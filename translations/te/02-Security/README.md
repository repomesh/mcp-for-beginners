# MCP భద్రత: AI సిస్టమ్ల కోసం సమగ్ర రక్షణ

[![MCP భద్రత ఉత్తమ పద్ధతులు](../../../translated_images/te/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(ఈ పాఠపు వీడియోను వీక్షించడానికి పై చిత్రం పై క్లిక్ చేయండి)_

భద్రత AI సిస్టమ్ డిజైన్ యొక్క ప్రాథమిక అంశంగా ఉంటుంది, కాబట్టి మేము దీన్ని మా రెండవ విభాగంగా ప్రాధాన్యం ఇస్తాము. ఇది మైక్రోసాఫ్ట్ యొక్క [సెక్యూర్ ఫ్యుట్యూర్ ఇనిషియేటివ్](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) నుండి **సెక్యూర్ బై డిజైన్** సిద్ధాంతంతో అనుకూలంగా ఉంటుంది.

మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్ (MCP) AI-వృద్ధి చెందిన అనువర్తనాలకు శక్తివంతమైన కొత్త సామర్థ్యాలను అందిస్తుంది, అయితే ఇది సంప్రదాయ సాఫ్ట్‌వేర్ ప్రమాదాల కంటే విస్తృతమైన ప్రత్యేక భద్రతా సవాళ్లను పరిచయం చేస్తుంది. MCP సిస్టమ్లు స్థాపిత భద్రతా సమస్యలు (సురక్షిత కోడింగ్, కనిష్ట ప్రివలిజ్, సప్లై చేయుని భద్రత) మరియు కొత్త AI-స్పెసిఫిక్ ముప్పులు, ఉదాహరణకు ప్రాంప్ట్ ఇంజెక్షన్, సాధనం విషపూరణ, సెషన్ హైలీజాకింగ్, కలచిపోయిన డెప్యూటీ దాడులు, టోకెన్ పాస్త్రూ లో పనితీరు లోపాలు మరియు డైనమిక్ సామర్థ్య సరిచూడే మార్పులు ఎదుర్కొంటాయి.

ఈ పాఠం MCP అమలులో అత్యంత తీవ్రమైన భద్రతా ప్రమాదాలను అన్వేషిస్తుంది - ధృవీకరణ, అనుమతి, అధిక పరిమితులు, పరోక్ష ప్రాంప్ట్ ఇంజెక్షన్, సెషన్ భద్రత, కలచిపోయిన డెప్యూటీ సమస్యలు, టోకెన్ నిర్వహణ మరియు సరఫరా గొలుసు లోపాలు. మీరు Microsoft పరిష్కారాలు, ఉదాహరణకు ప్రాంప్ట్ షీల్డ్స్, Azure కంటెంట్ సేఫ్టి, మరియు GitHub అడ్వాన్స్‌డ్ సెక్యూరిటీ ఉపయోగించి మీ MCP అమలు శక్తివంతం చేసే కార్యాచరణ నియంత్రణలు మరియు ఉత్తమ పద్ధతులను నేర్చుకుంటారు.

## అభ్యాస లక్ష్యాలు

ఈ పాఠం చివరికి, మీరు చేయగలుగుతారు:

- **MCP-స్పెసిఫిక్ ముప్పులను గుర్తించడం**: ప్రాంప్ట్ ఇంజెక్షన్, టూల్ విషపూరణ, అధిక అనుమతులు, సెషన్ హైలీజాకింగ్, కలచిపోయిన డెప్యూటీ సమస్యలు, టోకెన్ పాస్త్రూ లోపాలు మరియు సరఫరా గొలుసు ముప్పులు సహా MCP సిస్టమ్‌లలో ప్రత్యేక భద్రత సమస్యలను గుర్తించడం
- **భద్రత నియంత్రణలు వర్తించండి**: బలమైన ధృవీకరణ, కనిష్ట ప్ర Privilege ప్రవేశం, సురక్షిత టోకెన్ నిర్వహణ, సెషన్ భద్రత నియంత్రణలు, మరియు సరఫరా గొలుసు ధృవీకరణ వంటి సమర్థవంతమైన నివారణలను అమలు చేయండి
- **Microsoft భద్రతా పరిష్కారాలను ఉపయోగించండి**: MCP పని భారం రక్షణ కోసం Microsoft Prompt Shields, Azure Content Safety, మరియు GitHub Advanced Security ను అర్థం చేసుకుని అమలు చేయండి
- **టూల్ భద్రతను నిర్ధారించండి**: టూల్ మెటాడేటా ధృవీకరణ ముఖ్యత్వం, డైనమిక్ మార్పుల మానిటరింగ్ మరియు పరోక్ష ప్రాంప్ట్ ఇంజెక్షన్ దాడులకు రక్షణ
- **ఉత్తమ పద్ధతులను సమ్మిళితం చేయండి**: స్థాపిత భద్రతా మౌలికతలు (సురక్షిత కోడింగ్, సర్వర్ హార్డెనింగ్, జీరో ట్రస్ట్) తో MCP-స్పెసిఫిక్ నియంత్రణలను కలపడం ద్వారా సంపూర్ణ రక్షణ

# MCP భద్రత ఆర్కిటెక్చర్ & నియంత్రణలు

ఆధునిక MCP అమలు విధానాలు సంప్రదాయ సాఫ్ట్‌వేర్ భద్రత మరియు AI-సపెసిఫిక్ ముప్పులను పరిష్కరించే పలుకుబడి భద్రతా విధానాలను అవసరాన్ని కలిగి ఉంటాయి. తీవ్రమైన MCP ప్రత్యేకత తన భద్రతా నియంత్రణలను పెంపొందిస్తోంది, ప్రామాణిక కార్యాలయ భద్రత ఆర్కిటెక్చర్లతో మరియు స్థాపిత ఉత్తమ పద్ధతులతో మెరుగైన ఏకీకరణను సాధిస్తుంది.

[Microsoft Digital Defense Report](https://aka.ms/mddr) నుండి పరిశోధన **98% నివేదించిన ఉల్లంఘనలను బలమైన భద్రతా శుభ్రత ద్వారా నివారించవచ్చు** అని చూపిస్తుంది. అత్యంత సమర్థవంతమైన రక్షణ వ్యూహం ప్రాథమిక భద్రతా పద్ధతులతో MCP-స్పెసిఫిక్ నియంత్రణలను కలిపే విధానం - సాక్షాత్తు ప్రమాణ భద్రత చర్యలు మొత్తానికి భద్రతా ప్రమాదాన్ని తగ్గించడంలో ప్రధానముగా ఉంటాయి.

## ప్రస్తుత భద్రతా పరిస్థితి

> **గమనిక:** ఈ సమాచారం **ఫిబ్రవరి 5, 2026** నాటికి MCP భద్రతా ప్రమాణాలను ప్రతిబింబిస్తుంది, **MCP Specification 2025-11-25** కు అనుకూలంగా ఉంటుంది. MCP ప్రోటోకాల్ వేగంగా అభివృద్ధి చెందుతోంది, మరియు భవిష్యత్తు అమలు విధానాలు కొత్త ధృవీకరణ నమూనాలు మరియు అభివృద్ధి చెందిన నియంత్రణలను పరిచయం చేయవచ్చు. తాజా మార్గనిర్దేశకాలకు ప్రస్తుత [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), మరియు [భద్రత ఉత్తమ పద్ధతుల డాక్యుమెంటేషన్](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ను ఎప్పుడూ సందర్శించండి.

> **రాబోయే దిశ:** `2026-07-28` విడుదల అభ్యర్థి అధికృతిని మరింత బలపరిచింది — క్లయింట్లు అధికೃತಿ ప్రతిస్పందనలలో `iss` పరిమాణాన్ని ధృవీకరించాలి (RFC 9207), డైనమిక్ క్లయింట్ రిజిస్ట్రేషన్ సమయంలో OpenID Connect `application_type` ను ప్రకటించాలి, మరియు నమోదు చేసిన ఆధారాలు జారీ చేసిన అధికృతి సర్వర్‌కు బంధింపబడాలి. పూర్తిగా సూచన కోసం [MCPలో ఏమి మారింది: 2026-07-28 విడుదల అభ్యర్థి](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) చూడండి.

## 🏔️ MCP భద్రత శిఖర సమ్మేళన వర్క్‌షాప్ (షెర్పా)

**హ్యాండ్స్-ఆన్ భద్రతా శిక్షణ కోసం**, మేము **MCP భద్రత శిఖర సమ్మేళన వర్క్‌షాప్** (షెర్పా) ను గుండుగా సిఫారసు చేస్తాము - Microsoft Azureలో MCP సర్వర్లను భద్రపరచడం కోసం సమగ్ర మార్గదర్శక ప్రయాణం.

### వర్క్‌షాప్ అవలోకనం

[MCP భద్రత శిఖర సమ్మేళన వర్క్‌షాప్](https://azure-samples.github.io/sherpa/) నిరూపితమైన "భేద్యం → ఎక్స్‌ప్లాయిట్ → ఫిక్స్ → ధృవీకరణ" పద్ధతితో ప్రాక్టికల్, కార్యాచరణ భద్రత శిక్షణను అందిస్తుంది. మీరు:

- **పగులిస్తూ నేర్చుకోండి**: ఉద్దేశపూర్వకంగా అసురక్షిత సర్వర్లు ద్వారా లోపాలను అనుభవించండి
- **Azure-నేటివ్ భద్రత ఉపయోగించండి**: Azure Entra ID, కీ వల్లట్, API మేనేజ్మెంట్, మరియు AI కంటెంట్ సేఫ్టీని ఉపయోగించండి
- **డిఫెన్స్-ఇన్-డెత్ అనుసరించండి**: విస్తృత భద్రతా పొరలు నిర్మిస్తూ శిబిరాల ద్వారా ప్రగతి చేయండి
- **OWASP ప్రమాణాలు వర్తించండి**: ప్రతి సాంకేతికత [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) కి అనుగుణంగా ఉంటుంది
- **ఉత్పత్తి కోడ్ పొందండి**: పని చేసిన, పరీక్షించబడిన అమలు విధానాలతో వెళ్లిపోండి

| **MCP05** | కమాండ్ ఇంజెక్షన్ & అమలు | ఇన్పుట్ ధృవీకరణ, శాండ్బాక్సింగ్ |

### MCP ధృవీకరణ అభివృద్ది

MCP స్పెసిఫికేషన్ ధృవీకరణ మరియు అధికార విధానంలో గణనీయంగా అభివృద్ధి చెందింది:

- **మూల విధానం**: ప్రారంభ స్పెసిఫికేషన్లు అభివృద్ధి కర్తలు సొంత ధృవీకరణ సర్వర్లను అమలు చేయాలి అని చెప్పేవి, MCP సర్వర్లు OAuth 2.0 అధికృతి సర్వర్లుగా వ్యవహరిస్తూ వినియోగదారుల ధృవీకరణను నేరుగా నిర్వహించేవి
- **ప్రస్తుత ప్రమాణం (2025-11-25)**: నవీకరించిన స్పెసిఫికేషన్ MCP సర్వర్లకు బాహ్య గుర్తింపు ప్రదాతల (మైక్రోసాఫ్ట్ ఎన్‌ట్రా ID వంటి) ద్వారా ధృవీకరణను అప్పగించేందుకు అనుగుణంగా మారింది, భద్రత స్థితి మెరుగుపడినది మరియు అమలు సంక్లిష్టత తగ్గింది
- **ట్రాన్స్పోర్ట్ లేయర్ సెక్యూరిటీ**: స్థానిక (STDIO) మరియు రిమోట్ (స్ట్రీమబుల్ HTTP) కనెక్షన్లకు సరైన ధృవీకరణ నమూనాలతో భద్రతా ట్రాన్స్పోర్ట్ మెకానిజములకు మెరుగైన మద్దతు

## ధృవీకరణ & అధికృతి భద్రత

### ప్రస్తుత భద్రతా సవాళ్లు

ఆధునిక MCP అమలు విధానాలు అనేక ధృవీకరణ మరియు అధికృతి సవాళ్లను ఎదుర్కొంటాయి:

### ప్రమాదాలు మరియు ముప్పు దిశలు

- **తప్పుడు అధికృతి లాజిక్ అమలు**: MCP సర్వర్లలో లోపభూయిష్ఠ అధికృతి అమలు సున్నిత సమాచారాన్ని బయటపెట్టవచ్చు మరియు తప్పుగా యాక్సెస్ నియంత్రణలను వర్తింపచేస్తుంది
- **OAuth టోకెన్ చోరీ**: స్థానిక MCP సర్వర్ టోకెన్ దొంగతనంతో దాడికర్తలు సర్వర్లుగా నటించి దిగువ సేవలకు ప్రవేశించగలరు
- **టోకెన్ పాస్త్రూ లోపాలు**: సరైన టోకెన్ నిర్వహణ లేకపోవటం భద్రత నియంత్రణలకు మినహాయింపులు మరియు బాధ్యత లోపాలు సృష్టిస్తుంది
- **అధిక అనుమతులు**: అధిక ప్రివిలేజ్ MCP సర్వర్లు కనిష్ట ప్రివిలేజ్ సూత్రాలను ఉల్లంఘించి దాడి పరిమితులను పెంచుతాయి

#### టోకెన్ పాస్త్రూ: ఒక తీవ్రమైన వ్యతిరేక-న‌మూనా

**టోకెన్ పాస్త్రూ ప్రస్తుత MCP అధికృతి స్పెసిఫికేషన్‌లో స్పష్టంగా నిషేధించబడింది** తీవ్రమైన భద్రత వ్యతిరేక ప్రభావాల కారణంగా:

##### భద్రత నియంత్రణల గోడవొడుగు
- MCP సర్వర్లు మరియు దిగువ APIs కీలక భద్రత నియంత్రణలను (రేట్ పరిమితి, అభ్యర్ధన ధృవీకరణ, ట్రాఫిక్ మానిటరింగ్) అమలు చేస్తాయి అవి సరైన టోకెన్ ధృవీకరణపై ఆధారపడుతాయి
- నేరుగా కస్టమర్-తొ-API టోకెన్ వాడకము ఈ అవసరమైన రక్షణలను దాటిచెందుతుంది, భద్రతా నిర్మాణాన్ని దెబ్బతీస్తుంది

##### బాధ్యత మరియు ఆడిట్ సవాళ్లు  
- MCP సర్వర్లు ఆపర upstream-జారీ చేసిన టోకెన్లతో క్లయింట్లను గుర్తించలేవు, ఆడిట్ ట్రైలు తప్పిపోతోంది
- దిగువ వనరు సర్వర్ లాగులు అసలైన MCP సర్వర్ మధ్యవర్తుల స్థానానికి బదులుగా మోసపూరిత అభ్యర్ధన మూలాలను చూపుతాయి
- సంఘటనల గుర్తింపు మరియు అనుకూలత ఆడిట్‌లు ఎంతో కష్టమవుతాయి

##### డేటా ఎఫిల్ట్రేషన్ ప్రమాదాలు
- ధృవీకరించని టోకెన్ క్లెయిమ్‌లు దొంగతనమయిన టోకెన్లతో ఉన్న దుర్మార్గులు MCP సర్వర్లను డేటా ఎఫిల్ట్రేషన్ కోసం ప్రాక్సీలా ఉపయోగించగలరు
- నమ్మక మર્યాదా ఉల్లంఘన ఇది అనుమతించని యాక్సెస్ నమూనాలను మినహాయిస్తుంది భద్రత నియంత్రణలను దాటించి

##### బహుళ-సేవ దాడి మార్గాలు
- దెబ్బతిన్న టోకెన్లు అనేక సేవల ద్వారా అంగీకరించబడటం కలుపబడిన సిస్టమ్‌లలో పక్కపక్కగా కదలికను సులభతరం చేస్తుంది
- సర్వీసుల మధ్య నమ్మకం అనుమానాలు టోకెన్ మూలాలను నిర్ధారించలేని సమయంలో పాడవుతాయి

### భద్రత నియంత్రణలు & నివారణలు

**ముఖ్యమైన భద్రతా అవసరాలు:**

> **అత్యవసరం**: MCP సర్వర్లు **ఎలాంటి టోకెన్లను ఆమోదించకూడదు** అవి స్పష్టంగా MCP సర్వర్ కోసం జారీ చేయబడలేదు

#### ధృవీకరణ & అధికృతి నియంత్రణలు

- **బలమైన అధికృతి సమీక్ష**: MCP సర్వర్ అధికృతి లాజిక్ సంపూర్ణ ఆడిట్‌లను నిర్వహించి, ఉద్దేశించిన వినియోగదారులు మరియు క్లయింట్లు మాత్రమే సున్నిత వనరులకు ప్రవేశించగలిగేలా చూడండి
  - **అమలవిధి మార్గదర్శి**: [Azure API మేనేజ్మెంట్ MCP సర్వర్ల కోసం ధృవీకరణ గేట్వే](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **గుర్తింపు ఐక్యకరణ**: [MCP సర్వర్ ధృవీకరణ కోసం Microsoft Entra ID ఉపయోగించడం](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **సురక్షిత టోకెన్ నిర్వహణ**: [Microsoft టోకెన్ ధృవీకరణ మరియు జీవిత చక్ర ఉత్తమ పద్ధతులు](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) అమలు చేయండి
  - టోకెన్ ప్రేక్షకుల క్లెయిమ్‌లు MCP సర్వర్ గుర్తింపుతో సరిపోలుతాయని ధృవీకరించండి
  - సరైన టోకెన్ రొటేషన్ మరియు గడువు విధానాలను అమలు చేయండి
  - టోకెన్ పునఃప్రసారం దాడులు మరియు అనధికారిక వినియోగాన్ని నివారించండి

- **రక్షిత టోకెన్ నిల్వ**: విశ్రాంతి మరియు ప్రయాణంలో ఉండే టోకెన్‌లను గోప్యతతో భద్రపరచండి
  - **ఉత్తమ పద్ధతులు**: [సురక్షిత టోకెన్ నిల్వ మరియు గోప్యత మార్గదర్శకాలు](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### యాక్సెస్ కంట్రోల్ అమలు

- **కనిష్ట ప్రివిలేజ్ సూత్రం**: MCP సర్వర్‌లకు అవసరమైన వృత్తిపరమైన పనులకు కనిష్ట పరిమిత అనుమతులు ఇవ్వండి
  - అనుమతుల సమీక్షలు మరియు నవీకరణలు చేస్తూ అధిక అధికార పెరుగుదల నివారించండి
  - **Microsoft డాక్యుమెంటేషన్**: [సురక్షిత కనిష్ట-అనుమతులు యాక్సెస్](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **పాత్ర ఆధారిత యాక్సెస్ కంట్రోల్ (RBAC)**: సన్నిహిత పాత్ర నియామకాలు అమలు చేయండి
  - ప్రత్యేక వనరులు మరియు చర్యలకు పాత్రలను కఠినంగా పరిమితం చేయండి
  - దాడి పరిధిని విస్తరించే విస్తృత లేదా అవసరంలేని అనుమతులను నివారించండి

- **నిరంతర అనుమతి మానిటరింగ్**: కొనసాగుతున్న యాక్సెస్ ఆడిట్ మరియు పర్యవేక్షణను అమలు చేయండి
  - అనుమతి ఉపయోగ నమూనాలను అసాధారణతల కోసం పర్యవేక్షించండి
  - అధిక లేదా ఉపయోగించని అనుమతులను వెంటనే పరిష్కరించండి

## AI-స్పెసిఫిక్ భద్రతా ముప్పులు

### ప్రాంప్ట్ ఇంజెక్షన్ & సాధనం మానిప్యులేషన్ దాడులు

ఆధునిక MCP అమలులు సంప్రదాయ భద్రతా చర్యలు పూర్తిగా పరిష్కరించలేని సంక్లిష్ట AI-స్పెసిఫిక్ దాడి మార్గాలు ఎదుర్కొంటాయి:

#### **పరోక్ష ప్రాంప్ట్ ఇంజెక్షన్ (క్రాస్-డోమెన్ ప్రాంప్ట్ ఇంజెక్షన్)**

**పరోక్ష ప్రాంప్ట్ ఇంజెక్షన్** MCP-సక్రియ AI సిస్టమ్లలో అత్యంత తీవ్రమైన లోపాలలో ఒకటి. దాడి చేసే వ్యక్తులు విదేశీ కంటెంట్ — డాక్యుమెంట్లు, వెబ్ పేజీలు, ఈమెయిళ్లు లేదా డేటా మూలాలు — లో దుర్మార్గ ఆదేశాలను దాచి ఉంచుతారు, ఆపై AI సిస్టమ్లు వాటిని సరైన ఆదేశాలుగా ప్రాసెస్ చేస్తాయి.

**దాడి పరిస్థితులు:**
- **డాక్యుమెంట్-ఆధారిత ఇంజెక్షన్**: ప్రాసెస్ చేయబడిన డాక్యుమెంట్లలో దుర్మార్గ ఆదేశాలు, అవాంఛిత AI చర్యలు ప్రేరేపించేలా
- **వెబ్ కంటెంట్ వినియోగం**: దుర్మార్గ వెబ్ పేజీలు పొడవుగా ప్రాంప్ట్‌లను ಒಳపరుస్తాయి, AI ప్రవర్తనను గోప్యంగా మార్చడం కోసం స్క్రాప్ చేయబడినప్పుడు
- **ఈమెయిల్ ఆధారిత దాడులు**: ఈమెయిళ్లలో దుర్మార్గ ప్రాంప్ట్‌లు AI సహాయకులను సమాచారం లీక్ చేయడానికి లేదా అనధికారిక చర్యలు చేయడానికి కారణమవుతాయి
- **డేటా మూలాల కాలుష్యం**: దుర్మార్గమైన డేటాబేసులు లేదా APIs AI సిస్టమ్లకు కలుషితం అయిన కంటెంట్ ని అందిస్తాయి

**వాస్తవ ప్రపంచ ప్రభావం**: ఈ దాడులు డేటా ఎఫిల్ట్రేషన్, గోప్యత ఉల్లంఘన, హానికరమైన కంటెంట్ ఉత్పత్తి మరియు వాడుకరి సంభాషణల దుర్వినియోగానికి దారి తీస్తాయి. వివరమైన విశ్లేషణ కోసం, చూడండి [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).

#### **సాధనం విషపూరణ దాడులు**

**సాధనం విషపూరణ** MCP సాధనాన్ని నిర్వచించే మెటాడేటాను లక్ష్యంగా చేసుకుంటుంది, LLMs సాధన వివరణలు మరియు పరామితులను వివరించేటప్పుడు అమలు నిర్ణయాలకు ప్రభావం చూపే విధంగా.

**దాడి పద్ధతులు:**
- **మెటాడేటా మానిప్యులేషన్**: దాడికర్తలు సాధనం వివరణలు, పరామితి నిర్వచనలు లేదా వాడుక ఉదాహరణలలో దుర్మార్గ ఆదేశాలను చేర్పుతారు
- **గోప్య ఆదేశాలు**: సాధనం మెటాడేటాలో దాచిన ప్రాంప్ట్‌లు AI మోడళ్ల ద్వారా ప్రాసెస్ చేయబడతాయి కానీ మానవ వినియోగదారులకు కనిపించవు
- **డైనమిక్ సాధన సవరణ ("రగ్ పుల్స్")**: వినియోగదారులు ఆమోదించిన సాధనాలు తర్వాత వినియోగదారుడికి తెలియకుండానే దుర్మార్గ చర్యలకు మార్చబడతాయి
- **పరామితి ఇంజెక్షన్**: సాధన పరామితి స్కీమాలలో దుర్మార్గ కంటెంట్ చేరుస్తూ మోడల్ ప్రవర్తనను ప్రభావితం చేయడం


**హోస్ట్ చేయబడిన సర్వర్ ప్రమాదాలు**: రిమోట్ MCP సర్వర్లు ప్రారంభ వినియోగదారు ఆమోదం తర్వాత టూల్ నిర్వచనాలను నవీకరించగలగడం వలన పెరిగిన ప్రమాదాలను సూచిస్తాయి, ఇలాంటి సందర్భాలలో ముందుగా సురక్షితంగా భావించిన టూల్స్ దెబ్బతినేలా మారవచ్చు. సమగ్ర విశ్లేషణ కోసం, చూడండి [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks).

![Tool Injection Attack Diagram](../../../translated_images/te/tool-injection.3b0b4a6b24de6bef.webp)

#### **అదనపు AI దాడి మార్గాలు**

- **క్రాస్-డొమైన్ ప్రాంప్ట్ ఇంజెక్షన్ (XPIA)**: రక్షణ నియంత్రణలను దాటివేయడానికి బహుళ డొమైన్‌ల కంటెంట్‌ను ఉపయోగించే నిపుణుల దాడులు
- **డైనమిక్ సామర్థ్య మార్పిడి**: ప్రారంభ భద్రతా మూల్యాంకనాలను తప్పించడం కోసం సాధన సామర్థ్యాలలో రియల్-టైమ్ మార్పులు
- **కాంటెక్ట్స్ విండో విషపరిష్కరణ**: దెబ్బతిన్న సత్వర సూత్రాలను దాచేందుకు పెద్ద కాంటెక్ట్స్ విండోలను మార్చే దాడులు
- **మోడల్ కలత దాడులు**: మోడల్ పరిమితులను ఉపయోగించి అప్రిడిక్టబుల్ లేదా సురక్షితంగా లేని ప్రవర్తనలను సృష్టించడం


### AI భద్రతా ప్రమాద ప్రభావం

**పెద్ద జోక్యం కలిగిన ఫలితాలు:**
- **డేటా ఎక్స్‌ఫిల్ట్రేషన్**: అనధికార వినియోగం మరియు సున్నితమైన సంస్థ లేదా వ్యక్తిగత డేటా చోరీ
- **గోప్యత ఉల్లంఘనలు**: వ్యక్తిగత గుర్తింపు చేసుకోవచ్చిన సమాచార (PII) మరియు గోప్యమైన వ్యాపార డేటా ప్రదర్శన  
- **సిస్టమ్ మార్పులు**: ముఖ్యమైన సిస్టమ్‌లు మరియు వర్క్‌ఫ్లోలకు అనుకోకుండా మార్పులు
- **ప్రామాణీకరణ చోరీ**: గుర్తింపు టోకెన్లు మరియు సేవా గుర్తింపు చెలామణీ బాధ్యతలు
- **పక్క దిశలో కదలిక**: компрోమైజ్ అయిన AI సిస్టమ్‌లను విస్తృత నెట్‌వర్క్ దాడులకు ఉపయోగించడం

### Microsoft AI భద్రతా పరిష్కారాలు

#### **AI ప్రాంప్ట్ షీల్డ్స్: ఇంజెక్షన్ దాడులపై అధిక రక్షణ**

Microsoft **AI ప్రాంప్ట్ షీల్డ్స్** పలు భద్రతా స్థరాల ద్వారా ప్రత్యక్ష మరియు పరోక్ష ప్రాంప్ట్ ఇంజెక్షన్ దాడుల నుంచి సమగ్ర రక్షణను అందిస్తాయి:

##### **ప్రధాన రక్షణ తంత్రాలు:**

1. **అధునాతన గుర్తింపు & ఫిల్టరింగ్**
   - శత్రువుగా వ్యవహరించే సూచనలను బయట అనిన కంటెంట్‌లో గుర్తించడానికి మెషీన్ లెర్నింగ్ అల్గోరిథంలు మరియు NLP సాంకేతికతలు
   - డాక్యుమెంట్లు, వెబ్ పేజీలు, ఇమెయిల్‌లు మరియు డేటా మూలాల యొక్క లోపల వున్న ప్రమాదకర అంశాల సమయానుకూల విశ్లేషణ
   - సరైన మరియు దుష్ట ప్రాంప్ట్ నమూనాల పరిచయ నైపుణ్యం

2. **స్పాట్లైటింగ్ తంత్రాలు**  
   - విశ్వసనీయ వ్యవస్థ సూచనలు మరియు దెబ్బతిన్న బాహ్య ఇన్‌పుట్ల మధ్య భేదం చేయడం
   - మోడల్ సంబంధితతను మెరుగుపరుస్తూ దుష్ట కంటెంట్‌ను వేరు చేయడంలో పాఠ్య మార్పిడి పద్ధతులు
   - AI సిస్టంలను సక్రమ సూచనా క్రమాన్ని కలిగి ఉండేందుకు మరియు ఇంజెక్షన్ కమాండ్లను నిర్లక్ష్యం చేసేలా సహాయం చేయడం

3. **డీలిమిటర్ & డేటామార్కింగ్ వ్యవస్థలు**
   - విశ్వసనీయ సిస్టమ్ సందేశాలు మరియు బాహ్య ఇన్‌పుట్ టెక్స్ట్ మధ్య స్పష్టమైన సరిహద్దు నిర్వచనం 
   - విశ్వసనీయ మరియు అవిశ్వసనీయ డేటా మూలాల మధ్య సరిహద్దులను హైలైట్ చేయడానికి ప్రత్యేక గుర్తులు
   - సూచన కలత మరియు అనధికార ఆజ్ఞ పూర్తి నివారణకు స్పష్టమైన విభజన

4. **సతతమైన బెదిరింపు జ్ఞానం**
   - Microsoft కొత్త పెరుగుతున్న దాడి నమూనాలను పర్యవేక్షిస్తూ రక్షణలను నవీకరించడం
   - కొత్త ఇంజెక్షన్ సాంకేతికతలు మరియు దాడి మార్గాలపై క్రియాశీల బెదిరింపు వేటలు
   - అభివృద్ధి చెందుతున్న బెదిరింపులపై సమర్థతను నిలబెట్టుకునేందుకు ప regular రోగ భద్రతా మోడల్ నవీకరణలు

5. **Azure కంటెంట్ సేఫ్టీ సమగ్రత**
   - Azure AI కంటెంట్ సేఫ్టీ సూట్ యొక్క భాగం
   - జైల్‌బ్రేక్ ప్రయత్నాలు, హానికరమైన కంటెంట్ మరియు భద్రతా విధాన ఉల్లంఘనలపై అదనపు గుర్తింపు
   - AI అప్లికేషన్ భాగాల మీద ఏకరీకృత భద్రత నియంత్రణలు

## అభివృద్ధి చెందిన MCP భద్రతా బెదిరింపులు

### సెషన్ హైజాకింగ్ లోపాలు

**సెషన్ హైజాకింగ్** అనేది స్టేట్ఫుల్ MCP అమలాలలో ముఖ్యమైన దాడి మార్గం, ఇక్కడ అనధికార పార్టీలు సరైన సెషన్ గుర్తింపులను పొందాలని, అవినియోగం చేసి క్లయింట్లను అనుకరించి అనధికార చర్యలు చేపడతాయి.

#### **దాడి సన్నివేశాలు & ప్రమాదాలు**

- **సెషన్ హైజాక్ ప్రాంప్ట్ ఇంజెక్షన్**: దొంగిలించిన సెషన్ IDలతో దాడిదారులు సెషన్ స్థితిని పంచుకునే సర్వర్లలో దుష్ట సంఘటనలను ఇంజెక్ట్ చేస్తారు, ఇది హానికర చర్యలను విడుదల చేయవచ్చు లేదా సున్నితమైన డేటాకు యాక్సెస్ ఇస్తుంది
- **నేరుగా అనుకరణ**: దొంగిలించిన సెషన్ IDలు ప్రామాణీకరణను దాటిచేత MCP సర్వర్ కాల్స్ చేయడానికి వీలు కల్పించి దాడిదారులను సరైన వినియోగదారులుగా భావిస్తాయి
- **కంప్రమైజ్ అయ్యిన రిస్యూమబుల్ స్ట్రీమ్స్**: దాడిదారులు అభ్యర్థనలను ముందుగానే ముగిస్తారని, దేనివల్ల సరైన క్లయింట్లు ప్రమాదకర కంటెంట్‌తో తిరిగి ప్రారంభించవచ్చు

#### **సెషన్ నిర్వహణ కోసం భద్రత నియంత్రణలు**

**అత్యవసర అవసరాలు:**
- **అధికారపు ధృవీకరణ**: MCP సర్వర్లు అన్ని ఇటు వచ్చే అభ్యర్థనలను ధృవీకరించాలి మరియు ప్రామాణీకరణ కోసం సెషన్‌లపై ఆధారపడకూడదు
- **భద్రతా సెషన్ సృష్టి**: క్రిప్టోగ్రాఫిక్‌గా భద్రతగల, నిర్ధారించని సెషన్ IDలను భద్రతా రాండమ్ సంఖ్య జనరేటర్లతో రూపొందించాలి
- **వినియోగదారు-వ్యక్తిగత బైండింగ్**: సెషన్ IDలను వినియోగదారుకు సంబంధించిన సమాచారంతో `<user_id>:<session_id>` వంటి ఫార్మాట్లలో బైండ్ చేయాలి, తద్వారా వినియోగదారుల మద్య సెషన్ దుర్వినియోగం నివారించవచ్చు
- **సెషన్ జీవిత చక్ర నిర్వహణ**: కచ్చితమైన కాలపరిమితి, రొటేషన్, మరియు అమానతత్వం అమలు చేయడం
- **ట్రాన్స్పోర్ట్ సెక్యూరిటీ**: అన్ని కమ్యూనికేషన్లకు అనివార్యమైన HTTPS ఉపయోగించడం, సెషన్ ID పట్టుకోవడం నివారించడానికి

### కనఫ్యూజ్డ్ డెప్యూటీ సమస్య

**కనఫ్యూజ్డ్ డెప్యూటీ సమస్య** సంభవించే సందర్భంలో MCP సర్వర్లు క్లయింట్స్ మరియు మూడవ పక్ష సేవల మధ్య ప్రామాణీకరణ ప్రాక్సీలుగా పనిచేస్తాయి, ఇది స్థిర క్లయింట్ IDల దుర్వినియోగం ద్వారా అధికార దాటిచేతకు అవకాశం కల్పిస్తుంది.

#### **దాడి యంత్రాంగాలు & ప్రమాదాలు**

- **కుకీ ఆధారిత అనుమతి దాటిచేత**: పూర్వపు వినియోగదారు ప్రామాణీకరణ అనుమతి కుకీలు సృష్టిస్తుంది, దాడిదారులు సృష్టించిన రీడైరెక్ట్ URIలతో దుష్ట అధికార అభ్యర్థనల ద్వారా వాటిని దుర్వినియోగం చేస్తారు
- **అధికార కోడ్ చోరీ**: ఉన్న అనుమతి కుకీల వల్ల అధికారిక సర్వర్లు అనుమతి తెరలను దాటవేయవచ్చు, కోడులను దాడిదారుల నిర్వహణ గల ఎండ్రాయింట్‌లకు రాయవచ్చు  
- **అనధికార API యాక్సెస్**: దొంగిలించిన అధికార కోడులు అనుమతి లేకుండా టోకెన్ మార్పిడిని మరియు వినియోగదారు అనుకరణను సాధ్యం చేస్తాయి

#### **తగ్గింపు వ్యూహాలు**

**అనివార్య నియంత్రణలు:**
- **స్పష్ట అనుమతి అవసరాలు**: స్థిర క్లయింట్ IDలతో MCP ప్రాక్సీ సర్వర్లు ప్రతీ డైనమిక్ నమోదయిన క్లయింట్ కోసం వినియోగదారు అనుమతి పొందాలి
- **OAuth 2.1 భద్రత అమలు**: PKCE (ప్రూఫ్ కీ ఫర్ కోడ్ ఎక్స్‌చేంజ్) సహా ప్రస్తుత OAuth భద్రతా ఉత్తమ పద్ధతులను అనుసరించడం
- **కఠిన క్లయింట్ ధృవీకరణ**: రీడైరెక్ట్ URIలు మరియు క్లయింట్ గుర్తింపులను దుర్వినియోగం నివారణ కోసం గట్టిగా ధృవీకరించడం

### టోకెన్ పాస్‌త్రూ లోపాలు  

**టోకెన్ పాస్‌త్రూ** అనేది ఒక స్పష్టమైన ప్రతీకార పద్ధతి, దీని ద్వారా MCP సర్వర్లు సరైన ధృవీకరణ లేకుండా క్లయింట్ టోకెన్లను అంగీకరించి అవి దాదాపు APIsకు రవాణా చేస్తాయి, ఇది MCP అధికార నిర్దేశాలను ఉల్లంఘిస్తుంది.

#### **భద్రతా ప్రభావాలు**

- **నియంత్రణను దాటివేయడం**: నేరుగా క్లయింట్-కు-API టోకెన్ ఉపయోగించడం ప్రధాన రేటు పరిమితి, ధృవీకరణ మరియు పరిశీలన నియంత్రణలను దాటిస్తుంది
- **ఆడిట్ ట్రైల్ అవినియోగం**: వినియోగదారుని గుర్తించడం అసాధ్యం అవుతుంది, ఘటన పరిశోధన సామర్థ్యాలు దెబ్బతింటాయి
- **ప్రోక్సీ ఆధారిత డేటా ఎక్స్‌ఫిల్ట్రేషన్**: నిర్ధారించబడని టోకెన్లు దుష్ట వ్యక్తులకి అనధికార డేటా యాక్సెస్ కోసం సర్వర్లను ప్రోక్సీలు చేయడానికి ఉపకరిస్తాయి
- **ట్రస్ట్ బౌండరీ ఉల్లంఘనలు**: టోకెన్ మూలాలను ధృవీకరించలేని సమయంలో దిగువ సేవల ట్రస్ట్ అంచనాలు దెబ్బతింటాయి
- **బహు-సేవ దాడి వ్యాప్తి**: పలు సేవలపై అంగీకరించబడిన компрోమైజ్ టోకెన్లు పక్క దిశలో కదలికను అనుమతిస్తాయి

#### **అవసరమైన భద్రత నియంత్రణలు**

**అంగీకరించని అవసరాలు:**
- **టోకెన్ ధృవీకరణ**: MCP సర్వర్లు ప్రత్యేకంగా MCP సర్వర్ కోసం జారీ చేయని టోకెన్లను అంగీకరించకూడదు
- **ఆడియన్స్ ధృవీకరణ**: టోకెన్ ఆడియన్స్ క్లైమ్స్ MCP సర్వర్ గుర్తింపుతో సరిపోలేలా ఎల్లప్పుడూ ధృవీకరించాలి
- **సరైన టోకెన్ జీవిత చక్రం**: సురక్షిత మలుపులతో తక్కువ కాలం ఉన్న యాక్సెస్ టోకెన్లను అమలు చేయాలి


## AI సిస్టమ్‌ల సరఫరా గొలుసు భద్రత

సరఫరా గొలుసు భద్రత సాంప్రదాయ సాఫ్ట్‌వేర్ ఆధారపడిద్దల‌ను మించిపోయి మొత్తం AI వ్యవస్థాక్ఞానాన్ని కవరిస్తోంది. ఆధునిక MCP అమలులు అన్ని AI సంబంధిత భాగాలను గట్టి పరిశీలన మరియు పర్యవేక్షణ చేయాలి, ఎందుకంటే ప్రతి భాగం వ్యవస్థ సమగ్రతకు ప్రమాదకరమైన లోపాలు కలిగించవచ్చు.

### వీస్తరించిన AI సరఫరా గొలుసు భాగాలు

**సాంప్రదాయ సాఫ్ట్‌వేర్ ఆధారపాట్లు:**
- ఓపెన్-సోర్స్ లైబ్రరీలు మరియు ఫ్రేమ్‌వర్క్‌లు
- కంటైనర్ చిత్రాలు మరియు బేస్ సిస్టమ్‌లు  
- అభివృద్ధి సాధనాలు మరియు బిల్డ్ పైప్లైన్లు
- మౌలిక నిర్మాణ భాగాలు మరియు సేవలు

**AI-ఖచ్చితమైన సరఫరా గొలుసు అంశాలు:**
- **ఫౌండేషన్ మోడల్స్**: వివిధ ప్రొవైడర్లు అందించే ముందుగా శిక్షణ పొందిన మోడల్స్, ప్రాముఖ్యత ధృవీకరణ అవసరం
- **ఎంబెడ్డింగ్ సేవలు**: బాహ్య వెక్టరిజేషన్ మరియు సేమాంటిక్ శోధన సేవలు
- **ప్రాసంగిక ప్రొవైడర్లు**: డేటా మూలాలు, జ్ఞాన ఆధారాలు, మరియు డాక్యుమెంట్ నిల్వలు  
- **మూడవ పక్ష APIs**: బాహ్య AI సేవలు, ML పైప్లైన్లు, మరియు డేటా ప్రాసెసింగ్ ఎండ్రాయింట్లు
- **మోడల్ ఆర్టీఫాక్ట్లు**: బరువులు, కాన్ఫిగరేషన్లు, మరియు సరికొత్త మోడల్ వెర్షన్లు
- **శిక్షణ డేటా మూలాలు**: మోడల్ శిక్షణ మరియు సున్నితమైన డేటాసెట్లు

### సంపూర్ణ సరఫరా గొలుసు భద్రతా వ్యూహం

#### **భాగం ధృవీకరణ & నమ్మకం**
- **మూల ధృవీకరణ**: అన్ని AI భాగాల మూలం, లائسెన్సింగ్, మరియు సమగ్రతను ఒకీకృతం చేయక ముందే ధృవీకరించాలి
- **భద్రతా సమీక్ష**: మోడల్స్, డేటా మూలాలు, మరియు AI సేవలపై లోపాల స్కాన్లు మరియు భద్రతా సమీక్షలు నిర్వహించాలి
- **కీర్తి విశ్లేషణ**: AI సేవా ప్రొవైడర్ల భద్రతా ట్రాక్ రికార్డ్ మరియు విధానాలను అంచనా వేయాలి
- **అనుగుణత ధృవీకరణ**: అన్ని భాగాలు సంస్థ భద్రతా మరియు నియంత్రణ అవసరాలను తీర్చాలని నిర్ధారించాలి

#### **భద్రతా విడుదల పైప్లైన్‌లు**  
- **ఆటోమేటెడ్ CI/CD భద్రత**: ఆటోమేటెడ్ విడుదల పైప్లైన్‌లలో భద్రత స్కానింగ్ సరిపోల్చడం
- **ఆర్టీఫాక్ట్ సమగ్రత**: విజయవంతంగా విడుదల చేసిన ఆర్టీఫాక్ట్‌లు (కోడ్, మోడల్స్, కాన్ఫిగరేషన్లు) కోసం క్రిప్టోగ్రాఫిక్ ధృవీకరణను అమలు చేయాలి
- **దశాబ్దాలైన విడుదల**: ప్రతి దశలో భద్రత నిర్ధారణతో ప్రగతిశీల విడుదల వ్యూహాలను ఉపయోగించాలి
- **నమ్మకమైన ఆర్టీఫాక్ట్ నిల్వలు**: ధృవీకరించబడిన, భద్రతగల ఆర్టీఫాక్ట్ రిజిస్ట్రీలు మరియు నిల్వల నుండి మాత్రమే విడుదల చేయాలి

#### **సతత పర్యవేక్షణ & ప్రతిస్పందన**
- **ఆధారాల స్కానింగ్**: అన్ని సాఫ్ట్‌వేర్ మరియు AI భాగ ఆధారాలపై నిరంతర లోపాల పర్యవేక్షణ
- **మోడల్ పర్యవేక్షణ**: మోడల్ ప్రవర్తన, పనితీరు మార్పు మరియు భద్రత అసాధారణతలను నిరంతరం అంచనా వేయడం
- **సర్వీస్ ఆరోగ్య ట్రాకింగ్**: బాహ్య AI సేవలకు లభ్యత, భద్రత సంఘటనలు, మరియు విధాన మార్పులను పర్యవేక్షించడం
- **బెదిరింపు బుద్ధిమत्ता సమగ్రత**: AI మరియు ML భద్రతా ప్రమాదాలకు ప్రత్యేక బెదిరింపు ఫీడ్లను చేర్చడం

#### **ప్రవేశ నియంత్రణ & కనిష్టప్రవేశం**
- **భాగ స్థాయి అనుమతులు**: వ్యాపార అవసరాల ప్రకారం మోడల్స్, డేటా, మరియు సేవలపై యాక్సెస్ పరిమితం చేయడం
- **సర్వీస్ ఖాతా నిర్వహణ**: కనిష్ట సరైన అనుమతులతో ప్రత్యేక సర్వీస్ ఖాతాలను అమలు చేయడం
- **నెట్‌వర్క్ విభజన**: AI భాగాలను వేరుచేసి సేవల మధ్య నెట్‌వర్క్ యాక్సెస్‌ను పరిమితం చేయడం
- **API గేట్వే నియంత్రణలు**: బాహ్య AI సేవలకు యాక్సెస్‌ను నియంత్రించడానికి మరియు పర్యవేక్షించడానికి కేంద్రీకృత API గేట్వేలను ఉపయోగించడం

#### **సంఘటన ప్రతిస్పందన & పునరుద్ధరణ**
- **ద్రుత స్పందన విధానాలు**: компрోమైజ్ చేసిన AI భాగాలను ప్యాచ్ చేయడానికి లేదా భర్తీ చేయడానికి స్థాపిత ప్రక్రియలు
- **ప్రామాణిక తిరుగుళ్లు**: రహస్యాలు, API కీలు, మరియు సర్వీస్ క్రెడెంషియల్స్‌ను తేవడం కోసం ఆటోమేటెడ్ వ్యవస్థలు
- **రీస్టోర్ సామర్థ్యాలు**: AI భాగాలకు ముందు తెలిసిన-బాగున్న సంచికలకు త్వరగా తిరిగి రావలసిన సామర్థ్యం
- **సరఫరా గొలుసు ఉల్లంఘన పునరుద్ధరణ**: అప్‌స్ట్రీం AI సేవ компрోమైజ్‌లకు స్పందించడానికి ప్రత్యేక విధానాలు

## ప్రాథమిక భద్రత ఉత్తమ పద్ధతులు

MCP అమలులు మీ సంస్థ ఉన్న భద్రత స్థిరత్వాన్ని ירసుకోవడం మరియు మెరుగుపరచడం ద్వారా నిర్మించబడతాయి. ప్రాథమిక భద్రత పద్ధతులను బలోపేతం చేయడం AI సిస్టంల మరియు MCP అమలుల మొత్తం భద్రతను గణనీయంగా మెరుగుపరుస్తుంది.

### కోర్ భద్రతా మౌలిక సూత్రాలు

#### **భద్రతగల అభివృద్ధి పద్ధతులు**
- **OWASP అనుగుణత**: [OWASP టాప్ 10](https://owasp.org/www-project-top-ten/) వెబ్ అప్లికేషన్ లోపాల నుంచి రక్షణ
- **AI-ఖచ్చితమైన రక్షణలు**: [LLMs కోసం OWASP టాప్ 10](https://genai.owasp.org/download/43299/?tmstv=1731900559)పై నియంత్రణలు అమలు చేయడం
- **సురక్షిత రహస్య నిర్వహణ**: టోకెన్లు, API కీలు మరియు సున్నిత కాన్ఫిగరేషన్ డేటాకు ప్రత్యేక తోటాల ఉపయోగం
- **ఎండ్-టు-ఎండ్ గోప్యతా సంకేతీకరణ**: అన్ని అప్లికేషన్ భాగాలు మరియు డేటా ప్రవాహాలలో సురక్షిత సంబంధాలు అమలు చేయడం
- **ఇన్‌పుట్ ధృవీకరణ**: అన్ని వినియోగదారు ఇన్‌పుట్లు, API పరామితులు మరియు డేటా మూలాల దృవీకరణ కఠినంగా చేయడం

#### **అవస్థపరమైన దృఢీకరణ**
- **బహుళ-ఫ్యాక్టర్ గుర్తింపు**: అన్ని పరిపాలనా మరియు సర్వీస్ ఖాతాలకు తప్పనిసరి MFA
- **ప్యాచ్ నిర్వహణ**: ఆపరేటింగ్ సిస్టమ్‌లు, ఫ్రేమ్‌వర్క్‌లు మరియు ఆధారాల కోసం ఆటోమేటెడ్, సమయోచిత ప్యాచింగ్  
- **గుర్తింపు దాత సమగ్రత**: సంస్థ గుర్తింపు దాతల ద్వారా కేంద్రీకృత గుర్తింపు నిర్వహణ (Microsoft Entra ID, Active Directory)
- **నెట్‌వర్క్ విభజన**: MCP భాగాల లాజికల్ వేరుచేసుకోవడం మరియు పక్క దిశ కదలిక అవకాశం పరిమితం చేయడం
- **కనిష్ట అనుమతి సూత్రం**: అన్ని సిస్టమ్ భాగాలు మరియు ఖాతాల కోసం కనిష్ట అవసర అనుమతులను పాటించడం

#### **భద్రత పర్యవేక్షణ & గుర్తింపు**
- **సంపూర్ణ లాగింగ్**: MCP క్లయింట్-సర్వర్ చర్యలు సహా AI అప్లికేషన్ కార్యకలాపాల పూర్తి లాగింగ్
- **SIEM సమగ్రత**: అసాధారణ గుర్తింపుకు కేంద్రీకృత భద్రతా సమాచార మరియు సంఘటన నిర్వహణ
- **ప్రవర్తన విశ్లేషణ**: సిస్టమ్ మరియు వినియోగదారుల ప్రవర్తనలో అసాధారణ నమూనాల గుర్తింపుకు AI-చాలిత పర్యవేక్షణ
- **బెదిరింపు జ్ఞానం**: బాహ్య బెదిరింపు ఫీడ్లు మరియు సంకేతాల సమగ్రత  
- **సంఘటన ప్రతిస్పందన**: భద్రత సంఘటన గుర్తింపు, స్పందన మరియు పునరుద్ధరణకి నిర్వచిత ప్రక్రియలు

#### **జీరో ట్రస్ట్ నిర్మాణం**
- **ఎప్పుడూ నమ్మకపోవద్దు, ఎప్పుడూ ధృవీకరించు**: వినియోగదారులు, పరికరాలు మరియు నెట్‌వర్క్ కనెక్షన్ల యొక్క నిరంతర ధృవీకరణ
- **మైక్రో-విభజన**: ప్రతి వర్క్‌లోడ్ మరియు సేవలకు ప్రత్యేక నెట్‌వర్క్ నియంత్రణలు
- **గుర్తింపు-కేంద్ర భద్రత**: నెట్‌వర్క్ స్థానం ఆదారమైన కాకుండా ధృవీకరించబడిన గుర్తింపుల ఆధారంగా భద్రతా విధానాలు
- **సతత ప్రమాద మూల్యాంకనం**: ప్రస్తుత పరిస్తితి మరియు ప్రవర్తన ఆధారంగా డైనమిక్ భద్రత స్థితి మూల్యాంకనం
- **నిబంధన ప్రాప్యత**: ప్రమాద కారణాలు, స్థానం మరియు పరికర నమ్మకంపై ఆధారపడి యాక్సెస్ నియంత్రణలు

### సంస్థా సమగ్రత నమూనాలు

- **డేటా వర్గీకరణ**: AI వ్యవస్థలు ప్రాసెస్ చేసే సున్నితమైన డేటాను సరైన రీతిలో వర్గీకరించడం మరియు నిర్వహించడం
- **ఆడిట్ ట్రైల్స్**: నియంత్రణ అనుగుణత మరియు ఫోరెన్సిక్ విచారణ కోసం సమగ్రమైన లాగింగ్
- **గోప్యత నియంత్రణలు**: AI వ్యవస్థ నిర్మాణంలో ప్రైవసీ-బై-డిజైన్ సూత్రాల అమలు
- **మార్పు నిర్వహణ**: AI వ్యవస్థ మార్పుల భద్రత సమీక్షల కోసం అధికారపూర్వక ప్రక్రియలు

ఈ మౌలిక ఆచరణలు MCP-స్పెసిఫిక్ భద్రత నియంత్రణల ప్రభావాన్ని మెరుగుపరచుతూ AI-చालित అనువర్తనాలకు సమగ్రమైన రక్షణను అందించే దృఢమైన భద్రతా దోరణిని సృష్టిస్తాయి.

## ప్రాముఖ్యమైన భద్రతా అంశాలు

- **స్థరబద్ధమైన భద్రతా దృష్టికోణం**: మౌలిక భద్రతా ఆచరణలు (భద్రతతో కూడిన కోడింగ్, కనిష్ఠ привileged రహిత 접근, సరఫరా గొలుసు ధృవీకరణ, నిరంతర పర్యవేక్షణ) మరియు AI-ప్రత్యేక నియంత్రణలను కలిపి సమగ్రమైన రక్షణ

- **AI-ప్రత్యేక ప్రమాద చిత్రాలు**: MCP వ్యవస్థలు ప్రత్యేక ప్రమాదాలను ఎదుర్కొంటాయి, వీటిలో ప్రాంప్ట్ ఇంజెక్షన్, టూల్ విషప్రయోగం, సెషన్ హీజాకింగ్, కలగట్ల deputy సమస్యలు, టోకెన్ పాస్త్రూ దెబ్బతినే పాయింట్లు మరియు అధిక అనుమతులు ఉన్నాయి, వీటికి ప్రత్యేక నివారణ అవసరం

- **ప్రామాణీకరణ & అనుమతుల లోతైన అమలు**: Microsoft Entra ID వంటి బాహ్య ఐడెంటిటీ ప్రొవైడర్‌లను ఉపయోగించి బలమైన ప్రామాణీకరణను అమలు చేయండి, సరైన టోకెన్ ధృవీకరణను అమలు చేయండి, మరియు మీ MCP సర్వర్ కోసం స్పష్టంగా జారీ చేయని టోకన్లు ఎప్పుడూ అంగీకరించవద్దు

- **AI దాడులు నిరోధకత**: Microsoft Prompt Shields మరియు Azure Content Safety ను పరోక్ష ప్రాంప్ట్ ఇంజెక్షన్ మరియు టూల్ విషప్రయోగం దాడుల నుండి రక్షణకు ఉపయా గించండి, టూల్ మెటాడేటాను ధృవీకరించండి మరియు డైనమిక్ మార్పుల కోసం పర్యవేక్షించండి

- **సెషన్ & రవాణా భద్రత**: వాడుకరి గుర్తింపులకు బంధింపబడి ఉన్న క్రిప్టోగ్రాఫిక్‌గా సురక్షిత, నిర్దిష్టతలేని సెషన్ ID లను ఉపయోగించండి, సరైన సెషన్ లైఫ్ సైకిల్ నిర్వహణను అమలు చేయండి, మరియు ప్రామాణీకరణ కోసం సెషన్లు ఉపయోగించవద్దు

- **OAuth భద్రతా ఉత్తమ పద్ధతులు**: డైనమిక్‌గా నమోదు చేసిన క్లయింట్ల కోసం స్పష్టమైన వాడుకరి అనుమతితో confuse deputy దాడులను నివారించండి, PKCEతో OAuth 2.1 సరైన అమలును చేయండి మరియు కఠినమైన రీడైరెక్ట్ URI ధృవీకరణను పాటించండి

- **టోకెన్ భద్రతా పద్ధతులు**: టోకెన్ పాస్త్రూ యాంటీ-ప్యాటర్న్‌లను నివారించండి, టోకెన్ ప్రేక్షకుల క్లెయిమ్‌లను ధృవీకరించండి, సురక్షిత రొటేషన్‌తో తక్కువ కాలం కోసం ఉన్న టోకన్లను అమలు చేయండి, మరియు స్పష్టమైన విశ్వాస సరిహద్దులను నిర్వహించండి

- **సమగ్ర సరఫరా గొలుసు భద్రత**: అన్ని AI ఎకోసిస్టం భాగాలను (మోడల్స్, ఎంబెడింగ్స్, కాంటెక్స్ట్ ప్రొవైడర్లు, బాహ్య APIలు) సంప్రదాయ సాఫ్ట్‌వేర్ ఆధారిత సున్నితమైన విధానాలలోనాటి భద్రతా ప్రమాణాలతో ఆచరించండి

- **నిరంతర అభివృద్ధి**: వేగంగా అభివృద్ధిచెందుతున్న MCP స్పెసిఫికేషన్లను అనుసరించండి, భద్రతా సంఘ సమూహ ప్రమాణాలకి సహకరించండి, మరియు ప్రోటోకాల్ అభివృద్ధికి అనుగుణంగా అభివృద్ధిచేసుకునే భద్రతా దృక్కోణాలను పరిరక్షించండి

- **Microsoft భద్రతా సమిష్టి**: Microsoft యొక్క సమగ్ర భద్రతా పరిసరాలు (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) ను ఉపయోగించి MCP అమరికలకు మరింత రక్షణ సాధించండి
