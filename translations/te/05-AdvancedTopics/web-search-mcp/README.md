# పాఠం: వెబ్ సర్చ్ MCP సర్వర్ నిర్మాణం

ఈ అధ్యాయం బాహ్య API లతో అనుసంధానం చేస్తూ, విభిన్న డేటా రకాల నిర్వహణ, లోపాలను పరిష్కరించడం మరియు అనేక సాధనలను సమన్వయపరచడం వంటి సవాళ్లను ఎదుర్కొనే నిజ జీవిత AI ఏజెంట్‌ను నిర్మించడం ఎలా అనే దాన్ని చూపిస్తుంది—అన్ని ప్రొడక్షన్ సిద్ధమైన రూపంలో. మీరు చూడబోతున్నవి:

- **ప్రామాణీకరణ అవసరం ఉన్న బాహ్య API లతో అనుసంధానం**
- **ఎక్కువ ఎండ్‌పాయింట్ల నుండి వివిధ డేటా రకాలను నిర్వహించడం**
- **కఠినమైన లోపాల నిర్వహణ మరియు లాగ్ విధానాలు**
- **ఒకే సర్వర్‌లో బహుళ సాధనాల సమన్వయం**

చివరికి, మీరు అభివృద్ధి చెందిన AI మరియు LLM ఆధారిత అనువర్తనాల కోసం అవసరమైన మోడళ్ళు మరియు ఉత్తమ ఆచరణలపై ప్రాక్టికల్ అనుభవం పొందుతారు.

## పరిచయం

ఈ పాఠంలో, మీరు SerpAPI ఉపయోగించి నిజకాల వెబ్ డేటాతో LLM సామర్థ్యాలను విస్తరించే ఆధునిక MCP సర్వర్ మరియు క్లయింట్‌ను నిర్మించడం నేర్చుకుంటారు. ఇది అత్యాధునిక AI ఏజెంట్లను అభివృద్ధి చేసేందుకు ముఖ్యమైన నైపుణ్యం, వాటికి నవీకరించబడిన వెబ్ సమాచారం అందుబాటులో ఉంటుంది.

## నేర్చుకునే లక్ష్యాలు

ఈ పాఠం చివరికి, మీరు చేయగలుగుతారు:

- SerpAPI వంటి బాహ్య API లను సురక్షితంగా MCP సర్వర్‌లో అనుసంధానించడం
- వెబ్, వార్తలు, ఉత్పత్తుల శోధన మరియు ప్రశ్నల సమాధానాల కోసం బహుళ సాధనాలను అమలు చేయడం
- LLM వినియోగానికి నిర్మిత డేటాను విశ్లేషించి నిర్మాణాత్మకంగా చేయడం
- లోపాలను నిర్వహించడం మరియు API రేట్ పరిమితులను సక్రమంగా నిర్వహించడం
- ఆటోమేటెడ్ మరియు ఇంటరాక్టివ్ MCP క్లయింట్లను రూపొందించి పరీక్షించటం

## వెబ్ సర్చ్ MCP సర్వర్

ఈ విభాగం వెబ్ సర్చ్ MCP సర్వర్ యొక్క వాస్తవ్య నిర్మాణం మరియు లక్షణాలను పరిచయం చేస్తుంది. మీరు FastMCP మరియు SerpAPI కలసి LLM సామర్థ్యాలను నిజకాల వెబ్ డేటాతో ఎలా విస్తరించతాయో చూడగలుగుతారు.

### అవలೋಕనం

ఈ అమలు నాలుగు సాధనాలను కలిగి ఉంటుంది, MCP యొక్క సురక్షితంగా మరియు సమర్థవంతంగా వివిధ బాహ్య API ఆధారిత పనులను నిర్వహించే సామర్ధ్యాన్ని ప్రదర్శిస్తుంది:

- **general_search**: విస్తృత వెబ్ ఫలితాల కోసం
- **news_search**: తాజా శీర్షికల కోసం
- **product_search**: ఈ-కామర్స్ డేటాకు
- **qna**: ప్రశ్న-సమాధానం కొరకు

### లక్షణాలు
- **కోడ్ ఉదాహరణలు**: ప్రత్యేక భాషల కోడ్ బ్లాకులను Python కోసం (మరియు ఇతర భాషలకు సులభంగా పొడిగించదగిన) స్పష్టత కోసం కోడ్ పివార్ట్‌లతో కలిపి ఇక్కడ కలిగి ఉన్నాయి

###Python

```python
# general_search టూల్ యొక్క ఉదాహరణ వాడకం
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

క్లయింట్ నడిపించే ముందు, సర్వర్ ఏమి చేస్తుందో అర్థం చేసుకోవడం సహాయకరం. [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ఫైల్ MCP సర్వర్‌ను అమలుచేస్తూ, SerpAPIతో అనుసంధానం చేసి వెబ్, వార్తలు, ఉత్పత్తుల శోధన మరియు ప్రశ్నలు సమాధాన సాధనాలను అందిస్తుంది. ఇది అభ్యర్థనలు స్వీకరించి, API కాల్‌లను నిర్వహించి, ప్రతిస్పందనలను విశ్లేషించి, నిర్మిత ఫలితాలను క్లయింట్‌కు ఇస్తుంది.

మీరు పూర్తి అమలును [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py)లో సమీక్షించవచ్చు.

ఇక్కడ సర్వర్ ఎలా సాధనం నిర్వచించి నమోదు చేస్తుందో ఒక చిన్న ఉదాహరణ ఉంది:

### Python సర్వర్

```python
# server.py (అంశం)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...అమలు...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **బాహ్య API అనుసంధానం**: API కీలు మరియు బాహ్య అభ్యర్థనలను సురక్షితంగా నిర్వహించడం చూపిస్తుంది
- **నిర్మాత్మక డేటా విశ్లేషణ**: API ప్రతిస్పందనలను LLM అనుకూల ఫార్మట్లుగా మార్చటం చూపిస్తుంది
- **లోపాల నిర్వహణ**: సరైన లాగ్ తో మજબుత్ లోపాల నిర్వహణ
- **ఇంటరాక్టివ్ క్లయింట్**: ఆటోమేటెడ్ పరీక్షలతో పాటు పరీక్షించుకునే ఇంటరాక్టివ్ మోడ్‌ను కలిగి ఉంటుంది
- **సందర్భ నిర్వహణ**: అభ్యర్థనలకు లాగింగ్ మరియు ట్రాకింగ్ కొరకు MCP Context వినియోగం

## ముందస్తు అవసరాలు

మీరు ప్రారంభించే ముందు, మీ పరిసరాన్ని సక్రమంగా సెట్ చేయడానికి ఈ దశలను అనుసరించండి. ఇది అన్ని ఆధారాలు ఇన్‌స్టాల్ అయి, మీ API కీలు సరి గా కాన్ఫిగర్ అయి ఉండటం సునిశ్చితం చేస్తుంది, తద్వారా అభివృద్ధి మరియు పరీక్షలు విఘ్నం లేకుండా జరుగుతాయి.

- Python 3.8 లేదా అంతకంటే పైగా
- SerpAPI API కీ (Sign up at [SerpAPI](https://serpapi.com/) - ఉచిత టియర్ అందుబాటులో ఉంది)

## ఇన్‌స్టాలేషన్

ప్రారంభించేందుకు, మీ పరిసరాన్ని సెట్ చేసుకోడానికి ఈ దశలను అనుసరించండి:

1. అనుభంధాలను uv (సిఫార్సు చేయబడుతుంది) లేదా pip ద్వారా ఇన్‌స్టాల్ చేయండి:

```bash
# uv ఉపయోగించడం (సిఫార్సు చేయబడింది)
uv pip install -r requirements.txt

# పిప్ ఉపయోగించడం
pip install -r requirements.txt
```

2. ప్రాజెక్ట్ రూట్‌లో `.env` ఫైల్ సృష్టించి, మీ SerpAPI కీని అందించండి:

```
SERPAPI_KEY=your_serpapi_key_here
```

## వినియోగం

వెబ్ సర్చ్ MCP సర్వర్ ప్రధాన భాగం, ఇది SerpAPIతో అనుసంధానం చేసి వెబ్, వార్తలు, ఉత్పత్తుల శోధన మరియు ప్రశ్నల సాధనాలను అందిస్తుంది. ఇది అభ్యర్థనలను నిర్వహించి, API కాల్‌లను నిర్వహించి, ప్రతిస్పందనలను విశ్లేషించి, నిర్మిత ఫలితాలను క్లయింట్‌కు ఇస్తుంది.

మీరు పూర్తి అమలును [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py)లో సమీక్షించవచ్చు.

### సర్వర్ నడపడం

MCP సర్వర్ ప్రారంభించేందుకు ఈ కమాండ్‌ను ఉపయోగించండి:

```bash
python server.py
```

సర్వర్ stdio ఆధారిత MCP సర్వర్‌గా నడుస్తుంది, క్లయింట్ దానికి ప్రత్యక్షంగా కలవచ్చు.

### క్లయింట్ మోడ్స్

MCP సర్వర్‌తో పరస్పరం సాధించేందుకు క్లయింట్ (`client.py`) రెండు మోడ్స్‌ను మద్దతు ఇస్తుంది:

- **సాధారణ మోడ్**: అన్ని సాధనాలను పరీక్షించే ఆటోమేటెడ్ టెస్ట్లు నడుపుతుంది. సర్వర్ మరియు సాధనాలు సక్రమంగా పని చేస్తున్నాయో త్వరగా పరీక్షించుకోవడానికి ఇది ఉపయోగకరం.
- **ఇంటరాక్టివ్ మోడ్**: మెనూ ఆధారిత ఇంటర్‌ఫేస్ ప్రారంభించి, మానవీయంగా సాధనాలను ఎంపిక చేసి పిలవచ్చు, కస్టమ్ ప్రశ్నలు అందించి ఫలితాలను నిజకాలంలో చూడవచ్చు. సర్వర్ సామర్థ్యాలను అన్వేషించడానికి మరియు వివిధ ఇన్పుట్‌లతో ప్రయోగాలు చేయడానికి ఇది అద్భుతమైనది.

మీరు పూర్తి అమలును [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py)లో సమీక్షించవచ్చు.

### క్లయింట్ నడపడం

ఆటోమేటెడ్ టెస్ట్లను నడిపేందుకు (ఇది ఆటోమేటెడ్ గా సర్వర్‌ను కూడా ప్రారంభిస్తుంది):

```bash
python client.py
```

లేదా ఇంటరాక్టివ్ మోడ్‌లో నడపండి:

```bash

```

### విభిన్న పద్ధతులతో పరీక్షించడం

మీరు అవసరాన్ని మరియు వర్క్‌ఫ్లో ప్రకారం సర్వర్ అందించే సాధనాలతో పరీక్షించడం మరియు పరస్పరం సాధించడం కోసం అనేక విధానాలు ఉన్నాయి.

#### MCP Python SDKతో మీ స్వంత టెస్ట్ స్క్రిప్ట్‌లు వ్రాయడం

```python

            result = await session.call_tool("general_search", 
                                           arguments={"query": "your custom query"})
            # ఫలితాన్ని ప్రాసెస్ చేయండి
```

ఈ సందర్భంలో, "టెస్ట్ స్క్రిప్ట్" అంటే మీరు MCP సర్వర్ కోసం క్లయింట్‌గా వ్యవహరించే అనుకూల Python ప్రోగ్రామ్. ఇది బలవంతమైన యూనిట్ టెస్ట్ కాక, సర్వర్‌కు ప్రోగ్రామాటిక్గా కనెక్ట్ అవ్వడంతో పాటు, ఏ సాధనాన్ని అయినా మీరు ఎంచుకున్న పారామితులతో పిలవటం మరియు ఫలితాలను చూస్తూ ఉండటం. ఈ విధానం ఈ క్రింది పనులకు ఉపయోగకరం:
- సాధన పిలుపులపై ప్రోటోటైపింగ్ మరియు ప్రయోగం
- సర్వర్ వివిధ ఇన్పుట్‌లకు ఎలా ప్రతిస్పందిస్తుందో ధృవీకరించడం
- పునరావృత సాధన పిలుపులను ఆటోమేటెడ్ చేయడం
- MCP సర్వర్ పై మీ స్వంత వర్క్‌ఫ్లో లేదా అనుసంధానాలను నిర్మించడం

మీరు కొత్త ప్రశ్నలను త్వరగా ప్రయత్నించడానికి, సాధన ప్రవర్తనను డిబగ్ చేయడానికి లేదా మరింత అభివృద్ధి ఆటోమేషన్‌ కోసం ప్రారంభ బిందువుగా ఈ టెస్ట్ స్క్రిప్ట్‌లను ఉపయోగించవచ్చు. ఈ క్రింది ఉదాహరణ MCP Python SDK ఉపయోగించి అందువలన ఒక స్క్రిప్ట్ ఎలా చేయాలో చూపిస్తుంది:

## సాధన వివరణలు

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_general_search():

        command="python",

            print(result)
```

## ఈ పాఠంలో ఉన్న అగ్రగణ్య సిద్ధాంతాలు

మీరు నిర్మాణం మొదలుపెట్టేముందు, ఈ అధ్యాయం మొత్తం కనిపించే కొన్ని ముఖ్యమైన అత్యాధునిక సిద్ధాంతాలు ఇక్కడ ఉన్నాయి. ఇవి అర్థం చేసుకోవడం, మీరు కొత్తవారు అయినా, మీకు సహాయపడుతుంది:

- **బహుళ సాధన సమన్వయం**: ఇది ఒక MCP సర్వర్‌లో అనేక సాధనాలు (వెబ్ శోధన, వార్తా శోధన, ఉత్పత్తుల శోధన మరియు Q&A వంటివి) నడపడం అర్ధం. ఇది మీ సర్వర్‌కి ఒకటి కాకుండా వివిధ పనులను నిర్వహించేందుకు వీలు కలిగిస్తుంది.
- **API రేట్ పరిమితి నిర్వహణ**: SerpAPI వంటి అనేక బాహ్య APIలు మీరు ఒక నిర్దిష్ట సమయం లో ఎంత అభ్యర్థనలు చేస్తారో పరిమితం చేస్తాయి. మంచి కోడ్ ఈ పరిమితులను పరిశీలించి, అప్లికేషన్ విరమించకుండా అందులో ఆ పరిమితుల్ని సక్రమంగా నిర్వహిస్తుంది.
- **నిర్మాణాత్మక డేటా విశ్లేషణ**: API ప్రతిస్పందనలు చాలా సార్లు సంక్లిష్టంగా ఉండి పైన్హి జరగతాయి. ఈ సిద్ధాంతం వాటిని శుభ్రంగా, ఉపయోగించడానికి సులభమైన ఫార్మట్లుగా మార్చడం గురించి, LLM లకు లేదా ఇతర ప్రోగ్రాములకు స్నేహపూర్వకంగా.
- **లోపాల పునర్యుద్ధరణ**: అప్పుడప్పుడు సమస్యలు వస్తాయి—జాలి నెట్‌వర్క్ సమస్యలు లేదా API మీ ఆశించినంత మంచి సమాచారం ఇవ్వకపోవడం వంటివి. లోపాల పునర్యుద్ధరణ అంటే మీ కోడ్ ఈ సమస్యలను నిర్వహించి, ఒక విధమైన ఉపయోగకరమైన ప్రతిస్పందన ఇవ్వగలగడం, ఎరుగకుండా బగ్ అవ్వకుండా ఉండటం.
- **పారామితి ధ్రువీకరణ**: ఇది మీ సాధనాలకు పంపే అన్ని ఇన్పుట్లను సరిగ్గా మరియు భద్రమైన వివరంగా తనిఖీ చేయడం గురించి. ఇది డిఫాల్ట్ విలువలను సెట్ చేయడం, టైపు సరిపోతున్నాయని చూడటం వంటివి కలిగి ఉంటుంది, తద్వారా బగ్స్ మరియు తప్పుదోవల నుంచి రక్షణ కలుగుతుంది.

ఈ విభాగం మీరు వెబ్ సర్చ్ MCP సర్వర్‌తో పనిచేస్తున్నప్పుడు ఎదుర్కొనే సాధారణ సమస్యలను గుర్తించి పరిష్కరించడానికి సహాయపడుతుంది. మీరు లోపాలు లేదా అనుకోని ప్రవర్తన చూస్తే, ఈ ట్రబుల్‌షూటింగ్ భాగం అత్యవసర సమస్యలకు పరిష్కారాలు అందిస్తుంది. మరింత సహాయం కోసం వెళ్ళేముందు ఈ సూచనాలు చూడండి—ఇవి తరచుగా సమస్యలను త్వరగా పరిష్కరిస్తాయి.

## ట్రబుల్‌షూటింగ్

వెబ్ సర్చ్ MCP సర్వర్‌తో పని చేస్తుంటే, మీరు అనుకోకుండా కొన్ని సమస్యలు ఎదుర్కొనవచ్చు—ఇది బాహ్య API లతో మరియు కొత్త సాధనాలతో అభివృద్ధి సమయంలో సాధారణం. ఈ విభాగం అత్యంత సాధారణ సమస్యలకు ప్రాయోగిక పరిష్కారాలను ఇస్తుంది, తద్వారా మీరు త్వరగా సాధారణ దిశగా తిరిగి వచ్చేయగలరు. లోపం వచ్చినప్పుడు ఇక్కడి సూచనలు మొదలుపెట్టండి: క్రిందున్న చిట్కాలు ఎక్కువ మంది ఉపయోగించే సమస్యలకు సరిపోయి, అదనపు సహాయం లేకుండానే మీ సమస్యను తరచుగా పరిష్కరిస్తాయి.

### సాధారణ సమస్యలు

క్రింద ఉపయోగకర్తలు ఎక్కువగా ఎదుర్కొనే ముఖ్యమైన సమస్యలు మరియు వాటి క్లియర్ వివరణలు, పరిష్కార దశలను ఇచ్చారు:

1. **.env ఫైలులో SERPAPI_KEY లేని పరిస్థితి**
   - మీరు `SERPAPI_KEY environment variable not found` అనే లోపం చూడగలుగుతారు అంటే మీ అప్లికేషన్ SerpAPIకి అవసరమైన API కీ కనుగొనలేకపోతుంది అని అర్థం. దీన్ని సరిచేయడానికి, ప్రాజెక్ట్ యొక్క రూట్ ఫోల్డర్‌లో `.env` అనే ఫైల్ (ఇప్పటికీ 없다 అనుకుంటే) సృష్టించి `SERPAPI_KEY=your_serpapi_key_here` అటువంటి లైన్‌ను జోడించండి. `your_serpapi_key_here` ని మీ నిజమైన SerpAPI వెబ్‌సైట్ నుండి పొందిన కీతో మార్చడం మరియు అది స్పష్టంగా ఉండటం చాలా ముఖ్యం.

2. **మాడ్యూల్ కనబడలేదు లోపాలు**
   - `ModuleNotFoundError: No module named 'httpx'` వంటి లోపాలు అవసరమైన Python ప్యాకేజీ లభించకపోవడంతో ఉంటాయి. సాధారణంగా మీరు అన్ని ఆధారాలు ఇన్‌స్టాల్ చేయకపోతే ఇది జరుగుతుంది. దీన్ని సరి చేయడానికి టెర్మినల్‌లో `pip install -r requirements.txt` ఆదేశం నడిపి మీ ప్రాజెక్ట్‌కు అవసరమైన ప్రతిదీ ఇన్‌స్టాల్ చేసుకోండి.

3. **కనెక్షన్ సమస్యలు**
   - మీరు `Error during client execution` వంటి లోపం పొందితే, ఇది సాధారణంగా క్లయింట్ సర్వర్‌కు కనెక్ట్ కాలేకపోవడం లేదా సర్వర్ అంచనా ప్రకారం నడవడం లేదని అర్థం. క్లయింట్ మరియు సర్వర్ ఒకే వెర్షన్లు కాదా అని, `server.py` సరైన డైరెక్టరీలో ఉందా అని, సర్వర్ మరియు క్లయింట్ రెండింటినీ తిరిగి ప్రారంభించడం వంటి చర్యలు తీసుకోండి.

4. **SerpAPI లోపాలు**
   - `Search API returned error status: 401` అని అర్థం మీ SerpAPI కీ లేదని, తప్పు కీ ఇచ్చారని లేదా అవధి గడచిపోయిందని. మీ SerpAPI డాష్‌బోర్డ్‌కు వెళ్లి మీ కీ శుద్ధి చేసి `.env` ఫైలును అవసరమైతే నవీకరించండి. మీ కీ సరైనదయినా ఈ లోపం వస్తే, మీ ఉచిత టియర్ క్వాటా పూర్తయిందో చెక్ చేయండి.

### డీబగ్ మోడ్

సాధారణంగా, యాప్ కీలక సమాచారాన్ని మాత్రమే లాగ్ చేస్తుంది. మీరు మరింత వివరాలను (ఉదాహరణకు క్లిష్టమైన సమస్యల నిర్ధారణ కోసం) చూడాలనుకుంటే, DEBUG మోడ్‌ను ప్రారంభించవచ్చు. ఇది యాప్ తీసుకొనే ప్రతి దశ గురించి సమృద్ధిగా చూపిస్తుంది.

**ఉదాహరణ: సాధారణ అవుట్‌పుట్**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**ఉదాహరణ: DEBUG అవుట్‌పుట్**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

DEBUG మోడ్ HTTP అభ్యర్థనలు, ప్రతిస్పందనలు మరియు ఇతర అంతర్గత వివరాల గురించి అదనపు లైన్లను చూపించును. ఇది ట్రబుల్షూటింగ్‌కు చాలా ఉపయోగకరం.

DEBUG మోడ్ ప్రారంభించడానికి, `client.py` లేదా `server.py` టాప్‌లో లాగింగ్ స్థాయిని DEBUGగా సెట్ చేయండి:

# [Python](#tab/python-debug)


```python
# మీ client.py లేదా server.py యొక్క పైభాగంలో
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO నుండి DEBUG కి మార్చండి
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
