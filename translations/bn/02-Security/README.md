# MCP সিকিউরিটি: AI সিস্টেমের জন্য ব্যাপক সুরক্ষা

[![MCP Security Best Practices](../../../translated_images/bn/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(এই পাঠের ভিডিও দেখতে উপরের ছবিটি ক্লিক করুন)_

সিকিউরিটি AI সিস্টেম ডিজাইনের মৌলিক বিষয়, এজন্য আমরা এটিকে আমাদের দ্বিতীয় বিভাগ হিসাবে অগ্রাধিকার দিই। এটি Microsoft-এর [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) থেকে **Secure by Design** নীতির সাথে সামঞ্জস্যপূর্ণ।

Model Context Protocol (MCP) AI-চালিত অ্যাপ্লিকেশনগুলিতে শক্তিশালী নতুন ক্ষমতা নিয়ে আসে যা ঐতিহ্যবাহী সফটওয়্যার ঝুঁকি ছাড়িয়ে বিশেষ সুরক্ষা চ্যালেঞ্জ তৈরি করে। MCP সিস্টেমগুলি স্থাপিত সুরক্ষা উদ্বেগ মোকাবেলা করে (নিরাপদ কোডিং, ন্যূনতম সুবিধা, সরবরাহ চেন সুরক্ষা) এবং নতুন AI-নির্দিষ্ট হুমকি যেমন প্রম্পট ইনজেকশন, টুল পয়জনিং, সেশন হাইজ্যাকিং, বিভ্রান্ত ডেপুটি আক্রমণ, টোকেন পাসথ্রু দুর্বলতা, এবং গতিশীল ক্ষমতা পরিবর্তনকে অন্তর্ভুক্ত করে।

এই পাঠে MCP বাস্তবায়নে সবচেয়ে গুরুত্বপূর্ণ সুরক্ষা ঝুঁকিগুলি অন্বেষণ করা হয়েছে—যাতে প্রমাণীকরণ, অনুমোদন, অতিরিক্ত অনুমতি, অপ্রত্যক্ষ প্রম্পট ইনজেকশন, সেশন সিকিউরিটি, বিভ্রান্ত ডেপুটি সমস্যা, টোকেন ব্যবস্থাপনা, এবং সরবরাহ চেন দুর্বলতা অন্তর্ভুক্ত। আপনি Microsoft সমাধান যেমন Prompt Shields, Azure Content Safety, এবং GitHub Advanced Security ব্যবহার করে আপনার MCP ডিপ্লয়মেন্ট শক্তিশালী করার জন্য কার্যকর নিয়ন্ত্রণ এবং সেরা অনুশীলন শিখব।

## শেখার উদ্দেশ্যসমূহ

এই পাঠের শেষে, আপনি সক্ষম হবেন:

- **MCP-নির্দিষ্ট হুমকি সনাক্ত করুন**: MCP সিস্টেমের বিশেষ সুরক্ষা ঝুঁকি যেমন প্রম্পট ইনজেকশন, টুল পয়জনিং, অতিরিক্ত অনুমতি, সেশন হাইজ্যাকিং, বিভ্রান্ত ডেপুটি সমস্যা, টোকেন পাসথ্রু দুর্বলতা, এবং সরবরাহ চেন ঝুঁকি চিনতে পারবেন
- **সুরক্ষা নিয়ন্ত্রণ প্রয়োগ করুন**: শক্তিশালী প্রমাণীকরণ, ন্যূনতম সুবিধা প্রবেশাধিকার, নিরাপদ টোকেন ব্যবস্থাপনা, সেশন সুরক্ষা নিয়ন্ত্রণ, এবং সরবরাহ চেন যাচাইকরণ সহ কার্যকর প্রতিকারগুলি বাস্তবায়িত করুন
- **Microsoft সুরক্ষা সমাধান ব্যবহার করুন**: MCP ওয়ার্কলোড সুরক্ষার জন্য Microsoft Prompt Shields, Azure Content Safety, এবং GitHub Advanced Security বুঝে প্রয়োগ করুন
- **টুল সিকিউরিটি যাচাই করুন**: টুল মেটাডেটা যাচাইকরণের গুরুত্ব বুঝুন, গতিশীল পরিবর্তনের জন্য পর্যবেক্ষণ করুন এবং অপ্রত্যক্ষ প্রম্পট ইনজেকশন আক্রমণের বিরুদ্ধে সুরক্ষা নিন
- **সেরা অনুশীলন সমন্বিত করুন**: প্রতিষ্ঠিত সুরক্ষা মৌলিক বিষয়গুলি (নিরাপদ কোডিং, সার্ভার হার্ডেনিং, শূন্য বিশ্বাস) MCP-নির্দিষ্ট নিয়ন্ত্রণের সাথে মিলিয়ে ব্যাপক সুরক্ষা নিশ্চিত করুন

# MCP সিকিউরিটি আর্কিটেকচার ও নিয়ন্ত্রণ

আধুনিক MCP বাস্তবায়নগুলিতে বহস্তর সিকিউরিটি পদ্ধতির প্রয়োজন যা ঐতিহ্যবাহী সফটওয়্যার সুরক্ষা এবং AI-নির্দিষ্ট হুমকিগুলি মোকাবেলা করে। দ্রুত বিকাশমান MCP স্পেসিফিকেশন তার সুরক্ষা নিয়ন্ত্রণ উন্নত করে চলেছে, যা এন্টারপ্রাইজ সিকিউরিটি আর্কিটেকচারের সাথে এবং প্রতিষ্ঠিত সেরা অনুশীলনের সাথে উত্তম সমন্বয় সম্ভব করে তোলে।

[Microsoft Digital Defense Report](https://aka.ms/mddr) থেকে গবেষণা দেখায় যে **প্রতিবেদনকৃত ৯৮% নিরাপত্তা লঙ্ঘন কঠোর সুরক্ষা স্বচ্ছতার মাধ্যমে প্রতিরোধ করা যেত**। সবচেয়ে কার্যকর সুরক্ষা কৌশল হলো ভিত্তিপ্রস্তর সুরক্ষা অনুশীলন এবং MCP-নির্দিষ্ট নিয়ন্ত্রণ একত্রিত করা—স্বীকৃত বেসলাইন সুরক্ষা ব্যবস্থা সামগ্রিক ঝুঁকি কমানোর সবচেয়ে প্রভাবশালী উপায়।

## বর্তমান সুরক্ষা পটভূমি

> **তথ্য:** এই তথ্যটি MCP সিকিউরিটি স্ট্যান্ডার্ডগুলি জানিয়ে দেয় **৫ ফেব্রুয়ারি, ২০২৬** তারিখ পর্যন্ত, যা **MCP Specification 2025-11-25** এর সাথে সামঞ্জস্যপূর্ণ। MCP প্রোটোকল দ্রুত পরিবর্তিত হচ্ছে এবং ভবিষ্যত বাস্তবায়নে নতুন প্রমাণীকরণ প্যাটার্ন এবং উন্নত নিয়ন্ত্রণ অন্তর্ভুক্ত হতে পারে। সর্বদা বর্তমান [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), এবং [security best practices documentation](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) পরামর্শ করুন সর্বশেষ দিকনির্দেশনার জন্য।

> **দৃষ্টি নিবদ্ধ করুন:** `2026-07-28` রিলিজ ক্যান্ডিডেট অনুমোদন আরও কড়া করে — ক্লায়েন্টগণ অনুমোদন প্রতিক্রিয়ায় `iss` প্যারামিটার যাচাই করতে হবে (RFC 9207), Dynamic Client Registration সময় OpenID Connect এর `application_type` ঘোষণা করতে হবে, এবং নিবন্ধিত শংসাপত্র ইস্যুকৃত অনুমোদন সার্ভারের সাথে বাঁধা থাকবে। সময় তালিকা ও বিস্তারিত তথ্যের জন্য দেখুন [What's Changing in MCP: The 2026-07-28 Release Candidate](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md)।

## 🏔️ MCP সিকিউরিটি সামিট ওয়ার্কশপ (শেরপা)

**হ্যান্ডস-অন সিকিউরিটি প্রশিক্ষণের জন্য**, আমরা দৃঢ়ভাবে সুপারিশ করি **MCP Security Summit Workshop** (শেরপা) - Microsoft Azure-এ MCP সার্ভার সুরক্ষায় একটি ব্যাপক নির্দেশিত অভিযান।

### ওয়ার্কশপ ওভারভিউ

[MCP Security Summit Workshop](https://azure-samples.github.io/sherpa/) একটি প্রমাণিত "ভিউনারেবল → এক্সপ্লয়েট → ফিক্স → ভ্যালিডেট" পদ্ধতির মাধ্যমে ব্যবহারিক, কার্যকর সুরক্ষা প্রশিক্ষণ প্রদান করে। আপনি:

- **ভাঙা শিখুন**: ইচ্ছাকৃতভাবে নিরাপত্তাহীন সার্ভারে দুর্বলতাগুলি অনুধাবন করে শিখবেন
- **Azure-নেটিভ সিকিউরিটি ব্যবহার করুন**: Azure Entra ID, Key Vault, API Management, এবং AI Content Safety ব্যবহার করুন
- **ডিফেন্স-ইন-ডেপথ অনুসরণ করুন**: ক্যাম্পগুলোতে ব্যাপক সুরক্ষা স্তর তৈরি করুন
- **OWASP স্ট্যান্ডার্ড অনুসরণ করুন**: প্রতিটি কৌশল [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) এর সাথে মানানসই
- **প্রোডাকশন কোড পান**: কাজ করা, পরীক্ষাসাপেক্ষ বাস্তবায়ন নিয়ে যান

| **MCP02** | সুযোগ বিস্তার দ্বারা সুবিধা বৃদ্ধি | RBAC, Conditional Access |

| **MCP04** | সফটওয়্যার সরবরাহ চেইন আক্রমণ ও নির্ভরশীলতা পরিবর্তন | GitHub Advanced Security, নির্ভরশীলতা স্ক্যানিং |
| **MCP05** | কমান্ড ইনজেকশন ও কার্যকরকরণ | ইনপুট যাচাইকরণ, স্যান্ডবক্সিং |

### MCP প্রমাণীকরণের বিবর্তন

MCP স্পেসিফিকেশন প্রমাণীকরণ এবং অনুমোদনের পদ্ধতিতে উল্লেখযোগ্য উন্নতি লাভ করেছে:

- **মূল পদ্ধতি**: প্রারম্ভিক স্পেসিফিকেশনগুলো বিকাশকারীদের নিজস্ব প্রমাণীকরণ সার্ভার বাস্তবায়ন করতে বাধ্য করত, MCP সার্ভার OAuth 2.0 অনুমোদন সার্ভার হিসেবে সরাসরি ব্যবহারকারীর প্রমাণীকরণ পরিচালনা করত
- **বর্তমান মান (2025-11-25)**: আপডেটকৃত স্পেসিফিকেশন MCP সার্ভারগুলোকে বহিরাগত পরিচয় প্রদানকারীদের (যেমন Microsoft Entra ID) কাছে প্রমাণীকরণ অর্পণের অনুমতি দেয়, যা নিরাপত্তা অবস্থান উন্নত করে এবং বাস্তবায়ন জটিলতা কমায়
- **ট্রান্সপোর্ট লেয়ার সিকিউরিটি**: স্থানীয় (STDIO) এবং দূরবর্তী (Streamable HTTP) সংযোগের জন্য সঠিক প্রমাণীকরণ প্যাটার্ন সহ নিরাপদ পরিবহন প্রক্রিয়ার উন্নত সহায়তা

## প্রমাণীকরণ ও অনুমোদন সুরক্ষা

### বর্তমান সুরক্ষা চ্যালেঞ্জসমূহ

আধুনিক MCP বাস্তবায়নগুলি বিভিন্ন প্রমাণীকরণ ও অনুমোদন সমস্যা সম্মুখীন:

### ঝুঁকি ও হুমকি ভেক্টর

- **ভুলভাবে কনফিগারকৃত অনুমোদন লজিক**: MCP সার্ভারে ভুল অনুমোদন বাস্তবায়ন সংবেদনশীল ডেটা প্রকাশ করতে পারে এবং ভুলভাবে প্রবেশাধিকার নিয়ন্ত্রণ প্রয়োগ করতে পারে
- **OAuth টোকেন দখল**: স্থানীয় MCP সার্ভারের টোকেন চুরি আক্রমণকারীদের সার্ভার নকল করতে এবং নিম্নবর্তী পরিষেবাগুলিতে প্রবেশ করতে সক্ষম করে
- **টোকেন পাসথ্রু দুর্বলতা**: ভুল টোকেন পরিচালনা সুরক্ষা নিয়ন্ত্রণ বাইপাস এবং দায়িত্বের ফাঁক তৈরি করে
- **অতিরিক্ত অনুমতি**: অতিরিক্ত অধিকারপ্রাপ্ত MCP সার্ভার কম সুবিধা নীতিকে লঙ্ঘন করে এবং আক্রমণ পৃষ্ঠ বিস্তৃত করে

#### টোকেন পাসথ্রু: একটি গুরুত্বপূর্ণ বিরতি-প্যাটার্ন

**বর্তমান MCP অনুমোদন স্পেসিফিকেশনে টোকেন পাসথ্রু স্পষ্টভাবে নিষিদ্ধ** কারণ তা গুরুতর সুরক্ষা প্রভাব ফেলে:

##### সুরক্ষা নিয়ন্ত্রণ পরিহার করা
- MCP সার্ভার এবং নিম্নবর্তী API গুলো গুরুত্বপূর্ণ সুরক্ষা নিয়ন্ত্রণ (দর সীমাবদ্ধকরণ, অনুরোধ যাচাইকরণ, ট্রাফিক পর্যবেক্ষণ) বাস্তবায়ন করে যা সঠিক টোকেন যাচাইয়ের উপর নির্ভরশীল
- সরাসরি ক্লায়েন্ট থেকে API টোকেন ব্যবহার এই অপরিহার্য সুরক্ষা দুর্বল করে, সুরক্ষা আর্কিটেকচারের ক্ষতি করে

##### দায়িত্ব ও নিরীক্ষা সমস্যা  
- MCP সার্ভার ব্যবহারকারী ক্লায়েন্ট এবং আপস্ট্রিম ইস্যুকৃত টোকেন পৃথক করতে অক্ষম, যার ফলে নিরীক্ষা ট্রেইল ভঙ্গুর হয়
- নিম্নস্থিত রিসোর্স সার্ভার লগগুলিতে আসল MCP সার্ভার মধ্যস্থতাকারীর বদলে বিভ্রান্তিকর অনুরোধ উত্স দেখা যায়
- ঘটনা তদন্ত এবং সম্মতি নিরীক্ষা অত্যন্ত কঠিন হয়ে পড়ে

##### ডেটা চুরি ঝুঁকি
- অবিচ্ছিন্ন টোকেন দাবীগুলি চুরি করা টোকেনধারী দুর্বৃত্তদের MCP সার্ভারকে ডেটা চোরাগোপ্তা হিসেবে ব্যবহার করার সুযোগ দেয়
- বিশ্বাস সীমা লঙ্ঘনে অনুমোদনহীন প্রবেশাধিকার প্যাটার্ন গ্রহণযোগ্য হয় যা ইচ্ছাকৃত সুরক্ষা নিয়ন্ত্রণ বাইপাস করে

##### বহু-পরিষেবা আক্রমণ ভেক্টর
- গ্রহণযোগ্য টোকেন বহুসংখ্যক পরিষেবাতে গ্রহণযোগ্য হওয়া সংযুক্ত ব্যবস্থাগুলির মধ্যে পার্শ্বচলন সক্ষম করে
- পরিষেবাগুলির মধ্যে বিশ্বাস অনুমানগুলি লঙ্ঘিত হতে পারে যখন টোকেন উত্স যাচাই করা না যায়

### সুরক্ষা নিয়ন্ত্রণ ও প্রতিকার

**গুরুত্বপূর্ণ সুরক্ষা অনুরোধসমূহ:**

> **আবশ্যক:** MCP সার্ভার **অত্যধিকভাবে গ্রহণ করবেন না** কোনো টোকেন যা স্পষ্টভাবে MCP সার্ভারের জন্য ইস্যুকৃত নয়

#### প্রমাণীকরণ ও অনুমোদন নিয়ন্ত্রণ

- **কঠোর অনুমোদন পর্যালোচনা**: MCP সার্ভার অনুমোদন লজিকের ব্যাপক নিরীক্ষা করুন যাতে শুধুমাত্র নির্ধারিত ব্যবহারকারী এবং ক্লায়েন্ট সংবেদনশীল রিসোর্সে প্রবেশাধিকার পায়
  - **বাস্তবায়ন গাইড**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **পরিচয় সংযোজন**: [Using Microsoft Entra ID for MCP Server Authentication](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **নিরাপদ টোকেন ব্যবস্থাপনা**: [Microsoft-এর টোকেন যাচাইকরণ ও জীবনচক্র সেরা অনুশীলন](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) প্রয়োগ করুন
  - টোকেন অডিয়েন্স দাবীগুলি MCP সার্ভার পরিচয়ের সাথে মেলে কিনা যাচাই করুন
  - সঠিক টোকেন ঘূর্ণন ও মেয়াদ শেষ নীতি বাস্তবায়ন করুন
  - টোকেন পুনরাবৃত্তি আক্রমণ এবং অননুমোদিত ব্যবহারের প্রতিরোধ করুন

- **সুরক্ষিত টোকেন স্টোরেজ**: বিশ্রামে এবং যাতায়াতে উভয় ক্ষেত্রেই এনক্রিপশন সহ টোকেন সংরক্ষণে সুরক্ষা প্রদান করুন
  - **সেরা অনুশীলন**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### অ্যাক্সেস কন্ট্রোল বাস্তবায়ন

- **ন্যূনতম সুবিধার নীতি**: MCP সার্ভারকে শুধুমাত্র প্রয়োজনীয় সর্বনিম্ন অনুমতি দিন
  - নিয়মিত অনুমতি পর্যালোচনা এবং আপডেটের মাধ্যমে সুবিধা বিস্তার প্রতিরোধ করুন
  - **Microsoft ডকুমেন্টেশন**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **ভূমিকা-ভিত্তিক অ্যাক্সেস কন্ট্রোল (RBAC)**: সূক্ষ্মভাবে ভূমিকা বরাদ্দ করুন
  - নির্দিষ্ট রিসোর্স এবং ক্রিয়াগুলির জন্য ভূমিকা সীমাবদ্ধ করুন
  - আক্রমণ পৃষ্ঠ বৃদ্ধি করার মতো বিস্তৃত বা অপ্রয়োজনীয় অনুমতি এড়িয়ে চলুন

- **অবিচ্ছিন্ন অনুমতি পর্যবেক্ষণ**: চলমান প্রবেশাধিকার নিরীক্ষা ও পর্যবেক্ষণ প্রয়োগ করুন
  - অনুমতির ব্যবহার প্যাটার্নে অস্বাভাবিকতা খুঁজে বের করুন
  - অতিরিক্ত বা অব্যবহৃত সুবিধা দ্রুত সংশোধন করুন

## AI-নির্দিষ্ট সুরক্ষা হুমকি

### প্রম্পট ইনজেকশন ও টুল ম্যানিপুলেশন আক্রমণসমূহ

আধুনিক MCP বাস্তবায়নগুলো জটিল AI-নির্দিষ্ট আক্রমণ ভেক্টরগুলোর সম্মুখীন যা ঐতিহ্যগত সুরক্ষা ব্যবস্থায় পুরোপুরি মোকাবেলা করা যায় না:

#### **অপ্রত্যক্ষ প্রম্পট ইনজেকশন (ক্রস-ডোমেইন প্রম্পট ইনজেকশন)**

**অপ্রত্যক্ষ প্রম্পট ইনজেকশন** MCP-সক্ষম AI সিস্টেমে সবচেয়ে গুরুত্বপূর্ণ দুর্বলতা। আক্রমণকারীরা বহিরাগত সামগ্রীর মধ্যে—যেমন ডকুমেন্ট, ওয়েব পৃষ্ঠা, ইমেইল, বা তথ্য উৎসে—ক্ষতিকারক নির্দেশনা স্থাপন করে, যা AI সিস্টেমগুলি পরবর্তীতে বৈধ কমান্ড হিসেবে প্রক্রিয়া করে।

**আক্রমণের পরিস্থিতি:**
- **ডকুমেন্ট ভিত্তিক ইনজেকশন**: প্রক্রিয়াজাত ডকুমেন্টে লুকানো ক্ষতিকারক নির্দেশ যা অপ্রত্যাশিত AI ক্রিয়াকে ট্রিগার করে
- **ওয়েব কনটেন্ট শোষণ**: ক্ষতিগ্রস্থ ওয়েব পৃষ্ঠাগুলিতে এমবেড করা প্রম্পট যা স্ক্র্যাপ হলে AI আচরণ নিয়ন্ত্রণ করে
- **ইমেইল ভিত্তিক আক্রমণ**: ইমেইলে ক্ষতিকারক প্রম্পট যা AI সহকারীকে তথ্য ফাঁস করতে বা অননুমোদিত কাজ করতে বাধ্য করে
- **তথ্য উৎস দূষণ**: ক্ষতিগ্রস্থ ডাটাবেস বা API যা AI সিস্টেমে দূষিত সামগ্রী সরবরাহ করে

**বাস্তব জীবনের প্রভাব**: এই আক্রমণগুলো ডেটা চুরি, গোপনীয়তা লঙ্ঘন, ক্ষতিকর সামগ্রী উৎপাদন এবং ব্যবহারকারী মিথস্ক্রিয়ার প্রতি প্রভাব ফেলতে পারে। বিশদ বিশ্লেষণের জন্য দেখুন [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

#### **টুল পয়জনিং আক্রমণসমূহ**

**টুল পয়জনিং** MCP টুলস সংজ্ঞায়িত মেটাডেটার উপর লক্ষ্য করে, যা LLM গুলো কিভাবে টুলের বর্ণনা ও প্যারামিটার ব্যাখ্যা করে নির্বাহ সিদ্ধান্ত নেয় তার সুযোগ নেয়।

**আক্রমণ প্রক্রিয়া:**
- **মেটাডেটা ম্যানিপুলেশন**: আক্রমণকারীরা টুল বর্ণনা, প্যারামিটার সংজ্ঞা, বা ব্যবহারের উদাহরণের মধ্যে ক্ষতিকারক নির্দেশ যোগ করে
- **অদৃশ্য নির্দেশনা**: টুল মেটাডেটাতে লুকানো প্রম্পট যা AI মডেল প্রক্রিয়া করে কিন্তু মানুষের কাছে অদৃশ্য থাকে
- **গতিশীল টুল পরিবর্তন ("রাগ পুল")**: ব্যবহারকারীদের অনুমোদিত টুল পরে পরিবর্তিত হয় ক্ষতিকারক কাজ করার জন্য ব্যবহারকারীর অজান্তে
- **প্যারামিটার ইনজেকশন**: টুল প্যারামিটার স্কিমায় ক্ষতিকারক সামগ্রী এম্বেড যা মডেলের আচরণ প্রভাবিত করে


**হোস্টেড সার্ভার ঝুঁকি**: রিমোট MCP সার্ভারগুলি উচ্চতর ঝুঁকি উপস্থাপন করে কারণ টুল সংজ্ঞাগুলি ব্যবহারকারীর প্রাথমিক অনুমোদনের পরে আপডেট হতে পারে, যার ফলে আগের থেকে নিরাপদ টুলগুলি দুষ্টপ্রকৃতির হয়ে ওঠার পরিস্থিতি তৈরি হয়। বিস্তৃত বিশ্লেষণের জন্য দেখুন [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

- **ক্রস-ডোমেইন প্রম্পট ইনজেকশন (XPIA)**: বহু ডোমেইনের বিষয়বস্তু ব্যবহার করে নিরাপত্তা নিয়ন্ত্রণ বায়পাস করার জটিল আক্রমণ
- **ডায়নামিক ক্যাপাবিলিটি পরিবর্তন**: টুলের ক্ষমতাগুলোর রিয়েল-টাইম পরিবর্তন যা প্রাথমিক নিরাপত্তা মূল্যায়ন এড়ায়
- **কন্টেক্সট উইন্ডো পয়জনিং**: বড় কন্টেক্সট উইন্ডো ম্যানিপুলেট করে ক্ষতিকর নির্দেশাবলী লুকানোর আক্রমণ
- **মডেল কনফিউশন আক্রমণ**: মডেলের সীমাবদ্ধতাকে কাজে লাগিয়ে অনিয়ন্ত্রিত বা অনিরাপদ আচরণ তৈরি করা

## উন্নত MCP নিরাপত্তা হুমকি

### সেশন হাইজ্যাকিং দুর্বলতা

**সেশন হাইজ্যাকিং** একটি গুরত্বপূর্ণ আক্রমণ ভেক্টর যা স্টেটফুল MCP বাস্তবায়নে অবৈধ পক্ষকে বৈধ সেশন আইডি অর্জন ও অপব্যবহার করার মাধ্যমে ক্লায়েন্ট এর ভয়েস হয়ে অননুমোদিত ক্রিয়া পরিচালনা করতে দেয়।

#### **আক্রমণ পরিস্থিতি ও ঝুঁকি**

- **সেশন হাইজ্যাক প্রম্পট ইনজেকশন**: চুরি হওয়া সেশন আইডি সহ আক্রমণকারীরা সার্ভারে ম্যালিশিয়াস ইভেন্ট ইনজেক্ট করে, যা ক্ষতিকর কার্যকলাপ ঘটাতে বা সংবেদনশীল তথ্য অ্যাক্সেস করতে পারে
- **সরাসরি ছদ্মবেশ**: চুরি হওয়া সেশন আইডি দ্বারা ব্যতীত MCP সার্ভার কল করা যায় যা প্রমাণীকরণ পাস করে, আক্রমণকারীদের বৈধ ব্যবহারকারী হিসাবে গ্রহণ করে
- **কম্প্রোমাইজড রিসিউমেবল স্ট্রিমস**: আক্রমণকারীরা অনুরোধগুলো প্রাথমিকভাবে সংক্ষেপ করে যেখানে বৈধ ক্লায়েন্ট সম্ভাব্য ম্যালিশিয়াস সামগ্রীর সাথে পুনরায় শুরু করতে পারে

#### **সেশন ব্যবস্থাপনার জন্য নিরাপত্তা নিয়ন্ত্রণ**

**গুরুত্বপূর্ণ প্রয়োজনীয়তা:**
- **অধিকার যাচাই**: MCP সার্ভার যা অনুমোদন বাস্তবায়ন করে **প্রতিটি** ইনবাউন্ড রিকোয়েস্ট যাচাই করতে **হবেই** এবং সেশনকে প্রমাণীকরণের জন্য ব্যবহার করা উচিত নয়
- **নিরাপদ সেশন জেনারেশন**: ক্রিপ্টোগ্রাফিক্যালি নিরাপদ, অগণনীয় সেশন আইডি তৈরি করতে নিরাপদ র‍্যান্ডম নাম্বার জেনারেটর ব্যবহার করুন
- **ব্যবহারকারী-নির্দিষ্ট বাইন্ডিং**: সেশন আইডি ব্যবহারকারীর নির্দিষ্ট তথ্যের সাথে যুক্ত করুন `<user_id>:<session_id>` ফরম্যাটে, যাতে ব্যবহারকারীদের সেশন অপব্যবহার রোধ হয়
- **সেশন লাইফসাইকেল ব্যবস্থাপনা**: সমীচীন মেয়াদ শেষ, রোটেশন এবং বাতিলকরণ প্রয়োগ করুন যাতে ঝুঁকি কম হয়
- **ট্রান্সপোর্ট সিকিউরিটি**: সেশন আইডি আটকানো রোধ করতে সকল যোগাযোগে HTTPS বাধ্যতামূলক

### বিভ্রান্ত প্রতিরক্ষাকারী সমস্যা

**বিভ্রান্ত প্রতিরক্ষাকারী সমস্যা** ঘটে যখন MCP সার্ভার ক্লায়েন্ট ও তৃতীয় পক্ষের সেবাগুলির মধ্যে প্রমাণীকরণ প্রক্সি হিসাবে কাজ করে, যেখানে স্ট্যাটিক ক্লায়েন্ট আইডি ব্যবহার করে অনুমোদন বাইপাসের সুযোগ সৃষ্টি হয়।

#### **আক্রমণ প্রক্রিয়া ও ঝুঁকি**

- **কুকি-ভিত্তিক সম্মতি বাইপাস**: পূর্বের ব্যবহারকারী প্রমাণীকরণ সম্মতি কুকি তৈরি করে যা আক্রমণকারীরা ক্ষতিকর অনুমোদন অনুরোধের মাধ্যমে অপব্যবহার করে, তৈরি করা রিডাইরেক্ট URI সহ
- **অনুমোদন কোড চুরি**: বিদ্যমান সম্মতি কুকির কারণে অনুমোদন সার্ভার সম্মতি স্ক্রীন পাস করে, কোডগুলো আক্রমণকারী-নিয়ন্ত্রিত এন্ডপয়েন্টে রিডাইরেক্ট করে
- **অননুমোদিত API অ্যাক্সেস**: চুরি হওয়া অনুমোদন কোড টোকেন বিনিময় এবং ব্যবহারকারী ছদ্মবেশ অনুমতি দেয়

#### **নিরসন কৌশল**

**আবশ্যিক নিয়ন্ত্রণ:**
- **স্পষ্ট সম্মতি প্রয়োজনীয়তা**: MCP প্রক্সি সার্ভার যা স্ট্যাটিক ক্লায়েন্ট আইডি ব্যবহার করে **প্রতিটি** ডায়নামিক ভাবে নিবন্ধিত ক্লায়েন্টের জন্য ব্যবহারকারীর সম্মতি নিতে হবে
- **OAuth 2.1 নিরাপত্তা বাস্তবায়ন**: সর্বশেষ OAuth নিরাপত্তা অনুশীলন অনুসরণ করুন PKCE (Proof Key for Code Exchange) সহ সকল অনুমোদন অনুরোধের জন্য
- **কঠোর ক্লায়েন্ট যাচাই**: রিডাইরেক্ট URI ও ক্লায়েন্ট আইডেন্টিফায়ার কঠোর যাচাই প্রয়োগ করে অপব্যবহার প্রতিরোধ করুন

### টোকেন পাসথ্রু দুর্বলতা

**টোকেন পাসথ্রু** একটি স্পষ্ট বিরুদ্ধ অবস্থা যেখানে MCP সার্ভার ক্লায়েন্ট টোকেন সঠিক যাচাই ছাড়াই গ্রহণ করে এবং এগুলোকে নিম্নতর API তে ফরওয়ার্ড করে, যা MCP অনুমোদন স্পেসিফিকেশন লঙ্ঘন করে।

#### **নিরাপত্তা প্রভাব**

- **নিয়ন্ত্রণ ত্যাগ**: সরাসরি ক্লায়েন্ট থেকে API টোকেন ব্যবহারে গুরুত্বপূর্ণ রেট সীমাবদ্ধতা, যাচাই ও মনিটরিং নিয়ন্ত্রণ বাইপাস হয়
- **অডিট ট্রেইল ক্ষতি**: আপস্ট্রিম ইস্যুকৃত টোকেন ব্যবহার করলে ক্লায়েন্ট শনাক্তকরণ সম্ভব হয় না, ঘটনা তদন্তের সক্ষমতা বিঘ্নিত হয়
- **প্রক্সি-ভিত্তিক ডেটা চুরি**: অনিরীক্ষিত টোকেন ব্যবহার করে ক্ষতিকর পক্ষগুলি সার্ভারকে অননুমোদিত ডেটা অ্যাক্সেসের জন্য প্রোক্সি হিসেবে ব্যবহার করতে পারে
- **ট্রাস্ট বাউন্ডারি লঙ্ঘন**: টোকেন উৎস যাচাই করা না গেলে নিম্নতর সেবাগুলোর ট্রাস্ট অনুমান লঙ্ঘিত হতে পারে
- **বহু-সেবা আক্রমণ বিস্তার**: একাধিক সেবায় গ্রহণযোগ্য কম্প্রোমাইজড টোকেন লাটারাল মুভমেন্ট সক্ষম করে

#### **আবশ্যক নিরাপত্তা নিয়ন্ত্রণ**

**অপরিবর্তনীয় প্রয়োজনীয়তা:**
- **টোকেন যাচাই**: MCP সার্ভার স্পষ্টভাবে MCP সার্ভারের জন্য ইস্যুকৃত নয় এমন টোকেন গ্রহণ করা উচিত নয়
- **অডিয়েন্স যাচাই**: সর্বদা যাচাই করুন যে টোকেনের অডিয়েন্স দাবি MCP সার্ভারের পরিচয়ের সাথে মেলে
- **সঠিক টোকেন জীবনচক্র**: স্বল্পমেয়াদী অ্যাক্সেস টোকেন প্রয়োগ করুন নিরাপদ রোটেশন পদ্ধতি সহ


## AI সিস্টেমের জন্য সাপ্লাই চেইন নিরাপত্তা

সাপ্লাই চেইন নিরাপত্তা ঐতিহ্যগত সফটওয়্যার নির্ভরশীলতার বাইরে বিস্তৃত হয়ে পুরো AI ইকোসিস্টেম জুড়ে উন্নত হয়েছে। আধুনিক MCP বাস্তবায়নে সমস্ত AI-সংক্রান্ত উপাদানগুলি কঠোরভাবে যাচাই ও পর্যবেক্ষণ করা আবশ্যক, কারণ প্রতিটি সম্ভাব্য দুর্বলতা নিয়ে আসে যা সিস্টেমের অখণ্ডতা ক্ষতিগ্রস্থ করতে পারে।

### সম্প্রসারিত AI সাপ্লাই চেইন উপাদানসমূহ

**ঐতিহ্যগত সফটওয়্যার নির্ভরশীলতা:**
- ওপেন সোর্স লাইব্রেরি ও ফ্রেমওয়ার্ক
- কন্টেনার ইমেজ ও বেস সিস্টেম  
- ডেভেলপমেন্ট টুলস ও বিল্ড পাইপলাইন
- অবকাঠামো উপাদান ও সেবা

**AI-নির্দিষ্ট সাপ্লাই চেইন উপাদান:**
- **ফাউন্ডেশন মডেল**: বিভিন্ন পরিষেবা প্রদানকারীর থেকে প্রি-ট্রেইন্ড মডেল যাদের প্রোভেন্যান্স যাচাই প্রয়োজন
- **এম্বেডিং সেবা**: বাহ্যিক ভেক্টরাইজেশন ও সেমান্টিক সার্চ সেবা
- **কন্টেক্সট প্রদানকারী**: ডেটা উৎস, জ্ঞানভাণ্ডার ও ডকুমেন্ট রিপোজিটরি  
- **তৃতীয় পক্ষের API**: বাহ্যিক AI সেবা, ML পাইপলাইন ও ডেটা প্রক্রিয়াকরণ এন্ডপয়েন্ট
- **মডেল আর্টিফ্যাক্টস**: ওজন, কনফিগারেশন ও ফাইন-টিউন মডেল ভেরিয়েন্ট
- **ট্রেইনিং ডেটা উৎস**: মডেল প্রশিক্ষণ ও ফাইন-টিউনিং এর জন্য ব্যবহৃত ডেটাসেট

### ব্যাপক সাপ্লাই চেইন নিরাপত্তা কৌশল

#### **উপাদান যাচাই ও বিশ্বাসযোগ্যতা**
- **প্রোভেন্যান্স যাচাইকরণ**: সংযুক্তির আগে সমস্ত AI উপাদানের উত্স, লাইসেন্স ও অখণ্ডতা যাচাই করুন
- **নিরাপত্তা মূল্যায়ন**: মডেল, ডেটা উৎস এবং AI সেবার জন্য দুর্বলতা স্ক্যান ও নিরাপত্তা পর্যালোচনা পরিচালনা করুন
- **খ্যাতি বিশ্লেষণ**: AI সেবা প্রদানকারীর নিরাপত্তা ট্র্যাক রেকর্ড ও অনুশীলন মূল্যায়ন করুন
- **অবাধ্যতার যাচাইকরণ**: নিশ্চিত করুন সব উপাদান সংস্থার নিরাপত্তা এবং নিয়ন্ত্রক চাহিদা পূরণ করে

#### **নিরাপদ ডেপ্লয়মেন্ট পাইপলাইন**
- **স্বয়ংক্রিয় CI/CD নিরাপত্তা**: স্বয়ংক্রিয় ডেপ্লয়মেন্ট পাইপলাইনে নিরাপত্তা স্ক্যানিং সংযোজন
- **আর্টিফ্যাক্ট অখণ্ডতা**: সমস্ত ডেপ্লয়মেন্ট আর্টিফ্যাক্টস (কোড, মডেল, কনফিগারেশন) ক্রিপ্টোগ্রাফিক যাচাইকরণ প্রয়োগ
- **পর্যায়ক্রমিক ডেপ্লয়মেন্ট**: প্রতিটি পর্যায়ে নিরাপত্তা যাচাইকরণের সঙ্গে প্রগ্রেসিভ ডেপ্লয়মেন্ট কৌশল ব্যবহার
- **বিশ্বস্ত আর্টিফ্যাক্ট রিপোজিটরি**: যাচাইকৃত, নিরাপদ আর্টিফ্যাক্ট তালিকা ও রিপোজিটরি থেকে শুধুমাত্র ডেপ্লয় করুন

#### **অবিচ্ছিন্ন পর্যবেক্ষণ ও প্রতিক্রিয়া**
- **নির্ভরশীলতা স্ক্যানিং**: সব সফটওয়্যার ও AI উপাদান নির্ভরশীলতার জন্য চলমান দুর্বলতা পর্যবেক্ষণ
- **মডেল পর্যবেক্ষণ**: মডেল আচরণ, পারফরম্যান্স ড্রিফট এবং নিরাপত্তা অস্বাভাবিকতা অবিচ্ছিন্ন মূল্যায়ন
- **সেবা স্বাস্থ্য ট্র্যাকিং**: বাহ্যিক AI সেবার উপলব্ধতা, নিরাপত্তা ঘটনা ও নীতিমালা পরিবর্তন পর্যবেক্ষণ
- **হুমকি বুদ্ধিমত্তা ইন্টিগ্রেশন**: AI ও ML নিরাপত্তা ঝুঁকির জন্য বিশেষ হুমকি ফিড অন্তর্ভুক্তকরণ

#### **অ্যাক্সেস কন্ট্রোল ও ন্যূনতম সুবিধা**
- **উপাদান-স্তরের অনুমতি**: ব্যবসায়িক প্রয়োজন অনুসারে মডেল, ডেটা এবং সেবায় অ্যাক্সেস সীমাবদ্ধ করুন
- **সেবা অ্যাকাউন্ট ব্যবস্থাপনা**: সর্বনিম্ন প্রয়োজনীয় অনুমতির সঙ্গে নিবেদিত সেবা অ্যাকাউন্ট বাস্তবায়ন
- **নেটওয়ার্ক বিচ্ছিন্নকরণ**: AI উপাদান আলাদা করুন ও সেবাগুলো মধ্যে নেটওয়ার্ক অ্যাক্সেস সীমিত করুন
- **API গেটওয়ে নিয়ন্ত্রণ**: বাহ্যিক AI সেবায় অ্যাক্সেস নিয়ন্ত্রণ ও পর্যবেক্ষণের জন্য কেন্দ্রীভূত API গেটওয়ে ব্যবহার করুন

#### **ঘটনা প্রতিক্রিয়া ও পুনরুদ্ধার**
- **দ্রুত প্রতিক্রিয়া পদ্ধতি**: কম্প্রোমাইজড AI উপাদান প্রতিস্থাপন বা প্যাচ করার প্রতিষ্ঠিত প্রক্রিয়া
- **ক্রেডেনশিয়াল রোটেশন**: গোপনীয়তা, API কী এবং সেবা ক্রেডেনশিয়ালের জন্য স্বয়ংক্রিয় রোটেশন সিস্টেম
- **রোলব্যাক ক্ষমতা**: পূর্বের সুপরিচিত ভাল সংস্করণের AI উপাদানে দ্রুত ফিরে যাওয়ার যোগ্যতা
- **সাপ্লাই চেইন লঙ্ঘন পুনরুদ্ধার**: আপস্ট্রিম AI সেবা কম্প্রোমাইজের জন্য নির্দিষ্ট প্রতিক্রিয়া প্রক্রিয়া

## ভিত্তি নিরাপত্তা সেরা অনুশীলন

MCP বাস্তবায়নগুলি আপনার সংস্থার বিদ্যমান নিরাপত্তা অবস্থা গ্রহণ করে এবং এর উপর নির্মিত হয়। ভিত্তিমূল নিরাপত্তা অনুশীলনগুলি শক্তিশালীকরণ AI সিস্টেম ও MCP ডেপ্লয়মেন্টের সামগ্রিক নিরাপত্তা উল্লেখযোগ্যভাবে বৃদ্ধি করে।

### মূল নিরাপত্তা ভিত্তি

#### **নিরাপদ ডেভেলপমেন্ট অনুশীলন**
- **OWASP সম্মতি**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) ওয়েব অ্যাপ্লিকেশন দুর্বলতার বিরুদ্ধে সুরক্ষা
- **AI-নির্দিষ্ট সুরক্ষা**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) এর জন্য নিয়ন্ত্রণ বাস্তবায়ন
- **নিরাপদ সিক্রেটস ব্যবস্থাপনা**: টোকেন, API কী এবং সংবেদনশীল কনফিগারেশন তথ্যের জন্য নিবেদিত ভল্ট ব্যবহার
- **এন্ড-টু-এন্ড এনক্রিপশন**: সকল অ্যাপ্লিকেশন উপাদান ও ডেটা প্রবাহে নিরাপদ যোগাযোগ বাস্তবায়ন
- **ইনপুট যাচাই**: সমস্ত ব্যবহারকারী ইনপুট, API প্যারামিটার ও ডেটা উৎসের কঠোর যাচাই

#### **অবকাঠামোর শক্তিশালীকরণ**
- **মাল্টি-ফ্যাক্টর প্রমাণীকরণ**: সমস্ত প্রশাসনিক ও সেবা অ্যাকাউন্টের জন্য আবশ্যক MFA
- **প্যাচ ব্যবস্থাপনা**: অপারেটিং সিস্টেম, ফ্রেমওয়ার্ক এবং নির্ভরশীলতার জন্য স্বয়ংক্রিয় ও সময়মত প্যাচিং  
- **আইডেন্টিটি প্রদানকারী ইন্টিগ্রেশন**: এন্টারপ্রাইজ আইডেন্টিটি প্রদানকারী (Microsoft Entra ID, Active Directory) মাধ্যমে কেন্দ্রীকৃত পরিচয় ব্যবস্থাপনা
- **নেটওয়ার্ক বিচ্ছিন্নকরণ**: MCP উপাদান আলাদা করার জন্য যুক্তিসঙ্গত বিচ্ছিন্নকরণ যাতে লাটারাল মুভমেন্ট সীমাবদ্ধ হয়
- **ন্যূনতম সুবিধার নীতি**: সমস্ত সিস্টেম উপাদান ও অ্যাকাউন্টে ন্যূনতম প্রয়োজনীয় অনুমতি

#### **নিরাপত্তা পর্যবেক্ষণ ও সনাক্তকরণ**
- **বিস্তৃত লগিং**: AI অ্যাপ্লিকেশন কার্যকলাপের বিস্তারিত লগিং, যার মধ্যে MCP ক্লায়েন্ট-সার্ভার ইন্টারঅ্যাকশন অন্তর্ভুক্ত
- **SIEM ইন্টিগ্রেশন**: অস্বাভাবিকতা সনাক্তকরণের জন্য কেন্দ্রীভূত নিরাপত্তা তথ্য ও ঘটনা ব্যবস্থাপনা
- **আচরণ বিশ্লেষণ**: সিস্টেম ও ব্যবহারকারী আচরণের অস্বাভাবিক প্যাটার্ন সনাক্ত করতে AI-চালিত পর্যবেক্ষণ
- **হুমকি বুদ্ধিমত্তা**: বহিরাগত হুমকি ফিড ও কমপ্রোমাইজ সূচক (IOCs) এর সংযোজন
- **ঘটনা প্রতিক্রিয়া**: নিরাপত্তা ঘটনা সনাক্তকরণ, প্রতিক্রিয়া এবং পুনরুদ্ধারের সুসংজ্ঞায়িত প্রক্রিয়া

#### **জিরো ট্রাস্ট আর্কিটেকচার**
- **কখনই বিশ্বাস করবেন না, সর্বদা যাচাই করুন**: ব্যবহারকারী, ডিভাইস এবং নেটওয়ার্ক সংযোগের অবিচ্ছিন্ন যাচাই
- **মাইক্রো-সেগমেন্টেশন**: স্বতন্ত্র ওয়ার্কলোড ও সেবাগুলোকে বিচ্ছিন্ন করার জন্য সূক্ষ্ম নেটওয়ার্ক নিয়ন্ত্রণ
- **পরিচয়-কেন্দ্রিক নিরাপত্তা**: নেটওয়ার্ক অবস্থানের পরিবর্তে যাচাইকৃত পরিচয়ের উপর ভিত্তি করে নিরাপত্তা নীতি
- **অবিচ্ছিন্ন ঝুঁকি মূল্যায়ন**: বর্তমান প্রসঙ্গ ও আচরণের ভিত্তিতে গতিশীল নিরাপত্তা অবস্থা মূল্যায়ন
- **শর্তাধীন অ্যাক্সেস**: ঝুঁকি উপাদান, অবস্থান ও ডিভাইসের বিশ্বাসের উপর ভিত্তি করে অভিযোজিত প্রবেশ নিয়ন্ত্রণ

### এন্টারপ্রাইজ ইন্টিগ্রেশন প্যাটার্ন

#### **Microsoft নিরাপত্তা ইকোসিস্টেম ইন্টিগ্রেশন**
- **Microsoft Defender for Cloud**: বিস্তৃত ক্লাউড নিরাপত্তা অবস্থা পরিচালনা
- **Azure Sentinel**: AI ওয়ার্কলোড সুরক্ষার জন্য ক্লাউড-দেশীয় SIEM ও SOAR ক্ষমতা
- **Microsoft Entra ID**: শর্তাধীন অ্যাক্সেস নীতি সহ এন্টারপ্রাইজ পরিচয় ও অ্যাক্সেস ব্যবস্থাপনা
- **Azure Key Vault**: হার্ডওয়্যার নিরাপত্তা মডিউল (HSM) সহ কেন্দ্রীভূত সিক্রেট ম্যানেজমেন্ট
- **Microsoft Purview**: AI ডেটা উৎস ও ওয়ার্কফ্লোর জন্য ডেটা শাসন ও সম্মতি

#### **সম্মতি ও শাসন**
- **নিয়ন্ত্রক সামঞ্জস্যতা**: MCP বাস্তবায়ন শিল্প-নির্দিষ্ট সম্মতি প্রয়োজন (GDPR, HIPAA, SOC 2) মেনে চলা নিশ্চিত করুন

- **তথ্য শ্রেণীবদ্ধকরণ**: AI সিস্টেম দ্বারা প্রক্রিয়াজাত সংবেদনশীল তথ্যের সঠিক শ্রেণীবিভাগ এবং হ্যান্ডলিং
- **অডিট ট্রেইলস**: নিয়ন্ত্রক সম্মতি এবং ফরেনসিক তদন্তের জন্য ব্যাপক লগিং
- **গোপনীয়তা নিয়ন্ত্রণ**: AI সিস্টেম আর্কিটেকচারে গোপনীয়তা-বাই-ডিজাইন নীতিমালা প্রয়োগ
- **পরিবর্তন ব্যবস্থাপনা**: AI সিস্টেম সংশোধনের সুরক্ষা পর্যালোচনার জন্য আনুষ্ঠানিক প্রক্রিয়া

এই ভিত্তিপ্রস্তর অভ্যাসগুলি একটি দৃঢ় সুরক্ষা বেসলাইন তৈরি করে যা MCP-নির্দিষ্ট সুরক্ষা নিয়ন্ত্রণের কার্যকারিতা বাড়ায় এবং AI-চালিত অ্যাপ্লিকেশনগুলির জন্য ব্যাপক সুরক্ষা প্রদান করে।

## মূল সুরক্ষা বিষয়াবলী

- **স্তরভিত্তিক সুরক্ষা পদ্ধতি**: মৌলিক সুরক্ষা অনুশীলন (নিরাপদ কোডিং, ন্যূনতম অনুমতি, সরবরাহ শৃঙ্খল যাচাই, ক্রমাগত মনিটরিং) কে AI-নির্দিষ্ট নিয়ন্ত্রণগুলোর সাথে মিলিয়ে ব্যাপক সুরক্ষা প্রদান

- **AI-নির্দিষ্ট হুমকি প্রেক্ষাপট**: MCP সিস্টেমগুলো অনন্য ঝুঁকির সম্মুখীন যেমন প্রম্পট ইনজেকশন, টুল পয়জনিং, সেশন হাইজ্যাকিং, কনফিউজড ডেপুটি সমস্যা, টোকেন পাসথ্রু দুর্বলতা এবং অতিরিক্ত অনুমতির জন্য বিশেষায়িত প্রতিকার প্রয়োজন

- **প্রমাণীকরণ ও অনুমোদনে উৎকর্ষ**: শক্তিশালী প্রমাণীকরণ বাস্তবায়ন করুন বাইরের পরিচয় প্রদানকারী (Microsoft Entra ID) ব্যবহার করে, সঠিক টোকেন যাচাই নিশ্চিতে, এবং কখনও আপনার MCP সার্ভারের জন্য স্পষ্টভাবে ইস্যুকৃত না হওয়া টোকেন গ্রহণ করবেন না

- **AI আক্রমণ প্রতিরোধ**: Microsoft Prompt Shields এবং Azure Content Safety ব্যবহার করে পরোক্ষ প্রম্পট ইনজেকশন ও টুল পয়জনিং আক্রমণ প্রতিরোধ করুন, টুল মেটাডেটা যাচাই করুন এবং গতিশীল পরিবর্তন নজরদারি করুন

- **সেশন ও পরিবহন সুরক্ষা**: ব্যবহারকারী পরিচয়ের সাথে বাঁধাযোগ্য কমপক্ষে নির্ধারিত নয় এমন ক্রিপ্টোগ্রাফিকভাবে সুরক্ষিত সেশন আইডি ব্যবহার করুন, সেশন জীবচক্র ব্যবস্থাপনা সঠিকভাবে বাস্তবায়ন করুন, এবং কখনও সেশন প্রমাণীকরণের জন্য ব্যবহার করবেন না

- **OAuth সুরক্ষা সেরা অনুশীলন**: কনফিউজড ডেপুটি আক্রমণ প্রতিরোধ করুন স্পষ্ট ব্যবহারকারীর সম্মতি প্রাপ্তির মাধ্যমে ডায়নামিকভাবে নিবন্ধিত ক্লায়েন্টদের জন্য, PKCE সহ সঠিক OAuth 2.1 বাস্তবায়ন, এবং কঠোর রিডাইরেক্ট URI যাচাই  

- **টোকেন সুরক্ষা নীতিমালা**: টোকেন পাসথ্রু অ্যান্টি-প্যাটার্ন এড়িয়ে চলুন, টোকেন অডিয়েন্স ক্লেইম যাচাই করুন, নিরাপদ ঘূর্ণায়মান সহ স্বল্পস্থায়ী টোকেন বাস্তবায়ন করুন, এবং পরিষ্কার বিশ্বাস সীমা বজায় রাখুন

- **ব্যাপক সরবরাহ শৃঙ্খল সুরক্ষা**: সমস্ত AI ইকোসিস্টেম উপাদান (মডেল, এমবেডিংস, প্রসঙ্গ প্রদানকারী, বাইরের API) কে ঐতিহ্যবাহী সফটওয়্যার নির্ভরতার মতো একই সুরক্ষা কঠোরতায় বিবেচনা করুন

- **নিরবচ্ছিন্ন বিবর্তন**: দ্রুত পরিবর্তিত MCP স্পেসিফিকেশনগুলোর সাথে আপ টু ডেট থাকুন, সুরক্ষা সম্প্রদায়ের মান অনুসরণে অংশগ্রহণ করুন, এবং প্রোটোকল পরিপক্ক হওয়ার সাথে সাথে অভিযোজিত সুরক্ষা অবস্থান বজায় রাখুন

- **মাইক্রোসফট সুরক্ষা একত্রীকরণ**: Microsoft এর ব্যাপক সুরক্ষা ইকোসিস্টেম (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) ব্যবহার করুন MCP নিযুক্তির সুরক্ষা বৃদ্ধির জন্য
