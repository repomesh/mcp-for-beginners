# পাঠ: একটি ওয়েব সার্চ MCP সার্ভার তৈরি

এই অধ্যায়ে দেখানো হয়েছে কিভাবে বাহ্যিক API ইন্টিগ্রেট করে, বিভিন্ন ধরনের ডেটা হ্যান্ডেল করে, ত্রুটি পরিচালনা করে এবং একাধিক টুল সমন্বয় সাধন করে একটি বাস্তব সমস্যা সমাধানকারী AI এজেন্ট সৃষ্টি করা যায়—সবই একটি উৎপাদন-সজ্জিত ফরম্যাটে। আপনি দেখতে পাবেন:

- **প্রমাণীকরণের প্রয়োজনীয়তা সহ বাহ্যিক API গুলোর সাথে ইন্টিগ্রেশন**
- **বিভিন্ন এন্ডপয়েন্ট থেকে বিভিন্ন ধরনের ডেটার হ্যান্ডলিং**
- **কঠোর ত্রুটি হ্যান্ডলিং এবং লগিং কৌশল**
- **একক সার্ভারে একাধিক টুল সমন্বয়**

শেষে, আপনার কাছে উন্নত AI এবং LLM-চালিত অ্যাপ্লিকেশনের জন্য অপরিহার্য প্যাটার্ন ও সেরা অনুশীলন নিয়ে ব্যবহারিক অভিজ্ঞতা থাকবে।

## পরিচিতি

এই পাঠে, আপনি শিখবেন কিভাবে একটি উন্নত MCP সার্ভার ও ক্লায়েন্ট তৈরি করতে হয় যা SerpAPI ব্যবহার করে রিয়েল-টাইম ওয়েব ডেটার সাথে LLM ক্ষমতা বৃদ্ধি করে। এটি একটি গুরুত্বপূর্ণ দক্ষতা যা গতিশীল AI এজেন্ট তৈরি করতে সাহায্য করে যারা ওয়েব থেকে সর্বশেষ তথ্য অ্যাক্সেস করতে পারে।

## শেখার উদ্দেশ্য

এই পাঠ শেষে, আপনি সক্ষম হবেন:

- SerpAPI’র মতো বাহ্যিক API কে নিরাপদে MCP সার্ভারে সংযোজিত করা
- ওয়েব, সংবাদ, পণ্য অনুসন্ধান, এবং প্রশ্নোত্তর জন্য একাধিক টুল ব্যবহার করা
- কাঠামোবদ্ধ ডেটা পার্স এবং ফরম্যাট করে LLM প্রক্রিয়াজাতকরণের জন্য প্রস্তুত করা
- ত্রুটি পরিচালনা ও API রেট লিমিট দক্ষতার সাথে হ্যান্ডেল করা
- স্বয়ংক্রিয় এবং ইন্টারেক্টিভ MCP ক্লায়েন্ট উভয়ই তৈরি ও পরীক্ষা করা

## ওয়েব সার্চ MCP সার্ভার

এই অংশে ওয়েব সার্চ MCP সার্ভারের স্থাপত্য এবং বৈশিষ্ট্যগুলি উপস্থাপন করা হয়েছে। আপনি দেখতে পাবেন কিভাবে FastMCP এবং SerpAPI একত্রে LLM এর ক্ষমতা রিয়েল-টাইম ওয়েব ডেটার মাধ্যমে বাড়ায়।

### ওভারভিউ

এই বাস্তবায়নে চারটি টুল রয়েছে যা MCP এর শক্তিশালী বাহ্যিক API-চালিত কাজগুলি নিরাপদ এবং দক্ষতার সাথে পরিচালনার ক্ষমতা প্রদর্শন করে:

- **general_search**: বিস্তৃত ওয়েব অনুসন্ধানের জন্য
- **news_search**: সাম্প্রতিক শিরোনামের জন্য
- **product_search**: ই-কমার্স ডেটার জন্য
- **qna**: প্রশ্নোত্তর অংশের জন্য

### বৈশিষ্ট্যাবলী
- **কোড উদাহরণ**: পাইথন সহ ভাষা-নির্দিষ্ট কোড ব্লক অন্তর্ভুক্ত, সহজে অন্য ভাষায় সম্প্রসারণযোগ্য, বোধগম্যতার জন্য কোড পিভট ব্যবহার করে

### পাইথন

```python
# general_search টুলের উদাহরণ ব্যবহারের জন্য
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

ক্লায়েন্ট চালানোর আগে সার্ভার কি করে তা বোঝা গুরুত্বপূর্ণ। [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ফাইলটি MCP সার্ভার বাস্তবায়ন করে, যা SerpAPI ইন্টিগ্রেশন ব্যবহার করে ওয়েব, সংবাদ, পণ্য অনুসন্ধান এবং প্রশ্নোত্তর টুলগুলি প্রকাশ করে। এটি ইনকামিং অনুরোধ পরিচালনা করে, API কলগুলি নিয়ন্ত্রণ করে, প্রতিক্রিয়া পার্স করে এবং কাঠামোবদ্ধ ফলাফল ক্লায়েন্টকে ফিরিয়ে দেয়।

আপনি [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) এ সম্পূর্ণ বাস্তবায়ন দেখতে পারেন।

এখানে সার্ভার কীভাবে একটি টুল ডিফাইন ও রেজিস্টার করে তার সংক্ষিপ্ত উদাহরণ দেওয়া হল:

### পাইথন সার্ভার

```python
# server.py (উদ্ধৃতি)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...বাস্তবায়ন...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **বাহ্যিক API ইন্টিগ্রেশন**: API কী এবং বাহ্যিক অনুরোধগুলি নিরাপদে পরিচালনার প্রদর্শনী
- **কাঠামোবদ্ধ ডেটা পার্সিং**: API প্রতিক্রিয়াগুলিকে LLM-বন্ধুভাবেক ফরম্যাটে রূপান্তর করার উপায় দেখানো
- **ত্রুটি হ্যান্ডলিং**: যথাযথ লগিং সহ দৃঢ় ত্রুটি পরিচালনা
- **ইন্টারেক্টিভ ক্লায়েন্ট**: স্বয়ংক্রিয় টেস্ট ও ইন্টারেক্টিভ মোড উভয়ই অন্তর্ভুক্ত
- **কন্টেক্সট ম্যানেজমেন্ট**: MCP কন্টেক্সট ব্যবহার করে লগিং ও অনুরোধ ট্র্যাকিং

## প্রয়োজনীয়তা

শুরু করার আগে, নিশ্চিত করুন যে আপনার পরিবেশ সঠিকভাবে সেটআপ হয়েছে নিম্নলিখিত ধাপগুলি অনুসরণ করে। এটি নিশ্চিত করবে সব ডিপেন্ডেন্সি ইনস্টল হয়েছে এবং API কী সঠিকভাবে কনফিগার করা আছে উন্নয়ন ও পরীক্ষার জন্য।

- পাইথন 3.8 বা উপরে
- SerpAPI API কী ([SerpAPI](https://serpapi.com/) এ সাইন আপ করুন - ফ্রি টিয়ার উপলব্ধ)

## ইনস্টলেশন

শুরু করতে, আপনার পরিবেশ সেটআপের জন্য নিম্নলিখিত ধাপগুলি অনুসরণ করুন:

1. uv (পরামর্শকৃত) বা pip ব্যবহার করে ডিপেন্ডেন্সি ইনস্টল করুন:

```bash
# ইউভি ব্যবহার করা হচ্ছে (প্রস্তাবিত)
uv pip install -r requirements.txt

# পাইপ ব্যবহার করা হচ্ছে
pip install -r requirements.txt
```

2. প্রজেক্ট মূল ডিরেক্টরিতে `.env` ফাইল তৈরি করুন এবং এতে আপনার SerpAPI কী লিখুন:

```
SERPAPI_KEY=your_serpapi_key_here
```

আপনি [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) এ সম্পূর্ণ বাস্তবায়ন দেখতে পারেন।

### সার্ভার চালানো

MCP সার্ভার শুরু করতে, নিচের কমান্ডটি ব্যবহার করুন:

```bash
python server.py
```

সার্ভার stdio-ভিত্তিক MCP সার্ভার হিসেবে চলবে যা ক্লায়েন্ট সরাসরি সংযুক্ত হতে পারবে।

### ক্লায়েন্ট মোড

ক্লায়েন্ট (`client.py`) MCP সার্ভারের সাথে ইন্টারঅ্যাক্ট করার জন্য দুটি মোড সাপোর্ট করে:

- **নর্মাল মোড**: স্বয়ংক্রিয় টেস্ট চালায় যা সব টুল পরীক্ষা করে ও তাদের প্রতিক্রিয়া যাচাই করে। এটি দ্রুত যাচাইয়ের জন্য সহায়ক।
- **ইন্টারেক্টিভ মোড**: মেনু-চালিত ইন্টারফেস শুরু করে যেখানে ম্যানুয়ালি টুল নির্বাচন, কল, কাস্টম ক্যোয়ারী প্রবেশ করানো এবং রিয়েল-টাইমে ফলাফল দেখা যায়। এটি সার্ভারের সক্ষমতা অনুসন্ধান ও বিভিন্ন ইনপুট পরীক্ষা করার জন্য আদর্শ।

আপনি [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) এ সম্পূর্ণ বাস্তবায়ন দেখতে পারেন।

### ক্লায়েন্ট চালানো

স্বয়ংক্রিয় টেস্ট চালাতে (এটি সার্ভার স্বয়ংক্রিয়ভাবে শুরু করবে):

### বিভিন্ন পদ্ধতিতে পরীক্ষা

আপনার প্রয়োজন এবং কাজের ধারা অনুযায়ী সার্ভারের টুলগুলো পরীক্ষা ও ব্যবহার করার বিভিন্ন উপায় রয়েছে।

#### MCP পাইথন SDK দিয়ে কাস্টম টেস্ট স্ক্রিপ্ট লেখা
আপনি MCP পাইথন SDK ব্যবহার করে নিজস্ব টেস্ট স্ক্রিপ্টও তৈরি করতে পারেন:

```python
from mcp import ClientSession, StdioServerParameters

async def test_custom_query():

    )

            result = await session.call_tool("general_search", 
                                           arguments={"query": "your custom query"})
            # ফলাফল প্রক্রিয়া করুন
```

---


এই পরিপ্রেক্ষিতে, "টেস্ট স্ক্রিপ্ট" বলতে বুঝায় একটি কাস্টম পাইথন প্রোগ্রাম যা MCP সার্ভারের জন্য ক্লায়েন্ট হিসেবে কাজ করে। এটি ফরমাল ইউনিট টেস্ট না হলেও, আপনি প্রোগ্রাম্যাটিক্যালি সার্ভারের সাথে সংযোগ স্থাপন, যেকোনো টুল প্যারামিটার সহ কল করা এবং ফলাফল বিশ্লেষণ করতে পারবেন। এই পদ্ধতি উপযোগী:
- টুল কল প্রোটোটাইপিং এবং পরীক্ষা চালানোর জন্য
- সার্ভারের বিভিন্ন ইনপুটে প্রতিক্রিয়া যাচাই করার জন্য
- টুল পুনরাবৃত্ত কল স্বয়ংক্রিয় করার জন্য
- MCP সার্ভারের ওপরে নিজস্ব কার্যপ্রবাহ বা ইন্টিগ্রেশন নির্মাণের জন্য

আপনি দ্রুত নতুন কুয়েরি পরীক্ষা, টুল আচরণ ডিবাগ করতে, অথবা বেশি উন্নত অটোমেশন শুরু করার জন্য এই স্ক্রিপ্ট ব্যবহার করতে পারেন। নিচে MCP পাইথন SDK দিয়ে এমন একটি স্ক্রিপ্ট তৈরির উদাহরণ দেওয়া হল:

## টুল বর্ণনা

সার্ভার প্রদত্ত নিচের টুলগুলো বিভিন্ন ধরনের অনুসন্ধান ও অনুরোধ সম্পাদনে ব্যবহৃত হয়। প্রতিটি টুলের প্যারামিটার এবং উদাহরণসহ বর্ণনা দেওয়া হয়েছে।


এই অংশে প্রতিটি উপলব্ধ টুল এবং তাদের প্যারামিটার বিস্তারিত দেওয়া হয়েছে।

### general_search

একটি সাধারণ ওয়েব অনুসন্ধান সম্পাদন করে এবং প্রক্রিয়াজাত ফলাফল প্রদান করে।

**কিভাবে এই টুল কল করবেন:**

আপনি MCP পাইথন SDK ব্যবহার করে আপনার নিজস্ব স্ক্রিপ্ট থেকে `general_search` কল করতে পারেন, অথবা ইন্টারেক্টিভলি ইন্সপেক্টর বা ইন্টারেক্টিভ ক্লায়েন্ট মোড থেকে করতে পারেন। এখানে SDK ব্যবহার উদাহরণ:

# [Python Example](#tab/python-general-search)

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_general_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "latest AI trends"})
            print(result)
```

---

বিকল্পভাবে, ইন্টারেক্টিভ মোডে মেনু থেকে `general_search` নির্বাচন করে অনুরোধ করা প্রশ্ন লিখুন।

## এই পাঠে উন্নত ধারণা

তৈরি শুরু করার আগে, এখানে কিছু গুরুত্বপূর্ণ উন্নত ধারণা রয়েছে যা এই অধ্যায়ে বারবার আসবে। এগুলো বোঝলে আপনি সহজে অনুসরণ করতে পারবেন, এমনকি নতুন হলে:

- **একাধিক টুল সমন্বয়**: এর অর্থ হল একক MCP সার্ভারে বিভিন্ন টুল (যেমন ওয়েব সার্চ, সংবাদ সার্চ, পণ্য সার্চ, প্রশ্নোত্তর) পরিচালনা করা। এটি আপনার সার্ভারকে একাধিক কাজের জন্য সক্ষম করে তোলে, শুধুমাত্র একটি কাজের জন্য নয়।
- **API রেট লিমিট হ্যান্ডলিং**: অনেক বাহ্যিক API (যেমন SerpAPI) কতবার অনুরোধ করা যাবে তা সীমিত করে। ভালো কোড এই সীমাগুলো পরীক্ষা করে এবং সুষ্ঠুভাবে পরিচালনা করে, যাতে আপনি সীমা পূর্ণ হলে অ্যাপ্লিকেশন ক্র্যাশ না করে।
- **কাঠামোবদ্ধ ডেটা পার্সিং**: API থেকে প্রায়শই জটিল এবং নেস্টেড রেসপন্স আসে। এই ধারণা হল সেগুলোকে পরিষ্কার ও ব্যবহারযোগ্য ফরম্যাটে রূপান্তর করা যা LLM বা অন্য প্রোগ্রামগুলোর জন্য সহজ।
- **ত্রুটি পুনরুদ্ধার**: মাঝে মাঝে সমস্যা হয়—যেমন নেটওয়ার্ক ব্যর্থ, অথবা API কাঙ্ক্ষিত তথ্য আনে না। ত্রুটি পুনরুদ্ধার মানে আপনার কোড এসব সমস্যা সামলাতে পারে এবং উপযোগী প্রতিক্রিয়া দেয়, ক্র্যাশ করে না।
- **প্যারামিটার যাচাই**: টুলগুলোর সব ইনপুট সঠিক ও নিরাপদ কিনা পরীক্ষা করা। এতে ডিফল্ট মান নির্ধারণ এবং টাইপ সঠিক আছে কি না যাচাই থাকে, যা বাগ ও বিভ্রান্তি রোধ করে।

এই অংশ আপনাকে Web Search MCP সার্ভার ব্যবহারের সময় সাধারণ সমস্যা শনাক্ত ও সমাধানে সাহায্য করবে। আপনি যদি কোনো ত্রুটি বা অপ্রত্যাশিত আচরণ দেখতে পান, তবে এই সমাধানগুলো পরীক্ষা করুন - এগুলো সাধারণত দ্রুত সমস্যার উৎসমাধান করে।

## সমস্যার সমাধান

Web Search MCP সার্ভারের সাথে কাজ করার সময় মাঝে মাঝে সমস্যা হতে পারে—এটি স্বাভাবিক, বিশেষত বাহ্যিক API ও নতুন টুল ব্যবহারকালে। এই অংশে সবচেয়ে সাধারণ সমস্যাগুলোর কার্যকর সমাধান দেওয়া হয়েছে যাতে আপনি দ্রুত ফিরে আসতে পারেন। কোনো ত্রুটির সম্মুখীন হলে প্রথমে এখানে দেখুন: নিচের টিপসগুলি সবচেয়ে বেশি ব্যবহৃত সমস্যাগুলোর সমাধান দেয় এবং প্রায়শই অতিরিক্ত সাহায্য ছাড়াই কাজ করে।

### সাধারণ সমস্যা

নীচে সর্বাধিক ঘন ঘন সমস্যাগুলোর তালিকা, স্পষ্ট ব্যাখ্যা এবং সমাধানের ধাপ উল্লেখ করা হয়েছে:

১. **.env ফাইলে SERPAPI_KEY অনুপস্থিত**
   - যদি দেখেন `SERPAPI_KEY environment variable not found` ত্রুটি, তার মানে আপনার অ্যাপ্লিকেশন SerpAPI অ্যাক্সেস করার জন্য প্রয়োজনীয় API কী খুঁজে পাচ্ছে না। এটি ঠিক করতে, আপনার প্রজেক্টের মূল ডিরেক্টরিতে `.env` নামের ফাইল তৈরি করুন (যদি না থাকে) এবং এতে একটি লাইন যোগ করুন যেমন `SERPAPI_KEY=your_serpapi_key_here`। অবশ্যই `your_serpapi_key_here` আপনার আসল SerpAPI কী দিয়ে প্রতিস্থাপন করবেন।

২. **মডিউল ফাউন্ড না হওয়ার ত্রুটি**
   - যেমন `ModuleNotFoundError: No module named 'httpx'` ত্রুটি নির্দেশ করে একটি প্রয়োজনীয় পাইথন প্যাকেজ নেই। সাধারণত এই সমস্যাটি হয় যখন সব ডিপেন্ডেন্সি ইনস্টল করা হয়নি। এই সমস্যা সমাধানে টার্মিনালে চালান `pip install -r requirements.txt` যা আপনার প্রজেক্টের সব প্রয়োজনীয় প্যাকেজ ইনস্টল করবে।

৩. **কানেকশন সমস্যা**
   - যেমন `Error during client execution` ত্রুটি মানে ক্লায়েন্ট সার্ভারের সাথে কানেক্ট করতে পারছে না, অথবা সার্ভার প্রত্যাশিতভাবে চলছে না। নিশ্চিত করুন ক্লায়েন্ট ও সার্ভার উভয় ভার্সন সঙ্গতিপূর্ণ, `server.py` ফাইল সঠিক ডিরেক্টরিতে আছে ও চলছে। সার্ভার ও ক্লায়েন্ট পুনরায় শুরু করাও সাহায্য করতে পারে।

৪. **SerpAPI ত্রুটি**
   - `Search API returned error status: 401` দেখালে বোঝায় আপনার SerpAPI কী অনুপস্থিত, ভুল বা মেয়াদোত্তীর্ণ। SerpAPI ড্যাশবোর্ডে যান, কী যাচাই করুন এবং আপনার `.env` ফাইল আপডেট করুন। কী ঠিক থাকলেও এই ত্রুটি দেখা দিলে দেখুন ফ্রি টিয়ারের কোটার পরিমাণ শেষ হয়েছে কি না।

### ডিবাগ মোড

সাধারণত অ্যাপ কেবল গুরুত্বপূর্ণ তথ্য লগ করে। আপনি যদি আরো বিস্তারিত দেখতে চান (যেমন জটিল সমস্যা নির্ণয়ের জন্য), তাহলে DEBUG মোড চালু করতে পারেন। এতে অ্যাপের প্রতিটি ধাপের আরো বিস্তারিত তথ্য দেখাবে।

**উদাহরণ: সাধারণ আউটপুট**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**উদাহরণ: DEBUG আউটপুট**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

লক্ষ্য করুন DEBUG মোডে HTTP অনুরোধ, প্রতিক্রিয়া এবং অন্যান্য অন্তর্নিহিত তথ্যের অতিরিক্ত লাইন যুক্ত হয়। এটি সমস্যার সমাধানে খুবই সহায়ক।

DEBUG মোড চালু করতে, আপনার `client.py` অথবা `server.py` ফাইলের শুরুর দিকে লগিং লেভেল DEBUG সেট করুন:

# [Python](#tab/python-debug)


```python
# আপনার client.py বা server.py এর শীর্ষে
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO থেকে DEBUG এ পরিবর্তন করুন
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
