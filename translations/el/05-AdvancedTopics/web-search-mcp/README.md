# Μάθημα: Δημιουργία ενός MCP Server Αναζήτησης Ιστού

Αυτό το κεφάλαιο δείχνει πώς να χτίσετε έναν πρακτικό πράκτορα AI που ενσωματώνεται με εξωτερικά APIs, χειρίζεται διάφορους τύπους δεδομένων, διαχειρίζεται σφάλματα και οργανώνει πολλαπλά εργαλεία — όλα σε μια μορφή έτοιμη για παραγωγή. Θα δείτε:

- **Ενσωμάτωση με εξωτερικά APIs που απαιτούν αυθεντικοποίηση**
- **Χειρισμό διαφόρων τύπων δεδομένων από πολλαπλά endpoints**
- **Αξιόπιστες στρατηγικές διαχείρισης σφαλμάτων και καταγραφής**
- **Ορχήστρωση πολλαπλών εργαλείων σε έναν μόνο server**

Στο τέλος, θα έχετε πρακτική εμπειρία με πρότυπα και βέλτιστες πρακτικές που είναι απαραίτητες για προηγμένες εφαρμογές AI και LLM.

## Εισαγωγή

Σε αυτό το μάθημα, θα μάθετε πώς να χτίσετε έναν προχωρημένο MCP server και πελάτη που επεκτείνουν τις δυνατότητες LLM με δεδομένα ιστού σε πραγματικό χρόνο χρησιμοποιώντας το SerpAPI. Αυτή είναι μια κρίσιμη δεξιότητα για την ανάπτυξη δυναμικών πρακτόρων AI που μπορούν να έχουν πρόσβαση σε ενημερωμένες πληροφορίες από το διαδίκτυο.

## Στόχοι Μάθησης

Στο τέλος αυτού του μαθήματος, θα μπορείτε να:

- Ενσωματώνετε εξωτερικά APIs (όπως το SerpAPI) με ασφάλεια σε έναν MCP server
- Υλοποιείτε πολλαπλά εργαλεία για αναζήτηση στο διαδίκτυο, ειδήσεων, προϊόντων και ερωταπαντήσεων
- Αναλύετε και μορφοποιείτε δομημένα δεδομένα για την κατανάλωση από LLM
- Διαχειρίζεστε σφάλματα και όρια ρυθμού API αποτελεσματικά
- Δημιουργείτε και δοκιμάζετε τόσο αυτοματοποιημένους όσο και διαδραστικούς MCP πελάτες

## MCP Server Αναζήτησης Ιστού

Αυτή η ενότητα παρουσιάζει την αρχιτεκτονική και τα χαρακτηριστικά του MCP Server Αναζήτησης Ιστού. Θα δείτε πώς το FastMCP και το SerpAPI χρησιμοποιούνται μαζί για να επεκτείνουν τις δυνατότητες LLM με δεδομένα στον ιστό σε πραγματικό χρόνο.

### Επισκόπηση

Αυτή η υλοποίηση διαθέτει τέσσερα εργαλεία που παρουσιάζουν την ικανότητα του MCP να χειρίζεται με ασφάλεια και αποτελεσματικότητα διάφορα εξωτερικά API-driven καθήκοντα:

- **general_search**: Για ευρείες αποτελέσματα ιστού
- **news_search**: Για πρόσφατους τίτλους ειδήσεων
- **product_search**: Για δεδομένα ηλεκτρονικού εμπορίου
- **qna**: Για αποσπάσματα ερωτήσεων και απαντήσεων

### Χαρακτηριστικά
- **Παραδείγματα Κώδικα**: Περιλαμβάνει μπλοκ κώδικα ειδικά για Python (και εύκολα επεκτάσιμα σε άλλες γλώσσες) χρησιμοποιώντας code pivots για σαφήνεια

### Python

```python
# Παράδειγμα χρήσης του εργαλείου γενικής_αναζήτησης
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

- **Διαδραστική λειτουργία**: Ξεκινά μια μενού-οδηγούμενη διεπαφή όπου μπορείτε να επιλέγετε και να καλείτε εργαλεία χειροκίνητα, να εισάγετε προσαρμοσμένα ερωτήματα και να βλέπετε τα αποτελέσματα σε πραγματικό χρόνο. Ιδανικό για εξερεύνηση των δυνατοτήτων του server και πειραματισμό με διαφορετικές εισόδους.

## Προχωρημένες Έννοιες σε Αυτό το Μάθημα

Πριν ξεκινήσετε την κατασκευή, εδώ είναι μερικές σημαντικές προχωρημένες έννοιες που θα εμφανιστούν σε όλο το κεφάλαιο. Η κατανόησή τους θα σας βοηθήσει να ακολουθήσετε, ακόμα κι αν είναι καινούργιες για εσάς:

- **Ορχήστρωση Πολλαπλών Εργαλείων**: Αυτό σημαίνει ότι τρέχετε αρκετά διαφορετικά εργαλεία (όπως αναζήτηση στον ιστό, αναζήτηση ειδήσεων, αναζήτηση προϊόντων και Q&A) μέσα σε έναν μόνο MCP server. Επιτρέπει στον server σας να χειρίζεται ποικιλία εργασιών, όχι μόνο μία.
- **Διαχείριση Ορίου Ρυθμού API**: Πολλά εξωτερικά APIs (όπως το SerpAPI) περιορίζουν πόσα αιτήματα μπορείτε να κάνετε μέσα σε ένα χρονικό διάστημα. Ο καλός κώδικας ελέγχει αυτά τα όρια και τα χειρίζεται εύστοχα, ώστε η εφαρμογή σας να μη σπάει αν φτάσετε σε όριο.
- **Ανάλυση Δομημένων Δεδομένων**: Οι απαντήσεις API συχνά είναι περίπλοκες και εμφωλευμένες. Αυτή η έννοια αφορά τη μετατροπή αυτών των απαντήσεων σε καθαρές, εύχρηστες μορφές που είναι φιλικές προς LLMs ή άλλες εφαρμογές.
- **Ανάκαμψη από Σφάλματα**: Μερικές φορές κάτι πάει στραβά — ίσως η σύνδεση δικτύου αποτύχει ή το API δεν επιστρέψει αυτά που περιμένετε. Η ανάκαμψη από σφάλματα σημαίνει ότι ο κώδικας σας μπορεί να χειριστεί αυτά τα προβλήματα και να δίνει χρήσιμα μηνύματα, αντί να καταρρέει.
- **Επικύρωση Παραμέτρων**: Αυτό αφορά τον έλεγχο ότι όλες οι είσοδοι στα εργαλεία σας είναι σωστές και ασφαλείς στη χρήση. Περιλαμβάνει τον καθορισμό προεπιλεγμένων τιμών και την εξασφάλιση ορθότητας τύπων, κάτι που βοηθά στην αποφυγή σφαλμάτων και σύγχυσης.

Αν προκύψει σφάλμα, ξεκινήστε από εδώ: οι παρακάτω συμβουλές αντιμετωπίζουν τα θέματα που βλέπει η πλειονότητα των χρηστών και συχνά επιλύουν το πρόβλημά σας χωρίς επιπλέον βοήθεια.

### Συνήθη Προβλήματα

Ακολουθούν μερικά από τα πιο συχνά προβλήματα που αντιμετωπίζουν οι χρήστες, μαζί με σαφείς εξηγήσεις και βήματα για την επίλυσή τους:

1. **Λείπει το SERPAPI_KEY στο αρχείο .env**
   - Αν δείτε το σφάλμα `SERPAPI_KEY environment variable not found`, σημαίνει ότι η εφαρμογή σας δεν μπορεί να βρει το κλειδί API που χρειάζεται για να έχει πρόσβαση στο SerpAPI. Για να το διορθώσετε, δημιουργήστε ένα αρχείο με όνομα `.env` στη ρίζα του έργου σας (αν δεν υπάρχει ήδη) και προσθέστε μια γραμμή όπως `SERPAPI_KEY=το_κλειδί_σας_από_το_serpapi`. Φροντίστε να αντικαταστήσετε `το_κλειδί_σας_από_το_serpapi` με το πραγματικό κλειδί από την ιστοσελίδα του SerpAPI.

2. **Σφάλματα τύπου module not found**
   - Σφάλματα όπως `ModuleNotFoundError: No module named 'httpx'` δείχνουν ότι λείπει κάποιο απαιτούμενο πακέτο Python. Αυτό συμβαίνει συνήθως αν δεν έχετε εγκαταστήσει όλες τις εξαρτήσεις. Για να το διορθώσετε, τρέξτε την εντολή `pip install -r requirements.txt` στο τερματικό σας για να εγκαταστήσετε ό,τι χρειάζεται το έργο.

3. **Προβλήματα σύνδεσης**
   - Αν εμφανιστεί σφάλμα όπως `Error during client execution`, συχνά σημαίνει ότι ο πελάτης δεν μπορεί να συνδεθεί με τον server, ή ο server δεν τρέχει όπως αναμένεται. Ελέγξτε ότι και ο πελάτης και ο server είναι συμβατές εκδόσεις, και ότι το `server.py` υπάρχει και τρέχει στο σωστό φάκελο. Η επανεκκίνηση τόσο του server όσο και του πελάτη μπορεί επίσης να βοηθήσει.

4. **Σφάλματα SerpAPI**
   - Το μήνυμα `Search API returned error status: 401` σημαίνει ότι το κλειδί SerpAPI λείπει, είναι λανθασμένο ή έχει λήξει. Μπείτε στον πίνακα ελέγχου του SerpAPI, επαληθεύστε το κλειδί σας, και ενημερώστε το αρχείο `.env` αν χρειάζεται. Αν το κλειδί είναι σωστό αλλά βλέπετε ακόμα το σφάλμα, ελέγξτε αν το δωρεάν επίπεδο έχει εξαντληθεί.

### Λειτουργία Εντοπισμού Σφαλμάτων (Debug Mode)

Από προεπιλογή, η εφαρμογή καταγράφει μόνο σημαντικές πληροφορίες. Αν θέλετε να δείτε περισσότερες λεπτομέρειες για το τι συμβαίνει (π.χ., για να διαγνώσετε δύσκολα προβλήματα), μπορείτε να ενεργοποιήσετε τη λειτουργία DEBUG. Αυτή θα σας δείξει πολλές περισσότερες πληροφορίες για κάθε βήμα που κάνει η εφαρμογή.

**Παράδειγμα: Κανονική Έξοδος**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**Παράδειγμα: Έξοδος DEBUG**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

Παρατηρήστε πως η λειτουργία DEBUG περιλαμβάνει επιπλέον γραμμές σχετικά με αιτήματα HTTP, απαντήσεις και άλλες εσωτερικές λεπτομέρειες. Αυτό μπορεί να είναι πολύ χρήσιμο για την αντιμετώπιση προβλημάτων.

Για να ενεργοποιήσετε τη λειτουργία DEBUG, ορίστε το επίπεδο καταγραφής σε DEBUG στην κορυφή του `client.py` ή `server.py`:

# [Python](#tab/python-debug)


```python
# Στην κορυφή του client.py ή server.py σας
import logging
logging.basicConfig(
    level=logging.DEBUG,  # Αλλάξτε από INFO σε DEBUG
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
