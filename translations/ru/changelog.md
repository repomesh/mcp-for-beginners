### Усиление безопасности и исправление уязвимостей зависимостей

Выполнен полный аудит безопасности всех манифестов зависимостей и исходного кода примеров, затем устранены все выявленные предупреждения npm и одна проблема на уровне кода. После исправления `npm audit` сообщает о **0 уязвимостях** во всех проверенных директориях.

#### Уязвимости зависимостей npm (транзитивные) — исправлены

Произведён аудит всех 15 зафиксированных файлов `package-lock.json`. Уязвимости ограничивались транзитивными зависимостями, добавляемыми инструментом разработчика MCP Inspector, клиентом OpenAI и MCP SDK; все теперь устранены без нарушения работы примеров:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** и **lab3/code/weather_mcp/inspector**: Повышена версия `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), что устранило предупреждения по встроенным пакетам `ajv`, `brace-expansion`, `diff`, `path-to-regexp` и `ws`. Добавлена запись npm `overrides`, принудительно использующая исправленный `shell-quote@1.8.4` для устранения оставшегося критического предупреждения от `concurrently`; пересозданы оба lock-файла (теперь 0 уязвимостей)
- **03-GettingStarted/samples/typescript**: `npm audit fix` обновил транзитивный пакет `qs` (средний уровень) до исправленной версии
- **03-GettingStarted/samples/javascript**: `npm audit fix` обновил транзитивный пакет `hono` (средний уровень) до исправленной версии
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` обновил транзитивный пакет `form-data` (высокий уровень) до исправленной версии
- **03-GettingStarted/11-simple-auth/solution/typescript**: Сгенерирован отсутствующий файл `package-lock.json`, чтобы проект стал воспроизводимым и проверяемым (0 уязвимостей)

#### Исправление безопасности на уровне кода (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: Удалён параметр `shell=True` из инструмента `open_in_vscode`. Предыдущий вызов `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` позволял интерпретировать метасимволы оболочки в пути к папке через `cmd.exe` (вектор инъекции команд). Теперь запускается напрямую разрешённый `Code.exe` с папкой в качестве аргумента — без оболочки — что функционально эквивалентно и безопасно

#### Аудит зависимостей Python

- Произведён аудит каждого набора требований Python с помощью `pip-audit`. В `05-AdvancedTopics` и `03-GettingStarted/samples/python` не обнаружено известных уязвимостей (их диапазоны версий `mcp` / `httpx` / `pydantic` / `python-dotenv` разрешаются к текущим исправленным релизам)
- **09-CaseStudy/docs-mcp/solution/python/requirements.txt**: `pip-audit` выявил транзитивную зависимость **`werkzeug` 3.1.1** с тремя предупреждениями отказа в обслуживании (DoS) Windows device-name через `safe_join` — `CVE-2025-66221`, `CVE-2026-21860` и `CVE-2026-27199` (все исправлены в версии 3.1.6). Добавлен явный пин безопасности `werkzeug>=3.1.6`, чтобы разрешение выбирало исправленный релиз; проверено, что ограничение корректно разрешается в стеке `chainlit` / `mcp` / `semantic-kernel`
