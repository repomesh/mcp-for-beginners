# หัวข้อขั้นสูงใน MCP

[![Advanced MCP: Secure, Scalable, and Multi-modal AI Agents](../../../translated_images/th/06.42259eaf91fccfc6.webp)](https://youtu.be/4yjmGvJzYdY)

_(คลิกที่รูปภาพด้านบนเพื่อดูวิดีโอของบทเรียนนี้)_

บทนี้ครอบคลุมหัวข้อขั้นสูงหลายประการในการใช้งาน Model Context Protocol (MCP) รวมถึงการรวมระบบมัลติโมดอล, ความสามารถในการขยายระบบ, แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัย และการบูรณาการในองค์กร หัวข้อเหล่านี้มีความสำคัญสำหรับการสร้างแอปพลิเคชัน MCP ที่มั่นคงและพร้อมใช้งานในสภาพแวดล้อมการผลิตที่สามารถตอบสนองความต้องการของระบบ AI สมัยใหม่

## ภาพรวม

บทเรียนนี้สำรวจแนวคิดขั้นสูงในการใช้งาน Model Context Protocol โดยมุ่งเน้นการรวมระบบมัลติโมดอล, ความสามารถในการขยายระบบ, แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัย และการบูรณาการในองค์กร หัวข้อเหล่านี้จำเป็นสำหรับการสร้างแอป MCP ที่มีคุณภาพสำหรับการผลิตที่สามารถจัดการกับความต้องการที่ซับซ้อนในสภาพแวดล้อมองค์กร

> **มองไปข้างหน้า:** หัวข้อหลายรายการด้านล่างได้รับผลกระทบจากรุ่นผู้ทดสอบการเผยแพร่สเปค MCP วันที่ `2026-07-28` — Root Contexts (5.4) และ Sampling (5.6) สร้างบน primitives ที่รุ่นผู้ทดสอบการเผยแพร่จัดว่าเลิกใช้งานแล้ว และฟีเจอร์ Tasks ทดลองที่อ้างอิงใน Protocol Features (5.16) ย้ายไปยังส่วนขยาย Tasks เฉพาะ ดูเพิ่มเติมที่ [What's Changing in MCP: The 2026-07-28 Release Candidate](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md)

## วัตถุประสงค์การเรียนรู้

เมื่อจบบทเรียนนี้ คุณจะสามารถ:

- ใช้งานความสามารถมัลติโมดอลภายในเฟรมเวิร์ก MCP
- ออกแบบสถาปัตยกรรม MCP ที่สามารถขยายได้สำหรับสถานการณ์ความต้องการสูง
- ใช้แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยที่สอดคล้องกับหลักการความปลอดภัยของ MCP
- บูรณาการ MCP กับระบบ AI ขององค์กรและเฟรมเวิร์ก
- ปรับปรุงประสิทธิภาพและความน่าเชื่อถือในสภาพแวดล้อมการผลิต

## ประเด็นสำคัญที่ควรจำ

- การใช้งาน MCP แบบมัลติโมดอลขยายความสามารถ AI เกินกว่าการประมวลผลข้อความ
- ความสามารถในการขยายระบบจำเป็นสำหรับการใช้งานในองค์กรและสามารถจัดการได้ผ่านการขยายแนวนอนและแนวตั้ง
- มาตรการความปลอดภัยที่ครอบคลุมช่วยปกป้องข้อมูลและรักษาควบคุมการเข้าถึงอย่างเหมาะสม
- การรวมระบบองค์กรกับแพลตฟอร์มอย่าง Azure OpenAI และ Microsoft AI Foundry ช่วยเพิ่มขีดความสามารถของ MCP
- การใช้งาน MCP ขั้นสูงได้รับประโยชน์จากสถาปัตยกรรมที่ปรับให้เหมาะสมและการจัดการทรัพยากรอย่างรอบคอบ

## แบบฝึกหัด

ออกแบบการใช้งาน MCP ที่มีคุณภาพสำหรับองค์กรสำหรับกรณีใช้งานเฉพาะ:

1. ระบุความต้องการมัลติโมดอลสำหรับกรณีใช้งานของคุณ
2. ร่างการควบคุมความปลอดภัยที่จำเป็นเพื่อปกป้องข้อมูลที่ละเอียดอ่อน
3. ออกแบบสถาปัตยกรรมที่สามารถขยายได้ซึ่งรองรับโหลดที่แตกต่างกัน
4. วางแผนจุดบูรณาการกับระบบ AI ขององค์กร
5. จัดทำเอกสารกำหนดปัญหาคอขวดของประสิทธิภาพที่อาจเกิดขึ้นและกลยุทธ์การบรรเทาปัญหา
