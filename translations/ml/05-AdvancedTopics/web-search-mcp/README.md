# പാഠം: ഒരു വെബ് സെർച്ച് MCP സെർവർ നിർമ്മിക്കൽ

ഈ അധ്യായം എങ്ങനെ യഥാർത്ഥ ലോക AI ഏജന്റിനെ നിർമ്മിക്കാമെന്ന് കാണിക്കുന്നു, അതിൽ ബാഹ്യ API-കളുമായി സംയോജനം, വൈവിധ്യമാർന്ന ഡാറ്റാ തരം കൈകാര്യം ചെയ്യൽ, പിശകുകൾ നിയന്ത്രിക്കൽ, ഒറ്റ സെർവറിൽ ഒരു гораздо ലളിതമായ സംവിധാനങ്ങളുണ്ട്—all ഉത്പാദനത്തിന് തയ്യാറുളള രൂപത്തിൽ. നിങ്ങൾ കണ്ട് എടുക്കും:

- **Authentication ആവശ്യമായ ബാഹ്യ API-കളുമായി സംയോജനം**
- **പല എപ്പോൾ ക്കളിൽ നിന്നുള്ള വൈവിധ്യമാർന്ന ഡാറ്റാ തരം കൈകാര്യം ചെയ്യൽ**
- **ഹൃദയമായ പിശക് കൈകാര്യം ചെയ്യൽ കൂടാതെ ലോഗിംഗ് തന്ത്രങ്ങൾ**
- **ഒറ്റ സെർവറിൽ മൾട്ടി-ടൂൾ ഓർക്കസ്‌ട്രേഷൻ**

അവസാനം, വികസിത AI-യും LLM-ഓട്ടൊമേറ്റഡ് അപ്ലിക്കേഷനുകൾക്കും അത്യന്താപേക്ഷിതമായ മാതൃകകളും മികച്ച പ്രവർത്തനങ്ങളും നിങ്ങൾക്ക് പ്രായോഗിക പരിചയം ഉണ്ടായിരിക്കും.

## പരിചയം

ഈ പാഠത്തിൽ, SerpAPI ഉപയോഗിച്ച് യഥാർത്ഥ സമയ വെബ് ഡാറ്റ ഉപയോഗിച്ച് LLM കഴിവുകൾ വിപുലീകരിക്കുന്ന ഒരു ഉയർന്ന MCP സെർവർ, ക്ലയന്റ് നിർമ്മിക്കുന്നതിൻറെ രീതി പഠിക്കും. 이는 വെബിൽ നിന്നുള്ള تازى വിവരങ്ങൾ ലഭ്യമാക്കുന്ന ഡൈനാമിക് AI ഏജന്റുകൾ വികസിപ്പിക്കാൻ നിർണായകമായ കഴിവാണ്.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം പൂർത്തിയാക്കുമ്പോൾ, നിങ്ങൾക്ക് കഴിയും:

- SerpAPI പോലുള്ള ബാഹ്യ API-കൾ സുരക്ഷയോടെ MCP സെർവറിൽ സംയോജിപ്പിക്കുക
- വെബ്, വാർത്ത, ഉൽപ്പന്ന ശോധന, Q&A‌യ്ക്ക് വേണ്ടി പല ടൂളുകളും നടപ്പിലാക്കുക
- LLM ഉപഭോഗത്തിനായി ഘടിത ഡാറ്റ പാഴ്‌സ് ചെയ്യുക, ഫോർമാറ്റ് ചെയ്യുക
- പിശകുകൾ കൈകാര്യം ചെയ്യുക, API നിരക്ക് പരിധികൾ കാര്യക്ഷമമായി നിയന്ത്രിക്കുക
- ഓട്ടോമേറ്റഡ്, ഇന്ററാക്ടീവ് MCP ക്ലയന്റുകൾ നിർമ്മിച്ച് പരീക്ഷിക്കുക

## വെബ് സെർച്ച MCP സെർവർ

ഈ വിഭാഗം വെബ് സെർച്ച് MCP സെർവറിന്റെ ശില്പശാസ്ത്രവും സവിശേഷതകളും പരിചയപ്പെടുന്നു. FastMCP, SerpAPI എന്നിവ LLM കഴിവുകൾ യഥാർത്ഥ സമയ വെബ് ഡാറ്റയോടെ വിപുലീകരിക്കാൻ എങ്ങനെ ചേർന്ന് പ്രവർത്തിക്കുന്നു എന്ന് കാണിക്കും.

### അവലോകനം

ഈ നടപ്പാക്കൽ MCPയുടെ ബാഹ്യ API-നിർദേശിത വൈവിധ്യമാർന്ന ടാസ്‌കുകൾ സുരക്ഷിതവും കാര്യക്ഷമവുമായ håndle ചെയാനുള്ള ശേഷിക്കുന്ന നാല് ടൂളുകൾ സവിശേഷിക്കുന്നു:

- **general_search**: വ്യാപകമായ വെബ് ഫലങ്ങൾക്കായി
- **news_search**: പുതിയ തലക്കെട്ടുകൾക്കായി
- **product_search**: ഇ-കൊമേഴ്സ് ഡാറ്റക്കായി
- **qna**: ചോദ്യോത്തര ചെറിയ ഭാഗങ്ങൾക്കായി

### സവിശേഷതകൾ
- **കോട് ഉദാഹരണങ്ങൾ**: പൈത്തൺ ഉൾപ്പെടെ ഭാഷാ-നിർദ്ദിഷ്ട കോഡ് ബ്ലോക്കുകൾ ഉൾക്കൊള്ളുന്നു (മറ്റ് ഭാഷകളിലേക്ക് എളുപ്പത്തിൽ വിപുലീകരിക്കാൻ കഴിയും) വ്യക്തതയ്‌ക്കായി കോഡ് പിവോട്ടുകൾ ഉപയോഗിക്കുന്നു

### പൈത്തൺ

```python
# പൊതുവായ_തിരയൽ ഉപകരണം ഉപയോഗിക്കുന്ന ഉദാഹരണം
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

ക്ലയന്റ് ഓടിക്കുന്നതിന് മുൻപ്, സെർവർ എന്ത് ചെയ്യുന്നതാണ് എന്ന് മനസിലാക്കുന്നത് സഹായകമാണ്. [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ഫയൽ MCP സെർവറിനെ നടപ്പിലാക്കുന്നു, SerpAPI സംയോജിപ്പിച്ച് വെബ്, വാർത്ത, ഉൽപ്പന്ന ശോധന, Q&A ടൂളുകൾ കിഴിവ് ചെയ്യും. ഇത് വരുന്ന അഭ്യർത്ഥനകൾ കൈകാര്യം ചെയ്യുന്നു, API കോൾസ് നിയന്ത്രിക്കുന്നു, മറുപടികൾ പാഴ്‌സ് ചെയ്യുന്നു, ഘടിത ഫലങ്ങൾ ക്ലയന്റിലേക്ക് തിരിച്ച നൽകുന്നു.

നിങ്ങൾ മുഴുവൻ നടപ്പാക്കൽ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ൽ റിവ്യൂ ചെയ്യാം.

സെർവർ എങ്ങനെ ടൂൾ നിർവ്വചിച്ച് രജിസ്റ്റർ ചെയ്യുന്നതിന്റെ ലഘു ഉദാഹരണം ഇവിടെ:

### പൈത്തൺ സെർവർ

```python
# server.py (ഉദ്ധരണി)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...പ്രവർത്തനം...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **ബാഹ്യ API സംയോജനം**: API കീകൾ സുരക്ഷിതമായി കൈകാര്യം ചെയ്യുന്നതും ബാഹ്യ അഭ്യർത്ഥനകളും പ്രദർശിപ്പിക്കുന്നു
- **ഘടിത ഡാറ്റ പാഴ്‌സിംഗ്**: API മറുപടികളെ LLM സൗഹൃദ ഫോർമാറ്റിലേക്ക് മാറ്റുന്നത് കാണിക്കുന്നു
- **പിശക് കൈകാര്യം ചെയ്യൽ**: അനുയോജ്യമായ ലോഗിംഗ് സഹിതം ശക്തമായ പിശക് കൈകാര്യം ചെയ്യൽ
- **ഇന്ററാക്റ്റീവ് ക്ലയന്റ്**: ഓട്ടോമേറ്റഡ് ടെസ്റ്റുകളും ഇന്ററാക്റ്റീവ് മോഡും ഉൾപ്പെടുന്നു
- **സന്ദർഭം മാനേജ്മെന്റ്**: അഭ്യർത്ഥനകൾ ലോഗ് ചെയ്യാനും ട്രാക്ക് ചെയ്യാനും MCP Context ഉപയോഗിക്കുന്നു

## മുൻപരിചയങ്ങൾ

ആരംഭിക്കുന്നതിന് മുമ്പ്, നിങ്ങളുടെ പരിസരം ശരിയായി സജ്ജമാണെന്ന് ഉറപ്പാക്കുക. ഇത് എല്ലാ ആശ്രിതോപാധികളുമുണ്ടെന്നും API കീകൾ ശരിയായി ക്രമീകരിച്ചിട്ടുണ്ടെന്നും ഉറപ്പ് തരുന്നതാണ്.

- പൈത്തൺ 3.8 അല്ലെങ്കിൽ അതിൽക്കൂടി ഉയർന്ന വേർഷൻ
- SerpAPI API കീ ([SerpAPI](https://serpapi.com/) ൽ അംഗീകരിക്കാം - സൗജന്യ നിരപ്പ് ലഭ്യമാണ്)

## ഇൻസ്റ്റലേഷൻ

തുടക്കമാക്കാൻ, നിങ്ങളുടെ പരിസ്ഥിതി സജ്ജീകരിക്കാൻ ഈ ചുവടുകൾ പിന്തുടരുക:

1. uv (ശിപാർശചെയ്തത്) അല്ലെങ്കിൽ pip ഉപയോഗിച്ച് ആശ്രിതോപാധികൾ ഇൻസ്റ്റാൾ ചെയ്യുക:

```bash
# uv ഉപയോഗിക്കുക (ശുപാർശ ചെയ്‌തിരിക്കുന്നത്)
uv pip install -r requirements.txt

# পിപ് ഉപയോഗിക്കുന്നു
pip install -r requirements.txt
```

2. പ്രോജക്ട് റൂട്ടിൽ `.env` ഫയൽ സൃഷ്ടിച്ച് നിങ്ങളുടെ SerpAPI കീ ചേർക്കുക:

```
SERPAPI_KEY=your_serpapi_key_here
```

## ഉപയോഗം

വെബ് സെർച്ച് MCP സെർവർ SerpAPI സംയോജിപ്പിച്ച് വെബ്, വാർത്ത, ഉൽപ്പന്ന ശോധനം, Q&A ടൂളുകൾ തുറന്നിട്ടുള്ള പ്രധാന ഘടകമാണ്. ഇത് വരുമ്പോഴുള്ള അഭ്യർത്ഥനകൾ കൈകാര്യം ചെയ്യുന്നു, API കോൾസ് മാനേജു ചെയ്യുന്നു, മറുപടി പാഴ്‌സ് ചെയ്യുന്നു, ഘടിത ഫലങ്ങൾ ക്ലയന്റിലേക്ക് നൽകുന്നു.

നിങ്ങൾ മുഴുവൻ നടപ്പാക്കൽ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ൽ റിവ്യൂ ചെയ്യാം.

### സെർവർ ഓടിക്കൽ

MCP സെർവർ ആരംഭിക്കാൻ, താഴെ കാണുന്ന കമാൻഡ് ഉപയോഗിക്കുക:

```bash
python server.py
```

സെർവർ stdio പണിത MCP സെർവറായി പ്രവർത്തിക്കും, ക്ലയന്റ് നേരിട്ട് ബന്ധപ്പെടാൻ കഴിയും.

### ക്ലയന്റ് മോഡുകൾ

MCP സെർവറുമായി ഇടപഴകാൻ ക്ലയന്റ് (`client.py`) രണ്ട് മോഡുകൾ പിന്തുടരുന്നു:

- **സാധാരണ മോഡ്**: എല്ലാ ടൂളുകളും പ്രവർത്തിച്ച് മറുപടികൾ ശരിയെന്ന് പരിശോധിക്കുന്ന ഓട്ടോമേറ്റഡ് ടെസ്റ്റുകൾ ഓടിക്കുന്നു. സെർവർ, ടൂളുകൾ സാധൂകരിക്കുന്നുണ്ടെന്ന് ലഘുവിൽ പരിശോധിക്കാൻ ഇതു സഹായകമാണ്.
- **ഇന്ററാക്ടീവ് മോഡ്**: ഉപയോക്താവ് മെയ്‌നുവിൽ നിന്ന് ടൂളുകൾ തിരഞ്ഞെടുക്കാനും, ഇഷ്ടാനുസൃത ചോദ്യങ്ങൾ നൽകാനും, ഫലങ്ങൾ യഥാർത്ഥ സമയത്തിൽ കാണാനും കഴിയുന്ന ഇന്ററാക്ടീവ് ഇന്റർഫേസ് ആരംഭിക്കുന്നു. സെർവറിന്റെ കഴിവുകൾ അന്വേഷിക്കാനും വ്യത്യസ്ത ഇൻപുട്ടുകൾ പരീക്ഷിക്കാനും ഇത് അനുയോജ്യമാണ്.

നിങ്ങൾ മുഴുവൻ നടപ്പാക്കൽ [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) ൽ റിവ്യൂ ചെയ്യാം.

### ക്ലയന്റ് ഓടിക്കൽ

ഓട്ടോമേറ്റഡ് ടെസ്റ്റുകൾ പ്രവർത്തിപ്പിക്കാൻ (ഇത് സ്വയം സെർവർ ആരംഭിക്കും):

```bash
python client.py
```

അല്ലെങ്കിൽ ഇന്ററാക്ടീവ് മോഡിൽ ഓടിക്കുക:


### വ്യത്യസ്ത രീതികളിലൂടെ പരീക്ഷണം

സെർവർ നൽകിയ ടൂളുകൾ പരീക്ഷിക്കാനും ഇടപഴകാനുമുള്ള വിവിധ മാർഗങ്ങൾ ഉണ്ട്, നിങ്ങളുടെ ആവശ്യങ്ങൾക്കും പ്രവൃത്തി ശൈലിയ്ക്കും അനുസരിച്ച്.

MCP പൈത്തൺ SDK ഉപയോഗിച്ച് നിങ്ങളുടെ സ്വന്തമായ ടെസ്റ്റ് സ്ക്രിപ്റ്റുകൾ നിർമ്മിക്കാനും സാധിക്കും:

- ടൂൾ കോൾസിന്റെ പ്രോട്ടോട്ടൈപ്പിംഗ്, പരീക്ഷണം
- സെർവർ വ്യത്യസ്ത ഇൻപുട്ടുകൾക്ക് എങ്ങനെ പ്രതികരിക്കുന്നു എന്ന് ശരിയെന്ന് ഉറപ്പാക്കുക
- ആവർത്തിച്ചു ടൂൾ ഉപയോഗം ഓട്ടോമേറ്റുചെയ്യുക
- MCP സെർവറിനോട് മുകളിൽ നിങ്ങളുടെ സ്വന്തം വർക്ക്‌ഫ്ലോകൾ അല്ലെങ്കിൽ സംയോജിതങ്ങൾ നിർമിക്കുക

ടെസ്റ്റ് സ്ക്രിപ്റ്റുകൾ ഉപയോഗിച്ച് പുതിയ ചോദ്യങ്ങൾ പെട്ടെന്ന് പരീക്ഷിക്കാനോ, ടൂൾ പ്രവർത്തനം ഡീബഗ് ചെയ്‌യാനോ, അല്ലെങ്കിൽ കൂടുതൽ വികസിത ഓട്ടോമേഷൻ ആരംഭിക്കാനോ കഴിയും. താഴെ MCP പൈത്തൺ SDK ഉപയോഗിച്ച് അത്തരം ഒരു സ്ക്രിപ്റ്റ് എങ്ങനെ നിർമ്മിക്കാമെന്ന് ഉദാഹരണം നൽകിയിരിക്കുന്നു:

## ടൂൾ വിവരണങ്ങള്‍

സെർവർ നൽകിയ വിവിധ തരത്തിലുള്ള ശോധനകളും ചോദ്യങ്ങളും നിർവഹിക്കാൻ താഴെ കാണുന്ന ടൂളുകൾ ഉപയോഗിക്കാം. ഓരോ ടൂളിന്റെയും പാരാമീറ്ററുകളും ഉദാഹരണ ഉപയോഗവും താഴെ വിശദീകരിച്ചിരിക്കുന്നു.

## ഈ പാഠത്തിലെ ഉയർന്ന ആശയങ്ങൾ

നിർമ്മിക്കാൻ തുടങ്ങിയതിന് മുൻപ്, ഈ അധ്യായത്തിൽ ആവർത്തിച്ച് കാണപ്പെടുന്ന ചില ഉയർന്ന ആശയങ്ങൾ താഴെ പറയുന്നവയാണ്. ഇവ മനസിലാക്കുക നിങ്ങളുടെ പാഠം പിന്തുടരാൻ സഹായിക്കും, നിങ്ങൾ പുതിയവനാകുമ്പോഴും:

- **മൾട്ടി-ടൂൾ ഓർക്കസ്‌ട്രേഷൻ**: വെബ് സെർച്ച്, വാർത്ത ശോധനം, ഉൽപ്പന്ന ശോധനം, Q&A പോലുള്ള പല ടൂളുകളും ഒറ്റ MCP സെർവറിൽ പ്രവർത്തിക്കുന്നത്. ഇത് നിങ്ങളുടെ സെർവർ വ്യത്യസ്ത പ്രവർത്തനങ്ങൾ കൈകാര്യം ചെയ്യാൻ കഴിയും എന്നർത്ഥം.
- **API നിരക്ക് പരിധി കൈകാര്യം ചെയ്യൽ**: SerpAPI പോലുള്ള പല ബാഹ്യ API-കളും ഒരു നിർദ്ധിഷ്ട സമയത്തിനുള്ള അഭ്യർത്ഥന എണ്ണം നിയന്ത്രിക്കുന്നു. നല്ല കോഡ് ഈ പരിധികൾ പരിശോധിക്കുകയും, അതിക്രമിക്കാതിരിക്കാനും ഉചിതമായി കൈകാര്യം ചെയ്യുകയും ചെയ്യുന്നു, ആപ്പ് തകരാറിലേക്ക് പോകാതിരിക്കാൻ.
- **ഘടിത ഡാറ്റ പാഴ്‌സിംഗ്**: API മറുപടികൾ اکثر कठिन, നെട്ടിൽ ആയവയാണ്. ഈ ആശയം അവരെ LLMs അല്ലെങ്കിൽ മറ്റ് പ്രോഗ്രാമുകൾക്ക് സൗഹൃദമായ ഒരു ലളിതമായഫോർമാറ്റുകളിലേക്ക് മാറ്റുന്നതിനു సంబంధించినതാണ്.
- **പിശക് വീണ്ടെടുക്കൽ**: ചിലപ്പോൾ പ്രശ്നങ്ങൾ ഉണ്ടാകാം—ഉദാഹരണത്തിന് നെറ്റ്‌വർക്ക് പരാജയം, API പ്രതീക്ഷിച്ച പ്രദാനം നൽകാത്തത്. പിശക് വീണ്ടെടുക്കൽ നിങ്ങളുടെ കോഡ് ഈ പ്രശ്നങ്ങൾ കൈകാര്യം ചെയ്യാനും, തകരാറില്ലാതെ ഉപയോഗപ്രദമായ ഫീഡ്ബാക്ക് നൽകാനും കഴിയും എന്നർത്ഥം.
- **പാരാമീറ്റർ ശരിയായതായുണ്ടെന്ന് പരിശോധന**: നിങ്ങളുടെ ടൂളുകളിലേക്ക് നൽകുന്ന എല്ലാ ഇൻപുട്ടുകളും ശരിയാണെന്നും സുരക്ഷിതമായതാണെന്നും പരിശോധിക്കുന്നത്. ഇതിൽ മുൻ-നിർണ്ണയ മൂല്യങ്ങൾ നല്കാനും, ടൈപ്പുകൾ ശരിയാണെന്ന് ഉറപ്പുവരുത്താനും ഉൾപ്പെടുന്നു, ഇത് പിശകുകൾ തടയാനും ഒപ്പം ആശയക്കുഴപ്പം കുറയ്ക്കാനും സഹായിക്കുന്നു.

ഈ വിഭാഗം നിങ്ങളെ വെബ് സെർച്ച് MCP സെർവറുമായി പ്രവർത്തിക്കുമ്പോൾ നേരിടാവുന്ന സാധാരണ പ്രശ്നങ്ങൾ കണ്ടെത്താനും പരിഹരിക്കാനും സഹായിക്കും. നിങ്ങൾ പിശകുകൾക്കോ പ്രതീക്ഷിച്ചില്ലാത്ത പെരുമാറ്റത്തിനോ അഭിമുഖീകരിച്ചാൽ, ഈ പരിഹാരങ്ങൾ പൊതുവായി പ്രശ്നങ്ങൾ വേഗത്തിൽ പരിഹരിക്കുന്നു. കൂടുതൽ സഹായം തേടുന്നതിനു മുമ്പെങ്കിലും ഇവ പരിശോധിക്കുക.

## പ്രശ്നപരിഹാരങ്ങള്‍

വെബ് സെർച്ച് MCP സെർവറിനെ ഉപയോഗിക്കുമ്പോൾ ചിലപ്പോൾ പ്രശ്നങ്ങൾ ഉണ്ടാകാം—ഇത് ബാഹ്യ API-കൾക്കും പുതിയ ടൂളുകൾക്കും ഒപ്പം വികസിപ്പിക്കുമ്പോൾ സാധാരണമാണ്. ഈ വിഭാഗം സാധാരണ പ്രശ്നങ്ങൾക്ക് ഉപജീവ്യ പരിഹാരങ്ങൾ നൽകുന്നു, ഹ്രസ്വ സമയത്തിനുള്ളിൽ പുനഃസ്ഥാപനത്തിന് സഹായിക്കാനും. നിങ്ങൾ പിശക് അനുഭവിച്ചാൽ, ഇവിടം തുടങ്ങുക: താഴെ കാണിക്കുന്ന ഉപദേശം പല ഉപയോക്താക്കളും നേരിടുന്ന പ്രശ്നങ്ങൾ പരിഹരിക്കുന്നു, അധിക സഹായം വേണ്ടാതിരിക്കാൻ.

### സാധാരണ പ്രശ്നങ്ങൾ

താഴെ ഏറ്റവും സാധാരണമായി ഉപയോക്താക്കൾ നേരിടുന്ന ചില പ്രശ്നങ്ങളും വിശദീകരണങ്ങളും പരിഹാര നടപടിക്രമങ്ങളും കൊടുത്തിട്ടുണ്ട്:

1. **.env ഫയലിൽ SERPAPI_KEY ഇല്ലാത്തത്**
   - `SERPAPI_KEY environment variable not found` എന്ന പിശക് കാണിച്ചാൽ, നിങ്ങളുടേയൊരു API കീ SerpAPI-യിൽ പ്രവേശിക്കാൻ ലഭ്യമല്ല എന്ന് ഉദ്ദേശിക്കുന്നു. ഇത് പരിഹരിക്കാൻ, നിങ്ങളുടെ പ്രോജക്ട് റൂട്ടിൽ `.env` ഫയൽ ഉണ്ടാക്കുക (ഇത്രേ വരെ ഇല്ലെങ്കിൽ) അടിപൊളം വരി ചേർക്കുക: `SERPAPI_KEY=your_serpapi_key_here`. `your_serpapi_key_here`-നെ SerpAPI വെബ്സൈറ്റിൽ നിന്നുള്ള യഥാർത്ഥ കീ എന്നിങ്ങനെ മാറ്റുക.

2. **മോഡ്യൂൾ ലാഭ്യമല്ലാത്ത പിശകുകൾ**
   - `ModuleNotFoundError: No module named 'httpx'` പോലുള്ള പിശകുകൾ ആവശ്യമായ പൈത്തൺ പാക്കേജ് ഇല്ലാത്തതായി സൂചിപ്പിക്കുന്നു. സാധാരണയായി നിങ്ങൾ എല്ലാ ആശ്രിതോപാധികളും ഇൻസ്റ്റാൾ ചെയ്തിട്ടില്ലെങ്കിൽ ഇത് സംഭവിക്കും. ഇത് പരിഹരിക്കാൻ, നിങ്ങളുടെ ടർമിനലിൽ `pip install -r requirements.txt` ഓടിച്ചു എല്ലാ ആവശ്യകതകളും ഇൻസ്റ്റാൾ ചെയ്യുക.

3. **കണക്ഷൻ പ്രശ്നങ്ങൾ**
   - `Error during client execution` പോലുള്ള പിശകുകൾ കാണുമ്പോൾ സാധാരണയായി ക്ലയന്റ് സെർവറുമായി കണക്റ്റ് ചെയ്യാൻ കഴിയാത്തത് അല്ലെങ്കിൽ സെർവർ പ്രതീക്ഷിക്കപ്പെട്ടുപോലെ പ്രവർത്തിക്കാത്തതാകാം. ക്ലയന്റും സെർവറും യോജിച്ച വേർഷനുകളാണെന്ന് ഉറപ്പാക്കുക, ടയ്യും `server.py` ശരിയായ ഡയറക്ടറിയിൽ നിലകൊള്ളുകയും പ്രവർത്തിക്കുകയും ചെയ്യുന്നത് പരിശോധിക്കുക. സെർവറും ക്ലയന്റും പുനരാരംഭിക്കുന്നതും സഹായകമാണ്.

4. **SerpAPI പിശകുകൾ**
   - `Search API returned error status: 401` കാണുന്നത് SerpAPI കീ ഇല്ലാതെയോ തെറ്റായതോ കാലഹരണപ്പെട്ടതോ ആണെന്ന് സൂചിപ്പിക്കുന്നു. നിങ്ങളുടെ SerpAPI ഡാഷ്ബോർഡിലേക്ക് പോകുക, കീ ശരിയാണെന്ന് ഉറപ്പാക്കുക, ആവശ്യമായാൽ `.env` ഫയൽ അപ്ഡേറ്റ് ചെയ്യുക. കീ ശരിയെങ്കിലും ഈ പിശക് വരുന്നുവെങ്കിൽ, നിങ്ങളുടെ സൗജന്യ നിരപ്പിലെ കൂറ്റായുള്ള ക്വോട്ട പൂർത്തിയായിട്ടുണ്ടെന്ന് പരിശോധിക്കുക.

### ഡീബഗ് മോഡ്

സാധാരണയായി ആപ്പ് പ്രധാനപ്പെട്ട വിവരങ്ങൾ മാത്രമേ ലോഗ് ചെയ്യൂ. എന്താണ് നടക്കുന്നത് കൂടുതൽ വിശദമായി കാണാൻ (ജടിലമായ പ്രശ്‌നങ്ങൾ കണ്ടെത്താൻ), നിങ്ങൾക്ക് DEBUG മോഡ് सक्षम ചെയ്യാം. ഇത് ആപ്പ് എടുക്കുന്ന ഓരോ ഘട്ടത്തെക്കുറിച്ച് കൂടുതൽ വിവരങ്ങൾ കാണിക്കും.

**ഉദാഹരണം: സാധാരണ ഔട്ട്പുട്ട്**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**ഉദാഹരണം: DEBUG ഔട്ട്പുട്ട്**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

DEBUG മോഡ് HTTP അഭ്യർത്ഥനകൾ, മറുപടികൾ, മറ്റ് അന്തർഭാഗ വിവരങ്ങൾ സംബന്ധിച്ച അധിക വരികളും ഉൾക്കൊള്ളിക്കുന്നുവെന്ന് ശ്രദ്ധിക്കുക. ഇതാണ് പ്രശ്നപരിഹാരത്തിന് വളരെ സഹായകമായത്.

DEBUG മോഡ് സജ്ജമാക്കാൻ, നിങ്ങളുടെ `client.py` അല്ലെങ്കിൽ `server.py`-യുടെ മുകളിൽ ലോഗിംഗ് ലെവൽ DEBUG ആയി ക്രമീകരിക്കുക:

# [Python](#tab/python-debug)


```python
# നിങ്ങളുടെ client.py അല്ലെങ്കിൽ server.py ൽ മുകളിൽ
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO എന്നതിൽ നിന്ന് DEBUG ആയി മാറ്റുക
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
