# MCP സുരക്ഷ: AI സിസ്റ്റങ്ങൾക്കുള്ള സമഗ്രരക്ഷ

[![MCP Security Best Practices](../../../translated_images/ml/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(ഈ പാഠത്തിന്റെ വീഡിയോ കാണാൻ മുകളിൽ ചിത്രത്തിൽ ക്ലിക്കുചെയ്യുക)_

സുരക്ഷ AI സിസ്റ്റം ഡിസൈനിന്റെ അടിസ്ഥാനസ്തംഭമാണിത്, അതുകൊണ്ടാണ് ഇത് നമ്മുടെ രണ്ടാം വകുപ്പ് മുൻതൂക്കം നൽകുന്നത്. ഇത് Microsoftന്റെ [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) ൽ നിന്നുള്ള **Secure by Design** സിദ്ധാന്തത്തോടൊപ്പമാണ് പൊരുത്തപ്പെടുന്നത്.

മോഡൽ കോൺടെക്സ്റ്റ് പ്രോട്ടോക്കോൾ (MCP) ശക്തമായ പുതിയ ശേഷികളെ AI-നടത്തിപ്പ്രവർത്തിത ആപ്ലിക്കേഷനുകളിൽ കൊണ്ടുവരികയാണ് ചെയ്യുന്നത്, എന്നാൽ ഇതു പരമ്പരാഗത സോഫ്റ്റ്‌വയർ അപകടങ്ങളെക്കാൾ വ്യത്യസ്തമായ പ്രത്യേക സുരക്ഷാ വെല്ലുവിളികൾ സൃഷ്ടിക്കുന്നു. MCP സിസ്റ്റങ്ങൾ നിലവിലുള്ള സുരക്ഷാ പ്രശ്നങ്ങൾ (സുരക്ഷിത കോഡിംഗ്, കുറഞ്ഞ അവകാശങ്ങൾ, സപ്ലൈ ചെയിൻ സുരക്ഷ) കൂടാതെ പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ, ടൂൾ വിഷപ്പെടുത്തൽ, സെഷൻ ഹിജാക്കിംഗ്, കൺഫ്യൂസ്ഡ് ഡെപ്യൂട്ടി ആക്രമണങ്ങൾ, ടോക്കൺ പാസ്ത്രൂ വീഴ്ചകൾ, ഡൈനാമിക് ശേഷി മാറ്റം എന്നിവയുൾപ്പെടെയുള്ള പുതിയ AI-നിത്യമായ ഭീഷണികളെ അഭിമുഖീകരിക്കുന്നു.

ഈ പാഠത്തിൽ MCP നടപ്പാക്കലുകളിൽ ഏറ്റവും ഗുരുതരമായ സുരക്ഷാ അപകടങ്ങൾ പരിശോധിക്കുന്നു — അംഗീകാരം, അവകാശ നിയന്ത്രണം, 지나친 അനുമതികൾ, അപരോക്ഷ പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ, സെഷൻ സുരക്ഷ, കൺഫ്യൂസ്ഡ് ഡെപ്യൂട്ടി പ്രശ്നങ്ങൾ, ടോക്കൺ മാനേജ്മെന്റ്, സപ്ലൈ ചെയിൻ വീഴ്ചകൾ എന്നിവ ഉൾപ്പെടെ. ഈ അപകടങ്ങൾ കുറയ്ക്കാൻ ആവിഷ്ക്കരിക്കാവുന്ന നിയന്ത്രണങ്ങളും മികച്ച ആചാരങ്ങളും നിങ്ങൾക്ക് പഠിക്കാനും Microsoft പരിഹാരങ്ങൾ Prompt Shields, Azure Content Safety, GitHub Advanced Security എന്നിവ ഉപയോഗിച്ച് MCP നടപ്പാക്കൽ ഊർജ്ജിതമാക്കാനും സാധിക്കും.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം അവസാനിക്കുന്നതിന് മുകളിൽ, നിങ്ങൾക്കുയുക്തമാകും:

- **MCP-പ്രത്യേക ഭീഷണികളെ തിരിച്ചറിയുക**: MCP സിസ്റ്റങ്ങളിൽ ഉള്ള പ്രത്യേക സുരക്ഷാ അപകടങ്ങൾ തിരിച്ചറിഞ്ഞ് പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ, ടൂൾ വിഷപ്പെടുത്തൽ, 지나친 അനുമതികൾ, സെഷൻ ഹിജാക്കിംഗ്, കൺഫ്യൂസ്ഡ് ഡെപ്യൂട്ടി പ്രശ്നങ്ങൾ, ടോക്കൺ പാസ്ത്രൂ വീഴ്ചകൾ, സപ്ലൈ ചെയിൻ അപകടങ്ങൾ എന്നിവയെ ഉൾപ്പെടുത്തുക
- **സുരക്ഷാ നിയന്ത്രണങ്ങൾ പ്രയോഗിക്കുക**: ശക്തമായ അംഗീകാരം, കുറഞ്ഞ അവകാശ പദവി ആക്സസ്, സുരക്ഷിത ടോക്കൺ മാനേജ്മെന്റ്, സെഷൻ സുരക്ഷാ നിയന്ത്രണങ്ങൾ, സപ്ലൈ ചെയിൻ ശരിവരുത്തൽ എന്നിവയുൾപ്പെടെ ഫലപ്രദമായ പ്രതിരോധങ്ങൾ നടപ്പിലാക്കുക
- **Microsoft സുരക്ഷാ പരിഹാരങ്ങൾ ഉപയോഗിക്കുക**: MCP പ്രവർത്തനസംരക്ഷണത്തിനായി Microsoft Prompt Shields, Azure Content Safety, GitHub Advanced Security അറിയുകയും വിനിയോഗിക്കുകയും ചെയ്യുക
- **ടൂൾ സുരക്ഷ സാധൂകരിക്കുക**: ടൂൾ മെറ്റാഡാറ്റ സ്ഥിരീകരണം, ഡൈനാമിക് മാറ്റങ്ങൾ നിരീക്ഷണം, അപരോക്ഷ പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ ആക്രമണങ്ങൾ പ്രതിരോധിക്കുന്നതിന്റെ പ്രാധാന്യം തിരിച്ചറിയുക
- **മികച്ച ആചാരങ്ങൾ സംയോജിപ്പിക്കുക**: സ്ഥിരതയേറിയ സുരക്ഷാ അടിസ്ഥാനങ്ങള് (സുരക്ഷിത കോഡിംഗ്, സെർവർ ഹാർഡനിംഗ്, സീറോ ട്രസ്റ്റ്) MCP-പ്രത്യേക നിയന്ത്രണങ്ങളോടൊപ്പം സംയുക്തമാക്കുക സമഗ്രരക്ഷയ്ക്കായി

# MCP സുരക്ഷ ആർക്കിടെക്ചർ & നിയന്ത്രണങ്ങൾ

ആധുനിക MCP നടപ്പാക്കലുകൾ പരമ്പരാഗത സോഫ്റ്റ്‌വെയർ സുരക്ഷയും AI-പ്രത്യേക ഭീഷണികളും പരിഗണിക്കുന്ന പാളിമുറുക്കപ്പെട്ട സുരക്ഷാപദ്ധതികൾ ആവശ്യമാണ്. ദ്രുതഗതിയിലുള്ള MCP സ്പെസിഫിക്കേഷൻ അതിന്റെ സുരക്ഷാ നിയന്ത്രണങ്ങൾ മികവാർത്ത നൽകിക്കൊണ്ടിരിക്കുന്നു, ഇത് എന്റർപ്രൈസ് സുരക്ഷാ ആർക്കിടെക്ചറുകളുമായി ഒത്തുചേരലും നിലവിലുള്ള മികച്ച ആചാരങ്ങളും എളുപ്പമാക്കുന്നു.

[Microsoft Digital Defense Report](https://aka.ms/mddr) ൽ നിന്നുള്ള ഗവേഷണം കാണിക്കുന്നു **98% റിപ്പോർട്ട് ചെയ്ത ഭേദങ്ങൾ ശക്തമായ സുരക്ഷാ ശുദ്ധികരണത്തിന്റെ ഉപയോ​ഗത്തോടെ തടയപ്പെടുമെന്ന്**. ഏറ്റവും ഫലപ്രദമായ സംരക്ഷണ തന്ത്രം അടിസ്ഥാനസുരക്ഷാ രീതികളെയും MCP-പ്രത്യേക നിയന്ത്രണങ്ങളെയും സംയോജിപ്പിക്കലാണ് — തെളിയിക്കപ്പെട്ട അടിസ്ഥാന സുരക്ഷാ നടപടികൾ ശേഷിച്ചതിൽ സുരക്ഷാ അപകടങ്ങൾ കുറക്കുന്നതിൽ ഏറ്റവും സ്വാധീനമുള്ളവയാണ്.

## നിലവിലുള്ള സുരക്ഷാ പ്രസ്ഥിതി

> **കുറിപ്പ്:** ഈ വിവരം MCP സുരക്ഷാ മാനദണ്ഡങ്ങളെ **ഫെബ്രുവരി 5, 2026** നു നിലവിലുള്ള രീതിയിൽ പ്രതിഫലിപ്പിക്കുന്നു, **MCP Specification 2025-11-25** നെ അനുസരിച്ച്. MCP പ്രോട്ടോക്കോൾ വേഗത്തിൽ വികസിക്കുന്നു, ഭാവിയിലെ നടപ്പാക്കലുകൾ പുതിയ അംഗീകാരം രീതികളും മെച്ചപ്പെട്ട നിയന്ത്രണങ്ങളും കൊണ്ടു വരാം. ഏറ്റവും പുതിയ മാർഗ്ഗനിർദ്ദേശങ്ങൾക്ക് ഉദ്ദേശിച്ച് നിലവിലെ [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), [സുരക്ഷാ മികച്ച ആചാര ഡോക്യുമെന്റേഷൻ](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) എന്നിവ കാണുക.

> **ഭാവിയിൽ:** `2026-07-28` റിലീസ് കാൻഡിഡേറ്റ് അംഗീകാരം കൂടുതൽ ശക്തമാക്കുന്നു — ക്ലയന്റുകൾ `iss` പാരാമീറ്റർ അംഗീകാരം പ്രതീക്ഷയിൽ പരിശോധിക്കണം (RFC 9207), ഡൈനാമിക് ക്ലയന്റ് റജിസ്ട്രേഷനിൽ ഒരു OpenID Connect `application_type` ഡിക്ലയർ ചെയ്യണം, രജിസ്റ്റർ ചെയ്ത ക്രെഡൻഷ്യലുകൾ പ്രദാനംചെയ്യുന്ന അംഗീകാരം സർവറുമായി ബന്ധിപ്പിക്കണം. അംഗീകാര SEPs പൂർണ്ണമായറിയാൻ [What's Changing in MCP: The 2026-07-28 Release Candidate](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) കാണുക.

## 🏔️ MCP Security Summit വർക്ക്‌ഷോപ്പ് (ഷെർപ)

**പ്രായോഗിക സുരക്ഷാ പരിശീലനത്തിന്**, **MCP Security Summit Workshop** (ഷെർപ) വളരെ മാർഗ്ഗനിർദ്ദേശം നൽകുന്നു - Microsoft Azure-യിൽ MCP സെർവറുകൾ സുരക്ഷിതമാക്കാനുള്ള സമഗ്ര മാർഗ്ഗനിർദേശം.

### വർക്ക്‌ഷോപ്പ് അവലോകനം

[MCP Security Summit Workshop](https://azure-samples.github.io/sherpa/) പ്രായോഗികവും പ്രവർത്തനക്ഷമവുമായ സുരക്ഷാ പരിശീലനം ഉറപ്പാക്കുന്നു, ഇത് "ദുർബല → ആക്രമിക്കുക → പുനരുദ്ധരിക്കുക → സാധൂകരിക്കുക" എന്ന പരിച്ഛേദിത രീതി പിന്തുടരുന്നു. നിങ്ങൾ:

- **വിഷയങ്ങൾ തകരാറിലാക്കിയാണ് പഠിക്കുക**: മനസ്സുകൊണ്ട് സുരക്ഷിതമല്ലാത്ത സെർവറുകൾ ആക്രമിച്ച് ദുർബലതകളിൽ അഭ്യാസം നേടുക
- **Azure-നാതിവ് സുരക്ഷ ഉപയോഗിക്കുക**: Azure Entra ID, Key Vault, API Management, AI Content Safety എന്നിവ ഉപയോഗിക്കുക
- **പ്രതിരോധം-കഴിവ് തുടരെ ഉപയോഗിക്കുക**: ക്യാംപുകൾ വഴി പുരോഗമിച്ച് സമഗ്രസുരക്ഷാ പാളികൾ നിർമ്മിക്കുക
- **OWASP മാനദണ്ഡങ്ങൾ പാലിക്കുക**: എല്ലാ സാങ്കേതിക വിദ്യകളും [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) നെ അനുസരിക്കുന്നു
- **പ്രോഡക്ഷൻ കോഡ് കൈവരിക്കുക**: പ്രവർത്തിക്കുന്ന, പരീക്ഷിച്ച നടപ്പാക്കലുകൾ പ്രാപിക്കുക

### MCP അംഗീകാരം പരിണാമം

MCP സ്പെസിഫിക്കേഷൻ അംഗീകാരം, അവകാശാനുമതി സമീപനത്തിൽ ശ്രദ്ധേയമായി മാറിയിട്ടുണ്ട്:

- **മൂല സമയ മാർഗം**: തുടക്കത്തിലെ സ്പെസിഫിക്കേഷനുകൾ വികസിപ്പകർ ആയി കസ്റ്റം അംഗീകാര സെർവറുകൾ നിർമ്മിച്ചു, MCP സെർവറുകൾ നേരിട്ട് OAuth 2.0 അംഗീകാരം സർവറുകളുടെ ജോലി ചെയ്തിരുന്നത്
- **हाल का നിലവിലെ മാനദണ്ഡം (2025-11-25)**: MCP സെർവറുകൾ പുറത്തിറങ്ങിയ ഐഡന്റിറ്റി ദാതാക്കളെ (ഉദാ: Microsoft Entra ID) അംഗീകാരം ഡെലഗേറ്റ് ചെയ്യാൻ അംഗീകാരം നൽകുന്നു, സുരക്ഷ നിലനിർത്തുകയും പ്രവർത്തന സങ്കീർണത കുറയ്ക്കുകയും ചെയ്യുന്നു
- **ട്രാൻസ്പോർട്ട് ലെയർ സുരക്ഷ**: പ്രാദേശിക (STDIO) & റിമോട്ട് (Streamable HTTP) കണക്ഷനുകൾക്ക് ശരിയായ അംഗീകാര മോഡലുകളോടെ സുരക്ഷിത ട്രാൻസ്പോർട്ട് മെക്കാനിസങ്ങൾ വർദ്ധിപ്പിച്ചു

## അംഗീകാരം & അവകാശാനുമതി സുരക്ഷ

### നിലവിലെ സുരക്ഷാ വെല്ലുവിളികൾ

ആധുനിക MCP നടപ്പാക്കലുകൾ ചില അംഗീകാരം, അവകാശാനുമതി വെല്ലുവിളികൾ നേരിടുന്നു:

### അപകടങ്ങൾ & ഭീഷണി വക്രങ്ങൾ

- **അവകാശാനുമതി ലಾಜിക് തെറ്റായ രൂപകൽപ്പന**: MCP സെർവറുകളിൽ തെറ്റായ അവകാശാനുമതി നടപ്പാക്കൽ രഹസ്യ ഡാറ്റ എല്ലാവർക്കും ലഭ്യമാക്കുകയും അനിയന്ത്രിത ആക്സസ് നിയന്ത്രണങ്ങൾ പ്രയോഗിക്കുകയും ചെയ്യാം.
- **OAuth ടോക്കൺ കവർച്ച**: ലോക്കൽ MCP സെർവർ ടോക്കൺ മോഷണം ആക്രമකරികളെ സെർവർ കൈകാര്യം ചെയ്യായി ഓർമ്മപ്പെടുത്തുകയും താഴത്തെ സേവനങ്ങൾക്ക് ആക്സസ് നൽകുകയും ചെയ്യുന്നു
- **ടോക്കൺ പാസ്ത്രൂ വീഴ്ചകൾ**: തെറ്റായ ടോക്കൺ കൈകാര്യം ഒളികൂടൽ നിയന്ത്രണങ്ങൾ മറികടക്കുകയും ഉത്തരവാദിത്വം നഷ്ടപ്പെടുകയും ചെയ്യുന്നു
- **അതിरेकമായ അനുമതികൾ**: അധികാനുമതിയുള്ള MCP സെർവർ കുറവ് അവകാശത്തിന്റെ ഘടകനിയമങ്ങൾ ലംഘിച്ച് ആക്രമണ സാധ്യതകൾ വർദ്ധിപ്പിക്കുന്നു

#### ടോക്കൺ പാസ്ത്രൂ: ഗുരുതരമായ പൊരുത്തവില്ലായ്മ

നിലവിലെ MCP അംഗീകാരം സ്പെസിഫിക്കേഷനുമായി പ്രകാരം **ടോക്കൺ പാസ്ത്രൂ തെളിവുകൾ വിലക്കപ്പെട്ടിരിക്കുന്നു**, അതിന്റെ ഗുരുതര സുരക്ഷാ ഫലപ്രാപ്തികളാൽ:

##### സുരക്ഷാ നിയന്ത്രണം മറികടക്കൽ
- MCP സെർവറുകളും താഴെത്തരം APIകളും നിർണ്ണായക സുരക്ഷാ നിയന്ത്രണങ്ങൾ നടപ്പാക്കുന്നു (രേറ്റു ലിമിറ്റിംഗ്, അഭ്യർത്ഥന സ്ഥിരീകരണം, ട്രാഫിക് നിരീക്ഷണം) ടോക്കൺ ശരിയായ പരിശോധനയിൽ ആശ്രയിച്ചിരിക്കുന്നു
- നേരിട്ടുള്ള ക്ലയന്റ്-ടു-API ടോക്കൺ ഉപയോഗം ഈ പ്രധാന സംരക്ഷണം മറികടക്കുന്നു, സുരക്ഷാ ആർക്കിടെക്ചർ ദുർബലം ആക്കുന്നു

##### ഉത്തരവാദിത്വവും ഓഡിറ്റ് വെല്ലുവിളികളും
- MCP സെർവറുകൾ തങ്ങളുടെ ഇടപെടൽക്കാർ ഉപയോഗിക്കുന്ന ടോക്കണുകൾ തമ്മിൽ വേർതിരിക്കാൻ കഴിയില്ല, ഓഡിറ്റ് ട്രെയ്ൽ തകരാറിലാക്കുന്നു
- താഴെത്തരം റിസോഴ്‌സ് സെർവർ ലോഗുകൾ MCP സെർവർ ഇടനിലക്കാരെ മറികടന്ന് അഭ്യർത്ഥനയുടെ തെറ്റായ ഉറവിടങ്ങൾ കാണിക്കുന്നു
- സംഭവ പരിശോധനയും അനുസരണം ഓഡിറ്റും വളരെ പ്രയാസമാകുന്നു

##### ഡാറ്റാ പുറത്തെടുക്കൽ അപകടങ്ങൾ
- പരിശോധനയില്ലാത്ത ടോക്കൺ അവകാശങ്ങൾ മോഷണ ടോക്കണുകൾകൊണ്ടുള്ള ദുഷ്ഛക്രവാളികൾ MCP സെർവറുകൾ പ്രോക്സികൾ ആയി ഉപയോഗിച്ച് ഡാറ്റ പുറത്തെടുക്കൽ നടത്തുന്നു
- വിശ്വാസ പരിധി ലംഘനങ്ങൾ അനധികൃത ആക്സസ് മാതൃകകൾ അനുവദിക്കുന്നു, ലക്ഷ്യമിട്ട സുരക്ഷ നിയന്ത്രണങ്ങൾ മറികടക്കുന്നു

##### ബഹുസേവന ആക്രമണ വഴികൾ
- പല സേവനങ്ങളിലേക്കും അംഗീകരിച്ച ടോക്കണുകൾ ലാറ്ററൽ മൂവ്മെന്റ് അനുവദിച്ച് ബന്ധിച്ചായുള്ള സിസ്റ്റങ്ങളിൽ വ്യാപനത്തിന് വഴി തുറക്കുന്നു
- ടോക്കൺ ഉറവിടം ശരിയായി സ്ഥിരീകരിക്കാനാവാത്തപ്പോൾ സേവനങ്ങൾ തമ്മിലുള്ള വിശ്വാസ ധാരണകൾ ദുർബലപ്പെടുത്തപ്പെടാം

### സുരക്ഷാ നിയന്ത്രണങ്ങൾ & പ്രതിരോധങ്ങൾ

**അത്യാവശ്യ സുരക്ഷാ ആവശ്യങ്ങൾ:**

> **അനിവാര്യമാണ്**: MCP സെർവറുകൾ സ്വന്തമായി MCP സെർവർക്കായി സുതാര്യമായി നൽകിയതല്ലാത്ത ടോക്കണുകൾ **സ്വീകരിക്കരുത്**

#### അംഗീകാരം & അവകാശാനുമതി നിയന്ത്രണങ്ങൾ

- **കഠിനമായ അവകാശാനുമതി അവലംബം**: MCP സെർവർ അവകാശാനുമതി ലാജിക് സമഗ്രമായി ഓഡിറ്റ് ചെയ്ത് സംവേദ്യങ്ങളായ ഉപയോക്താക്കളിലേക്കും ക്ലയന്റുകളിലേക്കും മാത്രമേ സ്രവൺ ലഭ്യമാക്കാൻ പാടുള്ളൂ എന്ന് ഉറപ്പാക്കുക
  - **നടപ്പാക്കൽ മാർഗ്ഗനിർദേശം**: [MCP സെർവറുകളുടെ അംഗീകാരം ഗേറ്റ്വേ ആയി Azure API Management](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ഐഡന്റിറ്റി സംയോജനം**: [MCP സെർവർ അംഗീകാരം Microsoft Entra ID ഉപയോഗിച്ച്](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **സുരക്ഷിത ടോക്കൺ മാനേജ്മെന്റ്**: [Microsoft-ന്റെ ടോക്കൺ പരിശോധനയും ജീവിതചക്ര മികച്ച ആചാരങ്ങളും](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) നടപ്പിലാക്കുക
  - ടോക്കൺ ആഡിയൻസ് അവകാശങ്ങൾ MCP സെർവർ ഐഡന്റിറ്റിക്കൊപ്പം പൊരുത്തപ്പെട്ടു എന്ന് സ്ഥിരീകരിക്കുക
  - ശരിയായ ടോക്കൺ റോട്ടേഷൻ, കാലാവധി നയം നടപ്പാക്കുക
  - ടോക്കൺ റിപ്ലേ ആക്രമണവും അനധികൃത ഉപയോഗവും തടയുക

- **സംരക്ഷിത ടോക്കൺ സംഭരണം**: വിശ്രമത്തിന്റെയും ട്രാൻസിറ്റിന്റെയുംEncryption ഉപയോഗിച്ച് ടോക്കണുകൾ സുരക്ഷിതമായി സൂക്ഷിക്കുക
  - **മികച്ച ആചാരങ്ങൾ**: [സുരക്ഷിത ടോക്കൺ സംഭരണം, എൻക്രിപ്ഷൻ മാർഗ്ഗനിർദേശങ്ങൾ](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### ആക്സസ് നിയന്ത്രണ നടപ്പാക്കൽ

- **എന്തിന് ഏറ്റവും കുറഞ്ഞ അവകാശം**: MCP സെർവറുകൾ ആവശ്യമായ ഏറ്റവും കുറഞ്ഞ അനുമതികളേ നൽകുക
  - അനുമതി അവലോകനവും അപ്ഡേറ്റുകളും പതിവായി നടത്തുക
  - **Microsoft ഡോക്യുമെന്റേഷൻ**: [സുരക്ഷിത കുറഞ്ഞ അവകാശ ആക്സസ്](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **റോൾ ബേസ്‌ഡ് ആക്സസ് കണ്ട്രോൾ (RBAC)**: സൂക്ഷ്മമായി തസ്തിക വ്യവസ്ഥകൾ സജ്ജമാക്കുക
  - വ്യക്തമായ റിസോഴ്‌സുകളും പ്രവർത്തനങ്ങളും മാത്രം റോളുകൾക്ക് അനുവദിക്കുക
  - കൂടുതൽ അല്ലെങ്കിൽ അനാവശ്യ അനുമതികൾ നൽകാതിരിക്കുക, ഇത് ആക്രമണ സാധ്യതകൾ കൂട്ടിക്കും

- **തുടർച്ചയായ അനുമതി നിരീക്ഷണം**: ആക്സസ് ഓഡിറ്റ്, നിരീക്ഷണം നേരത്തേ നടപ്പാക്കുക
  - അനുമതി ഉപയോഗ പാറ്റേണുകളിൽ അസാധാരണങ്ങൾ കണ്ടെത്തുക
  - അധികമോ ഉപയോഗ ചെയ്യാത്ത അവകാശങ്ങൾ വേഗം പരിഹരിക്കുക

## AI-പ്രത്യേക സുരക്ഷ ഭീഷണികൾ

### പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ & ടൂൾ മാനിപ്പുലേഷൻ ആക്രമണങ്ങൾ

ആധുനിക MCP നടപ്പാക്കലുകൾ പരമ്പരാഗത സുരക്ഷാ നടപടികൾ സമ്പൂർണമായി പരിഹരിക്കാനിടയായെല്ലാത്ത സങ്കീർണ AI-പ്രത്യേക ആക്രമണ വഴികൾ നേരിടുന്നു:

#### **അപരോക്ഷ പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ (ക്രോസ്-ഡൊമെയിൻ പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ)**

**അപരോക്ഷ പ്രൊംപ്‌റ്റ് ഇൻജെക്ഷൻ** MCP സജ്ജമാക്കിയ AI സിസ്റ്റങ്ങളിൽ ഏറ്റവും ഗുരുതരമായ ദുർബലതകളിൽ ഒന്നാണ്. ആക്രമകർ ദുഷ്പ്രേരിത നിർദ്ദേശങ്ങൾ പുറത്തുള്ള ഉള്ളടക്കങ്ങളിൽ (ഡോക്യുമെന്റുകൾ, വെബ് പേജുകൾ, ഇമെയിലുകൾ, ഡാറ്റാ ഉറവിടങ്ങൾ) സ്വരൂപിക്കുകയും അതിനെ AI സിസ്റ്റങ്ങൾ യഥാർത്ഥ കമാൻഡുകളായി പ്രക്രിയ ചെയ്യും.

**ആക്രമണാവസ്ഥകൾ:**
- **ഡോക്യുമെന്റ് അടിസ്ഥാന പ്രൊംപ്‌റ്റ്:** പ്രോസസ് ചെയ്ത ഡോക്യുമെന്റിൽ മറച്ചിരിക്കുന്ന ദुष്‌പ്രേരിത നിർദ്ദേശങ്ങൾ AI യെ അനിഷ്ടമായ പ്രവർത്തനങ്ങൾക്ക് നയിക്കുന്നു
- **വെബ് ഉള്ളടക്കം ദുരുപയോഗം:** പിടിച്ചെടുത്ത വെബ് പേജുകൾ പ്രൊംപ്‌റ്റുകളോടെ എമ്പെഡ് ചെയ്യുന്നുണ്ടാക്കി AI പെരുമാറ്റം മാത്രമാക്കുന്നു
- **ഇമെയിൽ-അടിസ്ഥാനത്തെ ആക്രമണങ്ങൾ:** AI അസിസ്റ്റന്റുകൾക്ക് വിവരങ്ങൾ ചോർത്ത് നൽകുകയോ അനധികൃത പ്രവർത്തനങ്ങൾ ചെയ്യുകയോ ചെയ്യുന്ന ദുഷ്പ്രേരിത പ്രൊംപ്‌റ്റുകൾ ഉള്ള ഇമെയിലുകൾ
- **ഡാറ്റ ഉറവിടം മലിനീകരണം:** MCP സിസ്റ്റങ്ങളിലെ ഡാറ്റാവിനെ ആഘാതം ചെലുത്തുന്ന ദുഷ്പ്രേരിത ഡേറ്റാബേസുകൾ അല്ലെങ്കിൽ APIകൾ

**യാഥാർത്ഥ്യ പ്രതിഫലം**: ഈ ആക്രമണങ്ങൾ ഡാറ്റാ പുറത്തെടുക്കൽ, സ്വകാര്യത ലംഘനം, ഹാനികരമായ ഉള്ളടക്കം സൃഷ്ടിക്കൽ, ഉപയോക്തൃ ഇന്ററാക്ഷനുകൾ മാനിപ്പുലേറ്റ് ചെയ്യൽ എന്നിവക്ക് കാരണമാകാം. വിശദമായ വിശകലനത്തിന് [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) കാണുക.

![Prompt Injection Attack Diagram](../../../translated_images/ml/prompt-injection.ed9fbfde297ca877.webp)

#### **ടൂൾ വിഷപ്പെടുത്തൽ ആക്രമണങ്ങൾ**

**ടൂൾ വിഷപ്പെടുത്തി** MCP ടൂളുകൾ നിർവചിക്കുന്ന മെറ്റാഡേറ്റക്കെയാണ് ലക്ഷ്യമിടുന്നത്, LLMകൾ ടൂൾ വിവരണങ്ങളും പാരാമീറ്ററുകളും എങ്ങനെ വ്യാഖ്യാനിക്കുന്നു എന്നതിനെ സദുർ‌വിനിയോഗപ്പെടുത്തുന്നു.

**ആക്രമണ മാർഗ്ഗങ്ങൾ:**
- **മെറ്റാഡേറ്റ മാനിപ്പുലേഷൻ**: ആക്രമകർ ടൂൾ വിവരണങ്ങളിൽ, പാരാമീറ്റർ നിർവ്വചനങ്ങളിലും ഉപയോഗ ഉദാഹരണങ്ങളിലും ദുഷ്‌പ്രേരിത നിർദ്ദേശങ്ങൾ ചേർക്കുന്നു
- **അദൃശ്യ നിർദേശങ്ങൾ**: ആയിരം മനുഷ്യ ഉപയോക്താക്കൾക്ക് കാണാനാകാത്ത എങ്കിലും AI മോഡലുകൾ പ്രോസസ് ചെയ്യുന്ന ടൂൾ മെറ്റാഡേറ്റയിലെ മറഞ്ഞിരിക്കുന്ന പ്രൊംപ്‌റ്റുകൾ
- **ഡൈനാമിക് ടൂൾ മാറ്റം ("റഗ് പുൾസ്")**: ഉപയോക്താക്കൾ അംഗീകരിച്ച ടൂളുകൾ പിന്നീട് അന്ധമായ പ്രവർത്തനങ്ങൾ ചെയ്യാൻ മാധ്യമമാറ്റം ചെയ്യും
- **പാരാമീറ്റർ ഇൻജെക്ഷൻ**: ടെക്സ്റ്റ് പാരാമീറ്റർ സ്കീമകളിൽ മാലിന്യ ഉള്ളടക്കം ചേർക്കുന്നു, ഇത് മോഡൽ പെരുമാറ്റത്തെ സ്വാധീനിക്കുന്നു


**ഹോസ്റ്റഡ് സർവർ അപകടങ്ങൾ**: റിമോട്ട് MCP സർവർകൾ ഉപകരണ നിർവചനങ്ങൾ പ്രാഥമിക ഉപയോക്തൃ അംഗീകാരം കഴിഞ്ഞ് അപ്ഡേറ്റ് ചെയ്യപ്പെടുന്നതിനാൽ ഉയർന്ന അപകടങ്ങൾ സൃഷ്ടിക്കുന്നു, ഇതോടെ മുൻപ് സുരക്ഷിതമായ ഉപകരണങ്ങൾ ദുര്മുഖതയാവാം. സമഗ്ര വിശകലനത്തിനായി, [ടൂൾ പൊയിസണിംഗ് ആക്രമണങ്ങൾ (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) കാണുക.

![Tool Injection Attack Diagram](../../../translated_images/ml/tool-injection.3b0b4a6b24de6bef.webp)

#### **കൂടുതൽ AI ആക്രമണ വഴികൾ**

- **ക്രോസ്-ഡൊമെയ്ൻ പ്രോമ്പ്റ്റ് ഇൻജക്ഷൻ (XPIA)**: സുരക്ഷാ നിയന്ത്രണങ്ങൾ മറികടക്കാൻ ബഹുദോമെയ്ൻ ഉള്ളടക്കം ഉപയോഗിക്കുന്ന കഠിനാക്രമണങ്ങൾ  
- **ഡൈനാമിക് ക്യാപബിലിറ്റി മാറ്റം**: പ്രാരംഭ സുരക്ഷാ വിലയിരുത്തലുകൾ മറികടക്കുന്ന ഉപകരണ ശേഷികളിൽ യഥാർഥസമയ മാറ്റങ്ങൾ  
- **കോൺടെക്റ്റ് വിൻഡോ പൊയിസണിംഗ്**: ദുര്മുഖ നിർദേശങ്ങൾ മറയ്ക്കാൻ വലിയ കോൺടെക്റ്റ് വിൻഡോകൾ മാനിപ്പുലേറ്റ് ചെയ്യുന്ന ആക്രമണങ്ങൾ  
- **മോഡൽ കോൺഫ്യൂഷൻ ആക്രമണങ്ങൾ**: മോഡൽ പരിധികൾ അപകർഷിച്ച് അനിഷ്ടമായ അല്ലെങ്കിൽ അപകടകരമായ പെരുമാറ്റങ്ങൾ സൃഷ്ടിക്കൽ  


### AI സുരക്ഷാ അപകടത്തെ പ്രഭാവം

**ഉയർന്ന പ്രഭാവമുള്ള പ്രത്യാഘാതങ്ങൾ**:
- **ഡേറ്റ എക്സ്ഫില്ട്രേഷൻ**: അനധികൃത அண negligibleച്‌വും സംവേദനാസുദ്ധതയുള്ള സംരംഭത്തിന്റെയും വ്യക്തിഗത ഡേറ്റയുടെയും മോഷണം  
- **സ്വകാര്യത ഭേദഗതികൾ**: വ്യക്തിഗത തിരിച്ചറിയാനുള്ള വിവരങ്ങൾ (PII)യും രഹസ്യപ്പെട്ട ബിസിനസ് ഡാറ്റയും വെളിപ്പെടുത്തൽ  
- **സിസ്റ്റം മാനിപ്പുലേഷൻ**: നിർണ്ണായക സിസ്റ്റങ്ങൾക്കും വർക്ക്‌ഫ്ലോകൾക്കും അനിഷ്ടമായ മാറ്റങ്ങൾ  
- **അധികാരം മോഷണം**: ഓത്തെന്റിക്കേഷൻ ടോക്കണുകളും സർവീസ് ക്രെഡൻഷ്യലുകളും കവർച്ച ചെയ്യൽ  
- **ലാറ്ററൽ മൂവ്മെന്റ്**: വമ്പിച്ച നെറ്റ്വർക്ക് ആക്രമണങ്ങൾക്ക് പിവോട്ട് ആയി ഉപയോഗിക്കുന്ന കവർച്ച ചെയ്ത AI സിസ്റ്റങ്ങൾ  

### Microsoft AI സുരക്ഷാ പരിഹാരങ്ങൾ

#### **AI പ്രോമ്പ്റ്റ് ഷീൽഡുകൾ: ഇൻജക്ഷൻ ആക്രമണങ്ങളിൽ ആധുനിക സംരക്ഷണം**

Microsoft **AI പ്രോമ്പ്റ്റ് ഷീൽഡുകൾ** നേരിട്ട് അല്ലെങ്കിൽ പരോക്ഷമായി പ്രോമ്പ്റ്റ് ഇൻജക്ഷൻ ആക്രമണങ്ങളിൽ സമഗ്ര സംരക്ഷണം നൽകുന്നു വിവിധ സുരക്ഷാ പാളികളിലൂടെ:

##### **പ്രധാന സംരക്ഷണ സംവിധാനങ്ങൾ:**

1. **ആധുനിക കണ്ടെത്തൽ & ഫിൽറ്ററിംഗ്**
   - മഷീൻ ലേണിംഗ് ആlgോറിതങ്ങളിലും NLP സാങ്കേതികവിദ്യകളിലും പുറം ഉള്ളടക്കം അനിഷ്ട നിർദ്ദേശങ്ങൾ തിരിച്ചറിയുന്നു  
   - രേഖകൾ, വെബ് പേജുകൾ, ഇമെയിലുകൾ, ഡേറ്റ സ്രോതസ്സ് എന്നിവയുടെ യഥാർത്ഥസമയ പരിശോധനയ്ക്കായി  
   - സാധുവും ദുർമുഖവുമായ പ്രോമ്പ്റ്റ് പാറ്റേണുകളുടെ സാങ്കേതിക കണ്ടെത്തൽ  

2. **സ്പോട്ട്‌ലൈറ്റിംഗ് സാങ്കേതിക വിദ്യകൾ**  
   - വിശ്വാസാർഹമായ സിസ്റ്റം നിർദ്ദേശങ്ങളും സംശയാസ്പദമായ പുറം ഇൻപുട്ടുകളും വ്യത്യസിക്കുന്നു  
   - ടെക്സ്റ്റ് മാറ്റിപ്പറഞ്ഞ് മോഡൽ പ്രസക്തിയിലേക്കും ദുര്മുഖ ഉള്ളടക്കം വെത്യസ്തമാക്കാൻ സഹായിക്കുന്നു  
   - AI സിസ്റ്റങ്ങൾ ശരിയായ നിർദേശം നിരക്ക് പാലിക്കാനും ഇൻജക്‌ഷൻ കമാന്റുകൾ നിരീക്ഷിക്കാതിരിക്കാനും സഹായിക്കുന്നു  

3. **ഡെലിമിറ്റർ & ഡേറ്റാമാർക്കിങ്ങ് സിസ്റ്റങ്ങൾ**
   - വിശ്വാസാർഹമായ സിസ്റ്റം സന്ദേശങ്ങളും പുറം ഇൻപുട്ട് ടെക്സ്റ്റും തമ്മിൽ വ്യക്തമായ അതിർത്തി നിർവചിക്കൽ  
   - വിശ്വാസാർഹവും വിശ്വാസം ഇല്ലാത്ത ഡേറ്റ സ്രോതസ്സ് തമ്മിൽ പ്രത്യേക മാർക്കറുകൾ ഉപയോഗിച്ച് അടയാളപ്പെടുത്തൽ  
   - നിർദ്ദേശക്കളഞ്ഞി തടയുകയും അനധികൃത കമാൻഡ് നടപ്പാക്കൽ ഒഴിവാക്കുകയും ചെയ്യുന്നു  

4. **തിരിമുറിവിള മെച്ചപ്പെട്ട ഭീഷണി വിവരശേഖരണം**
   - Microsoft ഉപരിതലത്തിലിറങ്ങിയ ആക്രമണ മാതിരികൾ നിരന്തരം നിരീക്ഷിക്കുന്നു  
   - പുതിയ ഇൻജക്ഷൻ സാങ്കേതിക വിദ്യകൾക്കും ആക്രമണ വഴി തിരഞ്ഞെടുക്കലുകൾക്കും മുൻകരുതൽ തിരിച്ചറിവ്  
   - സുതാര്യ നിലയിൽ ഭീഷണികളെ പ്രതിരോധിക്കാൻ സുരക്ഷാ മോഡൽ അപ്ഡേറ്റുകൾ  

5. **Azure ഉള്ളടക്കം സുരക്ഷ ഒരു കൂടിക്കാഴ്ച്ച**
   - സമഗ്ര Azure AI ഉള്ളടക്കം സുരക്ഷ സ്യൂട്ടിന്റെ ഭാഗമാണ്  
   - ജെയിൽബ്രേക്ക് ശ്രമങ്ങൾ, ഹാനികരമായ ഉള്ളടക്കം, സുരക്ഷാ നയം ലംഘനങ്ങൾ കണ്ടെത്താൻ അധിക പിന്തുണ  
   - AI ആപ്ലിക്കേഷൻ ഘടകങ്ങളിൽ ഏകീകരിച്ച സുരക്ഷാ നിയന്ത്രണങ്ങൾ

## ഉയർന്ന MCP സുരക്ഷാ ഭീഷണികൾ

### സെഷൻ ഹൈജാക്കിംഗ് അപകടങ്ങൾ

**സെഷൻ ഹൈജാക്കിംഗ്** സംസ്ഥാനപരമായ MCP നടപ്പിൽ ഒരു സാരമായ ആക്രമണ മാർഗമാണ്, അനധികൃത പാർട്ടികൾ ശരിയായ സെഷൻ ഐഡന്റിഫയർ മോഷണം ചെയ്ത് ഉപഭോക്താക്കളായി മിന്നുന്നതും അനധികൃത പ്രവർത്തനങ്ങൾ നടത്തുന്നതും.

#### **ആക്രമണ സന്നിവേശങ്ങളും അപകടങ്ങളും**

- **സെഷൻ ഹൈജാക് പ്രോമ്പ്റ്റ് ഇൻജക്ഷൻ**: മോഷണമെടുത്ത സെഷൻ ഐഡികൾ ഉള്ള ആക്രമികൾ സെഷൻ അവസ്ഥ പങ്കുവെക്കുന്ന സർവറുകളിൽ ദുർമുഖ സംഭവങ്ങൾ ഇൻജക്ട് ചെയ്ത് ഹാനികരമായ പ്രവർത്തനങ്ങൾ പ്രേരിപ്പിക്കാമെന്നോ സംവേനശീല ഡേറ്റ കാണാമെന്നോ സാധ്യത  
- **നേരിട്ടുള്ള പ്രതിനിധാനം**: മോഷണമെടുത്ത സെഷൻ ഐഡികൾ അഥൻറിക്കേഷൻ മറികടന്ന് MCP സർവർ വിളികൾ നേരിട്ട് അനുവദിച്ച് ആക്രമികളെ സാധുവായ ഉപയോക്താക്കളായി കണക്കാക്കുന്നു  
- **കമ്പ്രോമൈസ്ഡ് റീസൂമേബിള്‍ സ്ട്രീംസ്**: ആക്രമികൾ അപേക്ഷകൾ സമയം കഴിഞ്ഞ് നിർത്തിക്കൊണ്ട് വ്യാജ ഉള്ളടക്കത്തോടെ സാധുവായ ഉപഭോക്താക്കളെ തുടർന്നുള്ള സംവേദനത്തിനു പ്രേരിപ്പിക്കും  

#### **സെഷൻ മാനേജ്‌മെന്റിന് സുരക്ഷാ നിയന്ത്രണങ്ങൾ**

**അത്യാവശ്യ ആവശ്യകതകൾ:**
- **അധികാരം പരിശോധന**: MCP സർവർകൾ എല്ലാ ഇൻബൗണ്ട് അപേക്ഷകളും പരിശോധിക്കണമെന്നും **സെഷനുകൾക്ക് അടിസ്ഥാനമാക്കിയുള്ള അഥൻറിക്കേഷൻ ഒഴിവാക്കണമെന്നും**  
- **സുരക്ഷിത സെഷൻ ജനറേഷൻ**: ക്രിപ്റ്റോഗ്രാഫിക് അടിസ്ഥാനത്തിൽ സുരക്ഷിതവും അനിശ്ചിതവുമായ സെഷൻ ഐഡികൾ ഉപയോഗിക്കുക  
- **ഉപയോക്തൃ-കേന്ദ്രിത ബന്ധം**: സെഷൻ ഐഡികൾ `<user_id>:<session_id>` പോലുള്ള ഫോർമാറ്റിൽ ഉപയോക്തൃ വിവരങ്ങളുമായി ബന്ധിപ്പിച്ച് ഉപയോക്തൃ ഇടപെടൽ തടയുക  
- **സെഷൻ ലൈഫ് സൈക്കിൾ മാനേജ്‌മെന്റ്**: കാലപരിധി, റൊട്ടേഷൻ, അസാധുവാക്കൽ എന്നിവ വഴിയുള്ള അപകട സാധ്യതകൾ കുറയ്ക്കൽ  
- **ട്രാൻസ്പോർട്ട് സുരക്ഷ**: സർവകലാശാലാ HTTPS ഉപയാഗം സെഷൻ ഐഡി ഇടപെടൽ തടയാൻ നിർബന്ധം  

### സംഘട്ടനം കാര്യമാക്കുന്ന പ്രതിനിധരുടെ പ്രശ്നം

**സംഘട്ടനം കാര്യമാക്കുന്ന പ്രതിനിധർ പ്രശ്നം** MCP സർവർകൾ ഉപയോക്താക്കളുടെയും മൂന്നാർ സേവനങ്ങളുടെയും ഇടയിൽ ഒത്താശാ പ്രോക്സി ആയിത്തീരുമ്പോൾ സ്ഥിതിയാകും, ഇത് സ്ഥിരമായ ക്ലയന്റ് ഐഡി ദുരുപയോഗത്തിന്റെ കാരണം അധികാരം മറികടക്കാനുള്ള അവസരങ്ങൾ സൃഷ്ടിക്കുന്നു.

#### **ആക്രമണ രീതി & അപകടങ്ങൾ**

- **കുക്കി അടിസ്ഥാനമാക്കിയുള്ള സമ്മതികൾ മറികടക്കൽ**: മുൻ ഉപയോക്തൃ അംഗീകാരം സൃഷ്ടിച്ച സമ്മതി കുക്കികളെ ആക്രമികൾ ദുര്മുഖമായ അംഗീകാരം അഭ്യർത്ഥനകളിലൂടെ ദുരുപയോഗിക്കുന്നു  
- **അധികാര കോഡ് മോഷണം**: നിലവിലുള്ള സമ്മതി കുക്കികൾ അംഗീകാര സെർവറുകളെ സമ്മതി സ്ക്രീനുകൾ ഒഴിവാക്കി ആക്രമിക്ക് നിയന്ത്രണത്തിലുള്ള എൻഡ്‌പോയിന്റിലേക്ക് കോഡ് റീഡയറക്ട് ചെയ്യാൻ പ്രേരിപ്പിക്കാം  
- **അനധികൃത API ആക്‌സസ്**: മോഷണമെടുത്ത അംഗീകാര കോഡുകൾ ടോക്കൺ എക്സ്ചേഞ്ചിന് പ്രയോജനപ്പെടുത്തി ഉപയോക്തൃ അപരിചയം നടത്തുന്നു  

#### **തിരുത്തൽ തന്ത്രങ്ങൾ**

**നിർബന്ധമായ നിയന്ത്രണങ്ങൾ:**
- **സ്പഷ്ടമായ സമ്മതി ആവശ്യങ്ങൾ**: സ്ഥിരമായ ക്ലയന്റ് ഐഡികൾ ഉപയോഗിക്കുന്ന MCP പ്രോക്സി സർവർകൾ ദൈനാമിക് റെജിസ്ട്രേഷൻ ചെയ്ത ക്ലയന്റിനായി ഓരോത്തും ഉപയോക്തൃ സമ്മതി നേടണം  
- **OAuth 2.1 സുരക്ഷ പ്രവർത്തനം**: എല്ലാ അംഗീകാരം അഭ്യർത്ഥനകൾക്കും PKCE (പ്രൂഫ് കീ ഫോർ കോഡ് എക്സ്ചേഞ്ച്) ഉൾപ്പെടെ നിലവിലെ OAuth മികച്ച സുരക്ഷാ രീതികൾ പാലിക്കുക  
- **ക്ലയന്റ് പരിശോധന കർശനമായത്**: റീഡയറക്ട് URI-കളും ക്ലയന്റ് ഐഡന്റിഫയർ-മാരും ദുരുപയോഗം തടയാൻ കർശന പരിശോധന നടപ്പാക്കുക  

### ടോക്കൺ പാസ്ത്രൂ അപായങ്ങൾ  

**ടോക്കൺ പാസ്ത്രൂ** MCP സർവർകൾ ക്ലയന്റ് ടോക്കണുകൾ ശരിയായ പരിശോധന കൂടാതെ സ്വീകരിച്ച് അവ ദുരുപയോഗം ചെയ്യാൻ അനുവദിക്കുന്ന ഒരു വിരുദ്ധ പാറ്റേൺ ആണ്, MCP അധികാരം നിർദേശങ്ങൾ ലംഘിക്കുന്നു.

#### **സുരക്ഷാ പ്രത്യാഘാതങ്ങൾ**

- **നിയന്ത്രണം മറികടക്കൽ**: നേരിട്ട് ക്ലയന്റ്-നിന്നും API ടോക്കൺ ഉപയോഗം അനിവാര്യമായ നിരക്ക് പരിധികളും പരിശോധനകളും നിരീക്ഷണങ്ങളും ഒഴിവാക്കുന്നു  
- **ഓഡിറ്റ് ട്രെയിൽ നശിപ്പിക്കൽ**: മേല്പറഞ്ഞ ടോക്കണുകൾ ക്ലയന്റ് തിരിച്ചറിയൽ അശേഷിപ്പിക്കുക, സംഭവപരിശോധനയ്ക്ക് തടസ്സം നൽകുന്നു  
- **പ്രോക്കി അടിസ്ഥാന രാജി**: പരിശോധനയില്ലാത്ത ടോക്കണുകൾ അനധികൃത ഡേറ്റ ആക്സസിനായി സർവർകൾ പ്രോക്കിയായി ഉപയോഗിക്കുന്നത് സൗകര്യപ്പെടുത്തി  
- **വിശ്വാസ അതിർത്തി ലംഘനം**: ടോക്കൺ ഉത്ഭവം പരിശോധിക്കാനാകാത്തപ്പോൾ ഡൗൺസ്ട്രീം സർവീസുകളുടെ വിശ്വാസ അവകാശങ്ങൾ ലംഘിക്കും  
- **മൾട്ടി-സർവീസ് ആക്രമണം വ്യാപിച്ചു**: കവർച്ച ചെയ്ത ടോക്കണുകൾ പല സർവീസുകളിലും സ്വീകരിച്ചു ലാറ്ററൽ മൂവ്മെന്റിനുള്ള വഴിയൊരുക്കും  

#### **ആവശ്യമായ സുരക്ഷ നിയന്ത്രണങ്ങൾ**

**അടിക്കുറപ്പായ ആവശ്യകതകൾ:**
- **ടോക്കൺ പരിശോധന**: MCP സർവർക്ക് പുറമെ മറ്റൊരാൾ ഇഷ്‌ടപ്പെട്ട ടോക്കണുകൾ സ്വീകരിക്കരുത്  
- **പ്രേക്ഷക പരിശോധന**: ടോക്കൺ പ്രേക്ഷക അവകാശങ്ങൾ MCP സർവറിന്റെ ഐഡന്റിറ്റിയെ പൊരുത്തപ്പെടുന്നുണ്ടെന്ന് എല്ലായ്പ്പോഴും പരിശോധിക്കുക  
- **സാധുവായ ടോക്കൺ ജീവിതചക്രം**: കുറഞ്ഞ കാലാവധി ആക്‌സസ് ടോക്കണുകൾ സുരക്ഷിത റൊട്ടേഷൻ പ്രവർത്തനങ്ങളോടെ നടപ്പിലാക്കുക  


## AI സിസ്റ്റങ്ങൾക്കുള്ള സപ്ലൈ ചെയിൻ സുരക്ഷ

പരമ്പരാഗത സ്വർകോഴ്സ് ആശ്രിതത്വങ്ങളെക്കാൾ കൂടുതലായി AI സമ്പൂർണ്ണ വാതിൽപ്പടകൾ ഉൾക്കൊള്ളുന്ന സപ്ലൈ ചെയിൻ സുരക്ഷ വികസിതമാണ്. ആധുനിക MCP നടപ്പാക്കലുകൾ AI-ുമായി ബന്ധപ്പെട്ട എല്ലാ ഘടകങ്ങളും കൃത്യമായി പരിശോധിക്കുകയും നിരീക്ഷിക്കുകയും വേണം, കാരണം ഓരോന്നും സിസ്റ്റം സമ്പൂർണതയ്ക്ക് ഭീഷണി സൃഷ്ടിക്കാം.  

### വിപുലീകരിച്ച AI സപ്ലൈ ചെയിൻ ഘടകങ്ങൾ

**പരമ്പരാഗത സോഫ്റ്റ്‌വെയർ ആശ്രിതത്വങ്ങൾ:**
- ഓപ്പൺ സോഴ്‌സ് ലൈബ്രററികളും ഫ്രെയിംവർകുകളും  
- കണ്ടെയ്നർ ഇമേജുകളും അടിസ്ഥാന സിസ്റ്റങ്ങളും  
- വികസന ഉപകരണങ്ങളും ബിൽഡ് പൈപ്പ്ലൈനുകളും  
- ഇൻഫ്രാസ്ട്രക്ചർ ഘടകങ്ങളും സേവനങ്ങളും  

**AI പ്രത്യേക സപ്ലൈ ചെയിൻ ഘടകങ്ങൾ:**
- **ഫൗണ്ടേഷൻ മോഡലുകൾ**: വ്യത്യസ്ത വിതരണം ചെയ്യുന്നവരിൽ നിന്നുള്ള പ്രീ-ട്രെയിനുചെയ്‌ത മോഡലുകൾ, ഉറവിട പരിശോധന ആവശ്യമാണ്  
- **എംബെഡ്ഡിംഗ് സേവനങ്ങൾ**: പുറം വെക്ടറൈസേഷൻ, സെമാന്റിക് സെർച്ച് സേവനങ്ങൾ  
- **കോൺടെകസ്റ്റ് പ്രൊവൈഡറുകൾ**: ഡേറ്റ സ്രോതസ്സുകൾ, നോളേജ് ബേസുകൾ, രേഖാ ശേഖരங்கள்  
- **മൂന്നാർ APIകൾ**: പുറം AI സേവനങ്ങൾ, ML പൈപ്പ്ലൈനുകൾ, ഡേറ്റ പ്രോസസ്സിംഗ് എൻഡ്‌പോയിന്റുകൾ  
- **മോഡൽ ആർട്ടിഫാക്റ്റുകൾ**: ഭാരം, കോൺഫിഗറേഷനുകൾ, ഫൈൻ-ട്യൂൺ ചെയ്ത മോഡൽ വേറിയന്റുകൾ  
- **ട്രെയ്നിംഗ് ഡാറ്റ സ്രോതസ്സുകൾ**: മോഡൽ പരിശീലനത്തിനും ഫൈൻ-ട്യൂണിംഗിനും ഉപയോഗിച്ചിട്ടുള്ള ഡാറ്റാസറ്റുകൾ  

### സമഗ്ര സപ്ലൈ ചെയിൻ സുരക്ഷാ തന്ത്രം

#### **ഘടക പരിശോധന & വിശ്വാസം**
- **ഉറവിട പരിശോധന**: എല്ലാ AI ഘടകങ്ങൾ സംയോജിപ്പിക്കുന്നതിന് മുമ്പ് അതിന്റെ ഉറവിടം, ലൈസൻസിംഗ്, സമഗ്രത പരിശോധിക്കുക  
- **സുരക്ഷാ അവലോകനം**: മോഡലുകൾ, ഡേറ്റ സ്രോതസ്സുകൾ, AI സേവനങ്ങൾക്ക് തോന്നുന്ന അപകടങ്ങൾ പരിശോധിക്കുക  
- **പ്രതിഷ്ഠാ വിശകലനം**: AI സേവന ദാതാക്കളുടെ സുരക്ഷാ രേഖകളും രീതികളും വിലയിരുത്തുക  
- **അംശസഹിഷ്ണുത പരിശോധന**: എല്ലാ ഘടകങ്ങളും സംഘടനാ സുരക്ഷാ, നിയമപരישോധന ആവശ്യകതകൾ പാലിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക  

#### **സുരക്ഷിത വിന്യാസ പൈപ്പ്ലൈനുകൾ**  
- **സ്വയംചാലിത CI/CD സുരക്ഷ**: എല്ലായ്പ്പോഴും സുരക്ഷാ സ്കാനിംഗ് പൈപ്പ്ലൈനുകളിൽ ഉൾപ്പെടുത്തുക  
- **ആർട്ടിഫാക്റ്റ് സമഗ്രത**: വിന്യസിക്കുന്ന എല്ലായ്‌തും (കോഡ്, മോഡലുകൾ, കോൺഫിഗറേഷനുകൾ) ക്രിപ്റോ പരിശോധന നടപ്പാക്കുക  
- **പടിവാതിൽ വിന്യാസം**: ഓരോ ഘട്ടത്തിലും സുരക്ഷാ പരിശോധനകൾ നടത്തുന്ന പ്രഗതിശീല വിന്യാസ തന്ത്രങ്ങൾ ഉപയോഗിക്കുക  
- **വിശ്വാസാർഹ ആർട്ടിഫാക്റ്റ് റജിസ്ട്രികൾ**: സരട്ടിഫിക്കുചെയ്ത, സുരക്ഷിത റെപോസിറ്ററികളിൽ നിന്നേ മാത്രം വിന്യസിക്കുക  

#### **നിരന്തരം നിരീക്ഷണവും പ്രതികരണവും**
- **ആശ്രിതത്വ പരിശോധന**: എല്ലാ സോഫ്റ്റ്‌വെയർ, AI ഘടക ആശ്രിതത്വത്തിനും തുടർച്ചയായ അപായ നിരീക്ഷണം  
- **മോഡൽ നിരീക്ഷണം**: മോഡൽ പെരുമാറ്റം, പ്രകടനത്തിന്റെ മാറ്റം, സുരക്ഷാ അസാധാരണതകൾ നിരന്തരം വിലയിരുത്തൽ  
- **സേവനാരോഗ്യ പിന്തുടർപ്പ്**: AI സേവനങ്ങളുടെ ലഭ്യത, സുരക്ഷാ സംഭവം, നയ മാറ്റങ്ങൾ നിരീക്ഷണം  
- **ഭീഷണി വിവരശേഖരണ സംയോജനം**: AI, ML സുരക്ഷാ അപകടങ്ങൾക്ക് പ്രത്യേകിച്ചുള്ള ഭീഷണി ഫീഡുകൾ ഉൾപ്പെടുത്തൽ  

#### **പ്രവേശന നിയന്ത്രണവും കുറവ് അവകാശവും**
- **ഘടക നിരീക്ഷണം അനുമതികൾ**: ബിസിനസ് ആവശ്യാനുസരണം മോഡലുകൾക്കും ഡേറ്റയ്ക്കും സേവനങ്ങൾക്കും അനുവദനവകാശങ്ങൾ നിയന്ത്രിക്കുക  
- **സർവീസ് അക്കൗണ്ട് മാനേജ്‌മെന്റ്**: കുറഞ്ഞ അവശ്യ അനുമതികളോടെ സമർപ്പിത സർവീസ് അക്കൗണ്ടുകൾ നടപ്പിലാക്കുക  
- **നെറ്റ്‌വര്‍ക്ക് വിഭജനം**: AI ഘടകങ്ങൾ വിട്ടുവീഴ്ച നിഷേധിക്കുക, സേവനങ്ങൾ തമ്മിലുള്ള നെറ്റ്‌വർക്ക് ആക്സസ് പരിമിക്കുക  
- **API ഗേറ്റ്‌വേ നിയന്ത്രണങ്ങൾ**: പുറം AI സേവനങ്ങളിൽ ആക്സസ് നിയന്ത്രണം, നിരീക്ഷണം നടപ്പാക്കാൻ കേന്ദ്രീകൃത API ഗേറ്റ്‌വേകൾ ഉപയോഗിക്കുക  

#### **സംഭവ പ്രതികരണവും പുനരധിവാസവും**
- **വേഗത്തിലുള്ള പ്രതികരണ നടപടികൾ**: കവർച്ചപ്പെട്ട AI ഘടകങ്ങളുടെ പാച്ചിംഗ് അല്ലെങ്കിൽ മാറ്റം സംബന്ധിച്ച സ്ഥാപിത നടപടിക്രമങ്ങൾ  
- **അധികാരശേഷി റൊട്ടേഷൻ**: രഹസ്യങ്ങൾ, API കീകൾ, സർവീസ് ക്രെഡൻഷ്യലുകൾ സ്വയംചാലിതമായി റൊട്ടേറ്റ് ചെയ്യാനുള്ള സംവിധാനം  
- **റോൾബാക്ക് സൗകര്യങ്ങൾ**: പണ്ടത്തെ അറിയപ്പെടുന്ന നല്ല പതിപ്പുകളിലേക്ക് എളുപ്പത്തിൽ മടങ്ങാനുള്ള കഴിവ്  
- **സപ്ലൈ ചെയിൻ ലംഘന പുനരധിവാസം**: മുകളിൽ നിന്നുള്ള AI സേവനങ്ങൾ കവർച്ച വന്നപ്പോൾ പ്രതികരിക്കാൻ പ്രത്യേക നടപടിക്രമങ്ങൾ

Microsoft എല്ലാ ഉற்பന്നങ്ങളിലും വ്യാപകമായ സപ്ലൈ ചെയിൻ സുരക്ഷാ രീതികൾ നടപ്പാക്കുന്നു. [Microsoft-ൽ സോഫ്റ്റ്വെയർ സപ്ലൈ ചെയിൻ സുരക്ഷിതമാക്കലിലേക്കുള്ള യാത്ര](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) പഠിക്കുക.  


## ഫൗണ്ടേഷൻ സുരക്ഷാ മികച്ച നിബന്ധനകൾ

MCP നടപ്പാക്കലുകൾ നിങ്ങളുടെ സംഘടനയുടെ നിലവിലുള്ള സുരക്ഷാ നിലമെടുത്തു വികസിപ്പിക്കുന്നു. അടിസ്ഥാന സുരക്ഷാ രീതികൾ ശക്തിപ്പെടുത്തുന്നത് AI സിസ്റ്റങ്ങളുടെയും MCP വിന്യാസങ്ങളുടെയും സമഗ്ര സുരക്ഷ വളരെയധികം മെച്ചപ്പെടുത്തും.

### പ്രധാന സുരക്ഷാ അടിസ്ഥാനങ്ങൾ

#### **സുരക്ഷിത വികസന രീതികൾ**
- **OWASP പാലന**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) വെബ് അപ്ലിക്കേഷൻ അപകടങ്ങൾക്കെതിരെ സംരക്ഷണം  
- **AI-വിശിഷ്ട സംരക്ഷണം**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) സുരക്ഷാ നിയന്ത്രണങ്ങൾ നടപ്പാക്കുക  
- **സുരക്ഷിത രഹസ്യ മാനേജ്‌മെന്റ്**: ടോക്കൺ, API കീകൾ, സംവേദനാപ്രധാന കോൺഫിഗറേഷൻ ഡാറ്റക്ക് സമർപ്പിത വാൾട്ടുകൾ ഉപയോഗിക്കുക  
- **എൻഡ്-ടു-എൻഡ് എൻകർപ്ഷൻ**: എല്ലായ്‌തും ആപ്ലിക്കേഷൻ ഘടകങ്ങളിലും ഡാറ്റ പാതകളിലും സുരക്ഷിത ആശയവിനിമയം നടപ്പിലാക്കുക  
- **ഇൻപുട്ട് മാനേജ്‌മെന്റ്**: എല്ലാ ഉപയോക്തൃ ഇൻപുട്ടുകൾ, API പാരാമീറ്ററുകൾ, ഡേറ്റാ സ്രോതസ്സുകൾ കർശനമായി പരിശോധിക്കുക  

#### **ഇൻഫ്രാസ്‌ട്രക്‌ചർ ഹാർഡനിംഗ്**
- **മൾട്ടി-ഫാക്ടർ ഓതൻറിക്കേഷൻ**: എല്ലാ അഡ്മിനിസ്ട്രേറ്റീവ്, സർവീസ് അക്കൗണ്ടുകൾക്കും നിർബന്ധമായ MFA  
- **പാച്ച് മാനേജ്‌മെന്റ്**: ഓപ്പറേറ്റിംഗ് സിസ്റ്റങ്ങൾ, ഫ്രെയിംവർക്, ആശ്രിതത്വങ്ങൾക്കായി സ്വയംചാലിത, സമയബന്ധിത പാച്ചിംഗ്  
- **ഐഡന്റിറ്റി പ്രൊവൈഡർ സംയോജനം**: Microsoft Entra ID, ആക്ടീവ് ഡയറക്ടറി പോലുള്ള സംരംഭ ഐഡന്റിറ്റി പ്രൊവൈഡർമാരിലൂടെ കേന്ദ്രീകൃത ഐഡന്റിറ്റി മാനേജ്‌മെന്റ്  
- **നെറ്റ്‌വർക്ക് വിഭജനം**: MCP ഘടകങ്ങൾ ലാറ്ററൽ മൂവ്മെന്റ് നിയന്ത്രിക്കാൻ ലജിക്കൽ ഐസൊലേഷനിൽ വെക്കുക  
- **കുറഞ്ഞ അവകാശ സിദ്ധാന്തം**: എല്ലാ സിസ്റ്റം ഘടകങ്ങൾക്കും അക്കൗണ്ടുകൾക്കും കുറഞ്ഞ ആവശ്യമുള്ള അനുമതികൾ  

#### **സുരക്ഷാ നിരീക്ഷണവും കണ്ടെത്തലും**
- **സമഗ്ര ലോഗിംഗ്**: MCP ക്ലയന്റ്-സർവർ ഇടപെടലുകൾ ഉൾപ്പെടെ AI ആപ്ലിക്കേഷൻ പ്രവർത്തനങ്ങളുടെ വിശദമായ രേഖപ്പെടുത്തൽ  
- **SIEM സംയോജനം**: ആномലി കണ്ടെത്തലിനായുള്ള കേന്ദ്രകൃത സുരക്ഷാ വിവര, ഇവന്റ് മാനേജുമെന്റ്  
- **നടപടികൾ വിശകലനം**: സിസ്റ്റം, ഉപയോക്തൃ പെരുമാറ്റത്തിലെ അസാധാരണ പാറ്റേണുകൾ കണ്ടെത്തുന്ന AI അധിഷ്ഠിത നിരീക്ഷണം  
- **ഭീഷണി വിവരശേഖരണം**: പുറം ഭീഷണി ഫീഡുകളും компрോമൈസ് സൂചനകളും ഉൾപ്പെടുത്തൽ  
- **സംഭവ പ്രതികരണം**: സുരക്ഷാ സംഭവം കണ്ടെത്തൽ, പ്രതികരണം, പുനരധിവാസം എന്നീ ശാസ്ത്രീയ നടപടിക്രമങ്ങൾ  

#### **സീറോ ട്രസ്റ്റ് ആർക്കിടെക्चर**
- **ഒന്നും വിശ്വസിക്കരുത്, എല്ലായ്പോഴും പരിശോദിക്കുക**: ഉപയോക്താക്കൾ, ഉപകരണങ്ങൾ, നെറ്റ്‌വർക്ക് കണക്ഷനുകൾ നിരന്തരം പരിശോധന  
- **മൈക്രോ-സെഗ്മെന്റേഷൻ**: വ്യക്തിഗത വർക്ക്ലോഡുകളും സേവനങ്ങളും ബഹിരാകാശമാക്കുന്ന സൂക്ഷ്മ നെറ്റ്‌വർക്ക് നിയന്ത്രണങ്ങൾ  
- **ഐഡന്റിറ്റി-കേന്ദ്രിത സുരക്ഷ**: നെറ്റ്‌വർക്ക് സ്ഥലം പകരം സ്ഥിരീകരിക്കപ്പെട്ട ഐഡന്റിറ്റികളുടെ അടിസ്ഥാനത്തിൽ സുരക്ഷാ നയങ്ങൾ  
- **നിരന്തരം അപകട വിലയിരുത്തൽ**: നിലവിലെ കോൺടെക്സ്‌റ്റും പെരുമാറ്റവും അടിസ്ഥാനമാക്കി സുസ്ഥിര സുരക്ഷ നില  
- **ശരിതപ്രാപ്തി ലഭ്യത**: അപകട ഘടകങ്ങൾ, സ്ഥലം, ഉപകരണം വിശ്വാസം അടിസ്ഥാനമാക്കി ആക്സസ് നിയന്ത്രണങ്ങൾ  

### സംരംഭങ്ങളുടെ സംയോജനം മാതൃകകൾ

#### **Microsoft സുരക്ഷാ പരിസരം സംയോജനം**
- **Microsoft Defender for Cloud**: സമഗ്ര ക്ലൗഡ് സുരക്ഷാ നില മാനേജ്‌മെന്റ്  
- **Azure Sentinel**: AI വർക്ക്‌ലോഡ് സംരക്ഷണത്തിനുള്ള ക്ലൗഡ്-സ്വദേശ SIEM, SOAR ശേഷികൾ  
- **Microsoft Entra ID**: വ്യവസായ ഐഡന്റിറ്റി, ആക്സസ് മാനേജ്‌മെന്റും ശരിതസ്പദ ആക്സസ് നയങ്ങളോടൊപ്പം  
- **Azure Key Vault**: ഹാർഡ്‌വെയർ സുരക്ഷാ മോഡ്യൂൾ (HSM) പിന്തുണയുളള കേന്ദ്രീകൃത രഹസ്യ മാനേജ്‌മെന്റ്  
- **Microsoft Purview**: AI ഡാറ്റാ സ്രോതസ്സുകളും വർക്ക്ലോകുകളും നിയന്ത്രിക്കുന്ന ഡാറ്റ ഗവർണൻസ്, അനുസരിച്ചു  

#### **അനംശാനുസൃതതയും ഗവേണൻസും**
- **നിയന്ത്രണ യാത്രയയപ്പ്**: MCP നടപ്പാക്കല들이 വ്യവസായ-സവിശേഷ അനുശാസന ആവശ്യകതകൾ (GDPR, HIPAA, SOC 2) പാലിക്കുന്നുവെന്ന് ഉറപ്പാക്കുക  

- **ഡേറ്റാ വർഗ്ഗീകരണം**: AI സിസ്റ്റങ്ങളാൽ പ്രോസസ്സ് ചെയ്യുന്ന സമ്പത് ഡാറ്റയുടെ ശരിയായ വർഗ്ഗീകരണവും കൈകാര്യം ചെയ്യലും  
- **ഓഡിറ്റ് ട്രെയിലുകൾ**: നിബന്ധനാപാലനവും ഫോറൻസിക് പരിശോധനക്കുമായി വ്യാപകമായ ലോ​ഗ്ഗിംഗ്  
- **സ്വകാര്യതാ നിയന്ത്രണങ്ങൾ**: AI സിസ്റ്റം ആർക്കിടെക്ചറിൽ സ്വകാര്യത-ഗണിത സിദ്ധാന്തങ്ങളുടെ നടപ്പാക്കൽ  
- **മാറ്റ മാനേജ്മെന്റ്**: AI സിസ്റ്റം മാറ്റങ്ങളുടെ സുരക്ഷാ അവലോകനങ്ങൾക്ക് ഔദ്യോഗിക പ്രക്രിയകൾ  

ഈ അടിസ്ഥാന പ്രാക്ടിസുകൾ MCP-നിഷ്ട സുരക്ഷാ നിയന്ത്രണങ്ങളുടെ ഫലപ്രാപ്തി വർദ്ധിപ്പിക്കുന്ന ശക്തമായ സുരക്ഷാ അടിസ്ഥാനരേഖ സൃഷ്ടിക്കുകയും AI-സഞ്ചാലിത അപേക്ഷകൾക്ക് സമഗ്ര സംരക്ഷണം നൽകുകയും ചെയ്യുന്നു.  

## പ്രധാന സുരക്ഷാ തെളിവുകൾ  

- **പാളിതല സുരക്ഷാ സമീപനം**: അടിസ്ഥാന സുരക്ഷാ പ്രാക്ടിസുകൾ (സുരക്ഷിത കോഡിംഗ്, ഏറ്റവും കുറഞ്ഞ അധികാരം, സപ്ലൈ ചെയിൻ പരിശോധന, തുടർച്ചയായ നിരീക്ഷണം) AI-നിഷ്ട നിയന്ത്രണങ്ങളുമായി സംയോജിപ്പിച്ച് സമഗ്ര സംരക്ഷണം ലഭ്യമാക്കുക  

- **AI-നിഷ്ട ഭീഷണി സന്നിവേശം**: MCP സിസ്റ്റങ്ങൾ കണക്കിലെടുക്കേണ്ട വ്യത്യസ്ത തരം അപകടങ്ങൾPrompt injection, Tool poisoning, Session hijacking, Confused deputy പ്രശ്‌നങ്ങൾ, Token passthrough ദുര്‍വിനിയോഗങ്ങൾ, അനാവശ്യ അധികാരങ്ങൾ എന്നിവയ്ക്കുള്ള പ്രത്യേക പരിഹാരങ്ങൾ ആവശ്യമുണ്ട്  

- **പരിചയകരണവും അംഗീകാരവും**: Microsoft Entra ID പോലെയുള്ള ബാഹ്യ ഐഡന്റിറ്റി പ്രൊവൈറ്റർമാരുടെ ഉറപ്പുള്ള ഓതന്റിക്കേഷൻ നടപ്പാക്കുക, ടോകൺ പരിശോധന കർശനമായി നടപ്പാക്കുക, MCP സർവറിന് സ്പെസിഫിക്കായി നൽകിയിട്ടില്ലാത്ത ടോകണുകൾ എപ്പോൾ കൂടി സ്വീകരിക്കരുത്  

- **AI ആക്രമണ പ്രതിരോധം**: Microsoft Prompt Shields, Azure Content Safety ഉപയോഗിച്ച് നേരിട്ടുള്ള പരോക്ഷ Prompt Injection, Tool Poisoning ആക്രമണങ്ങൾ നിന്ന് സംരക്ഷിക്കുക, Tool മെടാഡേറ്റ പരിശോധിക്കുകയും ഗതിശീല മാറ്റങ്ങൾ നിരീക്ഷിക്കുകയും ചെയ്യുക  

- **സെഷൻ & ട്രാൻസ്പോർട്ട് സുരക്ഷ**: ഉപയോക്തൃ ഐഡന്റിറ്റികളോട് ബന്ധിപ്പിച്ച ക്രിപ്‌റ്റോഗ്രാഫിക്‌గా സുരക്ഷിതവും അനിശ്ചിതവും ആയ സെഷൻ ഐഡികൾ ഉപയോഗിക്കുക, സെഷൻ എല്ലാ ഘട്ടങ്ങളിലും ശരിയായി കൈകാര്യം ചെയ്യുക, സെഷന്റിനായി ഓതന്റിക്കേഷൻ ഉപയോഗിക്കരുത്  

- **OAuth സുരക്ഷ മികച്ചപ്രവൃത്തികൾ**: ഡൈനാമിക് രജിസ്റ്റർ ചെയ്ത ക്ലയന്റുകൾക്ക് വ്യക്തമായ ഉപയോക്തൃ അനുമതി ഉൾപ്പെടെ Confused deputy ആക്രമണങ്ങൾ തടയുക, PKCE ഉള്ള OAuth 2.1 കർശനമായി നടപ്പാക്കുകയും, Redirect URI കർശനമായി പരിശോധിക്കുകയും ചെയ്യുക  

- **ടോക്കൺ സുരക്ഷ സിദ്ധാന്തങ്ങൾ**: ടോക്കൺ പാസ്സ്ത്രൂ ആന്റി-പാറ്റേണുകൾ ഒഴിവാക്കുക, ടോക്കൺ ഓഡിയൻസ് ക്ലെയിമുകൾ പരിശോധിക്കുക, സുരക്ഷിത റോട്ടേഷൻ ഉള്ള തന്നെ കുറച്ചുനാൾ മാത്രം സാധുവായ ടോക്കണുകൾ നടപ്പാക്കുക, വ്യക്തമായ ട്രസ്റ്റ് ബൗണ്ടറികൾ നിലനിര്‍ത്തുക  

- **വ്യാപകമായ സപ്ലൈ ചെയിൻ സുരക്ഷ**: AI അപരിസര ഘടകങ്ങൾ (മോഡലുകൾ, എംബെഡിങ്ങുകൾ, കോൺടെക്സ്റ്റ് പ്രൊവൈഡർമാർ, ബാഹ്യ API-കൾ) പരമ്പരാഗത സോഫ്റ്റ്വെയർ ആശ്രിതങ്ങളിൽ പോലെ സമగ్రമായ സുരക്ഷ പാലിക്കുക  

- **തുടർച്ചയായ മാറ്റങ്ങൾ**: വേഗത്തിൽ മാറുന്ന MCP മാനദണ്ഡങ്ങളിൽ പുതിയവ അറിയുകയും സുരക്ഷാ സമുദായ മാനദണ്ഡങ്ങളിൽ സംഭാവന നൽകുകയും പ്രോട്ടോക്കോൾ പുരോഗമിക്കുമ്പോളുള്ള അനുയോജ്യമായ സുരക്ഷാ നിലപാടുകൾ നിലനിർത്തുകയും ചെയ്യുക  

- **Microsoft സുരക്ഷ ഒരുമിച്ചുള്ള സംയോജനം**: Microsoft-ന്റെ സമഗ്ര സുരക്ഷാ പരിസ്ഥിതി (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) MCP വിന്യസനത്തെ കൂടുതൽ സംരക്ഷിക്കുക
