# MCP سیکیورٹی: AI سسٹمز کے لیے جامع تحفظ

[![MCP سیکیورٹی بہترین طریقے](../../../translated_images/ur/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(اس سبق کی ویڈیو دیکھنے کے لیے اوپر تصویر پر کلک کریں)_

سیکیورٹی AI نظام کے ڈیزائن کی بنیاد ہے، اسی لیے ہم اسے اپنا دوسرا سیکشن بناتے ہیں۔ یہ مائیکروسافٹ کے **Secure by Design** اصول سے مطابقت رکھتا ہے جو [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) کا حصہ ہے۔

ماڈل کانٹیکسٹ پروٹوکول (MCP) AI سے چلنے والی ایپلیکیشنز میں طاقتور نئی صلاحیتیں لاتا ہے، جبکہ روایتی سافٹ ویئر کے خطرات سے آگے منفرد سیکیورٹی چیلنجز بھی متعارف کراتا ہے۔ MCP سسٹمز کو نصب شدہ سیکیورٹی خدشات (محفوظ کوڈنگ، کم از کم اجازت، سپلائی چین سیکیورٹی) کے ساتھ ساتھ جدید AI مخصوص خطرات جیسے پرامپٹ انجیکشن، ٹول زہر آلودگی، سیشن ہائی جیکنگ، کنفیوزڈ ڈیپٹی حملے، ٹوکن پاس تھرو کمزوریاں، اور متحرک صلاحیت میں ترمیم کا سامنا ہوتا ہے۔

یہ سبق MCP کے نفاذ میں سب سے اہم سیکیورٹی خطرات کو دریافت کرتا ہے—جیسے کہ تصدیق، اجازت، غیر ضروری اجازتیں، بالواسطہ پرامپٹ انجیکشن، سیشن سیکیورٹی، کنفیوزڈ ڈیپٹی مسائل، ٹوکن مینجمنٹ، اور سپلائی چین کی کمزوریاں۔ آپ حفاظتی کنٹرولز اور بہترین طریقے سیکھیں گے تاکہ ان خطرات کو کم کیا جا سکے، ساتھ ہی مائیکروسافٹ کے حل جیسے Prompt Shields، Azure Content Safety، اور GitHub Advanced Security کو MCP کی تعیناتی کو مضبوط بنانے کے لیے استعمال کرنا سیکھیں گے۔

## سیکھنے کے مقاصد

اس سبق کے آخر تک، آپ قادر ہوں گے کہ:

- **MCP مخصوص خطرات کی شناخت کریں**: MCP سسٹمز میں منفرد سیکیورٹی خطرات کو پہچانیں جن میں پرامپٹ انجیکشن، ٹول زہر آلودگی، غیر ضروری اجازتیں، سیشن ہائی جیکنگ، کنفیوزڈ ڈیپٹی مسائل، ٹوکن پاس تھرو کمزوریاں، اور سپلائی چین خطرات شامل ہیں
- **سیکیورٹی کنٹرولز کا نفاذ کریں**: موثر تخفیف کے اقدامات نافذ کریں جن میں مضبوط تصدیق، کم از کم اجازت، محفوظ ٹوکن مینجمنٹ، سیشن سیکیورٹی کنٹرولز، اور سپلائی چین کی تصدیق شامل ہے
- **مائیکروسافٹ سیکیورٹی حل استعمال کریں**: مائیکروسافٹ کے Prompt Shields، Azure Content Safety، اور GitHub Advanced Security کو MCP کے کام کا تحفظ کے لیے سمجھیں اور نافذ کریں
- **ٹول سیکیورٹی کی توثیق کریں**: ٹول میٹا ڈیٹا کی توثیق کی اہمیت کو سمجھیں، متحرک تبدیلیوں کی نگرانی کریں، اور بالواسطہ پرامپٹ انجیکشن حملوں سے دفاع کریں
- **بہترین طریقے شامل کریں**: قائم شدہ سیکیورٹی بنیادی اصولوں (محفوظ کوڈنگ، سرور سختی، زیرو ٹرسٹ) کو MCP مخصوص کنٹرولز کے ساتھ جامع تحفظ کے لیے ملائیں

# MCP سیکیورٹی آرکیٹیکچر اور کنٹرولز

جدید MCP نفاذ کو ایسے تہہ دار سیکیورٹی طریقہ کار کی ضرورت ہے جو روایتی سافٹ ویئر کی سیکیورٹی اور AI مخصوص خطرات دونوں کو حل کرے۔ MCP کی تیزی سے ترقی کرتی ہوئی وضاحت اپنی سیکیورٹی کنٹرولز کو بہتر کر رہی ہے، جس سے اسے ادارتی سیکیورٹی آرکیٹیکچرز اور قائم شدہ بہترین طریقوں کے ساتھ بہتر ہم آہنگی ملتی ہے۔

[Microsoft Digital Defense Report](https://aka.ms/mddr) کی تحقیق سے ظاہر ہوتا ہے کہ **98% رپورٹ شدہ خلاف ورزیوں کو مضبوط سیکیورٹی حفظان صحت سے روکا جا سکتا ہے**۔ سب سے مؤثر تحفظ کی حکمت عملی بنیادی سیکیورٹی طریقوں کو MCP مخصوص کنٹرولز کے ساتھ ملاتی ہے—ثابت شدہ بنیادی حفاظتی تدابیر مجموعی سیکیورٹی خطرے کو کم کرنے میں سب سے مؤثر ہیں۔

## موجودہ سیکیورٹی منظر نامہ

> **نوٹ:** یہ معلومات MCP سیکیورٹی معیارات کی عکاسی کرتی ہے جیسا کہ **5 فروری 2026** کو ہے، جو **MCP Specification 2025-11-25** سے ہم آہنگ ہے۔ MCP پروٹوکول تیزی سے ارتقا پذیر ہے، اور آئندہ نفاذ میں ممکنہ طور پر نئی تصدیقی پیٹرن اور بہتر کنٹرولز متعارف ہوں گے۔ تازہ ترین رہنمائی کے لیے ہمیشہ موجودہ [MCP Specification](https://spec.modelcontextprotocol.io/)، [MCP GitHub repository](https://github.com/modelcontextprotocol)، اور [security best practices documentation](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) کو دیکھیں۔

> **آگے دیکھیں:** `2026-07-28` ریلیز امیدوار اجازت کو مزید مضبوط بناتا ہے — کلائنٹس کو اجازت کی جوابات پر `iss` پیرامیٹر کی تصدیق کرنی ہوگی (RFC 9207)، Dynamic Client Registration کے دوران OpenID Connect `application_type` کا اعلان کرنا ہوگا، اور رجسٹرڈ اسناد کو جاری کرنے والے اجازت سرور سے منسلک کرنا ہوگا۔ اجازت SEPs کی مکمل فہرست کے لیے دیکھیں [What's Changing in MCP: The 2026-07-28 Release Candidate](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md)۔

## 🏔️ MCP سیکیورٹی سمٹ ورکشاپ (شرپا)

**عملی سیکیورٹی ٹریننگ** کے لیے، ہم انتہائی سفارش کرتے ہیں **MCP سیکیورٹی سمٹ ورکشاپ** (شرپا) - Microsoft Azure میں MCP سرورز کو محفوظ بنانے کے لیے جامع رہنمائی۔

### ورکشاپ کا جائزہ

[MCP Security Summit Workshop](https://azure-samples.github.io/sherpa/) عملی، قابل عمل سیکیورٹی ٹریننگ فراہم کرتا ہے جو "کمزور → استحصال → درستگی → توثیق" کے ثابت شدہ طریقہ کار پر مبنی ہے۔ آپ:

- **خراب کرکے سیکھیں**: جان بوجھ کر غیر محفوظ سرورز کو استحصال کر کے کمزوریوں کا تجربہ کریں
- **Azure-نیٹو سیکیورٹی استعمال کریں**: Azure Entra ID، Key Vault، API Management، اور AI Content Safety کا فائدہ اٹھائیں
- **ڈیفنس-ان-ڈیپتھ اپروچ پر عمل کریں**: کیمپوں سے گزرتے ہوئے جامع سیکیورٹی تہہ بنائیں
- **OWASP معیارات اپنائیں**: ہر تکنیک [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) سے منسلک ہے
- **پیداواری کوڈ حاصل کریں**: کام کرنے والے، تجربہ شدہ نفاذ حاصل کریں

| خطرہ | وضاحت | Azure سے بچاؤ |

### MCP تصدیق کی ترقی

MCP وضاحت نے تصدیق اور اجازت کے لیے اپنے طریقے میں نمایاں ترقی کی ہے:

- **اصل طریقہ کار**: ابتدائی وضاحتیں ڈویلپرز کو اپنی مرضی کے مطابق تصدیقی سرورز بنانے کی ضرورت دیتی تھیں، جبکہ MCP سرورز OAuth 2.0 اجازت سرور کے طور پر کام کرتے تھے جو براہ راست صارف کی تصدیق کرتے تھے
- **موجودہ معیار (2025-11-25)**: اپ ڈیٹ وضاحت MCP سرورز کو اجازت دیتی ہے کہ وہ تصدیق کو بیرونی شناخت فراہم کنندگان (جیسے Microsoft Entra ID) کو تفویض کریں، جس سے سیکیورٹی کی حالت بہتر ہوتی ہے اور نفاذ کی پیچیدگی کم ہوتی ہے
- **ٹرانسپورٹ لیئر سیکیورٹی**: مقامی (STDIO) اور ریموٹ (Streamable HTTP) کنکشنز کے لیے مناسب تصدیقی پیٹرنز کے ساتھ محفوظ ٹرانسپورٹ میکانزم کی بہتر حمایت

## تصدیق اور اجازت کی سیکیورٹی

### موجودہ سیکیورٹی چیلنجز

جدید MCP نفاذ متعدد تصدیق اور اجازت کے چیلنجز کا سامنا کرتے ہیں:

### خطرات اور حملے کی سمتیں

- **غلط طریقے سے ترتیب دی گئی اجازت لاجک**: MCP سرورز میں خراب اجازت کا نفاذ حساس معلومات کو بے نقاب کر سکتا ہے اور غلط طریقے سے رسائی کنٹرول لگاتا ہے
- **OAuth ٹوکن کا سمجھوتہ**: مقامی MCP سرور کے ٹوکن کی چوری حملہ آوروں کو سرورز کی جعلی شناخت اور نیچے کے سروسز تک رسائی دیتی ہے
- **ٹوکن پاس تھرو کے نقصانات**: ٹوکن کی غلط ہینڈلنگ سیکیورٹی کنٹرولز کو بائی پاس کرتی ہے اور جوابدہی میں خلا پیدا کرتی ہے
- **زیادہ اجازتیں**: زیادہ مراعات یافتہ MCP سرور کم از کم مراعات کے اصول کی خلاف ورزی کرتے ہیں اور حملے کے امکانات بڑھاتے ہیں

#### ٹوکن پاس تھرو: ایک اہم منفی رجحان

موجودہ MCP اجازت وضاحت میں **ٹوکن پاس تھرو واضح طور پر ممنوع** ہے کیونکہ اس کے سنگین سیکیورٹی نتائج ہیں:

##### سیکیورٹی کنٹرولز کی گزرگاہ
- MCP سرورز اور نیچے کے APIs اہم سیکیورٹی کنٹرولز (جیسے ریٹ لمٹنگ، درخواست کی توثیق، ٹریفک کی نگرانی) نافذ کرتے ہیں جو مناسب ٹوکن تصدیق پر منحصر ہیں
- براہ راست کلائنٹ سے API ٹوکن کا استعمال ان ضروری حفاظتی تدابیر کو بائی پاس کر دیتا ہے، جس سے سیکیورٹی فن تعمیری متاثر ہوتا ہے

##### جوابدہی اور آڈٹ کے چیلنجز  
- MCP سرورز یہ فرق نہیں کر سکتے کہ کون سے کلائنٹس اپ اسٹریم جاری کردہ ٹوکن استعمال کر رہے ہیں، جو آڈٹ ٹریک کو توڑ دیتا ہے
- نیچے کے ریسورس سرور کے لاگز غلط درخواست کی اصل معلوم کراتے ہیں بجائے اس کے کہ اصل MCP سرور وسط کا پتہ چلائیں
- واقعہ کی تفتیش اور تعمیل کی آڈٹنگ بہت مشکل ہو جاتی ہے

##### ڈیٹا کی چوری کے خطرات
- غیر توثیق شدہ ٹوکن دعوے چوری کیے گئے ٹوکن کے ساتھ نقصان دہ فریقین کو MCP سرورز کو ڈیٹا خارج کرنے کے لیے پروکسی کے طور پر استعمال کرنے کی اجازت دیتے ہیں
- اعتماد کی حد کی خلاف ورزیاں غیر مجاز رسائی کے نمونے پیدا کرتی ہیں جو مقصود سیکیورٹی کنٹرولز کو بائی پاس کرتی ہیں

##### متعدد سروس حملے کی سمتیں
- سمجھوتہ شدہ ٹوکن متعدد سروسز میں قبول کیے جاتے ہیں جو منسلک نظاموں میں عرضی حرکت کو ممکن بناتے ہیں
- سروسز کے درمیان اعتماد کی بنیادیں ٹوٹ سکتی ہیں جب ٹوکن کی اصل کی تصدیق ممکن نہ ہو

### سیکیورٹی کنٹرولز اور تخفیف

**اہم حفاظتی تقاضے:**

> **لازمی:** MCP سرورز **کسی بھی ایسے ٹوکن کو قبول نہیں کریں گے جو خاص طور پر MCP سرور کے لیے جاری نہ کیے گئے ہوں

#### تصدیق اور اجازت کے کنٹرولز

- **کڑی اجازت کا جائزہ**: MCP سرور کی اجازت لاجک کا جامع آڈٹ کریں تاکہ صرف مستند صارفین اور کلائنٹس حساس وسائل تک رسائی حاصل کر سکیں
  - **نفاذ رہنمائی**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **شناخت کا انضمام**: [Using Microsoft Entra ID for MCP Server Authentication](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **محفوظ ٹوکن مینجمنٹ**: [مائیکروسافٹ کے ٹوکن کی توثیق اور lifecycle بہترین طریقے](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) نافذ کریں
  - ٹوکن آڈینس کے دعووں کی MCP سرور کی شناخت سے مطابقت کی تصدیق کریں
  - مناسب ٹوکن گردش اور معیاد ختم ہونے کی پالیسیاں نافذ کریں
  - ٹوکن ری پلے حملوں اور غیر مجاز استعمال کو روکیں

- **محفوظ ٹوکن ذخیرہ**: آرام کے وقت اور منتقلی کے دوران دونوں میں ٹوکن کے ذخیرہ کو انکرپشن کے ساتھ محفوظ بنائیں
  - **بہترین طریقے**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### رسائی کنٹرول کا نفاذ

- **کم از کم مراعات کا اصول**: MCP سرورز کو صرف مطلوبہ فعالیت کے لیے کم از کم اجازتیں دیں
  - مراعات کے جائزوں اور اپ ڈیٹس کو باقاعدگی سے کریں تاکہ مراعات میں اضافے کو روکا جا سکے
  - **مائیکروسافٹ دستاویزات**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **رول پر مبنی رسائی کنٹرول (RBAC)**: باریک بینی سے رول اسائنمنٹس کریں
  - مخصوص وسائل اور اعمال کے لیے رولز کو محدود کریں
  - وسیع یا غیر ضروری اجازتوں سے گریز کریں جو حملے کے امکانات بڑھائیں

- **مسلسل اجازتی نگرانی**: رسائی کا مستقل آڈٹ اور نگرانی نافذ کریں
  - اجازت استعمال کے نمونوں کی غیر معمولی چیزوں کے لیے نگرانی کریں
  - غیر ضروری یا غیر استعمال شدہ مراعات کو فوری طور پر درست کریں

## AI مخصوص سیکیورٹی خطرات

### پرامپٹ انجیکشن اور ٹول مینیپولیشن حملے

جدید MCP نفاذ جدید AI مخصوص حملہ آور سمتوں کا سامنا کرتے ہیں جنہیں روایتی سیکیورٹی تدابیر مکمل طور پر حل نہیں کر سکتیں:

#### **غیر مستقیم پرامپٹ انجیکشن (کراس ڈومین پرامپٹ انجیکشن)**

**غیر مستقیم پرامپٹ انجیکشن** MCP فعال AI سسٹمز میں سب سے اہم کمزوریوں میں سے ایک ہے۔ حملہ آور بیرونی مواد—دستاویزات، ویب صفحات، ای میلز، یا ڈیٹا ذرائع—میں نقصان دہ ہدایات چھپا دیتے ہیں جنہیں AI سسٹمز بعد میں جائز کمانڈز کے طور پر پروسیس کرتے ہیں۔

**حملہ کی صورتیں:**
- **دستاویز پر مبنی انجیکشن**: پروسیس ہونے والی دستاویزات میں پوشیدہ نقصان دہ ہدایات جو AI کو غیر ارادی کارروائیوں پر مجبور کرتی ہیں
- **ویب مواد کا استحصال**: متاثرہ ویب صفحات جن میں embedded prompts ہوتے ہیں جو scraped ہو کر AI رویہ کو کنٹرول کرتے ہیں
- **ای میل پر مبنی حملے**: ای میلز میں نقصان دہ پرامپٹس جو AI اسسٹنٹس کو معلومات افشاں کرنے یا غیر مجاز کارروائیاں کرنے پر مجبور کرتے ہیں
- **ڈیٹا سورس کی آلودگی**: متاثرہ ڈیٹا بیسز یا APIs جو AI سسٹمز کو آلودہ مواد فراہم کرتے ہیں

**حقیقی دنیا کا اثر**: یہ حملے ڈیٹا کی چوری، پرائیویسی کی خلاف ورزی، نقصان دہ مواد کی پیداوار، اور صارف کی تعاملات میں مداخلت کا سبب بن سکتے ہیں۔ تفصیلی تجزیہ کے لیے ملاحظہ کریں [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)۔

![پرامپٹ انجیکشن حملے کا خاکہ](../../../translated_images/ur/prompt-injection.ed9fbfde297ca877.webp)

#### **ٹول زہر آلودگی کے حملے**

**ٹول زہر آلودگی** MCP ٹولز کو بیان کرنے والے میٹا ڈیٹا کو نشانہ بناتی ہے، جس میں LLMs ٹول کی وضاحتوں اور پیرامیٹرز کی ترجمانی کرتے ہوئے عمل درآمد کے فیصلے کرتے ہیں۔

**حملہ کے طریقے:**
- **میٹا ڈیٹا کی چالاکی**: حملہ آور ٹول کی وضاحتوں، پیرامیٹر تعریفوں، یا استعمال کی مثالوں میں نقصان دہ ہدایات گھلاتے ہیں
- **غیر مرئی ہدایات**: ٹول میٹا ڈیٹا میں پوشیدہ پرامپٹس جو AI ماڈلز پروسیس کرتے ہیں لیکن انسان صارفین نہیں دیکھ سکتے
- **متحرک ٹول میں ترمیم ("رگ پل")**: صارفین کی منظوری کے بعد ٹولز میں نقصان دہ کارروائیاں کرنے کے لیے بعد میں تبدیلیاں کی جاتی ہیں بغیر صارف کی آگاہی کے
- **پیرامیٹر انجیکشن**: ٹول پیرامیٹر اسکیموں میں شامل کیا گیا نقصان دہ مواد جو ماڈل کے رویے کو متاثر کرتا ہے


**میزبان سرور کے خطرات**: ریموٹ MCP سرورز میں زیادہ خطرات ہوتے ہیں کیونکہ ٹول کی تعریفات کو صارف کی ابتدائی اجازت کے بعد اپ ڈیٹ کیا جا سکتا ہے، جس سے ایسے مناظر پیدا ہوتے ہیں جہاں پہلے محفوظ ٹولز نقصان دہ بن جاتے ہیں۔ جامع تجزیہ کے لیے، دیکھیں [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)۔

![Tool Injection Attack Diagram](../../../translated_images/ur/tool-injection.3b0b4a6b24de6bef.webp)

#### **اضافی AI حملے کے راستے**

- **کراس ڈومین پرامپٹ انجیکشن (XPIA)**: پیچیدہ حملے جو مختلف ڈومینز کے مواد کا فائدہ اٹھا کر سیکیورٹی کنٹرولز کو عبور کرتے ہیں
- **ڈائنامک صلاحیت کی تبدیلی**: ٹول کی صلاحیتوں میں حقیقی وقت کی تبدیلیاں جو ابتدائی سیکیورٹی جائزوں سے بچ جاتی ہیں
- **کانٹیکسٹ ونڈو زہر آلود کرنا**: ایسے حملے جو بڑے کانٹیکسٹ ونڈوز کو نقصان دہ ہدایات چھپانے کے لیے چالاکی سے استعمال کرتے ہیں
- **ماڈل کنفیوژن حملے**: ماڈل کی حدود کو استعمال کرتے ہوئے غیر متوقع یا غیر محفوظ رویے پیدا کرنا


### AI سیکیورٹی کے خطرات کا اثر

**اعلی اثرات کے نتائج:**
- **ڈیٹا نکلوانا**: حساس کاروباری یا ذاتی ڈیٹا تک غیر مجاز رسائی اور چوری
- **رازداری کی خلاف ورزیاں**: ذاتی شناختی معلومات (PII) اور رازدارانہ کاروباری ڈیٹا کا انکشاف
- **سسٹم میں چالاکی**: اہم نظاموں اور ورک فلو میں غیر ارادی تبدیلیاں
- **اسناد کی چوری**: توثیق ٹوکنز اور سروس اسناد کی خلاف ورزی
- **جانبی حرکت**: متاثرہ AI نظاموں کا استعمال وسیع نیٹ ورک حملوں کے لیے مرکز کے طور پر

### مائیکروسافٹ AI سیکیورٹی حل

#### **AI پرامپٹ شیلڈز: انجیکشن حملوں کے خلاف جدید تحفظ**

مائیکروسافٹ **AI پرامپٹ شیلڈز** براہ راست اور بالواسطہ پرامپٹ انجیکشن حملوں کے خلاف کثیر سطحی سیکیورٹی کے ذریعے مکمل دفاع فراہم کرتے ہیں:

##### **بنیادی حفاظتی طریقہ کار:**

1. **جدید شناخت اور فلٹرنگ**
   - مشین لرننگ الگورتھمز اور NLP تکنیکوں کے ذریعے خارجی مواد میں نقصان دہ ہدایات کا پتہ لگانا
   - ڈاکومنٹس، ویب پیجز، ای میلز، اور ڈیٹا ذرائع کا حقیقی وقت میں تجزیہ کرنا تاکہ شامل خطرات دریافت ہوں
   - جائز اور نقصان دہ پرامپٹ پیٹرنز کی سیاق و سباق کی سمجھ

2. **اسپاٹلائٹنگ تکنیکس**  
   - قابل اعتماد نظام کی ہدایات اور ممکنہ طور پر متاثرہ خارجی ان پٹس کے درمیان تمیز
   - ٹیکسٹ ٹرانسفارمیشن کے طریقے جو ماڈل کی مطابقت کو بہتر بناتے ہیں اور نقصان دہ مواد کو الگ کرتے ہیں
   - AI نظاموں کو درست ہدایتی ہئیرارکی برقرار رکھنے اور انجیکشن کمانڈز کو نظر انداز کرنے میں مدد دینا

3. **ڈیلمیٹر اور ڈیٹا مارکنگ سسٹمز**
   - قابل اعتماد نظام پیغامات اور خارجی ان پٹ ٹیکسٹ کے درمیان واضح حد بندی
   - خاص مارکرز جو قابل اعتماد اور غیر قابل اعتماد ڈیٹا ذرائع کے درمیان حدود کو نمایاں کرتے ہیں
   - واضح علیحدگی ہدایات کی الجھن اور غیر مجاز کمانڈز کی عملدرآمد کو روکتی ہے

4. **مسلسل خطرے کی معلومات**
   - مائیکروسافٹ نئے حملے کے نمونوں کی مسلسل نگرانی کرتا ہے اور دفاع کو اپ ڈیٹ کرتا ہے
   - نئی انجیکشن تکنیکوں اور حملے کے راستوں کے لیے پروایکٹو خطرہ شکار
   - بدلتے ہوئے خطرات کے خلاف مؤثر رہنے کے لیے باقاعدہ سیکیورٹی ماڈل اپ ڈیٹس

5. **Azure مواد کی حفاظت کا انضمام**
   - مکمل Azure AI مواد کی حفاظت کے سوٹ کا حصہ
   - جیل بریک کوششوں، نقصان دہ مواد، اور سیکیورٹی پالیسی کی خلاف ورزیوں کے لیے اضافی شناخت
   - AI ایپلیکیشن کے اجزاء کے درمیان متحدہ سیکیورٹی کنٹرولز

## جدید MCP سیکیورٹی خطرات

### سیشن ہائی جیکنگ کی کمزوریاں

**سیشن ہائی جیکنگ** ایک سنگین حملہ ہے جو اسٹیٹ فل MCP نفاذ میں ہوتا ہے جہاں غیر مجاز فریق جائز سیشن آئیڈینٹیفائرز حاصل کر کے کلائنٹس کی نقل کرتے ہیں اور غیر مجاز کارروائیاں کرتے ہیں۔

#### **حملے کے مناظر اور خطرات**

- **سیشن ہائی جیک پرامپٹ انجیکشن**: چوری شدہ سیشن IDs کے حامل حملہ آور سیشن اسٹیٹ شیئر کرنے والے سرورز میں نقصان دہ واقعات انجیکٹ کرتے ہیں، جو ممکنہ طور پر نقصان دہ کارروائیاں یا حساس ڈیٹا تک رسائی کا باعث بنتے ہیں
- **براہ راست نقل**: چوری شدہ سیشن IDs براہ راست MCP سرور کالز کی اجازت دیتی ہیں جو توثیق کو عبور کر کے حملہ آوروں کو جائز صارفین کی طرح برتاؤ کرنے دیتی ہیں
- **متاثر رشاری شدہ اسٹریمز**: حملہ آور درخواستوں کو وقت سے پہلے ختم کر سکتے ہیں، جس کی وجہ سے جائز کلائنٹس ممکنہ طور پر نقصان دہ مواد کے ساتھ دوبارہ شروع ہوتے ہیں

#### **سیشن مینجمنٹ کے لیے حفاظتی کنٹرولز**

**اہم تقاضے:**
- **اجازت کی تصدیق**: MCP سرورز جو اجازت پر عمل کرتے ہیں **تمام** آنے والی درخواستوں کی تصدیق کریں اور **سیشن پر تصدیق کے لیے انحصار نہ کریں**
- **محفوظ سیشن جنریشن**: کرپٹوگرافک طور پر محفوظ، غیر معین سیشن IDs کا استعمال جو محفوظ رینڈم نمبر جنریٹرز کے ساتھ تیار ہوں
- **صارف مخصوص پابندی**: سیشن IDs کو صارف مخصوص معلومات سے منسلک کریں جیسے `<user_id>:<session_id>` تاکہ کراس یوزر سیشن کا غلط استعمال روکا جا سکے
- **سیشن لائف سائیکل مینجمنٹ**: مناسب میعاد ختم ہونے، گردش، اور باطل کرنے کے اقدامات نافذ کریں تاکہ ممکنہ کمزوریوں کو محدود کیا جا سکے
- **ٹرانسپورٹ سیکیورٹی**: تمام مواصلات کے لیے HTTPS لازمی تاکہ سیشن ID اعتراض کو روکا جا سکے

### کنفیوذ ڈیپٹی مسئلہ

**کنفیوذ ڈیپٹی مسئلہ** اس وقت پیدا ہوتا ہے جب MCP سرورز کلائنٹس اور تیسرے فریق کی خدمات کے درمیان توثیق پروکسی کے طور پر کام کرتے ہیں، جو جامد کلائنٹ ID کے استحصال کے ذریعے اجازت کو عبور کرنے کے مواقع پیدا کرتا ہے۔

#### **حملے کے طریقہ کار اور خطرات**

- **کوکی بیسڈ رضامندی عبور**: سابقہ صارف کی توثیق رضامندی کوکیز بناتی ہے جس کا حملہ آور سازش شدہ ری ڈائریکٹ URIs کے ساتھ نقصان دہ اجازت درخواستوں کے ذریعے فائدہ اٹھاتا ہے
- **اجازت کوڈ چوری**: موجودہ رضامندی کوکیز اجازت سرورز کو رضامندی سکرینز چھوڑنے پر مجبور کرتی ہیں، کوڈز کو حملہ آور کے کنٹرول کردہ اینڈ پوائنٹس پر ری ڈائریکٹ کیا جاتا ہے
- **غیر مجاز API رسائی**: چوری شدہ اجازت کوڈز ٹوکن کے تبادلے اور صارف کی نقل کے لیے استعمال ہوتے ہیں بغیر واضح اجازت کے

#### **نقصان کو کم کرنے کی حکمت عملیاں**

**لازمی کنٹرولز:**
- **واضح رضامندی کی ضروریات**: MCP پراکسی سرورز جو جامد کلائنٹ IDs استعمال کرتے ہیں **ہر متحرک رجسٹر شدہ کلائنٹ کے لیے صارف کی رضامندی حاصل کریں**
- **OAuth 2.1 سیکیورٹی نفاذ**: تمام اجازت درخواستوں کے لیے موجودہ OAuth سیکیورٹی بہترین طریقے بشمول PKCE (پروف کیی فار کوڈ ایکسچینج) پر عمل کریں
- **سخت کلائنٹ کی تصدیق**: ری ڈائریکٹ URIs اور کلائنٹ شناخت کنندگان کی سخت تصدیق نافذ کریں تاکہ استحصال کو روکا جا سکے

### ٹوکن پاس تھرو کمزوریاں  

**ٹوکن پاس تھرو** ایک واضح اینٹی پیٹرن ہے جہاں MCP سرورز کلائنٹ ٹوکنز کو مناسب تصدیق کے بغیر قبول کرتے ہیں اور انہیں نیچے API کو بھیج دیتے ہیں، جس سے MCP اجازت کی وضاحتوں کی خلاف ورزی ہوتی ہے۔

#### **سیکیورٹی پیچیدگیاں**

- **کنٹرول کا خاتمہ**: کلائنٹ سے API تک براہ راست ٹوکن کا استعمال اہم ریٹ لمٹنگ، تصدیق، اور مانیٹرنگ کنٹرولز کو عبور کرتا ہے
- **آڈٹ ٹریل کی خرابی**: اپ اسٹریم جاری کردہ ٹوکنز صارف کی شناخت ناممکن بناتے ہیں، جس سے واقعہ کی تحقیقات متاثر ہوتی ہے
- **پراکسی کی بنیاد پر ڈیٹا نکالنا**: غیر تصدیق شدہ ٹوکن حملہ آوروں کو سرورز کو غیر مجاز ڈیٹا رسائی کے لیے پراکسی کے طور پر استعمال کرنے کی اجازت دیتے ہیں
- **اعتماد کی حد کی خلاف ورزی**: نیچے کی سطح کی خدمات کی اعتماد کی مفروضات ٹوکن کی اصلیت کی تصدیق نہ ہونے پر متاثر ہو سکتی ہیں
- **ملٹی سروس حملے کی توسیع**: چوری شدہ ٹوکنز کئی خدمات میں قبول کیے جانے پر جانبی حرکت کو ممکن بناتے ہیں

#### **لازمی حفاظتی کنٹرولز**

**ناقابل مذاکرہ تقاضے:**
- **ٹوکن کی تصدیق**: MCP سرورز **وہ ٹوکن قبول نہیں کریں گے جو واضح طور پر MCP سرور کے لیے جاری نہیں کیے گئے**
- **ناظرین کی تصدیق**: ہمیشہ ٹوکن کے ناظرین کے دعووں کی تصدیق کریں کہ وہ MCP سرور کی شناخت کے مطابق ہوں
- **صحیح ٹوکن لائف سائیکل**: مختصر مدت والے رسائی ٹوکنز کے محفوظ گردش کے طریقہ کار نافذ کریں


## AI نظاموں کے لیے سپلائی چین سیکیورٹی

سپلائی چین سیکیورٹی روایتی سافٹ ویئر انحصار سے آگے بڑھ کر پورے AI ماحولیاتی نظام کو گھیر لیتی ہے۔ جدید MCP نفاذ کو تمام AI متعلقہ اجزاء کی سخت جانچ اور نگرانی کرنی چاہیے کیونکہ ہر ایک ممکنہ کمزوریوں کا تعارف کراتا ہے جو نظام کی سالمیت کو متاثر کر سکتی ہیں۔

### توسیع شدہ AI سپلائی چین اجزاء

**روایتی سافٹ ویئر انحصار:**
- اوپن سورس لائبریریز اور فریم ورکس
- کنٹینر امیجز اور بنیادی نظام  
- ترقیاتی ٹولز اور بلڈ پائپ لائنز
- انفراسٹرکچر کے اجزاء اور خدمات

**AI مخصوص سپلائی چین عناصر:**
- **بنیادی ماڈلز**: مختلف فراہم کنندگان کے پری ٹرینڈ ماڈلز جن کی اصلیت کی تصدیق ضروری ہے
- **ایمبیڈنگ سروسز**: خارجی ویکٹرائزیشن اور معنوی تلاش کی سروسز
- **سیاق و سباق فراہم کرنے والے**: ڈیٹا ذرائع، علم کے اڈے، اور دستاویزات کے ذخیرے  
- **تیسری پارٹی API**: بیرونی AI خدمات، ML پائپ لائنز، اور ڈیٹا پروسیسنگ اینڈ پوائنٹس
- **ماڈل آرٹیفیکٹس**: وزن، کنفیگریشنز، اور فائن ٹونڈ ماڈل کے متغیرات
- **ٹریننگ ڈیٹا ذرائع**: ماڈل کی تربیت اور فائن ٹوننگ کے لیے ڈیٹا سیٹس

### جامع سپلائی چین سیکیورٹی حکمت عملی

#### **جزو کی تصدیق اور اعتماد**
- **اصل کی توثیق**: انضمام سے پہلے تمام AI اجزاء کی اصلیت، لائسنسنگ، اور سالمیت کی تصدیق کریں
- **سیکیورٹی جائزہ**: ماڈلز، ڈیٹا ذرائع، اور AI خدمات کے لیے کمزوری سکین اور سیکیورٹی جائزے کریں
- **ساکھ کا تجزیہ**: AI سروس فراہم کنندگان کی سیکیورٹی کا ریکارڈ اور طریقہ کار کا جائزہ لیں
- **مطابقت کی تصدیق**: یقینی بنائیں کہ تمام اجزاء تنظیمی سیکیورٹی اور قواعد و ضوابط کی ضروریات پر پورے اُترتے ہیں

#### **محفوظ ڈیپلائمنٹ پائپ لائنز**  
- **خودکار CI/CD سیکیورٹی**: خودکار ڈیپلائمنٹ پائپ لائنز میں سیکیورٹی سکیننگ کو شامل کریں
- **آرٹیفیکٹ سالمیت**: تمام تعینات شدہ آرٹیفیکٹس (کوڈ، ماڈلز، کنفیگریشنز) کے لیے کرپٹوگرافک تصدیق نافذ کریں
- **مرتبہ وار ڈیپلائمنٹ**: ہر مرحلے پر سیکیورٹی کی تصدیق کے ساتھ ترقیاتی ڈیپلائمنٹ حکمت عملیاں استعمال کریں
- **قابل اعتماد آرٹیفیکٹ ریپوزٹریز**: صرف تصدیق شدہ، محفوظ آرٹیفیکٹ ریجسٹریز اور ریپوز سے تعینات کریں

#### **مسلسل نگرانی اور ردعمل**
- **انحصار سکیننگ**: تمام سافٹ ویئر اور AI جزو انحصار کے لیے جاری کمزوری نگرانی
- **ماڈل کی نگرانی**: ماڈل کے رویے، کارکردگی میں تبدیلی، اور سیکیورٹی انومالیز کا مسلسل جائزہ
- **سروس ہیلتھ ٹریکنگ**: خارجی AI خدمات کی دستیابی، سیکیورٹی واقعات، اور پالیسی تبدیلیوں کی نگرانی
- **خطرے کی معلومات کا انضمام**: AI اور ML سیکیورٹی کے خطرات کے مخصوص خطرہ فیڈز کو شامل کریں

#### **رسائی کنٹرول اور کم از کم مراعات**
- **جزو کی سطح کی اجازتیں**: کاروباری ضرورت کی بنیاد پر ماڈلز، ڈیٹا، اور خدمات تک رسائی محدود کریں
- **سروس اکاؤنٹ مینجمنٹ**: کم سے کم مطلوبہ اجازتوں کے ساتھ مخصوص سروس اکاؤنٹس نافذ کریں
- **نیٹ ورک تقسیم**: AI اجزاء کو الگ کریں اور خدمات کے درمیان نیٹ ورک رسائی محدود کریں
- **API گیٹ وے کنٹرولز**: خارجی AI خدمات تک رسائی کو کنٹرول اور مانیٹر کرنے کے لیے مرکزی API گیٹ وے استعمال کریں

#### **واقعے کے ردعمل اور بحالی**
- **تیز رفتار ردعمل کے طریقہ کار**: متاثرہ AI اجزاء کو پیچ کرنے یا تبدیل کرنے کے لیے قائم شدہ عمل
- **اسناد کی گردش**: راز، API کلیدوں، اور سروس اسناد کو خودکار گردش کرنے والے نظام
- **رول بیک کی صلاحیتیں**: AI اجزاء کے پچھلے معروف اچھے ورژنز پر جلدی واپس جانے کی صلاحیت
- **سپلائی چین خلاف ورزی کی بحالی**: اپ اسٹریم AI سروس کے متاثر ہونے پر مخصوص ردعمل کے طریقہ کار

### مائیکروسافٹ سیکیورٹی ٹولز اور انضمام

**GitHub Advanced Security** مکمل سپلائی چین تحفظ فراہم کرتا ہے جس میں شامل ہیں:
- **راز سکیننگ**: ذخیرہ میں اسناد، API کلیدوں، اور ٹوکنز کا خودکار پتہ لگانا
- **انحصار سکیننگ**: اوپن سورس انحصار اور لائبریریز کے لیے کمزوری کا جائزہ
- **CodeQL تجزیہ**: سیکیورٹی کمزوریوں اور کوڈنگ مسائل کے لیے جامد کوڈ تجزیہ
- **سپلائی چین بصیرت**: انحصار کی صحت اور سیکیورٹی کی حالت کی واضح معلومات

**Azure DevOps اور Azure Repos انضمام:**
- مائیکروسافٹ کی ترقیاتی پلیٹ فارمز میں سیکیورٹی سکیننگ کا ہموار انضمام
- AI کام کے بوجھ کے لیے Azure پائپ لائنز میں خودکار سیکیورٹی چیکس
- محفوظ AI جزو کی تعیناتی کے لیے پالیسی نفاذ

**مائیکروسافٹ کے اندرونی طریقہ کار:**
مائیکروسافٹ تمام مصنوعات میں وسیع سپلائی چین سیکیورٹی طریقہ کار نافذ کرتا ہے۔ [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) میں ثابت شدہ طریقوں کے بارے میں جانیں۔


## بنیادی سیکیورٹی بہترین طریقے

MCP نفاذ آپ کی تنظیم کی موجودہ سیکیورٹی صورتحال کو اختیار کرتے ہیں اور اس پر تعمیر کرتے ہیں۔ بنیادی سیکیورٹی طریقوں کو مضبوط کرنا AI نظاموں اور MCP تعیناتیوں کی مجموعی سیکیورٹی کو نمایاں طور پر بڑھاتا ہے۔

### بنیادی سیکیورٹی اصول

#### **محفوظ ترقیاتی طریقے**
- **OWASP مطابقت**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) ویب ایپلیکیشن کمزوریوں سے حفاظت
- **AI مخصوص تحفظات**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) کے لیے کنٹرول نافذ کریں
- **محفوظ راز مینجمنٹ**: ٹوکنز، API کیز، اور حساس کنفیگریشن ڈیٹا کے لیے مخصوص والٹس کا استعمال
- **اینڈ ٹو اینڈ انکرپشن**: تمام ایپلیکیشن جزو اور ڈیٹا فلو میں محفوظ مواصلات نافذ کریں
- **ان پٹ کی توثیق**: تمام صارف ان پٹس، API پیرامیٹرز، اور ڈیٹا ذرائع کی سخت توثیق

#### **انفراسٹرکچر سخت کرنا**
- **کثیر العنصری توثیق**: تمام انتظامی اور سروس اکاؤنٹس کے لیے لازمی MFA
- **پیچ مینجمنٹ**: آپریٹنگ سسٹمز، فریم ورکس اور انحصار کے لیے خودکار اور بروقت پیچنگ  
- **شناخت فراہم کنندہ انضمام**: ادارہ جاتی شناخت فراہم کنندگان کے ذریعے مرکزی شناخت مینجمنٹ (Microsoft Entra ID، Active Directory)
- **نیٹ ورک تقسیم**: MCP کے اجزاء کی منطقی علیحدگی تاکہ جانبی حرکت کی صلاحیت محدود ہو
- **کم از کم اجازت کا اصول**: تمام نظام کے اجزاء اور اکاؤنٹس کے لیے کم سے کم ضروری اجازتیں

#### **سیکیورٹی نگرانی اور شناخت**
- **جامع لاگنگ**: AI ایپلیکیشن کی سرگرمیوں کی تفصیلی لاگنگ، بشمول MCP کلائنٹ-سرور تعاملات
- **SIEM انضمام**: غیر معمولی حالات کی شناخت کے لیے مرکزی سیکیورٹی معلومات اور واقعہ مینجمنٹ
- **رویے کا تجزیہ**: نظام اور صارف کے رویے میں غیر معمولی پیٹرنز کی شناخت کے لیے AI سے چلنے والی نگرانی
- **خطرے کی معلومات**: خارجی خطرے کے فیڈز اور خطرہ اشاریہ جات (IOCs) کا انضمام
- **واقعے کا ردعمل**: سیکیورٹی واقعات کی شناخت، ردعمل، اور بحالی کے لیے واضح طریقہ کار

#### **زیرو ٹرسٹ فن تعمیر**
- **کبھی نہ بھروسہ کریں، ہمیشہ تصدیق کریں**: صارفین، آلات، اور نیٹ ورک کنکشنز کی مسلسل تصدیق
- **مائیکرو تقسیم**: وھیل نیٹ ورک کنٹرولز جو انفرادی کام اور خدمات کو الگ کرتے ہیں
- **شناخت مرکزیت والی سیکیورٹی**: نیٹ ورک مقام کے بجائے تصدیق شدہ شناختوں پر مبنی سیکیورٹی پالیسیاں
- **مسلسل خطرے کا جائزہ**: موجودہ سیاق و سباق اور رویے کی بنیاد پر ڈائنامک سیکیورٹی صورتحال کی جانچ
- **مشروط رسائی**: خطرے کے عوامل، مقام، اور ڈیوائس اعتماد کے مطابق رسائی کنٹرولز

### ادارہ جاتی انضمام کے نمونے

#### **مائیکروسافٹ سیکیورٹی ماحولیاتی نظام انضمام**
- **Microsoft Defender for Cloud**: جامع کلاؤڈ سیکیورٹی صورتحال مینجمنٹ
- **Azure Sentinel**: AI کام کے بوجھ کی حفاظت کے لیے کلاؤڈ نیٹو SIEM اور SOAR صلاحیتیں
- **Microsoft Entra ID**: ادارہ جاتی شناخت اور رسائی مینجمنٹ کے لیے مشروط رسائی پالیسیاں
- **Azure Key Vault**: ہارڈویئر سیکیورٹی ماڈیول (HSM) کے ساتھ مرکزی راز مینجمنٹ
- **Microsoft Purview**: AI ڈیٹا ذرائع اور ورک فلو کے لیے ڈیٹا گورننس اور تعمیل

#### **مطابقت اور گورننس**
- **ریگولیٹری ہم آہنگی**: یقینی بنائیں کہ MCP نفاذ صنعت مخصوص تعمیل کی ضروریات کو پورا کرتے ہیں (GDPR، HIPAA، SOC 2)

- **ڈیٹا کی درجہ بندی**: AI نظاموں کے ذریعہ عمل کیے جانے والے حساس ڈیٹا کی مناسب درجہ بندی اور ہینڈلنگ  
- **آڈٹ ٹریلز**: ضوابط کی تعمیل اور عدالتی تحقیقات کے لیے جامع لاگنگ  
- **پرائیویسی کنٹرولز**: AI نظام کی ساخت میں پرائیویسی بذریعہ ڈیزائن اصولوں کا نفاذ  
- **تبدیلی کا انتظام**: AI نظام کی تبدیلیوں کے لیے سیکیورٹی جائزے کے باضابطہ عمل  

یہ بنیادی طریقے ایک مضبوط سیکیورٹی بنیاد قائم کرتے ہیں جو MCP مخصوص سیکیورٹی کنٹرولز کی مؤثریت کو بڑھاتے ہیں اور AI پر مبنی ایپلیکیشنز کے لیے جامع تحفظ فراہم کرتے ہیں۔  

## اہم سیکیورٹی نکات  

- **سطح بند سیکیورٹی طریقہ کار**: بنیادی سیکیورٹی طریقوں (محفوظ کوڈنگ، کم از کم استحقاق، سپلائی چین کی تصدیق، مسلسل نگرانی) کو AI مخصوص کنٹرولز کے ساتھ مربوط کریں تاکہ جامع تحفظ فراہم کیا جا سکے  

- **AI مخصوص خطرات کا منظرنامہ**: MCP نظام منفرد خطرات سے دوچار ہوتے ہیں جن میں پرامپٹ انجیکشن، ٹول زہریلا پن، سیشن ہائی جیکنگ، متشابہ ڈپٹی مسائل، ٹوکن پاس تھرو کی کمزوریاں، اور بہت زیادہ اجازتیں شامل ہیں جن کے لیے خاص بچاؤ کی ضرورت ہوتی ہے  

- **تصدیق اور اجازت نامہ کی عمدگی**: خارجی شناختی فراہم کنندگان (Microsoft Entra ID) کے ذریعے مضبوط تصدیق نافذ کریں، مناسب ٹوکن کی تصدیق کو یقینی بنائیں، اور کبھی بھی وہ ٹوکن قبول نہ کریں جو خاص طور پر آپ کے MCP سرور کے لیے جاری نہ کیے گئے ہوں  

- **AI حملوں کی روک تھام**: Microsoft Prompt Shields اور Azure Content Safety کا استعمال کرتے ہوئے غیر مستقیم پرامپٹ انجیکشن اور ٹول زہریلا پن حملوں سے دفاع کریں، ٹول میٹا ڈیٹا کی تصدیق کریں اور متحرک تبدیلیوں کی نگرانی کریں  

- **سیشن اور ٹرانسپورٹ سیکیورٹی**: صارف کی شناختوں سے منسلک، کرپٹوگرافک طور پر محفوظ، غیر تعیناتی سیشن آئی ڈیز استعمال کریں، مناسب سیشن لائف سائیکل مینجمنٹ نافذ کریں، اور سیشنز کو کبھی بھی تصدیق کے لیے استعمال نہ کریں  

- **OAuth سیکیورٹی بہترین طریقے**: متحرک طور پر رجسٹرڈ کلائنٹس کے لیے واضح صارف کی رضامندی کے ذریعے متشابہ ڈپٹی حملوں کی روک تھام کریں، PKCE کے ساتھ OAuth 2.1 کا درست نفاذ یقینی بنائیں، اور ری ڈائریکٹ URI کی سخت تصدیق کریں  

- **ٹوکن سیکیورٹی اصول**: ٹوکن پاس تھرو اینٹی پیٹرنز سے بچیں، ٹوکن آڈینس کلیمز کی تصدیق کریں، مختصر مدت کے ٹوکن استعمال کریں جن کی محفوظ گردش ہو، اور واضح اعتماد کی حدود برقرار رکھیں  

- **جامع سپلائی چین سیکیورٹی**: AI ماحولیاتی نظام کے تمام اجزاء (ماڈلز، ایمبیڈنگز، کانٹیکسٹ پرووائیڈرز، بیرونی APIs) کو روایتی سافٹ ویئر انحصار کی طرح ہی سخت سیکیورٹی کے ساتھ دیکھیں  

- **مسلسل ارتقاء**: تیزی سے تبدیل ہونے والی MCP وضاحتوں کے ساتھ اپ ٹو ڈیٹ رہیں، سیکیورٹی کمیونٹی کے معیارات میں حصہ ڈالیں، اور پروٹوکول کی ترقی کے ساتھ مطابقت پذیر سیکیورٹی رویے برقرار رکھیں  

- **مائیکروسافٹ سیکیورٹی انٹیگریشن**: Microsoft کے جامع سیکیورٹی ماحولیاتی نظام (Prompt Shields، Azure Content Safety، GitHub Advanced Security، Entra ID) سے استفادہ کریں تاکہ MCP تعیناتی کے تحفظ کو بہتر بنایا جا سکے
