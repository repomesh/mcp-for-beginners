# MCP பாதுகாப்பு: AI அமைப்புகளுக்கான விரிவான பாதுகாப்பு

[![MCP Security Best Practices](../../../translated_images/ta/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(இந்த பாடத்தின் வீடியோவை பார்க்க மேலுள்ள படத்தை கிளிக் செய்யவும்)_

பாதுகாப்பு AI அமைப்பு வடிவமைப்பிற்கு அடிப்படையாகும், அதனால் இது நமது இரண்டாவது பகுதியானது. இது Microsoft இன் [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) இலிருந்து **Secure by Design** கோரிக்கையோடு இணக்கமாக உள்ளது.

மாடல் Context Protocol (MCP) AI நடத்தும் பயன்பாடுகளில் சக்திவாய்ந்த புதிய திறன்களை கொண்டு வர while அசாதாரண பாதுகாப்பு சவால்களையும் அறிமுகப்படுத்துகிறது, இது பாரம்பரிய மென்பொருள் அபாயங்களுக்குத் துரிதமானது. MCP அமைப்புகள் இருந்தும் நிலையான பாதுகாப்பு கவலைகளையும் (பாதுகாப்பான குறியீடு எழுதுதல், குறைந்த அதிகாரம், வழங்கல் சங்கிலி பாதுகாப்பு) AI-க்கு தனித்துவமான அச்சுறுத்தல்கள், உதாரணமாக prompt injection, கருவி விஷம் சேர்க்கை, வர்த்தக தெறிப்பு, குழப்பம் படைத்த கீழ்த்தரவர் தாக்குதல், டோக்கன் பாத்திர வழிதவறல், மற்றும் தனிமாற்ற திறன் மாற்றங்களை எதிர்கொள்கின்றன.

இந்த பாடம் MCP பயன்பாடுகளில் மிக முக்கியமான பாதுகாப்பு அபாயங்களை ஆய்வு செய்கிறது — அங்கீகாரம், அங்கீகார உரிமை, மிகுதியான அனுமதிகள், மறைபுரிந்த prompt injection, செஷன் பாதுகாப்பு, குழப்பம் படைத்த கீழ்த்தரவர் பிரச்சினைகள், டோக்கன் மேலாண்மை, வழங்கல் சங்கிலி பாதிப்புகள் ஆகியவை உள்ளடங்கும். நீங்கள் Microsoft சமாளிப்பு கருவிகள் மாதிரி Prompt Shields, Azure Content Safety, மற்றும் GitHub Advanced Security போன்றவற்றை பயன்படுத்தி இந்த அபாயங்களை குறைப்பதில் செயல்படும் கட்டுப்பாடுகள் மற்றும் சிறந்த நடைமுறைகளை கற்றுக் கொள்வீர்கள்.

## கற்றல் இலக்குகள்

இந்த பாடத்தின் முடிவில், நீங்கள்:

- **MCP-க்கு தனித்துவமான அச்சுறுத்தல்களை அடையாளம் கண்டு கொள்ள வேண்டும்**: prompt injection, கருவி விஷம் சேர்க்கை, மிகுதியான அனுமதிகள், செஷன் தெறிப்பு, குழப்பம் படைத்த கீழ்த்தரவர் பிரச்சினைகள், டோக்கன் பாத்திர வழிதவறல், மற்றும் வழங்கல் சங்கிலி அபாயங்கள் போன்ற MCP அமைப்பு பாதுகாப்பு அபாயங்களை அறிந்துகொள்ள
- **பாதுகாப்பு கட்டுப்பாடுகளை நடைமுறைப்படுத்தவும்**: வலுவான அங்கீகாரம், குறைந்த அதிகார அணுகல், பாதுகாப்பான டோக்கன் மேலாண்மை, செஷன் பாதுகாப்பு கட்டுப்பாடுகள் மற்றும் வழங்கல் சங்கிலி சரிபார்ப்புகளை செயல்படுத்தவும்
- **Microsoft பாதுகாப்பு தீர்வுகளை பயன்படுத்து**: MCP வேலைபளுவுக்கான Microsoft Prompt Shields, Azure Content Safety மற்றும் GitHub Advanced Security ஐ புரிந்து கொண்டு பயன்படுத்தவும்
- **கருவி பாதுகாப்பை உறுதிப்படுத்தவும்**: கருவி மெட்டாடேட்டா சரிபார்ப்பு, மாற்றங்களுக்கான கண்காணிப்பு மற்றும் மறைபுரிந்த prompt injection தாக்குதல்களை எதிர்கொள்ள முக்கியத்துவத்தை உணரவும்
- **சிறந்த நடைமுறைகளை ஒருங்கிணைக்கவும்**: நிலையான பாதுகாப்பு அடிப்படைகள் (பாதுகாப்பான குறியீடு எழுத்து, சர்வர் கடுமைப்படுத்தல், பூஜ்ஜிய நம்பிக்கை) மற்றும் MCP தனிப்பட்ட கட்டுப்பாடுகளை சேர்ந்த முற்றிலும் பாதுகாப்பான ஒன்றிணைப்பை உருவாக்கவும்

# MCP பாதுகாப்பு கட்டமைப்பு & கட்டுப்பாடுகள்

புதுமையான MCP பயன்பாடுகள் பாரம்பரிய மென்பொருள் பாதுகாப்பும் AI-க்கு தனித்துவமான அச்சுறுத்தல்களையும் சமாளிக்கும் படிகள் அடிப்படையிலான பாதுகாப்பு அணுகுமுறையை தேவையாக்குகின்றன. வேகமாக வளர்ந்து வரும் MCP விவரக்குறிப்பு அதன் பாதுகாப்பு கட்டுப்பாடுகளை மேம்படுத்தி வருகிறது, நிறுவன பாதுகாப்பு கட்டமைப்புகள் மற்றும் நிலையான சிறந்த நடைமுறைகளுடன் சிறந்த ஒருங்கிணைப்பை உருவாக்குகிறது.

[Microsoft Digital Defense Report](https://aka.ms/mddr) ஆய்வுகள் காட்டுகின்றன **98% புகாரளிக்கப்பட்ட குறுக்குவிழிகள் வலுவான பாதுகாப்பு சீரமைப்பால் தடுப்பதற்கானவை**. மிகத் திறமையான பாதுகாப்பு நெறிகள் அடிப்படை பாதுகாப்பு நடைமுறைகளைக் கொண்டு MCP-க்கு தனிப்பட்ட கட்டுப்பாடுகளுடன் சேர்த்து செயல்படுத்த வேண்டும் — நிரூபிக்கப்பட்ட அடிப்படை பாதுகாப்பு நடவடிக்கைகள் மொத்த பாதுகாப்பு ஆபத்தை குறைக்க மிகச் சிறந்தவை.

## தற்போதைய பாதுகாப்பு நிலவரம்

> **குறிப்பு:** இந்த தகவல் MCP பாதுகாப்பு தரநிலைகளை **பிப்ரவரி 5, 2026** தேதி நிலவரப்படி பிரதிபலிக்கிறது, இது **MCP விவரக்குறிப்பு 2025-11-25** உடன் ஒத்துழைத்தது. MCP நெறிமுறை வேகமாக முன்னேறுகிறது, எதிர்கால பயன்பாடுகள் புதிய அங்கீகாரம் முறை மற்றும் மேம்பட்ட கட்டுப்பாடுகளை அறிமுகப்படுத்தலாம். எப்போதும் அண்மை [MCP விவரக்குறிப்பு](https://spec.modelcontextprotocol.io/), [MCP GitHub களஞ்சியம்](https://github.com/modelcontextprotocol), மற்றும் [பாதுகாப்பு சிறந்த நடைமுறைகளுக்கான ஆவணம்](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ஐப் பார்க்கவும்.

> **எதிர்பார்ப்பு:** `2026-07-28` வெளியீட்டு வழிகாட்டி அங்கீகாரத்தை மேலும் கடுமைப்படுத்துகிறது — கிளையண்ட்கள் அங்கீகார பதில்களில் உள்ள `iss` அளவுருவை சரிபார்க்க வேண்டும் (RFC 9207), Dynamic Client Registration போது OpenID Connect `application_type` அறிவிக்க வேண்டும், மற்றும் பதிவு செய்யப்பட்ட சான்றுகளை அங்கீகார வழங்கும் சர்வருடன் பிணைத்துக் கொள்ள வேண்டும். முழு அங்கீகார SEP க்கான விவரங்களுக்கு [MCP இல் என்ன மாற்றம்? 2026-07-28 வெளியீட்டு வழிகாட்டி](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) ஐப் பார்க்கவும்.

## 🏔️ MCP பாதுகாப்பு உச்சி பட்டறை (ஷெர்பா)

**வழங்கப்பட்ட பாதுகாப்பு பயிற்சிக்காக** நாங்கள் உயர்மொழிய MCP பாதுகாப்பு உச்சி பட்டறையை (ஷெர்பா) பரிந்துரைக்கிறோம் - Microsoft Azure இல் MCP சர்வர்கள் பாதுகாப்புக்கான விரிவான வழிகாட்டப்பட்ட பயணம்.

### பட்டறை குறுக்கெழுத்து

[MCP பாதுகாப்பு உச்சி பட்டறை](https://azure-samples.github.io/sherpa/) ஒரு நிலையான "தவறானது → தாக்குதல் → சரி → சரிபார்" செயல்முறை மூலம் நடைமுறை, செயல்படும் பாதுகாப்பு பயிற்சியை வழங்குகிறது. நீங்கள்:

- **தவறு ஏற்படுத்துவதன் மூலம் கற்றுக்கொள்ளவும்**: குறைந்தபட்ச பாதுகாப்புள்ள சர்வர்களை தாக்குதலுக்கு ஆளாக்கி விடுவீர்கள்
- **Azure நேடிவ் பாதுகாப்பை பயன்படுத்தவும்**: Azure Entra ID, Key Vault, API Management, மற்றும் AI Content Safety உபயோகி
- **டெஃபன்ஸ்-இன்-டெப்த் பின்தொடரவும்**: பாதுகாப்பு அடுக்கு கட்டியிடு முகாமுகள்
- **OWASP தரநிலைகளை பின்பற்றவும்**: ஒவ்வொரு தொழில்நுட்பமும் [OWASP MCP Azure பாதுகாப்பு கையேடு](https://microsoft.github.io/mcp-azure-security-guide/) உடன் இணைக்கப்பட்டுள்ளது
- **உற்பத்தி குறியீட்டை பெறவும்**: வேலை செய்கிறது, சோதிக்கப்பட்டவை பயன்பாட்டிற்கு கொண்டு செல்லப்படுகின்றன

| **MCP05** | கட்டளை உள்ளீடு & இயக்கம் | உள்ளீடு சரிபார்ப்பு, பாதுகாப்புSandboxing |
| **MCP06** | குறிக்கோள் ஓட்டம் முறிவு | Azure AI Content Safety, Prompt Shields |
| **MCP07** | போதாத அங்கீகாரம் மற்றும் அங்கீகார உரிமை | Azure Entra ID, OAuth 2.1 PKCE உடன் |
| **MCP08** | கணக்கீடு மற்றும் தொலைதூர மூலம் இல்லாமை | Azure Monitor, பயன்பாட்டு ஆன்களவை |
| **MCP09** | நிழல் MCP சர்வர்கள் | API சென்டர் நிர்வாகம், நெட்வொர்க் தனிமைப்படுத்தல் |
| **MCP10** | Context Injection & அதிக பகிர்வு | தரவு வகைப்படுத்தல், குறைந்த வெளிப்பாடு |

### MCP அங்கீகாரத்தின் வளர்ச்சி

MCP விவரக்குறிப்பு அங்கீகாரம் மற்றும் அங்கீகாரத்தில் பெரிதும் மேம்பட்டுள்ளது:

- **மூல அணுகுமுறை**: ஆரம்ப விவரக்குறிப்புகள் டெவலப்பர்கள் தனிப்பட்ட அங்கீகார சர்வர்களை உருவாக்க குழந்தைகள், MCP சர்வர்கள் OAuth 2.0 அங்கீகார சர்வர்களாக செயல்பட்டு பயனர் அங்கீகாரத்தை நேரடியாக நிர்வகிக்கிறது
- **தற்போதைய தரநிலை (2025-11-25)**: புதுப்பிக்கப்பட்ட விவரக்குறிப்பு MCP சர்வர்கள் வெளிப்புற ஐடெண்டிட்டி வழங்குநர்களுக்கு அங்கீகாரம் ஒப்படைக்க அனுமதிக்கிறது (Microsoft Entra ID போன்றவை) பாதுகாப்பு நிலையை மேம்படுத்தி நடப்பு பராமரிப்பை குறைக்கும்
- **போக்குவரத்து தர அடுக்கு பாதுகாப்பு**: உள்ளூர் (STDIO) மற்றும் தொலைநிலை (Streamable HTTP) இணைப்புகளுக்கு ஒப்பான அங்கீகாரம் முறை_PATTERN களை மேம்படுத்தியிருப்பது

## அங்கீகாரம் மற்றும் அங்கீகார பாதுகாப்பு

### தற்போதைய பாதுகாப்பு சவால்கள்

புதுமையான MCP பயன்பாடுகள் பல அங்கீகாரம் மற்றும் அங்கீகார சவால்களை எதிர்கொள்கின்றன:

### அபாயங்கள் மற்றும் தாக்கும் பாதைகள்

- **தவறான அங்கீகார லாஜிக் அமைப்பு**: MCP சர்வரின் எழுத்தில் ஏற்பட்ட பிழைகள் தரவுகளை வெளிப்படுத்தும் மற்றும் தவறான அணுகலை ஏற்படுத்தும்
- **OAuth டோக்கன் மோசடி**: உள்ளூர் MCP சர்வர் டோக்கன் திருட்டால் தாக்குதலாளர்கள் சர்வரை போல நடித்து பின்வரும் சேவைகளை அணுகலாம்
- **டோக்கன் பாத்திர வழித்தவறு**: தவறான டோக்கன் நிர்வாகம் பாதுகாப்பு கட்டுப்பாடுகளை தவிர்க்கும் மற்றும் பொறுப்பற்றை உருவாக்கும்
- **மிகுதியான அனுமதிகள்**: MCP சர்வர்கள் குறைந்த அதிகார கொள்கையை மீறி தாக்குதலை ஏற்படுத்துகிறது

#### டோக்கன் பாத்திர வழித்தவறு: ஒரு முக்கிய எதிர்முறைத் துறப்பு

**டோக்கன் பாத்திர வழித்தவறு தற்போது MCP அங்கீகார விதிமுறையில் கடுமையாக தடைசெய்யப்பட்டுள்ளது** அதன் தீவிர பாதுகாப்பு விளைவுகளுக்காக:

##### பாதுகாப்பு கட்டுப்பாடு தாண்டல்
- MCP சர்வர்கள் மற்றும் பின்வரும் API கள் முக்கிய பாதுகாப்பு கட்டுப்பாடுகளை (திரைவு வரம்பு, கோரிக்கை சரிபார்ப்பு, போக்குவரத்து கண்காணிப்பு) சரியான டோக்கன் சரிபார்ப்புக்கு சார்ந்துள்ளன
- நேரடி கிளையண்ட்-க்கு API டோக்கன் பயன்முறை இந்த முக்கிய பாதுகாப்புகளைத் தவிர்க்கின்றது, பாதுகாப்பு கட்டமைப்பை பாதிக்கிறது

##### பொறுப்புக்குறிப்பு மற்றும் கணக்காய்வு சிக்கல்கள்
- MCP சர்வர்கள் மேல் வழங்கப்பட்ட டோக்கன்களை பயன்படுத்தும் கிளையண்ட்களை வேறுபடுத்த முடியாது, கணக்காய்வு பாதிப்புகள் விளைவிக்கின்றன
- பின்வரும் வள சர்வர் பதிவுகள் உண்மையான MCP சர்வர் இடைமுகத்தை பதிலாக தவறான கோரிக்கை ஆதாரங்களை காட்டுகின்றன
- சம்பவ விசாரணைகள் மற்றும் பொருந்துச்செய்தல் கணக்காய்வுகள் மிகக் கடினமாக்கப்படுகின்றன

##### தரவிறக்க அபாயங்கள்
- சரிபார்க்கப்படாத டோக்கன் உரிமைகள் திருடப்பட்ட டோக்கன்களை கொண்டு MCP சர்வர்களை தரவு வெளியீட்டுக்கான வழி போல பயன்படுத்த அனுமதிக்கின்றன
- நம்பிக்கை எல்லையை மீறி திட்டமிட்ட பாதுகாப்பு கட்டுப்பாடுகளை தவிர்க்கும் அணுகல் உருவாகிறது

##### பன்முக சேவை தாக்கல் பாதைகள்
- பல சேவைகளால் ஏற்றுக்கொள்ளப்பட்ட மோசடியான டோக்கன்கள் இணைக்கப்பட்ட அமைப்புகளுக்கு பக்கசார்ந்த நகர்வை அனுமதிக்கின்றன
- டோக்கன் தோற்ற உதவியால் சேவைகள் இடையேயான நம்பிக்கை கொள்கைகள் மீறப்படும்

### பாதுகாப்பு கட்டுப்பாடுகள் மற்றும் குறைபாடுகள்

**கடுமையான பாதுகாப்பு தேவைகள்:**

> **கட்டாயம்**: MCP சர்வர்கள் **எந்தவொரு டோக்கன்களையும்**, குறிப்பாக MCP சர்வர் பக்கம் வழங்கப்படாதவை, ஏற்றுக்கொள்ள கூடாது

#### அங்கீகாரம் மற்றும் அங்கீகார கட்டுப்பாடுகள்

- **கடுமையான அங்கீகார ஆய்வு**: MCP சர்வர் அங்கீகார செயல்முறை முழுமையான ஆய்வுகளை செய்யவேண்டும், நீதிப்பூர்வமான பயனர்களும் கிளையண்ட்களும் மட்டுமே பாதுகாப்பான வளங்களை அணுகலாம் என்பதை உறுதிசெய்ய
  - **நடைமுறை வழிகாட்டி**: [Azure API Management இனை MCP சர்வர் அங்கீகாரம் வாயிலாக பயன்படுத்துதல்](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ஐடெண்டிட்டி ஒருங்கிணைப்பு**: [Microsoft Entra ID மூலம் MCP சர்வர் அங்கீகாரம்](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **பாதுகாப்பான டோக்கன் மேலாண்மை**: [Microsoft இன் டோக்கன் சரிபார்ப்பு மற்றும் ஆயுள் மேலாண்மை சிறந்த நடைமுறைகள்](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) செயல்படுத்தவும்
  - டோக்கன் பார்வையாளர் உரிமைகள் MCP சர்வர் ஐடெண்டிட்டியுடன் பொருந்துவதை சரிபார்க்கவும்
  - சரியான டோக்கன் சுழற்சி மற்றும் செயலிழப்பு கொள்கைகளை மேற்பார்வை செய்யவும்
  - டோக்கன் மீண்டும் பயன்படுத்துதல் மற்றும் அங்கீகாரம் இல்லாத பயன்பாடுகளை தடுக்கவும்

- **பாதுகாக்கப்பட்ட டோக்கன் சேமிப்பு**: ஓய்விலும் போக்குவரத்து (in-transit) இருக்கவும் குறியாக்கத்தில் டோக்கன் சேமிப்புடன் பாதுகாக்கவும்
  - **சிறந்த நடைமுறைகள்**: [பாதுகாப்பான டோக்கன் சேமிப்பு மற்றும் குறியாக்க வழிகாட்டி](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### அணுகல் கட்டுப்பாடு நடைமுறை

- **குறைந்த அதிகார கொள்கை**: MCP சர்வர்களுக்கு தேவைப்படும் குறைந்தபட்ச அனுமதிகளை மட்டும் வழங்கவும்
  - அதிகார பரப்புகளை தடுக்கும் விதமாக அனுமதி மீட்கைகள் மற்றும் அடையும்படி ஆய்வுகள் செய்யவும்
  - **Microsoft ஆவணம்**: [பாதுகாப்பான குறைந்த அதிகார அணுகல்](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **பங்கு அடிப்படையிலான அணுகல் கட்டுப்பாடு (RBAC)**: நுணுக்கமான பங்கு ஒதுக்கீடுகளை நடைமுறைப்படுத்தவும்
  - குறிப்பிட்ட வளங்கள் மற்றும் நடவடிக்கைகளுக்கு பங்குகளை படைப்பதை கட்டுப்படுத்தவும்
  - பெரிய அல்லது தேவையற்ற அனுமதிகளை தவிர்க்கவும்

- **தொடர்ச்சியான அனுமதி கண்காணிப்பு**: நிலையான அணுகல் கணக்காய்வு மற்றும் கண்காணிப்பை செயல்படுத்தவும்
  - சீரற்ற அனுமதி பயன்பாட்டை கண்காணித்து விரைந்து தீர்வளிக்கவும்
  - அதிகமாகவோ பயன்படுத்தப்படாத அனுமதிகளை உடனடியாக நீக்கவும்

## AI-க்கு தனித்துவமான பாதுகாப்பு அச்சுறுத்தல்கள்

### Prompt Injection மற்றும் கருவி அவதூறு தாக்கல்கள்

புதுமையான MCP பயன்பாடுகள் பாரம்பரிய பாதுகாப்பு நடவடிக்கைகளால் முழுமையாக கையாள முடியாது எனும் AI-க்கு தனித்துவமான அதிகமாக நுட்பமான தாக்குதல்களை எதிர்கொள்கின்றன:

#### **மறைபுரிந்த Prompt Injection (இலங்கை-புல Prompt Injection)**

**மறைபுரிந்த Prompt Injection** என்பது MCP இயங்கும் AI அமைப்புகளுக்கு மிக முக்கியமான பலவீனங்களுள் ஒன்றாகும். தாக்குதலாளிகள் தீங்கு விளைவிக்கும் அறிவுரைகளை வெளியில் உள்ள உள்ளடக்கங்களில் — ஆவணங்கள், வலைப்பக்கங்கள், மின்னஞ்சல்கள் அல்லது தரவு மூலங்கள் — தடுத்துவைத்து, AI அமைப்புகள் பின்னர் அதை சட்டபூர்வம் ஆன கட்டளைகளாக புரிந்து செயல்படுத்துகிறது.

**தாக்கல் நிலைகள்:**
- **ஆவண அடிப்படையிலான Injection**: செயல்படுத்தப்பட்ட ஆவணங்களில் மறைக்கப்பட்ட தீங்கு அறிவுரைகள் AI தவறான செயல்களை காலுவிக்கும்
- **இணைய உள்ளடக்க திருட்டு**: உட்புகுத்தப்பட்ட வலைப்பக்கங்களில் உள்ள prompt கள் AI நடத்தையை மாற்றுகின்றன
- **மின்னஞ்சல் அடிப்படையிலான தாக்கல்**: மின்னஞ்சலில் உள்ள தீங்கு அறிவுரைகள் AI உதவியாளர்களை தகவல் வெளியிட அல்லது அனுமதி இல்லாமல் செயல்படுத்த வைக்கின்றன
- **தரவு மூல அழுக்கு**: பாதிக்கப்பட்ட தரவுத்தளங்கள் அல்லது API கள் AI அமைப்புக்கு பாதிக்கப்பட்ட உள்ளடக்கங்களை வழங்குகின்றன

**உண்மையான உலக விளைவுகள்**: இத்தகைய தாக்கல்கள் தரவிறக்க, தனியுரிமை மீறல், தீங்கு விளைவிக்கும் உள்ளடக்கம் உருவாக்கல், மற்றும் பயனர் இடையூறுகள் போன்ற விளைவுகளை ஏற்படுத்தும். விரிவான பகுப்பாய்வுக்கு [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) பார்க்கவும்.

#### **கருவி விஷம் சேர்க்கை தாக்கல்கள்**

**கருவி விஷம் சேர்க்கை** MCP கருவிகளின் மெட்டாடேட்டாவை குறிக்கிறது, இது LLM கள் கருவி விளக்கங்களையும் அளவுருக்களையும் எப்படி புரிகின்றன என்பதில் தாக்குதலை ஏற்படுத்துகிறது.

**தாக்கல் முறைகள்:**
- **மெட்டாடேட்டா திருத்தம்**: தாக்குதலாளர்கள் கருவி விளக்கங்கள், அளவுரு வரையறைகள், அல்லது பயன்படுத்தும் எடுத்துக்காட்டுகளில் தீங்கு அறிவுரைகளை மறைக்கின்றனர்
- **காணாமலுள்ள அறிவுரைகள்**: கருவி மெட்டாடேட்டாவில் மறைக்கப்பட்ட prompt கள் AI மாதிரிகள் செயலாக்குகின்றன, ஆனால் மனித பயனர்கள் காணமுடியாது
- **தனிமாற்ற கருவி திருத்தங்கள் ("ரக் புல்ஸ்")**: பயனர்களால் ஒப்புக்கொள்ளப்பட்ட கருவிகள் பின்னர் தீங்கு செய்கின்ற செயல்களை பாவிக்க மாற்றப்படுகின்றன, பயனர் அறிவின்றி
- **அளவுரு Injection**: கருவி அளவுரு சுற்றுகளுக்குள் தீங்கு உள்ளடக்கங்கள் புகுத்தப்படுவதால் மாதிரி நடத்தையை பாதிக்கின்றன
**பராமரிக்கப்பட்ட சேவையகம் அபாயங்கள்**: தொலைநிலை MCP சேவைகள் உயர்ந்த அபாயங்களை உருவாக்குகின்றன ஏனெனில் கருவி வரையறைகள் ஆரம்ப பயனர் அனுமதித்த பிறகும் புதுப்பிக்கப்படக்கூடியவை, இதனால் முந்தைய பாதுகாப்பான கருவிகள் தீங்கானவையாக மாறும் வாய்ப்புகள் உருவாகும். முழுமையான பகுப்பாய்வுக்காக, [கருவி விஷப்பொருத்துதல் தாக்குதல்கள் (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) பார்க்கவும்.

- **மாதிரி குழப்பல் தாக்குதல்கள்**: மாதிரி வரம்புகளை பயன்படுத்தி கணிக்க முடியாத அல்லது பாதுகாப்பற்ற நடத்தைகளை உருவாக்குதல்  

### AI பாதுகாப்பு அபாய விளைவுகள்

**உயர் பாதிப்புகளைக் கொண்ட விளைவுகள்:**
- **தரவு திருட்டு**: அனுமதியற்ற அணுகல் மற்றும் தனிப்பட்ட மக்கள் அல்லது நிறுவன தரவின் திருட்டு  
- **தனியுரிமை மீறல்கள்**: தனித்துவமான தகவல்கள் (PII) மற்றும் biznes ரகசிய தகவல்களின் வெளிப்பாடு  
- **சிஸ்டம் மோசடி**: முக்கியமான அமைப்புகள் மற்றும் பண்முறைகளில் எதிர்பாராத மாற்றங்கள்  
- **அங்கீகாரத் திருட்டு**: ஆட்டு குறிகள் மற்றும் சேவை அங்கீகாரங்கள் மீறல்  
- **பக்கவிளைவுமான இயக்கங்கள்**: குறுக்கீடு செய்யப்பட்ட AI அமைப்புகளைப் பயன்படுத்தி விரிவான நெட்வொர்க் தாக்குதல்களுக்கு வாய்ப்பு

### அமர்வு கொள்ளை வசதிகள்

**அமர்வு கொள்ளை** என்பது அதிகாரவிரோதக் கட்சிகள் சரியான அமர்வுக் குறிப்புக்களைப் பெற்று அவற்றை தவறான முறையில் பயன்படுத்தி கிளையன்டுகளைத் தனிப்பயனாக்கி, அனுமதியில்லாத செயல்களைச் செய்யும் சூழ்நிலையாகும், குறிப்பாக நிலையான MCP அமர்வு செயலாக்கங்களில்.

#### **தாக்குதல் சூழல்கள் மற்றும் அபாயங்கள்**

- **அமர்வு கொள்ளை_prompt ஊழல்**: திருடப்பட்ட அமர்வு அடையாளங்களுடன் தாக்குதலாளர்கள் அமர்வு நிலையை பகிரும் சேவையகங்களில் தீங்கு விளைவிக்கும் நிகழ்வுகளை ஊட்டலாம், இது தீங்கு விளைவிக்கும் செயல்களை தொடக்கக்கூடும் அல்லது ரகசியத் தரவுகளுக்கு அணுகலாம்  
- **நேரடி தனிப்பயனாக்கல்**: திருடப்பட்ட அமர்வு அடையாளங்கள் MCP சேவையக அழைப்புகளை நேரடியாக செய்ய உதவி புரிந்து, அங்கீகாரத்தைத் தாண்டி தாக்குதலாளர்களை உண்மையான பயனர்களாகக் கண்டு கொள்ளச் செய்கிறது  
- **தவிர்க்கப்படக்கூடிய தொடர்ச்சியான ஓட்டைகள்**: தாக்குதலாளர்கள் கோரிக்கைகளை முன்கூட்டியே நிறுத்தி, சட்டவிரோத உள்ளடக்கத்துடன் மீண்டும் தொடர்வதற்கு வாய்ப்பு அளிக்கலாம்  

#### **அமர்வு மேலாண்மை பாதுகாப்பு கட்டுப்பாடுகள்**

**அத்தியாவசியத்தேர்வுகள்:**
- **அங்கீகார சரிபார்ப்பு**: MCP சேவைகள் அனைத்து உள்ளீட்டு கோரிக்கைகளையும் சரிபார்க்கவேண்டும் மற்றும் அங்கீகாரத்திற்கு அமர்வுகளுக்கு சாராமைக்கு இருக்கக்கூடாது  
- **பாதுகாப்பான அமர்வு உருவாக்கல்**: குறியாக்க பாதுகாப்பு மற்றும் தரவாகத் தானியங்கி உருவாக்கப்படும் ασφαதமிகு, பண்பற்ற முடியாத அமர்வு அடையாளங்களைப் பயன்படுத்துங்கள்  
- **பயனர்-சார்ந்த பிணைப்பு**: அமர்வு அடையாளங்களை `<user_id>:<session_id>` போன்ற வடிவமைப்பில் பயனரின் தனித்துவமான தகவலுடன் பிணைக்கவேண்டும், இதன் மூலம் பயனர் இடையே அமர்வு தாக்குதலைத் தடுக்கும்  
- **அமர்வு ஆயுள் மேலாண்மை**: காலாவதி, சுழற்சி மற்றும் தவிர்ப்பு முறைகளை செயல்படுத்தி பாதிப்பு நேரத்தை கட்டுப்படுத்தவும்  
- **கடத்தல் பாதுகாப்பு**: அனைத்து தொடர்புகளுக்கும் கடுமையான HTTPS பயன்படுத்துதல் அவசியம்  

### குழப்பப்பட்ட டெப்பி பிரச்சனை

**குழப்பப்பட்ட டெப்பி பிரச்சனை** என்பது MCP சேவைகள் கிளையண்டுகளுக்கும் மூன்றாம் தரப்பு சேவைகளுக்கும் இடையிலான அங்கீகார பிரதிநிதிகளாக செயல்படுத்தும்போது ஏற்படுகிறது, இதனால் நிலையான கிளையண்ட் ID களை துள்ளியடுத்தி அங்கீகாரம் தவிர்ப்பதற்கு வாய்ப்பு உருவாகிறது.

#### **தாக்குதல் இயந்திரவியல் மற்றும் அபாயங்கள்**

- **குக்கீ-அடிப்படையிலான ஒப்புதல் தவிர்ப்பு**: முந்தைய பயனர் அங்கீகாரம் ஒப்புதல் குக்கீகளை உருவாக்குகிறது, தாக்குதலாளர்கள் அவற்றை தீங்கு விளைவிக்கும் உரிமம் கோரிக்கைகளுடன் அனுப்பி ஓர் மாற்ற்ந்த மீண்டும் வழிமாற்ற URI களை பயன்படுத்தி பயன் பெறுவர்  
- **அங்கீகார குறியீடு திருட்டு**: நிலவும் ஒப்புதல் குக்கீகள் அங்கீகார சேவைகளை ஒப்புதல் திரைகளை தவிர்க்க வைக்கலாம், குறியீடுகளை தாக்குதல் கட்டுப்பாட்டுக் கட்டமைப்பிற்கு மாற்றி அனுப்புதல்  
- **அங்கீகாரமின்றி API அணுகல்**: திருடப்பட்ட அங்கீகார குறியீடுகள் அங்கீகாரமின்றி குறியீடு பரிமாற்றம் மற்றும் பயனர் impersonationக்கு உதவும்  

#### **தடுக்கும் முறைகள்**

**அவசியமான கட்டுப்பாடுகள்:**
- **தெளிவான ஒப்புதல் கோரிக்கைகள்**: MCP ப்ராக்சி சேவைகள் நிலையான கிளையண்ட் ID களை பயன்படுத்தும் போது ஒவ்வொரு ஜீவிலுள்ள பதிவுசெய்யப்பட்ட கிளையண்டுக்கும் பயனர் ஒப்புதலைப் பெற வேண்டும்  
- **OAuth 2.1 பாதுகாப்பு அமலாக்கம்**: அனைத்து அங்கீகார கோரிக்கைகளுக்கும் PKCE (Code பரிமாற்றத்திற்கு ஆதாரம்) உட்பட தற்போதைய OAuth பாதுகாப்பு நடைமுறைகளை பின்பற்றவும்  
- **கடுமையான கிளையண்ட் சரிபார்ப்பு**: மீண்டும் வழிமாற்ற URI மற்றும் கிளையண்ட் அடையாளங்களை கடுமையாக சரிபார்க்கவும்  

### டோக்கன் போட்டுக்கொடுக்கும் பாதுகாப்பு குறைகள்  

**டோக்கன் போட்டுக்கொடுக்கும்** என்பது MCP சேவைகள் கிளையண்ட் டோக்கன்களை சரிபார்ப்பின்றி ஏற்கும் மற்றும் அவற்றை கீழ்சுற்று API களுக்கு அனுப்பும் திறனற்ற நடைமுறையாகும், இது MCP அங்கீகார விதிகளுக்கு முரணானது.

#### **பாதுகாப்பு விளைவுகள்**

- **கட்டுப்பாடு விலக்கு**: நேரடியாக கிளையண்டிலிருந்து API க்கு டோக்கன் பயன்பாடு முக்கிய வேகம் கட்டுப்பாடு, சரிபார்ப்பு மற்றும் கண்காணிப்பு கட்டுப்பாடுகளுக்கு தடையாக இருக்கிறது  
- **ஆடிட் தடகளம்**: மேல்முனை டோக்கன்கள் காரணமாக கிளையண்ட் அடையாளம் சரியாக செல்லாது, நிகழ்வு விசாரணை சிக்கலாகிறது  
- **ப்ராக்சி அடிப்படையிலான தரவு திருட்டு**: சரிபார்ப்பற்ற டோக்கன்கள் சேவையகங்களை அனுமதியற்ற தரவு அனுப்பும் ப்ராக்ஸிகளாக பயன்படுத்த அனுமதிக்கின்றன  
- **நம்பிக்கை எல்லை மீறல்கள்**: கீழ்சுற்ற சேவைகள் டோக்கன் தோற்றத்தை சரிபார்க்க முடியாதபோது நம்பிக்கை கோட்பாடுகள் முரணடைகின்றன  
- **பல சேவை தாக்கல் விரிவாக்கம்**: தகர்மானிக்கப்பட்ட டோக்கன்கள் பல சேவைகளில் ஏற்கப்பட்டால் பரப்பாயுள்ள தாக்குதல்கள் ஏற்படும்  

#### **தேவையான பாதுகாப்பு கட்டுப்பாடுகள்**

**மறுக்க முடியாத கோரிக்கைகள்:**
- **டோக்கன் சரிபார்ப்பு**: MCP சேவைகள் MCP சேவைக்காக தெளிவாக வழங்கப்படவில்லை என்ற டோக்கன்களை ஏற்கக்கூடாது  
- **கோட்பாடு சரிபார்ப்பு**: டோக்கன் கோட்பாடு MCP சேவையின் அடையாளத்துடன் பொருந்துகிறதா என்பதை எப்போதும் சரிபார்க்கவேண்டும்  
- **சரியான டோக்கன் ஆயுள் காலம்**: குறுகிய ஆயுள் காலத்துடன் பாதுகாப்பான சுழற்சி நடைமுறைகளை செயல்படுத்தவும்  


## AI அமைப்புகளுக்கான சப்ளை செயின் பாதுகாப்பு

சப்ளை செயின் பாதுகாப்பு பாரம்பரிய மென்பொருள் சார்புகளுக்கு மேலாக வளர்ந்து முழு AI சூழலையும் உள்ளடக்கியுள்ளது. நவீன MCP செயலாக்கங்கள் அனைத்து AI தொடர்புடைய கூறுகளையும் கடுமையாக சரிபார்த்து கண்காணிக்க வேண்டும், ஏனெனில் ஒவ்வொன்றும் அமைப்பின் ஒருங்கிணைப்பைத் தோற்கடிக்கக்கூடிய அபாயங்களை உருவாக்கும்.

### விரிவடைந்த AI சப்ளை செயின் கூறுகள்

**பாரம்பரிய மென்பொருள் சார்புகள்:**
- திறந்த மூல நூலகங்கள் மற்றும் கட்டமைப்புகள்  
- கன்டெய்னர் படங்கள் மற்றும் அடிப்படைக் கணினிகள்  
- மேம்பாட்டு கருவிகள் மற்றும் கட்டுமான குழாயிகள்  
- செயலாக்க கூறுகள் மற்றும் சேவைகள்  

**AI-க்கு சிறப்பு வழங்கப்படும் சப்ளை செயின் கூறுகள்:**
- **அடித்தள மாதிரிகள்**: பலவகை வழங்குநர்களிடமிருந்து முன்பயிற்சி பெற்ற மாதிரிகள், வினியோக வரலாறு சரிபார்ப்பு தேவையுடன்  
- **எம்பெடிங் சேவைகள்**: வெளிப்புற வேக்டர் மாற்றம் மற்றும் சொந்த பொருள் தேடல் சேவைகள்  
- **சூழல் வழங்குநர்கள்**: தரவுத் தரவுகள், அறிவுத்தளங்கள் மற்றும் ஆவண சேமிப்பகங்கள்  
- **மூன்றாம் தரப்பு API கள்**: வெளிப்புற AI சேவைகள், எம்எல் குழாய்கள் மற்றும் தரவு செயலாக்க இறுதிக் கட்டங்கள்  
- **மாதிரி உருவாக்கங்கள்**: எடை, கட்டமைப்புகள் மற்றும் நுட்பமாக சரிசெய்யப்பட்ட மாதிரி வகைகள்  
- **பயிற்சி தரவை வழங்குநர்கள்**: மாதிரி பயிற்சி மற்றும் நுட்பமாக்கலில் பயன்படுத்தப்படும் தரவு தொகுதிகள்  

### முழுமையான சப்ளை செயின் பாதுகாப்பு மூலோபாயம்

#### **கூறு சரிபார்ப்பு & நம்பிக்கை**
- **மூலாரம்பு சரிபார்ப்பு**: அனைத்து AI கூறுகளின் மூலமும், உரிமமும், ஒருங்கிணைப்புக்கு முன் முழுமையுடனும் சரிபார்க்கப்பட வேண்டும்  
- **பாதுகாப்பு மதிப்பீடு**: மாதிரிகள், தரவுத் தரவுகள் மற்றும் AI சேவைகளுக்கென அபாய மதிப்பீடுகள் மற்றும் பாதுகாப்பு ஆய்வுகள் நடத்தல்  
- **புகைப்பட ஆய்வு**: AI சேவை வழங்குநர்களின் பாதுகாப்பு வரலாறு மற்றும் நடைமுறைகளை மதிப்பீடு செய்தல்  
- **கட்டுப்பாடெல்லாம் ஆய்வு**: அனைத்து கூறுகளும் நிறுவன பாதுகாப்பு மற்றும் ஒழுங்குமுறை தேவைகளை பூர்த்தி செய்கிறதா என்பதை உறுதி செய்தல்  

#### **பாதுகாப்பான பாய்வகைக் குழாயிகள்**  
- **தானியங்கி CI/CD பாதுகாப்பு**: தானியங்கி நிலைமையில் பாதுகாப்பு சோதனைகளை உள்ளடக்குதல்  
- **உருவாக்கத்தின் முழுமைத்தன்மை**: அனைத்து தளவமைப்பு படைப்புகளுக்கும் (கோடு, மாதிரிகள், கட்டமைப்புகள்) குறியீட்டு சரிபார்ப்பு  
- **பரிமாற்ற நிலையில் முன்னேற்றம்**: ஒவ்வொரு கட்டத்திலும் பாதுகாப்பு மதிப்பீடு கொண்ட முன்னேற்ற பாய்வகைகள்  
- **நம்பகமான உருவாக்கக் களஞ்சியங்கள்**: உறுதிப்படுத்தப்பட்ட மற்றும் பாதுகாப்பான களஞ்சியங்களைப் பயன்படுத்தி இயக்குதல்  

#### **தொடர்ச்சியான கண்காணிப்பு மற்றும் பதில்**
- **சார்பு தொகுதி சோதனை**: அனைத்து மென்பொருள் மற்றும் AI கூறுகளுக்கான அபாய கண்காணிப்பு

- **API வாயில் கட்டுப்பாடுகள்**: வெளிப்புற AI சேவைகளுக்கான அனுமதி கட்டுப்பாடுகள் மற்றும் கண்காணிப்பை ஒருங்கிணைக்கும் API வாயில்கள்  

#### **நிகழ்வு பதில் மற்றும் மீட்பு**
- **விரைவு பதில்கள் நடைமுறைகள்**: பாதிக்கப்பட்ட AI கூறுகளை பாதுகாப்பாக மாற்றும் அல்லது மீட்கும் செயல்முறைகள்

- **மீளச்செயல் திறன்கள்**: முன்பாக அறிந்த நல்ல பதிப்புகளுக்கு வேகமாக திருப்பி செல்லும் திறன்  
- **சப்ளை செயின் முறுக்கு மீட்பு**: மேன்மேலும் AI சேவை முறுக்கு பாதிப்புகளுக்கு எதிரான பிரத்யேக நடவடிக்கைகள்

## அடித்தள பாதுகாப்பு சிறந்த நடைமுறைகள்

MCP செயலாக்கங்கள் உங்கள் நிறுவனத்தின் தற்போதைய பாதுகாப்பு நிலையை அனுவர்த்தி மேம்படுத்தும். அடித்தள பாதுகாப்பு நடைமுறைகளை வலுப்படுத்துதல் AI அமைப்புகளின் மற்றும் MCP முதிர்வுகளின் மொத்த பாதுகாப்பை முக்கியமாக மேம்படுத்தும்.

- **முழுமையான குறியாக்கம்**: அனைத்து பயன்பாட்டு கூறுகளிலும் மற்றும் தரவு ஓட்டங்களில் பாதுகாப்பான தொடர்பு

- **நடத்தை பகுப்பாய்வு**: AI-ஐ அடிப்படையாகக் கொண்டு அமைப்பு மற்றும் பயனர் நடத்தை சிக்கல்களை அதிரடியாக கண்டறிதல்  
- **அச்சுறுத்தல் அறிவு**: வெளிப்புற அச்சுறுத்தல் தரவுகள் மற்றும் தோல்வி குறியீடுகளுடன் ஒருங்கிணைப்பு

- **தனியுரிமை கட்டுப்பாடுகள்**: AI அமைப்பு கட்டமைப்பில் தனியுரிமை-ஆல்-வடிவமைப்பு கொள்கைகள் அமல் செய்யப்படல்  
- **மாற்ற மேலாண்மை**: AI அமைப்பு மாற்றங்களுக்கான பாதுகாப்பு விமர்சனங்களுக்கான பற்பலமான செயல்முறைகள்  

இந்த அடித்தளம் நடைமுறைகள் MCP-ச் சிறப்பு பாதுகாப்பு கட்டுப்பாடுகளின் செயல்திறனை மேம்படுத்தும் ஒரு வலுவான பாதுகாப்பு அடித்தளத்தை உருவாக்குகின்றன மேலும் AI இயக்கப்பட்ட பயன்பாடுகளுக்கு முழுமையான பாதுகாப்பை வழங்குகின்றன.

## முக்கிய பாதுகாப்பு குறிப்புகள்

- **பொறுத்த படிமுறை பாதுகாப்பு அணுகுமுறை**: அடித்தளம் பாதுகாப்பு நடைமுறைகள் (பாதுகாப்பான குறியீடு உருவாக்கல், குறைந்த அனுமதி, வழங்கல் சங்கிலி சரிபார்ப்பு, தொடர்ச்சியான கண்காணிப்பு) ஐ AI-க்கு சொந்த கட்டுப்பாடுகளுடன் ஒன்றிணைத்து முழுமையான பாதுகாப்பை வழங்கல்  

- **AI-க்கு சிறப்பு அச்சுறுத்தல் நிலம்**: MCP அமைப்புகள் தனித்துவமான ஆபத்துக்களை எதிர்கொள்கின்றன; இதில் prompt injection, கருவி விசேடபடுத்து, அமர்வு கபடப்படுத்தல், குழப்பமான இடைமுக பிரச்சினைகள், டோக்கன் கடத்தல் பாதிப்புகள் மற்றும் அதிக அனுமதிகள் அடங்கும், இது சிறப்பு தடுப்பு நடவடிக்கைகள் தேவைப்படுகிறது  

- **அங்கீகாரம் மற்றும் அங்கீகார சிறப்புத்தன்மை**: வெளிப்புற அடையாள வழங்குநர்கள் (Microsoft Entra ID) மூலம் வலுவான அங்கீகாரத்தை அமல் செய்து சரியான டோக்கன் சரிபார்ப்பை மேற்கொண்டு, உங்கள் MCP சேவையகத்துக்காக தெளிவாக வழங்கப்பெறாத டோக்கன்களை எதிலும் ஏற்கக் கூடாது  

- **AI தாக்கத் தடுப்பு**: Microsoft Prompt Shields மற்றும் Azure Content Safety ஐ பயன்படுத்தி மறைமுக prompt injection மற்றும் கருவி விசேடபடுத்தல் தாக்குதல்களை எதிர் நிறுத்தி, கருவி மெட்டாடேட்டாவை சரிபார்த்து மற்றும் மாற்றங்கள் நிகழும்போது கண்காணித்தல்  

- **அமர்வும் பரிமாற்ற பாதுகாப்பும்**: பயனர் அடையாளங்களுடன் கட்டுப்படுத்தப்படும் குறியாக்க பாதுகாப்பான, முடிவற்ற அமர்வு ஐடியைக் கொண்டு செயல்படுத்து, சரியான அமர்வு வாழ்க்கைச் சுழற்சி மேலாண்மையை அமல் செய்து, அங்கீகாரத்திற்கு அமர்வுகளை பயன்படுத்தக் கூடாது  

- **OAuth பாதுகாப்பு சிறந்த நடைமுறைகள்**: குழப்பமான இடைமுக அட்டை தாக்குக்களை தடுக்க தெளிவான பயனர் ஒப்புதலைக் கொண்டு மாறுதலான பதிவு செய்யப்பட்ட கிளையன்ட்களுக்கு, OAuth 2.1 ஐ சீரான PKCE உடன் நடைமுறைப்படுத்தி, கடுமையான மறுவழி URI சரிபார்ப்பை நடத்துக  

- **டோக்கன் பாதுகாப்பு கொள்கைகள்**: டோக்கன் கடத்தல் எதிர்மறை வடிவங்களை தவிர்த்து, டோக்கன் கேட்கையாளர் தணிக்கை செய்து, குறுகிய கால வாழ்நாள் கொண்ட டோக்கன்களுடன் பாதுகாப்பான மாற்றத்தை மேற்கொள்ளவும் மற்றும் தெளிவான நம்பிக்கை எல்லைகளை பராமரித்து செயற்படு  

- **முழுமையான வழங்கல் சங்கிலி பாதுகாப்பு**: அனைத்து AI சூழல் கூறுகளையும் (மாதிரிகள், சேர்க்கைகள், கட்டமைப்பு வழங்கிகள், வெளிப்புற API கள்) பாரம்பரிய மென்பொருள் சார்ந்த பாதுகாப்பு கடமைகளுடன் சமமாக கையாளவும்  

- **தொடர்ச்சியான வளர்ச்சி**: விரைவில் மாறும் MCP விவரக்குறிப்புகளுக்கு ஏற்ப புதுப்பிக்கப்படுக, பாதுகாப்பு சமூக தரநிலைகளுக்கு பங்களிக்கவும் மற்றும் வழிகாட்டல் வளர்ச்சிக்கு ஏற்ப பாதுகாப்பு நிலைகளை பராமரிக்கவும்  

- **Microsoft பாதுகாப்பு ஒருங்கிணைப்பு**: Microsoft Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID போன்ற Microsoft முழுமையான பாதுகாப்பு சூழலை MCP இயக்க நீடிப்புக்கு வழங்க பெண்துணையாக பயன்படுத்துக
