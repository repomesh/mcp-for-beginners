# பாடம்: ஒரு வலைத் தேடல் MCP சேவையகம் கட்டுதல்

இந்த அத்தியாயம் வெளிப்புற APIகளை ஒருங்கிணைத்து, பலவகை தரவுகளை கையாள்ந்து, பிழைகளைக் கவனித்து, பல கருவிகளைக் ஒருங்கிணைக்கும் ஒரு நடைமுறை AI முகவரியை எப்படி உருவாக்குவது என்பதை காட்டுகிறது—all production-ready வடிவத்தில். நீங்கள் காண்பீர்கள்:

- **அங்கீகாரம் தேவைப்பட்டுக் கொள்ளும் வெளிப்புற APIகளை ஒருங்கிணைத்தல்**
- **பல முனைகளிலிருந்து பல்வேறு தரவுகளை கையாளுதல்**
- **தடுக்கமான பிழைக் கவனிப்பு மற்றும் பதிவு நெறிமுறைகள்**
- **ஒரே சேவையகத்தில் பல கருவிகள் ஒருங்கொடுக்கல்**

இறுதியில், நீங்கள் நவீன AI மற்றும் LLM-சக்தி வாய்ந்த பயன்பாடுகளுக்கான அவசியமான வடிவங்கள் மற்றும் சிறந்த பயிற்சிகளுடன் நடைமுறையான அனுபவத்தை பெற்றிருப்பீர்கள்.

## அறிமுகம்

இந்தப் பாடத்தில், நீங்கள் SerpAPI ஐப் பயன்படுத்தி நேரடி வலைத் தரவுடன் LLM திறன்களை விரிவாக்கும் மேம்பட்ட MCP சேவையகத்தையும் கிளையண்டையும் கட்டுவது எப்படி என்பதை கற்பீர்கள். இது இணையத்திலிருந்து புதுப்பிக்கப்பட்ட தகவல்களை அணுகக்கூடிய உறுப்பிகள் உருவாக்கும் முக்கிய திறனை ஆகும்.

## கற்றல் குறிக்கோள்கள்

இந்தப் பாடத்தின் இறுதியில், நீங்கள் செய்ய सक्षम ஆக இருப்பீர்கள்:

- MCP சேவையகத்தில் பாதுகாப்பான முறையில் வெளிப்புற APIகளை (SerpAPI போன்றவை) ஒருங்கிணைத்தல்
- வலை, செய்திகள், பொருள் தேடல் மற்றும் கேள்&பதில் பயன்பாட்டிற்கான பல கருவிகளை செயல்படுத்துதல்
- LLM பயனுக்கு அமைவாக கட்டமைக்கப்பட்ட தரவுகளை பகுப்பாய்வு செய்து வடிவமைத்தல்
- பிழைகளை கையாள்வதும் API வரம்புகளை திறமையாக நிர்வகிப்பதும்
- தானாக இயங்கும் மற்றும் இடைமுக கொண்ட MCP கிளையண்டுகளை கட்டி சோதனை செய்தல்

## வலைத் தேடல் MCP சேவையகம்

இந்த பகுதி Web Search MCP Server-ன் கட்டமைப்பு மற்றும் அம்சங்களை அறிமுகப்படுத்துகிறது. FastMCP மற்றும் SerpAPI ஒருங்கிணைந்து நேரடி வலைத் தரவுடன் LLM திறன்களை விரிவாக்குவதைக் காண்பீர்கள்.

### மேலோட்டம்

இந்த அமல்படுத்தலில், MCPவை பல்வேறு, பாதுகாப்பான மற்றும் திறமையான வெளிப்புற API அடிப்படையிலான பணிகளை கையாளும் நான்கு கருவிகள் காட்டப்படுகின்றன:

- **general_search**: பரபரப்பான வலை முடிவுகளுக்காக
- **news_search**: சமீபத்திய தலைப்புகளுக்காக
- **product_search**: மின்னணு வர்த்தக தரவுக்காக
- **qna**: கேள்வி-பதில் துணுக்குகளுக்காக

### அம்சங்கள்
- **குறியீடு உதாரணங்கள்**: பைத்தான் (மற்ற மொழிகளுக்கு எளிதில் விரிவாக்கக்கூடிய) மொழி-சார்ந்த குறியீடு தொகுதிகள், தெளிவுக்கு குறியீடு சுழற்சிகள் பயன்படுத்தி

### Python

```python
# general_search கருவியின் உதாரண பயன்பாடு
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

[`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) கோணு MCP சேவையகத்தை அமல்படுத்துகிறது, SerpAPI உடன் ஒருங்கிணைத்து வலை, செய்திகள், பொருள் தேடல் மற்றும் கேள்&பதில் கருவிகளுக்கு வாயிலாக திறவுகிறது. இது வரும் கோரிக்கைகளை கையாளும், API அழைப்புகளை நிர்வகிக்கும், பதில்களை பகுப்பாய்வு செய்கிறது மற்றும் கட்டமைக்கப்பட்ட முடிவுகளை கிளையண்டுக்கு அனுப்புகிறது.

முழு அமல்படுத்தலை நீங்கள் [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) இல் பார்வையிடலாம்.

சேவையகம் ஒரு கருவியை எப்படி வரையறுத்து பதிவு செய்கிறது என்பதற்கான சுருக்கமான எடுத்துக்காட்டு இதோ:

### Python Server

```python

from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...இடைமுகப்படுத்தல்...
```

- **வெளிப்புற API ஒருங்கிணைப்பு**: API சாவிகள் மற்றும் வெளிப்புற கோரிக்கைகள் பாதுகாப்பான முறையில் கையாளப்படுவதை காட்சிப்படுத்துகிறது
- **கட்டமைக்கப்பட்ட தரவு பகுப்பாய்வு**: API பதில்களை LLM-க்கு ஏற்ற வடிவங்களில் மாற்றுவது எப்படி என்பதை காட்டுகிறது
- **பிழைக் கவனிப்பு**: சரியான பதிவேட்டை உடைய தடுக்கமான பிழைக் கவனிப்பு
- **இடைகட்டுக் கிளையண்ட்**: தானாகச் செயல்படும் சோதனைகள் மற்றும் இடைகட்டுக் சோதனை முறையைச் சேர்க்கிறது
- **சூழல் மேலாண்மை**: MCP Context-ஐ பயன்படுத்தி கோரிக்கைகள் பதிவேற்றம் மற்றும் கண்காணிப்பில் உதவுகிறது

## முன்பயிற்சி தேவைகள்

தொடங்குவதற்கு முன்பு, உங்கள் சுற்றுப்புற சூழல் சரியாக அமைக்கப்பட்டுள்ளதா என கீழ்காணும் படி உறுதிப்படுத்திக் கொள்ளுங்கள். இதனால் அனைத்து சார்பு நூலகங்களும் நிறுவப்பட்டு, உங்கள் API சாவிகள் முறையாக காட்சியளிக்கப்படும்.

- Python 3.8 அல்லது அதற்கும் மேல் பதிப்பு
- SerpAPI API சாவி (சரிபார்க்க [SerpAPI](https://serpapi.com/) - இலவச நிலை உள்ளது)

## நிறுவல்

ஆரம்பிக்க கீழ்காணும் படிகளைப் பின்பற்றுங்கள்:

1. uv (பரிந்துரைக்கப்பட்டது) அல்லது pip கொண்டு சார்புகளை நிறுவவும்:

```bash
# uv பயன்படுத்துவது (பரிந்துரைக்கப்படுகிறது)
uv pip install -r requirements.txt

# pip பயன்படுத்துவது
pip install -r requirements.txt
```

2. திட்டத்தின் வேர் அடிக்குறியில் `.env` கோப்பை உருவாக்கி, உங்கள் SerpAPI சாவியை சேர்க்கவும்:

```
SERPAPI_KEY=your_serpapi_key_here
```

## பயன்பாடு

Web Search MCP சேவையகம் SerpAPI உடன் ஒருங்கிணைக்கப்பட்டு, வலை, செய்திகள், பொருள் தேடல் மற்றும் கேள்&பதில் கருவிகளை திறக்கின்ற முக்கிய கூறாகும். இது வரும் கோரிக்கைகளை கையாளும், API அழைப்புகளை நிர்வகிக்கும், பதில்களை பகுப்பாய்வு செய்கிறது மற்றும் கட்டமைக்கப்பட்ட முடிவுகளை கிளையண்டுக்கு அனுப்புகிறது.

முழு அமல்படுத்தலை நீங்கள் [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) இல் பார்வையிடலாம்.

### சேவையகத்தை இயக்குதல்

MCP சேவையகத்தை துவங்க கீழ்க்கண்ட கட்டளையை பயன்படுத்தவும்:

```bash
python server.py
```

சேவையகம் `stdio` அடிப்படையிலான MCP சேவையகமாக இயங்கும், கிளையண்ட் நேரடியாக அதனுடன் இணைக்க முடியும்.

### கிளையண்ட் நிலைகள்

கிளையண்ட் (`client.py`) MCP சேவையகத்துடன் தொடர்பு கொள்ள இரண்டு நிலைகள் ஆதரிக்கப்படுகிறது:

- **சாதாரண நிலை**: அனைத்து கருவிகளையும் பரிசோதித்து, பதில்களை சரிபார்க்க தானாக இயங்கும் சோதனைகள். இது சேவையகம் மற்றும் கருவிகள் எதிர்பார்ப்பின் படி இயங்குகிறதா என்பதை விரைவாகச் சரிபார்க்க உதவும்.
- **இடைகட்டுக் நிலை**: கைமுறையாக கருவிகளை தேர்வு செய்து அழைக்க, தனிப்பயன் கேள்விகளை உள்ளிடுகிறது மற்றும் முடிவுகளை நேரடி காண்பிக்கும் ஒரு மெனு இயக்கப்படுத்தி. இது சேவையக திறன்களை ஆராய மற்றும் பல்வேறு உள்ளீடுகளை பரிசோதிக்க சிறந்தது.

முழு அமல்படுத்தலை நீங்கள் [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) இல் பார்வையிடலாம்.

#### MCP Python SDK உடன் தனிப்பயன் சோதனை ஸ்கிரிப்ட்களை எழுதுதல்

இங்கு "சோதனை ஸ்கிரிப்ட்" என்பது MCP சேவையகத்தின் கிளையண்டாக செயற்படும் தனிப்பயன் Python திட்டத்தை குறிக்கும். இது அதிகாரப்பூர்வமான அலகு சோதனை அல்லாமல், உங்கள் தேர்ந்த பராமர்கள் கொண்டு சேவையகத்தின் எந்த கருவியையும் அழைக்கவும், முடிவுகளைக் கண்காணிக்கவும் உதவும். இது பயன்படும் அம்சங்கள்:

- கருவிச் அழைப்புகளை முன்மாதிரி மற்றும் பரிசோதனை செய்வது
- சேவையகம் வெவ்வேறு உள்ளீடுகளுக்கு எப்படி பதிலளிக்கிறது என்று சரிபார்த்தல்
- கருவிச் அழைப்புகளை தன்னான ஒதுக்கீட்டுக்காக தானாகச் செய்வது
- MCP சேவையகத்தின் மேலே சொந்த பணிமுறைகள் அல்லது ஒருங்கிணைப்புகளை கட்டுவது

செயல்பாடுகள் மாற்றங்களை விரைவில் முயற்சிக்க, கருவி நடத்தைக்கான பிழைகளை வெளிப்படுத்த, அல்லது மேம்பட்ட தானாக இயக்கத்திற்கான துவக்கப் புள்ளியாக இதைப் பயன்படுத்தலாம். கீழே MCP Python SDK பயன்படுத்தி அவ்வாறு ஒரு ஸ்கிரிப்ட் செய்யும் உதாரணம் உள்ளது:

## கருவி விளக்கங்கள்

## இந்தப் பாடத்தில் மேம்பட்ட கருத்துக்கள்

உருவாக்கத் தொடங்குவதற்கு முன்பு, இந்த அத்தியாயம் முழுவதும் தோன்றும் சில முக்கிய மேம்பட்ட கருத்துக்களை அறிந்து கொள்ளுங்கள். அவற்றை புரிந்துகொள்வதால் நீங்கள் பின்தொடர அனைத்து பாகங்களையும் தெளிவாக பின்பற்ற முடியும், புதியவராக இருந்தாலும்:

- **பல கருவிகளின் ஒருங்கிணைப்பு**: வலை தேடல், செய்திகள் தேடல், பொருள் தேடல், கேள்&பதில் போன்ற பல கருவிகளை ஒரே MCP சேவையகத்தில் இயக்குவதைக் குறிக்கும். இது உங்கள் சேவையகம் பல பணிகளை தாங்கும் வகையில் உருவாக உதவும்.
- **API வரம்பு நிர்வாகம்**: பல வெளிப்புற APIகள் (SerpAPI போன்றவை) குறிப்பிட்ட காலத்தில் அனுப்பக்கூடிய கோரிக்கைகள் எண்ணிக்கையை வரம்பிடும். நல்ல குறியீடு இந்த வரம்புகளை சரி பார்த்து, வரம்பு அடுத்தவுடன் மென்மையாக கையாளும், 앱 பாதிக்காமல் இருப்பதற்கான வளாகம்.
- **கட்டமைக்கப்பட்ட தரவு பகுப்பாய்வு**: API பதில்கள் பொதுவாக சிக்கலானவை மற்றும் அடர்த்தியானவை ஆகும். இந்த கருத்து, அந்த பதில்களை சுத்தமான, பயன்பாட்டுக்கு ஏற்ற வடிவுகளாக மாற்றுவது பற்றியது, குறிப்பாக LLMகளுக்கும் பிற நிரல் கருவிகளுக்கும் எளிதானபடி.
- **பிழை மீட்பு**: சில நேரங்களில் பிழைகள் நிகழலாம் — வாயில் பிரச்சினை, API எதிர்பார்த்ததை தராதது போன்றவை. பிழை மீட்பு எனும் கருத்து உங்கள் குறியீடு அந்தப் பிரச்சினைகளை கையாள கூடியதாகவும் பயனுள்ள பின்னூட்டம் தரக்கூடியதாகவும் இருக்கும் என்பது.
- **பராமரிப்பு சரிபார்ப்பு**: உங்கள் கருவிகளுக்கான அனைத்து உள்ளீடுகளும் சரியானவை மற்றும் பாதுகாப்பானவையாக உள்ளதை உறுதிப்படுத்துதல் ஆகும். இதில் இயல்புநிலை மதிப்புகளை அமைத்தல் மற்றும் தரவுத்தர வகைகளை சரிபார்ப்பதும் அடங்கும், இது பிழைகளையும் குழப்பங்களையும் தடுக்கும்.

இந்த பகுதி Web Search MCP சேவையகத்தோடு பணியாற்றும் போது நீங்கள் எதிர்கொள்ளக்கூடிய பொதுவான சிக்கல்களை கண்டறிந்து தீர்க்க உதவும். பிழைகள் அல்லது எதிர்பாராத செயல்பாடுகள் ஏற்பட்டால், இந்த பிரச்சினைத் தீர்க்கும் பகுதியை முதலில் பார்வையிடுங்கள் — பெரும்பாலான பிரச்சனைகள் இங்கே உள்ள ஆலோசனைகள் மூலம் விரைவில் தீரும்.

## பிரச்சினைகள் மற்றும் தீர்வுகள்

Web Search MCP சேவையகத்துடன் பணியாற்றும் போது சில நேரங்களில் சிக்கல்களை எதிர்கொள்ளலாம்—எது அனைத்து புதிய கருவிகளுடனும் வெளிப்புற APIகளுடனும் பணியாற்றும் போது சாதாரணமாக எதிர்பார்க்கத்தக்கது. இந்த பகுதி பொதுவான பிரச்சனைகளுக்கான நடைமுறை சந்தேகங்களில் தீர்வுகளை வழங்குவதாகும், இதனால் நீங்கள் விரைவாக வழிமுறையை திருப்பிக் கொள்ளலாம். நீங்கள் ஒரு பிழை சந்தித்தால், முதலில் இங்குள்ள பரிந்துரைகளைப் படியுங்கள்: இவை பெரும்பாலான பயனர்களின் சிக்கல்களை தீர்க்கும் மற்றும் கூடுதல் உதவி தேவைப்படாமல் தீர்வு தரும்.

### பொதுவான பிரச்சனைகள்

கீழே பயனர்கள் அடிக்கடி சந்திக்கும் சில பிரச்சனைகள், தெளிவான விளக்கங்களுடன் மற்றும் தீர்க்க நடவடிக்கைகள்:

1. **.env கோப்பில் SERPAPI_KEY இல்லை**
   - நீங்கள் `SERPAPI_KEY environment variable not found` என்ற பிழையைப் பார்க்கும்போது, உங்கள் பயன்பாடு SerpAPIயை அணுக API சாவியை கண்டுபிடிக்க முடியவில்லை. இதை சரிசெய்ய, உங்கள் திட்ட வேர் அடிக்குறியில் `.env` என்ற பெயரில் ஒரு கோப்பு உருவாக்கி, `SERPAPI_KEY=உங்கள்_serpapi_சாவி_இங்கே` என்ற வரியை சேர்க்கவும். உங்கள் உண்மையான SerpAPI சாவியுடன் `உங்கள்_serpapi_சாவி_இங்கே`யை மாற்றியிட வேண்டும்.

2. **Module not found பிழைகள்**
   - `ModuleNotFoundError: No module named 'httpx'` போன்ற பிழைகள் தேவையான Python தொகுப்புகள் இல்லை என்பதை குறிக்கும். இது எதற்கும் உங்கள் தேவையான அனைத்து சார்புகளையும் நிறுவாதபோது நேரும். இதை சரிசெய்ய, உங்கள் டெர்மினலில் `pip install -r requirements.txt` இயக்கு, அது உங்கள் திட்டத்திற்கு தேவையான எல்லாவற்றையும் நிறுவும்.

3. **இணைப்பு பிரச்சனைகள்**
   - `Error during client execution` போன்ற பிழை பெரும்பாலும் கிளையண்ட் சேவையகத்துடன் இணைக்க முடியவில்லை அல்லது சேவையகம் சரியாக இயங்கவில்லை என்பதைக் குறிக்கும். கிளையண்டும் சேவையகமும் பொருந்தக்கூடிய பதிப்புகளாக உள்ளனவா என்பதை மற்றும் `server.py` சரியான அடிக்குறியில் உள்ளது மற்றும் இயக்கப்படுகிறது என்பதை இருமுறை சரிபார்க்கவும். சேவையகத்தையும் கிளையண்டையும் மீண்டும் துவக்கவேண்டும்.

4. **SerpAPI பிழைகள்**
   - `Search API returned error status: 401` என்ற பிழை உங்கள் SerpAPI சாவி இல்லை, தவறானது அல்லது காலாவதியானது என்பதை குறிக்கும். உங்கள் SerpAPI டாஷ்போர்ட்டைப் பார்க்கவும், சாவியை சரிபார்த்து `.env` கோபையில் புதுப்பிக்கவும். உங்கள் சாவி சரியானபோதும் பிழை தொடர்ந்தால், இலவச வரம்பு முடிந்துவிட்டதா என பார்த்துக் கொள்ளவும்.

### பிழை கண்டறிதல் நிலை

இயல்பாக, பயன்பாடு முக்கிய தகவல்களை மட்டுமே பதிவேற்றுகிறது. உங்கள் செயல்பாட்டில் உள்ள விசேஷங்களை (முன்னேற்றுக்கான சிக்கல்களை) அறிய DEBUG நிலையை இயக்கலாம். இது பயன்பாடு எவ்வாறு செயல்படுகிறது என்பதை விரிவாகக் காட்டும்.

**எடுத்துக்காட்டு: சாதாரண வெளியீடு**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**எடுத்துக்காட்டு: DEBUG வெளியீடு**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

DEBUG நிலை HTTP கோரிக்கைகள், பதில்கள் மற்றும் உட்கட்டளை விவரங்கள் போன்ற கூடுதல் வரிசைகளை காட்டுகிறது. இது சிக்கல்களை தீர்க்க மிகவும் பயனுள்ளது.

DEBUG நிலையை இயக்க, உங்கள் `client.py` அல்லது `server.py` மேல் பகுதியிலே பதிவு நிலையை DEBUG ஆக அமைக்கவும்:

# [Python](#tab/python-debug)
```python
# உங்கள் client.py அல்லது server.py கோப்பின் மேல் பகுதியில்
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO-யில் இருந்து DEBUG-க்கு மாற்றவும்
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
