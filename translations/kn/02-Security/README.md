# MCP ಭದ್ರತೆ: AI ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಸಂಪೂರ್ಣ ರಕ್ಷಣೆ

[![MCP Security Best Practices](../../../translated_images/kn/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(ಈ ಪಾಠದ ವೀಡಿಯೊ ನೋಡಲು ಮೇಲಿನ ಚಿತ್ರವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಭದ್ರತೆ AI ವ್ಯವಸ್ಥೆ ವಿನ್ಯಾಸದ ಪ್ರಮುಖ ಅಂಶವಾಗಿದೆ, ಆದ್ದರಿಂದ ನಾವು ಅದನ್ನು ನಮ್ಮ ಎರಡನೆಯ ಭಾಗವಾಗಿ ಪ್ರಮುಖತೆ ನೀಡುತ್ತೇವೆ. ಇದು Microsoft ನ [ಸುರಕ್ಷಿತ ಭವಿಷ್ಯ ابتهاಜನೆ](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) ನ **Secure by Design** ತತ್ವದೊಡನೆ ಹೊಂದಿಕೆಯಾಗುತ್ತದೆ.

ಮಾದರಿ ಕಾಂಟೆಕ್ಸ್ಟ್ ಪ್ರೊಟೋಕಾಲ್ (MCP) AI ಚಾಲಿತ ಅನ್ವಯಿಕೆಗಳಿಗೆ ಶಕ್ತಿಶಾಲಿ ಹೊಸ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ತರುತ್ತದೆ ಮತ್ತು ಅದೇ ಸಮಯದಲ್ಲಿ ಪರಂಪರাগত ಸಾಫ್ಟ್‌ವೇರ್ જોખಮಗಳನ್ನು ಮೀರಿದ ಅದ್ಭುತ ಭದ್ರತಾ ಸವಾಲುಗಳನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ. MCP ವ್ಯವಸ್ಥೆಗಳು ಸ್ಥಾಪಿತ ಭದ್ರತಾ ಆತಂಕಗಳು (ಭದ್ರತೆ ಸಂಹಿತೆ, ಕನಿಷ್ಠ привಿಲೇಜ್, ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತೆ) ಮತ್ತು AI ಸ್ಪಷ್ಟವಾದ ಹೊಸ ಭರ್ತಿಗಳಾದ prompt injection, tool poisoning, session hijacking, confused deputy ದಾಳಿಗಳು, token passthrough ಭದ್ರತಾ ಅಪಾಯಗಳು ಮತ್ತು dynamic ಸಾಮರ್ಥ್ಯ ಬದಲಾವಣೆಗಳನ್ನು ಎದುರಿಸುತ್ತವೆ.

ಈ ಪಾಠವು MCP ಅನುಷ್ಠಾನಗಳಲ್ಲಿನ ಅತ್ಯಂತ ಪ್ರಮುಖ ಭದ್ರತಾ ಅಪಾಯಗಳನ್ನು ವಿಶೇಷವಾಗಿ ಪರಿಶೀಲಿಸುತ್ತದೆ — ಪ್ರಾಮಾಣಿಕತೆ, ඒಕಾಂಗಿಕೃತತೆ, ಅತಿರಕ್ತ ಅನುಮತಿಗಳು, ಪರೋಕ್ಷ prompt injection, ಸೆಷನ್ ಭದ್ರತೆ, confused deputy ಸಮಸ್ಯೆಗಳು, ಟೋಕನ್ ನಿರ್ವಹಣೆ ಮತ್ತು ಸರಬರಾಜು ಸರಪಳಿ ಅಪಾಯಗಳನ್ನು. ನೀವು Microsoft ಪರಿಹಾರಗಳು ಹಾಗು Prompt Shields, Azure Content Safety, GitHub Advanced Security ಬಳಸಿ MCP ನಿಯೋಜನೆಯನ್ನು ಬಲಪಡಿಸಲು ನಡವಳಿಕೆ ನಿಯಂತ್ರಣಗಳು ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಕಲಿಯುತ್ತೀರಿ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಸಾಧ್ಯವಾಗುತ್ತದೆ:

- **MCP-ನಿರ್ದಿಷ್ಟ ಭರ್ತಿಗಳನ್ನು ಗುರುತಿಸಿ**: prompt injection, tool poisoning, ಅತಿರಕ್ತ ಅನುಮತಿಗಳು, ಸೆಷನ್ ಹೈಜಾಕಿಂಗ್, confused deputy ಸಮಸ್ಯೆಗಳು, token passthrough ಅಪಾಯಗಳು ಮತ್ತು ಸರಬರಾಜು ಸರಪಳಿ ಅಪಾಯಗಳನ್ನು ಸೇರಿ MCP ವ್ಯವಸ್ಥೆಗಳ ವೈಶಿಷ್ಟ್ಯಭರಿತ ಭದ್ರತಾ ಅಪಾಯಗಳನ್ನು ಗುರುತಿಸಿ
- **ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ಅನ್ವಯಿಸಿ**: ಭದ್ರತೆಯ ಪರಿಣಾಮಕಾರಿ ತಡೆಯನ್ನು ಅನುಷ್ಠಾನ ಮಾಡಿ, ಶಕ್ತಿಶಾಲಿ ಪ್ರಾಮಾಣಿಕತೆ, ಕನಿಷ್ಠ ಪ್ರಿವಿಲೇಜ್ ಪ್ರವೇಶ, ಭದ್ರ ಟೋಕನ್ ನಿರ್ವಹಣೆ, ಸೆಷನ್ ಭದ್ರತೆ ನಿಯಂತ್ರಣಗಳು ಮತ್ತು ಸರಬರಾಜು ಸರಪಳಿಯ ಪರಿಶೀಲನೆ
- **Microsoft ಭದ್ರತಾ ಪರಿಹಾರಗಳನ್ನು ಉಪಯೋಗಿಸಿ**: MCP ಕಾರ್ಯಭಾರದ ರಕ್ಷಣೆಗೆ Microsoft Prompt Shields, Azure Content Safety ಮತ್ತು GitHub Advanced Security ಅನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ ಮತ್ತು ನಿಯೋಜಿಸಿ
- **ಟೂಲ್ ಭದ್ರತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ**: ಉಪಕರಣ ಮೆಟಾಡೇಟಾ ಪರಿಶೀಲನೆಯ ಮಹತ್ವವನ್ನು ಗುರುತಿಸಿ, dynamic ಬದಲಾವಣೆಗಳಿಗೆ ನಿಗಾ ವಹಿಸಿ ಮತ್ತು ಪರೋಕ್ಷ prompt injection ದಾಳಿಗಳ ವಿರುದ್ಧ ರಕ್ಷಣೆ ಮಾಡಿರಿ
- **ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಏಕೀಕೃತಗೊಳಿಸಿ**: ಸ್ಥಾಪಿತ ಭದ್ರತಾ ಮೂಲಭೂತಾಂಶಗಳು (ಭದ್ರತೆ ಸಂಹಿತೆ, ಸರ್ವರ್ ಬಲಪಡಿಸುವಿಕೆ, ಶೂನ್ಯ ನಂಬಿಕೆ) MCP-ನಿರ್ದಿಷ್ಟ ನಿಯಂತ್ರಣಗಳೊಂದಿಗೆ ಒದಗಿಸಿ ಸಮಗ್ರ ರಕ್ಷಣೆಗೆ

# MCP ಭದ್ರತಾ ವಿನ್ಯಾಸ ಮತ್ತು ನಿಯಂತ್ರಣಗಳು

ಆಧುನಿಕ MCP ಅನುಷ್ಠಾನಗಳು ಪರಂಪರাগত ಸಾಫ್ಟ್‌ವೇರ್ ಭದ್ರತೆ ಮತ್ತು AI-ನಿರ್ದಿಷ್ಟ ಅಪಾಯಗಳನ್ನು ಉಲ್ಲೇಖಿಸುವ ಬಹುಪದರ ಭದ್ರತಾ ವಿಧಾನಗಳನ್ನು ಅಗತ್ಯವಾಗಿವೆ. ವೇಗವಾಗಿ ಬೆಳಕುವ MCP ವಿಶೇಷಣೆ ಅದರ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ವೃದ್ಧಿಸುತ್ತಿದೆ, ಎಂಟರ್‌ಪ್ರೈಸ್ ಭದ್ರತಾ ವಿನ್ಯಾಸಗಳೊಂದಿಗೆ ಮತ್ತು ಸ್ಥಾಪಿತ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳೊಂದಿಗೆ ಉತ್ತಮ ಏಕರೂಪತೆಯನ್ನು ಸಾಧ್ಯವಾಗಿಸುತ್ತಿದೆ.

[Microsoft ಡಿಜಿಟಲ್ ಡಿಫೆನ್ಸ್ ವರದಿ](https://aka.ms/mddr) ನ ಸಂಶೋಧನೆ ಪ್ರಕಾರ ** ವರದಿಯಾದ 98% ಉಲ್ಲಂಘನೆಗಳು ಶಕ್ತಿಶಾಲಿ ಭದ್ರತಾ ಹೈಜೀನ್ ಮೂಲಕ ತಡೆಯಲ್ಪಡಬಹುದು**. ಪರಿಣಾಮಕಾರಿ ರಕ್ಷಣಾ ತಂತ್ರವು ಮೂಲಭೂತ ಭದ್ರತಾ ಅಭ್ಯಾಸಗಳೊಂದಿಗೆ MCP-ನಿರ್ದಿಷ್ಟ ನಿಯಂತ್ರಣಗಳನ್ನು ಸಂಯೋಜಿಸುವುದು ಆಗಿದ್ದು, ಸ್ಥಾಪಿತ ಮೂಲಭೂತ ಭದ್ರತಾ ಕ್ರಮಗಳು ಒಟ್ಟು ಭದ್ರತಾ ಅಪಾಯವನ್ನು ಕಡಿಮೆ ಮಾಡುವಲ್ಲಿ ಅತ್ಯಂತ ಪರಿಣಾಮಕಾರಿ.

## ಪ್ರಸ್ತುತ ಭದ್ರತಾ ದೃಶ್ಯ

> **ಗೌರವ ಯೋಜನೆ:** ಈ ಮಾಹಿತಿ ಈ MCP ಭದ್ರತಾ ಮಾನದಂಡಗಳನ್ನು **ಫೆಬ್ರವರಿ 5, 2026** ರಂತೆ ಪ್ರತಿಫಲಿಸುತ್ತದೆ, **MCP ವಿಶೇಷಣೆ 2025-11-25** ಗೆ ಹೊಂದಾಣಿಕೆ. MCP ಪ್ರೋಟೋಕಾಲ್ ವೇಗವಾಗಿ ಬೆಳೆಯುತ್ತಿದ್ದು, ಭವಿಷ್ಯದ ಅನುಷ್ಠಾನಗಳು ಹೊಸ ಪ್ರಾಮಾಣಿಕತೆ ಮಾದರಿಗಳನ್ನು ಮತ್ತು ಅಭಿವೃದ್ಧಿಪಡಿಸಿದ ನಿಯಂತ್ರಣಗಳನ್ನು ಪರಿಚಯಿಸಬಹುದು. ಸದಾ ಇತ್ತೀಚಿನ ಮಾರ್ಗದರ್ಶನಕ್ಕಾಗಿ [MCP ವಿಶೇಷಣೆ](https://spec.modelcontextprotocol.io/), [MCP GitHub ಶೇಖರಣೆ](https://github.com/modelcontextprotocol), ಮತ್ತು [ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ದಾಖಲೆ](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ನೋಡಿರಿ.

> **ಮುಂದನ್ನು ನೋಡಿ:** `2026-07-28` ಬಿಡುಗಡೆ ಅಭ್ಯರ್ಥಿ ಪ್ರಾಧಿಕರಣವನ್ನು ಇನ್ನೂ ಬಲಪಡಿಸುತ್ತದೆ — ಕ್ಲೈಂಟ್‌ಗಳು ಪ್ರಾಧಿಕರಣ ಪ್ರತಿಕ್ರಿಯೆಗಳಲ್ಲಿ `iss` ಪರಾಮೀಟರನ್ನು ಸತ್ಯಪಾಯಿಸಬೇಕು (RFC 9207), ಡೈನಾಮಿಕ್ ಕ್ಲೈಂಟ್ ನೋಂದಣಿಯ ಸಮಯದಲ್ಲಿ OpenID Connect `application_type` ಘೋಷಿಸಬೇಕು, ನೋಂದಾಯಿಸಿದ ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ಅವಳ ಪ್ರಾಧಿಕರಣ ಸರ್ವರ್‌ಗೆ ಅಂಟಿಸಬೇಕು. ಸಂಪূর্ণ ಪ್ರಾಧಿಕರಣ SEPs ಪಟ್ಟಿ ಇಲ್ಲಿದೆ [MCP ಯಲ್ಲಿ ಏನು ಬದಲಾಗುತ್ತಿದೆ: 2026-07-28 ಬಿಡುಗಡೆ ಅಭ್ಯರ್ಥಿ](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md).

## 🏔️ MCP ಭದ್ರತಾ ಶಿಖರ ಕಾರ್ಯಾಗಾರ (ಶೆರ್ಪಾ)

**ಪ್ರಾಯೋಗಿಕ ಭದ್ರತಾ ತರಬೇತಿಯಿಗಾಗಿ**, ನಾವು **MCP ಭದ್ರತಾ ಶಿಖರ ಕಾರ್ಯಾಗಾರ** (ಶೆರ್ಪಾ) ಅನ್ನು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ - ಇದು Microsoft Azure ನಲ್ಲಿ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಭದ್ರಗೊಳಿಸುವ ಸಮಗ್ರ ಮಾರ್ಗದರ್ಶನವುಳ್ಳ ಯಾತ್ರೆ.

### ಕಾರ್ಯಾಗಾರದ ಅವಲೋಕನ

[MCP ಭದ್ರತಾ ಶಿಖರ ಕಾರ್ಯಾಗಾರ](https://azure-samples.github.io/sherpa/) ಅನುಭವಾತ್ಮಕ, ನಡವಳಿಕಾಶೀಲ ಭದ್ರತಾ ತರಬೇತಿಯನ್ನು "ಸೂಲಭ್ಯ → ದುರ್ಬಲತೆ → ಪರಿಹಾರ → ಪರಿಶೀಲನೆ" ವಿಧಾನಸರಿಯಾಗಿ ಒದಗಿಸುತ್ತದೆ. ನೀವು:

- **ಭಂಗದ ಮೂಲಕ ಕಲಿಯಿರಿ**: ಸಂಯೋಜಿತ ಸುರಕ್ಷಿತ ಸರ್ವರ್‌ಗಳನ್ನು ದುರ್ಬಲತೆಗಳನ್ನು ಅನುಭವಿಸಿ
- **Azure ಮೂಲಭೂತ ಭದ್ರತೆ ಉಪಯೋಗಿಸಿ**: Azure Entra ID, Key Vault, API ನಿರ್ವಹಣೆ ಮತ್ತು AI ವಿಷಯ ಭದ್ರತೆ ಬಳಸಿ
- **ಆಳದಲ್ಲಿನ ರಕ್ಷಣೆಯನ್ನು ಅನುಸರಿಸಿ**: ಬಹುಪದರ ಭದ್ರತಾ ಪದರಗಳನ್ನು ರಚಿಸುವ ಚರಣಗಳಲ್ಲಿ ಪ್ರಗತಿಪಡಿಸಿ
- **OWASP ಮಾನದಂಡಗಳನ್ನು ಅನುಸರಿಸಿ**: ಪ್ರತಿ ತಂತ್ರಜ್ಞಾನವು [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) ಗೆ ಹೊಂದಿಕೆಯಾಗುತ್ತದೆ
- **ಉತ್ಪಾದನಾ ಕೋಡ್ ಪಡೆಯಿರಿ**: ಕೆಲಸ ಮಾಡುವ ಪರಿಗಣಿಸಲಾದ ಅನುಷ್ಠಾನಗಳನ್ನು ಪಡೆಯಿರಿ

### MCP ಪ್ರಾಮಾಣಿಕತೆ ಬೆಳವಣಿಗೆ

MCP ವಿಶೇಷಣೆ ಪ್ರಾಮಾಣಿಕತೆ ಮತ್ತು ಪ್ರಾಧಿಕರಣದ ಸಂಬಂಧದಲ್ಲಿ ಬಹಳಷ್ಟು ಬೆಳವಣಿಗೆ ಕಂಡಿದೆ:

- **ಮೂಲ ವಿಧಾನ**: ಆರಂಭಿಕ ವಿಶೇಷಣೆಗಳು ಅಭಿವೃದ್ಧಿಪಡಕರಿಗೆ ಸ್ವಯಂ ಪ್ರಾಮಾಣಿಕತಾ ಸರ್ವರ್‌ಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳ್ಳಬೇಕಾದದ್ದು, MCP ಸರ್ವರ್‌ಗಳು ನೇರವಾಗಿ OAuth 2.0 ಪ್ರಾಮಾಣಿಕತಾ ಸರ್ವರ್‌ಗಳಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿದ್ದವು
- **ಪ್ರಸ್ತುತ ಮಾನದಂಡ (2025-11-25)**: ನವೀಕೃತ ವಿಶೇಷಣೆ MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಬಾಹ್ಯ ಐಡೆಂಟಿಟಿ ಪ್ರದಾಯಕರ (ಉದಾ: Microsoft Entra ID) ಬಳಸಿ ಪ್ರಾಮಾಣಿಕತೆ ಮಂಜೂರು ಮಾಡಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ, ಭದ್ರತೆ ಭದ್ರಪಡಿಸಿ ಅನುಷ್ಠಾನ ಸරಳಗೊಳಿಸುತ್ತದೆ
- **ರವಾನಾ ಪದರ ಭದ್ರತೆ**: ಸ್ಥಳೀಯ (STDIO) ಮತ್ತು ದೂರಸ್ಥ (Streamable HTTP) ಸಂಪರ್ಕಗಳಿಗೆ ಸರಿಯಾದ ಪ್ರಾಮಾಣಿಕತೆ ಮಾದರಿಗಳನ್ನು ಹೊಂದಿರುವ ಸುರಕ್ಷಿತ ರವಾನಾ ವ್ಯವಸ್ಥೆಯ ಬೆಂಬಲವನ್ನು ಸುಧಾರಿಸಲಾಗಿದೆ

## ಪ್ರಾಮಾಣಿಕತೆ & ಪ್ರಾಧಿಕರಣೆ ಭದ್ರತೆ

### ಪ್ರಸ್ತುತ ಭದ್ರತಾ ಸವಾಲುಗಳು

ಆಧುನಿಕ MCP ಅನುಷ್ಠಾನಗಳು ಹಲವು ಪ್ರಾಮಾಣಿಕತೆ ಮತ್ತು ಪ್ರಾಧಿಕರಣೆ ಸವಾಲುಗಳನ್ನು ಎದುರಿಸುತ್ತವೆ:

### ಅಪಾಯಗಳು ಮತ್ತು ದಾಳಿ ಮಾರ್ಗಗಳು

- **ತಪ್ಪು ಸಂರಚಿಸಲಾದ ಪ್ರಾಧಿಕರಣೆ ತರ್ಕ**: MCP ಸರ್ವರ್‌ಗಳಲ್ಲಿ ದೋಷಪೂರ್ಣ ಪ್ರಾಧಿಕರಣೆ ವಸ್ತುನಿಷ್ಠ ಮಾಹಿತಿಯನ್ನು ಬಹಿರಂಗಪಡಿಸಿ ತಪ್ಪಾಗಿ ಪ್ರವೇಶ ನಿಯಂತ್ರಣಗಳನ್ನು ಅನ್ವಯಿಸಬಹುದು
- **OAuth ಟೋಕನ್ ಕಮ್ಪ್ರೋಮೈಸ್**: ಸ್ಥಳೀಯ MCP ಸರ್ವರ್ ಟೋಕನ್ ಕಳವು ಆಕ್ರಮಣಕಾರರಿಗೆ ಸರ್ವರ್‌ಗಳಂತೆ ಮರುಪರಿಚಯ ಮಾಡಿಕೊಳ್ಳಲು ಮತ್ತು ನಂತರದ ಸೇವೆಗಳಿಗೆ ಪ್ರವೇಶ ಮಾಡಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ
- **ಟೋಕನ್ನು ಪಾಸ್ತುರುತಮ್ ಅಪಾಯಗಳು**: ತಪ್ಪಾಗಿ ಟೋಕನ್ ನಿರ್ವಹಣೆ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ತಲಕಡೆಯಾಗಿ ಮತ್ತು ಜವಾಬ್ದಾರಿತ್ವದ ಗ್ಯಾಪ್‌ಗಳನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ
- **ಅತಿರಿಕ್ತ ಅನುಮತಿಗಳು**: ಅಧಿಕಾರಿತ MCP ಸರ್ವರ್‌ಗಳು ಕನಿಷ್ಠ ಪ್ರಿವಿಲೇಜ್ ತತ್ವಗಳನ್ನು ಉಲ್ಲಂಘಿಸಿ ದಾಳಿದಾರರ ವಿಸ್ತೃತ ದೃಷ್ಠಿಕೋಣವನ್ನು ಸೃಷ್ಟಿಸುತ್ತವೆ

#### ಟೋಕನ್ ಪಾಸ್ತುರುತಮ್: ಒಂದು ಪ್ರಮುಖ ವಿರೋಧ - ಮಾದರಿ

**ಟೋಕನ್ ಪಾಸ್ತುರುತಮ್ ಸ್ಪಷ್ಟವಾಗಿ ನಿರ್ಬಂಧಿಸಲಾಗಿದೆ** ಪ್ರಸ್ತುತ MCP ಪ್ರಾಧಿಕರಣ ವಿಶೇಷಣದಲ್ಲಿ ಗಂಭೀರ ಭದ್ರತಾ ಪರಿಣಾಮಗಳ ಕಾರಣ:

##### ಭದ್ರತಾ ನಿಯಂತ್ರಣ ತಿರಸ್ಕಾರದ
- MCP ಸರ್ವರ್‌ಗಳು ಮತ್ತು ನಂತರದ API ಗಳು ಮುಖ್ಯ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು (ರೇಟು ನಿಯಂತ್ರಣ, ವಿನಂತಿ ಪರಿಶೀಲನೆ, ಟ್ರಾಫಿಕ್ ನಿಗಾ) ಅನುಷ್ಠಾನಗೊಳ್ಳುತ್ತವೆ ಎಂಬುದನ್ನು ಸರಿಯಾದ ಟೋಕನ್ ಪರಿಶೀಲನೆ ಅವಲಂಬಿಸಿದೆ
- ನೇರ ಕ್ಲೈಂಟ್-ಟು-API ಟೋಕನ್ ಬಳಕೆ ಈ ಅಗತ್ಯ ರಕ್ಷಣೆಯನ್ನು ತಿರಸ್ಕರಿಸಿ ಭದ್ರತಾ ವಿನ್ಯಾಸವನ್ನು ದುರ್ಬಲಗೊಳಿಸುತ್ತದೆ

##### ಜವಾಬ್ದಾರಿತ್ವ ಮತ್ತು ಲೆಕ್ಕ ಪರಿಶೀಲನೆ ಸವಾಲುಗಳು  
- MCP ಸರ್ವರ್‌ಗಳು ಮೇಲ್ವಿಚಾರಣೆಯ ಟೋಕನ್ ಬಳಸುತ್ತಿರುವ ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ವಿಭೇದಿಸಲಾಗುವುದಿಲ್ಲ, ಲೆಕ್ಕ ಪರಿಶೀಲನೆ ಟ್ರೈಲುಗಳು ಭಂಗವಾಗುತ್ತವೆ
- ನಂತರದ ಸಂಪನ್ಮೂಲ ಸರ್ವರ್ ಲಾಗ್ಗಳು ವಿವರಣೆಬದಲ್ ವಾದ ವಿನಂತಿ ಮೂಲಗಳನ್ನು ತೋರಿಸುತ್ತವೆ, ವಾಸ್ತವ MCP ಸರ್ವರ್ ಮಧ್ಯವರ్తಿಗಳನ್ನು ಅಲ್ಲ
- ಘಟನೆ ತನಿಖೆ ಮತ್ತು ಅನುಷ್ಟಾನ ಲೆಕ್ಕ ಪರಿಶೀಲನೆ ಬಹಳ ಸವಾಲಾಗುತ್ತದೆ

##### ಡೇಟಾ ಹೊರತಾಗುವ ಅಪಾಯಗಳು
- ಪರಿಶೀಲಿಸದ ಟೋಕನ್ ಹಕ್ಕುಗಳು ಕಳ್ಳತನಗೊಂಡ ಟೋಕನ್ಗಳನ್ನು ಹೊಂದಿದ ದುರ್ಬೃತ್ತಿರು MCP ಸರ್ವರ್‌ಗಳನ್ನು ಡೇಟಾ ಹೊರತೆಗೆಯಲು ಪ್ರಾಕ್ಸಿಗಳಾಗಿ ಬಳಸಲು ಸುವರ್ಣಾವಕಾಶ ಕೊಡುತ್ತದೆ
- ನಂಬಿಕೆ ಸೀಮೆ ಉಲ್ಲಂಘನೆಗಳು ಅನುಮತಿಸಲಾರದ ಪ್ರವೇಶ ಮಾದರಿಗಳನ್ನು ಪಾರಾಗುವಂತೆ ಮಾಡುತ್ತವೆ

##### ಬಹು-ಸೇವೆ ದಾಳಿ ಮಾರ್ಗಗಳು
- ಹಾಳಾದ ಟೋಕನ್‌ಗಳು ಹಲವಾರು ಸೇವೆಗಳಲ್ಲಿ ಒಪ್ಪಿಗೆಯಾಗಿದ್ದು ಸಂಪರ್ಕಿತ ವ್ಯವಸ್ಥೆಗಳ ಗೋಡೆಗಳಲ್ಲಿನ ಚಲನೆಯಂತೆ ಚಲಿಸುತ್ತದೆ
- ಟೋಕನ್ ಮೂಲಗಳನ್ನು ಪರಿಶೀಲಿಸಲಾಗದಿರುವಾಗ ಸೇವೆಗಳ ನಡುವೆ ನಂಬಿಕೆ ಊಹೆಗಳು ಭಂಗವಾಗಬಹುದು

### ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು ಮತ್ತು ತಡೆಗಳು

**ಪ್ರಮುಖ ಭದ್ರತಾ ಅಗತ್ಯತೆಗಳು:**

> **ಕಡ್ಡಾಯ**: MCP ಸರ್ವರ್‌ಗಳು **ಸ್ವತಃ ಸ್ಥಳೀಯವಾಗಿ MCP ಸರ್ವರ್‌ಗೆ ನೀಡಲಾಗದ** ಯಾವುದೇ ಟೋಕನ್ಗಳನ್ನು ಸ್ವೀಕರಿಸಬಾರದು

#### ಪ್ರಾಮಾಣಿಕತೆ ಮತ್ತು ಪ್ರಾಧಿಕರಣೆ ನಿಯಂತ್ರಣಗಳು

- **ಕಠಿಣ ಪ್ರಾಧಿಕರಣ ಪರಿಶೀಲನೆ**: MCP ಸರ್ವರ್ ಪ್ರಾಧಿಕರಣೆ ತರ್ಕದ ಸಂಪೂರ್ಣ ಲೆಕ್ಕ ಪರಿಶೀಲನೆ ಮಾಡಿ, ನಿರ್ಧಾರಿತ ಬಳಕೆದಾರರು ಮತ್ತು ಕ್ಲೈಂಟ್‌ಗಳು ಮಾತ್ರ ಸಂವೇದನಾಶೀಲ ಸಂಪನ್ಮೂಲಗಳಿಗೆ ಪ್ರವೇಶಿಸಲು ಭದ್ರಪಡಿಸಿ
  - **ಅನುಷ್ಠಾನ ಮಾರ್ಗದರ್ಶಿ**: [MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಭದ್ರತಾ ಪ್ರವೇಶದ್ವಾರವಾಗಿ Azure API ನಿರ್ವಹಣೆ](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ಐಡೆಂಟಿಟಿ ಏಕೀಕರಣ**: [MCP ಸರ್ವರ್ ಪ್ರಾಮಾಣಿಕತೆಗಾಗಿ Microsoft Entra ID ಬಳಕೆ](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **ಭದ್ರ ಟೋಕನ್ ನಿರ್ವಹಣೆ**: [Microsoft ನ ಟೋಕನ್ ಪರಿಶೀಲನೆ ಮತ್ತು ಜೀವನಚಕ್ರ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) ಅನುಷ್ಠಾನ ಮಾಡಿ
  - ಟೋಕನ್ ಪ್ರೇಕ್ಷಕ ಹಕ್ಕುಗಳು MCP ಸರ್ವರ್ ಐಡೆಂಟಿಟಿಯೊಂದಿಗೆ ಹೊಂದಿಕೆಯಾಗಿರುವುದಾಗಿ ಪರಿಶೀಲನೆ ಮಾಡಿರಿ
  - ಸರಿಯಾದ ಟೋಕನ್ ತಿರುಗಾಟ ಮತ್ತು ಅವಧಿ ಮುಗಿಯುವ ನೀತಿಗಳನ್ನು ಅನುಷ್ಠಾನ ಮಾಡಿ
  - ಟೋಕನ್ ಮರುಪ್ರಯೋಜನ ದಾಳಿಗಳನ್ನು ತಡೆಗಟ್ಟಿರಿ ಮತ್ತು ಅನುಮತಿಸದ ಬಳಕೆಯನ್ನು ನಿರಾಕರಿಸಿ

- **ರಕ್ಷಿತ ಟೋಕನ್ ಸಂಗ್ರಹಣೆ**: ವಿಶ್ರಾಂತಕ್ಕಾಗಿಯೂ ಸಂಚಾರದಲ್ಲಿಯೂ ಎನ್‌ಕ್ರಿಪ್ಟ್ ಮಾಡಿದ ಟೋಕನ್ ಸಂಗ್ರಹಣೆಯನ್ನು ಭದ್ರಪಡಿಸಿ
  - **ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು**: [ಭದ್ರ ಟೋಕನ್ ಸಂಗ್ರಹಣೆ ಮತ್ತು ಎನ್‌ಕ್ರಿಪ್ಷನ್ ಮಾರ್ಗದರ್ಶಿ](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### ಪ್ರವೇಶ ನಿಯಂತ್ರಣ ಅನುಷ್ಠಾನ

- **ಕನಿಷ್ಠ ಪ್ರಿವಿಲೇಜ್ ತತ್ವ**: MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಕೇವಲ ಅಗತ್ಯವಿರುವ ಕನಿಷ್ಠ ಅನುಮತಿಗಳನ್ನು ನೀಡಿ
  - privilage creep ತಪ್ಪಿಸಲು ನಿಯಮಿತ ಅನುಮತಿ ಪರಿಶೀಲನೆ ಮತ್ತು ನವೀಕರಣ
  - **Microsoft ಡಾಕ್ಯುಮೆಂಟೇಶನ್**: [ಕನಿಷ್ಠ-ಅನುಮತಿಯನ್ನು ಸುರಕ್ಷಿತಗೊಳಿಸುವುದು](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **ಪಾತ್ರಾಧಾರಕ ಪ್ರವೇಶ ನಿಯಂತ್ರಣ (RBAC)**: ಸುಕ್ಷ್ಮ ಪಾತ್ರ ನಿಯೋಜನೆಗಳನ್ನು ಅನುಷ್ಠಾನ ಮಾಡಿ
  - ನಿರ್ದಿಷ್ಟ ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಕಾರ್ಯಗಳಿಗೆ ಪಾತ್ರಗಳನ್ನು ಕಟ್ಟುನಿಟ್ಟಾಗಿಡಿ
  - ದಾಳಿದಾರರ ವಿಸ್ತರಣೆಯನ್ನು ತಡೆಯಲು ಅಗತ್ಯವಿಲ್ಲದ ಅಥವಾ ವ್ಯಾಪಕ ಅನುಮತಿಗಳನ್ನು ತಪ್ಪಿಸಿ

- **ನಿರಂತರ ಅನುಮತಿ ಮಾನಿಟರಿಂಗ್**: ನಿರಂತರ ಪ್ರವೇಶ ಲೆಕ್ಕ ಪರಿಶೀಲನೆ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆಯನ್ನು ಅನುಷ್ಠಾನ ಮಾಡಿ
  - ಅನುಮತಿ ಬಳಕೆ ಮಾದರಿಗಳ ಮೇಲೆ ಅನಾಮಲಿಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ
  - ಅತಿರಿಕ್ತ ಅಥವಾ ಉಪಯೋಗವಾಗದ privilage ಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಪರಿಹರಿಸಿ

## AI-ನಿರ್ದಿಷ್ಟ ಭದ್ರತಾ ಅಪಾಯಗಳು

### prompt injection ಮತ್ತು ಟೂಲ್ ಮ್ಯಾನಿಪ್ಯುಲೇಷನ್ ದಾಳಿ

ಆಧುನಿಕ MCP ಅನುಷ್ಠಾನಗಳು ಸಾಂಪ್ರದಾಯಿಕ ಭದ್ರತಾ ಕ್ರಮಗಳು ಪೂರ್ಣವಾಗಿ ಪರಿಹರಿಸಲಾರದ ಸುಕ್ಷ್ಮ AI-ನಿರ್ದಿಷ್ಟ ದಾಳಿ ಮಾರ್ಗಗಳನ್ನು ಎದುರಿಸುತ್ತವೆ:

#### **ಪರೋಕ್ಷ prompt injection (ಕ್ರಾಸ್-ಡೊಮೇನ್ prompt injection)**

**ಪರೋಕ್ಷ prompt injection** MCP ಸಕ್ರಿಯ AI ವ್ಯವಸ್ಥೆಗಳಲ್ಲಿನ ಅತ್ಯಂತ ಪ್ರಮುಖ ದೋಷಪರಮದ ದುರ್ಬಲತೆ ಒಂದು. ದಾಳಿದಾರರು ಬಾಹ್ಯ ವಿಷಯಗಳಲ್ಲಿ—ಪತ್ರಿಕೆಗಳು, ವೆಬ್ ಪುಟಗಳು, ಇಮೇಲ್‌ಗಳು ಅಥವಾ ಡೇಟಾ ಮೂಲಗಳಲ್ಲಿ—ಹಾನಿಕರ ಸೂಚನೆಗಳನ್ನು ಒಳಗೊಂಡಿರುತ್ತಾರೆ, ಅವನ್ನು AI ವ್ಯವಸ್ಥೆಗಳು ನಂಬಪಡುವ ಆಜ್ಞೆಗಳಾಗಿ ಪ್ರಕ್ರಿಯೆ ಮಾಡುತ್ತವೆ.

**ದಾಳಿ ಘಟನೆಗಳು:**
- **ದಾಖಲೆ ಆಧಾರಿತ ಸಂಯೋಜನೆ**: ಪ್ರಕ್ರಿಯೆಗೆ ಒಳಗಾಗುವ ದಾಖಲೆಗಳಲ್ಲಿ ಮೊಸಗಿಸಿದ ಹಾನಿಕರ ಸೂಚನೆಗಳು ಅನಿರೀಕ್ಷಿತ AI ಕ್ರಿಯೆಗಳನ್ನು ಪ್ರೇರೇಪಿಸುತ್ತವೆ
- **ವೆಬ್ ವಿಷಯ ದುರುಪಯೋಗ**: ಸಂಶೋಧಿತಾಗಿರುವ ವೆಬ್ ಪುಟಗಳಲ್ಲಿ ಅಂಟಿಸಲಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳು AI ನಡೆಗಿನ ಮೇಲೆ ಪರಿಣಾಮ ಬೀರುತ್ತವೆ
- **ಇಮೇಲ್ ಆಧಾರಿತ ದಾಳಿಗಳು**: ಇಮೇಲ್‌ಗಳಲ್ಲಿ ಮೋಸದ ಪ್ರಾಂಪ್ಟ್‌ಗಳು AI ಸಹಾಯಕರಿಂದ ತಿಳಿವಾಗದ ಮಾಹಿತಿಯನ್ನು ಹೊರಹಾಕುವಂತೆ ಅಥವಾ ಅನುಮತಿಸದ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಲು ಪ್ರೇರೇಪಿಸುತ್ತವೆ
- **ಡೇಟಾ ಮೂಲ ಮಾಲಿನ್ಯ**: ಆಘಾತಗೊಳ್ಳಲಾದ ಡೇಟಾಬೇಸ್‌ಗಳು ಅಥವಾ APIಗಳು AI ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಹೊರ್‌ತು ಕಳುಹಿಸುವ ಹಾನಿಕರ ವಿಷಯವನ್ನು ಒದಗಿಸುತ್ತವೆ

**ವಾಸ್ತವ ಪ್ರಭಾವ:** ಈ ದಾಳಿಗಳು ಡೇಟಾ ಚೋರಿ, ಗೌಪ್ಯತೆ ಉಲ್ಲಂಘನೆ, ಅನಾಹುತಕಾರಿ ವಿಷಯ ಹುಟ್ಟುಹಾಕುವಿಕೆ ಮತ್ತು ಬಳಕೆದಾರ ಸಂವೇದನೆಗಳ ನಾಶಕ್ಕೆ ಕಾರಣವಾಗಬಹುದು. ವಿವರವಾದ ವಿಶ್ಲೇಷಣೆಗೆ ನೋಡಿರಿ [MCPಯಲ್ಲಿ prompt injection (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).

![Prompt Injection Attack Diagram](../../../translated_images/kn/prompt-injection.ed9fbfde297ca877.webp)

#### **ಟೂಲ್ ವಿಷಕಾರಿ ದಾಳಿಗಳು**

**ಟೂಲ್ ವಿಷಕಾರಿ** MCP ಉಪಕರಣಗಳನ್ನು ನಿರ್ಧರಿಸುವ ಮೆಟಾಡೇಟಾಗಳನ್ನು ಗುರಿಯಾಗಿಸುತ್ತದೆ, ಹೇಗೂ LLM ಗಳು ಉಪಕರಣ ವಿವರಣೆ ಮತ್ತು ವ್ಯತ್ಯಾಸಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದರ ಮೇಲೆ ಆಧಾರಿತ ಆಯ್ಕೆಯನ್ನು ಮಾಡುತ್ತವೆ.

**ದಾಳಿ ವಿಧಾನಗಳು:**
- **ಮೆಟಾಡೇಟಾ ನಿರ್ವಹಣೆ**: ದಾಳಿದಾರರು ಟೂಲಿನ ವಿವರಣೆ, ಪರಾಮೀಟರ್ ವ್ಯಾಖ್ಯಾನಗಳು ಅಥವಾ ಬಳಕೆ ಉದಾಹರಣೆಗಳಲ್ಲಿ ಹಾನಿಕರ ಸೂಚನೆಗಳನ್ನು ಸೇರಿಸುತ್ತಾರೆ
- **ಅವ್ಯಕ್ತ ಸೂಚನೆಗಳು**: ಮಾನವ ಬಳಕೆದಾರರಿಗೆ ಕಾಣದಿರುವ ಬದಲಿಗೆ AI ಮಾದರಿಗಳು ಸಂಸ್ಕರಿಸುವ ಲಿಪಿಸಲಾಗದ prompt‌ಗಳು
- **ಡೈನಾಮಿಕ್ ಟೂಲ್ ಬದಲಾವಣೆ ("ರಗ್풀್ಸ್")**: ಬಳಕೆದಾರರಿಂದ ಅನುಮೋದಿತ ಉಪಕರಣಗಳನ್ನು ಬದಲಾಗಿಸಿ ಫಲಾನುಭವಿಗಳ ತಿಳಿವಿಗೆ ಇಲ್ಲದೆ ಹಾನಿಕರ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಲು ಬದಲಾಯಿಸಲಾಗುವುದು
- **ಪರಾಮೀಟರ್ ಸಂಯೋಜನೆ**: ಮಾದರಿಯ ವರ್ತನೆಗೆ ಪ್ರಭಾವ ಬೀರುವ ಟೂಲ್ ಪರಾಮೀಟರ್ ವಿನ್ಯಾಸಗಳಲ್ಲಿ ಹಾನಿಕರ ವಿಷಯ ಸಂಯೋಜನೆ


**ಹೋಸ್ಟ್ செய்யப்பட்ட ಸರ್ವರ್ ಅಪಾಯಗಳು**: ದೂರದ MCP ಸರ್ವರ್‌ಗಳು ಉಪಕರಣ ವ್ಯಾಖ್ಯಾನಗಳನ್ನು ಆರಂಭಿಕ ಬಳಕೆದಾರ ಅನುಮೋದನೆಯ ನಂತರವೂ ನವೀಕರಿಸಬಹುದಾದ ಕಾರಣದಿಂದ ಹೆಚ್ಚಿದ ಅಪಾಯಕಾರಿ ಪರಿಸ್ಥಿತಿಗಳನ್ನು ಉಂಟುಮಾಡುತ್ತವೆ, ಹೀಗಾಗಿ ಹಿಂದಿನ ಭದ್ರಿತ ಉಪಕರಣಗಳು ಹಾನಿಕಾರಕವಾಗಬಹುದು. ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆಗೆ, [ಟೂಲ್ ಪಾಯಿಸನಿಂಗ್ ದಾಳಿಗಳು (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) ಅನ್ನು ನೋಡಿ.

![Tool Injection Attack Diagram](../../../translated_images/kn/tool-injection.3b0b4a6b24de6bef.webp)

#### **ಹೆಚ್ಚುವರಿ AI ದಾಳಿಗಳು**

- **ಕ್ರಾಸ್-ಡೊಮೈನ್ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ (XPIA)**: ಹಲವು ಡೊಮೇನ್‌ಗಳ ವಿಷಯವನ್ನು ಉಪಯೋಗಿಸಿದ ಸಂಕೀರ್ಣ ದಾಳಿಗಳು, ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ದಾಟಿಕೊಂಡು
- **ಡೈನಾಮಿಕ್ ಸಾಮರ್ಥ್ಯ ಪರಿಷ್ಕರಣೆ**: ಉಪಕರಣ ಸಾಮರ್ಥ್ಯಗಳಲ್ಲಿ ನಿಜಸಮಯ ಬದಲಾವಣೆಗಳು, ಆರಂಭಿಕ ಭದ್ರತಾ ಮೌಲ್ಯಮಾಪನಗಳಿಂದ ತಪ್ಪಿಸಿಕೊಂಡು
- **ಕಾಂಟೆಕ್ಸ್ಟ್ ವಿಂಡೋ ಪಾಯಿಸನಿಂಗ್**: ದೊಡ್ಡ ಕಾಂಟೆಕ್ಸ್ಟ್ ವಿಂಡೋವನ್ನು ನಿಯಂತ್ರಿಸಿಕೊಂಡು ಹಾನಿಕಾರಕ ಸೂಚನೆಗಳನ್ನು ಮರೆಮುಗಿಸಿ ನಡೆಸುವ ದಾಳಿಗಳು
- **ಮಾದರಿ ಗೊಂದಲ ದಾಳಿಗಳು**: ಮಾದರಿ ಮಿತಿಗಳನ್ನು ದುರುಪಯೋಗಿಸಿ ಅನಿರೀಕ್ಷಿತ ಅಥವಾ ಅಸುರಕ್ಷಿತ ವರ್ತನೆಗಳನ್ನು ಸೃಷ್ಟಿಸುವುದು


### AI ಭದ್ರತಾ ಅಪಾಯದ ಪ್ರಭಾವ

**ಗರಿಷ್ಠ ಪ್ರಭಾವಿತ ಪರಿಣಾಮಗಳು:**
- **ಡೇಟಾ ಕಳ್ಳತನ**: ಅನುಮತಿಸದ ಪ್ರವೇಶ ಮತ್ತು ಸಂತ೯ಪ್ತ ಕೈಗೆಡವಿ
- **ಗೌಪ್ಯತೆ ಉಲ್ಲಂಘನೆಗಳು**: ವ್ಯಕ್ತಿಗತ ಗುರುತಿಸಬಹುದಾದ ಮಾಹಿತಿಯ (PII) ಮತ್ತು ರಹಸ್ಯ ವ್ಯವಹಾರ ಡೇಟಾ ಬಹಿರ್ಗತಗೊಳ್ಳುವುದು  
- **ಸಿಸ್ಟಮ್ ಪರಿಷ್ಕರಣೆ**: ಪ್ರಮುಖ ಸಿಸ್ಟಮ್‌ಗಳು ಮತ್ತು ಕಾರ್ಯಪ್ರವಾಹಗಳಲ್ಲಿ ಅನಿರೀಕ್ಷಿತ ಸುಧಾರಣೆಗಳು
- **ಪ್ರಮಾಣಪತ್ರ ಕಳ್ಳತನ**: ದೃಢೀಕರಣ ಟೋಕನ್‌ಗಳು ಮತ್ತು ಸೇವಾ ಪ್ರಮಾಣಪತ್ರಗಳ ಭಂಗ
- **ಅಡ್ಡಸಂಚಾರ**: ಬಾಧಿತ AI ಸಿಸ್ಟಮ್‌ಗಳನ್ನು ಅಗಲವಾದ ನೆಟ್ವರ್ಕ್ ದಾಳಿಗಳಿಗಾಗಿ ಪಿವಟ್ಸ್ ಆಗಿ ಬಳಕೆ ಮಾಡುವುದು

### ಮೈಕ್ರೋಸಾಫ್ಟ್ AI ಭದ್ರತಾ ಪರಿಹಾರಗಳು

#### **AI ಪ್ರಾಂಪ್ಟ್ ಶೀಲ್ಡ್ಸ್: ಇಂಜೆಕ್ಷನ್ ದಾಳಿಗಳ ವಿರುದ್ಧ ಉನ್ನತ ರಕ್ಷಣೆ**

ಮೈಕ್ರೋಸಾಫ್ಟ್ **AI ಪ್ರಾಂಪ್ಟ್ ಶೀಲ್ಡ್‌ಗಳು** ನೇರ ಮತ್ತು ಪರೋಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ದಾಳಿಗಳ ವಿರುದ್ಧ ಹಲವಾರು ಭದ್ರತಾ ಹಂತಗಳ ಮೂಲಕ ಸಮಗ್ರ ರಕ್ಷಣೆಯನ್ನು ಒದಗಿಸುತ್ತವೆ:

##### **ಪ್ರಮುಖ ರಕ್ಷಣಾ ತಂತ್ರಗಳು:**

1. **ಅತ್ಯಾಧುನಿಕ ಗುರುತು ಮತ್ತು ಫಿಲ್ಟರಿಂಗ್**
   - ಯಂತ್ರ ಕಲಿಕೆ ಅನಾಲ್ಗೊರಿಥಮ್‌ಗಳು ಮತ್ತು NLP ತಂತ್ರಜ್ಞಾನಗಳು ಹೊರಗಿನ ವಿಷಯದಲ್ಲಿರುವ ಹಾನಿಕಾರಕ ಸೂಚನೆಗಳನ್ನು ಗುರುತಿಸುತ್ತವೆ
   - ದಾಖಲಾತಿಗಳು, ವೆಬ್ ಪುಟಗಳು, ಇಮೇಲ್‌ಗಳು ಮತ್ತು ಡೇಟಾ ಮೂಲಗಳನ್ನು ನಿಜಸಮಯದಲ್ಲಿ ವಿಶ್ಲೇಷಣೆ ಮಾಡುತ್ತದೆ
   - ಮಾನ್ಯ ಮತ್ತು ಹಾನಿಕಾರಕ ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳನ್ನು ಸಂಬಂಧಿಸಿದ ಪ್ರContextನಲ್ಲಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತದೆ

2. **સ્પોટ್ಲાઇટಿಂಗ್ ತಂತ್ರಗಳು**  
   - ನಂಬಬಹುದಾದ ಸಿಸ್ಟಮ್ ಸೂಚನೆಗಳು ಮತ್ತು ಬಾಧಿತ ಹೊರಗಿನ ಇನ್‌ಪುಟ್‌ಗಳ ನಡುವೆ ಭೇದಿಸುವುದು
   - ಮಾದರಿಯ ಪ್ರಸ್ತುತಿಕರಣವನ್ನು ಸುಧಾರಿಸುವ ಪಠ್ಯ ಪರಿವರ್ತನೆ ವಿಧಾನಗಳು ಹಾಗೂ ಹಾನಿಕಾರಕ ವಿಷಯವನ್ನು पृथಕಗೊಳಿಸುವುದು
   - AI ಸಿಸ್ಟಮ್‌ಗಳು ಸರಿಯಾದ ಸೂಚನೆ ಹಿರಾರ್ಕಿಯನ್ನು ನಿರ್ವಹಿಸಿ ಎಂಜೆಕ್ಟ್ ಆದ ಆಜ್ಞೆಗಳನ್ನು ನಿರ್ಲಕ್ಷಿಸುವಲ್ಲಿ ನೆರವಾಗುತ್ತದೆ

3. **ಡಿಲಿಮಿಟರ್ & ಡೇಟಾಮಾರ್ಕಿಂಗ್ ವ್ಯವಸ್ಥೆಗಳು**
   - ನಂಬಬಹುದಾದ ಸಿಸ್ಟಮ್ ಸಂದೇಶಗಳು ಮತ್ತು ಹೊರಗಿನ ಇನ್‌ಪುಟ್ ಪಠ್ಯದ ಮಧ್ಯೆ ಸ್ಪಷ್ಟ ಗಡಿಯ ವ್ಯಾಖ್ಯಾನ
   - ವಿಶೇಷ ಚಿಹ್ನೆಗಳು ನಂಬಬಹುದಾದ ಮತ್ತು ನಂಬಲಾರದ್ದು ಡೇಟಾ ಮೂಲಗಳ ನಡುವಿನ ಗಡಿಗಳನ್ನು ಕೊಟ್ಟಿಸುತ್ತವೆ
   - ಸ್ಪಷ್ಟ ವಿಭಜನೆ ಸೂಚನೆ ಗೊಂದಲ ಮತ್ತು ಅನುಮತಿಸದ ಆಜ್ಞೆಗಳ ಅನುಷ್ಠಾನವನ್ನು ತಡೆಯುತ್ತದೆ

4. **ನಿರಂತರ ಹಿಂಸೆ ಬುದ್ಧಿವಂತಿಕೆ**
   - ಮೈಕ್ರೋಸಾಫ್ಟ್ ಹೊಸದಾಗಿ ಏರುತ್ತಿರುವ ದಾಳಿ ಮಾದರಿಗಳನ್ನು ನಿರಂತರವಾಗಿ ಗಮನಿಸುತ್ತಿದ್ದು ರಕ್ಷಣೆಯನ್ನು ನವೀಕರಿಸುತ್ತದೆ
   - ಹೊಸ ಇಂಜೆಕ್ಷನ್ ತಂತ್ರಗಳು ಮತ್ತು ದಾಳಿಗಳಿಗೆ ಸಕ್ರಿಯವಾಗಿ ಧ್ವನಿಸಾಧನೆ
   - ಬದಲಾಗುತ್ತಿರುವ ಅಪಾಯಗಳ ವಿರುದ್ಧ ಪರಿಣಾಮಕಾರಿತ್ವವನ್ನು ಕಾಯ್ದುಕೊಳ್ಳಲು ನಿಯಮಿತ ಭದ್ರತಾ ಮಾದರಿ ನವೀಕರಣಗಳು

5. **ಅಜ್ಯೂರ್ ವಿಷಯ ಸುರಕ್ಷತೆ ಸಂಯೋಜನೆ**
   - ಸಮಗ್ರ ಅಜ್ಯೂರ್ AI ವಿಷಯ ಸುರಕ್ಷತೆ ಪ್ಯಾಕೇಜಿನ ಭಾಗ
   - ಜೈಲ್ಬ್ರೆಕ್ ಪ್ರಯತ್ನಗಳು, ಹಾನಿಕಾರಕ ವಿಷಯ ಮತ್ತು ಭದ್ರತಾ ನೀತಿನಿರ್ಲಂಘನೆಗಳ అదనపు ಗುರುತಿಸುತೆ ಆಯ್ಕೆ
   - AI ಅಪ್ಲಿಕೇಶನ್ ಘಟಕಗಳಾದ ಮೇಲೆ ഏകೀಕೃತ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು

## ಉನ್ನತ MCP ಭದ್ರತಾ ಬೆದರಿಕೆಗಳು

### ಸೆಷನ್ ಹೈಜಾಕಿಂಗ್ ದುರ್ಬಲತೆಗಳು

**ಸೆಷನ್ ಹೈಜಾಕಿಂಗ್** ಅವರು ಸ್ಥಿತಿಶೀಲ MCP ಅನುಷ್ಠಾನಗಳಲ್ಲಿ ಪ್ರಮುಖ ದಾಳಿ ಮಾರ್ಗವಾಗಿದೆ, ಅನಧಿಕೃತ ಪಕ್ಷಗಳು ಮಾನ್ಯ ಸೆಷನ್ ಗುರುತಿಗಳನ್ನು ಪಡೆದರೆ ಗ್ರಾಹಕರಂತೆ ನಟಿಸಿ ಅನುಮತಿಸದ ಕ್ರಿಯೆಗಳನ್ನು ನಡೆಸಬಹುದು.

#### **ದಾಳಿ ಸಂದರ್ಭಗಳು ಮತ್ತು ಅಪಾಯಗಳು**

- **ಸೆಷನ್ ಹೈಜಾಕ್ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್**: ಕಳ್ಳತನಗೊಂಡ ಸೆಷನ್ ಐಡಿಗಳಿಂದ ದಾಳಿಕಾರರು ಸೆಷನ್ ಸ್ಥಿತಿಯನ್ನು ಹಂಚುವ ಸರ್ವರ್‌ಗಳಿಗೆ ಹಾನಿಕಾರಕ ಘಟನೆಗಳನ್ನು इंजೆಕ್ಟ್ ಮಾಡುವ ಮೂಲಕ ಅಪಾಯಕಾರಿಯಾಗಿ ಡೇಟಾವನ್ನು ಪ್ರವೇಶಿಸಬಹುದು
- **ನೇರ ಅಂಶ ಪರ್ಯಾಯತೆ**: ಕಳ್ಳತನಗೊಂಡ ಸೆಷನ್ ಐಡಿಗಳು ನಿರ್ಬಂಧವನ್ನು ತಳ್ಳಿ MCP ಸರ್ವರ್ ಕರೆಗಳನ್ನು ನೇರವಾಗಿ ಮಾಡಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ, ದಾಳಿಕಾರರನ್ನು ಮಾನ್ಯ ಬಳಕೆದಾರರಂತೆ ಪರಿಗಣಿಸುತ್ತದೆ
- **ಬಾಧಿತ ಪುನರಾರಂಭಿಸುವ ಸ್ಟ್ರೀಮ್ಸ್**: ದಾಳಿಕಾರರು ವಿನಂತಿಗಳನ್ನು ಮುಂಚಿತವಾಗಿ ಕೊನೆಗೊಳಿಸುವ ಮೂಲಕ ಮಾನ್ಯ ಗ್ರಾಹಕರು ಹಾನಿಕಾರಕ ವಿಷಯದೊಂದಿಗೆ ಪುನರಾರಂಭಿಸಲು ಕಾರಣವಾಗಬಹುದು

#### **ಸೆಷನ್ ನಿರ್ವಹಣೆಯ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು**

**ಅತ್ಯಗತ್ಯ ಬೇಡಿಕೆಗಳು:**
- **ಅಧಿಕೃತ ಪರಿಶೀಲನೆ**: ಪರವಾನಗಿ ಅನುಷ್ಠಾನಗೊಂಡ MCP ಸರ್ವರ್‌ಗಳು ಎಲ್ಲ ಒಳಬರುವ ವಿನಂತಿಗಳ ಪರಿಶೀಲನೆ ಮಾಡಬೇಕು ಮತ್ತು ಪ್ರಮಾಣೀಕರಣಕ್ಕಾಗಿ ಸೆಷನ್‌ಗಳ ಮೇಲೆ ಅವಲಂಬಿಸಬಾರದು
- **ಸುರಕ್ಷಿತ ಸೆಷನ್ ತಯಾರಿ**: ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕಲ್ ಸುರಕ್ಷಿತ, ಅನಿಶ್ಚಿತ ಸೆಷನ್ ಐಡಿಗಳನ್ನು ಸುರಕ್ಷಿತ ಯಾದೃಚ್ಛಿಕ ಸಂಖ್ಯಾ ಜನರೇಟರ್‌ಗಳಿಂದ ರಚಿಸಬೇಕು
- **ಬಳಕೆದಾರ-ವಿಶಿಷ್ಟ ಬಾಂಧವ್ಯ**: ಸೆಷನ್ ಐಡಿಗಳನ್ನು ಬಳಕೆದಾರ-ವಿಶಿಷ್ಟ ಮಾಹಿತಿಗೆ ಬಾಂಧಿಸಿ `<user_id>:<session_id>` ಮೂಲಕ ಅಪರಸ್ಪರ ಬಳಕೆಬಿಟ್ಟೊಯ್ಯುವಿಕೆಯನ್ನು ತಡೆಯಬೇಕು
- **ಸೆಷನ್ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ**: ದುರ್ಬಲತಾ ಗಡಿಗಳನ್ನು ಸೀಮಿತಗೊಳಿಸಲು ಸೂಕ್ತ ಅವಧಿಮಿತಿ, ತಿರುಗುಪಡಿ ಮತ್ತು ಅಮಾನ್ಯೀಕರಣ ಕಾರ್ಯಗತಗೊಳಿಸಬೇಕು
- **ಸಂಪ್ರೇಷಣ ಭದ್ರತೆ**: ಸೆಷನ್ ಐಡಿ ಚೋರಿ ತಪ್ಪಿಸಲು ಎಲ್ಲಾ ಸಂವಹನ HTTPS ಅನಿವಾರ್ಯ

### ಗೊಂದಲಕ್ಕೊಳಗಾದ ಪ್ರತಿನಿಧಿ ಸಮಸ್ಯೆ

**ಗೊಂದಲಕ್ಕೊಳಗಾದ ಪ್ರತಿನಿಧಿ ಸಮಸ್ಯೆ** ಎಂದು MCP ಸರ್ವರ್‌ಗಳು ಗ್ರಾಹಕರು ಮತ್ತು ಮೂರನೇ-पಕ್ಷ ಸೇವೆಗಳ ನಡುವೆ ಪ್ರಮಾಣೀಕರಣ ಪ್ರಾಕ್ಸಿ ಆಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವಾಗ, ಸ್ಥಿರ ಗ್ರಾಹಕ ಐಡಿಯಿಂದ ನಿರ್ಬಂಧಗಳನ್ನು ತಪ್ಪಿಸಲು ಅವಕಾಶ ಸೃಷ್ಟಿಯಾಗುತ್ತದೆ.

#### **ದಾಳಿ ವಿಧಾನಗಳು ಮತ್ತು ಅಪಾಯಗಳು**

- **ಕುકી ಆಧಾರಿತ ಸಹಮತ ತಪ್ಪಿಸುವಿಕೆ**: ಹಿಂದಿನ ಬಳಕೆದಾರ ಪ್ರಮಾಣೀಕರಣದಿಂದ ಸಹಮತಿ ಕುಕಿಗಳು ಸೃಷ್ಟಿಯಾಗುತ್ತವೆ, ದಾಳಿಕಾರರು ಮೋಸಮಯ ಅಧಿಕೃತತಾ ವಿನಂತಿಗಳು ಮತ್ತು ರಿಡೈರೆಕ್ಟ್ URI ರ ಮೂಲಕ ದುರುಪಯೋಗ ಮಾಡುತ್ತಾರೆ
- **ಅಧಿಕೃತ ಕೋಡ್ ಕಳವು**: ಇದ್ದು ಕುಕಿಗಳು ಅಧಿಕೃತತಾ ಸರ್ವರ್‌ಗಳನ್ನು ಸಹಮತಿ ಪರದೆಗಳನ್ನು ದಾಟಲು ಕಾರಣವಾಗಬಹುದು ಮತ್ತು ಕೋಡ್‌ಗಳನ್ನು ದಾಳಿಕಾರರ ನಿಯಂತ್ರಿತ ಅಂತ್ಯಸ್ಥಳಗಳಿಗೆ ರೀಡೈರೆಕ್ಟ್ ಮಾಡುತ್ತದೆ  
- **ಅನುಮತಿಸದ API ಪ್ರವೇಶ**: ಕಳ್ಳತನಗೊಂಡ ಅಧಿಕೃತತಾ ಕೋಡ್‌ಗಳು ಟೋಕನ್ ವಿನಿಮಯ ಮತ್ತು ಬಳಕೆದಾರ ಕಾತರತನವನ್ನು ಅನುಮತಿಸದಿರಲಿ ಸಾಧ್ಯ

#### **ತಪ್ಪಿಸುವಿಕೆ ತಡೆಯುವ ತಂತ್ರಗಳು**

**ಅನ್ಯೋನ್ಯ ನಿಯಂತ್ರಣಗಳು:**
- **ಸ್ಪಷ್ಟ ಸಹಮತಿ ಬೇಡಿಕೆಗಳು**: ಸದೃಢ ಗ್ರಾಹಕ ಐಡಿಯೊಂದಿಗೆ MCP ಪ್ರಾಕ್ಸಿ ಸರ್ವರ್‌ಗಳು ಪ್ರತಿ ಡೈನಾಮಿಕ್ ನೋಂದಾಯಿತ ಗ್ರಾಹಕನಿಗೂ ಬಳಕೆದಾರ ಸಹಮತವನ್ನು ಪಡೆಯಬೇಕು
- **OAuth 2.1 ಭದ್ರತಾ ಅನುಷ್ಠಾನ**: ಎಲ್ಲಾ ಅಧಿಕೃತತಾ ವಿನಂತಿಗಳಿಗೂ PKCE (ಪ್ರೂಫ್ ಕೀ ಫಾರ್ ಕೋಡ್ ಎಕ್ಸ್ಚೇಂಜ್) ಸೇರಿದಂತೆ ಪ್ರಪ್ರಥಮ OAuth ಭದ್ರತಾ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸಬೇಕು
- **ದೃಢ ಗ್ರಾಹಕ ಪರಿಶೀಲನೆ**: ದುರುಪಯೋಗ ತಪ್ಪಿಸಲು ರಿಡೈರೆಕ್ಟ್ URIಗಳು ಮತ್ತು ಗ್ರಾಹಕ ಗುರುತಿನ ಸೂಕ್ಷ್ಮ ಪರಿಶೀಲನೆ ಮಾಡಬೇಕು

### ಟೋಕನ್ ಪಾಸ್ಥ್ರೂ ದುರ್ಬಲತೆಗಳು  

**ಟೋಕನ್ ಪಾಸ್ಥ್ರೂ** ಎಂದರೆ MCP ಸರ್ವರ್‌ಗಳು ಗ್ರಾಹಕ ಟೋಕನ್‌ಗಳನ್ನು ಸೂಕ್ತ ಪರಿಶೀಲನೆ ಇಲ್ಲದೆ ಸ್ವೀಕರಿಸಿ ಅದನ್ನು ಕೆಳಗಿನ APIಗಳಿಗೆ ಕಳುಹಿಸುವದು, ಇದು MCP ಅಧಿಕೃತಾ ನಿರ್ದಿಷ್ಟತೆಗೆ ವಿರುದ್ಧವಾಗಿದೆ.

#### **ಭದ್ರತಾ ಪರಿಣಾಮಗಳು**

- **ನಿಯಂತ್ರಣಗಳನ್ನು ತಪ್ಪಿಸುವಿಕೆ**: ನೇರ ಗ್ರಾಹಕದಿಂದ API ಗೆ ಟೋಕನ್ ಬಳಕೆ ಮೂಲ ನಿಯಂತ್ರಣಗಳು, ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆಯನ್ನು ತಪ್ಪಿಸುತ್ತದೆ
- **ಆಡಿಟ್ ಟ್ರೇಲ್ ನಲ್ಲಿ ಭಂಗ**: ಮೇಲಿನ ಟೋಕನ್ ಗಳ պատճառով ಗ್ರಾಹಕ ಗುರುತಿಸುವಿಕೆ ಸಾಧ್ಯವಿಲ್ಲ, ಘಟನೆ ಪರಿಶೋಧನೆಗೆ ತೊಂದರೆ ಉಂಟುಮಾಡುತ್ತದೆ
- **ಪ್ರಾಕ್ಸಿ ಮೂಲಕ ಡೇಟಾ ದಾಳಿಗಳು**: ಪರಿಶೀಲನೆ ಇಲ್ಲದ ಟೋಕನ್‌ಗಳು ದುಷ್ಕೃತ್ಯಕರರಿಗೆ ಸೇವಿಗಳನ್ನು ಪ್ರಾಕ್ಸಿಯಾಗಿ ಬಳಸುವ ಅವಕಾಶ ಕೊಟ್ಟಿರುತ್ತದೆ
- **ನಂಬಿಕೆ ಗಡಿಯು ಉಲ್ಲಂಘನೆಗಳು**: ಟೋಕನ್ ಮೂಲವನ್ನು ಪರಿಶೀಲಿಸಲು ಸಾಧ್ಯವಾಗದಾಗ ಕೆಳಗಿನ ಸೇವೆಗಳ ನಂಬಿಕೆ ಸಂಬಂಧ ಖಂಡಿತ
- **ಬಹು-ಸೇವಾ ದಾಳಿ ವಿಸ್ತರಣೆ**: ಹಾನಿಗೊಂಡ ಟೋಕನ್‌ಗಳನ್ನು ಹಲವು ಸೇವೆಗಳಲ್ಲಿ ಒಪ್ಪಿಗೆಯಿಂದ ಬಳಸಿ ಅಡ್ಡಸಂಚಾರ ದಾಳಿಯನ್ನು ಅನುಮತಿಸುತ್ತದೆ

#### **ಅತ್ಯಗತ್ಯ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು**

**ಬದಲಾವಣೆ ಅಸಾಧ್ಯ ಮಟ್ಟಕ್ಕಿಂತ ಮೇಲಿನ ಬೇಡಿಕೆಗಳು:**
- **ಟೋಕನ್ ಪರಿಶೀಲನೆ**: MCP ಸರ್ವರ್‌ಗಳು ಸ್ಪಷ್ಟವಾಗಿ MCP ಸರ್ವರ್‌ಗೆ ನೀಡಲಾಗದ ಟೋಕನ್‌ಗಳನ್ನು ಸ್ವೀಕರಿಸಲಾರವು
- **ಪ್ರೇಕ್ಷಕ ದೃಢೀಕರಣ**: MCP ಸರ್ವರ್ ಗುರುತಿಗೆ ಹೊಂದಿಕೆಯ ಟೋಕನ್ ಪ್ರೇಕ್ಷಕ ಹಕ್ಕುಗಳ ಪರಿಶೀಲನೆ ಮಾಡಬೇಕು
- **ಸರಿಯಾದ ಟೋಕನ್ ಜೀವನಚಕ್ರ**: ಸ್ವಲ್ಪ ಕಾಲದಲ್ಲাজারಗುವ ಪ್ರವೇಶ ಟೋಕನ್‌ಗಳು ಭದ್ರಿತ ತಿರುಗುಪಡಿಯೊಂದಿಗೆ ಅನುಷ್ಠಾನಗೊಳ್ಳಬೇಕು


## AI ಸಿಸ್ಟಮ್‌ಗಳ ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತೆ

ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತೆ ಸಾಂಪ್ರದಾಯಿಕ ಸಾಫ್ಟ್‌ವೇರ್ ಅವಲಂಬನೆಗಳನ್ನು ಮೀರಿಸಿಕೊಂಡು ಸಂಪೂರ್ಣ AI ಪರಿಸರವನ್ನು ಒಳಗೊಂಡಿರುತ್ತದೆ. ಆಧುನಿಕ MCP ಅನುಷ್ಠಾನಗಳು ಎಲ್ಲಾ AI ಸಂಬಂಧಿತ ಘಟಕಗಳನ್ನು ಕಟ್ಟೂಕಟ್ಟಾಗಿ ಪರಿಶೀಲನೆ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆಯಲ್ಲಿರಬೇಕು, ಏಕೆಂದರೆ ಪ್ರತಿಯೊಂದು ಘಟಕವು ವ್ಯವಸ್ಥೆಯ ಸ್ಥಿತಿಸ್ಥಾಪಕತೆಗೆ ಶಂಕಾರ್ಹ ಆಪಾಯಗಳನ್ನು ಪರಿಚಯಿಸುವುದರಿಂದ.

### ವಿಸ್ತೃತ AI ಸರಬರಾಜು ಸರಣಿ ಘಟಕಗಳು

**ಸಾಂಪ್ರದಾಯಿಕ ಸಾಫ್ಟ್‌ವೇರ್ ಅವಲಂಬನೆಗಳು:**
- ಓಪನ್-ಸೋರ್ಸ್ ಲೈಬ್ರರಿಗಳು ಮತ್ತು ಫ್ರೇಮ್‌ವರ್ಕ್‌ಗಳು
- ಕಂಟೈನರ್ ಇಮೇಜುಗಳು ಮತ್ತು ಬೇಸ್ ಸಿಸ್ಟಮ್‌ಗಳು  
- ಅಭಿವೃದ್ಧಿ ಉಪಕರಣಗಳು ಮತ್ತು ನಿರ್ಮಾಣ ಪೈಪ್ಲೈನ್‌ಗಳು
- ಮೂಲಸೌಕರ್ಯ ಘಟಕಗಳು ಮತ್ತು ಸೇವೆಗಳು

**AI-ನಿರ್ದಿಷ್ಟ ಸರಬರಾಜು ಸರಣಿ ಅಂಶಗಳು:**
- **ಫೌಂಡೇಶನ್ ಮಾದರಿಗಳು**: ವಿವಿಧ ಒದಗಿಸುವವರಿಂದ ಪೂರ್ವ-ಪ್ರಶಿಕ್ಷಣ ಪಡೆದ ಮಾದರಿಗಳು, ಮೂಲಸ್ಥಿತಿ ಪರಿಶೀಲನೆಯ ಅಗತ್ಯ
- **ಎಂಬೆಡ್ಡಿಂಗ್ ಸೇವೆಗಳು**: ಹೊರಗಿನ ವೇಕ್ಟರೈಸೇಶನ್ ಮತ್ತು ಅರ್ಥಪೂರ್ಣ ಶೋಧನ ಸೇವೆಗಳು
- **ಕಾಂಟೆಕ್ಸ್ಟ್ ಒದಗಿಸುವವರು**: ಡೇಟಾ ಮೂಲಗಳು, ಜ್ಞಾನ ತಾಣಗಳು ಮತ್ತು ದಾಖಲೆ ಸಂಗ್ರಹಗಳು  
- **ಮೂರು-ಪಕ್ಷ APIs**: ಹೊರಗಿನ AI ಸೇವೆಗಳು, ML ಪೈಪ್ಲೈನ್‌ಗಳು, ಡೇಟಾ ಪ್ರಕ್ರಿಯೆ ಅಂತ್ಯಸ್ಥಳಗಳು
- **ಮಾದರಿ ಕಲೆಗಳು**: ತೂಕಗಳು, ಸಂರಚನೆಗಳು ಮತ್ತು ಸೂಕ್ಷ್ಮ-ತಿದ್ದುಪಡಿ ಹೊಂದಿದ ಮಾದರಿ ತರಂಗಗಳು
- **ತರಬೇತಿ ಡೇಟಾ ಮೂಲಗಳು**: ಮಾದರಿ ತರಬೇತಿ ಮತ್ತು ಸೂಕ್ಷ್ಮ-ತಿದ್ದುಪಡಿ ಬಳಕೆಯಾದ ಡೇಟಾಸೆಟ್‌ಗಳು

### ಸಮಗ್ರ ಸರಬರಾಜು ಸರಣಿ ಭದ್ರತಾ ಯೋಜನೆ

#### **ಘಟಕ ಪರಿಶೀಲನೆ ಮತ್ತು ನಂಬಿಕೆ**
- **ಮೂಲಸ್ಥಿತಿ ಪರಿಶೀಲನೆ**: ಎಲ್ಲ AI ಘಟಕಗಳ ಮೂಲ, ಪರವಾನಗಿ ಮತ್ತು ಭದ್ರತೆಯ ಪೂರ್ಣತೆ ಪರಿಶೀಲಿಸಿ ಸಂಯೋಜನೆಯ ಮೊದಲು
- **ಭದ್ರತಾ ಮೌಲ್ಯಮಾಪನ**: ಮಾದರಿಗಳು, ಡೇಟಾ ಮೂಲಗಳು ಮತ್ತು AI ಸೇವೆಗಳಿಗೆ ದುರ್ಬಲತೆ ಸ್ಕ್ಯಾನ್ ಮತ್ತು ಭದ್ರತಾ ವಿಮರ್ಶೆ ನಡೆಸುವುದು
- **ಖ್ಯಾತಿ ವಿಶ್ಲೇಷಣೆ**: AI ಸೇವೆ ಒದಗಿಸುವವರ ಭದ್ರತಾ ದಾಖಲೆ ಮತ್ತು ಕ್ರಮಗಳನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡುವುದು
- **ಅನುಕೂಲ ಪರಿಶೀಲನೆ**: ಎಲ್ಲಾ ಘಟಕಗಳು ಸಂಸ್ಥೆಯ ಭದ್ರತಾ ಮತ್ತು ನಿಯಂತ್ರಣ ಅಗತ್ಯಗಳನ್ನು ಪೂರೈಸುವಂತೆ ನೋಡಿಕೊಳ್ಳುವುದು

#### **ಸುರಕ್ಷಿತ ನಿಯೋಜನೆ ಪೈಪ್ಲೈನ್‌ಗಳು**  
- **ಸ್ವಯಂಚಾಲಿತ CI/CD ಭದ್ರತೆ**: ಸ್ವಯಂಚಾಲಿತ ನಿಯೋಜನೆ ಪೈಪ್ಲೈನ್‌ಗಳಲ್ಲಿ ಭದ್ರತಾ ಸ್ಕ್ಯಾನ್ ಸಿಂಕ್ರೊನೈಸೇಶನ್
- **ಕಲೆ ಪೂರ್ಣತೆ**: ಎಲ್ಲ ನಿಯೋಜಿತ ಕಲೆಗಳು (ಕೋಡ್, ಮಾದರಿಗಳು, ಸಂರಚನೆ)ಗೆ ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಪರಿಶೀಲನೆ
- **ಹಂತಬದ್ಧ ನಿಯೋಜನೆ**: ಪ್ರತಿಯೊಂದು ಹಂತದಲ್ಲಿ ಭದ್ರತಾ ಪರಿಶೀಲನೆ ಮೂಲಕ ಪ್ರಗತಿಯಲ್ಲಿರುವ ನಿಯೋಜನೆ ತಂತ್ರಗಳು
- **ನಂಬಬಹುದಾದ ಕಲೆ ಸಂಗ್ರಹಣೆಗಳು**: ಪರಿಶೀಲನೆಗೊಂಡ ಸುರಕ್ಷಿತ ಕಲೆ ಸಂಗ್ರಹಣಾ ಕೇಂದ್ರಗಳಿಂದ ಮಾತ್ರ ನಿಯೋಜನೆ

#### **ನಿರಂತರ ನಿರೀಕ್ಷಣೆ ಮತ್ತು ಪ್ರತಿಕ್ರಿಯಾ**
- **ಅವಲಂಬನೆ ಸ್ಕ್ಯಾನಿಂಗ್**: ಎಲ್ಲಾ ಸಾಫ್ಟ್‌ವೇರ್ ಮತ್ತು AI ಘಟಕ ಅವಲಂಬನೆಗಳ ದುರ್ಬಲತೆ ಸತತ ಮಾನಿಟರಿಂಗ್
- **ಮಾದರಿ ನಿರೀಕ್ಷಣೆ**: ಮಾದರಿ ವರ್ತನೆ, ಕಾರ್ಯಕ್ಷಮತೆ ಭ್ರಂಶ ಮತ್ತು ಭದ್ರತಾ ಅನಾಮಲಿಗಳನ್ನು ನಿರಂತರ ಮೌಲ್ಯಮಾಪನ
- **ಸೇವೆ ಆರೋಗ್ಯ ಟ್ರ್ಯಾಕಿಂಗ್**: ಹೊರಗಿನ AI ಸೇವೆಗಳ ಲಭ್ಯತೆ, ಭದ್ರತಾ ಘಟನೆಗಳು ಮತ್ತು ನೀತಿ ಬದಲಾವಣೆಗಳ ಮೌಲ್ಯಮಾಪನ
- **ಹಿಂಸೆ ಬುದ್ಧಿವಂತಿಕೆ ಸಂಯೋಜನೆ**: AI ಮತ್ತು ML ಭದ್ರತಾ ಅಪಾಯಗಳಿಗೆ ನಿರ್ದಿಷ್ಟ ಭೀಷಣ ಸ್ವರೂಪಗಳ ಸಂಯೋಜನೆ

#### **ಪ್ರವೇಶ ನಿಯಂತ್ರಣ ಮತ್ತು ಕನಿಷ್ಟ ಹಕ್ಕು**
- **ಘಟಕ ಮಟ್ಟ ಅನುಮತಿಗಳು**: ಮಾದರಿಗಳು, ಡೇಟಾ ಮತ್ತು ಸೇವೆಗಳ ಪ್ರವೇಶವನ್ನು ವ್ಯವಹಾರ ಅಗತ್ಯಗಳ ಆಧಾರದ ಮೇಲೆ ನಿಯಂತ್ರಿಸುವುದು
- **ಸೇವೆ ಖಾತೆ ನಿರ್ವಹಣೆ**: ಕನಿಷ್ಠ ಅಗತ್ಯ ಅನುಮತಿಗಳೊಂದಿಗೆ ಸಮರ್ಪಕ ಸೇವಾ ಖಾತೆಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವುದು
- **ನೆಟ್ವರ್ಕ್ ವಿಭಜನೆ**: AI ಘಟಕಗಳನ್ನು ಪ್ರತ್ಯೇಕಿಸಿ ಸೇವೆಗಳ ನಡುವೆ ನೆಟ್ವರ್ಕ್ ಪ್ರವೇಶವನ್ನು ಕಡಿಮೆಮಾಡುವುದು
- **API ಗೇಟ್‌ವೇ ನಿಯಂತ್ರಣಗಳು**: ಹೊರಗಿನ AI ಸೇವೆಗಳ ಪ್ರವೇಶ ನಿರ್ವಹಣೆಗಾಗಿ केंद್ರೀಕೃತ API ಗೇಟ್‌ವೇಗಳನ್ನು ಬಳಸುವುದು

#### **ಆಘಾತ ಪ್ರತಿಕ್ರಿಯಾ ಮತ್ತು ಪುನಃಪ್ರಾಪ્તિ**
- **ತ್ವರಿತ ಪ್ರತಿಕ್ರಿಯೆ ಕ್ರಮಗಳು**: ಬಾಧಿತ AI ಘಟಕಗಳನ್ನು ಪ್ಯಾಚ್ ಅಥವಾ ಬದಲಿಸುವ ಸ್ಥಿರ ಪ್ರಕ್ರಿಯೆಗಳು
- **ಪ್ರಮಾಣಪತ್ರ ತಿರುಗುಪಡಿ**: ಗುಪ್ತತೆ, API ಕೀ ಮತ್ತು ಸೇವಾ ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ತಿರುಗಿಸುವ ವ್ಯವಸ್ಥೆಗಳು
- **ರೋಲ್ಬ್ಯಾಕ್ ಸಾಮರ್ಥ್ಯಗಳು**: ಮೊದಲು ತಿಳಿದ-ಉತ್ತಮ ಆವೃತ್ತಿಗಳಿಗೆ ವೇಗವಾಗಿ ಮರಳುವ ಸಾಮರ್ಥ್ಯ
- **ಸರಬರಾಜು ಸರಣಿ ಉಲ್ಲಂಘನೆಯ ಪುನಃಪ್ರಾಪ್ತಿ**: ಮೇಲಿನ AI ಸೇವೆ ಬಾಧಿತ ಆಘಾತಗಳಿಗೆ ಸ್ಪೆಷಲ್ ಪ್ರತಿಕ್ರಿಯಾ ವಿಧಾನಗಳು

### ಮೈಕ್ರೋಸಾಫ್ಟ್ ಭದ್ರತಾ ಸಲಕರಣೆಗಳು ಮತ್ತು ಸಂಯೋಜನೆ

**GitHub Advanced Security** ಸಮಗ್ರ ಸರಬರಾಜು ಸರಣಿ ರಕ್ಷಣೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ, ಇದರಲ್ಲಿ:
- **ಗುಪ್ತ ಕೀವನ್ನು ಸ್ಕ್ಯಾನಿಂಗ್**: ರೆಪೊಗಳಲ್ಲಿ ಪ್ರಮಾಣಪತ್ರಗಳು, API ಕೀಗಳು ಮತ್ತು ಟೋಕನ್‌ಗಳ ಸ್ವಯಂಚಾಲಿತ ಪತ್ತೆ
- **ಅವಲಂಬನೆ ಸ್ಕ್ಯಾನಿಂಗ್**: ಓಪನ್-ಸೋರ್ಸ್ ಅವಲಂಬನೆಗಳ ಮತ್ತು ಲೈಬ್ರರಿ ದುರ್ಬಲತೆ ಮೌಲ್ಯಮಾಪನ
- **CodeQL ವಿಶ್ಲೇಷಣೆ**: ಭದ್ರತಾ ದುರ್ಬಲತೆ ಮತ್ತು ಕೋಡಿಂಗ್ ಸಮಸ್ಯೆಗಳಿಗೆ ಸ್ಥಿರ ಕೋಡ್ ವಿಶ್ಲೇಷಣೆ
- **ಸರಬರಾಜು ಸರಣಿ ಅಂತರ್ನೋಟಗಳು**: ಅವಲಂಬನೆ ಆರೋಗ್ಯ ಮತ್ತು ಭದ್ರತಾ ಸ್ಥಿತಿಗೆ ದೃಷ್ಟಿಗೋಚರತೆ

**Azure DevOps & Azure Repos ಸಂಯೋಜನೆ:**
- ಮೈಕ್ರೋಸಾಫ್ಟ್ ಅಭಿವೃದ್ಧಿ ವೇದಿಕೆಗಳಾದಲ್ಲಿ ಸಮನ್ವಯ ಭದ್ರತಾ ಸ್ಕ್ಯಾನಿಂಗ್
- AI ಕಾರ್ಯಭಾರಗಳಿಗೆ ಅಜ್ಯೂರ್ ಪೈಪ್ಲೈನ್‌ಗಳಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತ ಭದ್ರತಾ ಪರಿಶೀಲನೆ
- ಸುರಕ್ಷಿತ AI ಘಟಕ ನಿಯೋಜನೆಗಾಗಿ ನೀತಿ ಜಾರಿಗೆ

**ಮೈಕ್ರೋಸಾಫ್ಟ್ ಆಂತರಿಕ ತಂತ್ರಗಳು:**
ಮೈಕ್ರೋಸಾಫ್ಟ್ ಎಲ್ಲಾ ಉತ್ಪನ್ನಗಳಲ್ಲಿ ವ್ಯಾಪಕ ಸರಬರಾಜು ಸರಣಿ ಭದ್ರತಾ ಕ್ರಮಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುತ್ತದೆ. [ಮೈಕ್ರೋಸಾಫ್ಟ್‌ನಲ್ಲಿ ಸಾಫ್ಟ್‌ವೇರ್ ಸರಬರಾಜು ಸರಣಿಯನ್ನು ಸುರಕ್ಷಿತಗೊಳಿಸುವ ಪ್ರಯಾಣ](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) ನಲ್ಲಿ ಪ್ರಮಾಣಿತ ವಿಧಾನಗಳನ್ನು ಕಲಿಯಿರಿ.


## ಮೂಲಭೂತ ಭದ್ರತಾ ಉತ್ತಮ ಪದ್ಧತಿಗಳು

- **AI-ನಿರ್ದಿಷ್ಟ ರಕ್ಷಣೆಗಳು**: [OWASP ಟಾಪ್ 10 ಫಾರ್ LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) ನಿರ್ವಹಣೆ
- **ಗೌಪ್ಯ ನಿರ್ವಹಣೆಯ ಭದ್ರತೆ**: ಟೋಕನ್‌ಗಳು, API ಕೀಲಿಗಳು ಮತ್ತು ಸಂವೇದನಾಶೀಲ ಸಂರಚನಾ ಡೇಟಾ ನೇಮಕ ವಾಲ್ಟ್‌ಗಳನ್ನು ಬಳಸಿ
- **ಎಂಡು-ಟು-ಎಂಡು ಸಂಕೇತನ**: ಎಲ್ಲ ಅಪ್ಲಿಕೇಶನ್ ಘಟಕಗಳು ಮತ್ತು ಡೇಟಾ ಹರಿವುಗಳಲ್ಲಿ ಭದ್ರ ಸಂವಹನ ಅನುಷ್ಠಾನ
- **ಒಳಿಸುವಿಕೆ ಮಾನ್ಯತೆ**: ಎಲ್ಲಾ ಬಳಕೆದಾರ ಒಳಿಸುವಿಕೆಗಳು, API ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು ಮತ್ತು ಡೇಟಾ ಮೂಲಗಳ ಕಠಿಣ ಪರಿಶೀಲನೆ

#### **ಮೂಲಸೌಕರ್ಯ ಬಲಪಡಿಸುವಿಕೆ**
- **ಬಹು-ಘಟಕ ಪ್ರಮಾಣೀಕರಣ**: ಎಲ್ಲಾ ನಿರ್ವಹಣಾ ಹಾಗೂ ಸೇವಾ ಖಾತೆಗಳಿಗೆ ಅನಿವಾರ್ಯ MFA
- **ಪ್ಯాచ్ ನಿರ್ವಹಣೆ**: ಆಪರೇಟಿಂಗ್ ಸಿಸ್ಟಮ್‌ಗಳು, ಫ್ರೇಮ್‌ವರ್ಕ್‌ಗಳು ಮತ್ತು ಅವಲಂಬನೆಗಳ ಪರಿಪಾಲನೆಗೆ ಸ್ವಯಂಚಾಲಿತ ಮತ್ತು ತ್ವರಿತ ಪ್ಯಾಚಿಂಗ್  
- **ಪರಿಚಯ ದಾತೃ ಸಂಯೋಜನೆ**: ಎಂಟರ್‌ಪ್ರೈಸ್ ಐಡೆಂಟಿಟಿ ದಾತೃಗಳ (Microsoft Entra ID, Active Directory) ಮೂಲಕ ಕೇಂದ್ರೀಕೃತರಲ್ಲಿ
- **ನೆಟ್ವರ್ಕ್ ವಿಭಜನೆ**: MCP ಘಟಕಗಳನ್ನು ತರ್ಕಸಂಗತ ಪ್ರತ್ಯೇಕಿಸುವ ಮೂಲಕ ಅಡ್ಡಸಂಚಾರ ಸಾಧ್ಯತೆಯನ್ನು ಕಡಿಮೆ ಮಾಡುವುದು
- **ಕನಿಷ್ಟ ಹಕ್ಕು ತತ್ತ್ವ**: ಎಲ್ಲಾ ಸಿಸ್ಟಮ್ ಘಟಕಗಳು ಮತ್ತು ಖಾತೆಗಳಿಗೆ ಕನಿಷ್ಟ ಅಗತ್ಯ ಅನಮತಿಗಳು ನೀಡುವುದು

#### **ಭದ್ರತಾ ಮೇಲ್ವಿಚಾರಣೆ ಮತ್ತು ಪತ್ತೆಮಾಡುವಿಕೆ**
- **ಸಮಗ್ರ ಲಾಗಿಂಗ್**: MCP ಕ್ಲೈಂಟ್-ಸರ್ವರ್ ಕ್ರಿಯೆಗಳ ಒಳಗೊಂಡಂತೆ AI ಅಪ್ಲಿಕೇಶನ್ ಚಟುವಟಿಕೆಗಳ ವಿವರವಾದ ಲಾಗಿಂಗ್
- **SIEM ಸಂಯೋಜನೆ**: ಅನಾಮಾಲಿ ಪತ್ತೆಗೆ ಕೇಂದ್ರೀಕೃತ ಭದ್ರತಾ ಮಾಹಿತಿ ಮತ್ತು ಘಟನೆ ನಿರ್ವಹಣೆ
- **ವರ್ತನಾ ವಿಶ್ಲೇಷಣೆ**: ವ್ಯವಸ್ಥೆ ಮತ್ತು ಬಳಕೆದಾರ ವರ್ತನೆಯಲ್ಲಿ ಅಸಾಮಾನ್ಯ ಮಾದರಿಗಳನ್ನು ಪತ್ತೆ ಮಾಡಲು AI ಚಾಲಿತ ಮೇಲ್ವಿಚಾರಣೆ
- **ಭೀಷಣ ಬುದ್ಧಿವಂತಿಕೆ**: ಬಾಹ್ಯ ಭೀಷಣ ಫೀಡ್‌ಗಳ ಸಂಯೋಜನೆ ಮತ್ತು ಒಪ್ಪಿಗೆಯ ಸೂಚಕ (IOCs)
- **ಆಘಾತ ಪ್ರತಿಕ್ರಿಯೆ**: ಭದ್ರತಾ ಘಟನೆ ಪತ್ತೆಮಾಡುವಿಕೆ, ಪ್ರತಿಕ್ರಿಯೆ ಮತ್ತು ಪುನಃಪ್ರಾಪ್ತಿ ಪ್ರಕ್ರಿಯೆಗಳ ವೆಲು ನಿರ್ದಿಷ್ಟತೆ

#### **ಜೀರು ಟ್ರಸ್ಟ್ ವಾಸ್ತುಶಿಲ್ಪ**
- **ಯಾವುದಕ್ಕೂ ನಂಬಿಕೆಬಿಡದಿರಿ, ಸದಾ ಪರಿಶೀಲಿಸಿರಿ**: ಬಳಕೆದಾರರು, ಸಾಧನಗಳು ಮತ್ತು ನೆಟ್ವರ್ಕ್ ಸಂಪರ್ಕಗಳ ನಿರಂತರ ಪರಿಶೀಲನೆ
- **ಮೈಕ್ರೋ ಸೆಗ್ಮೆಂಟೇಷನ್**: ವೈಯಕ್ತಿಕ ಕೆಲಸ ಮತ್ತು ಸೇವೆಗಳನ್ನು ವಿಭಜಿಸುವ ಸೂಕ್ಷ್ಮ ನೆಟ್ವರ್ಕ್ ನಿಯಂತ್ರಣಗಳು
- **ಐಡೆಂಟಿಟಿ-ಕೆಂದ್ರೀಕೃತ ಭದ್ರತೆ**: ನೆಟ್ವರ್ಕ್ ಸ್ಥಳಕ್ಕಿಂತ ಪರಿಶೀಲಿತ ಐಡೆಂಟಿಟಿಗಳ ಆಧಾರದ ಮೇಲೆ ಭದ್ರತಾ ನೀತಿಗಳು
- **ನಿರಂತರ ಅಪಾಯ ಮೌಲ್ಯಮಾಪನ**: ನಿನೂತ ಸನ್ನಿವೇಶ ಮತ್ತು ವರ್ತನೆಯ ಆಧಾರದ ಮೇಲೆ ಗುಣಾತ್ಮಕ ಭದ್ರತಾ ಸ್ಥಿತಿ ನಿರೀಕ್ಷಣೆ
- **ಶರತಿಯ ಇಳಿಚುಡಾಟ**: ಅಪಾಯ ಅಂಶಗಳು, ಸ್ಥಳ ಮತ್ತು ಸಾಧನ ನಂಬಿಕೆ ಆಧಾರದ ಮೇಲೆ ಪ್ರವೇಶ ನಿಯಂತ್ರಣಗಳು

### ಎಂಟರ್‌ಪ್ರೈಸ್ ಸಂಯೋಜನಾ ಮಾದರಿಗಳು

#### **ಮೈಕ್ರೋಸಾಫ್ಟ್ ಭದ್ರತಾ ಪರಿಸರ ಸಂಯೋಜನೆ**
- **ಮೈಕ್ರೋಸಾಫ್ಟ್ ಡಿಫೆಂಡರ್ ಫಾರ್ ಕ್ಲೌಡ್**: ಸಮಗ್ರ ಕ್ಲೌಡ್ ಭದ್ರತಾ ಸ್ಥಿತಿ ನಿರ್ವಹಣೆ
- **ಅಜ್ಯೂರ್ ಸೆಂಟಿನೆಲ್**: AI ಕಾರ್ಯಭಾರದ ರಕ್ಷಣೆಗಾಗಿ ಕ್ಲೌಡ್-ಸ್ವದೇಶಿ SIEM ಮತ್ತು SOAR ಸಾಮರ್ಥ್ಯಗಳು
- **ಮೈಕ್ರೋಸಾಫ್ಟ್ ಎಂಟ್ರಾ ಐಡಿ**: ಶರತಿಯ ಇಳಿಚುಡಾಟ ನೀತಿಗಳೊಂದಿಗೆ ಉದ್ಯಮ ನಿಯಂತ್ರಣ ಮತ್ತು ಪ್ರವೇಶ ನಿರ್ವಹಣೆ
- **ಅಜ್ಯೂರ್ ಕೀ ವಾಲ್ಟ್**: ದ್ರವ್ಯರಾಶಿ ನಿರ್ವಹಣೆಗೆ ಹಾರ್ಡ್‌ವೇರ್ ಭದ್ರತಾ ಘಟಕ (HSM) ಬೆಂಬಲದ ಕೇಂದ್ರೀಕೃತ ವ್ಯವಸ್ಥೆ
- **ಮೈಕ್ರೋಸಾಫ್ಟ್ ಪರ್ವ್ಯೂ**: AI ಡೇಟಾ ಮೂಲಗಳು ಮತ್ತು ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗಾಗಿ ಡೇಟಾ ಆಡಳಿತ ಮತ್ತು ಅನುಕೂಲ

#### **ಸಮ್ಮಿಖಿಲತ ಮತ್ತು ನಿರ್ವಹಣೆ**
- **ನಿಯಮಾನುಗುಣ ಹೊಂದಿಕೆ**: MCP ಅನುಷ್ಠಾನಗಳು ಕೈಗಾರಿಕಾ ಗುರ್ತಿನಿಗನುಗುಣ ಸಮ್ಮಿಖಿಲತೆಯ ಅಗತ್ಯಗಳನ್ನು (GDPR, HIPAA, SOC 2) ಪೂರೈಸುವಂತೆ ನೋಡಿಕೊಳ್ಳಬೇಕು

- **ಡೇಟಾ ವರ್ಗೀಕರಣ**: ಏಐ ವ್ಯವಸ್ಥೆಗಳ ಮೂಲಕ ಪ್ರಕ್ರಿಯೆಗೊಳಿಸಿದ ಸಂವೇದಿ ಡೇಟಾದ ಸರಿಯಾದ ವರ್ಗೀಕರಣ ಮತ್ತು ನಿರ್ವಹಣೆ
- **ಆಡಿಟ್ ಟ್ರೇಲ್ಸ್**: ನಿಯಂತ್ರಣ ಅನುಕೂಲತೆ ಮತ್ತು ಫೊರೆನ್ಸಿಕ್ ತನಿಖೆಗಾಗಿ ಸಂಪೂರ್ಣ ಲಾಗಿಂಗ್
- **ಗೋಪ್ಯತೆ ನಿಯಂತ್ರಣಗಳು**: ಏಐ ವ್ಯವಸ್ಥಾ वास्तುಶಿಲ್ಪದಲ್ಲಿ ಗೌಪ್ಯತೆ-ಡಿಜೈನ್ ತತ್ವಗಳನ್ನು ಜಾರಿಗೊಳ್ಳುವಿಕೆ
- **ಬದಲಾವಣೆ ನಿರ್ವಹಣೆ**: ಏಐ ವ್ಯವಸ್ಥೆ ಬದಲಾವಣೆಗಳ ಭದ್ರತಾ ವಿಮರ್ಶೆಗಾಗಿ ಅಧಿಕೃತ ಪ್ರಕ್ರಿಯೆಗಳು

ಈ ಆಧಾರಭೂತ ಅಭ್ಯಾಸಗಳು ಆರ್‌ಎಂಬಸ್ಟ್ ಭದ್ರತಾ ಮಾದರಿಯನ್ನು ಸೃಷ್ಟಿಸಿ MCP-ನಿರ್ದಿಷ್ಟ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳ ಪರಿಣಾಮಕಾರಿತ್ವವನ್ನು ಹೆಚ್ಚಿಸುತ್ತವೆ ಮತ್ತು ಏಐ ಚಾಲಿತ ಅನ್ವಯಿಕೆಗಳಿಗೆ ಸಂಪೂರ್ಣ ರಕ್ಷಣೆ ಒದಗಿಸುತ್ತವೆ.

## ಪ್ರಮುಖ ಭದ್ರತಾ ಒಳಹೊರತೆಗಳು

- **ಪೊರೆಯಾಗಿದೆ ಭದ್ರತಾ ದೃಷ್ಠಿಕೋನ**: ಆಧಾರಭೂತ ಭದ್ರತಾ ಅಭ್ಯಾಸಗಳನ್ನು (ಭದ್ರತಾ ಕೋಡಿಂಗ್, ಕನಿಷ್ಠ привಿಲೇಜ್, ಸರಬರಾಜು ಸರಪಳಿ ಪರಿಶೀಲನೆ, ನಿರಂತರ ಮಾನಿಟರಿಂಗ್) ಏಐ-ನಿರ್ದಿಷ್ಟ ನಿಯಂತ್ರಣಗಳೊಡನೆ ಸಂಯೋಜಿಸಿ ಸಂಪೂರ್ಣ ರಕ್ಷಣೆಗಾಗಿ

- **ಎಐ-ನಿರ್ದಿಷ್ಟ ಬೆದರಿಕೆ ದೃಶ್ಯಪಟ**: MCP ವ್ಯವಸ್ಥೆಗಳು ವಿಶಿಷ್ಟ ಅಪಾಯಗಳನ್ನು ಎದುರಿಸುತ್ತವೆ, ಇವುಗಳಲ್ಲಿ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್, ಟೂಲ್ ವಿಷಪ್ರಯೋಗ, ಸೆಷನ್ ಹೈಜಾಕಿಂಗ್, ಗೊಂದಲಕ್ಕೊಳಗಿನ ಡೆಪ್ಯೂ ಸಮಸ್ಯೆಗಳು, ಟೋಕನ್ ಪಾಸ್ತ್ರೂ ವನ್ ದುರ್ಬಲತೆಗಳು ಮತ್ತು ಅಧಿಕೃತ ಅನುಮತಿಗಳು ಸೇರಿದ್ದು, ವಿಶೇಷ ಪರಿಹಾರಗಳನ್ನು ಅಗತ್ಯ ಪಡಿಸುತ್ತವೆ

- **ಪರಿಚಯ ಮತ್ತು ಅಧಿಕೃತತೆ ಶ್ರೇಷ್ಠತೆ**: ಹೊರಗಿನ ಗುಣಪತ್ರ ಒದಗಿಸುವವರ (Microsoft Entra ID) ಬಳಕೆ ಮೂಲಕ ಬಲವಾದ ಪ್ರಾಮಾಣೀಕರಣ, ಸರಿಯಾದ ಟೋಕನ್ ಪರಿಶೀಲನೆ ಜಾರಿಗೊಳಿಸಿ, ನಿಮ್ಮ MCP ಸರ್ವರ್‌ಗೆ ಸ್ಪಷ್ಟವಾಗಿ ಮಾಡಲಾಗದ ಟೋಕನ್ಗಳನ್ನು ಎಂದಿಗೂ ಸ್ವೀಕರಿಸಬೇಡಿ

- **ಎಐ ದಾಳಿಗಳ ತಡೆಗೆ**: Microsoft Prompt Shields ಮತ್ತು Azure Content Safety ಬಳಸಿಕೊಂಡು ಪರೋಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ಮತ್ತು ಟೂಲ್ ವಿಷಪ್ರಯೋಗ ದಾಳಿಗಳನ್ನು ತಡೆಹಿಡಿಯಿರಿ, ಟೂಲ್ ಮೆಟಾಡೇಟಾವನ್ನು ಪರಿಶೀಲಿಸಿ ಮತ್ತು ಚಲಿಸುತ್ತದೆ ಬದಲಾವಣೆಯ ಮೇಲೆ ನಿಗ vigilanciaೆ ಇಡಿ

- **ಸೆಷನ್ ಮತ್ತು ಸಾರಿನ ಭದ್ರತೆ**: ಬಳಕೆದಾರರ ಗುರುತಿಗೆ ಬಿಂದಿಸುವ ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಭದ್ರ, ಅನಿಯೋಜ್ಯ ಸೆಷನ್ ಐಡಿಗಳನ್ನು ಬಳಸಿ, ಸೂಕ್ತ ಸೆಷನ್ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆಯನ್ನು ಜಾರಿಗೊಳಿಸಿ ಮತ್ತು ಪ್ರಾಮಾಣೀಕರಣಕ್ಕಾಗಿ ಸೆಷನ್‌ಗಳನ್ನು ಎಂದಿಗೂ ಬಳಸಬೇಡಿ

- **OAuth ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು**: ಪ್ರತ್ಯೇಕ ಬಳಕೆದಾರ ಅನುಮತಿಯನ್ನು ಬಳಸಿಕೊಂಡು ಗೊಂದಲಕ್ಕೊಳಗಿನ ಡೆಪ್ಯೂ ದಾಳಿಗಳನ್ನು ತಡೆಯಿರಿ, PKCE ಇರುವ ಸರಿಯಾದ OAuth 2.1 ಜಾರಿಗೊಳ್ಳುವಿಕೆ ಮತ್ತು ಕಠಿಣ ರೀಡೈರೆಕ್ಟ್ URI ಪರಿಶೀಲನೆ

- **ಟೋಕನ್ ಭದ್ರತಾ ತತ್ವಗಳು**: ಟೋಕನ್ ಪಾಸ್ತ್ರೂ ವಿಜೃಂಭಣೆಯ ಮೌಲ್ಯನಾಶಕಗಳನ್ನು ತಪ್ಪಿಸಿ, ಟೋಕನ್ ಆಗೋಡಿಯೆನ್ ಮೇಲೆ ಪ್ರಮಾಣೀಕರಿಸಿ, ಸುರಕ್ಷಿತವಾಗಿ ತಿರುಳುವ ಹಳೆ-ಕಾಲ ತೋಕೆನ್‌ಗಳನ್ನು ಜಾರಿಗೊಳಿಸಿ ಮತ್ತು ಸ್ಪಷ್ಟ ವಿಶ್ವಾಸ ಗಡಿಗಳನ್ನು ನಿರ್ವಹಿಸಿ

- **ಸಂಪೂರ್ಣ ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತೆ**: ಎಲ್ಲಾ ಏಐ ಪರಿಸರ ಘಟಕಗಳನ್ನು (ಮಾದರಿಗಳು, ಎಂಬೆಡಿಂಗ್ಸ್, ಪ್ರಾರಂಭ ಪ್ರದಾಯಕರು, ಬಾಹ್ಯ APIಗಳು) ಪರಂಪರাগত ಸಾಫ್ಟ್‌ವೇರ್ ಅವಲಂಬನೆಗಳಷ್ಟು ಭದ್ರತೆ ದೃಢತೆಯೊಂದಿಗೆ ಚಿಂತಿಸಬೇಕು

- **ನಿರಂತರ ಅಭಿವೃದ್ಧಿ**: ವೇಗವಾಗಿ ಬದಲಾಗುತ್ತಿರುವ MCP ವಿಶೇಷಣಗಳೊಂದಿಗೆ ನವೀನತೆಯನ್ನು ಹಿಡಿದಿಡಿ, ಭದ್ರತಾ ಸಮುದಾಯ ನಿಯಮಗಳಿಗೆ ಕೊಡುಗೆ ನೀಡಿ ಮತ್ತು ಪ್ರೋಟೋಕಾಲ್ ಪ್ರಗತಿಯಾಗಿ ಭದ್ರತಾ ಸ್ಥಿತಿಗಳನ್ನು ಹೊಂದಿರಿ

- **Microsoft ಭದ್ರತಾ ಸಂಯೋಜನೆ**: MCP ಜಾರಿಗೆ ಹೆಚ್ಚುವರಿ ರಕ್ಷಣೆಗೆ Microsoft-ನ ಸಂಪೂರ್ಣ ಭದ್ರತಾ ಪರಿಸರ (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) ಅಳವಡಿಸಿ
