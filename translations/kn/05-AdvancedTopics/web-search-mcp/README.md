# ಪಾಠ: ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್ ನಿರ್ಮಾಣ

ಈ ಅಧ್ಯಾಯವು ಬಾಹ್ಯ APIs ಜೊತೆಗೆ ಏಕೀಕೃತವಾಗಿರುವ, 다양한 ದತ್ತಾಂಶ ಪ್ರಕಾರಗಳನ್ನು ನಿರ್ವಹಿಸುವ, ದೋಷಗಳನ್ನು ನಿಭಾಯಿಸುವ ಮತ್ತು ಬಹುತೇಕ ಸಾಧನಗಳನ್ನು ಸಂಯೋಜಿಸುವ ನೈಜ-ಜಗತ್ತಿನ AI ಏಜೆಂಟ್ ಅನ್ನು ನಿರ್ಮಿಸುವ ವಿಧಾನವನ್ನು ತೋರಿಸುತ್ತದೆ—ಇವು ಎಲ್ಲವೂ ಉತ್ಪಾದನಾ-ಸಿದ್ಧ ಸ್ವರೂಪದಲ್ಲಿ. ನೀವು ನೋಡಬಹುದು:

- **ಪ್ರಾಮಾಣೀಕರಣ ಅಗತ್ಯವಿರುವ ಬಾಹ್ಯ API ಗಳೊಂದಿಗೆ ಏಕೀಕರಣ**
- **ಬಹುಳಿತ ಬಿಂದುಗಳಿಂದ ಬರುವ ವಿವಿಧ ದತ್ತಾಂಶ ಪ್ರಕಾರಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು**
- **ದೃಢವಾದ ದೋಷ ನಿರ್ವಹಣೆ ಮತ್ತು ಲಾಗಿಂಗ್ ತಂತ್ರಗಳು**
- **ಒಂದು ಸರ್ವರ್‌ನಲ್ಲಿ ಬಹು-ಸಾಧನ ಸಂಯೋಜನೆ**

ಅಂತ್ಯದವರೆಗೆ, ಉನ್ನತ AI ಮತ್ತು LLM-ಪ್ರೇರಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಅಗತ್ಯವಾದ ಮಾದರಿಗಳು ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ನೀವು ಪ್ರಾಯೋಗಿಕ ಅನುಭವವನ್ನು ಪಡೆಯಿರಿ.

## ಪರಿಚಯ

ಈ ಪಾಠದಲ್ಲಿ, ನೀವು SerpAPI ಬಳಸಿ ನೈಜ-ಕಾಲಿನ ವೆಬ್ ದತ್ತಾಂಶದಿಂದ LLM ಸಮರ್ಥತೆಗಳನ್ನು ವಿಸ್ತರಿಸುವ ಉನ್ನತ MCP ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಅನ್ನು ನಿರ್ಮಿಸುವುದನ್ನು ಕಲಿಯೋಿರಿ. ಇದು ವೆಬ್‌ನಿಂದ ಅಪ್ಡೇಟ್ ಆಗಿರುವ ಮಾಹಿತಿಯನ್ನು ಪಡೆದುಕೊಳ್ಳಬಹುದಾದ ಡೈನಾಮಿಕ್ AI ಏಜೆಂಟ್‌ಗಳನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸುವ ಪ್ರಮುಖ ಕೌಶಲ್ಯವಾಗಿದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ ನೀವು ಈ ಕೆಳಗಿನ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಲು সক্ষমರಾಗಿರುವಿರಿ:

- ಹೊರಗಿನ APIs (ಉದಾ: SerpAPI) ಅನ್ನು ಸುರಕ್ಷಿತವಾಗಿ MCP ಸರ್ವರ್‌ಗೆ ಏಕಿ­ಕೃತಗೊಳಿಸುವುದು
- ವೆಬ್, ಸುದ್ದಿ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು Q&A ಗಳಿಗಾಗಿ ಅನೇಕ ಸಾಧನಗಳನ್ನು ಕಾರ್ಯಗತರಿಸುವುದು
- LLM ಉಪಯೋಗಕ್ಕಾಗಿ ರೂಪುಗೊಂಡ ದತ್ತಾಂಶವನ್ನು ಪಾರ್ಸ್ ಮತ್ತು ರೂಪಾಂತರಿಸುವುದು
- ದೋಷಗಳನ್ನು ನಿಭಾಯಿಸುವುದು ಮತ್ತು API ದರ ಮಿತಿ (rate limit)ಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿರ್ವಹಿಸುವುದು
- ಸ್ವಯಂಚಾಲಿತ ಮತ್ತು ಇಂಟರೆಕ್ಟಿವ್ MCP ಕ್ಲೈಂಟ್‌ಗಳ ನಿರ್ಮಾಣ ಮತ್ತು ಪರೀಕ್ಷೆ

## ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್

ಈ ವಿಭಾಗವು ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್‌ನ ವಾಸ್ತುಶಿಲ್ಪ ಮತ್ತು ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ. ನಿಮಗೊಂದು ಒಟ್ಟಾರೆ FastMCP ಮತ್ತು SerpAPI ಬಳಸಿ ನೈಜ-ಕಾಲಿನ ವೆಬ್ ದತ್ತಾಂಶದಿಂದ LLM ಸಾಮರ್ಥ್ಯಗಳನ್ನು ವಿಸ್ತರಿಸುವ ವಿಧಾನವನ್ನು ಕಾಣಬಹುದು.

### ಅವಲೋಕನ

ಈ ಅನುಷ್ಠಾನದಲ್ಲಿ MCP-ನ ಸಾಮರ್ಥ್ಯವನ್ನು ಸುರಕ್ಷಿತವಾಗಿ ಮತ್ತು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಹಲವಾರು ಬಾಹ್ಯ API ಚಾಲಿತ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸುವುದಕ್ಕಾಗಿ ನಾಲ್ಕು ಸಾಧನಗಳನ್ನು ಒಳಗೊಂಡಿದೆ:

- **general_search**: ವ್ಯಾಪಕ ವೆಬ್ ಫಲಿತಾಂಶಗಳಿಗೆ
- **news_search**: ಇತ್ತೀಚಿನ ಶೀರ್ಷಿಕೆಗಳಿಗೆ
- **product_search**: ಇ-ವಾಣಿಜ್ಯ ದತ್ತಾಂಶಕ್ಕೆ
- **qna**: ಪ್ರಶ್ನೆ-ಮತ್ತು-ಉತ್ತರ ವಿವರಗಳಿಗೆ

### ವೈಶಿಷ್ಟ್ಯಗಳು
- **ಕೋಡ್ ಉದಾಹರಣೆಗಳು**: ಪೈಥಾನ್‌ಗಾಗಿ ಭಾಷಾ-विशೇಷ-ಕೋಡ್ ಬ್ಲಾಕ್ಗಳನ್ನು ಹೊಂದಿದೆ (ಮತ್ತು ಇತರ ಭಾಷೆಗಳಿಗೆ ಸುಲಭವಾಗಿ ವಿಸ್ತರಿಸಬಹುದಾಗಿದೆ) ಸ್ಪಷ್ಟತೆಗಾಗಿ ಕೋಡ್ ಪಿವೋಟ್‌ಗಳನ್ನು ಬಳಸಿ

### ಪೈಥಾನ್

```python
# ಸಾಮಾನ್ಯ_ಹುಡುಕು ಸಾಧನದ ಉದಾಹರಣೆಯ ಬಳಕೆ
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

ಕ್ಲೈಂಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡುವ ಮುನ್ನ, ಸರ್ವರ್ ಏನು ಮಾಡುತ್ತದೆ ಎಂಬುದನ್ನು ತಿಳಿದುಕೊಳ್ಳುವುದು ಸಹಾಯಕರಾಗಿರುತ್ತದೆ. [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ಫೈಲ್ MCP ಸರ್ವರ್ ಅನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುತ್ತದೆ, SerpAPI ಜೋಡಣೆ ಮೂಲಕ ವೆಬ್, ಸುದ್ದಿ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು Q&A ಸಾಧನಗಳನ್ನು ಪ್ರಕಟಿಸುತ್ತದೆ. ಇದು ಬಂದ್ ಬೇಟೆಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು, API ಕರೆಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು, ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪಾರ್ಸ್ ಮಾಡುವುದು ಮತ್ತು ಕ್ಲೈಂಟ್‌ಗೆ ರೂಪುಗೊಂಡ ಫಲಿತಾಂಶಗಳನ್ನು ಮರಳಿಸುವುದನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ.

ನೀವು ಸಂಪೂರ್ಣ ಅನುಷ್ಠಾನವನ್ನು [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ನಲ್ಲಿ ಪರಿಶೀಲಿಸಬಹುದು.

ಇಲ್ಲಿ ಸರ್ವರ್ ಸಾಧನಗಳನ್ನು ಹೇಗೆ นิShareಐದ್ದಾಗಿರುವುದು ಮತ್ತು ನೋಂದಣಿ ಮಾಡುತ್ತದೆ ಎಂಬ ಸಣ್ಣ ಉದಾಹರಣೆ ಇದೆ:

### ಪೈಥಾನ್ ಸರ್ವರ್

```python
# server.py (ಕExcerpt)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...ಕಾರ್ಯಾನ್ವಯಿಕೆ...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **ಬಾಹ್ಯ API ಏಕೀಕರಣ**: API ಕೀಲಿಗಳ ಸುರಕ್ಷಿತ ನಿರ್ವಹಣೆ ಮತ್ತು ಬಾಹ್ಯ ವಿನಂತಿಗಳನ್ನು ತೋರಿಸುತ್ತದೆ
- **ರೂಪಮಾನಿತ ದತ್ತಾಂಶ ಪಾರ್ಸಿಂಗ್**: API ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು LLM ಗೆ ಸ್ನೇಹಪೂರಿತ ಸ್ವರೂಪಗಳಿಗೆ ಪರಿವರ್ತಿಸುವುದನ್ನು ತೋರಿಸುತ್ತದೆ
- **ದೋಷ ನಿರ್ವಹಣೆ**: ಸೂಕ್ತ ಲಾಗಿಂಗಿನೊಂದಿಗೆ ದೃಢವಾದ ದೋಷ ನಿರ್ವಹಣೆ
- **ಇಂಟರೆಕ್ಟಿವ್ ಕ್ಲೈಂಟ್**: ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷೆಗಳು ಮತ್ತು ಪರೀಕ್ಷೆಗಾಗಿ ಇಂಟರೆಕ್ಟಿವ್ ಮೋಡ್ ಎರಡನ್ನೂ ಒಳಗೊಂಡಿದೆ
- **ಸಂದರ್ಭ ನಿರ್ವಹಣೆ**: ಲಾಗಿಂಗ್ ಮತ್ತು ವಿನಂತಿಗಳ ಟ್ರ್ಯಾಕಿಂಗ್‌ಗಾಗಿ MCP Context ನ್ನು ಉಪಯೋಗಿಸುತ್ತದೆ

## ಪೂರ್ವಾವಶ್ಯಕತೆಗಳು

ಆರಂಭಿಸುವ ಮೊದಲು, ನಿಮ್ಮ ವಾತಾವರಣವು ಸರಿಯಾದ ರೀತಿಯಲ್ಲಿ ಸೆಟ್ ಆಗಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಈ ಕ್ರಮಗಳು ಎಲ್ಲಾ ಅವಲಂಬನಗಳನ್ನು ಸ್ಥಾಪಿಸುವುದಕ್ಕೆ ಮತ್ತು ನಿಮ್ಮ API ಕೀಲಿಗಳನ್ನು ಸರಿಯಾಗಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಲು ಸಹಾಯ ಮಾಡುವವು.

- ಪೈಥಾನ್ 3.8 ಅಥವಾ ಹೆಚ್ಚಿನ ಸಂಸ್ಕರಣೆ
- SerpAPI API ಕೀಲಿ ([SerpAPI](https://serpapi.com/) ನಲ್ಲಿ ಸೈನ್ ಅಪ್ ಮಾಡಿ - ಉಚಿತ ಸ್ತರ ಲಭ್ಯವಿದೆ)

## ಸ್ಥಾಪನೆ

ಪ್ರಾರಂಭಿಸಲು, ನಿಮ್ಮ ವಾತಾವರಣವನ್ನು ಸೆಟ್ ಮಾಡುವ ಕ್ರಮಗಳು ಇಂತಿವೆ:

1. ಅವಲಂಬನೆಗಳನ್ನು uv (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ) ಅಥವಾ pip ಬಳಸಿ ಸ್ಥಾಪಿಸಿ:

```bash
# uv ಬಳಕೆ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)
uv pip install -r requirements.txt

# ಪಿಪ್ ಬಳಕೆ
pip install -r requirements.txt
```

2. ಪ್ರಾಜೆಕ್ಟ್ ರೂಟ್ ನಲ್ಲಿ ನಿಮ್ಮ SerpAPI ಕೀಲಿಯನ್ನು ಹೊಂದಿರುವ `.env` ಫೈಲ್ ರಚಿಸಿ:

```
SERPAPI_KEY=your_serpapi_key_here
```

## ಬಳಕೆ

ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್ ಮುಖ್ಯ ಘಟಕವಾಗಿದೆ, ಅದು SerpAPI ಜೋಡಣೆ ಹೊಂದುವ ಮೂಲಕ ವೆಬ್, ಸುದ್ದಿ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು Q&A ಸಾಧನಗಳನ್ನು ಪ್ರಕಟಿಸುತ್ತದೆ. ಇದು ಬಂದ್ ಬೇಟೆಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, API ಕರೆಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪಾರ್ಸ್ ಮಾಡುತ್ತದೆ ಮತ್ತು ರೂಪುಗೊಂಡ ಫಲಿತಾಂಶಗಳನ್ನು ಕ್ಲೈಂಟ್‌ಗೆ ನೀಡುತ್ತದೆ.

ನೀವು ಸಂಪೂರ್ಣ ಅನುಷ್ಠಾನವನ್ನು [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ನಲ್ಲಿ ಪರಿಶೀಲಿಸಬಹುದು.

### ಸರ್ವರ್ ಚಾಲನೆ

MCP ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಲು ಈ ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ಬಳಸಿ:

```bash
python server.py
```

ಸರ್ವರ್ ಯಾವುದೇ stdio ಆಧಾರಿತ MCP ಸರ್ವರ್ ಆಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವುದರಿಂದ ಕ್ಲೈಂಟ್ ನೇರವಾಗಿ ಸಂಪರ್ಕಿಸಬಹುದು.

### ಕ್ಲೈಂಟ್ ಮೋಡ್‌ಗಳು

ಕ್ಲೈಂಟ್ (`client.py`) MCP ಸರ್ವರ್‌ನೊಂದಿಗೆ ಸಂವಾದಿಸಲು ಎರಡು ಮೋಡ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ:

- **ಸಾಧಾರಣ ಮೋಡ್**: ಎಲ್ಲಾ ಸಾಧನಗಳನ್ನು ವ್ಯಾಯಾಮವಾಗಿ ಪರೀಕ್ಷಿಸುವ ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷೆಗಳನ್ನು ನಡೆಸುತ್ತದೆ ಮತ್ತು ಅವರ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ. ಇದು ಸರ್ವರ್ ಮತ್ತು ಸಾಧನಗಳು ನಿರೀಕ್ಷೆಯಂತೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿರುವುದನ್ನು ಶೀಘ್ರವಾಗಿ ಪರಿಶೀಲಿಸಲು ಉಪಯುಕ್ತವಾಗಿದೆ.
- **ಇಂಟರೆಕ್ಟಿವ್ ಮೋಡ್**: ನೀವು ಕೈಯಿಂದ ಸಾಧನಗಳನ್ನು ಆಯ್ಕೆಮಾಡಿ ಕರೆಮಾಡಲು, ಕಸ್ಟಮ್ ಕ್ವೆರಿ ನಮೂದಿಸಲು ಮತ್ತು ಫಲಿತಾಂಶಗಳನ್ನು ನೈಜ ಕಾಲದಲ್ಲಿ ನೋಡಲು ಮೆನು ಚಾಲಿತ ಇಂಟರ್ಫೇಸ್ ಪ್ರಾರಂಭಿಸುತ್ತದೆ. ಇದು ಸರ್ವರ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಅನ್ವೇಷಿಸಲು ಮತ್ತು ವಿಭಿನ್ನ ಇನ್‌ಪುಟ್‌ಗಳೊಂದಿಗೆ ಪ್ರಯೋಗಿಸಲು ಸೂಕ್ತವಾಗಿದೆ.

ನೀವು ಸಂಪೂರ್ಣ ಅನುಷ್ಠಾನವನ್ನು [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) ನಲ್ಲಿ ಪರಿಶೀಲಿಸಬಹುದು.

### ಬೇರೆಯ ವಿಧಾನಗಳಿಂದ ಪರೀಕ್ಷೆ

ಸರ್ವರ್ ಒದಗಿಸುವ ಸಾಧನಗಳೊಂದಿಗೆ ಪರೀಕ್ಷೆ ಮತ್ತು ಸಂವಾದ ಮಾಡಲು ಹಲವಾರು ಮಾರ್ಗಗಳಿವೆ, ನಿಮ್ಮ ಅಗತ್ಯಗಳು ಮತ್ತು ಕಾರ್ಯವಿಧಾನದ ಮೇಲೆ ಅವಲಂಬಿತವಾಗಿದೆ.

#### MCP ಪೈಥಾನ್ SDK ಬಳಸಿ ಕಸ್ಟಮ್ ಪರೀಕ್ಷಾ ಸ್ಕ್ರಿಪ್ಟುಗಳನ್ನು ಬರೆಯುವುದು
ನೀವು MCP ಪೈಥಾನ್ SDK ಬಳಸಿ ನಿಮ್ಮ ಸ್ವಂತ ಪರೀಕ್ಷಾ ಸ್ಕ್ರಿಪ್ಟುಗಳನ್ನು ಕೂಡ ನಿರ್ಮಿಸಬಹುದು:

# [Python](#tab/python-sdk)

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def test_custom_query():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            # ನಿಮ್ಮ ಕಸ್ಟಮ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ ಉಪಕರಣಗಳನ್ನು ಕರೆಿಸಿ
            result = await session.call_tool("general_search", 
                                           arguments={"query": "your custom query"})
            # ಫಲಿತಾಂಶವನ್ನು ಪ್ರಕ್ರಿಯೆಗೊಳಿಸಿ
```

---


ಈ ಸಂಧರ್ಭದಲ್ಲಿ, "ಪರೀಕ್ಷಾ ಸ್ಕ್ರಿಪ್ಟ್" ಅಂದರೆ ನೀವು ಬರೆಯುವ ಒಂದು ಕಸ್ಟಮ್ ಪೈಥಾನ್ ಪ್ರೋಗ್ರಾಂ, ಅದು MCP ಸರ್ವರ್‌ಗೆ ಕ್ಲೈಂಟ್ ಆಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ. ಇದು ಆಗಾನಾಗದ ಯುನಿಟ್ ಪಾರದರ್ಶಕ ಪರೀಕ್ಷೆಯಾಗಿಲ್ಲ; ಬದಲಿಗೆ, ಈ ಸ್ಕ್ರಿಪ್ಟ್ ನಿಮ್ಮ ಆಯ್ಕೆ ಪ್ರಕಾರ ಯಾವುದೇ ಸಾಧನಗಳನ್ನು ಕರೆಮಾಡಿ ಮತ್ತು ಫಲಿತಾಂಶಗಳನ್ನು ಪರಿಶೀಲಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಈ ವಿಧಾನ ಉಪಯುಕ್ತವಾಗಿದೆ:
- ಸಾಧನ ಕರೆಗಳನ್ನು ಪೂರ್ವಸಿದ್ಧಪಡಿಸುವುದು ಮತ್ತು ಪ್ರಯೋಗಿಸುವುದು
- ಸರ್ವರ್ ವಿಭಿನ್ನ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಹೇಗೆ ಪ್ರತಿಕ್ರಿಯಿಸುತ್ತಿದೆ ಎಂದು ಪರಿಶೀಲಿಸುವುದು
- ಪುನರಾವರ್ತಿತ ಸಾಧನ ಆಮೋದ್ಯವನಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸುವುದು
- MCP ಸರ್ವರ್ ಮೇಲ್ಭಾಗದಲ್ಲಿ ನಿಮ್ಮ ಸ್ವಂತ ಕಾರ್ಯನಿರ್ವಾಹಕಗಳು ಅಥವಾ ಏಕೀಕರಣಗಳನ್ನು ನಿರ್ಮಿಸುವುದು

ನೀವು ಹೊಸ ಕ್ವೇರಿಗಳನ್ನು ಶೀಘ್ರವಾಗಿ ಪ್ರಯತ್ನಿಸಲು, ಸಾಧನ ವರ್ತನೆಗಳನ್ನು ಡಿಬಗ್ಗ ಮಾಡಲು ಅಥವಾ ಇನ್ನಷ್ಟು ಉನ್ನತ ಸ್ವಯಂಚಾಲಿತತನಕ್ಕೆ ಪ್ರಾರಂಭಿಕವಾಗಿ ಬಳಸಬಹುದು. ಕೆಳಗಿನ ಉದಾಹರಣೆ MCP ಪೈಥಾನ್ SDK ಬಳಸಿ ಈ ರೀತಿಯ ಸ್ಕ್ರಿಪ್ಟ್ ಹೇಗೆ ರಚಿಸುವುದನ್ನು ತೋರಿಸುತ್ತದೆ:

## ಸಾಧನ ವಿವರಣೆಗಳು

ಸರ್ವರ್ ಒದಗಿಸುವ ಕೆಳಗಿನ ಸಾಧನಗಳನ್ನು ಬಳಸಿ ನೀವು ವಿಭಿನ್ನ ಬಗೆಯ ಹುಡುಕಾಟಗಳು ಮತ್ತು ಪ್ರಶ್ನೆಗಳಿಗೆ ಕಾರ್ಯಗತಗೊಳಿಸಬಹುದು. ಪ್ರತಿ ಸಾಧನವು ಅದರ ಪ್ಯಾರಾಮೀಟರ್ಸ್ ಮತ್ತು ಉದಾಹರಣೆಯ ಬಳಕೆಯೊಂದಿಗೆ ಕೆಳಗೆ ವಿವರಿಸಲಾಗಿದೆ.


ಈ ವಿಭಾಗದಲ್ಲಿ ಪ್ರತಿ ಲಭ್ಯವಿರುವ ಸಾಧನ ಮತ್ತು ಅದರ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ವಿವರ ನೀಡಲಾಗಿದೆ.

## ಈ ಪಾಠದಲ್ಲಿ ಉನ್ನತ ಕಲ್ಪನೆಗಳು

ನೀವು ನಿರ್ಮಿಸಲು ಪ್ರಾರಂಭಿಸುವ ಮುನ್ನ, ಈ ಅಧ್ಯಾಯದಲ್ಲಿ ಬಾರಿ ಕಾಣಿಸಿಕೊಳ್ಳುವ ಕೆಲವು ಪ್ರಮುಖ ಉನ್ನತ ಕಲ್ಪನೆಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳುವುದು ಮುಖ್ಯ. ನಿಮಗೆ ಅವು ಹೊಸದಾಗಿದ್ದರೂ ಸಹ ಅವುಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಈ ಭಾಗ ನೆರವಾಗಲಿದೆ:

- **ಬಹುಸಾಧನ ಸಂಯೋಜನೆ**: ಇದು ಒಂದೇ MCP ಸರ್ವರ್‌ನಲ್ಲಿ ವಿವಿಧ ಸಾಧನಗಳನ್ನು (ಉದಾ: ವೆಬ್ ಹುಡುಕಾಟ, ಸುದ್ದಿ ಹುಡುಕಾಟ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು Q&A) ನಿರ್ವಹಿಸುವುದಾಗಿದೆ. ಇದು ನಿಮ್ಮ ಸರ್ವರ್ ವಿವಿಧ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತಿರುವಂತೆ ಮಾಡುತ್ತದೆ, ಒಂದೇ ಕಾರ್ಯವಲ್ಲ.
- **API ದರ ಮಿತಿ (Rate Limit) ನಿರ್ವಹಣೆ**: ಅನೇಕ ಬಾಹ್ಯ API ಗಳಿಗೆ ನಿರ್ದಿಷ್ಟ ಸಮಯದಲ್ಲಿ ನೀವು ಮಾಡಿದ ವಿನಂತಿಗಳ ಸಂಖ್ಯೆ ಮೇಲೆ ಮಿತಿ ಇರುತ್ತದೆ. ಉತ್ತಮ ಕೋಡ್ ಈ ಮಿತಿಗಳನ್ನು ಪರಿಶೀಲಿಸಿ ಅವುಗಳನ್ನು ಮೃದುವಾಗಿ ನಿರ್ವಹಿಸುತ್ತದೆ, ನೀವು ಮಿತಿಯೊಂದಕ್ಕೆ ತಲುಪಿದಾಗ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಉಲ್ಲಂಘನೆಗೊಳ್ಳದಂತೆ.
- **ರೂಪಮಾನಿತ ದತ್ತಾಂಶ ಪಾರ್ಸಿಂಗ್**: API ಪ್ರತಿಕ್ರಿಯೆಗಳು ಬಿಕ್ಕುಕುಹೊಂದುದಾದವು ಮತ್ತು ಅಂತರಂಗ ರೂಪಗಳಿರಬಹುದು. ಈ ಕಲ್ಪನೆ LLM ಅಥವಾ ಇತರ ಪ್ರೋಗ್ರಾಮ್ ಗಳಿಗೆ ಸ್ನೇಹಪೂರಿತ, ಸುಲಭವಾಗಿ ಉಪಯೋಗಿಸುವ ಸ್ವರೂಪಗಳಿಗೆ ಇವುಗಳನ್ನು ಪರಿವರ್ತಿಸುವುದರ ಬಗ್ಗೆ.
- **ದೋಷ ಪುನಃಶ್ರೇಷ್ಠತೆ (Error Recovery)**: ಕೆಲವೊಮ್ಮೆ ಸಮಸ್ಯೆಗಳು ಉಂಟಾಗಬಹುದು—ಶಾಲಿನ ವ್ಯತ್ಯಾಯ, API ನಿರೀಕ್ಷಿತ ಪ್ರತಿಕ್ರಿಯೆ ನೀಡದಿರುವುದು. ದೋಷ ಪುನಃಶ್ರೇಷ್ಠತೆ ಅಂದರೆ ನಿಮ್ಮ ಕೋಡ್ ಈ ಸಮಸ್ಯೆಗಳನ್ನು ನಿಭಾಯಿಸಿ, ಕ್ರ್ಯಾಶ್ ಆಗದೆ ಉಪಯುಕ್ತ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ನೀಡುತ್ತದೆ.
- **ಪ್ಯಾರಾಮೀಟರ್ ಮಾನ್ಯತೆ (Validation)**: ಇದು ನಿಮ್ಮ ಸಾಧನಗಳಿಗೆ ನೀಡಲಾಗುವ ಎಲ್ಲ ಒಳಬರುತ್ತಿರುವ ಮಾಹಿತಿ ಸರಿಯೆಂದು ಪರಿಶೀಲಿಸುವ ಬಗ್ಗೆ. ಇದರಲ್ಲಿ ಮೊದಲಿನಿಂದ ನಿರ್ದಿಷ್ಟ ಮೌಲ್ಯಗಳನ್ನು ಹೊಂದಿಸುವುದು ಮತ್ತು ಟೈಪ್‌ಗಳು ಸರಿಯಾಗಿವೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸುವುದು ಸೇರಿವೆ, ಇದು ದೋಷಗಳು ಮತ್ತು ಗೊಂದಲವನ್ನು ತಡೆಯುತ್ತದೆ.

ಈ ವಿಭಾಗವು ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್ ಬಳಕೆ ವೇಳೆ ಸಾಮಾನ್ಯವಾಗಿ ಎದುರಾಗುವ ಸಮಸ್ಯೆಗಳನ್ನು ಗುರುತಿಸಲು ಮತ್ತು ಪರಿಹರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ನೀವು ದೋಷ ಅಥವಾ ಅನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಎದುರಿಸಿದಾಗ, ಈ ಸಮಸ್ಯಾ ಪರಿಹಾರ ವಿಭಾಗವು ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳಿಗೆ ಪರಿಹಾರಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ. ಹೆಚ್ಚಿನ ಸಹಾಯ ಕೋರುವು ಮೊದಲು ಈ ಸಲಹೆಗಳನ್ನು ಪರಿಶೀಲಿಸಿ—ಅವು ಬಹುಶಃ ಸಮಸ್ಯೆಗಳನ್ನು ಶೀಘ್ರವಾಗಿ ಪರಿಹರಿಸಬಹುದು.

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್ನ್ನು ಬಳಸುವಾಗ, ನೀವು ಕೆಲವೊಮ್ಮೆ ಸಮಸ್ಯೆಗಳನ್ನು ಎದುರಿಸಬಹುದು—ಇದು ಬಾಹ್ಯ API ಗಳು ಮತ್ತು ಹೊಸ ಸಾಧನಗಳನ್ನು ಬಳಸುವಾಗ ಸಾಮಾನ್ಯವಾಗಿದೆ. ಈ ವಿಭಾಗವು ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳಿಗೆ ಪ್ರಾಯೋಗಿಕ ಪರಿಹಾರಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ, ನಿಮ್ಮ ಸಂಪರ್ಕವನ್ನು ಬೇಗ ಮತ್ತೆ ಸರಿಹೊಂದಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ನೀವು ದೋಷ ಕಂಡರೆ, ಇಲ್ಲಿ ಪ್ರಾರಂಭಿಸಿ: ಕೆಳಗಿನ ಸಲಹೆಗಳು ಬಹುಶಃ ಬಹುಪಾಲು ಬಳಕೆದಾರರ ಎದುರಿಸುವ ಸಮಸ್ಯೆಗಳನ್ನು ಮುಕ್ತಾಯಗೊಳಿಸುತ್ತದೆ ಮತ್ತು ಹೆಚ್ಚಿನ ಸಹಾಯ ಅಗತ್ಯವಿಲ್ಲದಂತೆ ಮಾಡಬಹುದು.

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು

ಕೆಳಗಿನವು ಬಳಕೆದಾರರು ಸಾಮಾನ್ಯವಾಗಿ ಎದುರಿಸುವ ಕೆಲವು ಸಮಸ್ಯೆಗಳು, ಮತ್ತು ಅವುಗಳಿಗೆ ಸ್ಪಷ್ಟ ವಿವರಣೆಗಳು ಮತ್ತು ಪರಿಹಾರ ಕ್ರಮಗಳು:

1. **.env ಫೈಲ್‌ನಲ್ಲಿ SERPAPI_KEY ಇಲ್ಲದಿರುವುದು**
   - ನೀವು `SERPAPI_KEY environment variable not found` ದೋಷ ಸಂದೇಶವನ್ನು ನೋಡಿದರೆ, ಅಂದರೆ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ SerpAPIಗೆ ಪ್ರವೇಶಿಸಲು ಅಗತ್ಯವಿರುವ API ಕೀಲಿಯನ್ನು ಪತ್ತೆಹಚ್ಚಲಾಗುತ್ತಿಲ್ಲ. ಇದನ್ನು ಸರಿಪಡಿಸಲು, ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್ ರೂಟ್‌ನಲ್ಲಿ `.env` ಎಂಬ ಫೈಲ್ ರಚಿಸಿ ಮತ್ತು `SERPAPI_KEY=ನಿಮ್ಮ_serpapi_ಕೀಲಿ_ಇಲ್ಲಿ` ಎಂಬ ಸಾಲಿನಲ್ಲಿ ನಿಮ್ಮ ನಿಜವಾದ ಕೀಲಿಯನ್ನು SerpAPI ವೆಬ್‌ಸೈಟ್‌ನಿಂದ ಪಡೆದಂತೆ ಸೇರಿಸಿ.

2. **ಮಾಡ್ಯೂಲ್ ಕಾಣದೆ ದೋಷಗಳು**
   - `ModuleNotFoundError: No module named 'httpx'` ಹಾಗು ಇತರ ಕಳೆದದೋಷಗಳು ಸೂಚಿಸುತ್ತವೆ ನೀವು ಅಗತ್ಯವಿರುವ ಪೈಥಾನ್ ಪ್ಯಾಕೇಜ್‌ಗಳನ್ನು ಸ್ಥಾಪಿಸಿಲ್ಲ. ಇದಕ್ಕಾಗಿ `pip install -r requirements.txt` ಅನ್ನು ನಿಮ್ಮ ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ರನ್ ಮಾಡಿ ಎಲ್ಲಾ ಅವಲಂಬನಗಳನ್ನು ಸ್ಥಾಪಿಸಿ.

3. **ಸಂಪರ್ಕ ಸಮಸ್ಯೆಗಳು**
   - `Error during client execution` ಎಂಬ ದೋಷ ಬಂದರೆ, ತಮ್ಮ ಕ್ಲೈಂಟ್ ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸಲಾರದೆ ಅಥವಾ ಸರ್ವರ್ ನಿರೀಕ್ಷೆಯಂತೆ ಚಾಲನೆಗೊಳ್ಳುತ್ತಿಲ್ಲ. ಕ್ಲೈಂಟ್ ಮತ್ತು ಸರ್ವರ್ ಎರಡೂ ಹೊಂದಾಣಿಕೆಯ ಸಂಸ್ಕರಣೆ ಇದ್ದಾರೆ ಎಂದು ಪರಿಶೀಲಿಸಿ, ಮತ್ತು `server.py` ಫೈಲ್ ಸರಿಯಾದ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಇದ್ದು ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿದೆಯೇ ಎಂದು ದೃಢಪಡಿಸಿ. ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಎರಡು ಕಡೆಗಳನ್ನು ಪುನರಾರಂಭಿಸುವುದು ಸಹ ಸಹಾಯವಾಗಬಹುದು.

4. **SerpAPI ದೋಷಗಳು**
   - `Search API returned error status: 401` ಎಂಬ ದೋಷ ಕಂಡುಬಂದರೆ, ನಿಮ್ಮ SerpAPI ಕೀಲಿ ಇಲ್ಲ, ತಪ್ಪು ಅಥವಾ ಅವಧಿ ಮುಗಿದಿದ್ದು. ನಿಮ್ಮ SerpAPI ಡ್ಯಾಶ್ಬೋರ್ಡ್‌ಗೆ ಹೋಗಿ ಕೀಲಿ ಪರಿಶೀಲಿಸಿ, ಅಗತ್ಯವಿದ್ದರೆ `.env` ಫೈಲ್ ಅನ್ನು ನವೀಕರಿಸಿ. ನಿಮ್ಮ ಕೀಲಿ ಸರಿಯಾಗಿದೆಯೇ ಆದರೆ ಈ ದೋಷ ಮುಂದುವರೆದರೆ, ಉಚಿತ ಸ್ತರದಲ್ಲಿ ಔಟ್ ಆಗಿರುವ ದರ ಮಿತಿ (quota) ಹೊಂದಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಿ.

### ಡಿಬಗ್ ಮೋಡ್

ಸಾಮಾನ್ಯವಾಗಿ, ಅಪ್ಲಿಕೇಶನ್ ಕೆವಲ ಮುಖ್ಯ ಮಾಹಿತಿಗಳನ್ನು ಲಾಗ್ ಮಾಡುತ್ತದೆ. ನೀವು ಇನ್ನಷ್ಟು ವಿವರಗಳನ್ನು (ಉದಾ: ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಲು) ನೋಡಬೇಕಾದರೆ, ನೀವು ಡಿಬಗ್ ಮೋಡ್ ಸಕ್ರಿಯಗೊಳಿಸಬಹುದು. ಇದು ಅಪ್ಲಿಕೇಶನ್ ಕೈಗೊಳ್ಳುವ ಪ್ರತಿ ಹಂತದ ಬಗ್ಗೆ ಹೆಚ್ಚಿನ ಮಾಹಿತಿಗಳನ್ನು ತೋರಿಸುತ್ತದೆ.

**ಉದಾಹರಣೆ: ಸಾಮಾನ್ಯ ಔಟ್‌ಪುಟ್**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**ಉದಾಹರಣೆ: ಡಿಬಗ್ ಔಟ್‌ಪುಟ್**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

ಡಿಬಗ್ ಮೋಡ್ HTTP ವಿನಂತಿಗಳು, ಪ್ರತಿಕ್ರಿಯೆಗಳು ಮತ್ತು ಇತರ ಆಂತರಿಕ ವಿವರಗಳ ಬಗ್ಗೆ ಹೆಚ್ಚುವರಿ ಸಾಲುಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ಇದು ಸಮಸ್ಯೆ ಪರಿಹಾರಕ್ಕೆ ಬಹಳ ಸಹಕಾರಿ.

ಡಿಬಗ್ ಮೋಡ್ ಸಕ್ರಿಯಗೊಳಿಸಲು, `client.py` ಅಥವಾ `server.py` ಮೇಲ್ಭಾಗದಲ್ಲಿ ಲಾಗಿಂಗ್ ಮಟ್ಟವನ್ನು DEBUG-ಗೆ ಸೆಟ್ ಮಾಡಿ:

# [Python](#tab/python-debug)


```python
# ನಿಮ್ಮ client.py ಅಥವಾ server.py ನ κορ್ಶೆಯಲ್ಲಿ
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO ನಿಂದ DEBUG ಗೆ ಬದಲಿಸು
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
