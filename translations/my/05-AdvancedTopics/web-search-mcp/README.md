# သင်ခန်းစာ - ဝဘ်ရှာဖွေမှု MCP ဆာဗာတည်ဆောက်ခြင်း

ဒီအခန်းက အပြင် API များနှင့် ပေါင်းစည်းပြီး မတူညီတဲ့ဒေတာအမျိုးအစားတွေကို ကိုင်တွယ်နိုင်၊ အမှားများကို စီမံကြပ်မတ်နိုင်၊ တစ်ချိန်တည်းမှာ ကိရိယာများစွာကို စုပေါင်းစီမံနိုင်တဲ့ စက်မှုလုပ်ငန်းအသုံးပြုနိုင်တဲ့ နည်းလမ်းဖြင့် အမှန်တကယ်လုံခြုံသော AI ပုံစံကို ပြသပါလိမ့်မယ်။ သင်တွေ့မြင်မယ့်အရာတွေမှာ -

- **အတည်ပြုချက်လိုအပ်သော အပြင် API များနှင့် ပေါင်းစည်းခြင်း**
- **အမျိုးမျိုးသော ဒေတာအမျိုးအစားများကို စီမံခန့်ခွဲခြင်း**
- **ခိုင်မာသော အမှား ကိုင်တွယ်ခြင်းနှင့် မှတ်တမ်းတင်နည်းလမ်းများ**
- **တစ်ခုတည်းသော ဆာဗာတွင် ကိရိယာများစွာ စုပေါင်း စီမံခြင်း**

စတင်ပြီးနောက် သင်သည် အဆင့်မြင့် AI နှင့် LLM အခြေပြု အက်ပလီကေးရှင်းများအတွက် အရေးကြီးတဲ့ ပုံစံများနှင့် အသုံးချမှုအကောင်းဆုံးနည်းလမ်းများကို လက်တွေ့အသုံးပြုနိုင်မှာ ဖြစ်ပါတယ်။

## နိဒါန်း

ဒီသင်ခန်းစာမှာ SerpAPI ကို အသုံးပြု၍ LLM စွမ်းရည်များကို အချိန်နဲ့ တည့်တဲ့ ဝဘ်ဒေတာဖြင့် တိုးချဲ့နိုင်တဲ့ MCP ဆာဗာနှင့် ကလိုင်င့်ကို ဘယ်လိုတည်ဆောက်မယ်ဆိုတာ သင်ယူမှာ ဖြစ်ပါတယ်။ ဒါဟာ မကြာခဏ အချက်အလက်သစ်တွေကို ဝဘ်မှာရယူနိုင်တဲ့ နောက်ဆုံးပေါ် AI ကိုဖန်တီးရာမှာ အရေးကြီးတဲ့ ကျွမ်းကျင်မှု ဖြစ်ပါတယ်။

## သင်ယူရမယ့် ရည်မှန်းချက်များ

ဒီသင်ခန်းစာအဆုံးသတ်ရင် ကျောင်းသားတွေသည် အောက်ပါအရာများကို လုပ်နိုင်မယ်။

- MCP ဆာဗာထဲတွင် ဘေးပိတ်လုံခြုံစွာအပြင် API များ (ဥပမာ SerpAPI) ပေါင်းစည်းခြင်း
- ဝဘ်၊ သတင်း၊ ထုတ်ကုန်ရှာဖွေရေးနှင့် မေး-ဖြေ အတွက် ကိရိယာများစုံကို အကောင်အထည်ဖော်ခြင်း
- LLM အတွက် နှလုံးသားဖွဲ့စည်းသော ဒေတာများကို စိစစ်ပုံဖော်ခြင်း
- အမှားများကို ကိုင်တွယ်ပြီး API အရည်အသွေး အကန့်အသတ်များ စနစ်တကျ စီမံခြင်း
- အလိုအလျောက်စမ်းသပ်မှုများနှင့် ပြောဆိုရင်း စမ်းသပ်ပုံ အမျိုးမျိုး MCP ကလိုင်င့်များ တည်ဆောက်ခြင်းနှင့် စမ်းသပ်ခြင်း

## ဝဘ်ရှာဖွေရေး MCP ဆာဗာ

ဒီအပိုင်းမှာ ဝဘ်ရှာဖွေရေး MCP ဆာဗာ၏ မြေပုံပုံစံနှင့် လုပ်ဆောင်ချက်များကို မိတ်ဆက်ပါမယ်။ FastMCP နဲ့ SerpAPI ကို တွဲဖက်အသုံးပြုပြီး LLM စွမ်းရည်သစ်များကို အချိန်နဲ့တပြေးညီ ဝဘ်ဒေတာဖြင့် တိုးချဲ့ပုံကို မြင်ရမှာ ဖြစ်ပါတယ်။

### အခြေခံအကြောင်းအရာ

ဒီအကောင်အထည်ဖော်မှုမှာ MCP ကိရိယာလေးခုပါဝင်ပြီး အပြင် API များကို လုံခြုံပြီး ထိရောက်စွာ စီမံနိုင်မှုကို ဖော်ပြပါတယ်။

- **general_search**: ကျယ်ပြန့်သော ဝဘ် ရလဒ်များအတွက်
- **news_search**: နောက်ဆုံးပေါ် ခေါင်းစဉ်များအတွက်
- **product_search**: အီလက်ထရွန်းနစ်လက်လီ မိတ်ဆက်ချက်အတွက်
- **qna**: မေးခွန်းနှင့် ဖြေချက်စာတမ်း များအတွက်

### လုပ်ဆောင်ချက်များ
- **ကုဒ် နမူနာများ**: Python အတွက် ဘာသာစကားအထူးကုဒ်ကုံးများပါဝင်ပြီး အခြားဘာသာစကားများနှင့်လွယ်ကူစွာ ရှေ့လျှောက်ခွင့်ရှိသည်။

### Python

```python
# general_search ကိရိယာ၏ နမူနာအသုံးပြုမှု ဥပမာ
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

ကလိုင်င့်ကို အလုပ်လုပ်ခင်းမပြုမီ ဆာဗာက ဘာလုပ်နေသလဲ သိဖို့ အားကောင်းပါတယ်။ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ဖိုင်မှာ MCP ဆာဗာတင်ဆက်ထားပြီး SerpAPI နှင့် ပေါင်းစည်းကာ ဝဘ်၊ သတင်း၊ ထုတ်ကုန် ရှာဖွေရေး၊ မေး-ဖြေ ကိရိယာများကို ဖော်ပြစေပါတယ်။ ဖောက်သည်ဆီသို့ ဖော်ပြရန် တောင်းဆိုမှုများကို ကိုင်တွယ်ကာ API ခေါ်ဆိုမှုများ စီမံ၊ ဖြေကြားချက်များကို စိစစ်ပြီး ဖွဲ့စည်းတည်ဆောက်ထားသော ရလဒ်များ ပြန်ပေးပို့ပါတယ်။

ပြည့်စုံတဲ့ အကောင်အထည်ဖော်မှုကို [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) မှာ တိုက်ရိုက်ကြည့်ရှုပေးနိုင်ပါသည်။

ဆာဗာက ကိရိယာတစ်ခုသတ်မှတ်ပြီး မှတ်ပုံတင်ပုံနမူနာအနည်းငယ်ကို အောက်တွင် ဖော်ပြထားသည်။

### Python ဆာဗာ

```python
# server.py (အစိတ်အပိုင်း)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...အကောင်အထည်ဖော်မှု...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **အပြင် API ပေါင်းစည်းခြင်း**: API key များနှင့် အပြင်အပ်ဒိတ်များလုံခြုံစွာနှင့် တင်ပြမှု
- **ဖွဲ့စည်းတည်ဆောက်ထားသော ဒေတာ စိစစ်ခြင်း**: API ဖြေကြားချက်များကို LLM အတွက် သင့်လျော်အောင် ပြောင်းလဲပြသခြင်း
- **အမှားကိုင်တွယ်မှု**: သေချာနှင့် စနစ်တကျ မှတ်တမ်းတင်မှုနှင့်အတူ ခိုင်မာသော အမှားကိုင်တွယ်မှု
- **ပြောဆိုခွင့် Client**: အလိုအလျောက် စမ်းသပ်မှုများနှင့် မျက်မှောက်စမ်းသပ်မှုတိုတိုပါရှိသော အပြောရှင်းနည်း
- **အကြောင်းအရာ စီမံခန့်ခွဲမှု**: MCP Context အသုံးပြုပြီး မှတ်တမ်းတင်ခြင်းနှင့် တောင်းဆိုမှုများကို ကောက်နှုတ်ခြင်း

## လိုအပ်ချက်များ

စတင်မလုပ်မီ သင်၏ ပတ်ဝန်းကျင်ကို မှန်ကန်စွာဖွဲ့စည်းထားကြောင်း သေချာစေရန် အောက်ဖော်ပြပါ အဆင့်များကို လိုက်နာပါ။ ဒါဟာ ဧည့်ခံဖြေရှင်းမှုများ တပ်ဆင်ပြီး API key များကို မှန်ကန်စွာထည့်သွင်းဖို့ အတွက်လည်း အရေးကြီးပါသည်။

- Python 3.8 သို့မဟုတ် အထက်
- SerpAPI API key (https://serpapi.com/ မှ အခမဲ့ စတင် မှတ်ပုံတင်နိုင်သည်)

## တပ်ဆင်ခြင်း

စတင်ရန် အတွက် သင်၏ပတ်ဝန်းကျင်ကို ဖွဲ့စည်းရန် အောက်ပါအဆင့်များကို လိုက်နာပါ။

1. မူလတပ်ဆင်မှုများကို uv (အကြံပြု) သို့မဟုတ် pip ဖြင့် တပ်ဆင်ပါ။

```bash
# uv ကိုအသုံးပြုခြင်း (အကြံပြုသည်)
uv pip install -r requirements.txt

# pip ကိုအသုံးပြုခြင်း
pip install -r requirements.txt
```

2. ပရောဂျက် ရုပ်မြင့်အောက်၌ `.env` ဖိုင်တစ်ခု ဖန်တီးပြီး သင်၏ SerpAPI key ထည့်သွင်းပါ။

```
SERPAPI_KEY=your_serpapi_key_here
```

## အသုံးပြုမှု

ဝဘ်ရှာဖွေရေး MCP ဆာဗာသည် SerpAPI နှင့် ပေါင်းစည်းကာ ဝဘ်၊ သတင်း၊ ထုတ်ကုန် ရှာဖွေမှုနှင့် မေး-ဖြေ ကိရိယာများကို ဖော်ပြသည့် အဓိကပစ္စည်းဖြစ်ပါသည်။ တောင်းဆိုမှုများကို ကိုင်တွယ်ကာ API ခေါ်ဆိုမှုများ စီမံ၊ ဖြေကြားချက်များကို ဖော်ပြအောင် ဖွဲ့စည်းတည်ဆောက်ပါတယ်။

ပြည့်စုံသော အကောင်အထည်ဖော်မှုကို [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) မှ တွေ့ရှုနိုင်ပါသည်။

### ဆာဗာကို စတင်လည်ပတ်ခြင်း

MCP ဆာဗာကို ဖွင့်ရန် အောက်ပါ 명령ကို အသုံးပြုပါ။

```bash
python server.py
```

ဆာဗာသည် ဖောက်သည်နှင့် တိုက်ရိုက် ချိတ်ဆက်နိုင်သော stdio-အခြေခံ MCP ဆာဗာအဖြစ် လည်ပတ်ပါလိမ့်မယ်။

### ကလိုင်င့် အမျိုးအစားများ

ကလိုင်င့် (`client.py`) သည် MCP ဆာဗာနှင့် ဆက်သွယ်ရာတွင် အမျိုးအစားနှစ်မျိုးကို ပံ့ပိုးပေးပါသည်။

- **ပုံမှန်ပြုလုပ်မှု**: ကိရိယာများအားလုံးကို အသုံးပြုပြီး အလိုအလျောက် စမ်းသပ်မှုများ ပေါင်းစပ် လုပ်ဆောင်ပါသည်။ ဆာဗာနဲ့ ကိရိယာများ အလုပ်လုပ်မှုကို အမြန်စစ်ဆေးရန် အထူးအသုံးဝင်သည်။
- **ပြောဆိုနိုင်မူ့စ်ကို ဘား**: မီနူးစနစ်ဖြင့် လုပ်ဆောင်ချက်များအပေါ် တောင်းဆိုနိုင်ပြီး မိမိစိတ်တိုင်းကျ လေ့လာရုံဖြင့် ရလဒ်တွေကို တိုက်ရိုက် ကြည့်ရှုနိုင်သည်။ ဆာဗာကြီး၏ စွမ်းရည်များကို စူးစမ်းပြသရန်နှင့် မတူညီသော အကြောင်းအရာများကို စမ်းသပ်ရန် အတွက် အကောင်းဆုံးဖြစ်သည်။

ပြည့်စုံတဲ့ အကောင်အထည်ဖော်မှုကို [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) မှ တိုက်ရိုက်ကြည့်ရှုနိုင်ပါသည်။

### ကလိုင်င့်ကို လည်ပတ်ခြင်း

### ဘယ်လိုနည်းလမ်းတွေနဲ့ စမ်းသပ်မလဲ

ဆာဗာမှ စီစဉ်ပေးသည့် ကိရိယာများနှင့် မတူညီသော နည်းလမ်းများမှ ရွေးချယ်ပြုလုပ်ပြီး စမ်းသပ်မှုလုပ်နိုင်သည်။

#### MCP Python SDK ဖြင့် သင်၏ကိုယ်ပိုင် စမ်းသပ်မှု စာတမ်းရေးခြင်း
MCP Python SDK ကို အသုံးပြုပြီး မိမိတိုင်း လိုအပ်သော စမ်းသပ်မှု script များ ဖန်တီးနိုင်သည်။

# [Python](#tab/python-sdk)

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def test_custom_query():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            # သင်၏စိတ်ကြိုက်ပါရာမီတာများဖြင့် စက်များကိုခေါ်ပါ
            result = await session.call_tool("general_search", 
                                           arguments={"query": "your custom query"})
            # ရလဒ်ကို ပြုစုပျိုးထောင်ပါ
```

---


ဒီအ ကြောင်းမှာ "စမ်းသပ်မှု script" ဆိုတာ သင်သည် MCP ဆာဗာအတွက် ကလိုင်င့် အဖြစ် လုပ်ဆောင်မည့် Python ပရိုဂရမ် များဖြစ်သည်။ တရားဝင် ယူနစ် စမ်းသပ်မှု မဟုတ်ပေမယ့် ဆာဗာနှင့် ချိတ်ဆက်ကာ မိမိလိုချင်သလို ကိရိယာကို အီးပ Parameter များဖြင့်ခေါ်သုံးပြီး ရလဒ်ကို ကြည့်ရှုနိုင်သည်။ ဒီနည်းလမ်းက -
- ကိရိယာခေါ်ဆိုမှု များကို ချိတ်ဆက်၊ စမ်းသပ်ရန်
- ဆာဗာ၏ အကြောင်းတုံ့ပြန်မှု ချက်ချက်မှန်ကန်မှုကို အတည်ပြုရန်
- ကိရိယာများကို အကြိမ်ကြိမ် စမ်းသပ်ရန် အလိုအလျောက်ဆောင်ရွက်မှုများတည်ဆောက်ရန်
- MCP ဆာဗာအပေါ် ကိုယ်ပိုင် လည်ပတ်မှုစနစ်များ သို့မဟုတ် ပေါင်းစည်းမှုများ တည်ဆောက်ရန်

MCP Python SDK ကို အသုံးပြုပြီး စမ်းသပ် script များကို မြန်မြန်ဆန်ဆန် လုပ်ဆောင်နိုင်ပြီး မတူညီသော မေးချက်များ၊ ကိရိယာ လုပ်ဆောင်မှု တွေကို မှန်ကန်စွာ ရယူနိုင်မှာပါ။

## ကိရိယာ ရုပ်ထွက်ဖေါ်ပြချက်များ

ဆာဗာမှ ပေးသော ကိရိယာများကို အသုံးပြုပြီး မတူညီသော ရှာဖွေရေးနှင့် မေးခွန်းများကို စစ်ဆေးနိုင်ပါသည်။ ကိရိယာတိုင်း၏ parameter များနှင့် အသုံးပြုပုံ ဥပမာများကို အောက်တွင် ဖော်ပြထားသည်။


ဒီအပိုင်းတွင် ကိရိယာ များနှင့် parameter များအသေးစိတ်ဖော်ပြထားပါသည်။

### general_search

ယေဘုယျ ဝဘ်ရှာဖွေမှု ပြုလုပ်ပြီး format ပြီးသော ရလဒ်အဖြစ် ပြန်လည်ပေးပို့သည်။

**ဒီကိရိယာကို ဘယ်လိုခေါ်သုံးမလဲ**

MCP Python SDK ကို အသုံးပြုပြီး သင့်ကိုယ်ပိုင် script မှ `general_search` ကိုခေါ်သုံးနိုင်သည်၊ ဒါမှမဟုတ် Inspector သို့မဟုတ် ပြောဆိုနိုင်မူ့စ်ကို ဘားဖြင့်လည်း အသုံးပြုနိုင်သည်။ SDK နမူနာကို အောက်တွင် ဖော်ပြထားသည်။

# [Python Example](#tab/python-general-search)

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_general_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "latest AI trends"})
            print(result)
```

---

ဒါမှမဟုတ် ပြောဆိုနိုင်မူ့စ်ကို ဘားတွင် `general_search` ကို မီနူးမှ ရွေးချယ်ပြီး မေးခွန်းကို ထည့်သွင်းနိုင်သည်။

**Parameters:**
- `query` (string): ရှာဖွေမည့် မေးခွန်း

**ဥပမာ တောင်းဆိုမှု**:

```json
{
  "query": "latest AI trends"
}
```

## ဒီသင်ခန်းစာထဲက အဆင့်မြင့် အယူအဆများ

တည်ဆောက်ခင် ဒီအချက်များကို နားလည်ထားသင့်သည်။ ဒီအယူအဆများနားလည်ခြင်းက သင်ကို အဆင့်မြင့် နည်းပညာများ လေ့လာရာတွင် အထောက်အကူဖြစ်စေပါလိမ့်မယ်။

- **ကိရိယာစုပေါင်း စီမံခန့်ခွဲမှု**: သဘောတရားတခုမှာ MCP ဆာဗာတစ်ခုမှာ ဝဘ်ရှာဖွေမှု, သတင်းရှာဖွေမှု, ထုတ်ကုန်ရှာဖွေမှု တို့ကဲ့သို့ ကိရိယာအမျိုးမျိုးကို တပြိုင်နက်မှာ လည်ပတ်နိုင်ခြင်းဖြစ်သည်။ ဒီနည်းလမ်းက ဆာဗာကို ကိုယ်ပိုင်အလုပ်များ အများကြီးကောင်းမွန်စွာ စီမံခန့်ခွဲနိုင်စေသည်။
- **API အရည်အသွေး အကန့်အသတ် ကိုင်တွယ်မှု**: အပြင် API များစွာ (ဥပမာ SerpAPI) မှ တောင်းဆိုမှု အရေအတွက်ကို ကန့်သတ်ထားသည်။ မှန်ကန်သော ကုဒ်တွေက ဒီအကန့်အသတ်တွေကို စစ်ဆေးပြီး ပြဿနာမဖြစ်အောင် စီမံပေးနိုင်သည်။
- **ဖွဲ့စည်းတည်ဆောက်ထားသော ဒေတာ စိစစ်ခြင်း**: API ဖြေကြားချက်များသည် အများအားဖြင့် စိပ်စိပ်လေးနှင့် ရုပ်ပုံတောက်မှာ ဖြစ်သည်။ ဒီအယူအဆက ဒီအဖြေများကို LLM သို့မဟုတ် အခြားအစီအစဉ်သုံးစွဲရာတွင် လွယ်ကူအောင် စနစ်တကျ ပြောင်းလဲပေးတာဖြစ်သည်။
- **အမှား ပြန်လည်ကောင်းမွန်မှု**: အချို့အခါမှာ အကြောင်းအရာ မမှန်ဘဲ ကွန်ယက် ဒီဇာတ်လမ်းမရှိခြင်း သို့မဟုတ် API အဖြေရမရောက်ခြင်း စတဲ့ ပျောက်ကွယ်မှုများဖြစ်တတ်သည်။ ဒီနည်းလမ်းက ကုဒ်အမှားတွေကို ကိုင်တွယ်ပြီး ဖောက်သည်ကို အသုံးဝင်သော ပြန်လည်အသိပေးမှု ပေးပေးနို်င်စေသည်။
- **Parameter အတည်ပြုခြင်း**: ကိရိယာများသို့ ပေးပို့မည့် input အားလုံး၏ မှန်ကန်မှုနှင့် လုံခြုံမှုကို စစ်ဆေးခြင်းဖြစ်သည်။ အတည်ပြုချက်တွင် ပုံမှန်တန်ဖိုးသတ်မှတ်ခြင်းနှင့် အမျိုးအစားမွမ်းမံခြင်းတို့ ပါဝင်သည်။ ဒါက bugs တွေ ရှောင်ကြဉ်ရာတွင် အထောက်အကူဖြစ်ပါသည်။

ဒီအပိုင်းက သင် ဝဘ်ရှာ MCP ဆာဗာနဲ့ လုပ်ဆောင်တုန်း တွေ့ကြုံနိုင်တဲ့ ပြဿနာများကို ရှာဖွေ၊ ဖြေရှင်း ဆောင်ရွက်ပေးမှာဖြစ်သည်။ ပြဿနာတွေရှိခဲ့ရင် ဒီ troubleshooting အပိုင်းကို စတင် ပြန်လည်ကြည့်ရှုမှာ ကောင်းပါတယ် - အများအားဖြင့် ဒီနည်းတွေက ပြဿနာအများစုကို မြန်မြန် ဖြေရှင်းပေးနိုင်ပါသည်။

## ပြဿနာရှာဖွေရေးနှင့် ဖြေရှင်းခြင်း

ဝဘ်ရှာ MCP ဆာဗာနဲ့ လုပ်ကိုင်စဉ့်အခါ ခဏခဏ ပြဿနာတွေ ရှိနိုင်ပါတယ် - အပြင် API များနှင့် အသစ်ထည့်သွင်းမည့် ကိရိယာတွေကြောင်းဆိုတာ သဘာဝကျတဲ့အခြေအနေပါ။ ဒီအပိုင်းက ကြုံတွေ့ရမယ့် ထိပ်တန်းပြဿနာများကို စနစ်တကျ ဖြေရှင်းပေးပြီး သင့်အား လျင်မြန်စွာ ပြန်လည်လုပ်ဆောင်နိုင်စေမယ်။ ပြဿနာ ရှိရင် အောက်ပါ နည်းလမ်းတွေနဲ့ စတင်ဦးပါ - ဒီအကြံပြုချက်တွေက အသုံးပြုသူအများဆုံးကြုံတွေ့ရတဲ့ပြဿနာတွေကို ဖြေရှင်းပေးနိုင်သည်။

### လူကြုံတွေ့နိင်တဲ့ ပြဿနာများ

အသုံးပြုသူအများစု ကြုံတွေ့ရသော ပြဿနာများ အနည်းငယ်နှင့် တိကျရှင်းလင်းမှု၊ ဖြေရှင်းရေးနည်းလမ်းများက如下:

1. **.env ဖိုင်ထဲမှာ SERPAPI_KEY မရှိခြင်း**
   - `SERPAPI_KEY environment variable not found` ဆိုတဲ့ အမှားကြုံရင် သင့်အက်ပလီကေးရှင်းမှာ SerpAPI အသုံးပြုရန်လိုအပ်တဲ့ API key ကို မတွေ့ရှိတာဖြစ်သည်။ ပြန်လည်ဖြေရှင်းရန် သင့်ပရောဂျက်ရဲ့ မူလFolder တွင် `.env` ဖိုင် တစ်ခု ဖန်တီးပြီး `SERPAPI_KEY=your_serpapi_key_here` လိုပုံစံ ထည့်သွင်းပါ။ `your_serpapi_key_here` ကို SerpAPI ဝက်ဘ်ဆိုဒ်မှာ ရရှိထားတဲ့ သင့် key ဖြင့် အစားထိုးပေးပါ။

2. **Module မတွေ့ခြင်းမှားများ**
   - `ModuleNotFoundError: No module named 'httpx'` ကဲ့သို့သော မှားများ သတိပေးသည်ဆိုလျင် လိုအပ်သော Python package မတပ်ဆင်ရသေးခြင်း ဖြစ်နိုင်သည်။ ပြဿနာဖြေရှင်းရန် တမင် terminal မှာ `pip install -r requirements.txt` လိုအပ်သော dependency အားလုံးကို တပ်ဆင်ပေးပါ။

3. **ချိတ်ဆက်ဆက်သွယ်မှု ပြဿနာများ**
   - `Error during client execution` ကဲ့သို့သော အမှားတွေကြုံရင် ကလိုင်င့်နေရာမှ ဆာဗာကို ချိတ်ဆက်နိုင်ခြင်း မရှိခြင်း သို့မဟုတ် ဆာဗာ မလည်ပတ်ခြင်း ဖြစ်နိုင်ပါသည်။ ကလိုင်င့်နဲ့ဆာဗာ နှစ်ခုစလုံး ဗားရှင်းကိုက်ညီမှုရှိကြောင်း၊ `server.py` ဖိုင်ကို မှန်ကန်တဲ့ ဒိုင်းရက်တွင် ရှိနေနိုင်ကြောင်း စစ်ဆေးပါ။ ဆာဗာနှင့် ကလိုင်င့် နှစ်ဖက်လုံးကို ထပ်မံပြန်စတင်ပါ။

4. **SerpAPI အမှားများ**
   - `Search API returned error status: 401` ဆိုတာ သင့် SerpAPI key မရှိ၊ မှားနေသော်လည်း သို့မဟုတ် သက်တမ်းကုန်လွန်နေသည်ကို ပြောသည်။ SerpAPI dashboard ကို သွားပတ်ပြီး key ကို စစ်ဆေးပြီး `.env` ဖိုင်ကို ပြင်ဆင်ပေးပါ။ သင့် key မှန်ကန်သော်လည်း အခမဲ့အဆင့် ရုပ်သိမ်းနည်းတွင် ကြမ်းတမ်းနိုင်ပါသည်။

### Debug Mode

ပုံမှန်အားဖြင့် app က များသောသတင်းအချက်အလက်များသာ မှတ်တမ်းတင်ပေးပါသည်။ အပိုသတင်းအချက်အလက်များကို ကြည့်ချင်တယ်ဆိုရင် (ဥပမာပြဿနာတွေ ရှာဖွေရာတွင်) DEBUG mode ကို ဖွင့်နိုင်ပါတယ်။ ဒီ mode မှာ app လုပ်ဆောင်ချက် အဆင့်ဆင့်ကို ပိုမို အသေးစိတ် ပြသပေးပါလိမ့်မယ်။

**ဥပမာ: ပုံမှန် ထွက်မှု**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**ဥပမာ: DEBUG ထွက်မှု**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

DEBUG mode မှာ HTTP တောင်းဆိုမှုများ၊ ဖြေရှင်းချက်များနဲ့ အခြား ပြန်ကြားချက် လူတယ်များပါဝင်တာကို သတိထားပါ။ ဒါက ပြဿနာရှာဖွေရန် အထူးအသုံးဝင်ပါသည်။

DEBUG mode ဖွင့်ဖို့ `client.py` သို့မဟုတ် `server.py` အပေါ်တွင် logging level ကို DEBUG အဖြစ် သတ်မှတ်ပေးပါ။

# [Python](#tab/python-debug)


```python
# သင်၏ client.py သို့မဟုတ် server.py အပေါ်ဆုံးတွင်
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO မှ DEBUG သို့ ပြောင်းလဲပါ
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
