# ပြောင်းလဲမှုမှတ်တမ်းများ: MCP အတွက် စတင်သင်ကြားမှု သင်ရိုးအစီအစဉ်

ဤစာရွက်သည် Model Context Protocol (MCP) အတွက် စတင်သင်ကြားမှု သင်ရိုးအစီအစဉ်တွင် ပြုလုပ်ခဲ့သည့် အရေးကြီးပြောင်းလဲမှုများအားလုံး၏ မှတ်တမ်းအဖြစ် တင်ပြသည်။ ပြောင်းလဲမှုများသည် ဘာသာရပ်အသစ်များအတွက် နောက်ဆုံးမှရှေ့သို့ အစီအစဉ်သတ်မှတ်ပြီး (အသစ်ဆုံးပြောင်းလဲမှုများပထမဆုံး) မှတ်တမ်းတင်ထားသည်။

## ၂၀၂၆ ခုနှစ် ဇူလိုင်လ ၂ ရက်

### သင်ခန်းစာအသစ်: ၂၀၂၆-၀၇-၂၈ MCP စပ်တူညီချက် ထုတ်ပြန်ရန် လျှောက်ထားသူ

လာမည့် `၂၀၂၆-၀၇-၂၈` MCP စပ်တူညီချက် ထုတ်ပြန်ရန် လျှောက်ထားသူအကြောင်း (၂၀၂၆ မေ ၂၁ ရက်တွင် ကြေညာပြီး; နောက်ဆုံးထုတ်ပြန်မှုကို ဇူလိုင် ၂၈ ရက်တွင် စီစဉ်ထား) ကို [တရားဝင်ကြေညာချက် ဘလော့ဂ်စာမျက်နှာတွင်](https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/) မွေ့လျော်ဖော်ပြပြီး ထည့်သွင်းတင်ပြခဲ့သည်။ သင်ရိုး၏ အခြေခံဗဟုသုတမှာ **MCP စပ်တူညီချက် ၂၀၂၅-၁၁-၂၅** ဖြစ်၍ ဤသင်ခန်းစာသစ်ကို သက်ဆိုင်ရာ သင်ခန်းစာများကို ပြန်ရေးသားခြင်းမဟုတ်ဘဲ ဦးတည်ချက်သစ်အဖြစ် တင်ပြသည်။

## ၂၀၂၆ ခုနှစ် ဇွန် ၁၆ ရက်

### MCP စပ်တူညီချက်ကို သဟဇာတ စစ်ဆေးခြင်း & နမူနာ သက်သေပြုခြင်း

သင်ရိုးကို လက်ရှိ **MCP စပ်တူညီချက် ၂၀၂၅-၁၁-၂၅** နှင့် နောက်ဆုံးပေါ် တရားဝင် SDK များနှင့် သဟဇာတစွာ စစ်ဆေးပြီး၊ သက်ဆိုင်ရာ စပ်တူညီချက် မှတ်ချက်များအား ဆုံးဖြတ် ပြုပြင်ပြီး၊ အခြေခံ နမူနာများမှာ အောင်မြင်စွာ တည်ဆောက်ကာ တင်ဆက်နိုင်မှု သေချာမှု ရရှိကြောင်း သေချာစေခဲ့သည်။

#### စပ်တူညီချက် ဗားရှင်း ဖြည့်စွက်ပြုပြင်မှုများ (2025-06-18 / 2025-03-26 → 2025-11-25)

အနောက်ပိုင်းသက်တမ်း ကျန်နေသော စပ်တူညီချက်များ သက်ဆိုင်ရာ စံချိန်မပြည့်မှုကို ပြင်ဆင်ပြီး canonical `modelcontextprotocol.io` စပ်တူညီချက် လင့်ခ်များသို့ ပြန်ညွှန်ခဲ့သည်။
- **05-AdvancedTopics/mcp-security/README.md**: "လက်ရှိစံချိန်" ဘနာ၊ နိဒါန်း၊ အချက်အလက်လုံခြုံရေးအကြောင်း ခေါင်းစဉ်၊ မဖြစ်မနေလိုအပ်ချက်ခေါင်းစဉ်၊ Microsoft Entra ID အပိုင်း၊ သတင်းအချက်အလက် အရင်းအမြစ်လင့်ခ်များနှင့် လုံခြုံရေး အကြောင်းကြားချက် (၈ ခု) ကို ၂၀၂၅-၁၁-၂၅ သို့ပြင်ဆင်ခဲ့သည်။
- **05-AdvancedTopics/mcp-transport/README.md**: ထပ်တိုးအရင်းအမြစ် စပ်တူညီချက် လင့်ခ်နှင့် "လက်ရှိစံချိန်" ဘနာကို ၂၀၂၅-၁၁-၂၅ သို့ ပြင်ဆင်သည်။
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: အဘယ့် သက်တမ်းကုန်မှု ဖြစ်သော `2025-03-26` လုံခြုံရေးနှင့် ယုံကြည်မှု လင့်ခ်ကို လက်ရှိ ၂၀၂၅-၁١-၂၅ လုံခြုံရေးအကောင်းဆုံး လင့်ခ်ဖြင့် အစားထိုးသည်။
- **03-GettingStarted/14-sampling/README.md**: တရားဝင် Sampling စာရွက်စာတမ်းများ လင့်ခ်ကို ၂၀၂၅-၁၁-၂၅ သို့ ပြင်ဆင်ခဲ့သည်။
- **03-GettingStarted/05-stdio-server/README.md**: လက်ရှိ MCP စပ်တူညီချက် မူကြမ်းကို ပြောဆိုသော အချိန်ကာလ ဂုဏ်သတ္တိနှင့် ထပ်တိုးအရင်းအမြစ် လင့်ခ်ကို ၂၀၂၅-၁၁-၂၅ သို့ ကြားဖြတ်ပြင်ဆင်သည်။ (၎င်းတွင် SSE မသုံးတော့ခြင်း သမိုင်းရေးမှတ်စုများကို တိကျမှန်ကန်မှုအတွက် မတည်ရှိမှု မပြောင်းလဲ)။

#### လက်ရှိ SDK များနှင့် နမူနာ အသုံးပြုမှု ရှင်းလင်းမှု

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` မှာ `@modelcontextprotocol/sdk@1.29.0` ဖြေရှင်းပြီး `tsc --noEmit` သည် အမှားမရှိစွာ ဖြတ်သွားသည်။ ရှိပြီးသား `McpServer`/`StdioServerTransport` API များ စစ်ဆေးထားသည်။
- **Python (03-GettingStarted/01-first-server/solution/python)**: `.venv` တစ်ခုတွင် `mcp[cli]` (1.27.2) ဖြင့် စစ်ဆေးပြီး၊ `py_compile` ဖြတ်ကာ `FastMCP.list_tools()` သည် `add` နှင့် `subtract` ကိရိယာများကို မှန်ကန်စွာ ပြန်လည်ရရှိခဲ့သည်။
- သိရှိသော သုံးစွဲမှု `@modelcontextprotocol/sdk` ဗားရှင်းများ (`>=1.26.0` / `^1.26.0` / `^1.27.0`) သည် လက်ရှိဗားရှင်း `1.29.0` သို့ ကောက်နှုတ်ပြီး API ပြောင်းလဲမှု မရှိပါ။

#### လိုအပ်ချက်များ ဗားရှင်း လိုက်ဖက်မှု (ဗားရှင်း အားနည်းမှု ဖြေရှင်းခြင်း)

### MCP Specification 2025-11-25 ကို လိုက်နာမှု သုံးသပ်ခြင်း

#### အဓိက အတွေးအခေါ် တိုးတက်မှု (01-CoreConcepts/)
- **Client Primitive အသစ် - Roots**: Roots client primitive အကြောင်း လေ့လာမှု အသေးစိတ် ထည့်သွင်းထားပြီး ဆာဗာများသည် ဖိုင်စနစ် နယ်နိမိတ်နှင့် ခွင့်ပြုချက်များ နားလည်နိုင်စေရန် တိုးတက်မှု
- **ကိရိယာ အမှတ်အသားများ**: ကိရိယာ လုပ်ဆောင်မှု ဆုံးဖြတ်ချက်များအတွက် `readOnlyHint`, `destructiveHint` ဟူသော behavioral annotations အကြောင်းသုတေသနတင်ပြချက် ထည့်သွင်းသည်
- **Sampling အတွင်း ကိရိယာခေါ်ဆိုမှုပုံစံ**: Sampling စာရွက်စာတမ်းတွင် model မေးမြန်းမှုအတွင်း ကိရိယာ `tools` နှင့် `toolChoice` ပါရာမီတာများ ထည့်သွင်း အသုံးပြုမှု
- **URL Mode Elicitation**: ဆာဗာမှ စတင်၍ ပြင်ပ ဝက်ဘ်အင်တာနက် နှင့် ဆက်သွယ်မှု အတွက် URL အခြေခံ ဖော်ပြချက်များ ထည့်သွင်းထားသည်
- **Tasks (စမ်းသပ်ရေး)**: မြဲမြံစွာ လုပ်ဆောင်နိုင်သော wrapper များနှင့် deferred အဖြေ အာမခံချက် များ ပါဝင်သော Tasks မှ စမ်းသပ်လေ့လာမှု အပိုင်း အသစ်ထည့်သည်
- **သင်္ကေတများအား ထောက်ပံ့မှု**: ကိရိယာများ၊ ရင်းမြစ်များ၊ ရင်းမြစ်ပုံစံများနှင့် prompt များတွင် သင်္ကေတများ metadata အဖြစ် ထည့်သွင်းနိုင်ကြောင်း သတင်းပေးသည်

#### စာရွက်စာတမ်း အပ်ဒိတ်များ
- **README.md**: MCP Specification 2025-11-25 ဗားရှင်းအညွှန်းနှင့် ရက်စွဲအခြေခံ ဗားရှင်း တင်ပြချက် စာသား ထည့်သွင်းသည်
- **study_guide.md**: Core Concepts အပိုင်းတွင် Tasks နှင့် Tool Annotations ပါရှိစေရန် သင်တန်းလမ်းကြောင်း မြေပုံ အပ်ဒိတ်လုပ်ပြီး စာရွက်အချိန်စာရင်း ပြင်ဆင်သည်

#### သတ်မှတ်ချက်လိုက်နာမှု စစ်ဆေးခြင်း
- **Protocol ဗားရှင်း**: MCP Specification 2025-11-25 လက္ရှိ ဗားရှင်းနှင့် ကိုက်ညီရန် စာရွက်စာတမ်းများအားလုံး အတည်ပြုထားသည်
- **အင်ဂျင်နီယာနှင့် ကိုက်ညီမှု**: ၂-အလွှာ အင်ဂျင်နီယာ (ဒေတာအလွှာ + သယ်ယူပို့ဆောင်မှုအလွှာ) စာရွက်စာတမ်း တိကျမှု စိစစ်မှု ပြုလုပ်
- **Primitive စာရွက်စာတမ်းများ**: Server Primitives (Resources, Prompts, Tools) နှင့် Client Primitives (Sampling, Elicitation, Logging, Roots) တိကျစွာ စစ်ဆေးသည်
- **သယ်ယူပို့ဆောင်မှု နည်းစနစ်များ**: STDIO နှင့် Streamable HTTP သယ်ယူပို့ဆောင်မှု စာရွက်စာတမ်း တိကျမှု စစ်ဆေးသည်
- **လုံခြုံရေး ညွှန်ကြားချက်များ**: MCP လုံခြုံရေး အကောင်းဆုံး လုပ်ထုံးလုပ်နည်းများနှင့် ကိုက်ညီမှု အတည်ပြုထားသည်

#### MCP 2025-11-25 အဓိက အင်္ဂါရပ်များ စာရွက်စာတမ်း ထည့်သွင်းခြင်း
- **OpenID Connect ရှာဖွေမှု**: OIDC မှတဆင့် အကောင့် ဆာဗာ ရှာဖွေမှု
- **OAuth Client ID Metadata စာရွက်စာတမ်းများ**: Client စာရင်းသွင်းမှု အသုံးပြု နည်းပြချက်
- **JSON Schema 2020-12**: MCP schema တွင် အသုံးပြုသော မူရင်း ဒိုင်လက်
- **SDK Tiering System**: SDK Feature ပံ့ပိုးမှုနှင့် ပြုပြင်ထိန်းသိမ်းမှု အတွက် စနစ်တကျ လိုအပ်ချက်များ ဖော်ပြထားသည်
- **အစိုးရအဖွဲ့ အဖွဲ့စည်းပုံ**: MCP အစိုးရအဖွဲ့တွင် အလုပ်အဖွဲ့များနှင့် စိတ်ဝင်စားဖွဲ့များ ဖြစ်စဉ် ဖော်ပြချက်

- **စံချိန်လိုက်နာမှု**: MCP လုံခြုံရေးလိုအပ်ချက် အားလုံး (အမြဲလိုအပ်/မလိုအပ်) ကို နောက်ဆုံးစံချိန်နှိုင်းမှုနှင့်ကိုက်ညီစေရန် အာမခံ
- **အရင်းအမြစ်ရက်စွဲနောက်ဆုံးရေး**: Microsoft စာရွက်စာတမ်းများ၊ လုံခြုံရေးစံနှုန်းများနှင့်အကောင်အထည်ဖော်လမ်းညွှန်များသို့ ပြင်ဆင်ထားသော အပြင်ဆင်လင့်ခ်များအားလုံးကို စစ်ဆေး
- **အကောင်းဆုံးလေ့ကျင့်မှု ကာကွယ်မှု**: အတည်ပြုရန် authentication, authorization, AI သက်ဆိုင်စွမ်းအားများ၊ supply chain security နှင့် စီးပွားရေးသည် pattern များစွာကို ကာကွယ်ထား

## အောက်တိုဘာ 6, 2025

### စတင်မှု ဌာန ရိုးရှင်းပြီးခွဲခြား သောက်သစ်မှုများ အပိုင်းချဲ့ထွင်ခြင်း

#### အဆင့်မြင့် Server သုံးချက် (03-GettingStarted/10-advanced)
- **အပိုင်းအသစ် ထည့်သွင်းခြင်း**: MCP server အသုံးပြုမှုမြင့်မားပြီး လက်တွေ့ထားဆောင်ရန် လမ်းညွှန်မှု အသီးသီးမပါဝင်ပါသည်၊ ရိုးရိုး server နှင့် နိမ့်အဆင့် server စုစည်းမှု
  - **ပုံစံများ**: Python နှင့် TypeScript မှ ကိုးကားထားသည့် နမူနာကုဒ်များဖြင့် စုံလင်သော ရိုးရိုးနှင့် နိမ့်အဆင့် server နှိုင်းယှဉ်မှု
  - **Handler-အခြေပြု ဒီဇိုင်း**: scalable နှင့် flexible server အကောင်အထည်ဖော်ရန် tool/resource/prompt ကို handler မှတစ်ဆင့် စီမံခန့်ခွဲခြင်း ရှင်းလင်းချက်
  - **လက်တွေ့ pattern များ**: နိမ့်အဆင့် server pattern များကို အဆင့်မြှင့်အင်္ဂါရပ်များနှင့် အဆောက်အဦးများအတွက် လက်တွေ့အသုံးများသော နေရာများပေါ်လွန်းမှု

#### ရိုးရှင်းသော Authentication (03-GettingStarted/11-simple-auth)
- **အပိုင်းအသစ် ထည့်သွင်းခြင်း**: MCP server များတွင် ရိုးရှင်းသော authentication ထည့်သွင်းခြင်းအတွက် အဆင့်ဆင့် လမ်းညွှန်ချက်
  - **Auth အယူအဆ**: authentication နှင့် authorization တို့၏ ဖွင့်ဆိုခြင်း လက်တွေ့ခံယူစိတ်များ
  - **Basic Auth အကောင်အထည်**: Python (Starlette) နှင့် TypeScript (Express) များတွင် middleware အခြေပြု authentication ပုံစံ များ နှင့် နမူနာကုဒ်များ
  - **အဆင့်မြင့်လုံခြုံရေးသို့ တိုးတက်ခြင်း**: ရိုးရှင်းသော auth မှစ၍ OAuth 2.1 နှင့် RBAC သို့ တိုးတက်မှုအတွက် ညွှန်ကြားချက်များ၊ အဆင့်မြင့်လုံခြုံရေး module များကို ကိုးကားထား

ဤထည့်သွင်းမှုများသည် MCP server အကောင်အထည်ဖော်ခြင်းတွင် ပိုမိုခိုင်မာ၊ လုံခြုံပြီး၊ လွယ်ကူသော ဆက်စပ်မှုများရှိအောင် လက်တွေ့ လမ်းညွှန်ချက်များကိုပေးပြီး အခြေခံအယူအဆများနှင့် အဆင့်မြင့်ထုတ်လုပ်မှု pattern များကို ဆက်သွယ်ပေးသည်။

#### MCP Custom Transports (05-AdvancedTopics/mcp-transport/) - အဆင့်မြင့် အကောင်အထည်ဖော်လမ်းညွှန်အသစ်
- **README.md**: Custom MCP သယ်ယူပို့ဆောင်ရေး မော်ဒယ်များအတွက် ပြည့်စုံသော အကောင်အထည်ဖော်လမ်းညွှန်
  - **Azure Event Grid သယ်ယူပို့ဆောင်မှု**: Serverless event-driven သယ်ယူပို့ဆောင်မှု အကောင်အထည်ဖော်ခြင်း
    - C#, TypeScript နှင့် Python နမူနာများ၊ Azure Functions ပေါင်းစည်းမှုပါဝင်
    - Scalable MCP ဖြေရှင်းမှုအတွက် Event-driven architecture pattern များ
    - Webhook လက်ခံသူများနှင့် ပို့ဆောင်မှုအခြေပြု စာတိုက်ဆက်သွယ်မှု တွင် အာရုံစိုက်မှု
  - **Azure Event Hubs သယ်ယူပို့ဆောင်မှု**: မြင့်မားသောထုတ်လွှင့်နှုန်း ဝိုင်းစည်း စာရင်းသွင်းပေးခြင်း အကောင်အထည်ဖေါ်ခြင်း
    - နိမ့် latency အတွက် အချိန်နှင့်တပြေးညီ streaming စနစ်များ
    - Partitioning နည်းလမ်းများနှင့် checkpoint ဖြတ်တောက်မှု စီမံခန့်ခွဲမှု
    - စာတိုက်စနစ်အစုလိုက်နှင့် စွမ်းဆောင်ရည် မြှင့်တင်မှု
  - **စီးပွားရေး ပေါင်းစည်းမှု pattern များ**: ထုတ်လုပ်မှုအဆင့် အေတ္တပထားဆက်စပ်ပုံများ
    - Azure Functions များအရေအတွက်၍ MCP ဖြန့်ခွဲမှု
    - နည်းလမ်းအမျိုးမျိုး ပေါင်းစည်းထားသော Hybrid သယ်ယူပို့ဆောင်မှု ဖွဲ့စည်းမှုများ
    - စာတိုက်ကြည့်ရှုမှု၊ ယုံကြည်စိတ်ချရမှုနှင့် အမှားကိုင်တွယ်မှု နည်းလမ်းများ
  - **လုံခြုံရေးနှင့် စောင့်ကြည့်မှု**: Azure Key Vault ပေါင်းစည်းမှုနှင့် ကြည့်ရှုနိုင်မှု pattern များ
    - Managed identity အတည်ပြုမှုနှင့် အနည်းဆုံးခွင့်ပြုမှု ဝင်ရောက်မှု
    - Application Insights telemetry နှင့် စွမ်းဆောင်ရည် စောင့်ကြည့်မှု
    - Circuit breaker များနှင့် အမှားခံနိုင်ရည် pattern များ
  - **စမ်းသပ်မှု Framework များ**: Custom transport များအတွက် စမ်းသပ်မှုအတွက် စနစ်တကျ ရှာဖွေရေးများ
    - test double များနှင့် mocking framework များဖြင့် unit စမ်းသပ်ခြင်း
    - Azure Test Containers ဖြင့် ပေါင်းစည်း စမ်းသပ်ခြင်း
    - စွမ်းဆောင်ရည်နှင့် အလုပ်စွမ်းဆောင်မှု စမ်းသပ်ခြင်း ထည့်သွင်းရန်သူ

#### Context Engineering (05-AdvancedTopics/mcp-contextengineering/) - မာနအသစ် AI ဌာန
- **README.md**: တိုးတက်လာသော Discipline အသစ်တစ်ရပ်အနေဖြင့် context engineering ကို ပြည့်စုံစွာ ရှင်းလင်းထုတ်ဖေါ်ပြုစုခြင်း
  - **အဓိကသဘောတရားများ**: context များအပြည့်နားလည်ချက်၊ လုပ်ဆောင်ချက်ဆုံးဖြတ်မှု အသိပညာနှင့် context ပတ်ရံထားမှုစီမံခန့်ခွဲမှု
  - **MCP ပရိုတိုကော နှိုင်းယှဉ်ချက်**: MCP ဒီဇိုင်းသည် context engineering စိန်ခေါ်မှုများကို မည်သို့ ဖြေရှင်းထားသည်
    - context ပတ်ဝန်းကျင် ကန့်သတ်ချက်များနှင့် progressive loading နည်းလမ်းများ
    - သက်ဆိုင်မှုတိုးတက်ရေးနှင့် dynamic context ရယူခြင်း
    - Multi-modal context ကိုင်တွယ်ခြင်းနှင့် လုံခြုံရေးအချက်များ
  - **အကောင်အထည်ဖော် နည်းလမ်းများ**: single-threaded နှင့် multi-agent architecture များ
    - context ချပ်ခြင်းနှင့် ဦးစားပေးမှု နည်းလမ်းများ
    - progressive context loading နှင့် compression နည်းလမ်းများ
    - အလွှာခွဲ context နည်းလမ်းများနှင့် ရယူခြင်း အဆင်ပြေမှု မြှင့်တင်မှု
  - **တိုင်းတာရေး Framework**: context အသုံးပြုအကျိုးသက်ရောက်မှု မီထရစ်အသစ်များ
    - input ထိရောက်မှု၊ စွမ်းဆောင်ရည်၊ အရည်အသွေးနှင့် အသုံးပြုသူ အတွေ့အကြုံ အချက်များ
    - context optimization နှင့် ပညာရပ် အသစ်များ ဆန်းစစ်ခြင်း
    - ဖောက်ပြန်မှု ခွဲခြမ်းစိတ်ဖြာခြင်းနှင့် တိုးတက်မှုပုံစံများ

    - ဖြစ်ပွားမှုတုံ့ပြန်မှုနှင့် ပြန်လည်ကယ်ဆယ်ခြင်း အလိုအလျောက်စွမ်းရည်များ
    - လိုက်လျောညီထွေမှုနှင့် အုပ်ချုပ်မှု စည်းမျဉ်းများနှင့် ကိုက်ညီမှု
    - ကြီးစိုးမှုလုံခြုံရေးထိန်းချုပ်မှုများ ဖြင့် သုညယုံကြည်မှု ဖွဲ့စည်းပုံ
    - Microsoft လုံခြုံရေး ပတ်ဝန်းကျင် ပေါင်းစည်းမှု အပြည့်အစုံ

  - **Microsoft လုံခြုံရေး ဖြေရှင်းချက်များ**: Prompt Shields, Azure Content Safety, Entra ID, GitHub Advanced Security အတွက် ပိုမိုကောင်းမွန်သော ပေါင်းစည်းမှု လမ်းညွှန်ချက်များ

#### အဆင့်မြင့်လုံခြုံရေး ထိန်းချုပ်မှုများ (02-Security/) - လုပ်ငန်းအဆင့် အကောင်အထည်ဖော်မှု
- **MCP-SECURITY-CONTROLS-2025.md**: လုပ်ငန်းအဆင့် လုံခြုံရေး ဖွဲ့စည်းပုံ အပြည့်အစုံနှင့် ပြန်လည်ထူထောင်ခြင်း
  - **လုံခြုံရေး နယ်ပယ် ၉ ခု**: မူလထိန်းချုပ်မှုများမှ လုပ်ငန်းအဆင့် အသေးစိတ် ဖွဲ့စည်းပုံသို့တိုးချဲ့
    - Microsoft Entra ID ပြန်လည်ပေါင်းစည်းသည့် အဆင့်မြင့် အတည်ပြုခြင်းနှင့် ခွင့်ပြုခြင်း
    - တိုကင် လုံခြုံရေးနှင့် မလိုလားအပ်သော လမ်းကြောင်း တားဆီးမှုများ စစ်ဆေးခြင်းပြည့်စုံ
    - အစည်းအဝေး လုံခြုံရေး ထိန်းချုပ်မှုများ နှင့် တိုက်ခိုက်မှု တားဆီးခြင်း
    - AI-သီးသန့် လုံခြုံရေး ထိန်းချုပ်မှုများ၊ prompt injection နှင့် ကိရိယာမှောက်မှု တားဆီးမှု
    - Confused Deputy မှတဆင့် တိုက်ခိုက်မှုကို OAuth proxy လုံခြုံရေးဖြင့် ကာကွယ်ခြင်း
    - ကိရိယာ အကောင်အထည်ဖော်ခြင်း လုံခြုံရေး sandbox နှင့် isolation ဖြင့် ကာကွယ်ခြင်း
    - ပစ္စည်းစီးဆင်းမှု လုံခြုံရေး ထိန်းချုပ်မှုများ စစ်ဆေးခြင်း
    - စောင့်ကြည့်ခြင်းနှင့် စိစစ်ခြင်း ထိန်းချုပ်မှုများ SIEM ပေါင်းစည်းရေးဖြင့်
    - ဖြစ်ပွားမှုတုံ့ပြန်မှုနှင့် ပြန်လည်ကယ်ဆယ်ခြင်း အလိုအလျောက်စွမ်းရည်များ
  - **အကောင်အထည်ဖော်မှု နမူနာများ**: YAML ဖောင့်ကွန်ဖစ်ရေးရှင်းများနှင့် ကုဒ်နမူနာအသေးစိတ် ထည့်သွင်း
  - **Microsoft ဖြေရှင်းချက်များ ပေါင်းစည်းမှု**: Azure လုံခြုံရေး ဝန်ဆောင်မှုများ, GitHub Advanced Security နှင့် လုပ်ငန်းအိုင်ဒင်တစ်စနစ်စီမံမှု အပြည့်အစုံဖုံးလွှမ်းမှု

#### အဆင့်မြင့် အကြောင်းအရာ လုံခြုံရေး (05-AdvancedTopics/mcp-security/) - ထုတ်လုပ်မှုအဆင်သင့် အကောင်အထည်ဖော်မှု

  - ကွန်ပြူတာများကို MCP ဖောက်သည် ချိတ်ဆက်မှုများကို ဦးစားပေး စီမံခန့်ခွဲသော AI အက်ပလီကေးရှင်းများအဖြစ် ပြတ်သားဖော်ပြ
  - ဖောက်သည်များကို ဆာဗာများနှင့် တစ်ဦးချင်းချိတ်ဆက်မှုကို ထိန်းသိမ်းသည့် ပရိုတိုကေါလ် ချိတ်ဆက်သူများအနေနဲ့ ဖော်ပြ
  - ဆာဗာများကို နေရာဒေသနှင့် ဝေးလံအေရာင်း များအတွက် ပြန်လည်သတ်မှတ် ပြောင်းလဲ ပြင်ဆင်မှုများ ဆောင်ရွက်ထား
- **ပရိုတိုကေါလ် နည်းဗျူဟာစနစ် လုံးဝပြောင်းလဲခြင်း**: ဆာဗာနှင့် ဖောက်သည်မူလအတိုင်း အခြေခံသဏ္ဍာန် ပြုပြင်ပြောင်းလဲခြင်း
  - ဆာဗာ မူလအမှုခံများ: အရင်းအမြစ်အချက်အလက်များ (ဒေတာ ရင်းမြစ်များ), ပုံဆွဲများ (ပုံစံများ), ကိရိယာများ (အလုပ်လုပ်နိုင်သော လုပ်ဆောင်ချက်များ) နဲ့ ပေါင်းစည်းပြီး အသေးစိတ်ရှင်းလင်းချက် နှင့် နမူနာများ
  - ဖောက်သည် မူလအမှုခံများ: စမ်းသပ်မှု (LLM ဖြည့်စည်းမှုများ), လျှောက်လွှာခြင်း (အသုံးပြုသူ အချက်အလက် ထည့်သွင်းမှု), မှတ်တမ်းတင်မှု (အမှားရှာဖွေမှု/ စောင့်ကြည့်မှု)
  - ပစ္စည်းရှာဖွေရေး (`*/list`), ရယူမှု (`*/get`), နှင့် အကောင်အထည်ဖော်မှု (`*/call`) နည်းဗျူဟာများကို ယနေ့ MCP ပရိုတိုကေါလ် စံနှုန်းအား ဖြည့်စွက်
- **ပရိုတိုကေါလ် ဖွဲ့စည်းပုံ**: နှစ် ဆင့်ဖွဲ့စည်းပုံ မော်ဒယ် တီထွင်
  - ဒေတာအလွှာ: JSON-RPC 2.0 အခြေခံ ဖြင့် ဘဝထက်စစ်ဆေးမှုနှင့် မူလအမှုခံများ ပါဝင်
  - သယ်ယူပို့ဆောင်ရေးအလွှာ: STDIO (ဒေသတွင်း) နှင့် Streamable HTTP နှင့် SSE (ဝေးကွာ) သယ်ယူပို့ဆောင်ရေးစနစ်များ
- **လုံခြုံရေး ဖွဲ့စည်းပုံ**: အသုံးပြုသူ သဘောတူညီမှု စာရွက်စာတမ်း တိကျခြင်း ၊ ဒေတာ ကိုယ်ပိုင်မှု ကာကွယ်မှု ၊ ကိရိယာ အကောင်အထည်ဖော်မှု လုံခြုံမှု ၊ သယ်ယူပို့ဆောင်ရေးအလွှာ လုံခြုံမှုတို့ ပါဝင်သော လုံခြုံရေး နည်းဗျူဟာအပြည့်အစုံ
- **ဆက်သွယ်ရေး ပုံစံများ**: ပရိုတိုကေါလ် သတင်းပို့စာများကို စတင်မှု၊ ရှာဖွေရေး၊ အကောင်အထည်ဖော်မှု နှင့် အသိပေးမှု လှုပ်ရှားမှုများ အသစ် နိုင်ငံတကာ လိုက်နာမှု ပြုလုပ်ထားသည်
- **ကုဒ်နမူနာများ**: MCP SDK စတိုင် နာမည်အလိုက် .NET, Java, Python, JavaScript များ အတွက် နမူနာများ ပြန်လည်ပြင်ဆင်ထားသည်

#### လုံခြုံရေး (02-Security/) - လုံခြုံရေး ပြုပြင်ပြောင်းလဲမှု ပြည့်စုံ
- **စံချိန်စံညွှန်း ကိုက်ညီမှု**: MCP စံချိန်စံညွှန်း 2025-06-18 လုံခြုံရေး တာဝန်ခံချက်များအတိုင်း အပြည့်အစုံ ကိုက်ညီမှု
- **အတည်ပြုခွင့် တိုးတက်မှု**: စိတ်ကြိုက် OAuth ဆာဗာများမှ အပြင်မှ အိုင်ဒင်တစ်ဖိုင် ဒေသခံမှ ကူးပြောင်းသည့် Microsoft Entra ID အား စာတမ်းတွင် ဖော်ပြထားသည်
- **AI သီးသန့် အန္တရာယ် ခြုံငုံအကဲဖြတ်မှု**: ခေတ်မီ AI တိုက်ခိုက်မှု နယ်ပယ်များကို ပိုမိုသေချာလေ့လာစီမံမှု
  - အမှန်တကယ် ကြုံတွေ့နေရသော prompt injection တိုက်ခိုက်မှု တိကျသော အကြောင်းအရာများဖြင့်
  - ကိရိယာ မှောက်မှုနည်းလမ်းများနှင့် "rug pull" တိုက်ခိုက်မှု နမူနာများ
  - ပတ်ဝန်းကျင်ဝင်းဒို စွန့်စားခြင်းနှင့် မော်ဒယ် ရှုပ်ထွေးမှု တိုက်ခိုက်မှုများ
- **Microsoft AI လုံခြုံရေး ဖြေရှင်းချက်များ**: Microsoft လုံခြုံရေး ပတ်ဝန်းကျင် အပြည့်အစုံဖုံးလွှမ်းမှု
  - AI Prompt Shields ပါဝင်သော ထိရောက်သော ရှာဖွေမှု၊ ထင်ဟပ်ပြသမှု နှင့် ခွဲခြားနည်းစနစ်များ
  - Azure Content Safety ပေါင်းစည်းရေး လမ်းညွှန်ချက်များ
  - GitHub Advanced Security ပစ္စည်းစီးဆင်းမှုကာကွယ်မှု
- **အဆင့်မြင့် အန္တရာယ်လျှော့ချခြင်း**: အရေးပါသော လုံခြုံရေး ထိန်းချုပ်မှုများ
  - MCP အထူး တိုက်ခိုက်မှု ပညာရပ်နှင့် ဆက်စပ် Roleback (session hijacking) ကာကွယ်ရေး
  - MCP Proxy တွင် ဖြစ်ပေါ်နိုင်သည့် Confused Deputy ချို့ယွင်းချက်များ နှင့် သဘောတူညီချက် ပြည့်စုံမှု
  - Token passthrough ကျဆင်းမှုများနှင့် လိုအပ်သော သေချာစစ်ဆေးမှု ထိန်းချုပ်မှုများ
- **ပစ္စည်းစီးဆင်းမှု လုံခြုံရေး**: အခြေခံ မော်ဒယ်များ, embeddings ဝန်ဆောင်မှုများ, ပတ်ဝန်းကျင် ပံ့ပိုးသူများ နှင့် တတိယ အချက်အလက် API များ အပြည့်အစုံ တိုးချဲ့ ခွဲခြားထားသည်
- **တည်နေရာလုံခြုံရေး**: သုညယုံကြည်မှု ဖွဲ့စည်းပုံနှင့် Microsoft လုံခြုံရေး ပတ်ဝန်းကျင်များနှင့် ပိုမိုကောင်းမွန်သော ပေါင်းစည်းမှု
- **ရင်းမြစ် စီမံခန့်ခွဲမှု**: အမျိုးအစားအလိုက် (တရားဝင် စာရွက်စာတမ်းများ၊ စံချိန်စံညွှန်းများ၊ သုတေသန၊ Microsoft ဖြေရှင်းချက်များ၊ အကောင်အထည်ဖော်လမ်းညွှန်များ) အလိုက် စုစည်းထားသည်

### စာရွက်စာတမ်း အရည်အသွေး တိုးတက်မှုများ
- **ဖွဲ့စည်းထားသော သင်ယူမှု ရည်မှန်းချက်များ**: အထူးသဖြင့် လုပ်ဆောင်ရန် အသေးစိတ် ရလဒ်ဖြစ်စေသော ရည်မှန်းချက်များ တိုးတက်အပ်ဒိတ်ပြုလုပ်
- **ဆက်စပ် အကြောင်းအရာများအတွက် ချိတ်ဆက်မှုများ**: လုံခြုံရေးနှင့် အဓိကအကြောင်းအရာများ တူညီသော သက်ဆိုင်မှုတွေ ကြားတွင် ချိတ်ဆက်ထားသည်
- **လက်ရှိ သတင်းအချက်အလက်များ**: ရက်စွဲများနှင့် စံချိန်စံညွှန်းလင့်ခ်များအားလုံး လက်ရှိ စံများအတိုင်း အပ်ဒိတ်ပြုလုပ်ထားသည်
- **အကောင်အထည်ဖော်လမ်းညွှန်ချက်များ**: နှစ်ဦးမလိုက်ဖက်သော ဌာနပေါင်းများစွာတွင် လုပ်ဆောင်ရန် အသေးစိတ် ရွေးချယ်လမ်းညွှန်ချက်များ ဖြည့်စွက်ထားသည်

- ပံ့ပိုးမှု နမူနာများကို ပိုမိုတိကျမြင့်မားသော ကုဒ်နမူနာများဖြင့် တိုးမြှင့်ထားသည်

### အဆင့်မြင့် အကြောင်းအရာများ (05-AdvancedTopics/)
- အထူးပြု အကြောင်းအရာ ဖိုင်တွဲများကို နာမည်အရ အစဉ်တကျ စုပုံတင်ထားသည်
- context engineering ပစ္စည်းများ နှင့် နမူနာများ ထည့်သွင်းထားသည်
- Foundry agent ပေါင်းစည်းမှု စာရွက်စာတမ်း ထည့်သွင်းခဲ့သည်
- Entra ID လုံခြုံရေး ပေါင်းစည်းမှုစာရွက်စာတမ်းများ တိုးချဲ့ထားသည်
