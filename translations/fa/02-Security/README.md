# امنیت MCP: حفاظت جامع برای سیستم‌های هوش مصنوعی

[![روش‌های بهتر امنیت MCP](../../../translated_images/fa/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(برای مشاهده ویدیو این درس، روی تصویر بالا کلیک کنید)_

امنیت بنیان طراحی سیستم‌های هوش مصنوعی است، که به همین دلیل ما آن را به عنوان بخش دوم خود در اولویت قرار داده‌ایم. این با اصل **امنیت بر اساس طراحی** مایکروسافت از [ابتکار آینده امن](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) همسو است.

پروتکل زمینه مدل (MCP) قابلیت‌های قدرتمند جدیدی را به برنامه‌های مبتنی بر هوش مصنوعی می‌آورد و در عین حال چالش‌های امنیتی منحصر به فردی که فراتر از خطرات نرم‌افزاری سنتی هستند را ایجاد می‌کند. سیستم‌های MCP با نگرانی‌های امنیتی رایج (کدنویسی ایمن، حداقل حق دسترسی، امنیت زنجیره تامین) و تهدیدات مخصوص هوش مصنوعی مانند تزریق فرمان، مسمومیت ابزار، ربودن نشست، حملات سردرگم کننده نماینده، آسیب‌پذیری‌های عبور توکن و تغییرات پویا در قابلیت‌ها روبرو هستند.

این درس مهم‌ترین ریسک‌های امنیتی در پیاده‌سازی‌های MCP را بررسی می‌کند—شامل احراز هویت، مجوزدهی، مجوزهای اضافی، تزریق فرمان غیرمستقیم، امنیت نشست، مشکلات نماینده سردرگم، مدیریت توکن و آسیب‌پذیری‌های زنجیره تامین. شما کنترل‌های عملی و بهترین روش‌ها برای کاهش این خطرات را یاد می‌گیرید و در عین حال از راه‌حل‌های مایکروسافت مانند سپرهای فرمان، ایمنی محتوای آژور و امنیت پیشرفته گیت‌هاب برای تقویت استقرار MCP خود بهره‌مند خواهید شد.

## اهداف یادگیری

تا پایان این درس، شما قادر خواهید بود:

- **شناسایی تهدیدات خاص MCP**: شناخت ریسک‌های امنیتی منحصر به MCP شامل تزریق فرمان، مسمومیت ابزار، مجوزهای اضافی، ربودن نشست، مشکلات نماینده سردرگم، آسیب‌پذیری‌های عبور توکن و ریسک‌های زنجیره تامین
- **اعمال کنترل‌های امنیتی**: اجرای راهکارهای مؤثر شامل احراز هویت قوی، حداقل دسترسی مجاز، مدیریت ایمن توکن، کنترل‌های امنیت نشست و راستی‌آزمایی زنجیره تامین
- **بهره‌گیری از راهکارهای امنیتی مایکروسافت**: درک و استقرار سپرهای فرمان مایکروسافت، ایمنی محتوای آژور و امنیت پیشرفته گیت‌هاب برای حفاظت از بار کاری MCP
- **اعتبارسنجی امنیت ابزار**: شناخت اهمیت اعتبارسنجی فراداده ابزار، پایش تغییرات پویا و دفاع در برابر حملات تزریق فرمان غیرمستقیم
- **ادغام بهترین روش‌ها**: ترکیب اصول پایه امنیتی تثبیت شده (کدنویسی ایمن، سخت‌کردن سرور، اعتماد صفر) با کنترل‌های خاص MCP برای محافظت جامع

# معماری و کنترل‌های امنیتی MCP

پیاده‌سازی‌های مدرن MCP نیازمند رویکردهای امنیتی لایه‌ای هستند که هم امنیت نرم‌افزار سنتی و هم تهدیدات مخصوص هوش مصنوعی را پوشش دهند. مشخصات سریع‌التطور MCP به تکامل کنترل‌های امنیتی آن ادامه می‌دهد که امکان ادغام بهتر با معماری‌های امنیت سازمانی و بهترین روش‌های تثبیت شده را فراهم می‌کند.

تحقیقات گزارش [مدیریت دیجیتال مایکروسافت](https://aka.ms/mddr) نشان می‌دهد که **98٪ از نفوذهای گزارش شده توسط بهداشت امنیتی قوی جلوگیری می‌شوند**. مؤثرترین راهکار حفاظتی ترکیبی از تمرین‌های پایه امنیتی با کنترل‌های خاص MCP است—اقدامات امنیتی پایه اثبات‌شده مهم‌ترین تأثیر را در کاهش ریسک کلی ایفا می‌کنند.

## چشم‌انداز کنونی امنیت

> **توجه:** این اطلاعات مطابق با استانداردهای امنیتی MCP در تاریخ **5 فوریه 2026** و بر اساس **مشخصات MCP نسخه 2025-11-25** است. پروتکل MCP به سرعت در حال تکامل است و نسخه‌های آینده ممکن است الگوهای احراز هویت جدید و کنترل‌های پیشرفته‌تری ارائه دهند. همواره به [مشخصات MCP](https://spec.modelcontextprotocol.io/)، [مخزن گیت‌هاب MCP](https://github.com/modelcontextprotocol) و [مستندات بهترین شیوه‌های امنیتی](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) برای راهنمایی‌های به‌روز مراجعه کنید.

> **نگاهی به آینده:** نسخه کاندید انتشار `2026-07-28` مجوزدهی را تقویت می‌کند — کلاینت‌ها باید پارامتر `iss` در پاسخ‌های مجوزدهی را اعتبارسنجی کنند (RFC 9207)، نوع برنامه OpenID Connect را در ثبت کلاینت دینامیک اعلام کنند و گواهی‌نامه‌های ثبت‌شده را به سرورمجوزدهی صادرکننده متصل کنند. لیست کامل SEPs مجوزدهی را در [چه تغییراتی در MCP است: نسخه کاندید انتشار 2026-07-28](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) ببینید.

- **کد عملیاتی دریافت کنید**: با پیاده‌سازی‌های کاری و تست‌شده کار خود را به پایان ببرید

### تکامل احراز هویت MCP

مشخصات MCP در رویکرد خود به احراز هویت و مجوزدهی بسیار تکامل یافته است:

- **رویکرد اولیه**: مشخصات اولیه توسعه‌دهندگان را ملزم به ساخت سرورهای احراز هویت سفارشی می‌کرد، در حالی که سرورهای MCP به‌عنوان سرورهای مجوزدهی OAuth 2.0 عمل می‌کردند که مستقیم احراز هویت کاربر را مدیریت می‌کردند
- **استاندارد فعلی (2025-11-25)**: مشخصات به‌روزشده اجازه می‌دهد سرورهای MCP احراز هویت را به ارائه‌دهندگان هویت خارجی (مانند Microsoft Entra ID) تفویض کنند، که موضع امنیتی را بهبود داده و پیچیدگی پیاده‌سازی را کاهش می‌دهد
- **امنیت لایه انتقال**: پشتیبانی تقویت‌شده برای مکانیزم‌های انتقال ایمن با الگوهای احراز هویت مناسب برای اتصالات محلی (STDIO) و راه دور (HTTP قابل جریان)

## امنیت احراز هویت و مجوزدهی

### چالش‌های امنیتی کنونی

پیاده‌سازی‌های مدرن MCP با چند چالش احراز هویت و مجوزدهی مواجه هستند:

### ریسک‌ها و مسیرهای تهدید

- **منطق مجوزدهی اشتباه پیکربندی شده**: اجرای ناقص مجوزدهی در سرورهای MCP می‌تواند داده‌های حساس را افشا کرده و کنترل‌های دسترسی را به ‌اشتباه اعمال کند
- **نفوذ توکن OAuth**: سرقت توکن سرور محلی MCP به مهاجم امکان جعل سرورها و دسترسی به سرویس‌های پایین‌دستی را می‌دهد
- **آسیب‌پذیری‌های عبور توکن**: مدیریت نادرست توکن منجر به دور زدن کنترل‌های امنیتی و خلأهای حسابرسی می‌شود
- **مجوزهای اضافی**: سرورهای MCP با مجوز بیش از حد، اصول حداقل دسترسی را نقض کرده و سطوح حمله را گسترش می‌دهند

#### عبور توکن: یک ضدالگوی بحرانی

**عبور توکن به صراحت در مشخصات مجوزدهی MCP جاری ممنوع است** به دلیل پیامدهای امنیتی شدید:

##### دور زدن کنترل‌های امنیتی
- سرورهای MCP و APIهای پایین‌دستی کنترل‌های امنیتی حیاتی (محدودیت نرخ، اعتبارسنجی درخواست، پایش ترافیک) را پیاده‌سازی می‌کنند که به اعتبارسنجی درست توکن متکی است
- استفاده مستقیم کلاینت‌ها از توکن به API این محافظت‌های اساسی را دور زده و معماری امنیت را تضعیف می‌کند

##### چالش‌های حسابرسی و پاسخگویی  
- سرورهای MCP نمی‌توانند بین کلاینت‌هایی که از توکن‌های صادرشده بالادستی استفاده می‌کنند تمایز قائل شوند، که زنجیره‌های حسابرسی را می‌شکند
- گزارش‌های سرور منابع پایین‌دستی مبدأ نادرست درخواست‌ها را به جای واسطه‌های واقعی سرور MCP نشان می‌دهند
- بررسی حادثه و ممیزی انطباق به طور قابل توجهی دشوارتر می‌شود

##### خطرات استخراج داده
- ادعاهای توکن تأییدنشده به بازیگران مخرب با توکن‌های سرقت‌شده اجازه می‌دهد از سرورهای MCP به عنوان پروکسی برای استخراج داده استفاده کنند
- نقض مرز اعتماد الگوهای دسترسی غیرمجاز را ایجاد می‌کند که کنترل‌های امنیتی مد نظر را دور می‌زند

##### مسیرهای حمله چندسرویسی
- توکن‌های به خطر افتاده که توسط چند سرویس پذیرفته می‌شوند امکان حرکت جانبی در سیستم‌های متصل را فراهم می‌کنند
- فرض‌های اعتماد بین سرویس‌ها ممکن است هنگام تأیید نشدن مبدأ توکن نقض شود

### کنترل‌های امنیتی و کاهش خطر

**الزامات حیاتی امنیتی:**

> **الزامی**: سرورهای MCP **نباید** هیچ توکنی را قبول کنند که صراحتاً برای سرور MCP صادر نشده باشد

#### کنترل‌های احراز هویت و مجوزدهی

- **بازبینی دقیق مجوزدهی**: انجام ممیزی جامع منطق مجوزدهی سرور MCP برای اطمینان از اینکه تنها کاربران و کلاینت‌های مدنظر به منابع حساس دسترسی دارند
  - **راهنمای پیاده‌سازی**: [مدیریت API آژور به عنوان دروازه احراز هویت برای سرورهای MCP](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ادغام هویت**: [استفاده از Microsoft Entra ID برای احراز هویت سرور MCP](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **مدیریت ایمن توکن**: اجرای [بهترین روش‌های اعتبارسنجی توکن و چرخه عمر مایکروسافت](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens)
  - اعتبارسنجی ادعاهای مخاطب توکن مطابق با هویت سرور MCP
  - اجرای سیاست‌های چرخش و انقضای مناسب توکن
  - جلوگیری از حملات تکرار توکن و استفاده غیرمجاز

- **ذخیره‌سازی محافظت شده توکن**: ذخیره‌سازی ایمن توکن با رمزگذاری هم در حالت استراحت و هم در انتقال
  - **بهترین روش‌ها**: [راهنمای ذخیره‌سازی و رمزگذاری ایمن توکن](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### پیاده‌سازی کنترل دسترسی

- **اصل حداقل امتیاز**: فقط حداقل مجوزهای لازم برای عملکرد مدنظر به سرورهای MCP اعطا شود
  - بازبینی و به‌روزرسانی منظم مجوزها برای جلوگیری از افزایش بی‌رویه امتیازات
  - **مستندات مایکروسافت**: [دسترسی با حداقل مجوز ایمن](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **کنترل دسترسی مبتنی بر نقش (RBAC)**: پیاده‌سازی تخصیص نقش دقیق
  - تعیین دقیق حوزه نقش‌ها برای منابع و اقدامات مشخص
  - اجتناب از مجوزهای گسترده یا غیرضروری که سطوح حمله را افزایش می‌دهد

- **نظارت مستمر مجوزها**: اجرای ممیزی و پایش دسترسی به صورت مداوم
  - پایش الگوهای استفاده از مجوز برای شناسایی ناهنجاری‌ها
  - رفع سریع مجوزهای اضافی یا استفاده نشده

## تهدیدات امنیتی مخصوص هوش مصنوعی

### حملات تزریق فرمان و دستکاری ابزار

پیاده‌سازی‌های مدرن MCP با مسیرهای حمله پیشرفته مخصوص هوش مصنوعی مواجه هستند که تدابیر امنیتی سنتی به طور کامل قادر به مقابله با آن‌ها نیستند:

#### **تزریق فرمان غیرمستقیم (تزریق فرمان بین‌دامنه‌ای)**

**تزریق فرمان غیرمستقیم** یکی از بحرانی‌ترین آسیب‌پذیری‌ها در سیستم‌های هوش مصنوعی فعال‌شده با MCP است. مهاجمین دستورات مخرب را در محتوای خارجی—اسناد، صفحات وب، ایمیل‌ها یا منابع داده—نهفته می‌کنند که سیستم‌های هوش مصنوعی بعداً به عنوان دستورهای معتبر پردازش می‌کنند.

**سناریوهای حمله:**
- **تزریق مبتنی بر سند**: دستورات مخرب مخفی در اسناد پردازش‌شده که باعث اقدامات ناخواسته هوش مصنوعی می‌شوند
- **استفاده از محتوای وب**: صفحات وب آلوده حاوی فرمان‌های جاسازی شده که رفتار هوش مصنوعی را هنگام جمع‌آوری دستکاری می‌کنند
- **حملات مبتنی بر ایمیل**: فرمان‌های مخرب در ایمیل‌ها که باعث نشت اطلاعات یا انجام اعمال غیرمجاز توسط دستیارهای هوش مصنوعی می‌شود
- **آلوده‌سازی منابع داده**: پایگاه‌های داده یا APIهای آلوده که محتوای آلوده را به سیستم‌های هوش مصنوعی ارائه می‌دهند

**تأثیر دنیای واقعی**: این حملات می‌توانند باعث استخراج داده، نقض حریم خصوصی، تولید محتوای مضر و دستکاری تعاملات کاربران شوند. برای تحلیل دقیق‌تر، به [تزریق فرمان در MCP (سایمون ویلیسون)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) مراجعه کنید.

![نمودار حمله تزریق فرمان](../../../translated_images/fa/prompt-injection.ed9fbfde297ca877.webp)

#### **حملات مسمومیت ابزار**

**مسمومیت ابزار** متاداده‌ای که ابزارهای MCP را تعریف می‌کند هدف قرار می‌دهد، و از نحوه تفسیر مدل‌های زبان بزرگ ابزار و پارامترها برای تصمیم‌گیری در اجرا سوءاستفاده می‌کند.

**مکانیزم حمله:**
- **دستکاری متاداده**: مهاجمین دستورات مخرب را در توصیف ابزار، تعریف پارامترها یا نمونه‌های استفاده تزریق می‌کنند
- **دستورات نامرئی**: فرمان‌های مخفی در متاداده ابزار که توسط مدل‌های هوش مصنوعی پردازش می‌شود اما برای کاربران انسانی نامرئی است
- **تغییرات پویا ابزار ("کشیدن زیر پای کاربر")**: ابزارهایی که توسط کاربران تأیید شده‌اند بعداً برای اجرای اعمال مخرب بدون اطلاع کاربر تغییر داده می‌شوند
- **تزریق پارامتر**: محتوای مخرب جاسازی‌شده در طرح‌های پارامتر ابزار که رفتار مدل را تحت تأثیر قرار می‌دهد


**ریسک‌های سرور میزبانی‌شده**: سرورهای راه دور MCP ریسک‌های بالاتری دارند زیرا تعاریف ابزار می‌توانند پس از تأیید اولیه کاربر به‌روزرسانی شوند که سناریوهایی ایجاد می‌کند که در آنها ابزارهایی که قبلاً ایمن بودند به ابزارهای مخرب تبدیل می‌شوند. برای تحلیل کامل، به [حملات مسموم‌سازی ابزار (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) مراجعه کنید.

![Tool Injection Attack Diagram](../../../translated_images/fa/tool-injection.3b0b4a6b24de6bef.webp)

#### **بردارهای حمله اضافی هوش مصنوعی**

- **تزریق فرمان بین‌دامنه (XPIA)**: حملات پیچیده‌ای که از محتوای چند دامنه برای عبور از کنترل‌های امنیتی استفاده می‌کنند
- **تغییر دینامیک قابلیت‌ها**: تغییرات بلادرنگ در قابلیت‌های ابزار که از ارزیابی‌های اولیه امنیتی فرار می‌کنند
- **مسموم‌سازی پنجره زمینه**: حملاتی که پنجره‌های بزرگ زمینه را دستکاری می‌کنند تا دستورالعمل‌های مخرب را پنهان کنند
- **حملات سردرگمی مدل**: سوءاستفاده از محدودیت‌های مدل برای ایجاد رفتارهای غیرقابل پیش‌بینی یا ناامن


### تأثیر ریسک‌های امنیتی هوش مصنوعی

**عواقب با تاثیر بالا:**
- **خروج غیرمجاز داده‌ها**: دسترسی و سرقت غیرمجاز داده‌های حساس سازمانی یا شخصی
- **نقض حریم خصوصی**: افشای اطلاعات شناسایی شخصی (PII) و داده‌های محرمانه کسب‌وکار  
- **دستکاری سیستم**: اصلاحات ناخواسته در سیستم‌ها و فرایندهای حیاتی
- **سرقت اعتبارنامه‌ها**: به خطر انداختن توکن‌های احراز هویت و اعتبارنامه‌های سرویس
- **حرکت جانبی**: استفاده از سیستم‌های هوش مصنوعی به خطر افتاده به عنوان نقاط واسطه برای حملات شبکه گسترده‌تر

### راهکارهای امنیتی هوش مصنوعی مایکروسافت

#### **سپرهای فرمان هوش مصنوعی: محافظت پیشرفته در برابر حملات تزریق**

مایکروسافت **سپرهای فرمان هوش مصنوعی** دفاع جامعی در برابر حملات تزریق فرمان مستقیم و غیرمستقیم از طریق چندین لایه امنیتی فراهم می‌کنند:

##### **مکانیسم‌های اصلی محافظت:**

1. **شناسایی و فیلترینگ پیشرفته**
   - الگوریتم‌های یادگیری ماشین و تکنیک‌های پردازش زبان طبیعی دستورالعمل‌های مخرب در محتوای خارجی را شناسایی می‌کنند
   - تحلیل بلادرنگ اسناد، صفحات وب، ایمیل‌ها و منابع داده برای تهدیدهای جاسازی شده
   - درک زمینه‌ای الگوهای فرمان مشروع در مقابل مخرب

2. **تکنیک‌های برجسته‌سازی**  
   - تمایز بین دستورالعمل‌های سیستم مورد اعتماد و ورودی‌های خارجی احتمالا به خطر افتاده
   - روش‌های تبدیل متن که مرتبط بودن مدل را افزایش می‌دهند در حالی که محتوای مخرب را ایزوله می‌کنند
   - کمک به سیستم‌های هوش مصنوعی برای حفظ سلسله‌مراتب دستورالعمل صحیح و نادیده گرفتن فرمان‌های تزریقی

3. **سیستم‌های علامت‌گذاری و تعیین مرز**
   - تعریف مرزهای واضح بین پیام‌های سیستم مورد اعتماد و متن ورودی خارجی
   - نشانگرهای خاص که مرز بین منابع داده معتبر و غیرمعتبر را نشان می‌دهند
   - جداسازی شفاف که باعث جلوگیری از سردرگمی دستورالعمل و اجرای غیرمجاز فرمان می‌شود

4. **اطلاعات تهدید مستمر**
   - مایکروسافت به طور مستمر الگوهای حمله جدید را رصد کرده و دفاع‌ها را به‌روزرسانی می‌کند
   - شکار تهدید پیشگیرانه برای تکنیک‌ها و بردارهای حمله تزریق جدید
   - به‌روزرسانی منظم مدل‌های امنیتی برای حفظ اثربخشی در برابر تهدیدهای در حال تحول

5. **ادغام Azure Content Safety**
   - بخشی از مجموعه کامل Azure AI Content Safety
   - تشخیص اضافی برای تلاش‌های دورزدن (jailbreak)، محتوای مضر و نقض‌های سیاست امنیتی
   - کنترل‌های امنیتی یکپارچه در سرتاسر اجزای برنامه‌های هوش مصنوعی

**منابع پیاده‌سازی**: [مستندات Microsoft Prompt Shields](https://learn.microsoft.com/azure/ai-services/content-safety/concepts/jailbreak-detection)

## تهدیدات پیشرفته امنیت MCP

### آسیب‌پذیری‌های ربودن نشست (Session Hijacking)

**ربودن نشست** نمایانگر بردار حمله بحرانی در پیاده‌سازی‌های حالت‌دار MCP است که در آن اشخاص غیرمجاز شناسه‌های نشست معتبر را به دست آورده و سوءاستفاده می‌کنند تا به جای مشتریان جا زده و اقدامات غیرمجاز انجام دهند.

#### **سناریوها و ریسک‌های حمله**

- **تزریق فرمان ربودن نشست**: مهاجمان با شناسه نشست دزدیده شده، رویدادهای مخرب را به سرورهایی که حالت نشست را به اشتراک می‌گذارند تزریق می‌کنند که ممکن است منجر به اجرای اقدامات مضر یا دسترسی به داده‌های حساس شود
- **جایگزینی مستقیم**: شناسه نشست‌های دزدیده شده امکان تماس مستقیم با سرور MCP بدون احراز هویت را می‌دهد و مهاجمان را به عنوان کاربران معتبر می‌شناسد
- **جریان‌های قابل از سرگیری به خطر افتاده**: مهاجمان می‌توانند درخواست‌ها را زودتر خاتمه دهند که باعث ادامه‌ی ناخواسته درخواست‌ها توسط مشتریان با محتوای احتمالا مخرب شود

#### **کنترل‌های امنیتی مدیریت نشست**

**الزامات حیاتی:**
- **تأیید مجوز**: سرورهای MCP که مجوز را پیاده‌سازی می‌کنند **باید** همه درخواست‌های ورودی را تأیید کرده و **نباید** برای احراز هویت به نشست‌ها متکی باشند
- **تولید نشست امن**: استفاده از شناسه‌های نشست رمزنگاری‌شده، غیر قطعی و تولید شده با استفاده از مولدهای عدد تصادفی امن
- **پیوند اختصاصی کاربر**: اتصال شناسه‌های نشست به اطلاعات خاص کاربر با قالب‌هایی مانند `<user_id>:<session_id>` برای جلوگیری از سوءاستفاده نشست بین کاربران مختلف
- **مدیریت چرخه عمر نشست**: پیاده‌سازی تاریخ انقضا، چرخش و اعتبارسنجی مناسب برای محدود کردن پنجره‌های آسیب‌پذیری
- **امنیت انتقال**: استفاده اجباری از HTTPS برای تمام ارتباطات برای جلوگیری از رهگیری شناسه نشست

### مسئله نماینده گیج شده (Confused Deputy)

مسئله **نماینده گیج شده** وقتی رخ می‌دهد که سرورهای MCP به عنوان پراکسی‌های احراز هویت بین مشتریان و سرویس‌های ثالث عمل می‌کنند و فرصت‌های دورزدن مجوز از طریق سوءاستفاده از شناسه‌های کلاینت ایستا ایجاد می‌کنند.

#### **مکانیک و ریسک‌های حمله**

- **دورزدن رضایت مبتنی بر کوکی**: احراز هویت قبلی کاربر کوکی‌های رضایت ایجاد می‌کند که مهاجمان از طریق درخواست‌های غیرمجاز با URlهای بازگشتی ساختگی سوءاستفاده می‌کنند
- **سرقت کد مجوز**: کوکی‌های رضایت موجود ممکن است سرورهای مجوز را به رد صفحات رضایت و ارسال کدها به نقاط انتهایی تحت کنترل مهاجم سوق دهد  
- **دسترسی غیرمجاز به API**: کدهای مجوز دزدیده شده امکان تبادل توکن و وانمود کردن به کاربران بدون تأیید صریح را فراهم می‌کند

#### **استراتژی‌های کاهش**

**کنترل‌های اجباری:**
- **الزامات رضایت صریح**: سرورهای پراکسی MCP که از شناسه‌های کلاینت ایستا استفاده می‌کنند **باید** برای هر کلاینت ثبت‌شده پویا رضایت کاربر را کسب کنند
- **پیاده‌سازی امنیت OAuth 2.1**: پیروی از بهترین شیوه‌های امنیتی جاری OAuth شامل PKCE برای همه درخواست‌های مجوز
- **اعتبارسنجی سختگیرانه کلاینت**: پیاده‌سازی تأیید دقیق URIهای بازگشتی و شناسه‌های مربوط به کلاینت برای جلوگیری از سوءاستفاده

### آسیب‌پذیری‌های عبور توکن  

**عبور توکن** یک الگوی ضدصریح است که در آن سرورهای MCP توکن‌های کلاینت را بدون اعتبارسنجی مناسب قبول کرده و به APIهای پایین‌دست ارسال می‌کنند که نقض مشخصات مجوز MCP است.

#### **پیامدهای امنیتی**

- **دورزدن کنترل‌ها**: استفاده مستقیم کلاینت به API از توکن‌ها باعث دور زدن محدودیت‌های نرخ، اعتبارسنجی و کنترل‌های نظارتی حیاتی می‌شود
- **خرابی مسیر ممیزی**: توکن‌های صادر شده بالا دست موجب ناممکن شدن شناسایی کلاینت شده و توانایی تحقیقات حادثه را مختل می‌کند
- **خروج داده از طریق پراکسی**: توکن‌های تأییدنشده به بازیگران مخرب اجازه می‌دهد از سرور به عنوان پراکسی برای دسترسی غیرمجاز به داده‌ها استفاده کنند
- **نقض مرز اعتماد**: فرضیات اعتماد سرویس‌های پایین‌دست ممکن است با عدم توانایی در تأیید منشاء توکن نقض شود
- **گسترش حمله چندسرویسی**: توکن‌های به خطر افتاده که در چندین سرویس پذیرفته می‌شوند امکان حرکت جانبی را فراهم می‌کنند

#### **کنترل‌های امنیتی مورد نیاز**

**الزامات غیرقابل مذاکره:**
- **اعتبارسنجی توکن**: سرورهای MCP **نباید** توکن‌هایی را که به صراحت برای سرور MCP صادر نشده‌اند قبول کنند
- **تأیید مخاطب**: همیشه باید ادعاهای مخاطب توکن با هویت سرور MCP مطابقت داشته باشد
- **چرخه عمر مناسب توکن**: پیاده‌سازی توکن‌های دسترسی کوتاه مدت با روش‌های چرخش امن


## امنیت زنجیره تأمین برای سیستم‌های هوش مصنوعی

امنیت زنجیره تأمین فراتر از وابستگی‌های نرم‌افزاری سنتی به کل اکوسیستم هوش مصنوعی توسعه یافته است. پیاده‌سازی‌های مدرن MCP باید به دقت تمام اجزای مرتبط با هوش مصنوعی را تایید و نظارت کنند، زیرا هر کدام آسیب‌پذیری‌هایی دارند که می‌تواند یکپارچگی سیستم را به خطر بیندازد.

### اجزای توسعه یافته زنجیره تأمین هوش مصنوعی

**وابستگی‌های نرم‌افزاری سنتی:**
- کتابخانه‌ها و فریم‌ورک‌های متن باز
- تصاویر کانتینری و سیستم‌های پایه  
- ابزارهای توسعه و خطوط ساخت
- اجزا و خدمات زیرساختی

**عناصر خاص زنجیره تأمین هوش مصنوعی:**
- **مدل‌های پایه**: مدل‌های آموزش‌دیده قبلی از ارائه‌دهندگان مختلف که نیاز به تأیید منشاء دارند
- **خدمات جاسازی**: خدمات بردارسازی و جستجوی معنایی خارجی
- **تأمین‌کنندگان زمینه**: منابع داده، پایگاه‌های دانش و مخازن اسناد  
- **APIهای شخص ثالث**: خدمات هوش مصنوعی خارجی، خطوط ML و نقاط پایانی پردازش داده
- **اثرات مدل**: وزن‌ها، پیکربندی‌ها و نسخه‌های مدل تخصصی
- **منابع داده آموزشی**: مجموعه داده‌های استفاده شده برای آموزش و تنظیم مدل

### استراتژی جامع امنیت زنجیره تأمین

#### **تأیید و اعتماد به اجزا**
- **اعتبارسنجی منشاء**: تأیید منشاء، مجوز و یکپارچگی تمام اجزای هوش مصنوعی قبل از یکپارچه‌سازی
- **ارزیابی امنیتی**: انجام اسکن‌های آسیب‌پذیری و بررسی‌های امنیتی برای مدل‌ها، منابع داده و خدمات هوش مصنوعی
- **تحلیل شهرت**: ارزیابی سابقه امنیتی و روش‌های ارائه‌دهندگان خدمات هوش مصنوعی
- **اعتبارسنجی تطابق مقررات**: اطمینان از اینکه همه اجزا با الزامات امنیتی و مقررات سازمانی مطابقت دارند

#### **خطوط استقرار امن**  
- **امنیت خودکار CI/CD**: ادغام اسکن امنیتی در کل خطوط استقرار خودکار
- **یکپارچگی اثرات**: پیاده‌سازی تأیید رمزنگاری برای تمام اثاثه‌های مستقر شده (کد، مدل‌ها، پیکربندی‌ها)
- **استقرار مرحله‌ای**: استفاده از استراتژی‌های استقرار تدریجی با اعتبارسنجی امنیتی در هر مرحله
- **مخازن اثرات مورد اعتماد**: استقرار فقط از مخازن و رجیستری‌های امن و تأیید شده

#### **نظارت و پاسخ مستمر**
- **اسکن وابستگی‌ها**: نظارت مداوم آسیب‌پذیری برای تمام وابستگی‌های نرم‌افزاری و اجزای هوش مصنوعی
- **نظارت مدل**: ارزیابی مستمر رفتار مدل، انحراف عملکرد و ناهنجاری‌های امنیتی
- **پیگیری سلامت سرویس‌ها**: نظارت بر خدمات هوش مصنوعی خارجی برای در دسترس بودن، حوادث امنیتی و تغییرات سیاست
- **ادغام اطلاعات تهدید**: ترکیب فیدهای تهدید خاص به ریسک‌های امنیتی هوش مصنوعی و یادگیری ماشین

#### **کنترل دسترسی و حداقل امتیاز**
- **مجوزهای سطح اجزا**: محدود کردن دسترسی به مدل‌ها، داده‌ها و خدمات بر اساس ضرورت کسب‌وکار
- **مدیریت حساب سرویس**: پیاده‌سازی حساب‌های سرویس اختصاصی با حداقل مجوزهای لازم
- **تفکیک شبکه**: ایزوله کردن اجزای هوش مصنوعی و محدود کردن دسترسی شبکه بین خدمات
- **کنترل‌های دروازه API**: استفاده از دروازه‌های API متمرکز برای کنترل و نظارت بر دسترسی به خدمات هوش مصنوعی خارجی

#### **پاسخ و بازیابی حادثه**
- **روش‌های پاسخ سریع**: فرآیندهای مستقر برای وصله کردن یا جایگزینی اجزای هوش مصنوعی به خطر افتاده
- **چرخش اعتبارنامه‌ها**: سیستم‌های خودکار برای چرخش اسرار، کلیدهای API و اعتبارنامه‌های سرویس
- **قابلیت‌های بازگشت**: توانایی بازگشت سریع به نسخه‌های قبلی شناخته‌شده سالم اجزای هوش مصنوعی
- **بازیابی نقض زنجیره تأمین**: فرآیندهای خاص برای پاسخ به نقص‌های خدمات هوش مصنوعی بالادستی

### ابزارها و ادغام امنیتی مایکروسافت

**GitHub Advanced Security** محافظت جامعی از زنجیره تأمین ارائه می‌دهد، شامل:
- **اسکن اسرار**: تشخیص خودکار اعتبارنامه‌ها، کلیدهای API و توکن‌ها در مخازن
- **اسکن وابستگی‌ها**: ارزیابی آسیب‌پذیری برای وابستگی‌ها و کتابخانه‌های متن باز
- **تحلیل CodeQL**: تحلیل ایستا کد برای آسیب‌پذیری‌های امنیتی و مسائل برنامه‌نویسی
- **بینش‌های زنجیره تأمین**: شفافیت در سلامت وابستگی‌ها و وضعیت امنیتی

**ادغام Azure DevOps & Azure Repos:**
- ادغام بی‌وقفه اسکن امنیتی در تمام پلتفرم‌های توسعه مایکروسافت
- بررسی‌های امنیتی خودکار در Azure Pipelines برای بارهای کاری هوش مصنوعی
- اجرای سیاست‌ها برای استقرار امن اجزای هوش مصنوعی

**رویه‌های داخلی مایکروسافت:**
مایکروسافت رویه‌های امنیتی گسترده زنجیره تأمین را در تمام محصولات پیاده‌سازی می‌کند. با روش‌های اثبات‌شده در [سفر به سوی امن‌سازی زنجیره تأمین نرم‌افزار در مایکروسافت](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) آشنا شوید.


## بهترین روش‌های امنیت پایه

پیاده‌سازی‌های MCP جایگاه امنیتی موجود سازمان شما را به ارث برده و بر آن بنا می‌کنند. تقویت شیوه‌های امنیتی بنیادین به طور قابل توجهی امنیت کلی سیستم‌های هوش مصنوعی و استقرار MCP را افزایش می‌دهد.

### اصول امنیتی اصلی

#### **شیوه‌های توسعه امن**
- **مطابقت با OWASP**: محافظت در برابر آسیب‌پذیری‌های وب برنامه‌های کاربردی [ده آسیب‌پذیری برتر OWASP](https://owasp.org/www-project-top-ten/)
- **محافظت‌های خاص هوش مصنوعی**: پیاده‌سازی کنترل‌ها برای [ده آسیب‌پذیری برتر OWASP برای LLMها](https://genai.owasp.org/download/43299/?tmstv=1731900559)
- **مدیریت امن اسرار**: استفاده از درگاه‌های اختصاصی برای توکن‌ها، کلیدهای API و داده‌های حساس پیکربندی
- **رمزنگاری انتها به انتها**: پیاده‌سازی ارتباطات امن در همه اجزای برنامه و جریان‌های داده
- **اعتبارسنجی ورودی**: اعتبارسنجی دقیق همه ورودی‌های کاربر، پارامترهای API و منابع داده

#### **سخت‌سازی زیرساخت**
- **احراز هویت چندعاملی**: الزام MFA برای همه حساب‌های مدیریتی و سرویس
- **مدیریت وصله**: وصله خودکار و به موقع برای سیستم‌عامل‌ها، فریم‌ورک‌ها و وابستگی‌ها  
- **ادغام ارائه‌دهنده هویت**: مدیریت متمرکز هویت از طریق ارائه‌دهندگان هویت سازمانی (Microsoft Entra ID، Active Directory)
- **تفکیک شبکه**: ایزوله منطقی اجزای MCP برای محدود کردن پتانسیل حرکت جانبی
- **اصل حداقل امتیاز**: حداقل مجوزهای لازم برای تمام اجزا و حساب‌های سیستم

#### **نظارت و شناسایی امنیتی**
- **ثبت جامع**: ثبت دقیق فعالیت‌های برنامه هوش مصنوعی، شامل تعاملات کلاینت-سرور MCP
- **ادغام SIEM**: مدیریت متمرکز اطلاعات و رویدادهای امنیتی برای شناسایی ناهنجاری‌ها
- **تحلیل رفتاری**: نظارت مبتنی بر هوش مصنوعی برای تشخیص الگوهای غیرعادی در سامانه و رفتار کاربر
- **اطلاعات تهدید**: ادغام فیدهای تهدید خارجی و شاخص‌های نفوذ (IOCs)
- **پاسخ به حادثه**: رویه‌های تعریف شده برای شناسایی، پاسخ و بازیابی در قبال حوادث امنیتی

#### **معماری اعتماد صفر**
- **هرگز اعتماد نکن، همیشه تأیید کن**: تأیید مستمر کاربران، دستگاه‌ها و اتصالات شبکه
- **ریزتفکیک**: کنترل‌های شبکه دقیق که بارهای کاری و خدمات جداگانه را ایزوله می‌کند
- **امنیت مبتنی بر هویت**: سیاست‌های امنیتی بر اساس هویت‌های تأیید شده به جای موقعیت شبکه
- **ارزیابی ریسک مستمر**: ارزیابی پویای وضعیت امنیتی بر اساس زمینه و رفتار فعلی
- **دسترسی مشروط**: کنترل‌های دسترسی که بر اساس عوامل ریسک، مکان و اعتماد دستگاه تغییر می‌کنند

### الگوهای یکپارچه‌سازی سازمانی

#### **ادغام اکوسیستم امنیتی مایکروسافت**
- **Microsoft Defender for Cloud**: مدیریت جامع وضعیت امنیتی ابری
- **Azure Sentinel**: قابلیت‌های بومی ابری SIEM و SOAR برای محافظت از بارهای کاری هوش مصنوعی
- **Microsoft Entra ID**: مدیریت هویت و دسترسی سازمانی با سیاست‌های دسترسی مشروط
- **Azure Key Vault**: مدیریت متمرکز اسرار با پشتیبانی از ماژول امنیتی سخت‌افزاری (HSM)
- **Microsoft Purview**: حاکمیت داده و انطباق برای منابع داده و فرایندهای هوش مصنوعی

#### **انطباق و حاکمیت**
- **همسویی با مقررات**: اطمینان از اینکه پیاده‌سازی‌های MCP مطابق الزامات انطباق خاص صنعت (GDPR، HIPAA، SOC 2) هستند

- **طبقه‌بندی داده‌ها**: دسته‌بندی و مدیریت صحیح داده‌های حساس پردازش‌شده توسط سیستم‌های هوش مصنوعی  
- **ردیابی بررسی‌ها**: ثبت کامل برای تطابق با مقررات و تحقیقات قانونی  
- **کنترل‌های حفظ حریم خصوصی**: پیاده‌سازی اصول حفظ حریم خصوصی از طریق طراحی در معماری سیستم هوش مصنوعی  
- **مدیریت تغییرات**: فرآیندهای رسمی برای بازبینی امنیتی تغییرات سیستم هوش مصنوعی  

این روش‌های پایه یک مبنای امنیتی قوی ایجاد می‌کنند که اثربخشی کنترل‌های امنیتی خاص MCP را افزایش داده و حفاظت جامع برای برنامه‌های مبتنی بر هوش مصنوعی فراهم می‌نماید.  

## نکات کلیدی امنیتی  

- **رویکرد امنیتی چندلایه**: ترکیب روش‌های پایه امنیتی (کدنویسی ایمن، کمترین سطح دسترسی، بررسی زنجیره تامین، نظارت مستمر) با کنترل‌های خاص هوش مصنوعی برای حفاظت جامع  

- **چشم‌انداز تهدیدات خاص هوش مصنوعی**: سیستم‌های MCP با خطرات منحصر به‌فردی مانند تزریق فرمان، مسمومیت ابزار، ربایش نشست، مشکلات نماینده اشتباه، آسیب‌پذیری‌های گذر توکن و مجوزهای بیش از حد مواجه هستند که نیاز به راهکارهای تخصصی دارند  

- **برتری احراز هویت و مجوزدهی**: اجرای احراز هویت قوی با استفاده از ارائه‌دهندگان هویت خارجی (Microsoft Entra ID)، اعمال اعتبارسنجی دقیق توکن، و هرگز پذیرش توکن‌هایی که به صراحت برای سرور MCP شما صادر نشده‌اند  

- **پیشگیری از حملات هوش مصنوعی**: به‌کارگیری Microsoft Prompt Shields و Azure Content Safety برای دفاع در برابر تزریق فرمان غیرمستقیم و مسمومیت ابزار، همراه با اعتبارسنجی متادیتای ابزار و نظارت بر تغییرات پویا  

- **امنیت نشست و انتقال داده‌ها**: استفاده از شناسه‌های نشست رمزنگاری‌شده، غیرقابل پیش‌بینی و متصل به هویت کاربران، اجرای مدیریت چرخه عمر مناسب نشست، و هرگز استفاده از نشست به عنوان روش احراز هویت  

- **بهترین شیوه‌های امنیت OAuth**: جلوگیری از حملات نماینده اشتباه با کسب رضایت صریح کاربر برای کلاینت‌های ثبت‌نام شده پویا، پیاده‌سازی صحیح OAuth 2.1 با PKCE، و اعتبارسنجی سخت‌گیرانه آدرس‌های URI بازگردانی  

- **اصول امنیت توکن‌ها**: اجتناب از الگوهای ضدگذر توکن، اعتبارسنجی ادعاهای مخاطب توکن، پیاده‌سازی توکن‌های کوتاه‌مدت با چرخش امن، و حفظ مرزهای اعتماد واضح  

- **امنیت جامع زنجیره تامین**: همه اجزای اکوسیستم هوش مصنوعی (مدل‌ها، جاسازی‌ها، ارائه‌دهندگان زمینه، APIهای خارجی) را با همان سختگیری امنیتی به عنوان وابستگی‌های نرم‌افزاری سنتی در نظر بگیرید  

- **تحول مستمر**: با مشخصات MCP به‌سرعت در حال تحول همراه باشید، به استانداردهای جامعه امنیتی کمک کنید، و حالت‌های امنیتی تطبیقی را با پیشرفت پروتکل حفظ نمایید  

- **ادغام امنیتی مایکروسافت**: از اکوسیستم جامع امنیتی مایکروسافت (Prompt Shields، Azure Content Safety، GitHub Advanced Security، Entra ID) برای افزایش حفاظت در استقرار MCP بهره ببرید
