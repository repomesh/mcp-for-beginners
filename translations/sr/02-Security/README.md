# MCP Безбедност: Свеобухватна заштита за AI системе

[![Најбоље праксе MCP безбедности](../../../translated_images/sr/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(Кликните на горњу слику да бисте видели видео овог часа)_

Безбедност је основна у дизајну AI система, због чега јој дајемо приоритет као другој теми. Ово је усклађено са Microsoft-овом **Secure by Design** принципом из [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/).

Протокол Контекста Модела (MCP) доноси моћне нове могућности AI-покретаним апликацијама уз увођење јединствених безбедносних изазова који превазилазе традиционалне ризике софтвера. MCP системи се суочавају и са утврђеним безбедносним питањима (сигуран код, принцип најмањих привилегија, безбедност ланца снабдевања) и новим AI-специфичним претњама као што су инјекције упита, тровање алата, препирање сесија, напади збуњеног заменика, рањивости проласка токена и динамичке модификације могућности.

Овај час истражује најкритичније безбедносне ризике у имплементацијама MCP-а — покривајући аутентификацију, ауторизацију, превелика овлашћења, индиректну инјекцију упита, безбедност сесије, проблеме збуњеног заменика, управљање токенима и рањивости ланца снабдевања. Научите применљиве контроле и најбоље праксе за ублажавање ових ризика уз коришћење Microsoft решења као што су Prompt Shields, Azure Content Safety и GitHub Advanced Security за јачање ваше MCP имплементације.

## Циљеви учења

На крају овог часа моћи ћете да:

- **Идентификујете MCP-специфичне претње**: Препознате јединствене безбедносне ризике у MCP системима укључујући инјекције упита, тровање алата, превелика овлашћења, препирање сесија, проблеме збуњеног заменика, рањивости проласка токена и ризике ланца снабдевања
- **Примените безбедносне контроле**: Спроведете ефикасне мере ублажавања укључујући снажну аутентификацију, приступ по принципу најмањих привилегија, безбедно управљање токенима, контроле безбедности сесија и верификацију ланца снабдевања
- **Искористите Microsoft безбедносна решења**: Разумете и примењујете Microsoft Prompt Shields, Azure Content Safety и GitHub Advanced Security за заштиту MCP радног оптерећења
- **Валидирате безбедност алата**: Препознате значај валидације метаподатака алата, надгледање динамичких промена и одбрану од индиректних инјекција упита
- **Интегришете најбоље праксе**: Комбинујете утврђене безбедносне основе (сигурно кодирање, ојачање сервера, zero trust) са MCP-специфичним контролама за свеобухватну заштиту

# MCP Безбедносна архитектура и контроле

Савремене MCP имплементације захтевају слојевите безбедносне приступе који покривају и традиционалну софтверску безбедност и AI-специфичне претње. Брзо еволуирајућа MCP спецификација наставља да усавршава безбедносне контроле, омогућавајући бољу интеграцију са архитектурама предузећа и утврђеним најбољим праксама.

Истраживање из [Microsoft Digital Defense Report](https://aka.ms/mddr) показује да **98% пријављених безбедносних пропуста може бити спречено чврстом хигијеном безбедности**. Најефикаснија стратегија заштите комбинује основне безбедносне праксе са MCP-специфичним контролама — доказане базне мере безбедности остају најутемељеније у смањењу укупног безбедносног ризика.

## Тренутни безбедносни пејзаж

> **Белешка:** Ове информације одражавају MCP безбедносне стандарде стања на **5. фебруар 2026.**, у складу са **MCP спецификацијом 2025-11-25**. MCP протокол се брзо развија, а будуће имплементације могу увести нове обрасце аутентификације и побољшане контроле. Увек консултујте актуелну [MCP спецификацију](https://spec.modelcontextprotocol.io/), [MCP GitHub репозиторијум](https://github.com/modelcontextprotocol) и [документацију најбољих пракси безбедности](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) за најновије смернице.

> **Поглед у будућност:** кандидат за издање `2026-07-28` додатно ојачава ауторизацију — клијенти сада морају да верификују параметар `iss` у одговорима о ауторизацији (RFC 9207), декларацију OpenID Connect `application_type` током Dynamic Client Registration, и да повежу регистроване акредитације са издавајућим ауторизационим сервером. Погледајте [Шта се мења у MCP: Кандидат за издање 2026-07-28](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) за комплетан списак SEPs о ауторизацији.

### Еволуција MCP аутентификације

MCP спецификација је значајно напредовала у приступу аутентификацији и ауторизацији:

- **Оригинални приступ**: Ране спецификације су захтевале од програмера да имплементирају прилагођене аутентификационе сервере, при чему су MCP сервери деловали као OAuth 2.0 ауторизациони сервери који директно управљају аутентификацијом корисника
- **Тренутни стандард (2025-11-25)**: Ажурирана спецификација омогућава MCP серверима делегирање аутентификације спољним провајдерима идентитета (као што је Microsoft Entra ID), побољшавајући безбедносни положај и смањујући сложеност имплементације
- **Безбедност слоја транспорта**: Побољшана подршка за безбедне транспортне механизме са одговарајућим образцима аутентификације и за локалне (STDIO) и удаљене (Streamable HTTP) везе

## Безбедност аутентификације и ауторизације

### Тренутни безбедносни изазови

Савремене MCP имплементације се суочавају са неколико изазова аутентификације и ауторизације:

### Ризици и претње

- **Погрешно конфигурисана ауторизациона логика**: Недостаци у ауторизационој имплементацији MCP сервера могу излагати осетљиве податке и неправилно применити контролу приступа
- **Компромитовани OAuth токени**: Крађа локалних MCP сервер токена омогућава нападачима да се лажно представљају као сервери и приступају сервисима у низу
- **Рњивости проласка токена**: Неадекватно руковање токенима ствара безбедносне заобилазнице и недостатке у одговорности
- **Превелика овлашћења**: MCP сервери са превеликим привилегијама крше принцип најмањих привилегија и шире површине напада

#### Пролазак токена: Критичан анти-шаблон

**Пролазак токена је изричито забрањен** у тренутној MCP ауторизационој спецификацији због озбиљних безбедносних последица:

##### Заобилажење безбедносних контрола
- MCP сервери и downstream API-ји имплементирају кључне безбедносне контроле (ограничење брзине, валидација захтева, надзор саобраћаја) које зависе од правилне валидације токена
- Директно коришћење токена клијента према API-ју заобилази ове виталне мере заштите, поткопавајући безбедносну архитектуру

##### Изазови у одговорности и ревизији  
- MCP сервери не могу разликовати међу клијентима који користе токене издате узводно, што ремети ревизијске тракове
- Записи downstream сервера ресурса приказују обмањујуће порекло захтева уместо стварних MCP сервер посредника
- Истраживање инцидената и ревизија усклађености постају знатно тежи

##### Ризици извлачења података
- Невалидиране тврдње токена омогућавају злонамерним актерима са украденим токенима да користе MCP сервере као проксије за извлачење података
- Прекорачења поверења дозвољавају неовлашћене обрасце приступа који заобилазе намењене безбедносне контроле

##### Векторски напади на више сервиса
- Примљени компромитовани токени у више сервиса омогућавају бочно кретање кроз повезане системе
- Претпоставке поверења између сервиса могу бити нарушене када порекло токена не може бити потврђено

### Безбедносне контроле и ублажавања

**Критични безбедносни захтеви:**

> **ОБАВЕЗНО**: MCP сервери **НЕ СМЕЈУ** прихватати било које токене који нису изричито издати за тај MCP сервер

#### Контроле аутентификације и ауторизације

- **Детаљна ревизија ауторизације**: Спроводите свеобухватне аудите ауторизационе логике MCP сервера како бисте осигурали да само намењени корисници и клијенти приступају осетљивим ресурсима
  - **Упутство за имплементацију**: [Azure API Management као аутентификациони пролаз за MCP сервере](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **Интеграција идентитета**: [Коришћење Microsoft Entra ID за аутентификацију MCP сервера](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **Безбедно управљање токенима**: Имплементирајте [Microsoft-ове најбоље праксе за проверу и животни циклус токена](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens)
  - Валидација тврдњи о публици токена који се поклапају са идентитетом MCP сервера
  - Имплементација одговарајуће ротације и истека токена
  - Превенција поновног коришћења токена и неовлашћене употребе

- **Заштићено чување токена**: Сигурно чување токена са шифровањем како у миру тако и у преносу
  - **Најбоље праксе**: [Приручник за безбедно чување и шифровање токена](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### Имплементација контроле приступа

- **Принцип најмањих привилегија**: Доделите MCP серверима само минимална овлашћења потребна за намењену функционалност
  - Редовне ревизије овлашћења и ажурирања како би се спречило продирање привилегија
  - **Microsoft документација**: [Сигуран приступ са најмањим привилегијама](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **Управљање приступом засновано на улогама (RBAC)**: Имплементирајте детаљне доделе улога
  - Уско обухватите улоге за специфичне ресурсе и радње
  - Избегавајте широка или непотребна овлашћења која шире површину напада

- **Континуирано праћење овлашћења**: Имплементирајте сталне аудите и надзор приступа
  - Пратите обрасце коришћења овлашћења ради откривања аномалија
  - Брзо отклоните превелика или неискоришћена овлашћења

## AI-специфичне безбедносне претње

### Инјекције упита и напади манипулације алатима

Савремене MCP имплементације се суочавају са софистицираним AI-специфичним нападима које традиционалне безбедносне мере не могу у потпуности да отклоне:

#### **Индиректна инјекција упита (Cross-Domain Prompt Injection)**

**Индиректна инјекција упита** представља једну од најкритичнијих рањивости у AI системима опремљеним MCP-ом. Нападачи убацују злонамерна упутства у спољни садржај — документа, веб странице, е-поруке или изворе података — које AI системи након тога обрађују као легитимне команде.

**Сценарији напада:**
- **Инјекција у документима**: Злонамерна упутства скривена у обрађеним документима која покрећу нежељене AI акције
- **Искоришћавање веб садржаја**: Компромитоване веб странице са уграђеним упитима који манипулишу понашањем AI када их системи прегледају
- **Напади путем е-поште**: Злонамерни упити у е-порукама који узрокују да AI асистенти цуре информације или извршавају неовлашћене радње
- **Контаминација извора података**: Компромитоване базе података или API-ји који служе загађени садржај AI системима

**Утицај у стварном свету**: Ови напади могу довести до извлачења података, кршења приватности, генерисања штетног садржаја и манипулације корисничким интеракцијама. За детаљну анализу погледајте [Prompt Injection у MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).

![Дијаграм напада инјекције упита](../../../translated_images/sr/prompt-injection.ed9fbfde297ca877.webp)

#### **Напади тровања алата**

**Тровање алата** циља на метаподатке који дефинишу MCP алате, експлоатишући начин на који LLM модели тумаче описе и параметре алата за доношење одлука о извршењу.

**Механизми напада:**
- **Манипулација метаподацима**: Нападачи убацују злонамерна упутства у описе алата, дефиниције параметара или примере употребе
- **Невидљива упутства**: Скривени упити у метаподацима алата који се обрађују од стране AI модела али су невидљиви људским корисницима
- **Динамичке модификације алата („Rug Pulls“)**: Алати које корисници одобре касније се мењају да извршавају злонамерне радње без њиховог сазнања
- **Инјекција параметара**: Злонамерни садржај уграђен у шеме параметара алата који утичу на понашање модела


**Ризици хостованих сервера**: Удаљени MCP сервери представљају повећане ризике јер дефиниције алата могу бити ажуриране након почетног одобрења корисника, стварајући сценарије у којима претходно безбедни алати постају злонамерни. За детаљну анализу, видети [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks).

![Tool Injection Attack Diagram](../../../translated_images/sr/tool-injection.3b0b4a6b24de6bef.webp)

#### **Додатни вектори напада везани за АИ**

- **Cross-Domain Prompt Injection (XPIA)**: Сложени напади који користе садржај из више домена да би заобишли безбедносне контроле
- **Динамичка измена способности**: Промене способности алата у реалном времену које избегавају почетне безбедносне процене
- **Отровање контекстног прозора**: Напади који манипулишу великим контекстним прозорима да би сакрили злонамерна упутства
- **Напади конфузије модела**: Искоришћавање ограничења модела да би се створило непредвидиво или небезбедно понашање


### Утицај безбедносних ризика везаних за АИ

**Последице великог утицаја:**
- **Извлачење података**: Неовлашћени приступ и крађа осетљивих пословних или личних података
- **Прекршаји приватности**: Излагање лично идентификовачких информација (PII) и поверљивих пословних података  
- **Манипулација системом**: Непланиране измене критичних система и токова рада
- **Крађа акредитива**: Компромитовање токена за аутентикацију и сервисних акредитива
- **Латерално померање**: Користење компромитованих АИ система као полазних тачака за шире нападе на мрежу

## Напредне безбедносне претње MCP-а

### Ранљивости по отимању сесије

**Отимање сесије** представља критичан вектор напада у државно-зависним MCP имплементацијама где неовлашћене стране добијају и злоупотребљавају легитимне идентификаторе сесије ради имитације клијената и извођења неовлашћених радњи.

#### **Сценарији напада и ризици**

- **Инјекција упита у отету сесију**: Нападачи са украденим ID сесије инјектују злонамерне догађаје у сервере који деле стање сесије, потенцијално изазивајући штетне радње или приступ осетљивим подацима
- **Директна имитација**: Украдени ID сесије омогућавају директне позиве MCP серверу који заобилазе аутентикацију, третирајући нападаче као легитимне кориснике
- **Компромитовани настављиви токови**: Нападачи могу прерано прекинути захтеве, узрокујући да легитимни клијенти наставе са потенцијално злонамерним садржајем

#### **Контроле безбедности за управљање сесијама**

**Критични захтеви:**
- **Провера овлашћења**: MCP сервери који имплементирају овлашћење **МОРAJУ** проверити СВЕ улазне захтеве и **НЕ СМЕЈУ** се ослањати на сесије за аутентикацију
- **Сигурна генерација сесија**: Користити криптографски сигурне, недетерминисане ID сесије генерисане помоћу сигурних генератора случајних бројева
- **Веза са корисничким подацима**: Везати ID сесије за корисничке податке користећи формате као што је `<user_id>:<session_id>` да би се спречила злоупотреба сесија између корисника
- **Управљање животним циклусом сесије**: Имплементирати правилно истекање, ротацију и неважење ради ограничења прозора рањивости
- **Безбедност транспорта**: Обавезан HTTPS за сву комуникацију да би се спречило пресретање ID сесије

### Проблем збуњеног помоћника

Проблем **збуњеног помоћника** јавља се када MCP сервери делују као прокси за аутентикацију између клијената и трећих услуга, стварајући могућности за заобилажење овлашћења кроз експлоатацију статичких ID клијената.

#### **Механизми напада и ризици**

- **Заобилажење приступа путем колачића**: Претходна корисничка аутентикација ствара колачиће за сагласност које нападачи злоупотребљавају кроз злонамерне захтеве за овлашћење са израђеним URI за преусмеравање
- **Крађа кода за овлашћење**: Постојећи колачићи сагласности могу изазвати да сервери овлашћења прескоче екране сагласности, преусмеравајући кодове на контролисане крајње тачке злонамера
- **Неовлашћени приступ API-ју**: Украдени кодови за овлашћење омогућавају размену токена и имитацију корисника без експлицитног одобрења

#### **Стратегије ублажавања**

**Обавезне контроле:**
- **Изричити захтеви за сагласност**: MCP прокси сервери који користе статичке ID клијената **МОРAJУ** за сваки динамички регистровани клијент добити сагласност корисника
- **Имплементација OAuth 2.1 безбедности**: Пратити најбоље тренутне праксе OAuth безбедности укључујући PKCE (Proof Key for Code Exchange) за све захтеве за овлашћење
- **Строга валидација клијента**: Имплементирати ригорозну валидацију URI за преусмеравање и идентификатора клијента да би се спречила експлоатација

### Ранљивости услед прослеђивања токена  

**Прослеђивање токена** представља директан анти-патерн где MCP сервери прихватају токене клијената без правилне провере и прослеђују их даљим API-јима, кршећи MCP спецификације овлашћења.

#### **Безбедносне импликације**

- **Заобилажење контроле**: Директна употреба токена клијента у API-ју заобилази критична ограничења брзине, валидацију и контроле надзора
- **Корупција ревизорског ланца**: Токени издати изнад нису идентификовали клијента, што онемогућује истрагу инцидената
- **Извлачење података преко проксија**: Несигурни токени омогућавају злонамерним актерима да користе сервере као прокси за неовлашћени приступ подацима
- **Кршења граница поверења**: Услуге на даљем нивоу поверења могу бити прекршене када није могуће потврдити порекло токена
- **Ширења напада на више услуга**: Компромитовани токени прихваћени у више услуга омогућују латерално померање

#### **Потребне безбедносне контроле**

**Неодложни захтеви:**
- **Валидација токена**: MCP сервери **НЕ СМЕЈУ** прихватати токене који нису експлицитно издати за MCP сервер
- **Провера публике**: Увек проверавати да ли се тврдње о публици у токену поклапају са идентитетом MCP сервера
- **Правилно управљање животним циклом токена**: Имплементирати краткорочне приступне токене са сигурним праксама ротације


## Безбедност ланца снабдевања за АИ системе

Безбедност ланца снабдевања је еволуирала да обухвати читаву АИ екосистем уместо традиционалних софтверских зависности. Савремене MCP имплементације морају темељно проверавати и пратити све АИ компоненте, јер свака уноси потенцијалне рањивости које могу угрозити интегритет система.

### Проширене компоненте ланца снабдевања за АИ

**Традиционалне софтверске зависности:**
- Отворене библиотеке и фрејмворкови
- Слике контејнера и основни системи  
- Развојни алати и производни процеси
- Инфраструктурне компоненте и сервиси

**Елементи ланца снабдевања специфични за АИ:**
- **Основни модели**: Предтренирани модели од различитих добављача који захтевају валидацију порекла
- **Услуге уграђивања**: Спољне услуге векторизације и семантичке претраге
- **Добављачи контекста**: Извори података, базе знања и депоа докумената  
- **API-ји трећих страна**: Спољашње АИ услуге, ML производни ланци и крајње тачке обраде података
- **Артефакти модела**: Тежине, конфигурације и прецизиране варијанте модела
- **Извори података за обуку**: Скуп података коришћен за обуку и прецизирање модела

### Свеобухватна стратегија безбедности ланца снабдевања

#### **Верификација компонената и поверење**
- **Валидација порекла**: Проверити порекло, лиценце и интегритет свих АИ компоненти пре интеграције
- **Безбедносна процена**: Спроводити проверу рањивости и безбедносне ревизије модела, извора података и АИ услуга
- **Анализа репутације**: Оценити безбедносну праксу и записе добављача АИ услуга
- **Верификација усклађености**: Осигурати да све компоненте задовољавају безбедносне и регулаторне захтеве организације

#### **Сигурни производни ланци**  
- **Аутоматизована CI/CD безбедност**: Интегрисати безбедносне скенере у све аутоматизоване производне ланце
- **Интегритет артефаката**: Имплементирати криптографску проверу свих дистрибуираних артефаката (код, модели, конфигурације)
- **Фазно распоређивање**: Користити стратешко постепено увођење уз безбедносну проверу на сваком кораку
- **Регистри поузданих артефаката**: Дистрибуирати искључиво из проверењеих и сигурних регистара и складишта

#### **Континуирани надзор и одговор**
- **Скенирање зависности**: Континуирано праћење рањивости свих софтверских и АИ зависности
- **Мониторинг модела**: Континуирана процена понашања модела, одступања перформанси и безбедносних аномалија
- **Праћење здравља сервиса**: Надзор спољних АИ услуга за доступност, безбедносне инциденте и промене политика
- **Интеграција обавештајних података о претњама**: Укључивање токова претњи специфичних за АИ и ML безбедносне ризике

#### **Контрола приступа и најмањи привилегијум**
- **Дозволе на нивоу компоненти**: Ограничити приступ моделима, подацима и услугама у складу са пословном потребом
- **Управљање сервисним налозима**: Имплементирати посебне сервисне налоге са минималним потребним дозволама
- **Сегментација мреже**: Изоловање АИ компоненти и ограничење приступа мрежи између услуга
- **Контроле API гатеваја**: Користити централизоване API гатеваје за контролу и надзор приступа спољним АИ услугама

#### **Одговор на инциденте и опоравак**
- **Поступци брзог одзива**: Успостављени процеси за закрпљивање или замену компромитованих АИ компоненти
- **Ротација акредитива**: Аутоматизовани системи за ротацију тајни, API кључева и сервисних акредитива
- **Могућност враћања**: Способност брзог враћања на претходне познато добре верзије АИ компоненти
- **Опоравак од повреда ланца снабдевања**: Посебни поступци за реаговање на компромитацију горњих АИ услуга

## Најбоље праксе за основну безбедност

MCP имплементације наслеђују и надограђују постојећи безбедносни положај ваше организације. Јачање основних безбедносних пракси значајно побољшава укупну безбедност АИ система и MCP имплементација.

### Основни безбедносни принципи

#### **Праксе безбедног развоја**
- **Усклађеност са OWASP-ом**: Заштита од [OWASP Top 10](https://owasp.org/www-project-top-ten/) рањивости веб апликација
- **Заштите специфичне за АИ**: Имплементирати контроле за [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559)
- **Сигурно управљање тајнама**: Користити посебне ковчеге за токене, API кључеве и осетљиве конфигурационе податке
- **Крај-до-краја енкрипција**: Имплементирати сигурну комуникацију у свим компонентама апликације и токовима података
- **Валидација уноса**: Ригорозна валидација свих корисничких уноса, API параметара и извора података

#### **Ојачавање инфраструктуре**
- **Мултифакторска аутентикација**: Обавезна MFA за све администраторске и сервисне налоге
- **Управљање закрпама**: Аутоматизовано и благовремено закрпљивање оперативних система, фрејмворкова и зависности  
- **Интеграција добављача идентитета**: Централизовано управљање идентитетом кроз предузећа (Microsoft Entra ID, Active Directory)
- **Сегментација мреже**: Логичка изолација MCP компоненти ради ограничења латералног кретања
- **Принцип најмањег привилегијума**: Минималне потребне дозволе за све системске компоненте и налоге

#### **Праћење и детекција безбедности**
- **Свеобухватно логовање**: Детаљно логовање активности АИ апликација, укључујући интеракције MCP клијената и сервера
- **SIEM интеграција**: Централизовано управљање безбедносним информацијама и догађајима за детекцију аномалија
- **Бихевиорална аналитика**: Надзор потпомогнут АИ-јем за откривање необичних образаца у понашању система и корисника
- **Обавештајни подаци о претњама**: Интеграција спољних токова претњи и индикатора компромитације (IOC)
- **Одговор на инциденте**: Јасно дефинисани поступци за детекцију, одговор и опоравак од безбедносних инцидената

#### **Архитектура нултог поверења**
- **Никад не веруј, увек потврђуј**: Континуирана верификација корисника, уређаја и мрежних веза
- **Микро-сегментација**: Грануларне мрежне контроле које изолују појединачне радне задатке и сервисе
- **Безбедност усмерена на идентитет**: Безбедносне политике засноване на провереним идентитетима а не на мрежној локацији
- **Континуирана процена ризика**: Динамичка процена безбедносног положаја на основу тренутног контекста и понашања
- **Условни приступ**: Контроле приступа које се прилагођавају на основу фактора ризика, локације и поверења у уређај

### Обрасци интеграције у предузећима

#### **Интеграција Microsoft безбедносног екосистема**
- **Microsoft Defender for Cloud**: Свеобухватно управљање безбедносним положајем у облаку
- **Azure Sentinel**: Облачно родни SIEM и SOAR могућности за заштиту АИ радних оптерећења
- **Microsoft Entra ID**: Управљање идентитетом и приступом у предузећу са политикама условног приступа
- **Azure Key Vault**: Централизовано управљање тајнама уз подршку хардверског модула безбедности (HSM)
- **Microsoft Purview**: Управљање подацима и усклађеност за АИ изворе података и токове рада

#### **Усклађеност и управљање**
- **Усклађеност са регулативом**: Осигурати да MCP имплементације испуњавају захтеве индустријске регулативе (GDPR, HIPAA, SOC 2)

- **Класификација података**: Правилна категоризација и руковање осетљивим подацима који се обрађују у системима вештачке интелигенције
- **Аудиторски трагови**: Свеобухватно евидентирање за регулаторну усаглашеност и форензичку истрагу
- **Контроле приватности**: Имплементација принципа приватности по замисли у архитектуру система вештачке интелигенције
- **Управљање променама**: Формални процеси за безбедносне прегледе измена у системима вештачке интелигенције

Ове темељне праксе стварају робусну безбедносну основу која побољшава ефективност безбедносних контрола специфичних за MCP и пружа свеобухватну заштиту за апликације које покреће вештачка интелигенција.

## Кључне безбедносне препоруке

- **Слојевити приступ безбедности**: Комбинујте темељне безбедносне праксе (сигурно кодирање, најмање привилегије, верификација ланца снабдевања, континуирани надзор) са контролама специфичним за вештачку интелигенцију за свеобухватну заштиту

- **Пејзаж претњи специфичних за вештачку интелигенцију**: MCP системи се суочавају са јединственим ризицима као што су инјекција упита, отровање алата, преузимање сесије, проблеми „збуњеног повереника“, рањивости пролаза токена и претеране дозволе које захтевају специјализована ублажавања

- **Изврсна аутентификација и ауторизација**: Имплементирајте робусну аутентификацију користећи спољне добављаче идентитета (Microsoft Entra ID), спроводите правилну валидацију токена и никада не прихватајте токене који нису изричито издати за ваш MCP сервер

- **Превенција напада на вештачку интелигенцију**: Распоредите Microsoft Prompt Shields и Azure Content Safety да браните од индиректне инјекције упита и напада отровања алата, при томе валидајући метаподатке алата и пратећи динамичке промене

- **Безбедност сесија и транспорта**: Користите криптографски сигурне, недетерминистичке ИД-е сесије везане за идентитете корисника, имплементирајте правилно управљање животним циклусом сесије и никада не користите сесије за аутентификацију

- **Најбоље праксе OAuth безбедности**: Спријечите нападе "збуњеног повереника" кроз изричиту сагласност корисника за динамички регистроване клијенте, правилну имплементацију OAuth 2.1 са PKCE и строгу валидацију URI преусмеравања  

- **Принципи безбедности токена**: Избегавајте анти-патерне пролаза токена, валидајте захтеве за публику токена, имплементирајте краткотрајне токене са сигурном ротацијом и одржавајте јасне границе поверења

- **Свеобухватна безбедност ланца снабдевања**: Третирајте све компоненте AI екосистема (моделе, уградње, добављаче контекста, спољашње API-је) са истом безбедносном строгошћу као традиционалне софтверске зависности

- **Континуирани развој**: Будите у току са брзо еволуирајућим MCP спецификацијама, доприносијте стандардима заједнице за безбедност и одржавајте прилагодљиве безбедносне ставове како протокол сазрева

- **Интеграција Microsoft безбедности**: Искористите свеобухватни Microsoft безбедносни екосистем (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) за побољшану заштиту дејства MCP
