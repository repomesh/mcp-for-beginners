### Појачавање безбедности и исправљање рањивости зависности

Извршена је потпуна безбедносна провера у свим манифестима зависности и изворним примерима, након чега су поправљена сва пријављена npm упозорења и једна откривена рањивост у коду. Након исправке, `npm audit` приказује **0 рањивости** у сваком ревидираном директоријуму.

#### npm рањивости зависности (преносне) — Поправљено

Ревидирано свих 15 унетих `package-lock.json` фајлова. Ранљивости су биле ограничене на преносне зависности које су увезене од стране MCP Inspector алата за развој, OpenAI клијента и MCP SDK-а; све су сада решене без прекида примера:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** и **lab3/code/weather_mcp/inspector**: Подигнут `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), што је очистило упозорења за уграђене `ajv`, `brace-expansion`, `diff`, `path-to-regexp` и `ws`. Додат npm унос `overrides` који приморава патчовани `shell-quote@1.8.4` да елиминише преостало критично упозорење које је доносила `concurrently`; поново генерисани оба локационих фајлова (сада 0 рањивости)
- **03-GettingStarted/samples/typescript**: `npm audit fix` обновио преносни `qs` (умерено) на патчирано издање
- **03-GettingStarted/samples/javascript**: `npm audit fix` обновио преносни `hono` (умерено) на патчирано издање
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` обновио преносни `form-data` (високо) на патчирано издање
- **03-GettingStarted/11-simple-auth/solution/typescript**: Генерисан недостајући `package-lock.json` тако да је пројекат репродуцибилан и проверљив (0 рањивости)

#### Безбедносна исправка на нивоу кода (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: Уклоњен `shell=True` из алата `open_in_vscode`. Претходни `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` је омогућавао да се метакарактерима шкољке у путањи фасцикле тумаче преко `cmd.exe` (вектор за инјекцију команди). Сада директно покреће разрешени `Code.exe` са фасциклом као аргументом — без шкољке — што је функционално еквивалентно и безбедно.

#### Python ревизија зависности

- Проверавани сви Python скуп зависности помоћу `pip-audit`. `05-AdvancedTopics` и `03-GettingStarted/samples/python` су пријавили **нема познатих рањивости** (њихови `mcp` / `httpx` / `pydantic` / `python-dotenv` распони resolve-ују се до актуелних патчираних верзија)

Додат експлицитни сигурносни pin `werkzeug>=3.1.6` тако да је патчирана верзија решена; потврђено да ограничење чисто resolve-ује са `chainlit` / `mcp` / `semantic-kernel` стогом

  - Транспортни слој: STDIO (локални) и Streamable HTTP са SSE (удаљени) транспортни механизми
- **Безбедносни оквир**: Опсежни безбедносни принципи укључујући експлицитни кориснички пристанак, заштиту приватности података, безбедност извршења алата и сигурност транспортног слоја
- **Обрасци комуникације**: Ажуриране поруке протокола које приказују иницијацију, откривање, извршење и обавештења
- **Примери кода**: Освежени мултијезички примери (.NET, Java, Python, JavaScript) како би одсликали актуелне MCP SDK обрасце

#### Безбедност (02-Security/) - Опсежна ревизија безбедности  
- **Усклађеност са стандардима**: Потпуна усклађеност са безбедносним захтевима MCP спецификације 2025-06-18
- **Еволуција аутентификације**: Документована еволуција од прилагођених OAuth сервера до делегације спољног провајдера идентитета (Microsoft Entra ID)
- **Анализа претњи специфичних за AI**: Побољшано покриће савремених AI вектора напада
  - Детаљни сценарији напада prompt injection са примерима из стварног света
  - Механизми труљења алата и обрасци напада "rug pull"
  - Отровање context window и напади конфузије модела
- **Microsoft AI безбедносна решења**: Опсежно покриће Microsoft безбедносног екосистема
  - AI Prompt Shields са напредном детекцијом, фокусирањем и техникама делимитера
  - Обрасци интеграције Azure Content Safety
  - GitHub Advanced Security за заштиту ланца снабдевања
- **Напредне мере против претњи**: Детаљне безбедносне контроле за
  - Отмицу сесије са MCP-специфичним сценаријима напада и криптографским захтевима за session ID
  - Проблеме Confused Deputy у MCP прокси сценаријима са експлицитним захтевима за пристанак
  - Ранџивости у пролазу токена са обавезним контролама валидације
- **Безбедност ланца снабдевања**: Проширено покриће AI ланца снабдевања укључујући foundation моделе, услуге ембедднинга, провајдере контекста и API-је трећих страна
- **Безбедност основе**: Побољшана интеграција са безбедносним обрасцима предузећа укључујући zero trust архитектуру и Microsoft безбедносни екосистем
- **Организација ресурса**: Категорисани опсежни линкови ресурса по типу (Званични документи, стандарди, истраживања, Microsoft решења, водичи за имплементацију)
