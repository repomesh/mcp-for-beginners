Овај поглавље приказује како изградити реалног AI агента који интегрише са екстерним API-јима, обрађује различите типове података, управља грешкама и оркестрира више алата — све у формату спремном за продукцију. Видећете:

- **Интеграција са екстерним API-јима који захтевају аутентификацију**
- **Обрада различитих типова података из више крајњих тачака**
- **Робусне стратегије за управљање грешкама и логовање**

На крају, имаћете практично искуство са обрасцима и најбољим праксама које су неопходне за напредне AI и LLM-покретане апликације.

## Web Search MCP сервер

Овај одељак уводи архитектуру и карактеристике Web Search MCP сервера. Видећете како се FastMCP и SerpAPI користе заједно да прошире могућности LLM-а са подацима из веба у реалном времену.

### Преглед

Ова имплементација укључује четири алата који показују способност MCP-а да безбедно и ефикасно обрађује различите задатке засноване на екстерним API-јима:

- **general_search**: За широке резултате претраге веба
- **news_search**: За најновије наслове вести
- **product_search**: За податке из е-комерц маркетплаце-а
- **qna**: За питања и одговоре

### Карактеристике
- **Примери кôда**: Укључује блокове кôда за језик Python (и лако прошириве на друге језике) користећи кодне пивоте за јасноћу

### Python

```python
# Пример коришћења алата general_search
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

Пре покретања клијента, корисно је разумети шта сервер ради. Фајл [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) имплементира MCP сервер, излажући алате за претрагу веба, новости, производа и питања и одговора путем интеграције са SerpAPI. Он обрађује долазне захтеве, управља API позивима, парсира одговоре и враћа структуриране резултате клијенту.

Можете погледати пуну имплементацију у [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py).

Ево кратког примера како сервер дефинише и региструје алат:

### Python сервер

```python
# server.py (исцечак)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...имплементација...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **Интеграција екстерног API**: Приказује безбедну обраду API кључева и екстерних захтева
- **Парсирање структурираних података**: Показује како трансформисати одговоре API-ја у формате погодне за LLM
- **Управљање грешкама**: Робусно руковање грешкама са одговарајућим логовањем
- **Интерактивни клијент**: Укључује и аутоматизоване тестове и интерактивни режим за тестирање
- **Управљање контекстом**: Користи MCP Context за логовање и праћење захтева

Да бисте покренули MCP сервер, користите следећу команду:

```bash

```

Сервер ће радити као stdio-базиран MCP сервер на који се клијент може директно повезати.

### Режими клијента

Клијент (`client.py`) подржава два режима интеракције са MCP сервером:

- **Нормални режим**: Покреће аутоматизоване тестове који проверавају све алате и њихове одговоре. Ово је корисно за брзу проверу да ли сервер и алати раде како се очекује.
- **Интерактивни режим**: Покреће мени интерфејс где ручно бирате и позивате алате, уносите прилагођене упите и видите резултате у реалном времену. Ово је идеално за истраживање могућности сервера и експериментисање са различитим уносима.

Можете погледати пуну имплементацију у [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py).

### Покретање клијента

За покретање аутоматизованих тестова (ово ће аутоматски покренути сервер):

```bash

```

Или покрените у интерактивном режиму:

```bash

```

У овом контексту, "тест скрипта" значи прилагођени Python програм који пишете да бисте деловали као клијент MCP сервера. Уместо формалног јединичног теста, ова скрипта вам омогућава да програмски повежете на сервер, позивате било који од његових алата са параметрима по избору и прегледате резултате. Овај приступ је користан за:
- Прототипирање и експериментисање са позивима алата
- Валидацију како сервер одговара на различите уносе
- Аутоматизацију поновљених позива алата
- Креирање сопствених радних токова или интеграција на врху MCP сервера

Можете користити тест скрипте за брзо испробавање нових упита, дебаговање понашања алата, или чак као полазну основу за напреднију аутоматизацију. Испод је пример како користити MCP Python SDK за креирање такве скрипте:

## Напредни појмови у овој лекцији

Пре него што почнете са изградњом, ево неколико важних напредних појмова који ће се појавити кроз цело ово поглавље. Разумевање ових помоћи ће вам да пратите, чак и ако су вам нови:

- **Оркестрација више алата**: Ово значи покретање више различитих алата (као што су претрага веба, претрага новости, претрага производа и питања и одговори) унутар једног MCP сервера. Омогућава вашем серверу да обради разноврсне задатке, а не само један.
- **Управљање ограничењима API позива**: Многи екстерни API-ји (као SerpAPI) имају ограничења колико захтева можете послати у одређеном временском периоду. Добар кôд проверава ова ограничења и поступа са њима на уредан начин, тако да ваша апликација не пукне ако досегнете лимит.
- **Парсирање структурираних података**: Одговори API-ја су често сложени и угнежђени. Овај појам се односи на претварање тих одговора у чисте, лаке за употребу формате погодне за LLM или друге програме.
- **Опоравак од грешке**: Понекад нешто пође наопако — можда мрежа не ради, или API не враћа оно што очекујете. Опоравак од грешке значи да ваш кôд може да се носи са тим проблемима и да и даље пружи корисне повратне информације, уместо да се сруши.
- **Валидација параметара**: Ово је провера да су сви улази у ваше алате исправни и безбедни за коришћење. Укључује подешавање подразумеваних вредности и проверавање типова, што помаже у спречавању грешака и конфузија.

Овај одељак ће вам помоћи да дијагностикујете и решите уобичајене проблеме са којима се можете сусрести док радите са Web Search MCP сервером. Уколико наиђете на грешке или неочекивано понашање приликом рада са Web Search MCP сервером, овај одељак за решавање проблема пружа решења за најчешће проблеме. Прегледајте ове савете пре него што тражите додатну помоћ — често брзо решавају проблеме.

## Решавање проблема

Када радите са Web Search MCP сервером, могу се повремено појавити проблеми — ово је нормално приликом развоја са екстерним API-јима и новим алатима. Овај одељак пружа практична решења за најчешће проблеме, тако да се брзо вратите на прави пут. Уколико наиђете на грешку, почните овде: савети испод обрађују проблеме са којима се већина корисника сусреће и често могу решити ваш проблем без додатне помоћи.

### Чести проблеми

Испод су неки од најчешћих проблема са којима се корисници сусрећу, са јасним објашњењима и корацима за решавање:

1. **Недостаје SERPAPI_KEY у .env фајлу**
   - Ако видите грешку `SERPAPI_KEY environment variable not found`, то значи да ваша апликација не може да пронађе API кључ потребан за приступ SerpAPI-ју. Да бисте то поправили, направите `.env` фајл у корену вашег пројекта (ако га већ немате) и додајте линију као `SERPAPI_KEY=your_serpapi_key_here`. Обавезно замените `your_serpapi_key_here` са вашим стварним кључем са SerpAPI сајта.

2. **Грешке да модул није пронађен**
   - Грешке као `ModuleNotFoundError: No module named 'httpx'` указују да је потребан Python пакет недостаје. Ово се обично дешава ако нисте инсталирали све зависности. Да бисте то решили, покрените `pip install -r requirements.txt` у вашем терминалу да бисте инсталирали све што ваш пројекат треба.

3. **Проблеми са конекцијом**
   - Ако добијете грешку попут `Error during client execution`, често то значи да се клијент не може повезати на сервер, или сервер не ради како је очекивано. Двоструко проверите да ли су верзије клијента и сервера компатибилне, као и да ли је `server.py` присутан и покренут у исправном директоријуму. Поновно покретање и сервера и клијента такође може помоћи.

4. **Грешке SerpAPI-ја**
   - Ако видите `Search API returned error status: 401`, то значи да ваш SerpAPI кључ недостаје, није исправан или је истекао. Идите на ваш SerpAPI контролни панел, проверите кључ и ажурирајте `.env` фајл по потреби. Ако је кључ исправан али и даље видите ову грешку, проверите да ли вам је бесплатна квота истекла.

### DEBUG режим

По подразумеваној вредности, апликација бележи само важне информације. Ако желите да видите више детаља о томе шта се дешава (на пример, да дијагностикујете компликоване проблеме), можете укључити DEBUG режим. Он ће вам показати много више о сваком кораку који апликација предузима.

**Пример: нормални исход**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**Пример: DEBUG исход**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

Приметите како DEBUG режим укључује додатне линије о HTTP захтевима, одговорима и другим интерним детаљима. Ово може бити врло корисно за решавање проблема.

Да бисте укључили DEBUG режим, подесите ниво логовања на DEBUG на врху вашег `client.py` или `server.py`:

# [Python](#tab/python-debug)


```python
# На врху вашег client.py или server.py
import logging
logging.basicConfig(
    level=logging.DEBUG,  # Промените са INFO на DEBUG
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
