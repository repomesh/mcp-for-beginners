## जुन १६, २०२६

### MCP विशिष्टता संरेखण र नमूना प्रमाणीकरण

पाठ्यक्रमलाई वर्तमान **MCP Specification 2025-11-25** र सबैभन्दा नयाँ आधिकारिक SDK हरूसँग प्रमाणीकरण गरियो, त्यसपछिका पुराना विशिष्टता सन्दर्भहरू सुधार गरियो र कोर नमूनाहरू अझै बनिरहेका र चलिरहेका छन् भन्ने पुष्टि गरियो।

#### विशिष्टता संस्करण सुधार (2025-06-18 / 2025-03-26 → 2025-11-25)

पुरानो विशिष्टता पुनरीक्षणलाई *हालको/नयाँतम* मानक भनेर अझै दाबी गर्ने अंग्रेजी सामग्री अद्यावधिक गरियो र लिंकहरूलाई आधिकारिक `modelcontextprotocol.io` विशिष्टता मार्गहरूमा पुनःनिर्देशन गरियो:
- **05-AdvancedTopics/mcp-security/README.md**: "Current Standard" ब्यानर, परिचय, कोर सुरक्षा सिद्धान्तहरू शीर्षक, अनिवार्य आवश्यकताहरू शीर्षक, Microsoft Entra ID सेक्षन, References & Resources लिंकहरू, र समापन सुरक्षा सूचना (८ सन्दर्भहरू) २०२५-११-२५ मा अद्यावधिक
- **05-AdvancedTopics/mcp-transport/README.md**: अतिरिक्त स्रोतहरूको विशिष्टता लिंक र "Current Standard" ब्यानर २०२५-११-२५ मा अद्यावधिक
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: पुरानो `2025-03-26` सुरक्षा र विश्वास लिंकलाई हालको २०२५-११-२५ सुरक्षा उत्तम अभ्यास पेजले प्रतिस्थापन गरियो
- **03-GettingStarted/14-sampling/README.md**: आधिकारिक sampling डकुमेन्ट लिंक २०२५-११-२५ मा अद्यावधिक
- **03-GettingStarted/05-stdio-server/README.md**: वर्तमान काल (present-tense) "current MCP specification" सन्दर्भ र अतिरिक्त स्रोत विशिष्टता लिंक २०२५-११-२५ मा अद्यावधिक (इतिहासबोध SSE-डिप्रिसेसन नोटहरू यथावत राखियो)

#### वर्तमान SDK हरूसँग नमूना प्रमाणीकरण

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` ले `@modelcontextprotocol/sdk@1.29.0` समाधान गर्यो; `tsc --noEmit` बिना प्रकार त्रुटिहरू पास भयो — हालको `McpServer`/`StdioServerTransport` API हरू मान्य छन्
- **Python (03-GettingStarted/01-first-server/solution/python)**: `mcp[cli]` (1.27.2) सहित अलग `.venv` मा प्रमाणीकरण गरियो; `py_compile` पास भयो र `FastMCP.list_tools()` ले `add` र `subtract` उपकरणहरू सही रूपमा फर्कायो
- सबै नमूना `@modelcontextprotocol/sdk` संस्करण दायरा (`>=1.26.0` / `^1.26.0` / `^1.27.0`) ले हालको `1.29.0` साफसित गर्ने र कुनै API परिवर्तन नगर्ने पुष्टि गरियो

पुराना SDK पिनहरू बढाइयो जसले हरेक नमूनाले वर्तमान MCP रिलीज ट्र्याक गर्दछ, र पञ्जीकरण-द्वारा-ब्यापार सम्मेलन अनुसार:

- **10-StreamliningAIWorkflows.../lab3/code/weather_mcp/pyproject.toml** र **lab4/code/github_mcp_server/pyproject.toml**: ठीक पिन `mcp==1.23.0` लाई `mcp>=1.26.0` मा बढाइयो; दुवै `uv.lock` फाइलहरू पुन: उत्पादन गरियो (`uv lock`) जसले लकफाइलहरूलाई हालको `mcp 1.27.2` सम्म समाधान र समानता राख्छ

### सुरक्षा कडाइ र निर्भरता जोखिम न्यूनीकरण

हरेक निर्भरता म्यानिफेस्ट र नमूना स्रोत कोडमा पूर्ण सुरक्षा समीक्षात्मक पास गरियो, त्यसपछि सबै रिपोर्ट गरिएका npm सल्लाहहरू र एक कोड स्तरको खोजी सुधारियो। सुधारपछि, `npm audit` ले हरेक परिक्षण गरिएको निर्देशिकामा **0 जोखिमहरू** रिपोर्ट गर्यो।

#### npm निर्भरता जोखिम (अरु स्रोतहरूबाट) — मर्मत गरिएको

सबै १५ दर्ता गरिएको `package-lock.json` फाइलहरू अडिट गरियो। जोखिमहरू MCP Inspector विकसक उपकरण, OpenAI क्लाइन्ट, र MCP SDK द्वारा ल्याइएका चुनौतीपूर्ण निर्भरता सीमित थिए; सबै अझै नमूनाहरू नटुट्ने गरी समाधान गरिए:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** र **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) बढाइयो जसले बन्डल गरिएको `ajv`, `brace-expansion`, `diff`, `path-to-regexp` र `ws` सल्लाहहरू हटायो। `concurrently` बाट आएको बाँकी गम्भीर सल्लाहलाई हटाउन npm `overrides` प्रविष्टि थपियो `shell-quote@1.8.4` को पैच गरिएको संस्करणमा; दुवै लकफाइलहरू पुनः उत्पादित (अब 0 जोखिमहरू)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ले मध्यवर्ती `qs` लाई पैच गरिएको रिलिजमा अद्यावधिक गर्यो
- **03-GettingStarted/samples/javascript**: `npm audit fix` ले मध्यवर्ती `hono` लाई पैच गरिएको रिलिजमा अद्यावधिक गर्यो
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ले मध्यवर्ती `form-data` (उच्च) लाई पैच गरिएको रिलिजमा अद्यावधिक गर्यो
- **03-GettingStarted/11-simple-auth/solution/typescript**: छुटेको `package-lock.json` निर्माण गरियो ताकि परियोजना पुनरुत्पादनयोग्य र अडिटयोग्य होस् (० जोखिमहरू)

#### कोड-स्तर सुरक्षा सुधार (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` उपकरणबाट `shell=True` हटाइयो। पहिला `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` ले फोल्डर पथमा shell मेटाचरेक्तर्सलाई `cmd.exe` द्वारा व्याख्या गर्न अनुमति दिएको थियो (कमाण्ड-इन्जेक्शन जोखिम)। अब यो सिधै समाधान गरिएको `Code.exe` फोल्डरलाई तर्कको रूपमा दिएर सुरु गर्छ — कुनै shell छैन — जुन कार्यात्मक रूपमा समतुल्य र सुरक्षित छ

### सुरूवात खण्ड विस्तार – उन्नत सर्भर प्रयोग र सरल प्रमाणीकरण

#### उन्नत सर्भर प्रयोग (03-GettingStarted/10-advanced)
- **नयाँ अध्याय थपियो**: उन्नत MCP सर्भर प्रयोगको समग्र मार्गदर्शक प्रस्तुत गरियो, दुवै नियमित र निम्न-स्तर सर्भर संरचनाहरूलाई समेटी
  - **नियमित vs. निम्न-स्तर सर्भर**: दुवै प्रविधिहरूका लागि पाइथन र टाइपस्क्रिप्टमा विस्तृत तुलना र कोड उदाहरणहरू
  - **ह्यान्डलर-आधारित डिजाइन**: स्केलेबल, लचिलो सर्भर कार्यान्वयनका लागि ह्यान्डलर-आधारित उपकरण/स्रोत/प्रॉम्प्ट व्यवस्थापनको व्याख्या
  - **व्यावहारिक ढाँचाहरू**: जहाँ निम्न-स्तर सर्भर ढाँचाहरू उन्नत सुविधाहरू र वास्तुकलाका लागि लाभदायक छन् त्यस्ता वास्तविक संसारका परिदृश्यहरू

#### सरल प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नयाँ अध्याय थपियो**: MCP सर्भरहरूमा सरल प्रमाणीकरण लागू गर्ने चरण-द्वारा-चरण मार्गदर्शक
  - **प्रमाणीकरण अवधारणाहरू**: प्रमाणीकरण र प्राधिकरण बीच स्पष्ट व्याख्या र प्रमाणपत्र ह्यान्डलिंग
  - **मूल प्रमाणीकरण कार्यान्वयन**: पाइथन (Starlette) र टाइपस्क्रिप्ट (Express) मा मध्यस्थता-आधारित प्रमाणीकरण ढाँचाहरू, कोड नमूनाहरूसहित
  - **उन्नत सुरक्षामा प्रगति**: सरल प्रमाणीकरणबाट OAuth 2.1 र RBAC तर्फ प्रगति गर्ने मार्गदर्शन, उन्नत सुरक्षा मोड्युलहरूमा सन्दर्भहरू सहित

यी थपहरू भरपर्दो, सुरक्षित, र लचिलो MCP सर्भर कार्यान्वयनहरू निर्माण गर्न व्यावहारिक, ह्यान्ड्स-अन निर्देशनहरू प्रदान गर्दछन्, आधारभूत अवधारणाहरूलाई उन्नत उत्पादन ढाँचाहरूसँग जोड्दै।

#### MCP सुरक्षा सर्वोत्तम अभ्यासहरू (०२-सुरक्षा/) - पूर्ण आधुनिकीकरण
- **MCP-SECURITY-BEST-PRACTICES-2025.md**: MCP विनिर्देशन २०२५-०६-१८ अनुरूप पूर्ण पुनर्लेखन
  - **अनिवार्य आवश्यकता**: आधिकारिक विनिर्देशनबाट स्पष्ट MUST/MUST NOT आवश्यकताहरू यथार्थ दृश्य संकेतहरूसहित थपियो
  - **१२ मुख्य सुरक्षा अभ्यासहरू**: १५-वस्तुको सूचीबाट पूर्ण सुरक्षा डोमेनहरूमा पुनर्संरचना
    - टोकन सुरक्षा र प्रमाणीकरण बाह्य पहिचान प्रदायक समाकलनसहित
    - सत्र व्यवस्थापन र यातायात सुरक्षा क्रिप्टोग्राफिक आवश्यकताहरू सहित
    - AI-विशिष्ट खतरा संरक्षण Microsoft Prompt Shields समाकलनसहित
    - पहुँच नियन्त्रण र अनुमति न्यूनतम विशेषाधिकारको सिद्धान्तसहित
    - सामग्री सुरक्षा र अनुगमन Azure Content Safety समाकलनसहित
    - आपूर्ति श्रृंखला सुरक्षा समग्र कम्पोनेन्ट प्रमाणीकरणसहित
    - OAuth सुरक्षा र Confused Deputy रोकथाम PKCE कार्यान्वयनसहित
    - घटना प्रतिक्रिया र पुनरुत्थान स्वचालित क्षमता सहित
    - अनुपालन र शासन नियामक समायोजनसहित
    - उन्नत सुरक्षा नियन्त्रणहरू शून्य भरोसा वास्तुकलासहित
    - Microsoft सुरक्षा इकोसिस्टम समाकलन समग्र समाधानहरू सहित
    - सतत सुरक्षा विकास अनुकुल अभ्यासहरू सहित
  - **Microsoft सुरक्षा समाधानहरू**: Prompt Shields, Azure Content Safety, Entra ID, र GitHub Advanced Security को समेकित निर्देशन सुधारिएको
  - **कार्यान्वयन स्रोतहरू**: आधिकारिक MCP दस्तावेज, Microsoft सुरक्षा समाधानहरू, सुरक्षा मापदण्डहरू, र कार्यान्वयन मार्गदर्शनहरूको वर्गीकरण गरिएको समग्र स्रोत लिंकहरू

#### उन्नत सुरक्षा नियन्त्रणहरू (०२-सुरक्षा/) - उद्यम कार्यान्वयन
- **MCP-SECURITY-CONTROLS-2025.md**: उद्यम-स्तरको सुरक्षा फ्रेमवर्कसहित पूर्ण पुनर्गठन
  - **९ समग्र सुरक्षा डोमेनहरू**: आधारभूत नियन्त्रणहरूबाट विस्तारित गरी विस्तृत उद्यम फ्रेमवर्क
    - उन्नत प्रमाणीकरण र प्राधिकरण Microsoft Entra ID समाकलनसहित
    - टोकन सुरक्षा र एंटी-पासथ्रू नियन्त्रणहरू समग्र प्रमाणीकरणसहित
    - सत्र सुरक्षा नियन्त्रणहरू अपहरण रोकथामसहित
    - AI-विशिष्ट सुरक्षा नियन्त्रणहरू प्रम्प्ट इन्जेक्सन र उपकरण विषाक्तता रोकथामसहित
    - Confused Deputy आक्रमण रोकथाम OAuth प्रोक्सी सुरक्षा सहित
    - उपकरण कार्यान्वयन सुरक्षा स्यान्डबक्सिङ र पृथक्करणसहित
    - आपूर्ति श्रृंखला सुरक्षा नियन्त्रणहरू निर्भरता प्रमाणीकरणसहित
    - अनुगमन र पत्ता लगाउने नियन्त्रणहरू SIEM समाकलनसहित
    - घटना प्रतिक्रिया र पुनरुत्थान स्वचालित क्षमता सहित
  - **कार्यान्वयन उदाहरणहरू**: विस्तृत YAML कन्फिगरेसन ब्लक र कोड उदाहरणहरू थपियो
  - **Microsoft समाधान समाकलन**: Azure सुरक्षा सेवाहरू, GitHub Advanced Security, र उद्यम पहिचान व्यवस्थापनको व्यापक समावेशीकरण

#### उन्नत विषयहरू सुरक्षा (०५-AdvancedTopics/mcp-security/) - उत्पादन-तयार कार्यान्वयन
- **README.md**: उद्यम सुरक्षा कार्यान्वयनका लागि पूर्ण पुनर्लेखन
  - **वर्तमान विनिर्देशन अनुरूप**: MCP विनिर्देशन २०२५-०६-१८ मा अनुकूलित अनिवार्य सुरक्षा आवश्यकतासहित अद्यावधिक गरिएको
  - **सशक्त प्रमाणीकरण**: Microsoft Entra ID समाकलन समग्र .NET र Java Spring Security उदाहरणहरूसहित
  - **AI सुरक्षा समाकलन**: Microsoft Prompt Shields र Azure Content Safety कार्यान्वयन विस्तृत Python उदाहरणहरूसहित
  - **उन्नत खतरा न्यूनीकरण**: विस्तृत कार्यान्वयन उदाहरणहरूको लागि
    - Confused Deputy आक्रमण रोकथाम PKCE र प्रयोगकर्ता सहमति प्रमाणीकरणसहित
    - टोकन पासथ्रू रोकथाम दर्शक प्रमाणीकरण र सुरक्षित टोकन व्यवस्थापनसहित
    - सत्र अपहरण रोकथाम क्रिप्टोग्राफिक बन्धन र व्यवहार विश्लेषणसहित
  - **उद्यम सुरक्षा समाकलन**: Azure Application Insights अनुगमन, खतरा पत्ता लगाउने पाइपलाइनहरू, र आपूर्ति श्रृंखला सुरक्षा
  - **कार्यान्वयन जाँचसूची**: स्पष्ट अनिवार्य बनाम सिफारिस सुरक्षा नियन्त्रणहरू Microsoft सुरक्षा इकोसिस्टम लाभहरूसहित

### दस्तावेजीकरण गुणस्तर र मापदण्ड अनुरूपता
- **विनिर्देशन सन्दर्भहरू**: सबै सन्दर्भहरू अद्यावधिक गरी वर्तमान MCP विनिर्देशन २०२५-०६-१८ मा मिलाइएको
- **Microsoft सुरक्षा इकोसिस्टम**: सबै सुरक्षा दस्तावेजीकरणभर समेकन निर्देशन सुधारिएको
- **व्यावहारिक कार्यान्वयन**: विस्तृत कोड उदाहरणहरू थपिएका .NET, Java, र Python मा उद्यम ढाँचाहरूसहित
- **स्रोत संगठन**: आधिकारिक दस्तावेज, सुरक्षा मापदण्ड, र कार्यान्वयन मार्गदर्शिकाहरूको समग्र वर्गीकरण
- **दृश्य संकेतहरू**: अनिवार्य आवश्यकताहरू र सिफारिस अभ्यासहरू स्पष्ट चिन्हित


#### मुख्य अवधारणाहरू (०१-CoreConcepts/) - पूर्ण आधुनिकीकरण
- **प्रोटोकल संस्करण अद्यावधिक**: वर्तमान MCP विनिर्देशन २०२५-०६-१८ मा सन्दर्भित, मिति-आधारित संस्करण (YYYY-MM-DD ढाँचा)
- **वास्तुकला सुधार**: Hosts, Clients, र Servers को वर्णन सुधारिएको वर्तमान MCP वास्तुकला ढाँचाहरूसँग मेल खाने गरी
  - Hosts अब स्पष्ट रूपमा AI एप्लिकेसनहरू रूपमा परिभाषित जसले धेरै MCP क्लाइन्ट जडानहरू समन्वय गर्छन्
  - Clients एक-देखि-एक सर्भर सम्बन्ध कायम राख्ने प्रोटोकल जडानकर्ता रूपमा वर्णित
  - Servers स्थानीय र दूरस्थ कार्यान्वयन परिदृश्यहरूको साथ सुधारिएको
- **प्रिमिटिभ पुनर्संरचना**: सर्भर र क्लाइन्ट प्रिमिटिभहरूको पूर्ण पुनर्गठन
  - सर्भर प्रिमिटिभहरू: स्रोतहरू (डाटा स्रोतहरू), प्रॉम्प्टहरू (ढाँचा), उपकरणहरू (कार्यान्वयन योग्य कार्यहरू) विस्तृत व्याख्या र उदाहरणहरूसहित
  - क्लाइन्ट प्रिमिटिभहरू: स्याम्पलिङ (LLM पूर्तिहरू), प्रयास (प्रयोगकर्ता इनपुट), लगिङ (डिबगिङ/अनुगमन)
  - वर्तमान डिस्कवरी (`*/list`), पुनःप्राप्ति (`*/get`), र कार्यान्वयन (`*/call`) विधि ढाँचाहरू अपडेट गरिएको
- **प्रोटोकल वास्तुकला**: दुई-स्तर वास्तुकला मोडेल परिचय
  - डाटा तह: JSON-RPC 2.0 आधार जीवनचक्र प्रबंधन र प्रिमिटिभहरू सहित
  - यातायात तह: STDIO (स्थानीय) र Streamable HTTP SSE (दूरस्थ) यातायात संयन्त्रहरू
- **सुरक्षा फ्रेमवर्क**: स्पष्ट प्रयोगकर्ता सहमति, डाटा गोपनीयता संरक्षण, उपकरण कार्यान्वयन सुरक्षा, र यातायात तह सुरक्षा सहित समग्र सुरक्षा सिद्धान्तहरू
- **सञ्चार ढाँचाहरू**: प्रोटोकल सन्देशहरू अद्यावधिक गरेर आरम्भ, खोज, कार्यान्वयन, र सूचना प्रवाह देखाइएको
- **कोड उदाहरणहरू**: वर्तमान MCP SDK ढाँचाहरू प्रतिबिम्बित गर्ने बहु-भाषी उदाहरणहरू (.NET, Java, Python, JavaScript) ताजा गरिएको

#### सुरक्षा (०२-सुरक्षा/) - व्यापक सुरक्षा पुनर्गठन  
- **मापदण्ड अनुरूपता**: MCP विनिर्देशन २०२५-०६-१८ सुरक्षा आवश्यकतासँग पूर्ण मिलान
- **प्रमाणीकरण विकास**: कस्टम OAuth सर्भरहरूबाट बाह्य पहिचान प्रदायक प्रतिनिधित्व (Microsoft Entra ID) सम्म विकास दस्तावेजीकरण
- **AI-विशिष्ट खतरा विश्लेषण**: आधुनिक AI आक्रमण भेक्टरहरूको सामर्थ्य विस्तार
  - वास्तविक विश्व उदाहरणहरूसहित विस्तृत प्रम्प्ट इन्जेक्सन आक्रमण परिदृश्यहरू
  - उपकरण विषाक्तता यन्त्रहरू र "रग पुल" आक्रमण ढाँचाहरू
  - सन्दर्भ विन्डो विषाक्तता र मोडेल भ्रम आक्रमणहरू
- **Microsoft AI सुरक्षा समाधानहरू**: Microsoft सुरक्षा इकोसिस्टमको समग्र समावेशीकरण
  - AI Prompt Shields प्रगत पत्ता लगाउने, स्पटलाईटिङ, र सीमालेखक प्रविधिहरूसहित
  - Azure Content Safety समाकलन ढाँचाहरू
  - आपूर्ति श्रृंखला सुरक्षा लागि GitHub Advanced Security
- **उन्नत खतरा न्यूनीकरण**: विस्तृत सुरक्षा नियन्त्रणहरूका लागि
  - MCP-विशिष्ट सत्र अपहरण आक्रमण परिदृश्यहरू र क्रिप्टोग्राफिक सत्र ID आवश्यकताहरू सहित
  - MCP प्रोक्सी परिदृश्यहरूमा Confused Deputy समस्याहरू स्पष्ट सहमति आवश्यकताहरू सहित
  - टोकन पासथ्रू भल्नरेबिलिटीहरू आवश्यक प्रमाणीकरण नियन्त्रणहरूसहित
- **आपूर्ति श्रृंखला सुरक्षा**: AI आपूर्ति श्रृंखला कभरेज विस्तारित गरिएको, जसमा फाउन्डेशन मोडेलहरू, एम्बेडिङ सेवा, सन्दर्भ प्रदायकहरू, र तेस्रो-पक्ष API हरू
- **फाउन्डेशन सुरक्षा**: उद्यम सुरक्षा ढाँचाहरू समावेश गरेर सुधारिएको समाकलन, जसमा शून्य भरोसा वास्तुकला र Microsoft सुरक्षा इकोसिस्टम रहेका छन्
- **स्रोत संगठन**: प्रकार अनुसार वर्गीकृत व्यापक स्रोत लिंकहरू (आधिकारिक कागजातहरू, मापदण्डहरू, अनुसन्धान, Microsoft समाधानहरू, कार्यान्वयन मार्गनिर्देशनहरू)

### दस्तावेजीकरण गुणस्तर सुधारहरू
- **संरचित सिकाइ उद्देश्यहरू**: विशिष्ट, कार्यान्वयन योग्य परिणामहरू सहित सिकाइ उद्देश्यहरूको विस्तार गरिएको
- **पारस्परिक सन्दर्भहरू**: सम्बन्धित सुरक्षा र मुख्य अवधारणा विषयहरूबीच लिंकहरू थपिएका
- **वर्तमान सूचना**: सबै मिति सन्दर्भहरू र विनिर्देशन लिंकहरू वर्तमान मापदण्डहरूमा अद्यावधिक गरिएको
- **कार्यान्वयन मार्गदर्शन**: दुवै अनुभागहरुमा विशिष्ट, कार्यान्वयन योग्य मार्गदर्शन थपिएको

- यो समग्र परिवर्तन विवरण थपियो

### समुदाय योगदानहरू (०६-CommunityContributions/)
- छवि निर्माणका लागि MCP सर्भरहरूको विस्तृत जानकारी थपियो
- VSCode मा Claude प्रयोग गर्ने समग्र खण्ड थपियो
- Cline टर्मिनल क्लाइन्ट सेटअप र प्रयोग निर्देशहरू थपियो
- MCP क्लाइन्ट अनुभाग सबै लोकप्रिय क्लाइन्ट विकल्पहरू समावेश गर्न अद्यावधिक गरियो
- योगदान उदाहरणहरू अझ सटीक कोड नमुनाहरू सहित सुधारियो

### उन्नत विषयहरू (०५-AdvancedTopics/)
- सबै विशेषज्ञ विषय फोल्डरहरू सुसंगत नामकरणसहित व्यवस्था गरियो
- सन्दर्भ अभियांत्रिकी सामग्रीहरू र उदाहरणहरू थपियो
- Foundry एजेन्ट समाकलन दस्तावेजीकरण थपियो
- Entra ID सुरक्षा समाकलन दस्तावेजीकरण सुधारियो
