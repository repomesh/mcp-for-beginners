# MCP ਸੁਰੱਖਿਆ: ਏਆਈ ਸਿਸਟਮਾਂ ਲਈ ਵਿਸਤਾਰਕ ਸੁਰੱਖਿਆ  

[![MCP Security Best Practices](../../../translated_images/pa/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)  

_(ਇਸ ਪਾਠ ਦੀ ਵੀਡੀਓ ਵੇਖਣ ਲਈ ਉਪਰ ਦਿੱਤੀ ਤਸਵੀਰ 'ਤੇ ਕਲਿਕ ਕਰੋ)_  

ਸੁਰੱਖਿਆ ਏਆਈ ਸਿਸਟਮ ਡਿਜ਼ਾਈਨ ਦਾ ਮੁਢਲਾ ਤੱਤ ਹੈ, ਇਸ ਲਈ ਅਸੀਂ ਇਸ ਨੂੰ ਆਪਣੇ ਦੂਜੇ ਭਾਗ ਵਜੋਂ ਪ੍ਰਾਥਮਿਕਤਾ ਦੇਂਦੇ ਹਾਂ। ਇਹ ਮਾਇਕ੍ਰੋਸੌਫਟ ਦੇ **ਸੁਰੱਖਿਅਤ ਡਿਜ਼ਾਈਨ ਦੁਆਰਾ** ਸਿਧਾਂਤ ਨਾਲ ਮੇਲ ਖਾਂਦਾ ਹੈ ਜੋ [ਸੇਕਿਊਰ ਫਿਊਚਰ ਇਨੀਸ਼ੀਏਟਿਵ](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) ਤੋਂ ਹੈ।  

ਮਾਡਲ ਕਾਂਟੈਕਸਟ ਪ੍ਰੋਟੋਕੋਲ (MCP) ਏਆਈ-ਸංਚਾਲਿਤ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਸ਼ਕਤੀਸ਼ਾਲੀ ਨਵੀਆਂ ਸਮਰੱਥਾਵਾਂ ਲਿਆਉਂਦਾ ਹੈ, ਜਦਕਿ ਰਵਾਇਤੀ ਸਾਫਟਵੇਅਰ ਖਤਰਿਆਂ ਤੋਂ ਅੱਗੇ ਵਧ ਕੇ ਵਿਲੱਖਣ ਸੁਰੱਖਿਆ ਚੁਣੌਤੀਆਂ ਵੀ ਪੇਸ਼ ਕਰਦਾ ਹੈ। MCP ਸਿਸਟਮਾਂ ਨੂੰ ਸਥਾਪਿਤ ਸੁਰੱਖਿਆ ਚਿੰਤਾਵਾਂ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ, ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ) ਦੇ ਨਾਲ-ਨਾਲ ਨਵੇਂ ਏਆਈ-ਖਾਸ ਖਤਰਿਆਂ ਦਾ ਸਾਹਮਣਾ ਕਰਨਾ ਪੈਂਦਾ ਹੈ, ਜਿਸ ਵਿੱਚ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਹਮਲੇ, ਟੋਕਨ ਪਾਸਥਰੂ ਦੁਰਬਲਤਾਵਾਂ ਅਤੇ ਗਤੀਸ਼ੀਲ ਸਮਰੱਥਾ ਸੋਧ ਸ਼ਾਮਲ ਹਨ।  

ਇਸ ਪਾਠ ਵਿੱਚ MCP ਲਾਗੂ ਕਰਦੇ ਸਮੇਂ ਸਭ ਤੋਂ ਜ਼ਰੂਰੀ ਸੁਰੱਖਿਆ ਖਤਰਿਆਂ ਦੀ ਜਾਂਚ ਕੀਤੀ ਗਈ ਹੈ—ਜਿਸ ਵਿੱਚ ਪ੍ਰਮਾਣਿਕਤਾ, ਪ੍ਰਮਾਣਿਕਰਨ, ਅਤਿ-ਅਧਿਕਾਰ, ਪ੍ਰਤੱਖ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ, ਸੈਸ਼ਨ ਸੁਰੱਖਿਆ, ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪ੍ਰਬੰਧਨ ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਦੇ ਖਤਰਿਆਂ ਨੂੰ ਕਵਰ ਕੀਤਾ ਗਿਆ ਹੈ। ਤੁਸੀਂ ਇਹ ਸਿੱਖੋਗੇ ਕਿ ਕਿਵੇਂ ਇਹਨਾਂ ਖਤਰਿਆਂ ਨੂੰ ਘਟਾਉਣ ਲਈ ਕਾਰਗਰ ਨਿਯੰਤਰਣ ਅਤੇ ਸਭ ਤੋਂ ਵਧੀਆ ਕਾਇਦੇ-ਕਾਨੂੰਨ ਅਪਣਾਏ ਜਾ ਸਕਦੇ ਹਨ, ਜਿੱਥੇ ਮਾਇਕ੍ਰੋਸੌਫਟ ਦੇ ਹੱਲਾਂ ਜਿਵੇਂ ਪ੍ਰੰਪਟ ਸ਼ੀਲਡਜ਼, ਅਜ਼ੂਰੇ ਕੰਟੈਂਟ ਸੇਫਟੀ ਅਤੇ ਗਿੱਟਹੱਬ ਐਡਵਾਂਸਡ ਸੁਰੱਖਿਆ ਤੁਹਾਡੇ MCP ਡਿਪਲੌਇਮੈਂਟ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਦੇ ਹਨ।  

## ਸਿੱਖਣ ਦੇ ਲਕੜ  

ਇਸ ਪਾਠ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:  

- **MCP-ਖ਼ਾਸ ਖਤਰਿਆਂ ਦੀ ਪਹਿਚਾਣ ਕਰੋ**: MCP ਸਿਸਟਮਾਂ ਵਿੱਚ ਵਿਲੱਖਣ ਸੁਰੱਖਿਆ ਖਤਰਿਆਂ ਨੂੰ ਸਮਝੋ, ਜਿਸ ਵਿੱਚ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ, ਅਤਿ-ਅਧਿਕਾਰ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪਾਸਥਰੂ ਦੁਰਬਲਤਾਵਾਂ ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਦੇ ਖਤਰਿਆਂ ਸ਼ਾਮਲ ਹਨ  
- **ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰੋ**: ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਰੋਕथਾਮਾਂ ਦਾ ਇੰਪਲਿਮੈਂਟ ਕਰੋ, ਜਿਸ ਵਿੱਚ ਮਜ਼ਬੂਤ ਪ੍ਰਮਾਣਿਕਤਾ, ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਐਕਸੈੱਸ, ਸੁਰੱਖਿਅਤ ਟੋਕਨ ਪ੍ਰਬੰਧਨ, ਸੈਸ਼ਨ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਪੁਸ਼ਟੀਕਰਨ ਸ਼ਾਮਲ ਹੈ  
- **ਮਾਇਕ੍ਰੋਸੌਫਟ ਸੁਰੱਖਿਆ ਹੱਲਾਂ ਦੀ ਵਰਤੋਂ ਕਰੋ**: MCP ਵਾਲ੍ਕਲੋਡ ਦੀ ਰੱਖਿਆ ਲਈ ਮਾਇਕ੍ਰੋਸੌਫਟ ਪ੍ਰੰਪਟ ਸ਼ੀਲਡਜ਼, ਅਜ਼ੂਰੇ ਕੰਟੈਂਟ ਸੇਫਟੀ ਅਤੇ ਗਿੱਟਹੱਬ ਐਡਵਾਂਸਡ ਸੁਰੱਖਿਆ ਨੂੰ ਸਮਝੋ ਅਤੇ ਲਾਗੂ ਕਰੋ  
- **ਟੂਲ ਸੁਰੱਖਿਆ ਨੂੰ ਵੈਰੀਫਾਈ ਕਰੋ**: ਟੂਲ ਮੈਟਾਡੇਟਾ ਵੈਰੀਫਿਕੇਸ਼ਨ, ਡਾਇਨਾਮਿਕ ਬਦਲਾਵਾਂ ਦੀ ਮਾਨੀਟਰਨਗ ਅਤੇ ਪਰੋਕ्सੀ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ ਹਮਲਿਆਂ ਤੋਂ ਬਚਾਅ ਕਰਨ ਦੇ ਮਹੱਤਵ ਨੂੰ ਜਾਣੋ  
- **ਸਰੋਤ ਸਰਭਾਜਨ ਅਭਿਆਸਾਂ ਨਾਲ ਇੰਟੈਗ੍ਰੇਟ ਕਰੋ**: ਸਥਾਪਿਤ ਸੁਰੱਖਿਆ ਮੂਲ(such as secure coding, server hardening, zero trust) ਨੂੰ MCP-ਖਾਸ ਨਿਯੰਤਰਣਾਂ ਨਾਲ ਜੋੜੋ ਤਾਂ ਜੋ ਵਿਆਪਕ ਸੁਰੱਖਿਆ ਮਿਲੇ  

# MCP ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰ ਅਤੇ ਨਿਯੰਤਰਣ  

ਆਧੁਨਿਕ MCP ਲਾਗੂਆਂ ਨੂੰ ਪਰਤਦਾਰ ਸੁਰੱਖਿਆ ਤਰੀਕਿਆਂ ਦੀ ਲੋੜ ਹੈ ਜੋ ਰਵਾਇਤੀ ਸਾਫਟਵੇਅਰ ਸੁਰੱਖਿਆ ਅਤੇ ਏਆਈ ਖ਼ਾਸ ਖਤਰਿਆਂ ਦੋਹਾਂ ਨੂੰ ਸੰਬੋਧਨ ਕਰਦੇ ਹਨ। ਤੇਜ਼ੀ ਨਾਲ ਬਦਲ ਰਹੀ MCP ਵਿਸ਼ੇਸ਼ਣ ਆਪਣੇ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਿਹਤਰ ਕਰ ਰਹੀ ਹੈ, ਜਿਸ ਨਾਲ ਏਨਟਰਪਰਾਈਜ਼ ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰਾਂ ਅਤੇ ਮੰਨੀਹੂਈਆਂ ਸਭ ਤੋਂ ਵਧੀਆਂ ਅਭਿਆਸਾਂ ਨਾਲ ਵਧੀਆ ਇੰਟੈਗ੍ਰੇਸ਼ਨ ਹੋ ਸਕਦਾ ਹੈ।  

[Microsoft Digital Defense Report](https://aka.ms/mddr) ਵਿੱਚ ਕੀਤੇ ਗਏ 리ਸਰਚ ਦਿਖਾਉਂਦਾ ਹੈ ਕਿ **ਰਿਪੋਰਟ ਕੀਤੀਆਂ 98% ਲੰਘਣਾ ਭਲਕੇ ਸੁਰੱਖਿਅਤ ਸਫਾਈ ਨਾਲ ਰੋਕੀਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ**। ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਸੁਰੱਖਿਆ ਰਣਨੀਤੀ ਮੂਲ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ ਨੂੰ MCP-ਖਾਸ ਨਿਯੰਤਰਣਾਂ ਨਾਲ ਜੋੜਦੀ ਹੈ—ਸਾਬਤ ਬੇਸਲਾਈਨ ਸੁਰੱਖਿਆ ਉਪਾਅ ਅਜੇ ਵੀ ਸਮੂਹ ਸੁਰੱਖਿਆ ਖਤਰੇ ਨੂੰ ਘਟਾਉਣ ਵਿੱਚ ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਹਨ।  

## ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਦ੍ਰਿਸ਼  

> **ਨੋਟ:** ਇਹ ਜਾਣਕਾਰੀ MCP ਸੁਰੱਖਿਆ ਮਿਆਰਾਂ ਨੂੰ ਫਰਵਰੀ 5, 2026 ਤੱਕ ਦਰਸਾਉਂਦੀ ਹੈ, ਜੋ MCP Specification 2025-11-25 ਦੇ ਨਾਲ ਮੇਲ ਖਾਂਦੀ ਹੈ। MCP ਪ੍ਰੋਟੋਕੋਲ ਤੇਜ਼ੀ ਨਾਲ ਵਿਕਸਤ ਹੋ ਰਿਹਾ ਹੈ ਅਤੇ ਭਵਿੱਖੀ ਲਾਗੂਆਂ ਵਿੱਚ ਨਵੇਂ ਪ੍ਰਮਾਣਿਕਤਾ ਪੈਟਰਨ ਅਤੇ ਸੁਧਾਰਤ ਨਿਯੰਤਰਣ ਸ਼ਾਮਲ ਹੋ ਸਕਦੇ ਹਨ। ਹਮੇਸ਼ਾਂ ਤਾਜ਼ਾ ਮਦਦ ਲਈ ਮੌਜੂਦਾ [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), ਅਤੇ [ਸੁਰੱਖਿਆ ਸਭ ਤੋਂ ਵਧੀਆ ਕਾਇਦੇ-ਕਾਨੂੰਨ ਦੀ ਡੌਕੂਮੈਂਟੇਸ਼ਨ](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ਨੂੰ ਵੇਖੋ।  

> **ਅਗਲੇ ਇਕੱਤਰਾਂ ਨੂੰ ਦੇਖਦੇ ਹੋਏ:** `2026-07-28` ਰਿਲੀਜ਼ ਕੈਂਡਿਡੇਟ ਅਧਿਕਾਰ ਵਧਾਉਂਦਾ ਹੈ — ਗ੍ਰਾਹਕਾਂ ਨੂੰ ਅਧਿਕਾਰ ਜਵਾਬਾਂ `iss` ਪੈਰਾਮੀਟਰ ਨੂੰ ਵੈਰੀਫਾਈ ਕਰਨਾ ਲਾਜ਼ਮੀ ਹੈ (RFC 9207), ਡਾਇਨਾਮਿਕ ਕਲਾਇੰਟ ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਦੌਰਾਨ OpenID Connect ਦਾ `application_type` ਘੋਸ਼ਿਤ ਕਰਨਾ ਅਤੇ ਰਜਿਸਟਰਡ ਕ੍ਰੈਡੈਂਸ਼ਲ ਨੂੰ ਜਾਰੀ ਕਰਨ ਵਾਲੇ ਅਧਿਕਾਰ ਸਰਵਰ ਨਾਲ ਜੁੜਨਾ। ਪੂਰੀ ਸੂਚੀ ਲਈ [What's Changing in MCP: The 2026-07-28 Release Candidate](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) ਵੇਖੋ।  

## 🏔️ MCP ਸੁਰੱਖਿਆ ਸਮਿਟ ਵਰਕਸ਼ਾਪ (ਸ਼ੇਰਪਾ)  

**ਪ੍ਰਯੋਗਕਾਰੀ ਸੁਰੱਖਿਆ ਟਰੇਨਿੰਗ** ਲਈ, ਅਸੀਂ ਬਹੁਤ ਸਿਫਾਰਸ਼ ਕਰਦੇ ਹਾਂ **MCP Security Summit Workshop** (ਸ਼ੇਰਪਾ) - ਮਾਇਕ੍ਰੋਸੌਫਟ ਅਜ਼ੂਰੇ ਵਿੱਚ MCP ਸਰਵਰਾਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਕਰਨ ਲਈ ਇੱਕ ਵਿਸਤਾਰਕ ਮਾਰਗਦਰਸ਼ਿਤ ਰਾਹਦਾਰੀ।  

### ਵਰਕਸ਼ਾਪ ਦਾ ਜਾਇਜ਼ਾ  

[MCP Security Summit Workshop](https://azure-samples.github.io/sherpa/) ਇੱਕ ਪ੍ਰਯੋਗਕਾਰੀ ਅਤੇ ਕਾਰਗਰ ਸੁਰੱਖਿਆ ਟਰੇਨਿੰਗ ਦਿੰਦਾ ਹੈ ਜੋ “ਦੁਰਬਲ → ਐਕਸਪਲੋਇਟ → ਫਿਕਸ → ਚੈੱਕ” ਵਿਧੀ ਤੇ ਆਧਾਰਿਤ ਹੈ। ਤੁਸੀਂ:  

- **ਤੁਟੂ-ਫੁਟੂ ਕਰਕੇ ਸਿੱਖੋ**: ਜ਼ਾਹਿਰ ਤੌਰ 'ਤੇ ਉਦਾਸੀਨ ਸਰਵਰਾਂ ਨੂੰ ਹਮਲਿਆਂ ਦੇ ਕੇ ਦੁર્શਾਂਤ ਕਰੋ  
- **ਅਜ਼ੂਰੇ ਦੇ ਨੈਟਿਵ ਸੁਰੱਖਿਆ ਸੰਦ ਵਰਤੋ**: ਅਜ਼ੂਰੇ ਏਂਟਰਾ ਆਈਡੀ, ਕੀ ਵੌਲਟ, ਏਪੀਆਈ ਪ੍ਰਬੰਧਨ, ਅਤੇ ਏਆਈ ਕੰਟੈਂਟ ਸੇਫਟੀ ਦੀ ਵਰਤੋਂ ਕਰੋ  
- **ਡਿਫੈਂਸ-ਇਨ-ਡੈਪਥ ਅਭਿਆਸਾਂ ਨੂੰ ਫਾਲੋ ਕਰੋ**: ਕੈਂਪਾਂ ਵਿੱਚ ਇੰਨ੍ਹਾਂ ਸੁਰੱਖਿਆ ਪਰਤਾਂ ਨੂੰ ਬਣਾਉਣ ਲਈ ਤਰੱਕੀ ਕਰੋ  
- **OWASP ਮਿਆਰ ਲਾਗੂ ਕਰੋ**: ਹਰ ਤਕਨੀਕ [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) ਨਾਲ ਮੇਲ ਖਾਂਦੀ ਹੈ  
- **ਉਤਪਾਦਨ ਕੋਡ ਲੈ ਕੇ ਜਾਓ**: ਕਾਰਗਰ ਅਤੇ ਟੈਸਟ ਕੀਤੇ ਗਏ ਲਾਗੂ ਕਰਨ ਵਾਲੇ ਹੱਲ ਨਾਲ

### MCP ਪ੍ਰਮਾਣਿਕਤਾ ਦਾ ਵਿਕਾਸ  

MCP ਵਿਸ਼ੇਸ਼ਣ ਨੇ ਪ੍ਰਮਾਣਿਕਤਾ ਤੇ ਅਧਿਕਾਰਣ ਦੇ ਤਰੀਕਿਆਂ ਵਿੱਚ ਕਾਫੀ ਵਿਕਾਸ ਕੀਤਾ ਹੈ:  

- **ਮੂਲ ਤਰੀਕਾ**: ਪਹਿਲੀ ਵਿਸ਼ੇਸ਼ਣਾਂ ਵਿੱਚ ਵਿਕਾਸਕਾਰਾਂ ਨੂੰ ਆਪਣੇ ਗਾਹਕ ਪ੍ਰਮਾਣਿਕਤਾ ਸਰਵਰ ਬਣਾਉਣੇ ਪੈਂਦੇ ਸਨ, MCP ਸਰਵਰ OAuth 2.0 ਅਧਿਕਾਰਣ ਸਰਵਰ ਵਜੋਂ ਕੰਮ ਕਰਦੇ ਸਨ ਅਤੇ ਸਿੱਧਾ ਯੂਜ਼ਰ ਪ੍ਰਮਾਣਿਕਤਾ ਸੰਭਾਲਦੇ ਸਨ  
- **ਮੌਜੂਦਾ ਮਿਆਰ (2025-11-25)**: ਅੱਪਡੇਟ ਵਿਸ਼ੇਸ਼ਣ MCP ਸਰਵਰਾਂ ਨੂੰ ਬਾਹਰੀ ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾਵਾਂ (ਜਿਵੇਂ ਮਾਇਕ੍ਰੋਸੌਫਟ ਏਂਟਰਾ ਆਈਡੀ) ਨੂੰ ਪ੍ਰਮਾਣਿਕਤਾ ਸੌਂਪਣ ਦੀ ਆਗਿਆ ਦਿੰਦੀ ਹੈ, ਜਿਸ ਨਾਲ ਸੁਰੱਖਿਆ ਦੀ ਸਥਿਤੀ ਬੇਹਤਰ ਹੁੰਦੀ ਹੈ ਅਤੇ ਲਾਗੂ ਕਰਨਾ ਆਸਾਨ ਬਣਦਾ ਹੈ  
- **ਟ੍ਰਾਂਸਪੋਰਟ ਲੇਅਰ ਸੁਰੱਖਿਆ**: ਸਥਾਨਕ (STDIO) ਅਤੇ ਰਿਮੋਟ (Streamable HTTP) ਕਨੈਕਸ਼ਨਾਂ ਲਈ ਢੁੱਕਵੀਂ ਪ੍ਰਮਾਣਿਕਤਾ ਮਾਡਲਾਂ ਨਾਲ ਸੁਰੱਖਿਅਤ ਟ੍ਰਾਂਸਪੋਰਟ ਮਕੈਨਿਜ਼ਮ ਦੇ ਲਈ ਵਧਾਇਆ ਗਿਆ ਸਹਿਯੋਗ  

## ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਪ੍ਰਮਾਣਿਕਰਨ ਸੁਰੱਖਿਆ  

### ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਚੁਣੌਤੀਆਂ  

ਆਧੁਨਿਕ MCP ਲਾਗੂਆਂ ਨੂੰ ਕਈ ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਪ੍ਰਮਾਣਿਕਰਨ ਸਮੱਸਿਆਵਾਂ ਦਾ ਸਾਹਮਣਾ ਹੈ:  

### ਖਤਰਿਆਂ ਅਤੇ ਹਮਲਾ ਵੈਕਟਰ  

- **ਗਲਤ ਤਰੀਕੇ ਨਾਲ ਅਧਿਕਾਰਣ ਲੋਗਿਕ**: MCP ਸਰਵਰਾਂ ਵਿੱਚ ਖ਼ਰਾਬ ਅਧਿਕਾਰਣ ਲਾਗੂ ਕਰਨਾ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਨੂੰ ਦਰਸਾ ਸਕਦਾ ਹੈ ਅਤੇ ਐਕਸੈੱਸ ਨਿਯੰਤਰਣ ਗਲਤ ਲਾਗੂ ਹੋ ਸਕਦੇ ਹਨ  
- **OAuth ਟੋਕਨ ਚੋਰੀ**: ਸਥਾਨਕ MCP ਸਰਵਰ ਦਾ ਟੋਕਨ ਚੋਰੀ ਕਰਨ ਨਾਲ ਹਮਲਾਵਰ ਸਰਵਰਾਂ ਦੀ ਨਕਲ ਕਰ ਸਕਦੇ ਹਨ ਅਤੇ ਡਾਊਨ ਸਟਰੀਮ ਸਭਾਵਾਂ ਤੱਕ ਪਹੁੰਚ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦੇ ਹਨ  
- **ਟੋਕਨ ਪਾਸਥਰੂ ਦੁਰਬਲਤਾਵਾਂ**: ਗਲਤ ਟੋਕਨ ਹੈਂਡਲਿੰਗ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਬਾਈਪਾਸ ਅਤੇ ਜ਼ਿੰਮੇਵਾਰੀ ਖ਼ਤਮ ਕਰ ਸਕਦੀ ਹੈ  
- **ਅਤਿ-ਅਧਿਕਾਰ**: MCP ਸਰਵਰ ਜੋ ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਦੇ ਸਿਧਾਂਤ ਦਾ ਉਲੰਘਨ ਕਰਦੇ ਹਨ ਅਤੇ ਹਮਲੇ ਦੇ ਸਤਹਾਂ ਨੂੰ ਵਧਾਉਂਦੇ ਹਨ  

#### ਟੋਕਨ ਪਾਸਥਰੂ: ਇੱਕ ਮੁੱਖ ਖ਼ਿਲਾਫ਼-ਨਮੂਨਾ  

**ਟੋਕਨ ਪਾਸਥਰੂ ਮੌਜੂਦਾ MCP ਅਧਿਕਾਰਣ ਵਿਸ਼ੇਸ਼ਣ ਵਿੱਚ ਸਪਸ਼ਟ ਤੌਰ ਤੇ ਮਨਾਹੀ ਹੈ** ਕਿਉਂਕਿ ਇਸਦੇ ਗੰਭੀਰ ਸੁਰੱਖਿਆ ਪ੍ਰਭਾਵ ਹਨ:  

##### ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਤਿਆਗਣਾ  
- MCP ਸਰਵਰ ਅਤੇ ਡਾਊਨਸਟਰੀਮ API ਸੰਵਿਧਾਨਿਕ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ (ਰੇਟ ਲਿਮਿਟਿੰਗ, ਬੇਨਤੀ ਵੈਰੀਫਿਕੇਸ਼ਨ, ਟ੍ਰੈਫਿਕ ਮਾਨੀਟਰਿੰਗ) ਨੂੰ ਲਾਗੂ ਕਰਦੇ ਹਨ ਜੋ ਸਹੀ ਟੋਕਨ ਵੈਰੀਫਿਕੇਸ਼ਨ 'ਤੇ ਨਿਰਭਰ ਕਰਦੇ ਹਨ  
- ਸਿੱਧਾ ਕਲਾਇੰਟ-ਟੁ-API ਟੋਕਨ ਵਰਤੋਂ ਇਹ ਅਹੰਕਾਰਕ ਸੁਰੱਖਿਆ ਬਚਾਅਆਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੀ ਹੈ ਅਤੇ ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰ ਨੂੰ ਨੁਕਸਾਨ ਪੁਚਾਉਂਦੀ ਹੈ  

##### ਜ਼ਿੰਮੇਵਾਰੀ ਅਤੇ ਆਡਿਟ ਚੁਣੌਤੀਆਂ  
- MCP ਸਰਵਰ ਅਪਸਟਰੀਮ ਜਾਰੀ ਕੀਤੇ ਗਏ ਟੋਕਨਾਂ ਵਾਲੇ ਕਲਾਇੰਟਾਂ ਵਿੱਚ ਫਰਕ ਨਹੀਂ ਕਰ ਸਕਦੇ, ਜਿਸ ਨਾਲ ਆਡਿਟ ਟ੍ਰੇਲ ਟੁੱਟ ਜਾਂਦਾ ਹੈ  
- ਡਾਊਨਸਟਰੀਮ ਸਰੋਤ ਸਰਵਰ ਲੌਗ MCP ਸਰਵਰ ਮੱਝਲੀ ਪਰਤ ਨਾਲੋਂ ਉਮੀਦ ਵਿਰੁੱਧ ਬੇਨਤੀ ਦੇ ਮੂਲ ਦਰਸਾਉਂਦੇ ਹਨ  
- ਘਟਨਾ ਦੀ ਜਾਂਚ ਅਤੇ ਅਨੁਕੂਲਤਾ ਆਡਿਟਿੰਗ ਨੂੰ ਕਾਫੀ ਦਰਦਨਾਕ ਬਣਾ ਦਿੰਦਾ ਹੈ  

##### ਡਾਟਾ ਚੋਰੀ ਦੇ ਖਤਰੇ  
- ਅਣਵੈਰੀਫਾਈਡ ਟੋਕਨ ਦਾਅਵੇ ਨਾਲ ਨਕਲੀ ਟੋਕਨਾਂ ਵਾਲੇ ਦੁਸ਼ਮਣ MCP ਸਰਵਰਾਂ ਨੂੰ ਡਾਟਾ ਚੋਰੀ ਲਈ ਪ੍ਰੋਕਸੀ ਵਜੋਂ ਵਰਤ ਸਕਦੇ ਹਨ  
- ਭਰੋਸਾ ਸੀਮਾ ਦੇ ਉਲੰਘਣ ਅਣਔਥੋਰਾਈਜ਼ਡ ਪਹੁੰਚ ਪੈਟਰਨ ਦੀ ਇਜਾਜ਼ਤ ਦਿੰਦੇ ਹਨ ਜੋ ਮੰਤਵੀਤ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੇ ਹਨ  

##### ਬਹੁ-ਸੇਵਾ ਹਮਲਿਆਂ ਦੇ ਰਾਹ  
- ਗਲਤ ਟੋਕਨ ਕਈ ਸੇਵਾਵਾਂ ਵੱਲੋਂ ਮੰਨਿਆ ਜਾਣਾ ਲਗਾਤਾਰ ਪ੍ਰणਾਲੀਆਂ ਵਿੱਚ ਪਾਸੇ ਖਿਸਕਣ ਦੀ ਆਗਿਆ ਦਿੰਦਾ ਹੈ  
- ਜਦੋਂ ਟੋਕਨ ਮੂਲਾਂ ਦੀ ਪਛਾਣ ਸਹੀ ਨਾ ਹੋਵੇ ਤਾਂ ਸੇਵਾਵਾਂ ਦਰਮਿਆਨ ਭਰੋਸੇ ਦੇ ਅਨੁਮਾਨ ਤੋੜੇ ਜਾ ਸਕਦੇ ਹਨ

## ਏਆਈ-ਖਾਸ ਸੁਰੱਖਿਆ ਖਤਰੇ  

### ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ ਅਤੇ ਟੂਲ ਮਨਿਪੂਲੇਸ਼ਨ ਹਮਲੇ  

ਆਧੁਨਿਕ MCP ਲਾਗੂਆਂ ਨੂੰ ਐਸੀ ਏਆਈ ਖ਼ਾਸ ਹਮਲਿਆਂ ਦਾ ਸਾਹਮਣਾ ਹੈ ਜਿਨ੍ਹਾਂ ਨੂੰ ਰਵਾਇਤੀ ਸੁਰੱਖਿਆ ਉਪਾਅ ਪੂਰੀ ਤਰ੍ਹਾਂ ਤੋਂ ਸੰਭਾਲ ਨਹੀਂ ਸਕਦੇ:  

#### **ਪਰੋਕਸੀ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ (ਕ੍ਰਾਸ-ਡੋਮੇਨ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ)**  

**ਪਰੋਕਸੀ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ** MCP ਯੋਗ ਏਆਈ ਸਿਸਟਮਾਂ ਵਿੱਚ ਸਭ ਤੋਂ ਗੰਭੀਰ ਦੁਰਬਲਤਾਵਾਂ ਵਿੱਚੋਂ ਇੱਕ ਹੈ। ਹਮਲਾਵਰ ਬਾਹਰੀ ਸਮੱਗਰੀ ਵਿੱਚ ਹਾਨਿਕਾਰਕ ਹੁਕਮ ਛੁਪਾ ਕੇ (ਜਿਵੇਂ ਦਸਤਾਵੇਜ਼, ਵੈੱਬ ਪੇਜ, ਈਮੇਲ ਜਾਂ ਡਾਟਾ ਸਰੋਤ), ਏਆਈ ਸਿਸਟਮਾਂ ਨੂੰ ਠੁਕਵੀਂਆਂ ਹੁਕਮਾਂ ਵਜੋਂ ਸੰਭਾਲਣ ਲੱਗਦੇ ਹਨ।  

**ਹਮਲੇ ਦੇ ਮਾਮਲੇ:**  
- **ਦਸਤਾਵੇਜ਼ ਅਧਾਰਿਤ ਇੰਜੈਕਸ਼ਨ**: ਸੰਬੰਧਿਤ ਦਸਤਾਵੇਜ਼ਾਂ ਵਿੱਚ ਲੁਕਿਆ ਹਾਨਿਕਾਰਕ ਹੁਕਮ ਜੋ ਨਾਦਾਨਸੀ ਦੇ ਕਾਰਜ ਕਰਨ ਵਾਲੇ ਏਆਈ ਕਾਰਜ ਚਾਲੂ ਕਰਦੇ ਹਨ  
- **ਵੈੱਬ ਸਮੱਗਰੀ ਦੁਰਵਰਤੋਂ**: ਸੰਕ੍ਰਮਿਤ ਵੈੱਬ ਸਫ਼ੇ ਜਿਹੜੇ ਮੋਮੀ ਪ੍ਰੰਪਟ ਜୁੜੇ ਹੁੰਦੇ ਹਨ ਜੋ ਏਆਈ ਵਰਤਾਰਾ ਦੌਰਾਨ ਹੇਠਾਂ ਲਿਆਂਦੇ ਜਾਂਦੇ ਹਨ  
- **ਈਮੇਲ ਅਧਾਰਿਤ ਹਮਲੇ**: ਈਮੇਲਾਂ ਵਿਚ ਲੁਕਾਏ ਹਾਨਿਕਾਰਕ ਪ੍ਰੰਪਟ ਜੋ ਏਆਈ ਸਹਾਇਕਾਂ ਨੂੰ ਜਾਣਕਾਰੀ ਲੀਕ ਕਰਨ ਜਾਂ ਬਿਨਾਂ ਅਧਿਕਾਰ ਦੇ ਕੰਮ ਕਰਨ ਲਈ ਮਜ਼ਬੂਰ ਕਰਦੇ ਹਨ  
- **ਡਾਟਾ ਸਰੋਤ ਦੀ ਗੰਦਗੀ**: ਸੰਕ੍ਰਮਿਤ ਡੇਟਾਬੇਸ ਜਾਂ API, ਜੋ ਹਾਨੀਕਾਰਕ ਸਮੱਗਰੀ ਏਆਈ ਨੂੰ ਪ੍ਰਦਾਨ ਕਰਦੇ ਹਨ  

**ਅਸਲ ਦੁਨੀਆਂ 'ਤੇ ਪ੍ਰਭਾਵ**: ਇਹ ਹਮਲੇ ਡਾਟਾ ਚੋਰੀ, ਪ੍ਰਾਈਵੇਸੀ ਦੀਆਂ ਉਲੰਘਣਾਂ, ਨੁਕਸਾਨਦਾਇਕ ਸਮੱਗਰੀ ਦਾ ਸਿਰਜਣ ਅਤੇ ਯੂਜ਼ਰ ਇੰਟਰਐਕਸ਼ਨਾਂ ਦੀ ਚਾਲਾਕੀ ਨਾਲ ਮੈਨਪੁਲੇਸ਼ਨ ਦਾ ਕਾਰਨ ਬਣ ਸਕਦੇ ਹਨ। ਵਿਸਥਾਰ ਲਈ ਦੇਖੋ [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).  

![Prompt Injection Attack Diagram](../../../translated_images/pa/prompt-injection.ed9fbfde297ca877.webp)  

#### **ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ ਹਮਲੇ**  

**ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ** MCP ਟੂਲਾਂ ਦੇ ਮੈਟਾਡੇਟਾ ਨੂੰ ਨਿਸ਼ਾਨਾ ਬਣਾਉਂਦਾ ਹੈ, ਜੋ LLMs ਟੂਲ ਵੇਰਵਿਆਂ ਅਤੇ ਪੈਰਾਮੀਟਰਾਂ ਨੂੰ ਕਿਵੇਂ ਸਮਝਦੇ ਹਨ ਅਤੇ ਕਾਰਜ ਕਰਦੇ ਹਨ, ਨਾਲ ਖੇਡਦਾ ਹੈ।  

**ਹਮਲੇ ਦੇ ਤਰੀਕੇ:**  
- **ਮੈਟਾਡੇਟਾ ਦੀ ਚਾਲਾਕੀ ਨਾਲ ਸੋਧ**: ਹਮਲਾਵਰ ਟੂਲ ਦੇ ਵੇਰਵਿਆਂ, ਪੈਰਾਮੀਟਰ ਪਰਿਭਾਸ਼ਾਵਾਂ ਜਾਂ ਵਰਤੋਂ ਉੱਦਾਹਰਣਾਂ ਵਿੱਚ ਹਾਨਿਕਾਰਕ ਹੁਕਮ ਸ਼ਾਮਲ ਕਰਦੇ ਹਨ  
- **ਅਦ੍ਰਿਸ਼ਟ ਹੁਕਮ**: ਨਜ਼ਰ ਨਾ ਆਉਣ ਵਾਲੇ ਪ੍ਰੰਪਟ ਟੂਲ ਮੈਟਾਡੇਟਾ ਵਿੱਚ ਜਿਹੜੇ ਏਆਈ ਮਾਡਲ ਪਰੋਸੈਸ ਕਰਦੇ ਹਨ, ਪਰ ਮਨੁੱਖੀ ਉਪਭੋਗਤਾਵਾਂ ਨੂੰ ਨਹੀਂ ਦਿਖਾਈ ਦੇਂਦੇ  
- **ਡਾਇਨਾਮਿਕ ਟੂਲ ਸੋਧ ("ਰੱਗ ਪੁੱਲਜ਼")**: ਉਪਭੋਗਤਾਵਾਂ ਦੁਆਰਾ ਮਨਜ਼ੂਰ ਕੀਤੇ ਟੂਲ ਬਾਅਦ ਵਿੱਚ ਬਿਨਾਂ ਉਨ੍ਹਾਂ ਦੀ ਜਾਣਕਾਰੀ ਦੇ ਹਾਨਿਕਾਰਕ ਕਾਰਜ ਕਰਨ ਲਈ ਸੋਧੇ ਜਾਂਦੇ ਹਨ  
- **ਪੈਰਾਮੀਟਰ ਇੰਜੈਕਸ਼ਨ**: ਟੂਲ ਦੇ ਪੈਰਾਮੀਟਰ ਸਕੀਮਾਂ ਵਿੱਚ ਛੁਪਾਈ ਗਈ ਹਾਨਿਕਾਰਕ ਸਮੱਗਰੀ ਜੋ ਮਾਡਲ ਦੇ ਵਰਤਾਰੇ ਨੂੰ ਪ੍ਰਭਾਵਿਤ ਕਰਦੀ ਹੈ  


**ਹੋਸਟ ਕੀਤਾ ਸਰਵਰ ਜੋਖਮ**: ਰਿਮੋਟ MCP ਸਰਵਰ ਉੱਚੇ ਜੋਖਮ ਪੇਸ਼ ਕਰਦੇ ਹਨ ਕਿਉਂਕਿ ਟੂਲ ਪਰਿਭਾਸ਼ਾਵਾਂ ਨੂੰ ਮੁਲਾਕਾਤੀ ਉਪਯੋਗਕਾਰ ਦੀ ਮੰਜੂਰੀ ਤੋਂ ਬਾਅਦ ਅਪਡੇਟ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ, ਜਿਸ ਨਾਲ ਪਹਿਲਾਂ ਸੁਰੱਖਿਅਤ ਟੂਲ ਮਾਲਿਸ਼ਸ ਬਣ ਸਕਦੇ ਹਨ। ਵਿਸ਼ਤਰੀਤ ਵਿਸ਼ਲੇਸ਼ਣ ਲਈ, ਵੇਖੋ [ਟੂਲ ਪੋਇਜ਼ਨਿੰਗ ਹਮਲੇ (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

#### **ਵਾਧੂ AI ਹਮਲਾ ਵੇਕਟਰ**

- **ਕ੍ਰਾਸ-ਡੋਮੇਨ ਪ੍ਰੋਂਪਟ ਇੰਜੈਕਸ਼ਨ (XPIA)**: ਸੁਚੱਜੇ ਹਮਲੇ ਜੋ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਨ ਲਈ ਕਈ ਡੋਮੇਨਾਂ ਦੀ ਸਮੱਗਰੀ ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹਨ
- **ਗਤੀਸ਼ੀਲ ਸਮਰੱਥਾ ਸੋਧ**: ਟੂਲ ਸਮਰੱਥਾਵਾਂ ਵਿੱਚ ਰੀਅਲ-ਟਾਈਮ ਬਦਲਾਵ ਜੋ ਸ਼ੁਰੂਆਤੀ ਸੁਰੱਖਿਆ ਮੁਲਾਂਕਣ ਨੂੰ ਛੱਡ ਜਾਂਦੇ ਹਨ
- **ਸੰਦੇਸ਼ ਖਿੜਕੀ ਪੋਇਜ਼ਨਿੰਗ**: ਵੱਡੇ ਸੰਦੇਸ਼ ਖਿੜਕੀਆਂ ਦਾ ਦੁਰਪਯੋਗ ਕਰਕੇ ਮਾਲਿਸ਼ਸ ਹੁਕਮਾਂ ਨੂੰ ਛੁਪਾਉਣ ਵਾਲੇ ਹਮਲੇ
- **ਮਾਡਲ ਗੁੰਝਲਦਾਰ ਹਮਲੇ**: ਮਾਡਲ ਦੀਆਂ ਸੀਮਾਵਾਂ ਨੂੰ ਵਰਤ ਕੇ ਅਣਭਵ ਸੰਵਿਕਲਪ ਜਾਂ ਅਸੁਰੱਖਿਅਤ ਰਵੱਈਏ ਬਣਾਉਣਾ


### AI ਸੁਰੱਖਿਆ ਜੋਖਮ ਪ੍ਰਭਾਵ

**ਉੱਚ ਪ੍ਰਭਾਵ ਵਾਲੇ ਨਤੀਜੇ:**
- **ਡਾਟਾ ਚੋਰੀ**: বেআইਨੀ ਪਹੁੰਚ ਅਤੇ ਸੰਵੇਦਨਸ਼ੀਲ ਉਦਯੋਗਕ ਜਾਂ ਵਿਅਕਤੀਗਤ ਡਾਟਾ ਦੀ ਚੋਰੀ
- **ਪ੍ਰਾਈਵੇਸੀ ਨੁਕਸਾਨ**: ਨਿੱਜੀ ਪਛਾਣ ਵਾਲੀ ਜਾਣਕਾਰੀ (PII) ਅਤੇ ਗੁਪਤ ਵਪਾਰਕ ਡਾਟਾ ਦਾ ਪ੍ਰਕਾਸ਼
- **ਸਿਸਟਮ ਮੈਨਿਪੂਲੈਸ਼ਨ**: ਅਹਮ ਸਿਸਟਮਾਂ ਅਤੇ ਰੁਤਬਿਆਂ ਵਿੱਚ ਇਛਾਉਂਦੇ ਬਿਨਾਂ ਸੋਧ ਬਦਲਾਅ
- **ਪ੍ਰਮਾਣ ਪੱਤਰ ਚੋਰੀ**: ਪ੍ਰਮਾਣਿਕਤਾ ਟੋਕਨ ਅਤੇ ਸੇਵਾ ਪ੍ਰਮਾਣ ਪੱਤਰਾਂ ਦਾ ਸਮਰਥਨ
- **ਲੈਟਰਲ ਮੂਵਮੈਂਟ**: ਵੱਡੇ ਨੈੱਟਵਰਕ ਹਮਲਿਆਂ ਲਈ ਸਮਰਪਿਤ AI ਸਿਸਟਮਾਂ ਦੀ ਵਰਤੋਂ

### ਮਾਈਕਰੋਸਾਫਟ AI ਸੁਰੱਖਿਆ ਹੱਲ

#### **AI ਪ੍ਰੋਂਪਟ ਸ਼ੀਲਡਜ਼: ਇੰਜੈਕਸ਼ਨ ਹਮਲਿਆਂ ਦੇ ਖ਼ਿਲਾਫ਼ ਉੱਨਤ ਸੁਰੱਖਿਆ**

ਮਾਈਕਰੋਸਾਫਟ **AI ਪ੍ਰੋਂਪਟ ਸ਼ੀਲਡਜ਼** ਸਿੱਧੇ ਅਤੇ ਅਥਾਹ ਪ੍ਰੋਂਪਟ ਇੰਜੈਕਸ਼ਨ ਹਮਲਿਆਂ ਖ਼ਿਲਾਫ਼ ਕਈ ਸੁਰੱਖਿਆ ਪਰਤਾਂ ਰਾਹੀਂ ਵਿਸ਼ਤਰੀਤ ਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰਦੇ ਹਨ:

##### **ਮੁੱਖ ਰੱਖਿਆ ਮਕੈਨਿਜ਼ਮ:**

1. **ਅੱਗੇ ਵਧੀਆ ਪਛਾਣ ਅਤੇ ਫਿਲਟਰੇਸ਼ਨ**
   - ਮਸ਼ੀਨ ਲਰਨਿੰਗ ਅਲਗੋਰਿਦਮਾਂ ਅਤੇ NLP ਤਕਨੀਕਾਂ ਬਾਹਰੀ ਸਮੱਗਰੀ ਵਿੱਚ ਮਾਲਿਸ਼ਸ ਹੁਕਮਾਂ ਦੀ ਪਛਾਣ ਕਰਦੀਆਂ ਹਨ
   - ਦਸਤਾਵੇਜ਼, ਵੈੱਬ ਪੰਨੇ, ਈਮੇਲਾਂ, ਅਤੇ ਡਾਟਾ ਸਰੋਤਾਂ ਦੀ ਰੀਅਲ-ਟਾਈਮ ਵਿਸ਼ਲੇਸ਼ਣ ਕਰਦੇ ਹਨ
   - ਵੈਧ ਅਤੇ ਮਾਲਿਸ਼ਸ ਪ੍ਰੋਂਪਟ ਮਾਡਲ ਪੈਟਰਨਾਂ ਦੀ ਸੰਦਰਭ ਸੋਝੀ

2. **ਸਪੌਟਲਾਈਟਿੰਗ ਤਕਨੀਕਾਂ**  
   - ਭਰੋਸੇਯੋਗ ਸਿਸਟਮ ਹੁਕਮਾਂ ਅਤੇ ਸੰਭਾਵਤ ਤੌਰ 'ਤੇ ਸਮਰਪਿਤ ਬਾਹਰੀ ਇਨਪੁਟਾਂ ਵਿੱਚ ਫਰਕ ਕਰਦਾ ਹੈ
   - ਮਾਡਲ ਪ੍ਰਸੰਗਤਾ ਵਧਾਉਣ ਵਾਲੇ ਟੈਕਸਟ ਰੂਪਾਂਤਰਣ ਵਿਧੀਆਂ, ਜਦਕਿ ਮਾਲਿਸ਼ਸ ਸਮੱਗਰੀ ਨੂੰ ਅਲੱਗ ਕਰਦੀਆਂ ਹਨ
   - AI ਸਿਸਟਮਾਂ ਨੂੰ ਢੰਗ ਨਾਲ ਹੁਕਮ ਹਾਇਰਾਰਕੀ ਪ੍ਰਾਪਤ ਕਰਨ ਅਤੇ ਇੰਜੈਕਟ ਕੀਤੇ ਕਮਾਂਡਾਂ ਨੂੰ ਅਨदेखਾ ਕਰਨ ਵਿੱਚ ਮਦਦ ਕਰਦਾ ਹੈ

3. **ਡੈਲੀਮੀਟਰ ਅਤੇ ਡੇਟਾ ਮਾਰਕਿੰਗ ਸਿਸਟਮ**
   - ਭਰੋਸੇਯੋਗ ਸਿਸਟਮ ਸੁਨੇਹੇ ਅਤੇ ਬਾਹਰੀ ਇਨਪੁਟ ਟੈਕਸਟ ਵਿਚਕਾਰ ਸਪਸ਼ਟ ਸੀਮਾ ਪਰਿਭਾਸ਼ਾ
   - ਖ਼ਾਸ ਨਿਸ਼ਾਨਾਂ ਨੇ ਭਰੋਸੇਯੋਗ ਅਤੇ ਬੇਭਰੋਸੇ ਡਾਟਾ ਸਰੋਤਾਂ ਵਿੱਚ ਹੱਦਾਂ ਦਰਸਾਈਆਂ ਹਨ
   - ਸਪਸ਼ਟ ਵੱਖ-ਵੱਖਤਾ ਹੁਕਮਾਂ ਦੇ ਭ੍ਰਮ ਨੂੰ ਰੋਕਦੀ ਹੈ ਅਤੇ ਬਿਨਾਂ ਅਧਿਕਾਰ ਦੇ ਕਮਾਂਡ ਨਾਲ ਕੰਮ ਕਰਨ ਤੋਂ ਬਚਾਉਂਦੀ ਹੈ

4. **ਲਗਾਤਾਰ ਖਤਰਾ ਬੂਝ**
   - ਮਾਈਕਰੋਸਾਫਟ ਨਵੇਂ ਹਮਲਾ ਪੈਟਰਨਾਂ ਦੀ ਨਿਗਰਾਨੀ ਕਰਦਾ ਹੈ ਅਤੇ ਰੱਖਿਆ ਨੂੰ ਅਪਡੇਟ ਕਰਦਾ ਹੈ
   - ਨਵੇਂ ਇੰਜੈਕਸ਼ਨ ਤਰ੍ਹਾਂ ਅਤੇ ਹਮਲਾ ਵੇਕਟਰਾਂ ਲਈ ਪ੍ਰੋਐਕਟਿਵ ਖ਼ਤਰਾ ਹੰਟਿੰਗ
   - ਵਿਕਾਸਸ਼ੀਲ ਖਤਰਿਆਂ ਖ਼ਿਲਾਫ਼ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਸੁਰੱਖਿਆ ਮਾਡਲ ਅਪਡੇਟਾਂ ਦੀ ਨਿਯਮਤ ਰੀਤ ਨਾਲ ਚਾਲਿਤਾ

5. **ਅਜ਼ੂਰ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਸਮੀਕਰਨ**
   - ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਸੂਟ ਦਾ ਹਿੱਸਾ
   - ਜੇਲਬ੍ਰੇਕ ਕੋਸ਼ਿਸ਼ਾਂ, ਨੁਕਸਾਨਦੇਹ ਸਮੱਗਰੀ, ਅਤੇ ਸੁਰੱਖਿਆ ਨੀਤੀ ਖਿਲਾਫ਼ ਵਾਧੂ ਪਛਾਣ
   - AI ਐਪਲੀਕੇਸ਼ਨ ਉਪਾਧਿਆਂ ਵਿੱਚ ਇਕਜੁਟ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ

## ਉੱਨਤ MCP ਸੁਰੱਖਿਆ ਖਤਰੇ

### ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ ਨਜ਼ੁਕੀਆਂ

**ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ** ਰਾਜਦਾਰ MCP ਲਾਗੂਆਂ ਵਿੱਚ ਇਕ ਮਹੱਤਵਪੂਰਨ ਹਮਲਾ ਵੈਕਟਰ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ ਜਿੱਥੇ ਬਿਨਾਂ ਅਧਿਕਾਰ ਵਾਲੇ ਪਾਸੇ वैਧ ਸੈਸ਼ਨ ਆਈਡੈਂਟਿਫਾਇਰ ਲੈ ਕੇ ਗ੍ਰਾਹਕਾਂ ਦੀ ਨਕਲ ਕਰਦੇ ਅਤੇ ਬਿਨਾਂ ਅਧਿਕਾਰ ਵਾਲੇ ਕਾਮ ਕਰਦੇ ਹਨ।

#### **ਹਮਲਾ ਦ੍ਰਿਸ਼ ਅਤੇ ਜੋਖਮ**

- **ਸੈਸ਼ਨ ਹਾਈਜੈਕ ਪ੍ਰੋਂਪਟ ਇੰਜੈਕਸ਼ਨ**: ਚੋਰੀ ਹੋਏ ਸੈਸ਼ਨ ID ਵਾਲੇ ਹਮਲਾਕਾਰ ਸਰਵਰਾਂ ਵਿੱਚ ਮਾਲਿਸ਼ਸ ਇਵੈਂਟ Inject ਕਰਦੇ ਹਨ ਜੋ ਸੈਸ਼ਨ ਸਟੇਟ ਸਾਂਝਾ ਕਰਦੇ ਹਨ, ਸੰਭਾਵਤ ਤੌਰ ਤੇ ਨੁਕਸਾਨਕਾਰੀ ਕਾਰਵਾਈ ਦਾ ਕਾਰਨ ਬਨ ਸਕਦਾ ਹੈ ਜਾਂ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਤੱਕ ਪਹੁੰਚ 
- **ਸਿੱਧਾ ਨਕਲ**: ਚੋਰੀ ਹੋਏ ਸੈਸ਼ਨ ID MCP ਸਰਵਰਾਂ ਦੀ ਸਿੱਧੀ ਕਾਲ ਨੂੰ ਯੋਗ ਬਣਾਉਂਦੇ ਹਨ ਜੋ ਪ੍ਰਮਾਣਿਕਤਾ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੇ ਹਨ, ਹਮਲਾਕਾਰਾਂ ਨੂੰ ਵੈਧ ਉਪਭੋਗਤਾਵਾਂ ਵਾਂਗ ਵੇਖਦੇ ਹਨ
- **ਬਿਗੜੇ ਰੀਜੂਮਬਲ ਸਟ੍ਰੀਮਾਂ**: ਹਮਲਾਕਾਰੀ ਬੇਹੋਵਰੇ ਨਾਲ ਬੇਨਤੀ ਸਮਾਪਤ ਕਰ ਸਕਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਵੈਧ ਗ੍ਰਾਹਕ ਸੰਭਵਤ ਮਾਲਿਸ਼ਸ ਸਮੱਗਰੀ ਨਾਲ ਮੁੜ ਸ਼ੁਰੂ ਕਰਦੇ ਹਨ

#### **ਸੈਸ਼ਨ ਪ੍ਰਬੰਧਨ ਲਈ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ**

**ਮੁੱਢਲੇ ਲੋੜਾਂ:**
- **ਅਧਿਕਾਰ ਪੜਤਾਲ**: ਜਿਸ MCP ਸਰਵਰ ਅਧਿਕਾਰ ਲਾਗੂ ਕਰਦਾ ਹੈ ਉਹ ਸਾਰੇ ਆਉਣ ਵਾਲੇ ਬੇਨਤੀਆਂ ਦੀ ਜਾਂਚ ਕਰੇ ਅਤੇ ਪ੍ਰਮਾਣਿਕਤਾ ਲਈ ਸੈਸ਼ਨਾਂ 'ਤੇ ਨਿਰਭਰ ਨਾ ਕਰੇ
- **ਸੁਰੱਖਿਅਤ ਸੈਸ਼ਨ ਜਨਰੇਸ਼ਨ**: ਗੁਪਤ ਲਘੂ, ਗੈਰ-ਢਾਂਚਾਕਾਰਕ ਸੈਸ਼ਨ ID ਨੂੰ ਸੁਰੱਖਿਅਤ ਰੈਂਡਮ ਨੰਬਰ ਜਨਰੇਟਰ ਨਾਲ ਬਣਾਓ
- **ਵਿਉਂਤ ਸੂਚਨਾ ਨਾਲ ਸੈਸ਼ਨ ID ਬੰਨ੍ਹੋ**: ਸੈਸ਼ਨ ID ਨੂੰ ਯੂਜ਼ਰ-ਵਿਸ਼ੇਸ਼ ਜਾਣਕਾਰੀ ਨਾਲ ਬਾਈਂਡ ਕਰੋ ਜਿਵੇਂ `<user_id>:<session_id>` ਕਰ ਕੇ ਵਰਤੋਂਕਾਰਾਂ ਵਿਚਕਾਰ ਸੈਸ਼ਨ ਦੀ ਗਲਤ ਵਰਤੋਂ ਨੂੰ ਰੋਕੋ
- **ਸੈਸ਼ਨ ਜੀਵਨ ਚਕ੍ਰ ਪ੍ਰਬੰਧਨ**: ਅਨੁਕੂਲ ਸਮਾਪਤੀ, ਘੁੰਮਾਅ, ਅਤੇ ਅਵੈਧਤਾ ਕਾਰਵਾਈ ਸੰਘ ਦੀਆਂ ਨਿਯਮਤਾਵਾਂ ਲਾਗੂ ਕਰੋ
- **ਟ੍ਰਾਂਸਪੋਰਟ ਸੁਰੱਖਿਆ**: ਸਾਰੀਆਂ ਸੰਚਾਰਾਂ ਲਈ HTTPS ਜ਼ਰੂਰੀ ਹੈ ਤਾਂ ਕਿ ਸੈਸ਼ਨ ID ਦੀ ਚੋਰੀ ਤੋਂ ਬਚਿਆ ਜਾ ਸਕੇ

### ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮਸਿਆ

**ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮਸਿਆ** ਉਸ ਸਮੇਂ ਹੁੰਦੀ ਹੈ ਜਦੋਂ MCP ਸਰਵਰ ਗ੍ਰਾਹਕਾਂ ਅਤੇ ਤੀਜੇ-ਪੱਖੀ ਸੇਵਾਵਾਂ ਦੇ ਦਰਮਿਆਨ ਪ੍ਰਮਾਣਿਕਤਾ ਪ੍ਰੋਕਸੀ ਵਜੋਂ ਕੰਮ ਕਰਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਸਥਿਰ ਕਲਾਇੰਟ ID ਦੇ ਦੁਰਪਯੋਗ ਰਾਹੀਂ ਅਧਿਕਾਰ ਬਾਈਪਾਸ ਕਰਨ ਦੇ ਮੌਕੇ ਬਣਦੇ ਹਨ।

#### **ਹਮਲਾ ਤਕਨੀਕਾਂ ਅਤੇ ਜੋਖਮ**

- **ਕੁਕੀ-ਅਧਾਰਿਤ ਸਹਿਮਤੀ ਬਾਈਪਾਸ**: ਪਿਛਲੀ ਵਰਤੋਂਕਾਰ ਪ੍ਰਮਾਣਿਕਤਾ ਸਹਿਮਤੀ ਕੁਕੀਜ਼ ਬਣਾਉਂਦੀ ਹੈ ਜੋ ਹਮਲਾਕਾਰ ਬਨਾਵਟੀ ਰੀਡਾਇਰੈਕਟ URI ਨਾਲ ਖ਼ਤਰਨਾਕ ਅਧਿਕਾਰਕ ਬੇਨਤੀਆਂ ਦੁਆਰਾ ਦੁਰਪਯੋਗ ਕਰਦੇ ਹਨ
- **ਅਧਿਕਾਰਕ ਕੋਡ ਚੋਰੀ**: ਮੌਜੂਦਾ ਸਹਿਮਤੀ ਕੁਕੀਜ਼ ਅਧਿਕਾਰ ਸਰਵਰਾਂ ਨੂੰ ਸਹਿਮਤੀ ਸਕਰੀਨ ਨੂੰ ਛੱਡ ਕੇ ਕੋਡਾਂ ਨੂੰ ਹਮਲਾਕਾਰ-ਨਿਯੰਤਰਿਤ ਐਂਡਪੋਇੰਟ ਤੱਕ ਭੇਜਣ ਲਈ ਪ੍ਰੇਰਿਤ ਕਰ ਸਕਦੀਆਂ ਹਨ  
- **ਬਿਨਾਂ ਅਧਿਕਾਰ ਵਾਲੀ API ਪਹੁੰਚ**: ਚੋਰੀ ਹੋਏ ਅਧਿਕਾਰਕ ਕੋਡ ਟੋਕਨ ਤਬਾਦਲਾ ਅਤੇ ਯੂਜ਼ਰ ਨਕਲ ਨੂੰ ਬਿਨਾਂ ਖ਼ਾਸ ਮਨਜ਼ੂਰੀ ਤੋਂ ਯੋਗ ਬਣਾਉਂਦੇ ਹਨ

#### **ਰੋਕਥਾਮ ਰਣਨੀਤੀਆਂ**

**ਲਾਜ਼ਮੀ ਨਿਯੰਤਰਣ:**
- **ਸਪਸ਼ਟ ਸਹਿਮਤੀ ਲੋੜਾਂ**: MCP ਪ੍ਰੋਕਸੀ ਸਰਵਰ ਜੋ ਸਥਿਰ ਕਲਾਇੰਟ ID ਵਰਤਦੇ ਹਨ ਉਹ ਹਰ ਡਾਇਨਾਮਿਕ ਰਜਿਸਟਰਡ ਕਲਾਇੰਟ ਲਈ ਯੂਜ਼ਰ ਦੀ ਸਹਿਮਤੀ ਲੈਣ ਲਾਜ਼ਮੀ ਹੈ
- **OAuth 2.1 ਸੁਰੱਖਿਆ ਲਾਗੂਅਤ**: ਸਾਰੇ ਅਧਿਕਾਰ ਕਾਲਾਂ ਲਈ ਨਵੀਨਤਮ OAuth ਸੁਰੱਖਿਆ ਸਰਬੋਤਮ ਚਰਚਾ ਅਨੁਸਾਰ PKCE (ਪ੍ਰੂਫ਼ ਕੀ ਫਾਰ ਕੋਡ ਐਕਸਚੇਂਜ) ਲਾਗੂ ਕਰੋ
- **ਕੜੀ ਕਲਾਇੰਟ ਵੈਰੀਫਿਕੇਸ਼ਨ**: ਦੁਰਪਯੋਗ ਨੂੰ ਰੋਕਣ ਲਈ ਰੀਡਾਇਰੈਕਟ URI ਅਤੇ ਕਲਾਇੰਟ ਪਛਾਣਕਾਰਾਂ ਦੀ ਸਖ਼ਤ ਜਾਂਚ ਕਰੋ

### ਟੋਕਨ ਪਾਸਥਰੂ ਨਜ਼ੁਕੀਆਂ  

**ਟੋਕਨ ਪਾਸਥਰੂ** ਇੱਕ ਖ਼ਾਸ ਵਿਰੋਧੀ-ਪੈਟਰਨ ਹੈ ਜਿੱਥੇ MCP ਸਰਵਰ ਗ੍ਰਾਹਕ ਟੋਕਨਾਂ ਨੂੰ ਬਿਨਾ ਢੰਗ ਨਾਲ ਜਾਂਚ ਕੀਤੇ ਸਵੀਕਾਰ ਕਰਦੇ ਹਨ ਅਤੇ ਉਨ੍ਹਾਂ ਨੂੰ ਡਾਊਨਸਟਰੀਮ API ਤੇ ਅੱਗੇ ਭੇਜਦੇ ਹਨ, ਜੋ MCP ਅਧਿਕਾਰ ਨਿਰਦੇਸ਼ਾਂ ਦਾ ਉਲੰਘਣ ਹੈ।

#### **ਸੁਰੱਖਿਆ ਅਰਥ**

- **ਨਿਯੰਤਰਣ ਚਾਲਾਕ ਰਾਹੀ ਬਚਾਵ**: ਸਿੱਧਾ ਕਲਾਇੰਟ-ਤੋਂ-API ਟੋਕਨ ਵਰਤੋਂ ਮਹੱਤਵਪੂਰਨ ਦਰ-, ਜਾਂਚ ਅਤੇ ਨਿਗਰਾਨੀ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੀ ਹੈ
- **ਆਡਿਟ ਟ੍ਰੇਲ ਵਿਗੜਣਾ**: ਉੱਪਰੀ ਪਾਸੇ ਜਾਰੀ ਕੀਤੇ ਟੋਕਨ ਗ੍ਰਾਹਕ ਪਛਾਣ ਨੂੰ ਅਸੰਭਵ ਬਣਾਦੇ ਹਨ, ਘਟਨਾ ਜਾਂਚ ਸਮਰੱਥਾ ਨੂੰ ਤੋੜਦਾ ਹੈ
- **ਪ੍ਰਾਕਸੀ-ਅਧਾਰਿਤ ਡਾਟਾ ਚੋਰੀ**: ਬਿਨਾ ਜਾਂਚੇ ਟੋਕਨ ਮਾਲਿਸ਼ਸ ਅਦਾਕਾਰਾਂ ਨੂੰ ਸਰਵਰਾਂ ਨੂੰ ਪ੍ਰਾਕਸੀ ਵਜੋਂ ਵਰਤ ਕੇ ਬਿਨਾ ਅਧਿਕਾਰ ਚੋਰੀ ਦਾ ਮੌਕਾ ਦਿੰਦੇ ਹਨ
- **ਭਰੋਸੇਯੋਗ ਸੀਮਾ ਉਲੰਘਣ**: ਜਦ ਟੋਕਨ ਮੂਲ ਦੀ ਪੁਸ਼ਟੀ ਨਹੀ ਹੋ ਸਕਦੀ ਤਾਂ ਡਾਊਨਸਟਰੀਮ ਸਰਵਿਸਿਜ਼ ਦੀ ਭਰੋਸਾ ਧਾਰਣਾ ਟੁੱਟ ਸਕਦੀ ਹੈ
- **ਬਹੁ-ਸੇਵਾ ਹਮਲਾ ਵਿਰਤੀਕਰਨ**: ਕਈ ਸਰਵਿਸਿਜ਼ ਉੱਤੇ ਸਵੀਕਾਰ ਕੀਤੇ ਗਏ ਸਮਰਪਿਤ ਟੋਕਨ ਲੈਟਰਲ ਮੂਵਮੈਂਟ ਨੂੰ ਯੋਗ ਬਣਾਉਂਦੇ ਹਨ

#### **ਲਾਜ਼ਮੀ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ**

**ਬੇਵਾਲਣਯੋਗ ਲੋੜਾਂ:**
- **ਟੋਕਨ ਵੈਰੀਫਿਕੇਸ਼ਨ**: MCP ਸਰਵਰ ਉਹ ਟੋਕਨ ਸਵੀਕਾਰ ਨਾ ਕਰੇ ਜੋ ਖ਼ਾਸ ਤੌਰ ਤੇ MCP ਸਰਵਰ ਲਈ ਜਾਰੀ ਨਾ ਕੀਤੇ ਗਏ ਹੋਣ
- **ਦਰਸ਼ਕ ਦੀ ਜਾਂਚ**: ਹਮੇਸ਼ਾਂ ਟੋਕਨ ਦਰਸ਼ਕ ਦਾਵਿਆਂ ਦੀ ਜਾਂਚ ਕਰੋ ਕਿ ਉਹ MCP ਸਰਵਰ ਦੀ ਪਛਾਣ ਨਾਲ ਮਿਲਦੇ ਹਨ
- **ਠੀਕ ਟੋਕਨ ਜੀਵਨ ਚੱਕਰ**: ਛੋਟੇ ਸਮੇਂ ਵਾਲੇ ਪਹੁੰਚ ਟੋਕਨਾਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਘੁੰਮਾਅ ਅਭਿਆਸਾਂ ਨਾਲ ਲਾਗੂ ਕਰੋ


## AI ਸਿਸਟਮਾਂ ਲਈ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ

ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਰਵਾਇਤੀ ਸਾਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ ਤੋਂ ਅੱਗੇ ਵਧ ਕੇ ਪੂਰੇ AI ਪਰਿਵਾਰ ਨੂੰ ਲੰਮਦੀ ਹੈ। ਆਧੁਨਿਕ MCP ਲਾਗੂਆਂ ਨੂੰ ਸਾਰੇ AI ਨਾਲ ਸੰਬੰਧਤ ਉਪਕਰਣਾਂ ਦੀ ਪੂਰੀ ਜਾਂਚ ਅਤੇ ਨਿਗਰਾਨੀ ਸ਼ੁੱਧਤਾ ਨਾਲ ਕਰਨੀ ਚਾਹੀਦੀ ਹੈ, ਕਿਉਂਕਿ ਹਰ ਇਕ ਸੰਭਵ ਜੋਖਮ ਲਿਆਉਂਦਾ ਹੈ ਜੋ ਸਿਸਟਮ ਦੀ ਠੀਕ ਢੰਗ ਨਾਲ ਚੱਲਣ ਨੂੰ ਖ਼ਤਰੇ ਵਿੱਚ ਪਾ ਸਕਦਾ ਹੈ।

### ਵਧੇਰੇ AI ਸਪਲਾਈ ਚੇਨ ਘਟਕ

**ਰਵਾਇਤੀ ਸਾਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ:**
- ਕੁੱਲ-ਵਰਗੀ ਲਾਈਬ੍ਰੇਰੀਆਂ ਅਤੇ ਫਰੇਮਵਰਕ
- ਕੰਟੇਨਰ ਚਿੱਤਰ ਅਤੇ ਬੇਸ ਸਿਸਟਮ
- ਵਿਕਾਸ ਟੂਲਾਂ ਅਤੇ ਬਿਲਡ ਪਾਈਪਲਾਈਨਾਂ
- ਢਾਂਚਾਗਤ ਉਪਕਰਣ ਅਤੇ ਸੇਵਾਵਾਂ

**AI-ਖ਼ਾਸ ਸਪਲਾਈ ਚੇਨ ਅੰਗ:**
- **ਫਾਉਂਡੇਸ਼ਨ ਮਾਡਲ**: ਵੱਖ-ਵੱਖ ਪ੍ਰਦਾਤਾਵਾਂ ਦੇ ਪ੍ਰੋ-ਟਰੇਨ ਮਾਡਲ ਜਿਨ੍ਹਾਂ ਦੀ ਮੂਲਤਾ ਦੀ ਜਾਂਚ ਜ਼ਰੂਰੀ ਹੈ
- **ਐਂਬੈਡਿੰਗ ਸੇਵਾਵਾਂ**: ਬਾਹਰੀ ਵੇਕਟਰਾਈਜ਼ੇਸ਼ਨ ਅਤੇ ਅਰਥਪੂਰਨ ਖੋਜ ਸੇਵਾਵਾਂ
- **ਸੰਦਰਭ ਪ੍ਰਦਾਤਾ**: ਡਾਟਾ ਸਰੋਤ, ਗਿਆਨ ਅਧਾਰ, ਅਤੇ ਦਸਤਾਵੇਜ਼ ਸੰਗ੍ਰਹਿ
- **ਤੀਸਰੇ-ਪੱਖੀ API**: ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ, ML ਪਾਈਪਲਾਈਨਾਂ, ਅਤੇ ਡਾਟਾ ਪ੍ਰોસੈਸਿੰਗ ਐਂਡਪੋਇੰਟ
- **ਮਾਡਲ ਆਰਟੀਫੈਕਟ**: ਭਾਰ, ਸੰਰਚਨਾਵਾਂ, ਅਤੇ ਬਹੁਤ ਬਹੁਤ ਫਾਈਨ-ਟਿਊਨ ਮਾਡਲ ਵੈਰੀਅੰਟ
- **ਟ੍ਰੇਨਿੰਗ ਡਾਟਾ ਸਰੋਤ**: ਮਾਡਲ ਟ੍ਰੇਨਿੰਗ ਅਤੇ ਫਾਈਨ-ਟਿਊਨਿੰਗ ਲਈ ਵਰਤੇ ਜਾਣ ਵਾਲੇ ਡਾਟਾਸੈੱਟ

### ਤਮਾਮ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਰਣਨੀਤੀ

#### **ਘਟਕ ਵੈਰੀਫਿਕੇਸ਼ਨ ਅਤੇ ਭਰੋਸਾ**
- **ਮੂਲਤਾ ਦੀ ਪੁਸ਼ਟੀ**: ਸਾਰੇ AI ਘਟਕਾਂ ਦੀ ਮੂਲਤਾ, ਲਾਇਸੈਂਸ ਅਤੇ ਅਖੰਡਤਾ ਨੂੰ ਏਕੀਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਸਾਬਤ ਕਰੋ
- **ਸੁਰੱਖਿਆ ਮੁਲਾਂਕਣ**: ਮਾਡਲਾਂ, ਡਾਟਾ ਸਰੋਤਾਂ, ਅਤੇ AI ਸੇਵਾਵਾਂ ਲਈ ਕਮਜ਼ੋਰੀ ਸਕੈਨਿੰਗ ਅਤੇ ਸੁਰੱਖਿਆ ਸਮੀਖਿਆ ਕਰੋ
- **ਪ੍ਰਤਿਸ਼ਠਾ ਵਿਸ਼ਲੇਸ਼ਣ**: AI ਸੇਵਾ ਪ੍ਰਦਾਤਾਵਾਂ ਦੀ ਸੁਰੱਖਿਆ ਪ੍ਰੈਕਟਿਸ ਅਤੇ ਟ੍ਰੈਕ ਰਿਕਾਰਡ ਦੀ ਸਮੀਖਿਆ ਕਰੋ
- **ਅਨੁਕੂਲਤਾ ਪੁਸ਼ਟੀ**: ਸਾਰੇ ਘਟਕ ਸੰਸਥਾਨਕ ਸੁਰੱਖਿਆ ਅਤੇ ਨਿਯੰਤਰਣ ਲੋੜਾਂ ਨੂੰ ਪੂਰਾ ਕਰਦੇ ਹੋਣ

#### **ਸੁਰੱਖਿਆ ਵਾਲੇ ਤਾਇਨਾਤੀ ਪਾਈਪਲਾਈਨ**  
- **ਸੁਚਾਲਿਤ CI/CD ਸੁਰੱਖਿਆ**: ਸੁਚਾਲਿਤ ਤਾਇਨਾਤੀ ਪਾਈਪਲਾਈਨਾਂ ਵਿੱਚ ਸੁਰੱਖਿਆ ਸਕੈਨਿੰਗ ਨੂੰ ਸ਼ਾਮਿਲ ਕਰੋ
- **ਆਰਟੀਫੈਕਟ ਅਖੰਡਤਾ**: ਸਾਰੇ ਤਾਇਨਾਤ ਕੀਤੇ ਆਰਟੀਫੈਕਟਾਂ (ਕੋਡ, ਮਾਡਲ, ਸੰਰਚਨਾ) ਲਈ ਗੁਪਤਚਰ ਸੰਖਿਆਕੀ ਪੁਸ਼ਟੀ ਲਾਗੂ ਕਰੋ
- **ਕਦਮ-ਦਰ ਕਦਮ ਤਾਇਨਾਤੀ**: ਹਰ ਮੰਜ਼ਿਲ ਤੇ ਸੁਰੱਖਿਆ ਪੁਸ਼ਟੀ ਨਾਲ ਪ੍ਰੋਗਰੈਸੀਵ ਤਾਇਨਾਤੀ ਰਣਨੀਤੀਆਂ ਵਰਤੋ
- **ਭਰੋਸੇਯੋਗ ਆਰਟੀਫੈਕਟ ਰਿਪਾਜ਼ਟਰਿਜ**: ਸਿਰਫ਼ ਪ੍ਰਮਾਣਿਤ, ਸੁਰੱਖਿਅਤ ਆਰਟੀਫੈਕਟ ਰਜਿਸਟਰੀਆਂ ਅਤੇ ਰਿਪਾਜ਼ਟਰਿਜ ਤੋਂ ਹੀ ਤਾਇਨਾਤ ਕਰੋ

#### **ਲਗਾਤਾਰ ਨਿਗਰਾਨੀ ਅਤੇ ਜਵਾਬ**
- **ਨਿਰਭਰਤਾ ਸਕੈਨਿੰਗ**: ਸਾਰੇ ਸਾਫਟਵੇਅਰ ਅਤੇ AI ਘਟਕ ਨਿਰਭਰਤਾ ਲਈ ਨਿਰੰਤਰ ਕਮਜ਼ੋਰੀ ਨਿਗਰਾਨੀ
- **ਮਾਡਲ ਨਿਗਰਾਨੀ**: ਮਾਡਲ ਦੇ ਵਿਹਾਰ, ਪ੍ਰਦਰਸ਼ਨ ਡ੍ਰਿਫਟ, ਅਤੇ ਸੁਰੱਖਿਆ ਅਸਧਾਰਨਤਾਵਾਂ ਦੀ ਲਗਾਤਾਰ ਮੁਲਾਂਕਣ
- **ਸੇਵਾ ਸਿਹਤ ਟ੍ਰੈਕਿੰਗ**: ਉਪਲਬਧਤਾ, ਸੁਰੱਖਿਆ ਘਟnāਆਂ, ਅਤੇ ਨੀਤੀ-ਬਦਲਾਅ ਲਈ ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ ਦੀ ਨਿਗਰਾਨੀ
- **ਖ਼ਤਰਾ ਬੁੱਧੀ ਸਮੀਕਰਨ**: AI ਅਤੇ ML ਸੁਰੱਖਿਆ ਜੋਖਮਾਂ ਲਈ ਵਿਸ਼ੇਸ਼ ਖ਼ਤਰਾ ਫੀਡਾਂ ਨੂੰ ਸ਼ਾਮਿਲ ਕਰੋ

#### **ਪਹੁੰਚ ਨਿਯੰਤਰਣ ਅਤੇ ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ**
- **ਘਟਕ-ਸਤਰ ਅਧਿਕਾਰ**: ਕਾਰੋਬਾਰੀ ਲੋੜ ਮੁਤਾਬਕ ਮਾਡਲਾਂ, ਡਾਟਾ, ਅਤੇ ਸੇਵਾਵਾਂ ਤੱਕ ਪਹੁੰਚ ਸੀਮਿਤ ਕਰੋ
- **ਸੇਵਾ ਖਾਤਾ ਪ੍ਰਬੰਧਨ**: ਘੱਟੋ-ਘੱਟ ਲੋੜੀਂਦੇ ਅਧਿਕਾਰਾਂ ਵਾਲੇ ਸਮਰਪਿਤ ਸੇਵਾ ਖਾਤੇ ਲਾਗੂ ਕਰੋ
- **ਨੈੱਟਵਰਕ ਵਿਭਾਜਨ**: AI ਘਟਕਾਂ ਨੂੰ ਅਲੱਗ ਰੱਖੋ ਅਤੇ ਸੇਵਾਵਾਂ ਵਿਚਕਾਰ ਨੈੱਟਵਰਕ ਪਹੁੰਚ ਸੀਮਿਤ ਕਰੋ
- **API ਗੇਟਵੇ ਨਿਯੰਤਰਣ**: ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ ਲਈ ਕੇਂਦਰੀਕ੍ਰਿਤ API ਗੇਟਵੇ ਵਰਤ ਕੇ ਪਹੁੰਚ ਨਿਯੰਤਰਿਤ ਕਰੋ ਅਤੇ ਨਿਗਰਾਨੀ ਕਰੋ

#### **ਘਟਨਾ ਰਿਪਲਾਈ ਅਤੇ ਪੁਨਰੁੱਥਾਨ**
- **ਤੁਰੰਤ ਰਿਪਲਾਈ ਪ੍ਰਕਿਰਿਆਵਾਂ**: ਸਪਲਾਈ ਚੇਨ ਦੇ ਖਰਾਬ ਹੋਏ AI ਘਟਕਾਂ ਨੂੰ ਪੈਚ ਕਰਨ ਜਾਂ ਬਦਲਣ ਲਈ ਸਥਾਪਤ ਪ੍ਰਕਿਰਿਆਵਾਂ
- **ਪ੍ਰਮਾਣ ਪੱਤਰ ਘੁੰਮਾਅ**: ਸਿਕ୍ਰੇਟਾਂ, API ਕੁੰਜੀਆਂ, ਅਤੇ ਸੇਵਾ ਪ੍ਰਮਾਣ ਪੱਤਰਾਂ ਦੀ ਆਟੋਮੈਟਿਕ ਭੁੱਲਵਟ
- **ਰੋਲਬੈਕ ਸਮਰੱਥਾ**: ਪਿਛਲੇ ਜਾਣੇ-ਪਹਚਾਣੇ-ਚੰਗੇ ਵਰਜ਼ਨਾਂ ਵਿੱਚ ਤੇਜ਼ੀ ਨਾਲ ਵਾਪਸੀ ਕਰਨ ਦੀ ਸਮਰੱਥਾ
- **ਸਪਲਾਈ ਚੇਨ ਤੋੜਫੋੜ ਪੁਨਰੁੱਥਾਨ**: ਉੱਪਰਲੇ AI ਸੇਵਾ ਖਰਾਬੀਆਂ ਨੂੰ ਸੰਬੋਧਨ ਕਰਨ ਲਈ ਵਿਸ਼ੇਸ਼ ਪ੍ਰਕਿਰਿਆਵਾਂ

## ਬੁਨਿਆਦੀ ਸੁਰੱਖਿਆ ਚੰਗੀਆਂ ਅਭਿਆਸਾਂ

MCP ਲਾਗੂਆਂ ਤੁਹਾਡੇ ਸੰਸਥਾ ਦੇ ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਦ੍ਰਸ਼ਟੀਕੋਣ ਤੋਂ ਵਾਧਾ ਅਤੇ ਉਸ ਤੇ ਆਧਾਰਿਤ ਹੁੰਦਾ ਹੈ। ਬੁਨਿਆਦੀ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ ਦੇ ਮਜ਼ਬੂਤ ਹੋਣ ਨਾਲ AI ਸਿਸਟਮਾਂ ਅਤੇ MCP ਤਾਇਨਾਤੀਆਂ ਦੀ ਕੁੱਲ ਸੁਰੱਖਿਆ ਭਾਰੀ ਤੌਰ 'ਤੇ ਵਧਦੀ ਹੈ।

### ਮੁੱਖ ਸੁਰੱਖਿਆ ਮੂਲ ਚੀਜ਼ਾਂ

#### **ਸੁਰੱਖਿਅਤ ਵਿਕਾਸ ਅਭਿਆਸ**
- **OWASP ਅਨੁਕੂਲਤਾ**: [OWASP ਟਾਪ 10](https://owasp.org/www-project-top-ten/) ਵੈੱਬ ਐਪਲੀਕੇਸ਼ਨ ਕਮਜ਼ੋਰੀਆਂ ਤੋਂ ਸੁਰੱਖਿਆ
- **AI-ਖ਼ਾਸ ਸੁਰੱਖਿਆ**: [OWASP ਟਾਪ 10 LLMs ਲਈ](https://genai.owasp.org/download/43299/?tmstv=1731900559) ਲਈ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰੋ
- **ਸੁਰੱਖਿਅਤ ਗੁਪਤ ਪ੍ਰਬੰਧਨ**: ਟੋਕਨ, API ਕੁੰਜੀਆਂ, ਅਤੇ ਸੰਵੇਦਨਸ਼ੀਲ ਸੰਰਚਨਾ ਡਾਟਾ ਲਈ وقف ਵੌਲਟ ਵਰਤੋ
- **ਆਖ਼ਰੀ ਤੱਕ ਗੁਪਤਕਰਨ**: ਸਾਰੇ ਐਪਲੀਕੇਸ਼ਨ ਉਪਾਧੀਆਂ ਅਤੇ ਡਾਟਾ ਫਲੋ ਵਿੱਚ ਸੁਰੱਖਿਅਤ ਸੰਚਾਰ ਲਾਗੂ ਕਰੋ
- **ਇਨਪੁਟ ਤਸਦੀਕ**: ਸਾਰੇ ਯੂਜ਼ਰ ਇਨਪੁਟ, API ਪੈਰਾਮੀਟਰ, ਅਤੇ ਡਾਟਾ ਸਰੋਤਾਂ ਦੀ ਗੁចੜ-ਗੁੱਟੀ ਜਾਂਚ

#### **ਇੰਫਰਾਸਟਰੱਕਚਰ ਮਜ਼ਬੂਤੀ**
- **ਮੇਲਟੀ-ਫੈਕਟਰ ਪ੍ਰਮਾਣਿਕਤਾ**: ਸਾਰੇ ਪ੍ਰਸ਼ਾਸਕੀ ਅਤੇ ਸੇਵਾ ਖਾਤਿਆਂ ਲਈ ਜ਼ਰੂਰੀ MFA
- **ਪੈਚ ਪ੍ਰਬੰਧਨ**: ਓਪਰੇਟਿੰਗ ਸਿਸਟਮਾਂ, ਫਰੇਮਵਰਕਾਂ ਅਤੇ ਨਿਰਭਰਤਾਵਾਂ ਲਈ ਆਟੋਮੈਟਿਕ ਜਾਂ ਸਮੇਂ ਸਿਰ ਪੈਚਿੰਗ  
- **ਆਈਡੈਂਟੀਟੀ ਪ੍ਰਦਾਤਾ ਏਕੀਕਰਨ**: ਸੰਸਥਾਨਕ ਆਈਡੈਂਟੀਟੀ ਪ੍ਰਦਾਤਾਵਾਂ (ਮਾਈਕਰੋਸਾਫਟ Entra ID, ਐਕਟਿਵ ਡਾਇਰੈਕਟਰੀ) ਦੁਆਰਾ ਕੇਂਦਰੀਕ੍ਰਿਤ ਪਛਾਣ ਪ੍ਰਬੰਧਨ
- **ਨੈੱਟਵਰਕ ਵਿਭਾਜਨ**: MCP ਘਟਕਾਂ ਦੀ ਲਾਜ਼ਮੀਅਤ ਮੁਤਾਬਕ ਲੈਟਰਲ ਮੂਵਮੈਂਟ ਪ੍ਰਤਿਬੰਧਿਤ ਕਰਨ ਲਈ ਤਰੱਕੀਯਾਬੰਦ ਅਲੱਗਥਾਂ
- **ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਦਾ ਮੂਲ ਮੰਤਵ**: ਸਾਰੇ ਸਿਸਟਮ ਘਟਕਾਂ ਅਤੇ ਖਾਤਿਆਂ ਲਈ ਘੱਟੋ-ਘੱਟ ਲੋੜੀਂਦੇ ਅਧਿਕਾਰ

#### **ਸੁਰੱਖਿਆ ਨਿਗਰਾਨੀ ਅਤੇ ਪਛਾਣ**
- **ਵਿਆਪਕ ਲੋਗਿੰਗ**: AI ਐਪਲੀਕੇਸ਼ਨ ਗਤੀਵਿਧੀਆਂ ਦੀ ਵਿਸ਼ਤਰੀਤ ਲੋਗਿੰਗ, ਜਿਸ ਵਿੱਚ MCP ਕਲਾਇੰਟ-ਸਰਵਰ ਪਰਸਪਰਕਿਰਿਆਵਾਂ ਸ਼ਾਮਿਲ ਹਨ
- **SIEM ਏਕੀਕਰਨ**: ਅਸਾਧਾਰਣਤਾ ਪਛਾਣ ਲਈ ਕੇਂਦਰੀਕ੍ਰਿਤ ਸੁਰੱਖਿਆ ਜਾਣਕਾਰੀ ਅਤੇ ਘਟਨਾ ਪ੍ਰਬੰਧਨ
- **ਵਿਹਾਰਵਿਗਿਆਨ ਵਿਸ਼ਲੇਸ਼ਣ**: ਸਿਸਟਮ ਅਤੇ ਯੂਜ਼ਰ ਵਿਹਾਰ ਵਿੱਚ ਅਜੀਬ ਪੈਟਰਨ ਪਛਾਣਨ ਲਈ AI-ਚਲਿਤ ਨਿਗਰਾਨੀ
- **ਖ਼ਤਰਾ ਬੁੱਧੀ**: ਬਾਹਰੀ ਖ਼ਤਰਾ ਫੀਡਾਂ ਅਤੇ ਸੰਕਟਕੇ ਅਸਰਾਂ (IOC) ਦਾ ਏਕੀਕਰਨ
- **ਘਟਨਾ ਜਵਾਬ**: ਸੁਰੱਖਿਆ ਘਟਨਾ ਦੀ ਪਛਾਣ, ਜਵਾਬ, ਅਤੇ ਪੁਨਰੁੱਥਾਨ ਲਈ ਸਪਸ਼ਟ ਪ੍ਰਕਿਰਿਆਵਾਂ

#### **ਜ਼ੀਰੋ ਟਰੱਸਟ ਆਰਕੀਟੈਕਚਰ**
- **ਕਦੇ ਭਰੋਸਾ ਨਾ ਕਰੋ, ਹਮੇਸ਼ਾਂ ਜਾਂਚੋ**: ਯੂਜ਼ਰਾਂ, ਡਿਵਾਈਸਾਂ, ਅਤੇ ਨੈੱਟਵਰਕ ਕਨੈਕਸ਼ਨਾਂ ਦੀ ਲਗਾਤਾਰ ਪੁਸ਼ਟੀ
- **ਮਾਈਕ੍ਰੋ-ਸੇਗਮੈਂਟੇਸ਼ਨ**: ਵਿਅਕਤੀਗਤ ਵਰਕਲੋਡ ਅਤੇ ਸੇਵਾਵਾਂ ਨੂੰ ਅਲੱਗ ਰੱਖਣ ਵਾਲੇ ਨੈੱਟਵਰਕ ਨਿਯੰਤਰਣ
- **ਪਛਾਣ-ਕੇਂਦ੍ਰਿਤ ਸੁਰੱਖਿਆ**: ਸੁਰੱਖਿਆ ਨੀਤੀਆਂ ਜੋ ਨੈੱਟਵਰਕ ਸਥਿਤੀ ਦੀ ਬਜਾਏ ਪੁਸ਼ਟੀ ਕੀਤੀਆਂ ਪਛਾਣਾਂ ਤੇ ਅਧਾਰਿਤ ਹੁੰਦੀਆਂ ਹਨ
- **ਲਗਾਤਾਰ ਜੋਖਮ ਮੁਲਾਂਕਣ**: ਮੌਜੂਦਾ ਸੰਦਰਭ ਅਤੇ ਵਿਹਾਰ ਅਧਾਰਿਤ ਗਤੀਸ਼ੀਲ ਸੁਰੱਖਿਆ ਦ੍ਰਸ਼ਟੀਕੋਣ ਮੁਲਾਂਕਣ
- **ਸ਼ਰਤੀ ਪਹੁੰਚ**: ਜੋਖਮ ਕਾਰਕ, ਸਥਾਨ, ਅਤੇ ਡਿਵਾਈਸ ਭਰੋਸੇ 'ਤੇ ਆਧਾਰਿਤ ਪਹੁੰਚ ਨਿਯੰਤਰਣ

### ਉਦਯੋਗ ਏਕੀਕਰਨ ਪੈਟਰਨ

#### **ਮਾਈਕਰੋਸਾਫਟ ਸੁਰੱਖਿਆ ਪ੍ਰਣਾਲੀ ਏਕੀਕਰਨ**
- **Microsoft Defender for Cloud**: ਵਿਸ਼ਤਰੀਤ ਕਲਾਊਡ ਸੁਰੱਖਿਆ ਦ੍ਰਸ਼ਟੀਕੋਣ ਪ੍ਰਬੰਧਕਤਾ
- **Azure Sentinel**: AI ਵਰਕਲੋਡ ਸੁਰੱਖਿਆ ਲਈ ਕਲਾਊਡ-ਨੇਟਿਵ SIEM ਅਤੇ SOAR ਸਮਰੱਥਾਵਾਂ
- **Microsoft Entra ID**: ਸ਼ਰਤੀ-ਅਧਾਰਿਤ ਪਹੁੰਚ ਨੀਤੀਆਂ ਨਾਲ ਕਾਰੋਬਾਰੀ ਪਛਾਣ ਅਤੇ ਪਹੁੰਚ ਪ੍ਰਬੰਧਨ
- **Azure Key Vault**: ਹਾਰਡਵੇਅਰ ਸੁਰੱਖਿਆ ਮਾਡਿਊਲ (HSM) ਦੇ ਨਾਲ ਕੇਂਦਰੀਕ੍ਰਿਤ ਰਾਜ ਦਫਤਰ ਪ੍ਰਬੰਧਨ
- **Microsoft Purview**: AI ਡਾਟਾ ਸਰੋਤਾਂ ਅਤੇ ਰੁਤਬਿਆਂ ਲਈ ਡਾਟਾ ਸ਼ਾਸਨ ਅਤੇ ਅਨੁਕੂਲਤਾ

#### **ਅਨੁਕੂਲਤਾ ਅਤੇ ਸ਼ਾਸਨ**
- **ਨਿਯਮਕ ਅਨੁਕੂਲਤਾ**: ਯਕੀਨੀ ਬਣਾਓ ਕਿ MCP ਲਾਗੂਆਂ ਉਦਯੋਗ-ਖਾਸ ਅਨੁਕੂਲਤਾ ਲੋੜਾਂ (GDPR, HIPAA, SOC 2) ਨੂੰ ਪੂਰਾ ਕਰਦੇ ਹਨ

- **ਡਾਟਾ ਵਰਗੀਕਰਨ**: AI ਸਿਸਟਮਾਂ ਦੁਆਰਾ ਪ੍ਰਕਿਰਿਆਵਤ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਦੀ ਸਹੀ ਵਰਗੀਕਰਨ ਅਤੇ ਸੰਭਾਲ
- **ਆਡਿਟ ਟਰੈਲਸ**: ਨਿਯਮਕ ਕਾਇਦੇ ਅਨੁਸਾਰ ਪਾਲਣਾ ਅਤੇ ਫੋਰੇਂਸਿਕ ਜਾਂਚ ਲਈ ਵਿਸਤ੍ਰਿਤ ਲੌਗਿੰਗ
- **ਪ੍ਰਾਈਵੇਸੀ ਕੰਟਰੋਲਸ**: AI ਸਿਸਟਮ ਆਰਕੀਟੈਕਚਰ ਵਿੱਚ ਪ੍ਰਾਈਵੇਸੀ-ਬਾਈ-ਡਿਜ਼ਾਈਨ ਸਿਧਾਂਤਾਂ ਦੀ ਲਾਗੂਆਈ
- **ਬਦਲਾਅ ਪ੍ਰਬੰਧਨ**: AI ਸਿਸਟਮ ਦੇ ਤਬਦੀਲੀਆਂ ਦੀ ਸੁਰੱਖਿਆ ਸਮੀਖਿਆ ਲਈ ਆਧਿਕਾਰਿਕ ਪ੍ਰਕਿਰਿਆਵਾਂ

ਇਹ ਨਿਰੀਖਣਾਤਮਕ ਅਭਿਆਸ ਇੱਕ ਮਜਬੂਤ ਸੁਰੱਖਿਆ ਬੇਸਲਾਈਨ ਬਣਾਉਂਦੇ ਹਨ ਜੋ MCP-ਵਿਸ਼ੇਸ਼ ਸੁਰੱਖਿਆ ਕੰਟਰੋਲਾਂ ਦੀ ਪ੍ਰਭਾਵਸ਼ੀਲਤਾ ਨੂੰ ਵਧਾਉਂਦਾ ਹੈ ਅਤੇ AI-ਚਲਿਤ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਵਿਸਤ੍ਰਿਤ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ।

## ਮੁੱਖ ਸੁਰੱਖਿਆ ਨਤੀਜੇ

- **ਪ੍ਰਤਿ-ਪੜਤਾਲੀ ਸੁਰੱਖਿਆ ਪਹੁੰਚ**: ਮੂਲ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਘੱਟ ਤੋਂ ਘੱਟ ਅਧਿਕਾਰ, ਸਪਲਾਈ ਚੇਨ ਤਸਦੀਕ, ਲਗਾਤਾਰ ਨਿਗਰਾਨੀ) ਨੂੰ AI-ਵਿਸ਼ੇਸ਼ ਕੰਟਰੋਲਾਂ ਨਾਲ ਜੋੜ ਕੇ ਵਿਸਤ੍ਰਿਤ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰੋ

- **AI-ਵਿਸ਼ੇਸ਼ ਖਤਰੇ ਦਾ ਦ੍ਰਿਸ਼ਯ**: MCP ਸਿਸਟਮਾਂ ਨੂੰ ਵਿਲੱਖਣ ਜੋਖਮਾਂ ਦਾ ਸਾਹਮਣਾ ਹੈ ਜਿਵੇਂ ਕਿ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜਿਹਰੀਲੀਕਰਨ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗ਼ਲਤ ਡਿਪਟੀ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪਾਸਥਰੂ ਕਮਜ਼ੋਰੀਆਂ ਅਤੇ ਜ਼ਿਆਦਾ ਅਧਿਕਾਰ ਜੋ ਖਾਸ ਰੋਕਥਾਮ ਦੀ ਮੰਗ ਕਰਦੇ ਹਨ

- **ਪਹਿਚਾਣ ਅਤੇ ਅਧਿਕਾਰਿਤ ਕਰਨ ਵਿੱਚ ਸ਼੍ਰੇਸ਼ਟਤਾ**: ਬਾਹਰੀ ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾਵਾਂ (Microsoft Entra ID) ਦੀ ਵਰਤੋਂ ਨਾਲ ਮਜ਼ਬੂਤ ਤਰੀਕੇ ਨਾਲ ਪਰਖ ਕਰੋ, ਸਹੀ ਟੋਕਨ ਮਾਨਤਾ ਲਾਗੂ ਕਰੋ, ਅਤੇ ਕਦੇ ਵੀ ਆਪਣੇ MCP ਸਰਵਰ ਲਈ ਖਾਸ ਤੌਰ 'ਤੇ ਜਾਰੀ ਨਾ ਕੀਤੇ ਟੋਕਨ ਸਵੀਕਾਰ ਨਾ ਕਰੋ

- **AI ਹਮਲੇ ਰੋਕਥਾਮ**: ਪ੍ਰਤਿਆਸ਼ੀਤ ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ ਅਤੇ ਟੂਲ ਜਿਹਰੀਲੀਕਰਨ ਹਮਲਿਆਂ ਤੋਂ ਬਚਾਅ ਲਈ Microsoft Prompt Shields ਅਤੇ Azure Content Safety ਦੀ ਵਰਤੋਂ ਕਰੋ, ਟੂਲ ਮੈਟਾਡੇਟਾ ਦੀ ਜਾਂਚ ਕਰੋ ਅਤੇ ਗਤਿਸ਼ੀਲ ਬਦਲਾਅ ਲਈ ਨਿਗਰਾਨੀ ਕਰੋ

- **ਸੈਸ਼ਨ ਅਤੇ ਟ੍ਰਾਂਸਪੋਰਟ ਸੁਰੱਖਿਆ**: ਯੂਜ਼ਰ ਪਹਿਚਾਣਾਂ ਨਾਲ ਜੁੜੇ ਕ੍ਰਿਪਟੋਗ੍ਰਾਫਿਕਲ ਸੁਰੱਖਿਅਤ, ਗੈਰ-ਨਿਸ਼ਚਿਤ ਸੈਸ਼ਨ ਆਈਡੀਜ਼ ਦੀ ਵਰਤੋਂ ਕਰੋ, ਉਚਿਤ ਸੈਸ਼ਨ ਜੀਵਨਚੱਕਰ ਪ੍ਰਬੰਧਨ ਲਾਗੂ ਕਰੋ, ਅਤੇ ਕਦੇ ਵੀ ਸੈਸ਼ਨਾਂ ਨੂੰ ਪਹਿਚਾਣ ਲਈ ਵਰਤੋਂ ਨਾ ਕਰੋ

- **OAuth ਸੁਰੱਖਿਆ ਦੇ ਸਹੀ ਅਭਿਆਸ**: ਚਲਦੀਆਂ ਰਜਿਸਟਰ ਕੀਤੀ ਕਲਾਇੰਟਾਂ ਲਈ ਸਪਸ਼ਟ ਯੂਜ਼ਰ ਸਹਿਮਤਿ ਦੇ ਕੇ ਗ਼ਲਤ ਡਿਪਟੀ ਹਮਲਿਆਂ ਨੂੰ ਰੋਕੋ, PKCE ਦੇ ਨਾਲ OAuth 2.1 ਨੂੰ ਠੀਕ ਤਰੀਕੇ ਨਾਲ ਲਾਗੂ ਕਰੋ, ਅਤੇ ਕੰਠਕ URI ਦੀ ਕਟੜ ਤਸਦੀਕ ਕਰੋ

- **ਟੋਕਨ ਸੁਰੱਖਿਆ ਮੂਲ ਸਿਧਾਂਤ**: ਟੋਕਨ ਪਾਸਥਰੂ ਐਂਟੀ-ਪੈਟਰਨ ਤੋਂ ਬਚੋ, ਟੋਕਨ ਦਰਸ਼ਕ ਦਾਅਵੇ ਦੀ ਜਾਂਚ ਕਰੋ, ਸੁਰੱਖਿਅਤ ਘੁੰਮਾਵ ਵਾਲੀਆਂ ਛੋਟੀਆਂ ਮਿਆਦ ਵਾਲੀਆਂ ਟੋਕਨ ਲਾਗੂ ਕਰੋ, ਅਤੇ ਸਪਸ਼ਟ ਭਰੋਸਾ ਸੀਮਾਵਾਂ ਬਣਾਈ ਰੱਖੋ

- **ਵਿਸਤ੍ਰਿਤ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ**: ਸਾਰੇ AI ਪਰਿਵੇਸ਼ ਘਟਕਾਂ (ਮਾਡਲ, ਇੰਬੈੱਡਿੰਗਜ਼, ਸੰਦਰਭ ਪ੍ਰਦਾਤਾ, ਬਾਹਰੀ API) ਨੂੰ ਰਵਾਇਤੀ ਸੌਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ ਜਿਵੇਂ ਦੇ ਸੁਰੱਖਿਆ ਤੇਜ਼ੀ ਨਾਲ ਸੰਭਾਲੋ

- **ਲਗਾਤਾਰ ਵਿਕਾਸ**: ਤੇਜ਼ੀ ਨਾਲ ਬਦਲ ਰਹੀਆਂ MCP ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਨਾਲ ਅੱਪ-ਟੂ-ਡੇਟ ਰਹੋ, ਸੁਰੱਖਿਆ ਕਮਿਊਨਿਟੀ ਮਿਆਰਾਂ ਵਿੱਚ ਯੋਗਦਾਨ ਪਾਓ, ਅਤੇ ਜਿਵੇਂ-ਜਿਵੇਂ ਪ੍ਰੋਟੋਕੋਲ ਪੱਕਾ ਹੁੰਦਾ ਜਾ ਰਿਹਾ ਹੈ, ਐਡਾਪਟਿਵ ਸੁਰੱਖਿਆ ਪੋਸਟਿਊਰ ਬਣਾਈ ਰੱਖੋ

- **Microsoft ਸੁਰੱਖਿਆ ਏਕੀਕਰਨ**: MCP ਡਿਪਲੋਇਮੈਂਟ ਸੁਰੱਖਿਆ ਵਿੱਚ ਵਾਧਾ করার ਲਈ Microsoft ਦੇ ਵਿਸਤ੍ਰਿਤ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਤਾ ਤੰਤਰ (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) ਦਾ ਲਾਭ ਉਠਾਓ
