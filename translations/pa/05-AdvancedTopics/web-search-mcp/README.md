# ਪਾਠ: ਵੈੱਬ ਖੋਜ MCP ਸਰਵਰ ਬਣਾਉਣਾ

ਇਸ ਅਧਿਆਇ ਵਿੱਚ ਦਿਖਾਇਆ ਗਿਆ ਹੈ ਕਿ ਕਿਵੇਂ ਇੱਕ ਅਸਲੀ ਦੁਨੀਆ ਦਾ AI ਏਜੰਟ ਬਣਾਇਆ ਜਾਵੇ ਜੋ ਬਾਹਰੀ APIs ਨਾਲ ਇੰਟੀਗ੍ਰੇਟ ਕਰਦਾ ਹੈ, ਵੱਖ-ਵੱਖ ਡੇਟਾ ਪ੍ਰਕਾਰਾਂ ਨੂੰ ਸੰਭਾਲਦਾ ਹੈ, ਗਲਤੀਆਂ ਦਾ ਪ੍ਰਬੰਧ ਕਰਦਾ ਹੈ, ਅਤੇ ਕਈ ਉਪਕਰਨਾਂ ਦੀ ਸਾਂਥ-ਸੰਚਾਲਨਾ ਕਰਦਾ ਹੈ—ਸਾਰਾ ਕੁਝ ਇੱਕ ਪ੍ਰੋਡਕਸ਼ਨ-ਤੁਆਰ ਸਰੂਪ ਵਿੱਚ। ਤੁਸੀਂ ਵੇਖੋਗੇ:

- **ਪ੍ਰਮਾਣਿਕਤਾ ਦੀ ਲੋੜ ਵਾਲੇ ਬਾਹਰੀ APIs ਨਾਲ ਇੰਟੀਗ੍ਰੇਸ਼ਨ**
- **ਕਈ ਐਂਡਪੌਇੰਟਾਂ ਤੋਂ ਵੱਖ-ਵੱਖ ਡੇਟਾ ਪ੍ਰਕਾਰਾਂ ਦੀ ਸੰਭਾਲ**
- **ਦ੍ਰਿੜ ਗਲਤੀ ਸੰਭਾਲ ਅਤੇ ਲਾਗਿੰਗ ਦੀਆਂ ਰਣਨੀਤੀਆਂ**
- **ਇੱਕ ਹੀ ਸਰਵਰ ਵਿੱਚ ਬਹੁ-ਉਪਕਰਨ ਸਾਂਥ-ਸੰਚਾਲਨਾ**

ਅੰਤ ਵਿੱਚ, ਤੁਹਾਡੇ ਕੋਲ ਪ্যাটਰਨ ਅਤੇ ਸ੍ਰੇਸ਼্ঠ ਅਭਿਆਸਾਂ ਦਾ ਵਿਹਾਰਕ ਅਨੁਭਵ ਹੋਵੇਗਾ ਜੋ ਉन्नਤ AI ਅਤੇ LLM ਚਲਾਏ ਜਾਨ ਵਾਲੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਜ਼ਰੂਰੀ ਹਨ।

## ਜਾਣ-ਪਛਾਣ

ਇਸ ਪਾਠ ਵਿੱਚ, ਤੁਸੀਂ ਸਿੱਖੋਗੇ ਕਿਵੇਂ ਇੱਕ ਉੱਨਤ MCP ਸਰਵਰ ਅਤੇ ਕਲਾਇੰਟ ਬਣਾਇਆ ਜਾਵੇ ਜੋ SerpAPI ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਓਪਰੇਟਿੰਗ ਸਮੇਂ ਵੈੱਬ ਡੇਟਾ ਨਾਲ LLM ਦੀ ਸ਼ਕਤੀਆਂ ਵਧਾਉਂਦਾ ਹੈ। ਇਹ ਇੱਕ ਮਹੱਤਵਪੂਰਨ ਹੁਨਰ ਹੈ ਜਿਸ ਨਾਲ ਗਤੀਸ਼ੀਲ AI ਏਜੰਟ ਬਣਾਏ ਜਾ ਸਕਦੇ ਹਨ ਜੋ ਵੈੱਬ ਤੋਂ ਤਾਜ਼ਾ ਜਾਣਕਾਰੀ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦੇ ਹਨ।

## ਸਿੱਖਣ ਦੇ ਉਦੇਸ਼

ਇਸ ਪਾਠ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:

- SerpAPI ਵਰਗੇ ਬਾਹਰੀ APIs ਨੂੰ ਸੁਰੱਖਿਅਤ ਤਰੀਕੇ ਨਾਲ MCP ਸਰਵਰ ਵਿੱਚ ਇੰਟੀਗ੍ਰੇਟ ਕਰਨਾ
- ਵੈੱਬ, ਖ਼ਬਰਾਂ, ਉਤਪਾਦ ਖੋਜ ਅਤੇ ਸਵਾਲ-ਜਵਾਬ ਲਈ ਕਈ ਉਪਕਰਨ ਲਾਗੂ ਕਰਨਾ
- LLM ਲਈ ਲੇਖੀ ਡੇਟਾ ਨੂੰ ਪਾਰਸ ਅਤੇ ਫਾਰਮੈਟ ਕਰਨਾ
- ਗਲਤੀਆਂ ਨੂੰ ਸੰਭਾਲਣਾ ਅਤੇ API ਦੀ ਦਰ ਸੀਮਾ ਦਾ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਪ੍ਰਬੰਧਨ ਕਰਨਾ
- ਦੋਹਾਂ ਆਟੋਮੇਟਿਕ ਅਤੇ ਇੰਟਰਐਕਟਿਵ MCP ਕਲਾਇੰਟਸ ਬਣਾਉਣਾ ਅਤੇ ਟੈਸਟ ਕਰਨਾ

## ਵੈੱਬ ਖੋਜ MCP ਸਰਵਰ

ਇਸ ਹਿੱਸੇ ਵਿੱਚ ਵੈੱਬ ਖੋਜ MCP ਸਰਵਰ ਦੀ ਆਰਕੀਟੈਕਚਰ ਅਤੇ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਦਿਖਾਈਆਂ ਗਈਆਂ ਹਨ। ਤੁਸੀਂ ਵੇਖੋਗੇ ਕਿ ਕਿਵੇਂ FastMCP ਅਤੇ SerpAPI ਨੂੰ ਇੱਕਸਾਥ ਵਰਤ ਕੇ LLM ਦੀਆਂ ਸਮਰੱਥਾਵਾਂ ਵਧਾਈਆਂ ਜਾਂਦੀਆਂ ਹਨ।

### ਜਾਇਜ਼ਾ

ਇਸ ਇੰਪਲਿਮੈਂਟੇਸ਼ਨ ਵਿੱਚ ਚਾਰ ਉਪਕਰਨ ਹਨ ਜੋ MCP ਦੀ ਸੁਧਾਰਿਤ ਅਤੇ ਸੁਰੱਖਿਅਤ ਤਰੀਕੇ ਨਾਲ ਵੱਖ-ਵੱਖ ਬਾਹਰੀ API-ਚਲਾਏ ਕੰਮ ਸੰਭਾਲਣ ਦੀ ਯੋਗਤਾ ਦਿਖਾਉਂਦੇ ਹਨ:

- **general_search**: ਵਿਆਪਕ ਵੈੱਬ ਨਤੀਜਿਆਂ ਲਈ
- **news_search**: ਤਾਜ਼ਾ ਸਿਰਲੇਖਾਂ ਲਈ
- **product_search**: ਈ-ਕਾਮਰਸ ਡੇਟਾ ਲਈ
- **qna**: ਸਵਾਲ-ਜਵਾਬ ਸਨਿੱਪੇਟਸ ਲਈ

### ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ
- **ਕੋਡ ਉਦਾਹਰਨਾਂ**: ਪਾਥਾਨ ਸਮੇਤ ਭਾਸ਼ਾਵਾਂ ਲਈ ਕੋਡ ਬਲਾਕ ਹਨ (ਹੋਰ ਭਾਸ਼ਾਵਾਂ ਵਿੱਚ ਬੜ੍ਹਾਏ ਜਾ ਸਕਦੇ ਹਨ) ਜੋ ਸਪਸ਼ਟਤਾ ਲਈ ਕੋਡ ਪਿਵਟਸ ਵਰਤਦੇ ਹਨ

### ਪਾਥਾਨ

```python
# general_search ਟੂਲ ਦੀ ਉਦਾਹਰਨ ਵਰਤੋਂ
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

ਕਲਾਇੰਟ ਚਲਾਉਣ ਤੋਂ ਪਹਿਲਾਂ, ਇਹ ਜਾਣਨਾ ਲਾਭਕਾਰੀ ਹੈ ਕਿ ਸਰਵਰ ਕੀ ਕਰਦਾ ਹੈ। [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ਫਾਇਲ MCP ਸਰਵਰ ਨੂੰ ਲਾਗੂ ਕਰਦੀ ਹੈ, ਜੋ SerpAPI ਨਾਲ ਇੰਟੀਗ੍ਰੇਟ ਕਰਕੇ ਵੈੱਬ, ਖ਼ਬਰਾਂ, ਉਤਪਾਦ ਖੋਜ ਅਤੇ Q&A ਲਈ ਉਪਕਰਨ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ। ਇਹ ਆਉਣ ਵਾਲੀਆਂ ਬੇਨਤੀਆਂ ਨੂੰ ਸਾਂਭਦਾ ਹੈ, API ਕਾਲਾਂ ਦਾ ਪ੍ਰਬੰਧ ਕਰਦਾ ਹੈ, ਪ੍ਰਤੀਕਿਰਿਆਆਂ ਨੂੰ ਪਾਰਸ ਕਰਦਾ ਹੈ ਅਤੇ ਗਾਢਾ ਨਤੀਜੇ ਕਲਾਇੰਟ ਨੂੰ ਵਾਪਸ ਭੇਜਦਾ ਹੈ।

ਤੁਸੀਂ ਪੂਰੀ ਇੰਪਲਿਮੈਂਟੇਸ਼ਨ ਨੂੰ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ਵਿੱਚ ਦੇਖ ਸਕਦੇ ਹੋ।

ਇਹ ਰਹੀ ਇੱਕ ਸੰਖੇਪ ਉਦਾਹਰਨ ਕਿ ਕਿਵੇਂ ਸਰਵਰ ਇੱਕ ਉਪਕਰਨ ਨੂੰ ਪਰਿਭਾਸ਼ਿਤ ਅਤੇ ਰਜਿਸਟਰ ਕਰਦਾ ਹੈ:

### ਪਾਥਾਨ ਸਰਵਰ

```python
# server.py (ਛਾਂਟ)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...ਲਾਗੂ ਕਰਨ...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **ਬਾਹਰੀ API ਇੰਟੀਗ੍ਰੇਸ਼ਨ**: API ਕੁੰਜੀਆਂ ਅਤੇ ਬਾਹਰੀ ਬੇਨਤੀਆਂ ਦੀ ਸੁਰੱਖਿਅਤ ਸੰਭਾਲ ਦਿਖਾਉਂਦਾ ਹੈ
- **ਲਿਖਤੀ ਡੇਟਾ ਪਾਰਸਿੰਗ**: API ਪ੍ਰਤੀਕਿਰਿਆਵਾਂ ਨੂੰ LLM-ਮਿੱਤਰ ਫਾਰਮੈਟ ਵਿੱਚ ਬਦਲਣ ਦਾ ਤਰੀਕਾ ਦਿਖਾਉਂਦਾ ਹੈ
- **ਗਲਤੀ ਸੰਭਾਲ**: ਮਜ਼ਬੂਤ ਗਲਤੀ ਸੰਭਾਲ ਨਾਲ ਯੋਗ ਲਾਗਿੰਗ
- **ਇੰਟਰਐਕਟਿਵ ਕਲਾਇੰਟ**: ਆਟੋਮੇਟਿਕ ਟੈਸਟ ਅਤੇ ਟੈਸਟਿੰਗ ਲਈ ਇੰਟਰਐਕਟਿਵ ਮੋਡ ਦੋਹਾਂ ਸ਼ਾਮਲ
- **ਸੰਦਰਭ ਪ੍ਰਬੰਧਨ**: ਲਾਗਿੰਗ ਅਤੇ ਬੇਨਤੀ ਟ੍ਰੈਕਿੰਗ ਲਈ MCP ਸੰਦਰਭ ਦੀ ਵਰਤੋਂ

## ਲੋੜੀਂਦੇ ਉਪਕਰਨ

ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਇਹ ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਹਾਡਾ ਵਾਤਾਵਰਨ ਢੰਗ ਨਾਲ ਸੈੱਟ ਹੈ। ਇਸ ਨਾਲ ਸਾਰੀਆਂ ਨਿਰਭਰਤਾਵਾਂ ਸਥਾਪਿਤ ਹੋ ਜਾਣਗੀਆਂ ਅਤੇ ਤੁਹਾਡੀਆਂ API ਕੁੰਜੀਆਂ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਸੰਰਚਿਤ ਹੋਣਗੀਆਂ, ਤਾਕਿ ਵਿਕਾਸ ਅਤੇ ਟੈਸਟਿੰਗ ਵਿੱਚ ਕਿਸੇ ਪ੍ਰਕਾਰ ਦੀ ਰੁਕਾਵਟ ਨਾ ਆਵੇ।

- ਪਾਥਾਨ 3.8 ਜਾਂ ਇਸ ਤੋਂ ਉੱਚਾ
- SerpAPI API ਕੁੰਜੀ (ਇੱਥੇ ਸਾਈਨ ਅਪ ਕਰੋ [SerpAPI](https://serpapi.com/) - ਮੁਫ਼ਤ ਟੀਅਰ ਉਪਲਬਧ)

## ਸਥਾਪਨਾ

ਸ਼ੁਰੂ ਕਰਨ ਲਈ, ਆਪਣਾ ਵਾਤਾਵਰਨ ਸੈੱਟ ਕਰਨ ਲਈ ਇਹ ਕਦਮ ਅਪਣਾਓ:

1. ਆਧਾਰਭੂਤ ਲਾਇਬ੍ਰੇਰੀਜ਼ uv (ਮੁਢਲੀ ਪਸੰਦ) ਜਾਂ pip ਨਾਲ ਇੰਸਟਾਲ ਕਰੋ:

```bash
# uv ਵਰਤੋਂ ਕਰਨਾ (ਸਿਫਾਰਸ਼ੀ)
uv pip install -r requirements.txt

# pip ਵਰਤੋਂ ਕਰਨਾ
pip install -r requirements.txt
```

2. ਪਰੋਜੈਕਟ ਦੀ ਰੂਟ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਇੱਕ `.env` ਫਾਇਲ ਬਣਾਓ ਅਤੇ ਉਸ ਵਿੱਚ ਆਪਣੀ SerpAPI ਕੁੰਜੀ ਸ਼ਾਮਲ ਕਰੋ:

```
SERPAPI_KEY=your_serpapi_key_here
```

## ਵਰਤੋਂ

ਵੈੱਬ ਖੋਜ MCP ਸਰਵਰ ਪ੍ਰਮੁੱਖ ਘੱਟ-ਮੱਧ-ਪੱਧਰੀ ਘਟਕ ਹੈ ਜੋ ਵੈੱਬ, ਖ਼ਬਰਾਂ, ਉਤਪਾਦ ਖੋਜ ਅਤੇ Q&A ਲਈ ਉਪਕਰਨ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ, SerpAPI ਨਾਲ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਕਰਕੇ। ਇਹ ਆਉਣ ਵਾਲੀਆਂ ਬੇਨਤੀਆਂ ਨੂੰ ਸੰਭਾਲਦਾ ਹੈ, API ਕਾਲਾਂ ਦਾ ਪ੍ਰਬੰਧ ਕਰਦਾ ਹੈ, ਪ੍ਰਤੀਕਿਰਿਆਵਾਂ ਨੂੰ ਪਾਰਸ ਕਰਦਾ ਹੈ ਅਤੇ ਗਾਢਾ ਨਤੀਜੇ ਕਲਾਇੰਟ ਨੂੰ ਵਾਪਸ ਭੇਜਦਾ ਹੈ।

ਤੁਸੀਂ ਪੂਰੀ ਇੰਪਲਿਮੈਂਟੇਸ਼ਨ ਨੂੰ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ਵਿੱਚ ਦੇਖ ਸਕਦੇ ਹੋ।

### ਸਰਵਰ ਚਲਾਉਣਾ

MCP ਸਰਵਰ ਸ਼ੁਰੂ ਕਰਨ ਲਈ, ਹੇਠਾਂ ਦਿੱਤਾ ਕਮਾਂਡ ਵਰਤੋਂ:

```bash
python server.py
```

ਸਰਵਰ stdio-ਅਧਾਰਿਤ MCP ਸਰਵਰ ਵਜੋਂ ਚੱਲੇਗਾ ਜਿਸ ਨਾਲ ਕਲਾਇੰਟ ਸਿੱਧਾ ਜੁੜ ਸਕਦਾ ਹੈ।

### ਕਲਾਇੰਟ ਮੋਡ

ਕਲਾਇੰਟ (`client.py`) MCP ਸਰਵਰ ਨਾਲ ਦੁੱਧਰਾ ਪ੍ਰਦਰਸ਼ਨ ਲਈ ਦੋ ਮੋਡਸ ਸਹਾਇਤਾ ਕਰਦਾ ਹੈ:

- **ਸਧਾਰਣ ਮੋਡ**: ਸਾਰੇ ਉਪਕਰਨਾਂ ਨੂੰ ਕਵਰ ਕਰਦੇ ਹੋਏ ਆਟੋਮੇਟਿਕ ਟੈਸਟ ਚਲਾਉਂਦਾ ਹੈ ਅਤੇ ਉਨ੍ਹਾਂ ਦੀ ਪ੍ਰਤੀਕਿਰਿਆ ਦੀ ਪੁਸ਼ਟੀ ਕਰਦਾ ਹੈ। ਇਹ ਸਰਵਰ ਅਤੇ ਉਪਕਰਨ ਸਹੀ ਕੰਮ ਕਰ ਰਹੇ ਹਨ ਜਾਂ ਨਹੀਂ ਇਸਦੀ ਤੇਜ਼ ਜਾਂਚ ਲਈ ਲਾਭਕਾਰੀ ਹੈ।
- **ਇੰਟਰਐਕਟਿਵ ਮੋਡ**: ਮੀਨੂ ਆਧਾਰਿਤ ਇੰਟਰਫੇਸ ਸ਼ੁਰੂ ਕਰਦਾ ਹੈ ਜਿੱਥੇ ਤੁਸੀਂ ਹੱਥੋਂ ਛੇਤੀ ਉਪਕਰਨਾਂ ਨੂੰ ਚੁਣ ਸਕਦੇ ਹੋ, ਕਸਟਮ ਪੁੱਛਤाछ ਦਾਖਲ ਕਰ ਸਕਦੇ ਹੋ ਅਤੇ ਤੁਰੰਤ ਨਤੀਜੇ ਦੇਖ ਸਕਦੇ ਹੋ। ਇਹ ਸਰਵਰ ਦੀਆਂ ਯੋਗਤਾਵਾਂ ਨੂੰ ਪਰਖਣ ਅਤੇ ਵੱਖ-ਵੱਖ ਇਨਪੁੱਟ ਨਾਲ ਪ੍ਰਯੋਗ ਕਰਨ ਲਈ ਉੱਤਮ ਹੈ।

ਤੁਸੀਂ ਪੂਰੀ ਇੰਪਲਿਮੈਂਟੇਸ਼ਨ ਨੂੰ [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) ਵਿੱਚ ਵੇਖ ਸਕਦੇ ਹੋ।

### ਕਲਾਇੰਟ ਚਲਾਉਣਾ

ਆਟੋਮੇਟਿਕ ਟੈਸਟ ਚਲਾਉਣ ਲਈ (ਇਹ ਆਪਣੇ ਆਪ ਸਰਵਰ ਸ਼ੁਰੂ ਕਰੇਗਾ):

```bash
python client.py
```

ਜਾਂ ਇੰਟਰਐਕਟਿਵ ਮੋਡ ਵਿੱਚ ਚਲਾਓ:

```bash
python client.py --interactive
```

### ਵੱਖ-ਵੱਖ ਤਰੀਕਿਆਂ ਨਾਲ ਟੈਸਟਿੰਗ

## ਇਸ ਪਾਠ ਵਿੱਚ ਉੱਨਤ ਧਾਰਣਾਵਾਂ

ਨਿਰਮਾਣ ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਇੱਥੇ ਕੁਝ ਮਹੱਤਵਪੂਰਨ ਉੱਨਤ ਧਾਰਣਾਵਾਂ ਹਨ ਜੋ ਇਸ ਅਧਿਆਇ ਦੌਰਾਨ ਆਉਣਗੀਆਂ। ਇਨ੍ਹਾਂ ਨੂੰ ਸਮਝਣਾ ਤੁਹਾਡੇ ਲਈ ਸਹਾਇਕ ਹੋਵੇਗਾ, ਭਾਵੇਂ ਤੁਸੀਂ ਨਵੇਂ ਹੋ:

- **ਬਹੁ ਉਪਕਰਨ ਸਾਂਥ-ਸੰਚਾਲਨਾ**: ਇਸ ਦਾ ਅਰਥ ਹੈ ਕਿ ਇੱਕ ਹੀ MCP ਸਰਵਰ ਵਿੱਚ ਕਈ ਉਪਕਰਨਾਂ (ਜਿਵੇਂ ਵੈੱਬ ਖੋਜ, ਖ਼ਬਰਾਂ ਖੋਜ, ਉਤਪਾਦ ਖੋਜ, ਅਤੇ Q&A) ਚਲਾਏ ਜਾਣ। ਇਹ ਤੁਹਾਡੇ ਸਰਵਰ ਨੂੰ ਵੱਖ-ਵੱਖ ਕੰਮਾਂ ਨੂੰ ਸੰਭਾਲਣ ਦੀ ਯੋਗਤਾ ਦਿੰਦਾ ਹੈ, ਨਾ ਕਿ ਸਿਰਫ਼ ਇੱਕ।
- **API ਦਰ ਸੀਮਾ ਸੰਭਾਲ**: ਕਈ ਬਾਹਰੀ APIs (ਜਿਵੇਂ SerpAPI) ਇਹ ਸੀਮਤ ਕਰਦੇ ਹਨ ਕਿ ਤੁਸੀਂ ਕਿਸੇ ਨਿਰੀਖਤ ਸਮੇਂ ਵਿੱਚ ਕਿੰਨੀ ਬੇਨਤੀਆਂ ਕਰ ਸਕਦੇ ਹੋ। ਚੰਗਾ ਕੋਡ ਇਹਨਾਂ ਸੀਮਾਵਾਂ ਦੀ ਜਾਂਚ ਕਰਦਾ ਹੈ ਅਤੇ ਉਹਨਾਂ ਨੂੰ ਸੰਭਾਲਦਾ ਹੈ, ਤਾਂ ਜੋ ਜੇ ਤੁਸੀਂ ਸੀਮਾ ਪਾਰ ਕਰੋ ਤਾਂ ਤੁਹਾਡਾ ਐਪ ਟੁੱਟੇ ਨਾ।
- **ਲਿਖਤੀ ਡੇਟਾ ਪਾਰਸਿੰਗ**: API ਪ੍ਰਤੀਕਿਰਿਆਵਾਂ ਅਕਸਰ ਜਟਿਲ ਅਤੇ ਨੇਸਟਡ ਹੁੰਦੀਆਂ ਹਨ। ਇਹ ਧਾਰਣਾ ਇਹ ਹੈ ਕਿ ਉਹ ਪ੍ਰਤੀਕਿਰਿਆਵਾਂ ਨੂੰ ਸਾਫ਼-ਸੁਥਰੇ ਅਤੇ ਵਰਤਣ ਯੋਗ ਫਾਰਮੈਟ ਵਿੱਚ ਬਦਲ ਦਿੱਤਾ ਜਾਵੇ ਜੋ LLMs ਜਾਂ ਹੋਰ ਪ੍ਰੋਗਰਾਮਾਂ ਲਈ ਉਪਯੋਗੀ ਹੋਣ।
- **ਗਲਤੀ ਬਹਾਲੀ**: ਕਈ ਵਾਰੀ ਚੀਜ਼ਾਂ ਗਲਤ ਹੋ ਜਾਂਦੀਆਂ ਹਨ—ਸ਼ਾਇਦ ਨੈੱਟਵਰਕ ਫੇਲ, ਜਾਂ API ਉਮੀਦ ਅਨੁਸਾਰ ਜਵਾਬ ਨਾ ਦੇਵੇ। ਗਲਤੀ ਬਹਾਲੀ ਦਾ ਅਰਥ ਹੈ ਕਿ ਤੁਹਾਡਾ ਕੋਡ ਇਹ ਸਮੱਸਿਆਵਾਂ ਨੂੰ ਸੰਭਾਲ ਸਕੇ ਅਤੇ ਫਿਰ ਵੀ ਉਪਯੋਗੀ ਫੀਡਬੈਕ ਦੇਵੇ, ਨਾ ਕਿ ਸਿਸਟਮ ਡਾਊਨ ਹੋ ਜਾਵੇ।
- **ਪੈਰਾਮੀਟਰ ਸੱਚਾਈ ਦੀ ਪੁਸ਼ਟੀ**: ਇਹ ਇਹ ਚੇਕ ਕਰਨ ਬਾਰੇ ਹੈ ਕਿ ਤੁਹਾਡੇ ਉਪਕਰਨਾਂ ਨੂੰ ਦਿੱਤੇ ਗਏ ਸਾਰੇ ਇਨਪੁੱਟ ਸਹੀ ਅਤੇ ਸੁਰੱਖਿਅਤ ਹਨ। ਇਸ ਵਿੱਚ ਡਿਫਾਲਟ ਮੁੱਲ ਦਿੱਤੇ ਜਾਣ ਅਤੇ ਟਾਈਪਾਂ ਦੀ ਜਾਂਚ ਸ਼ਾਮਲ ਹੈ, ਜੋ ਬੱਗ ਅਤੇ ਗਲਤ ਫਹਿਮੀਆਂ ਤੋਂ ਬਚਾਉਂਦਾ ਹੈ।

ਇਹ ਹਿੱਸਾ ਤੁਹਾਡੀ ਮਦਦ ਕਰੇਗਾ Web Search MCP Server ਨਾਲ ਕੰਮ ਕਰਦੇ ਸਮੇਂ ਆਮ ਸਮੱਸਿਆਵਾਂ ਨਿਰਧਾਰਿਤ ਕਰਨ ਅਤੇ ਸੁਧਾਰਨ ਵਿੱਚ। ਜੇ ਤੁਸੀਂ ਵੈੱਬ ਖੋਜ MCP ਸਰਵਰ ਨਾਲ ਕੰਮ ਕਰਦੇ ਸਮੇਂ ਗਲਤੀਆਂ ਜਾਂ ਅਣਪਛਾਤੀ ਵਿਹਾਰ ਦਾ ਸਾਹਮਣਾ ਕਰਦੇ ਹੋ, ਤਾਂ ਇਹ ਟ੍ਰਬਲਸ਼ੂਟਿੰਗ ਹਿੱਸਾ ਸਭ ਤੋਂ ਆਮ ਸਮੱਸਿਆਵਾਂ ਲਈ ਹੱਲ ਪੇਸ਼ ਕਰਦਾ ਹੈ। ਅਗਲੀ ਮਦਦ ਲਈ ਅੱਗੇ ਜਾਣ ਤੋਂ ਪਹਿਲਾਂ ਇਹ ਸੁਝਾਵ ਪੜ੍ਹੋ—ਇਹ اکثر ਸਮੱਸਿਆਵਾਂ ਨੂੰ ਤੇਜ਼ੀ ਨਾਲ ਹੱਲ ਕਰ ਦਿੰਦੇ ਹਨ।

## ਸਮੱਸਿਆ ਨਿਵਾਰਣ

ਜਦੋਂ ਤੁਸੀਂ ਵੈੱਬ ਖੋਜ MCP ਸਰਵਰ ਨਾਲ ਕੰਮ ਕਰ ਰਹੇ ਹੋ, ਕਈ ਵਾਰੀ ਸਮੱਸਿਆਵਾਂ ਆ ਸਕਦੀਆਂ ਹਨ—ਇਹ ਸਧਾਰਣ ਹੈ ਜਦੋਂ ਤੁਸੀਂ ਬਾਹਰੀ APIs ਅਤੇ ਨਵੇਂ ਉਪਕਰਨਾਂ ਨਾਲ ਵਿਕਾਸ ਕਰ ਰਹੇ ਹੋ। ਇਹ ਭਾਗ ਸਭ ਤੋਂ ਆਮ ਸਮੱਸਿਆਵਾਂ ਲਈ ਲਾਗੂ ਹੱਲ ਤੱਕ ਪਹੁੰਚ ਦਿੰਦਾ ਹੈ, ਤਾਂ ਜੋ ਤੁਸੀਂ ਤੇਜ਼ੀ ਨਾਲ ਕੰਮ ਤੇ ਵਾਪਸ ਆ ਸਕੋ। ਜੇ ਤੁਸੀਂ ਕੋਈ ਗਲਤੀ ਭੱਟਕਦੇ ਹੋ, ਤਾਂ ਇੱਥੇ ਸ਼ੁਰੂ ਕਰੋ: ਹੇਠਾਂ ਦਿੱਤੇ ਸੁਝਾਵ ਅਕਸਰ ਉਪਭੋਗਤਾਵਾਂ ਦੀਆਂ ਸਮੱਸਿਆਵਾਂ ਦਾ ਹੱਲ ਹੁੰਦੇ ਹਨ ਅਤੇ ਬਿਨਾਂ ਵਾਧੂ ਮਦਦ ਦੇ ਤੁਹਾਡੀ ਸਮੱਸਿਆ ਸੁਲਝਾ ਸਕਦੇ ਹਨ।

### ਆਮ ਸਮੱਸਿਆਵਾਂ

ਹੇਠਾਂ ਕੁਝ ਸਭ ਤੋਂ ਬਹੁਤ ਵਰਤੀ ਜਾਂਦੀਆਂ ਸਮੱਸਿਆਵਾਂ ਹਨ, ਜਿਨ੍ਹਾਂ ਦੇ ਸਾਫ਼ ਵਿਆਖਿਆ ਅਤੇ ਹੱਲ ਦੇ ਕਦਮ ਦਿੱਤੇ ਗਏ ਹਨ:

1. **.env ਫਾਇਲ ਵਿੱਚ SERPAPI_KEY ਗੈਂਭੀਰਤਾ ਨਾਲ ਗਾਇਬ**
   - ਜੇ ਤੁਸੀਂ `SERPAPI_KEY environment variable not found` ਗਲਤੀ ਵੇਖਦੇ ਹੋ, ਤਾਂ ਇਸਦਾ ਅਰਥ ਹੈ ਕਿ ਤੁਹਾਡੀ ਐਪਲੀਕੇਸ਼ਨ ਨੂੰ SerpAPI ਨੂੰ ਪਹੁੰਚ ਕਰਨ ਲਈ ਲੋੜੀਂਦੀ API ਕੁੰਜੀ ਨਹੀਂ ਮਿਲ ਰਹੀ। ਇਸ ਨੂੰ ਠੀਕ ਕਰਨ ਲਈ, ਆਪਣੇ ਪ੍ਰੋਜੈਕਟ ਦੀ ਰੂਟ ਵਿੱਚ `.env` ਨਾਮ ਦਾ ਫਾਇਲ ਬਣਾਓ (ਜੇ ਪਹਿਲਾਂ ਮੌਜੂਦ ਨਹੀਂ ਹੈ) ਅਤੇ ਇੱਕ ਲਾਈਨ ਜਮ੍ਹਾਂ ਕਰੋ `SERPAPI_KEY=your_serpapi_key_here`। ਯਕੀਨੀ ਬਣਾਓ ਕਿ `your_serpapi_key_here` ਨੂੰ ਆਪਣੇ ਅਸਲ SerpAPI ਕੁੰਜੀ ਨਾਲ ਬਦਲਿਆ ਹੋਇਆ ਹੈ ਜੋ SerpAPI ਵੈੱਬਸਾਈਟ ਤੋਂ ਮਿਲੀ ਹੈ।

2. **Module not found ਗਲਤੀਆਂ**
   - ਜਿਵੇਂ `ModuleNotFoundError: No module named 'httpx'` ਵਰਗੀਆਂ ਗਲਤੀਆਂ ਦਿਖਾਉਂਦੀਆਂ ਹਨ ਕਿ ਇੱਕ ਲੋੜੀਂਦਾ ਪਾਥਾਨ ਪੈਕੇਜ ਗਾਇਬ ਹੈ। ਇਸਦਾ ਕਾਰਨ ਅਕਸਰ ਇਹ ਹੁੰਦਾ ਹੈ ਕਿ ਤੁਸੀਂ ਸਾਰੀਆਂ ਨਿਰਭਰਤਾਵਾਂ ਇੰਸਟਾਲ ਨਹੀਂ ਕੀਤੀਆਂ। ਇਸਨੂੰ ਠੀਕ ਕਰਨ ਲਈ, ਆਪਣੇ ਟਰਮੀਨਲ ਵਿੱਚ `pip install -r requirements.txt` ਰਨ ਕਰੋ ਤਾਂ ਜੋ ਤੁਹਾਡੇ ਪ੍ਰੋਜੈਕਟ ਦੀਆਂ ਸਾਰੀਆਂ ਲੋੜੀਂਦੀਆਂ ਲਾਇਬ੍ਰੇਰੀਆਂ ਇੰਸਟਾਲ ਹੋ ਜਾਣ।

3. **ਕਨੈਕਸ਼ਨ ਸਮੱਸਿਆਵਾਂ**
   - ਜੇ ਤੁਹਾਨੂੰ `Error during client execution` ਵਰਗੀ ਗਲਤੀ ਮਿਲਦੀ ਹੈ, ਤਾਂ ਇਹ ਅਕਸਰ ਦਰਸਾਉਂਦਾ ਹੈ ਕਿ ਕਲਾਇੰਟ ਸਰਵਰ ਨਾਲ ਜੁੜ ਨਹੀਂ ਪਾ ਰਿਹਾ, ਜਾਂ ਸਰਵਰ ਉਮੀਦ ਅਨੁਸਾਰ ਚੱਲ ਨਹੀਂ ਰਿਹਾ। ਦੋਹਾਂ ਕਲਾਇੰਟ ਅਤੇ ਸਰਵਰ ਦੇ ਵਰਜਨ ਇੱਕ ਦੂਜੇ ਨਾਲ ਮੈਚ ਕਰਦੇ ਹਨ ਜਾਂ ਨਹੀਂ, ਅਤੇ `server.py` ਮੌਜੂਦ ਹੈ ਅਤੇ ਠੀਕ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਚੱਲ ਰਿਹਾ ਹੈ, ਇਹ ਚੈੱਕ ਕਰੋ। ਦੋਹਾਂ ਨੂੰ ਰੀਸਟਾਰਟ ਕਰਨ ਨਾਲ ਵੀ ਮਦਦ ਮਿਲ ਸਕਦੀ ਹੈ।

4. **SerpAPI ਗਲਤੀਆਂ**
   - ਜੇ ਤੁਸੀਂ `Search API returned error status: 401` ਵੇਖਦੇ ਹੋ, ਤਾਂ ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ ਤੁਹਾਡੀ SerpAPI ਕੁੰਜੀ ਗਾਇਬ, ਗਲਤ ਜਾਂ ਮਿਆਦ ਖਤਮ ਹੋ ਚੁਕੀ ਹੈ। ਆਪਣੇ SerpAPI ਡੈਸ਼ਬੋਰਡ ਨੂੰ ਵੇਖੋ, ਆਪਣੀ ਕੁੰਜੀ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ ਅਤੇ ਜਰੂਰਤ ਹੋਣ ਤੇ `.env` ਫਾਇਲ ਅੱਪਡੇਟ ਕਰੋ। ਜੇ ਕੁੰਜੀ ਸਹੀ ਹੈ ਪਰ ਫਿਰ ਵੀ ਗਲਤੀ ਆ ਰਹੀ ਹੈ, ਤਾਂ ਤਕਨੀਕੀ ਟੀਅਰ ਦਾ ਕੋਟਾ ਖ਼ਤਮ ਹੋਣ ਦੀ ਸਮਭਾਵਨਾ ਜਾਂਚੋ।

### ਡੀਬੱਗ ਮੋਡ

ਮੂਲ ਰੂਪ ਵਿੱਚ, ਐਪ ਕੇਵਲ ਜ਼ਰੂਰੀ ਜਾਣਕਾਰੀ ਹੀ ਲਾਗ ਕਰਦਾ ਹੈ। ਜੇ ਤੁਸੀਂ ਜਾਣਨਾ ਚਾਹੁੰਦੇ ਹੋ ਕਿ ਕੀ ਹੋ ਰਿਹਾ ਹੈ (ਜਿਵੇਂ ਕਿ ਗੁੰਝਲਦਾਰ ਸਮੱਸਿਆਵਾਂ ਦਾ ਪਤਾ ਕਰਨ ਲਈ), ਤਾਂ ਤੁਸੀਂ ਡੀਬੱਗ ਮੋਡ ਚਾਲੂ ਕਰ ਸਕਦੇ ਹੋ। ਇਸ ਨਾਲ ਤੁਹਾਨੂੰ ਹਰ ਕਦਮ ਬਾਰੇ ਵਧੇਰੇ ਡੀਟੇਲ ਮਿਲੇਗੀ ਜੋ ਐਪ ਲੈ ਰਹਾ ਹੈ।

**ਉਦਾਹਰਨ: ਸਧਾਰਣ ਨਤੀਜਾ**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**ਉਦਾਹਰਨ: ਡੀਬੱਗ ਨਤੀਜਾ**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

ਧਿਆਨ ਦਿਓ ਕਿ ਡੀਬੱਗ ਮੋਡ ਵਿੱਚ HTTP ਬੇਨਤੀਆਂ, ਪ੍ਰਤੀਕਿਰਿਆਵਾਂ ਅਤੇ ਹੋਰ ਅੰਦਰੂਨੀ ਬਿਆਨਾਂ ਬਾਰੇ ਵਾਧੂ ਲਾਈਨਾਂ ਸ਼ਾਮਲ ਹੁੰਦੀਆਂ ਹਨ। ਇਹ ਟ੍ਰਬਲਸ਼ੂਟਿੰਗ ਲਈ ਬਹੁਤ ਮਦਦਗਾਰ ਹੋ ਸਕਦਾ ਹੈ।

ਡੀਬੱਗ ਮੋਡ ਚਾਲੂ ਕਰਨ ਲਈ, ਆਪਣੀ `client.py` ਜਾਂ `server.py` ਦੇ ਸਰਵਪ੍ਰਥਮ ਸਤਰ ਤੇ ਲਾਗਿੰਗ ਸਤਰ DEBUG 'ਤੇ ਸੈੱਟ ਕਰੋ:

# [ਪਾਥਾਨ](#tab/python-debug)


```python
# ਤੁਹਾਡੇ client.py ਜਾਂ server.py ਦੇ ਸਿਖਰ 'ਤੇ
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO ਤੋਂ DEBUG ਵਿੱਚ ਬਦਲੋ
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
