## १६ जून, २०२६

### MCP स्पेसिफिकेशन संरेखन आणि नमुना पडताळणी

अभ्यासक्रमाची सध्याच्या **MCP Specification 2025-11-25** आणि नवीनतम अधिकृत SDKs सह पडताळणी केली, नंतर शिल्लक प्राचीन स्पेसिफिकेशन संदर्भ दुरुस्त केले आणि मुख्य नमुने अजूनही निर्मित आणि चालत असल्याची पुष्टी केली.

#### स्पेसिफिकेशन आवृत्ती दुरुस्त्या (२०२५-०६-१८ / २०२५-०३-२६ → २०२५-११-२५)

इंग्रजी सामग्री अद्ययावत केली जिथे अजूनही जुनी आवृत्ती *सध्याचा/नवीनतम* मानक म्हणून नमूद होती आणि दुवे canonical `modelcontextprotocol.io` स्पेसिफिकेशन पथांकडे पुनर्निर्देशित केले:
- **05-AdvancedTopics/mcp-security/README.md**: "सध्याचा मानक" बॅनर, प्रस्तावना, मुख्य सुरक्षा तत्त्वे शीर्षक, अनिवार्य आवश्यकता शीर्षक, Microsoft Entra ID विभाग, संदर्भ & संसाधने दुवे, आणि सुरक्षा सूचना (८ संदर्भ) २०२५-११-२५ पर्यंत अद्ययावत केले
- **05-AdvancedTopics/mcp-transport/README.md**: अतिरिक्त संसाधने स्पेक लिंक आणि "सध्याचा मानक" बॅनर २०२५-११-२५ पर्यंत अद्ययावत केले
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: जुना `२०२५-०३-२६` सुरक्षा-आणि-विश्वास लिंक नवीन २०२५-११-२५ सुरक्षा सर्वोत्तम पध्दती पृष्ठाने बदलला
- **03-GettingStarted/14-sampling/README.md**: अधिकृत Sampling दस्तऐवजीकरण लिंक २०२५-११-२५ पर्यंत अद्ययावत केले
- **03-GettingStarted/05-stdio-server/README.md**: वर्तमान काळातील "सध्याचा MCP स्पेसिफिकेशन" संदर्भ आणि अतिरिक्त संसाधने स्पेक लिंक २०२५-११-२५ पर्यंत अद्ययावत केले (इतिहास SSE-निषेध नोट्स अचूकतेसाठी कायम ठेवले)

#### सध्याच्या SDKs विरुद्ध नमुना पडताळणी

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` ने `@modelcontextprotocol/sdk@1.29.0` सोडवले; `tsc --noEmit` कोणतीही प्रकाराची चूक न देता उत्तीर्ण — विद्यमान `McpServer`/`StdioServerTransport` APIs वैध राहिले
- **Python (03-GettingStarted/01-first-server/solution/python)**: एक वेगळ्या `.venv` मध्ये `mcp[cli]` (1.27.2) सह पडताळले; `py_compile` उत्तीर्ण झाला आणि `FastMCP.list_tools()` ने योग्यरित्या `add` आणि `subtract` टूल परत केले
- सर्व नमुना `@modelcontextprotocol/sdk` संस्करण श्रेण्या (`>=1.26.0` / `^1.26.0` / `^1.27.0`) सध्याच्या `1.29.0` पर्यंत स्वच्छपणे सोडवल्या गेल्या, कोणतेही API बदल नाहीत याची पुष्टी केली

जुन्या SDK पिन्स बम्प केल्या ज्यामुळे प्रत्येक नमुना सध्याच्या MCP रिलीजसह जुळतो, रिपो-मुलभूत संकल्पनेशी सुसंगत:
- **03-GettingStarted/05-stdio-server/solution/typescript/package.json**: `@modelcontextprotocol/sdk` ची आवृत्ती `^1.8.0` पासून `>=1.26.0` पर्यंत बंप केली आणि जुन्या `"updated for MCP 2025-06-18"` पॅकेज वर्णनाला `"aligned with MCP Specification 2025-11-25"` मध्ये अपडेट केले
- **10-StreamliningAIWorkflows.../lab3/code/weather_mcp/pyproject.toml** आणि **lab4/code/github_mcp_server/pyproject.toml**: अचूक पिन `mcp==1.23.0` → `mcp>=1.26.0` पर्यंत बंप केली; दोन्ही `uv.lock` फाइल्स (uv lock) पुन्हा तयार केल्या, ज्यामुळे लॉकफाईल्स सध्याच्या `mcp 1.27.2` पर्यंत सोडवतात आणि मॅनिफेस्टसह समक्रमित राहतात

### सुरक्षा कडक करणे & अवलंबित्व धोके दुरुस्ती

प्रत्येक अवलंबित्व मॅनिफेस्ट आणि नमुना स्रोत कोडवर संपूर्ण सुरक्षा पाहणी केली, नंतर सर्व npm सल्लागार आणि एक कोड स्तर शोध ठीक केले. सुधारणा नंतर, `npm audit` प्रत्येक तपासलेल्या निर्देशिकेमध्ये **0 धोके** रिपोर्ट करते.

#### npm अवलंबित्व धोके (संक्रमणशील) — दुरुस्त

सर्व १५ कमिट केलेल्या `package-lock.json` फाइल्स तपासल्या. धोके मूळतः MCP Inspector डेव्हलपर टूल, OpenAI क्लायंट, आणि MCP SDK मधून आलेले संक्रमणशील अवलंबित्वांसह मर्यादित होते; सर्व आता नमुने न मोडता सोडवले आहेत:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** आणि **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) बम्प केला, ज्याने बंडल केलेले `ajv`, `brace-expansion`, `diff`, `path-to-regexp` आणि `ws` सल्लागार क्लिअर केले. npm `overrides` प्रविष्टी जोडली ज्यामुळे `shell-quote@1.8.4` ची दुरुस्ती बाधित करून `concurrently` द्वारे टिकवलेला गंभीर सल्ला नष्ट झाला; दोन्ही लॉकफाईल्स पुन्हा तयार केल्या (आता 0 धोके)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ने संक्रमणशील `qs` (मध्यम) दुरुस्तीत अद्ययावत केले
- **03-GettingStarted/samples/javascript**: `npm audit fix` ने संक्रमणशील `hono` (मध्यम) दुरुस्तीत अद्ययावत केले
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ने संक्रमणशील `form-data` (उच्च) दुरुस्तीत अद्ययावत केले
- **03-GettingStarted/11-simple-auth/solution/typescript**: प्रोजेक्ट पुनरुत्पादनयोग्य आणि तपासण्याजोगा करण्यासाठी हरवलेली `package-lock.json` तयार केली (0 धोके)

#### कोड-स्तरीय सुरक्षा दुरुस्ती (OWASP A03: इंजेक्शन)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` टूलमधून `shell=True` काढले. मागील `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` सेल मेटकॅरेक्टर्स फोल्डर मार्गात `cmd.exe` द्वारे समजू शकले (कमांड-इंजेक्शनचा मार्ग). आता ते सोडवलेले `Code.exe` थेट फोल्डर ऑर्ग्युमेंटसह सुरू करते — कोणतीही शेल नाही — जे कार्यात्मकदृष्ट्या समतुल्य आणि सुरक्षित आहे

#### Python अवलंबित्व तपासणी

- प्रत्येक Python आवश्यकतांचे सेट `pip-audit` सह तपासले. `05-AdvancedTopics` आणि `03-GettingStarted/samples/python` यांनी **कोणतेही ज्ञात धोके नाहीत** (त्यांचे `mcp` / `httpx` / `pydantic` / `python-dotenv` श्रेण्या सध्याच्या दुरुस्त आवृत्त्यांशी सुसंगत)

### सुरूवात विभाग विस्तार – प्रगत सर्व्हर वापर आणि सोपी प्रमाणीकरण

#### प्रगत सर्व्हर वापर (03-GettingStarted/10-advanced)
- **नवे प्रकरण जोडले**: प्रगत MCP सर्व्हर वापरासाठी सविस्तर मार्गदर्शक सादर केला, सामान्य आणि लो-लेव्हल सर्व्हर आर्किटेक्चर दोन्हींचा समावेश
  - **नियमित विरुद्ध लो-लेव्हल सर्व्हर**: दोन्ही पद्धतींसाठी Python आणि TypeScript मधील सविस्तर तुलना आणि कोड उदाहरणे
  - **हँडलर-आधारित डिझाइन**: हँडलर-आधारित साधन/संसाधन/प्रॉम्प्ट व्यवस्थापनाचे स्पष्टीकरण जे स्केलेबल, लवचीक सर्व्हर अंमलबजावणी साठी उपयुक्त आहे
  - **व्यावहारिक नमुने**: वास्तविक जगातील परिस्थिती जिथे लो-लेव्हल सर्व्हर नमुने प्रगत वैशिष्ट्ये आणि आर्किटेक्चर साठी फायदेशीर आहेत

#### सोपी प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नवे प्रकरण जोडले**: MCP सर्व्हरमध्ये सोपी प्रमाणीकरण अंमलबजावणीसाठी चरण-दर-चरण मार्गदर्शक
  - **प्रमाणीकरण संकल्पना**: प्रमाणीकरण विरुद्ध अधिकृतता आणि क्रेडेन्शियल हाताळणीचे स्पष्ट स्पष्टीकरण
  - **मूलभूत प्रमाणीकरण अंमलबजावणी**: Python (Starlette) आणि TypeScript (Express) मधील मिडलवेअर-आधारित प्रमाणीकरण नमुने, कोड नमुन्यांसहित
  - **प्रगत सुरक्षा कडे प्रगती**: सोपी प्रमाणीकरण पासून प्रारंभ करून OAuth 2.1 आणि RBAC कडे मार्गदर्शन, प्रगत सुरक्षा मॉड्यूल संदर्भांसह

या जोडणींमुळे मजबूत, सुरक्षित आणि लवचीक MCP सर्व्हर अंमलबजावण्या तयार करण्यासाठी व्यावहारिक, हाताळणी मार्गदर्शन मिळते, मूलभूत संकल्पनांपासून प्रगत उत्पादन नमुन्यांपर्यंतचा पूल तयार करतात.

#### मूळ संकल्पना (०१-CoreConcepts/) - संपूर्ण आधुनिकीकरण
- **प्रोटोकॉल आवृत्ती अद्ययतन**: सध्याच्या MCP तपशील २०२५-०६-१८ संदर्भानुसार दिनांक-आधारित आवृत्ती (YYYY-MM-DD फॉरमॅट)
- **आर्किटेक्चर सुधारणा**: होस्ट्स, क्लायंट्स, आणि सर्व्हर्स चा वर्णन सुधारित आणि सध्याच्या MCP आर्किटेक्चर नमुन्यांसह
  - होस्ट्स आता स्पष्टपणे AI अनुप्रयोग म्हणून, एकाधिक MCP क्लायंट कनेक्शन समन्वयित करतात
  - क्लायंट्स प्रोटोकॉल कनेक्टर्स म्हणून वर्णन, जे एक ते एक सर्व्हर नातेसंबंध राखतात
  - सर्व्हर्स स्थानिक विरुद्ध दूरस्थ तैनात परिस्थितीसह सुधारित
- **प्रिमिटिव्ह पुनर्रचना**: सर्व्हर व क्लायंट प्रिमिटिव्हची संपूर्ण सुधारण
  - सर्व्हर प्रिमिटिव्ह: संसाधने (डेटा स्रोत), प्रॉम्प्ट (टेम्पलेट्स), टूल्स (संचालित कार्ये) सविस्तर स्पष्टीकरण व उदाहरणांसह
  - क्लायंट प्रिमिटिव्ह: सॅम्पलिंग (LLM पूर्णता), एलीसिटेशन (वापरकर्ता इनपुट), लॉगिंग (डिबगिंग/निरीक्षण)
  - सध्या शोध (`*/list`), पुनर्प्राप्ती (`*/get`), आणि अंमलबजावणी (`*/call`) पद्धत नमुन्यांसह अद्यतनित
- **प्रोटोकॉल आर्किटेक्चर**: दोन-स्तरीय आर्किटेक्चर मॉडेल सादर केले
  - डेटा लेयर: JSON-RPC 2.0 पाया, जीवनचक्र व्यवस्थापन व प्रिमिटिव्हसह
  - ट्रान्सपोर्ट लेयर: STDIO (स्थानिक) आणि स्ट्रीम करू शकणारा HTTP SSE (दूरस्थ) ट्रान्सपोर्ट मेकॅनिझम
- **सुरक्षा फ्रेसवर्क**: स्पष्ट वापरकर्ता संमती, डेटा गोपनीयता, टूल अंमलबजावणी सुरक्षितता, व ट्रान्सपोर्ट लेयर सुरक्षा यांसह सर्वसमावेशक सुरक्षा तत्त्वे
- **संप्रेषण नमुने**: प्रारंभ, शोध, अंमलबजावणी, आणि सूचना प्रवाह दर्शविणारे अद्यतनित प्रोटोकॉल संदेश
- **कोड उदाहरणे**: सध्याच्या MCP SDK नमुन्यांसह बहुभाषिक उदाहरणे (.NET, Java, Python, JavaScript) ताजी केली

#### सुरक्षा (०२-सिक्युरिटी/) - संपूर्ण सुरक्षा पुनर्रचना  
- **मानक संरेखन**: MCP तपशील २०२५-०६-१८ सुरक्षा आवश्यकता पूर्णतः संरेखित

  - प्रगत शोध, हायलाइटिंग, व सीमांकन तंत्रांसह AI प्रॉम्प्ट शील्डस
  - Azure Content Safety एकत्रीकरण नमुने
  - पुरवठा साखळी संरक्षणासाठी GitHub Advanced Security

  - MCP-विशिष्ट हल्ला परिस्थिती व क्रिप्टोग्राफिक सत्र ID आवश्यकता असलेले सत्र हाइजॅकिंग  
  - स्पष्ट संमती आवश्यकतांसह MCP प्रॉक्सी परिस्थितीत संभ्रमित डिप्युटी समस्या  
  - आवश्यक पडताळणी नियंत्रणांसह टोकन पासथ्रू असुरक्षितता  
- **सप्लाय चेन सुरक्षा**: आधारभूत मॉडेल, एम्बेडिंग सेवा, संदर्भ पुरवठादार, व तृतीय पक्ष API समाविष्ट करून AI पुरवठा साखळी कव्हरेज विस्तारित
