# धडा: वेब शोध MCP सर्व्हर तयार करणे

हा अध्याय एक वास्तविक AI एजंट कसा तयार करावा हे दाखवतो जो बाह्य API सह समाकलित होतो, विविध डेटा प्रकार हाताळतो, चुका व्यवस्थापित करतो, आणि एकाधिक साधने योजतो—तेही उत्पादनासाठी तयार स्वरूपात. तुम्हाला खालील गोष्टी दिसतील:

- **प्रमाणीकरण आवश्यक असलेल्या बाह्य API समाकलन**
- **एकाधिक एंडपॉइंट्समधून विविध डेटा प्रकार हाताळणे**
- **मजबूत त्रुटी हाताळणी आणि लॉगिंग धोरणे**
- **एकाच सर्व्हरमध्ये बहु-साधन योजनेचा वापर**

शेवटी, तुम्हाला प्रगत AI आणि LLM-चालित अनुप्रयोगांसाठी आवश्यक नमुने आणि सर्वोत्तम पद्धतींबाबत व्यावहारिक अनुभव मिळेल.

## परिचय

या धड्यामध्ये, तुम्ही कसे प्रगत MCP सर्व्हर आणि क्लायंट तयार करू शकता हे शिकाल जे SerpAPI वापरून वास्तविक-वेळ वेब डेटासह LLM क्षमता वाढवतो. हे वेबमधून अद्ययावत माहिती मिळवू शकणारे गतिशील AI एजंट विकसित करण्यासाठी महत्त्वाचे कौशल्य आहे.

## शिकण्याचे उद्दिष्टे

या धड्याच्या शेवटी, तुम्हाला हे करण्यास सक्षम असाल:

- सुरक्षितपणे बाह्य API (जसे SerpAPI) MCP सर्व्हरमध्ये समाकलित करणे
- वेब, बातम्या, उत्पादन शोध, आणि प्रश्नोत्तरासाठी अनेक साधने लागू करणे
- LLM वापरासाठी संरचित डेटाचे पार्सिंग आणि स्वरूपन करणे
- त्रुटी हाताळणे आणि API दर मर्यादा प्रभावीपणे व्यवस्थापित करणे
- स्वयंचलित आणि संवादात्मक MCP क्लायंट दोन्ही तयार करणे आणि चाचणी करणे

## वेब शोध MCP सर्व्हर

हा विभाग वेब शोध MCP सर्व्हरच्या आर्किटेक्चर आणि वैशिष्ट्यांची ओळख करून देतो. तुम्हाला FastMCP आणि SerpAPI कसे वापरले जातात हे दिसेल जे LLM क्षमता वास्तविक-वेळ वेब डेटासह वाढवतात.

### आढावा

या अंमलबजावणीत चार साधने आहेत जी MCP ची बाह्य API-चालित विविध कार्ये सुरक्षित आणि कार्यक्षमतेने हाताळण्याची क्षमता दर्शवतात:

- **general_search**: व्यापक वेब परिणामांसाठी
- **news_search**: अलीकडील प्रमुख बातम्यांसाठी
- **product_search**: ई-कॉमर्स डेटा साठी
- **qna**: प्रश्न-आणि-उत्तर तुकड्यांसाठी

### वैशिष्ट्ये
- **कोड उदाहरणे**: पायथनसाठी भाषानुसार कोड ब्लॉक्स (आणि इतर भाषांसाठी सहज वाढवता येणारे) स्पष्टतेसाठी कोड पिव्हॉटकसह समाविष्ट आहेत

### पायथन

```python
# general_search टूलचा उदाहरण वापर
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

क्लायंट चालवण्यापूर्वी, सर्व्हर काय करतो हे समजून घेणे उपयुक्त आहे. [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) फाइल MCP सर्व्हरची अंमलबजावणी करते, जी SerpAPI सह समाकलित होऊन वेब, बातम्या, उत्पादन शोध आणि प्रश्नोत्तरासाठी साधने उपलब्ध करून देते. ती आलेल्या विनंत्या हाताळते, API कॉल्स व्यवस्थापित करते, प्रतिसाद पार्स करते, आणि क्लायंटला संरचित निकाल परत देते.

तुम्ही पूर्ण अंमलबजावणी [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) मध्ये पाहू शकता.

येथे एक संक्षिप्त उदाहरण दिले आहे की सर्व्हर कसा साधन परिभाषित आणि नोंदवतो:

### पायथन सर्व्हर

```python
# server.py (उद्धरण)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...अमलबजावणी...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **बाह्य API समाकलन**: API कीज आणि बाह्य विनंत्या सुरक्षितपणे हाताळण्याचे प्रदर्शन
- **संरचित डेटा पार्सिंग**: API प्रतिसाद LLM-मैत्रीपूर्ण स्वरूपांमध्ये कसे रूपांतरित करायचे हे दर्शवते
- **त्रुटी हाताळणी**: योग्य लॉगिंगसह मजबूत त्रुटी हाताळणी
- **संवादात्मक क्लायंट**: स्वयंचलित चाचण्या आणि संवादात्मक मोड दोन्हीचा समावेश
- **संदर्भ व्यवस्थापन**: विनंत्या लॉगिंग आणि ट्रॅकिंगसाठी MCP संदर्भाचा वापर

## पूर्वअटी

सुरुवात करण्यापूर्वी, तुमचे वातावरण योग्यरित्या सेट केले आहे याची खात्री करा खालील पावले पाळून. यामुळे सर्व अवलंबित्वे स्थापित होतील आणि तुमच्या API की योग्यरित्या कॉन्फिगर होतील जेणेकरून विकास आणि चाचणी सुरळीत पार पडतील.

- पायथन 3.8 किंवा त्याहून वरची आवृत्ती
- SerpAPI API की ([SerpAPI](https://serpapi.com/) येथे साइन अप करा - मोफत स्तर उपलब्ध)

## स्थापना

सुरू करण्यासाठी, तुमचे वातावरण सेट करण्यासाठी खालील चरणांचे अनुसरण करा:

1. uv (शिफारसीय) किंवा pip वापरून अवलंबित्वे इंस्टॉल करा:

```bash
# uv वापरणे (शिफारस केलेले)
uv pip install -r requirements.txt

# pip वापरणे
pip install -r requirements.txt
```

2. प्रोजेक्ट रूटमध्ये `.env` फाइल तयार करा आणि त्यात तुमची SerpAPI की ठेवा:

```
SERPAPI_KEY=your_serpapi_key_here
```

## वापर

वेब शोध MCP सर्व्हर हा मुख्य घटक आहे जो SerpAPI सह समाकलित होऊन वेब, बातम्या, उत्पादन शोध, आणि प्रश्नोत्तरासाठी साधने उपलब्ध करून देतो. तो आलेल्या विनंत्या हाताळतो, API कॉल्स व्यवस्थापित करतो, प्रतिसाद पार्स करतो, आणि क्लायंटला संरचित निकाल परत करतो.

तुम्ही पूर्ण अंमलबजावणी [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) मध्ये पाहू शकता.

### सर्व्हर चालविणे

MCP सर्व्हर सुरू करण्यासाठी खालील आदेश वापरा:

```bash
python server.py
```

सर्व्हर एक stdio-आधारित MCP सर्व्हर म्हणून चालेल ज्याला क्लायंट थेट कनेक्ट होऊ शकतो.

### क्लायंट मोड्स

क्लायंट (`client.py`) MCP सर्व्हरशी संवाद साधण्यासाठी दोन मोड्सना समर्थन देतो:

- **सामान्य मोड**: सर्व साधने तपासणाऱ्या आणि त्यांच्या प्रतिसादांची पुष्टी करणाऱ्या स्वयंचलित चाचण्या चालवतो. सर्व्हर आणि साधने अपेक्षेप्रमाणे काम करत आहेत का ते पटकन तपासण्यासाठी उपयुक्त.
- **संवादात्मक मोड**: मेनू-चालित इंटरफेस सुरू करतो जिथे तुम्ही मॅन्युअली साधने निवडू शकता, कॉल करू शकता, सानुकूल प्रश्न प्रविष्ट करू शकता, आणि निकाल पाहू शकता प्रत्यक्ष वेळेत. हा सर्व्हरच्या क्षमता शोधण्यासाठी आणि वेगवेगळ्या इनपुटसह प्रयोग करण्यासाठी उत्तम.

तुम्ही पूर्ण अंमलबजावणी [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) मध्ये पाहू शकता.

### क्लायंट चालविणे

स्वयंचलित चाचण्या चालविण्यासाठी (हे आपोआप सर्व्हर सुरू करेल):

```bash
python client.py
```

किंवा संवादात्मक मोडमध्ये चालवा:

```bash
python client.py --interactive
```

या संदर्भात, "चाचणी स्क्रिप्ट" म्हणजे तुम्ही लिहिलेला एक सानुकूल पायथन प्रोग्राम जो MCP सर्व्हरचा क्लायंट म्हणून कार्य करतो. हे औपचारिक युनिट टेस्ट न राहता, तुम्हाला प्रोग्रामॅटिकली सर्व्हरशी कनेक्ट होऊन त्याच्या कोणत्याही साधनांना निवडलेल्या पॅरामीटर्ससह कॉल करण्यास आणि निकाल तपासण्यास परवानगी देते. हा दृष्टिकोन यासाठी उपयुक्त आहे:
- साधन कॉलची प्रोटोटायपिंग आणि प्रयोग करणे
- सर्व्हर वेगवेगळ्या इनपुटवर कसा प्रतिसाद देतो ते पडताळणे
- साधने पुन्हा पुन्हा कॉल करण्याचे स्वयंचलीकरण करणे
- MCP सर्व्हरवर तुमचे स्वतःचे कार्यप्रवाह किंवा समाकलन तयार करणे

तुम्ही चाचणी स्क्रिप्ट्सचा वेगवान वापर करून नवीन प्रश्नांची चाचणी करू शकता, साधनांची वर्तन तपासू शकता, किंवा प्रगत स्वयंचलिकरणासाठी सुरुवातीसाठी वापरू शकता. खाली MCP Python SDK वापरून अशी स्क्रिप्ट कशी तयार करायची याचे उदाहरण आहे:

## साधन वर्णने

तुम्ही खालील साधने वापरून वेगवेगळ्या प्रकारच्या शोध आणि प्रश्नांची पूर्तता करू शकता. प्रत्येक साधनाचे त्याचे पॅरामीटर्स आणि उदाहरण वापर संलग्न आहे.


हा विभाग उपलब्ध प्रत्येक साधन आणि त्यांचे पॅरामीटर्स याबाबत तपशील प्रदान करतो.

### general_search

सामान्य वेब शोध करतो आणि स्वरूपित निकाल परत करतो.

**हे साधन कसे कॉल करावे:**

तुम्ही तुमच्या स्वतःच्या स्क्रिप्टमधून MCP Python SDK वापरून `general_search` कॉल करू शकता, किंवा Inspector किंवा संवादात्मक क्लायंट मोड वापरून संवादात्मकपणे वापरू शकता. SDK वापरून कोड उदाहरण येथे आहे:

# [Python उदाहरण](#tab/python-general-search)

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_general_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "latest AI trends"})
            print(result)
```

---

पर्यायी, संवादात्मक मोडमध्ये, मेनूमधून `general_search` निवडा आणि विनंती केल्यावर आपला प्रश्न प्रविष्ट करा.

**पॅरामीटर्स:**
- `query` (स्ट्रिंग): शोध प्रश्न

**उदाहरण विनंती:**

```json
{
  "query": "latest AI trends"
}
```

```python
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("product_search", arguments={"query": "best AI gadgets 2025"})
            print(result)
```

## या धड्यातील प्रगत संकल्पना

बांधकाम सुरू करण्यापूर्वी, येथे काही महत्त्वाच्या प्रगत संकल्पना दिल्या आहेत ज्या या अध्यायादरम्यान दिसतील. यांचे समज तुम्हाला या विषयावर नवीन असतील तरीही सोपे होईल:

- **बहु-साधन योजनेचा वापर**: म्हणजे वेगवेगळ्या साधनांचे (जसे वेब शोध, बातम्या शोध, उत्पादन शोध, आणि प्रश्नोत्तर) एकाच MCP सर्व्हरमध्ये चालवणे. यामुळे तुमचा सर्व्हर फक्त एका नव्हे तर विविध कार्ये हाताळू शकतो.
- **API दर मर्यादा हाताळणी**: अनेक बाह्य API (जसे SerpAPI) ठराविक वेळेत किती विनंत्या करता येतील यावर मर्यादा घालतात. चांगला कोड या मर्यादांना तपासतो आणि काळजीपूर्वक हाताळतो, ज्यामुळे मर्यादा येताच अॅप तुटत नाही.
- **संरचित डेटा पार्सिंग**: API प्रतिसाद बहुतेकदा गुंतागुंतीचे आणि नेस्टेड असतात. ही संकल्पना म्हणजे त्या प्रतिसादांना स्वच्छ, वापरण्यास सोप्या स्वरूपांमध्ये रूपांतर करणे जे LLM किंवा इतर प्रोग्रामसाठी अनुकूल असतात.
- **त्रुटी पुनर्प्राप्ती**: कधी कधी काहीतरी चुकीचे होते—कदाचित नेटवर्क अपयशी ठरते, किंवा API अपेक्षित डेटा परत करत नाही. त्रुटी पुनर्प्राप्ती म्हणजे तुमचा कोड अशा समस्यांना हाताळू शकतो आणि उपयुक्त अभिप्राय देतो, ऐवजी क्रॅश होण्याऐवजी.
- **पॅरामीटर वैधता तपासणी**: याचा अर्थ तुमच्या साधनांसाठी सर्व इनपुट्स योग्य आणि सुरक्षित आहेत की नाही हे तपासणे. यात डीफॉल्ट मूल्ये सेट करणे आणि प्रकारांची पडताळणी करणे याचा समावेश आहे, जे बग्स आणि गोंधळ रोखण्यास मदत करतात.

हा विभाग तुम्हाला वेब शोध MCP सर्व्हर वापरताना येणाऱ्या सामान्य समस्या ओळखायला आणि सोडवायला मदत करेल. जर तुम्हाला त्रुटी किंवा अनपेक्षित वर्तन बघायला मिळाले, तर ही समस्या निवारण विभाग उपयोगी ठरेल. पुढील मदतीसाठी संपर्क करण्यापूर्वी या टिपा तपासा—खरंच बर्‍याचदा समस्या लवकरच सोडवतात.

## समस्या निवारण

वेब शोध MCP सर्व्हर वापरताना कधी कधी तुम्हाला समस्या येऊ शकतात—होय, बाह्य API आणि नवीन साधने वापरताना ही सामान्य बाब आहे. हा विभाग सामान्य समस्या सोडवण्यासाठी व्यावहारिक उपाय देतो, जेणेकरून तुम्ही लवकर रनिंगमध्ये येऊ शकाल. त्रुटी आली तर, येथे प्रारंभ करा: खालील टिप्स बहुतेक वापरकर्त्यांना येणाऱ्या समस्या सोडवतात आणि बहुतेक वेळा अतिरिक्त मदतीशिवाय सोडवतात.

### सामान्य समस्या

खाली वापरकर्त्यांना सर्वसामान्यपणे येणाऱ्या काही समस्या, त्यांचे स्पष्ट स्पष्टीकरण आणि सोडवण्याचे पावले दिली आहेत:

1. **`.env` फाईलमध्ये SERPAPI_KEY नसणे**
   - जर तुम्हाला त्रुटी दिसली `SERPAPI_KEY environment variable not found`, तर तुमची अॅप्लिकेशन API की शोधू शकत नाही जी SerpAPI वापरण्यासाठी आवश्यक आहे. यासाठी, प्रोजेक्ट रूट मध्ये `.env` नावाची फाइल तयार करा (जर आधी नसेल तर) आणि त्यात ओळीप्रमाणे जोडा `SERPAPI_KEY=तुमची_serpapi_key_इथे`. नक्की करा की `तुमची_serpapi_key_इथे` हा तुमच्या प्रत्यक्ष कीने बदलला आहे जो SerpAPI वेबसाइटवरून मिळाला आहे.

2. **मॉड्यूल नाही मिळाल्याचे त्रुटी**
   - `ModuleNotFoundError: No module named 'httpx'` सारखे त्रुटी दाखवते की आवश्यक पायथन पॅकेज इंस्टॉल झालेले नाही. हे बहुतेक वेळा घडते जेव्हा तुम्ही सर्व अवलंबित्वे इंस्टॉल केलेली नाहीत. हे सोडवण्यासाठी, टर्मिनलमध्ये `pip install -r requirements.txt` चालवा जेणेकरून तुमच्या प्रोजेक्टसाठी आवश्यक सर्व गोष्टी इंस्टॉल होतील.

3. **कनेक्शन समस्या**
   - जर तुम्हाला त्रुटी आली `Error during client execution`, तर याचा अर्थ क्लायंट सर्व्हरशी कनेक्ट होऊ शकत नाही किंवा सर्व्हर अपेक्षेनुसार चालू नाही. खात्री करा की दोन्ही क्लायंट आणि सर्व्हर सुसंगत आवृत्त्या आहेत, आणि `server.py` आवश्यक योग्य फोल्डरमध्ये आहे व चालू आहे. सर्व्हर आणि क्लायंट दोन्ही रीस्टार्ट करणे देखील उपयोगी ठरू शकते.

4. **SerpAPI त्रुटी**
   - `Search API returned error status: 401` असाच त्रुटी येत असल्यास, तुमची SerpAPI की गायब, चुकीची किंवा कालबाह्य आहे. तुमच्या SerpAPI डॅशबोर्डमध्ये जाऊन की तपासा, आणि आवश्यक असल्यास `.env` फाइल अपडेट करा. जर की बरोबर असेल तरी त्रुटी येत असल्यास, तपासा की तुमच्या मोफत टियरने कोटा संपुष्टात आणलेला नाही.

### डीबग मोड

सामान्यपणे, अॅप फक्त महत्त्वाची माहिती लॉग करते. तुम्हाला जर अधिक तपशील पाहायचे असतील (उदा., गुंतागुंतीच्या समस्या ओळखण्यासाठी), तर तुम्ही DEBUG मोड सक्षम करू शकता. यामुळे अॅप प्रत्येक टप्प्याविषयी अधिक माहिती दाखवेल.

**उदाहरण: सामान्य आउटपुट**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**उदाहरण: DEBUG आउटपुट**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

लक्षात घ्या की DEBUG मोडमध्ये HTTP विनंत्या, प्रतिसाद, आणि इतर अंतर्गत तपशीलाचे अतिरिक्त ओळी असतात. हे समस्या सोडवण्यासाठी खूप उपयुक्त ठरू शकते.

DEBUG मोड सक्षम करण्यासाठी, तुमच्या `client.py` किंवा `server.py` च्या शीर्षस्थानी लॉगिंग पातळी DEBUG वर सेट करा:

# [Python](#tab/python-debug)


```python
# आपल्या client.py किंवा server.py च्या शीर्षस्थानी
import logging
logging.basicConfig(
    level=logging.DEBUG,  # INFO वरून DEBUG मध्ये बदला
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
