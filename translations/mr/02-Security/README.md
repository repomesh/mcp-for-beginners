# MCP सुरक्षा: AI प्रणालींसाठी सर्वसमावेशक संरक्षण

[![MCP सुरक्षा उत्तम पद्धती](../../../translated_images/mr/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(या धड्याचा व्हिडिओ बघण्यासाठी वर दाखवलेली प्रतिमा क्लिक करा)_

सुरक्षा ही AI प्रणाली डिझाइनसाठी मूलभूत असते, म्हणूनच आम्ही त्याला आमच्या दुसऱ्या विभागात प्राधान्य देतो. हे मायक्रोसॉफ्टच्या **डिझाइनद्वारे सुरक्षित** तत्वाशी सुसंगत आहे, जे [सिक्युअर फ्यूचर इनिशिएटिव्ह](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) कडून आले आहे.

मॉडेल कॉन्टेक्स्ट प्रोटोकॉल (MCP) AI-चालित अनुप्रयोगांसाठी सामर्थ्यशाली नवीन क्षमता आणतो तर पारंपरिक सॉफ्टवेअर जोखमींपलीकडे विस्तृत असलेल्या अनन्य सुरक्षा आव्हानांना देखील जन्म देतो. MCP प्रणालींना पारंपरिक सुरक्षा चिंतांसह (सुरक्षित कोडिंग, किमान अधिकार, पुरवठा साखळी सुरक्षा) तसेच AI-विशिष्ट धोके जसे की प्रॉम्प्ट इंजेक्शन, टूल विषबाधा, सत्र हायजॅकिंग, भ्रमित डिप्युटी हल्ले, टोकन पासथ्रू असुरक्षितता, आणि गतिशील क्षमता बदल यांचा सामना करावा लागतो.

हा धडा MCP अंमलबजावणीतील सर्वात महत्त्वाच्या सुरक्षा धोके तपासतो—प्रमाणीकरण, प्राधिकरण, जास्त परवानग्या, अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन, सत्र सुरक्षा, भ्रमित डिप्युटी समस्या, टोकन व्यवस्थापन, आणि पुरवठा साखळी असुरक्षितता यांवर चर्चा करतो. तुम्ही या धोके कमी करण्यासाठी कृतीक्षम नियंत्रण आणि उत्तम पद्धती शिकाल, तसेच Microsoft च्या Prompt Shields, Azure Content Safety, आणि GitHub Advanced Security सारख्या उपायांचे वापर करून तुमच्या MCP तैनातीसाठी बळकट संरक्षण मिळवाल.

## शिकण्याचे उद्दिष्टे

या धड्याच्या शेवटी, तुम्ही सक्षम असाल:

- **MCP-विशिष्ट धमक्या ओळखणे**: MCP प्रणालीतील अनन्य सुरक्षा धोके ओळखणे जसे की प्रॉम्प्ट इंजेक्शन, टूल विषबाधा, जास्त परवानग्या, सत्र हायजॅकिंग, भ्रमित डिप्युटी समस्या, टोकन पासथ्रू असुरक्षितता, आणि पुरवठा साखळी धोके
- **सुरक्षा नियंत्रण लागू करणे**: प्रभावी प्रतिबंधक उपाय राबविणे ज्यात मजबूत प्रमाणीकरण, किमान अधिकार, सुरक्षित टोकन व्यवस्थापन, सत्र सुरक्षा नियंत्रण, आणि पुरवठा साखळी पडताळणी यांचा समावेश आहे
- **Microsoft सुरक्षा उपायांचा लाभ घेणे**: MCP वर्कलोड संरक्षणासाठी Microsoft Prompt Shields, Azure Content Safety, आणि GitHub Advanced Security समजून घेणे आणि तैनात करणे
- **टूल सुरक्षा प्रमाणपत्र देणे**: टूल मेटाडेटा प्रमाणीकरणाचे महत्त्व ओळखणे, गतिशील बदलांसाठी निरीक्षण करणे, आणि अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन हल्ल्यांपासून संरक्षण करणे
- **उत्तम पद्धती एकत्रित करणे**: स्थापित सुरक्षा मूलभूत तत्त्वे (सुरक्षित कोडिंग, सर्व्हर हार्डनिंग, झिरो ट्रस्ट) MCP-विशिष्ट नियंत्रणांसह संपूर्ण संरक्षणासाठी एकत्र करणे

# MCP सुरक्षा आर्किटेक्चर व नियंत्रण

आधुनिक MCP अंमलबजावण्या थोड्या थरावरील सुरक्षा दृष्टिकोनांची गरज ठेवतात, ज्या पारंपरिक सॉफ्टवेअर सुरक्षा तसेच AI-विशिष्ट धमक्यांना हाताळतात. जलद विकसित होणारी MCP तपशीलवार सुरक्षात्मक नियंत्रण सुधारते, ज्यामुळे एंटरप्राइझ सुरक्षा आर्किटेक्चर आणि स्थापित उत्तम पद्धतींसह उत्तम समाकलन शक्य होते.

[Microsoft Digital Defense Report](https://aka.ms/mddr) मधील संशोधन दर्शविते की **98% नोंदवलेल्या भंगांवर मजबूत सुरक्षा निगडीत धोरणांमुळे प्रतिबंध होऊ शकतो**. सर्वोत्तम संरक्षण धोरण हे आधारभूत सुरक्षा पद्धती व MCP-विशिष्ट नियंत्रण यांचे संयोजन आहे—प्रमाणित सुरक्षात्मक उपाय हे एकूण सुरक्षा धोका कमी करण्यासाठी सर्वात प्रभावी ठरतात.

## वर्तमान सुरक्षा वातावरण

MCP प्रोटोकॉल जलद प्रगत होत आहे, आणि भविष्यातील अंमलबजावणी नवीन प्रमाणीकरण पद्धती आणि सुधारित नियंत्रण आणू शकतात. सद्याच्या मार्गदर्शनासाठी सद्य [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), आणि [सुरक्षा उत्तम पद्धती दस्तऐवज](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) यांचा संदर्भ घ्या.

> **भावी दृष्टीने:** `2026-07-28` रिलिझ कॅंडिडेट अधिक कडक प्राधिकरण आणतो—ग्राहकांना `iss` पॅरामीटर अवलंबीत करणे आवश्यक आहे (RFC 9207), डायनामिक क्लायंट नोंदणीदरम्यान OpenID Connect `application_type` जाहीर करणे, आणि नोंदणीकृत प्रमाणपत्रे जारी करणाऱ्या प्राधिकरण सर्व्हरशी बांधणे आवश्यक आहे. सर्व प्राधिकरण SEP सूची पाहण्यासाठी [What's Changing in MCP: The 2026-07-28 Release Candidate](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) वाचा.

### कार्यशाळेचा आढावा

[MCP सुरक्षा शिखर सम्मेलन कार्यशाळा](https://azure-samples.github.io/sherpa/) एक सिद्ध "सुरक्षात्मक त्रुटी → वापर → दुरुस्ती → पडताळणी" पद्धतीद्वारे प्रायोगिक आणि कृतीक्षम सुरक्षा प्रशिक्षण प्रदान करते. तुम्ही:

- **तुटण्याने शिकणे**: जानून अनुचितपणे असुरक्षित सर्व्हरचा वापर करून भेद्यता अनुभवा
- **Azure-देशी सुरक्षा वापरणे**: Azure Entra ID, Key Vault, API Management, आणि AI Content Safety चा लाभ घ्या
- **डिफेन्स-इन-डेप्थ अनुसरणे**: सर्व सुरक्षा स्तर तयार करणाऱ्या छावण्या यातून पुढे जा
- **OWASP मानके लागू करा**: प्रत्येक तंत्र [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) शी जुळते
- **उत्पादन कोड मिळवा**: कार्यरत, चाचणी केलेल्या अंमलबजावणीसह कार्यशाळेचे समारोप करा

### MCP प्रमाणीकरणाचा विकास

MCP तपशीलवार प्रमाणीकरण व प्राधिकरणात महत्त्वपूर्ण प्रगती केली आहे:

- **मूळ पद्धत**: सुरुवातीच्या तपशीलवार डेव्हलपर्सना सानुकूल प्रमाणीकरण सर्व्हर तयार करण्याची गरज होती, MCP सर्व्हर OAuth 2.0 प्राधिकरण सर्व्हर म्हणून थेट वापरकर्त्य़ांचे प्रमाणीकरण व्यवस्थापित करत होते
- **सध्याचा मानक (2025-11-25)**: सुधारित तपशीलवार MCP सर्व्हरना बाह्य ओळख प्रदात्यांकडे (जसे की Microsoft Entra ID) प्रमाणीकरण डेलीगेट करण्याची परवानगी देते, ज्यामुळे सुरक्षा स्थिती सुधारते आणि अंमलबजावणीची गुंतागुंत कमी होते
- **ट्रान्सपोर्ट लेयर सुरक्षा**: स्थानिक (STDIO) आणि दूरस्थ (Streamable HTTP) कनेक्शनसाठी योग्य प्रमाणीकरण नमुन्यांसह सुरक्षित पारगमन यंत्रणेचा विस्तार

## प्रमाणीकरण व प्राधिकरण सुरक्षा

### सध्याच्या सुरक्षा आव्हाने

आधुनिक MCP अंमलबजावण्यांना काही प्रमाणीकरण आणि प्राधिकरण आव्हानांचा सामना करावा लागतो:

### धोके व धमक्या

- **चुकीची प्राधिकरण लॉजिक**: MCP सर्व्हरमधील त्रुटीमुळे संवेदनशील डेटा उघड होऊ शकतो आणि प्रवेश नियंत्रण अयोग्यपणे लागू होऊ शकते
- **OAuth टोकन दुरुपयोग**: स्थानिक MCP सर्व्हर टोकन चोरल्यास हल्लेखोर सर्व्हरची नक्कल करून डाउनस्ट्रीम सेवा वापरू शकतात
- **टोकन पासथ्रू असुरक्षितता**: अनुचित टोकन हाताळणीमुळे सुरक्षा नियंत्रण बायपास होऊ शकतात आणि जबाबदारी गहाळ राहू शकते
- **जास्त परवानग्या**: अतिपरवानगी असलेले MCP सर्व्हर किमान अधिकार तत्त्वांचे उल्लंघन करतात आणि हल्ल्यांच्या संधी वाढवतात

#### टोकन पासथ्रू: एक गंभीर प्रतिमूल्य नमुना

**टोकन पासथ्रू स्पष्टपणे मनाई करण्यात आली आहे** सध्याच्या MCP प्राधिकरण तपशीलवारमध्ये गंभीर सुरक्षा परिणामांमुळे:

##### सुरक्षा नियंत्रण फोडणे
- MCP सर्व्हर आणि डाउनस्ट्रीम API महत्त्वाचे सुरक्षा नियंत्रण राबवतात (दर मर्यादा, विनंती पडताळणी, ट्रॅफिक निरीक्षण) जे योग्य टोकन पडताळणीवर अवलंबून असतात
- थेट क्लायंट-टू-API टोकन वापर या आवश्यक संरक्षणांची फोड घालते, सुरक्षा आर्किटेक्चर कमजोर करते

##### जबाबदारी व लेखापरीक्षा आव्हाने  
- MCP सर्व्हर क्लायंट जवळून आलेले टोकन ओळखू शकत नाहीत, ज्यामुळे लेखापरीक्षा ट्रेल तुटतो
- डाउनस्ट्रीम रिसोर्स सर्व्हर लॉगमध्ये चुकीचे विनंतीचे स्त्रोत दिसतात, वास्तविक MCP सर्व्हर मध्यस्थेऐवजी
- घटना तपासणी आणि अनुपालन लेखापरीक्षा अत्यंत कठीण होते

##### माहिती चोरी धोके
- अव्यवस्थित टोकन दावा मालकांना त्यांचे टोकन चोरी झाले तरी MCP सर्व्हर प्रॉक्सी म्हणून वापरून डेटा चोरी करण्याची परवानगी मिळू शकते
- विश्वासाचा मर्यादा उल्लंघन होऊन अप्रतिष्ठित प्रवेश प्रकार सुरक्षात्मक नियंत्रणांपासून वाचू शकतात

##### बहु-सेवा हल्ल्याचे मार्ग
- दुरुपयोग झालेले टोकन अनेक सेवांकडून स्वीकारले जातात, ज्यामुळे संबंधित प्रणालींमध्ये क्षैतिज हालचाल शक्य होते
- सेवा दरम्यान विश्वास-गृहितके भंग होऊ शकतात जेव्हा टोकन उत्पत्ती तपासली जाऊ शकत नाही

## AI-विशिष्ट सुरक्षा धमक्या

### प्रॉम्प्ट इंजेक्शन व टूल हस्तक्षेप हल्ले

आधुनिक MCP अंमलबजावण्या अशा प्रगत AI-विशिष्ट हल्ल्यांच्या मार्गाचा सामना करतात ज्यांना पारंपरिक सुरक्षा उपाय पूर्णपणे हाताळू शकत नाहीत:

#### **अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन (क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन)**

**अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन** हा MCP-सक्षम AI प्रणालीतील सर्वात गंभीर असुरक्षिततांपैकी एक आहे. हल्लेखोर बाह्य सामग्री (दस्तऐवज, वेब पृष्ठे, ईमेल, डेटा स्रोत) मध्ये दुष्ट सूचना लपवतात, ज्या AI प्रणाली नंतर वैध आदेश म्हणून प्रक्रियावतात.

**हल्ल्याचे संदर्भ:**
- **दस्तऐवज-आधारित इंजेक्शन**: प्रक्रियात दस्तऐवजांमध्ये लपलेल्या दुष्ट सूचनांमुळे अनपेक्षित AI क्रिया सुरु होतात
- **वेब सामग्री शोषण**: संक्रमित वेब पृष्ठांमध्ये एम्बेड केलेले प्रॉम्प्ट जे स्क्रॅप केल्यावर AI वर्तनाला प्रभावित करतात
- **ईमेल-आधारित हल्ले**: ईमेलमधील दुष्ट प्रॉम्प्ट AI सहाय्यकांना माहिती चुकार करण्यास किंवा अनधिकृत क्रिया करण्यास भाग पाडतात
- **डेटा स्रोत प्रदूषण**: संक्रमित डेटाबेस किंवा API जी दूषित सामग्री AI प्रणालींना पुरवते

**वास्तविक परिणाम**: हे हल्ले डेटा चुकारणे, गोपनीयता भंग, हानिकारक सामग्री निर्मिती, आणि वापरकर्ता संवादांचे विळखीकरण करू शकतात. तपशीलवार विश्लेषणासाठी, बघा [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).

![प्रॉम्प्ट इंजेक्शन हल्ला आरेख](../../../translated_images/mr/prompt-injection.ed9fbfde297ca877.webp)

#### **टूल विषबाधा हल्ले**

**टूल विषबाधा** MCP टूल्सची व्याख्या करणाऱ्या मेटाडेटावर लक्ष्य करते, तेव्हा LLM टूल वर्णन आणि पॅरामीटर कसे समजतात हे दुरुपयोग करते.

**हल्ल्याचे तंत्र:**
- **मेटाडेटा फेरफार**: हल्लेखोर टूल वर्णने, पॅरामीटर व्याख्या, किंवा वापर उदाहरणांमध्ये दुष्ट सूचना टाकतात
- **दृश्य नसलेली सूचना**: टूल मेटाडेटामध्ये लपलेले प्रॉम्प्ट AI मॉडेलद्वारे प्रक्रिया केले जातात पण मानव वापरकर्त्यांना दिसत नाहीत
- **गतिशील टूल बदल ("रग पुल्स")**: वापरकर्त्यांनी मंजूर केलेले टूल नंतर वापरकर्त्यांच्या अनभिज्ञतेत दुष्ट क्रिया करण्यासाठी बदलले जातात
- **पॅरामीटर इंजेक्शन**: टूल पॅरामीटर स्कीमामध्ये एम्बेड केलेली दुष्ट सामग्री जी मॉडेलच्या वर्तनावर परिणाम करते


**होस्ट केलेल्या सर्व्हरचा धोका**: रिमोट MCP सर्व्हरमध्ये टूल डिफिनिशन्स वापरकर्त्याच्या सुरुवातीच्या मंजुरीनंतर अद्यतनित होऊ शकतात, ज्यामुळे अशा परिस्थती निर्माण होतात जिथे पूर्वी सुरक्षित असलेले टूल्स धोकादायक बनतात. सर्वसमावेशक विश्लेषणासाठी, पाहा [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks).

![Tool Injection Attack Diagram](../../../translated_images/mr/tool-injection.3b0b4a6b24de6bef.webp)

#### **अतिरिक्त AI हल्ल्याचे मार्ग**

- **क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन (XPIA)**: अनेक डोमेन्समधील सामग्रीचा वापर करून सुरक्षा नियंत्रण ओलांडणारे कुशल हल्ले
- **डायनामिक क्षमता बदल**: टूल क्षमतांमध्ये वास्तविक-वेळेतील बदल जे सुरुवातीच्या सुरक्षा मूल्यांकनांपासून सुटका करतात
- **कॉन्क्स्ट विंडो पॉइझनिंग**: मोठ्या संदर्भ खिडक्यांचे दुरुपयोग करून धोकादायक सूचना छुपविणारे हल्ले
- **मॉडेल कन्फ्युजन हल्ले**: मॉडेलच्या मर्यादा वापरून अनपेक्षित किंवा असुरक्षित वर्तन तयार करणे


### AI सुरक्षा धोका परिणाम

**उच्च परिणामकारक परिणाम:**
- **डेटा चोरी**: संवेदनशील एंटरप्राइझ किंवा वैयक्तिक डेटाचा अनधिकृत प्रवेश व चोरी
- **गोपनीयता भंग**: वैयक्तिक ओळख पटवणारी माहिती (PII) आणि गोपनीय व्यवसाय डेटा उघड होणे  
- **सिस्टम हस्तक्षेप**: महत्त्वाच्या सिस्टम्स आणि कार्यप्रवाहांमध्ये अनपेक्षित बदल
- **प्रमाणपत्र चोरी**: प्रमाणीकरण टोकन्स आणि सेवा प्रमाणपत्रांचे भंग
- **लॅटरल मूव्हमेंट**: धोका पोहोचलेल्या AI सिस्टमचा वापर व्यापक नेटवर्क हल्ल्यांसाठी नोड म्हणून

## प्रगत MCP सुरक्षा धोके

### सत्र हायजॅकिंग असुरक्षितता

**सत्र हायजॅकिंग** ही राज्यपूर्ण MCP अंमलबजावणीत एक गंभीर हल्ला मार्ग आहे जिथे अनधिकृत पक्ष योग्य सत्र आयडेंटिफायर्स मिळवून ग्राहकांप्रती impersonate करतात आणि अनधिकृत क्रिया करतात.

#### **हल्ल्याच्या परिस्थिति आणि धोके**

- **सत्र हायजॅक प्रॉम्प्ट इंजेक्शन**: चोरी केलेल्या सत्र आयडीसह हल्लेखोर सर्व्हरवर मालिशस घटना इंजेक्ट करतात, ज्यामुळे हानिकारक क्रिया होऊ शकतात किंवा संवेदनशील डेटामध्ये प्रवेश होऊ शकतो
- **प्रत्यक्ष impersonation**: चोरी केलेल्या सत्र आयडींनी प्रमाणीकरण टाळून थेट MCP सर्व्हर कॉल करणे ज्यामुळे हल्लेखोरांना वैध वापरकर्ते समजले जाते
- **बाधित पुनरारंभयोग्य स्ट्रीम्स**: हल्लेखोर विनंत्या तिचा पूर्वीच बंद करु शकतात, ज्यामुळे वैध ग्राहकांना संभाव्य धोकादायक सामग्रीसह पुन्हा सुरूवात करावी लागू शकते

#### **सत्र व्यवस्थापनासाठी सुरक्षा नियंत्रण**

**महत्त्वाच्या गरजा:**
- **प्राधिकरण पडताळणी**: प्रमाणीकृत MCP सर्व्हरने सर्व येणाऱ्या विनंत्यांची पडताळणी करणे *आवश्यक* आहे आणि सत्रांवर प्रमाणीकरणासाठी अवलंबून राहू नये
- **सुरक्षित सत्र निर्मिती**: क्रिप्टोग्राफिकली सुरक्षित, अनिश्चित सत्र IDs तयार करण्यासाठी सुरक्षित रँडम नंबर जनरेटरचा वापर करा
- **वापरकर्ता-विशिष्ट बांधणी**: वापरकर्त्यासाठी विशिष्ट माहितीसह सत्र आयडी बांधणी करा जसे की `<user_id>:<session_id>` जेणेकरून वेगवेगळ्या वापरकर्त्यांच्या सत्राचा गैरवापर टाळता येईल
- **सत्र जीवनचक्र व्यवस्थापन**: सत्र कालबाह्यता, रोटेशन आणि निषेधांसाठी योग्य अंमलबजावणी करा जेणेकरून धोका विंडो मर्यादित राहील
- **वाहतूक सुरक्षा**: सर्व संवादांसाठी HTTPS अनिवार्य करा जेणेकरून सत्र आयडी चोरट्यापासून संरक्षण मिळेल

### निराकरण समस्या (Confused Deputy Problem)

**निराकरण समस्या** तेव्हा उद्भवते जेव्हा MCP सर्व्हर ग्राहक आणि तृतीय-पक्ष सेवा यांच्यात प्रमाणीकरण प्रॉक्सी म्हणून कार्य करतो, ज्यामुळे स्थिर ग्राहक आयडीचा गैरवापर करून प्राधिकरण टाळण्याची संधी निर्माण होते.

#### **हल्ल्याचा मेकॅनिझम आणि धोके**

- **कुकी-आधारित संमती टाळणे**: पूर्वीच्या वापरकर्त्याच्या प्रमाणीकरणामुळे संमती कुकीज तयार होतात ज्याचा वापर हल्लेखोर निकृष्ट प्रमाणीकरण विनंत्यांसह घोंगूस redirect URIs वापरून करतात
- **प्राधिकरण कोड चोरी**: विद्यमान संमती कुकीजमुळे प्राधिकरण सर्व्हर संमती स्क्रीन टाळू शकतात, कोड हल्लेखोरांच्या नियंत्रणातील एंडपॉइंटवर पुनर्निर्देशित करतात  
- **अनधिकृत API प्रवेश**: चोरी केलेले प्राधिकरण कोड टोकन देवाणघेवाणसाठी परवानगी देतात आणि वापरकर्त्याचे impersonation शक्य करतात

#### **प्रतिबंधात्मक धोरणे**

**अनिवार्य नियंत्रण:**
- **स्पष्ट संमती आवश्यकता**: स्थिर ग्राहक आयडी वापरणारे MCP प्रॉक्सी सर्व्हर प्रत्येक गतिशील नोंदणीकृत ग्राहकासाठी वापरकर्त्याची संमती मिळवणे *आवश्यक* आहे
- **OAuth 2.1 सुरक्षा अंमलबजावणी**: सर्व प्राधिकरण विनंत्यांसाठी सध्याच्या OAuth सुरक्षा सर्वोत्तम पद्धतींचे अनुसरण करा ज्यामध्ये PKCE (प्रूफ की फॉर कोड एक्सचेंज) समाविष्ट आहे
- **कडक ग्राहक पडताळणी**: redirect URIs आणि ग्राहक ओळखपत्रांचे कडक पडताळणी करण्यासाठी अंमलबजावणी करा जेणेकरून गैरवापर टाळता येईल

### टोकन पासथ्रू असुरक्षितता  

**टोकन पासथ्रू** हा एक स्पष्ट विरोधी नमुना आहे जिथे MCP सर्व्हर योग्य पडताळणी न करता ग्राहक टोकन्स स्वीकारतात आणि नंतरच्या API कडे पाठवतात, जे MCP प्राधिकरण तपशीलांचे उल्लंघन करते.

#### **सुरक्षा परिणाम**

- **नियंत्रण फाटणे**: थेट ग्राहक-ते-API टोकन वापर दर मर्यादा, पडताळणी, आणि निरीक्षण नियंत्रणांना टाळते
- **ऑडिट ट्रेल भ्रष्टाचार**: अपस्ट्रीम जारी केलेले टोकन्स ग्राहक ओळख शकत नाहीत, ज्यामुळे घटना तपासणी अशक्य होते
- **प्रॉक्सी आधारित डेटा चोरी**: न तपासलेले टोकन्स अनधिकृत डेटा प्रवेशासाठी सर्व्हरला प्रॉक्सी म्हणून वापरण्याची परवानगी देतात
- **विश्वास सीमा भंग**: टोकन स्रोत तपासता येत नाहीत त्या वेळी नंतरच्या सेवेची विश्वास गृहितके भंग होऊ शकतात
- **अनेक सेवा हल्ल्याचा विस्तार**: अनेक सेवांमध्ये स्वीकारलेले कमकुवत टोकन्स लॅटरल मूव्हमेंटला अनुमती देतात

#### **आवश्यक सुरक्षा नियंत्रण**

**अनाकिपूर्ण गरजा:**
- **टोकन पडताळणी**: MCP सर्व्हरने अशा टोकन्स स्वीकारू नयेत जे नेमके MCP सर्व्हरसाठी जारी केलेले नाहीत
- **प्रेक्षक पडताळणी**: टोकन प्रेक्षक दावा नेहमीच MCP सर्व्हरच्या ओळखीशी जुळतात याची पडताळणी करा
- **योग्य टोकन जीवनचक्र**: सुरक्षित रोटेशन प्रथांसह अल्पकालीन प्रवेश टोकन्सची अंमलबजावणी करा


## AI सिस्टमसाठी सप्लाय चेन सुरक्षा

सप्लाय चेन सुरक्षा पारंपरिक सॉफ्टवेअर अवलंबनांपेक्षा पुढे वाढली आहे आणि पूर्ण AI परिसंस्थेचा समावेश करते. आधुनिक MCP अंमलबजावण्या सर्व AI-संबंधित घटकांची काटेकोरपणे पडताळणी आणि निरीक्षण करणे आवश्यक आहे कारण प्रत्येक घटक संभाव्य असुरक्षितता निर्माण करतो ज्यामुळे सिस्टम अखंडता धोक्यात येऊ शकते.

### विस्तारित AI सप्लाय चेन घटक

**पारंपरिक सॉफ्टवेअर अवलंबने:**
- मुक्त स्रोत लायब्ररी आणि फ्रेमवर्क्स
- कंटेनर प्रतिमा आणि बेस सिस्टम  
- विकास साधने आणि बिल्ड पाईपलाईन्स
- पायाभूत सुविधा घटक आणि सेवा

**AI-विशिष्ट सप्लाय चेन घटक:**
- **फाउंडेशन मॉडेल**: विविध पुरवठादारांकडून प्री-ट्रेंड मॉडेल्स, प्रामाणिकता पडताळणी आवश्यक
- **एम्बेडिंग सेवा**: बाह्य वेक्टरायझेशन आणि सेमॅंटिक शोध सेवा
- **संदर्भ पुरवठादार**: डेटा स्रोत, ज्ञानाधारे, आणि दस्तऐवज संग्रह  
- **तृतीय-पक्ष API**: बाह्य AI सेवा, ML पाईपलाईन्स, आणि डेटा प्रक्रिया एंडपॉइंट्स
- **मॉडेल कलाकृती**: वजन, कॉन्फिगरेशन्स, आणि फाईन-ट्यून केलेले मॉडेल प्रकार
- **प्रशिक्षण डेटा स्रोत**: मॉडेल प्रशिक्षण आणि फाईन-ट्यूनिंगसाठी वापरलेले डेटासेट्स

### सर्वसमावेशक सप्लाय चेन सुरक्षा धोरण

#### **घटक पडताळणी आणि विश्वास**
- **मूळ तपासणी**: सर्व AI घटकांचे स्रोत, परवानगी, आणि अखंडता एकत्रीकरणापूर्वी पडताळा
- **सुरक्षा मूल्यांकन**: मॉडेल, डेटा स्रोत, आणि AI सेवा यांच्यावर असुरक्षितता स्कॅन आणि सुरक्षा पुनरावलोकने करा
- **प्रतिष्ठा विश्लेषण**: AI सेवा पुरवठादारांच्या सुरक्षा नोंदी आणि व्यवहारांचे मूल्यांकन करा
- **अनुपालन पडताळणी**: सर्व घटक संघटनेच्या सुरक्षा आणि नियमपालन आवश्यकतांची पूर्तता करतात याची खात्री करा

#### **सुरक्षित तैनाती पाईपलाईन्स**  
- **स्वयंचलित CI/CD सुरक्षा**: स्वयंचलित तैनाती पाईपलाईन्समध्ये सुरक्षा स्कॅनिंग समाकलित करा
- **कलाकृती अखंडता**: सर्व तैनात कलाकृतींसाठी (कोड, मॉडेल, कॉन्फिगरेशन) क्रिप्टोग्राफिक पडताळणी अंमलबजावणी करा
- **स्तरीय तैनाती**: प्रत्येक टप्यात सुरक्षा पडताळणीसह प्रगत तैनात धोरणे वापरा
- **विश्वासार्ह कलाकृती संग्रह**: फक्त पडताळलेले, सुरक्षित कलाकृती रजिस्ट्री आणि संचयकांमधून तैनात करा

#### **सतत निरीक्षण आणि प्रतिसाद**
- **अवलंबन स्कॅनिंग**: सर्व सॉफ्टवेअर आणि AI घटक अवलंबनांसाठी सतत असुरक्षितता निरीक्षण
- **मॉडेल निरीक्षण**: मॉडेल वर्तन, कार्यप्रदर्शनात बदल, आणि सुरक्षा असामान्यता यांचे सातत्याने मूल्यांकन
- **सेवा आरोग्य ट्रॅकिंग**: बाह्य AI सेवांसाठी उपलब्धता, सुरक्षा घटना, आणि धोरण बदल यांचे निरीक्षण
- **धोका बुद्धिमत्ता समाकलन**: AI आणि ML सुरक्षा धोका संकेतांसाठी विशिष्ट थेट फीड्स समाकलित करा

#### **प्रवेश नियंत्रण आणि किमान विशेषाधिकार**
- **घटक-स्तरीय परवानग्या**: व्यवसाय गरजेनुसार मॉडेल, डेटा, आणि सेवा प्रवेश मर्यादित करा
- **सेवा खाते व्यवस्थापन**: किमान आवश्यक परवानग्यांसह समर्पित सेवा खाते अंमलबजावणी करा
- **नेटवर्क विभाजन**: AI घटक वेगळे करा आणि सेवा दरम्यान नेटवर्क प्रवेश मर्यादित करा
- **API गेटवे नियंत्रण**: बाह्य AI सेवांसाठी प्रवेश नियंत्रित आणि निरीक्षण करण्यासाठी केंद्रीकृत API गेटवे वापरा

#### **घटना प्रतिसाद आणि पुनर्प्राप्ती**
- **त्वरित प्रतिसाद प्रक्रिया**: बाधित AI घटक दुरुस्त किंवा बदलेण्यासाठी स्थापन प्रक्रिया
- **प्रमाणपत्र फेरफटका**: रहस्य, API किल्ली, आणि सेवा प्रमाणपत्रे स्वयंचलित फेरफटका प्रणाली
- **रोलबॅक क्षमता**: AI घटकांच्या पूर्वीचे ज्ञात चांगले आवृत्त्यांकडे जलद परत जाण्याची क्षमता
- **सप्लाय चेन उल्लंघन पुनर्प्राप्ती**: अपस्ट्रीम AI सेवा अधिकाऱ्यांसाठी विशिष्ट प्रतिसाद प्रक्रिया

### Microsoft सुरक्षा साधने आणि एकत्रीकरण

**GitHub एडव्हान्स्ड सुरक्षा** पूर्ण सप्लाय चेन संरक्षण पुरवते ज्यामध्ये समाविष्ट आहेत:
- **गुपित स्कॅनिंग**: रेपॉझिटरीजमधील प्रमाणपत्रे, API कीज, आणि टोकन्सचा स्वयंचलित शोध
- **अवलंबन स्कॅनिंग**: मुक्त स्रोत अवलंबने आणि लायब्रऱ्यांसाठी असुरक्षितता मूल्यांकन
- **CodeQL विश्लेषण**: सुरक्षा असुरक्षितता आणि कोडिंग समस्यांसाठी स्थिर कोड विश्लेषण
- **सप्लाय चेन अंतर्दृष्टी**: अवलंबनांची आरोग्य आणि सुरक्षा स्थितीची दृश्यता

**Azure DevOps आणि Azure Repos एकत्रीकरण:**
- Microsoft विकास व्यासपीठांवर अखंड सुरक्षा स्कॅनिंग समाकलन
- Azure पाईपलाईन्समध्ये AI कार्यभारासाठी स्वयंचलित सुरक्षा तपासणी
- सुरक्षित AI घटक तैनातीसाठी धोरण अंमलबजावणी

**Microsoft अंतर्गत पद्धती:**
Microsoft सर्व उत्पादनेभर मोठ्या प्रमाणावर सप्लाय चेन सुरक्षा पद्धती अंमलबजावते. तज्ञ दृष्टीकोनासाठी पाहा [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/).


## पाया सुरक्षा सर्वोत्तम पद्धती

MCP अंमलबजावण्या आपल्या संस्थेच्या विद्यमान सुरक्षा स्थितीवर आधारित आणि त्याचा विकास करतात. पाया सुरक्षा पद्धती मजबूत केल्याने AI सिस्टम्स आणि MCP तैनातींच्या एकूण सुरक्षेत लक्षणीय वाढ होते.

### मुख्य सुरक्षा मूलतत्वे

#### **सुरक्षित विकास पद्धती**
- **OWASP अनुपालन**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) वेब अनुप्रयोग असुरक्षिततेपासून संरक्षण
- **AI-विशिष्ट संरक्षणे**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) साठी नियंत्रण अंमलबजावणी
- **सुरक्षित रहस्य व्यवस्थापन**: टोकन्स, API कीज, आणि संवेदनशील कॉन्फिगरेशन डेटासाठी समर्पित कणा वापरा
- **एकापासून-एकापर्यंत एन्क्रिप्शन**: सर्व अनुप्रयोग घटक व डेटा प्रवाहांमध्ये सुरक्षित संवाद अंमलबजावणी
- **इनपुट प्रमाणीकरण**: सर्व वापरकर्ता इनपुट, API पॅरामीटर्स, आणि डेटा स्रोतांची काटेकोर पडताळणी

#### **पायाभूत सुविधा कडक करणे**
- **मल्टी-फॅक्टर प्रमाणीकरण**: सर्व प्रशासकीय आणि सेवा खात्यांसाठी अनिवार्य MFA
- **पॅच व्यवस्थापन**: ऑपरेटिंग सिस्टम्स, फ्रेमवर्क्स, आणि अवलंबने यासाठी स्वयंचलित, वेळेवर पॅचिंग  
- **ओळख प्रदाता एकत्रीकरण**: एंटरप्राइझ ओळख प्रदाते (Microsoft Entra ID, Active Directory) द्वारे केंद्रीकृत ओळख व्यवस्थापन
- **नेटवर्क विभाजन**: एका-इतराला दूर ठेवण्याकरिता MCP घटकांचे तार्किक वेगळेपणा
- **किमान विशेषाधिकार तत्त्व**: सर्व सिस्टम घटक व खात्यांसाठी आवश्यक किमान परवानग्या

#### **सुरक्षा निरीक्षण आणि शोध**
- **संपूर्ण लॉगिंग**: AI अनुप्रयोग क्रियाकलापांचे सविस्तर लॉगिंग, ज्यात MCP क्लायंट-सर्व्हर संवाद समाविष्ट आहेत
- **SIEM समाकलन**: विचित्र वर्तन शोधण्यासाठी केंद्रीकृत सुरक्षा माहिती व घटना व्यवस्थापन
- **वर्तन विश्लेषण**: सिस्टम व वापरकर्त्याच्या वर्तनातील असामान्य नमुन्यांचे AI-शक्तीने निरीक्षण
- **धोका बुद्धिमत्ता**: बाह्य धोका फीड्स व नुकसान निर्देशक (IOCs) यांचे समाकलन
- **घटना प्रतिसाद**: सुरक्षा घटना ओळखणे, प्रतिसाद देणे, आणि पुनर्प्राप्तीसाठी नीटनेटक्या प्रक्रियांची अंमलबजावणी

#### **झेरो ट्रस्ट आर्किटेक्चर**
- **कधीही विश्वास न ठेवणे, नेहमी पडताळणी करणे**: वापरकर्ता, उपकरणे, आणि नेटवर्क कनेक्शन्सची सतत पडताळणी
- **सूक्ष्म-विभाजन**: व्यक्तिगत कार्यभार आणि सेवांना वेगळे करणारे सूक्ष्म नेटवर्क नियंत्रण
- **ओळख-केंद्रित सुरक्षा**: नेटवर्क स्थानापेक्षा पडताळलेल्या ओळखीवर आधारित सुरक्षा धोरणे
- **सतत धोका मूल्यांकन**: चालू संदर्भ व वर्तनावर आधारित गतिमान सुरक्षा स्थितीचे मूल्यांकन
- **अटींसह प्रवेश**: धोका घटक, स्थान, आणि उपकरण विश्वासावर आधारित प्रवेश नियंत्रण

### एंटरप्राइझ एकत्रीकरण नमुने

#### **Microsoft सुरक्षा परिसंस्था एकत्रीकरण**
- **Microsoft Defender for Cloud**: सर्वसमावेशक क्लाउड सुरक्षा स्थिती व्यवस्थापन
- **Azure Sentinel**: AI कार्यभार सुरक्षा साठी क्लाउड-नेटिव्ह SIEM आणि SOAR क्षमता
- **Microsoft Entra ID**: अटींसह प्रवेश धोरणांसह एंटरप्राइझ ओळख व प्रवेश व्यवस्थापन
- **Azure Key Vault**: हार्डवेअर सुरक्षा मॉड्यूल (HSM) समर्थित केंद्रीकृत गुपित व्यवस्थापन
- **Microsoft Purview**: AI डेटा स्रोत आणि कार्यप्रवाहांसाठी डेटा प्रशासन आणि नियम पालन

#### **अनुपालन व शासन**
- **नियमात्मक साडेसाती**: MCP अंमलबजावण्या उद्योग-विशिष्ट नियमपालन आवश्यकतांकडे जसे की GDPR, HIPAA, SOC 2 याकडे पाळतील याची खात्री करा

- **डेटा वर्गीकरण**: AI प्रणालींनी प्रक्रिया केलेल्या संवेदनशील डेटाचे योग्य वर्गीकरण आणि हाताळणी
- **ऑडिट ट्रेल्स**: नियामक अनुपालन आणि फॉरेन्सिक तपासणीसाठी सर्वसमावेशक लॉगिंग
- **गोपनीयता नियंत्रण**: AI प्रणालीच्या रचनेमध्ये प्रायव्हसी-बाय-डिझाइन तत्त्वांची अंमलबजावणी
- **बदल व्यवस्थापन**: AI प्रणालीतील सुधारणा सुरक्षात्मक पुनरावलोकनांसाठी औपचारिक प्रक्रिया

हे पाया प्रथमतः एक मजबूत सुरक्षा आधार तयार करतात ज्यामुळे MCP-विशिष्ट सुरक्षा नियंत्रणांची कार्यक्षमता वाढते आणि AI-चालित अनुप्रयोगांसाठी सर्वसमावेशक संरक्षण प्रदान होते.

## मुख्य सुरक्षा मुद्दे

- **स्तरीकृत सुरक्षा दृष्टिकोन**: पाया सुरक्षा प्रक्रियांचा (सुरक्षित कोडिंग, किमान विशेषाधिकार, पुरवठादार साखळी पडताळणी, सतत निरीक्षण) AI-विशिष्ट नियंत्रणांसह एकत्रित करा, ज्यामुळे सर्वसमावेशक संरक्षण मिळते

- **AI-विशिष्ट धोके**: MCP प्रणाली विशिष्ट धोके सहन करतात ज्यात प्रॉम्प्ट इंजेक्शन, टूल पॉइझनिंग, सेशन हाईजॅकिंग, गोंधळलेले डेप्युटी समस्या, टोकन पासथ्रू कमकुवतपणा, आणि जास्त परवानग्या असतात ज्यासाठी विशेष प्रतिबंध आवश्यक आहेत

- **प्रमाणीकरण आणि अधिकृतता उत्कृष्टता**: बाह्य ओळख पुरवठादार (Microsoft Entra ID) वापरून मजबूत प्रमाणीकरण करा, योग्य टोकन मान्यता लागू करा, आणि तुमच्या MCP सर्वरला स्पष्टपणे दिलेले नाही असे टोकन कधीही स्वीकारू नका

- **AI हल्ला प्रतिबंध**: Microsoft Prompt Shields आणि Azure Content Safety चा वापर करून अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन आणि टूल पॉइझनिंग हल्ल्यांपासून संरक्षण करा, टूल मेटाडेटा तपासा आणि गतिशील बदलांसाठी लक्ष ठेवा

- **सेशन आणि वाहतूक सुरक्षा**: क्रिप्टोग्राफिकली सुरक्षित, गैर-निर्णायक सेशन आयडी वापरा जे वापरकर्ता ओळखांशी बांधलेले आहेत, योग्य सेशन जीवनचक्र व्यवस्थापन लागू करा, आणि प्रमाणीकरणासाठी कधीही सेशन्स वापरू नका

- **OAuth सुरक्षा सर्वोत्तम प्रथाः** गतिशील नोंदणी केलेल्या क्लायंटसाठी स्पष्ट वापरकर्ता संमतीद्वारे गोंधळलेल्या डेप्युटी हल्ल्यापासून प्रतिबंध करा, PKCE सह योग्य OAuth 2.1 अंमलबजावणी करा, आणि कडक पुनर्निर्देशित करा URI ची पडताळणी करा  

- **टोकन सुरक्षा तत्त्वे**: टोकन पासथ्रू विरोधी नमुन्यांपासून बचाव करा, टोकन प्रेक्षक दावे मान्य करा, सुरक्षित फेरफटका सहित अल्पकालीन टोकनांची अंमलबजावणी करा, आणि स्पष्ट विश्वास सीमारेषा राखा

- **सर्वसमावेशक पुरवठा साखळी सुरक्षा**: सर्व AI पारिस्थितिकी तंत्र घटक (मॉडेल्स, एम्बेडिंग्ज, संदर्भ पुरवठादार, बाह्य API) पारंपारिक सॉफ्टवेअर अवलंबनांसारख्या सुरक्षा काटेकोरतेने हाताळा

- **सतत विकास**: जलद विकसित होत असलेल्या MCP तपशीलांसोबत अद्ययावत रहा, सुरक्षा समुदाय मानकांमध्ये योगदान द्या, आणि प्रोटोकॉल प्रगल्भ झाल्यामुळे अनुकूल सुरक्षा स्थिती ठेवा

- **Microsoft सुरक्षा समाकलन**: Microsoft च्या सर्वसमावेशक सुरक्षा परिसंस्थेचा (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) वापर करून MCP तैनातीचे संरक्षण वाढवा
