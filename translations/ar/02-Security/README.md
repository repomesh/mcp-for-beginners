# أمان MCP: الحماية الشاملة لأنظمة الذكاء الاصطناعي

[![أفضل ممارسات أمان MCP](../../../translated_images/ar/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(انقر على الصورة أعلاه لمشاهدة فيديو الدرس)_

الأمان هو أساس تصميم أنظمة الذكاء الاصطناعي، ولهذا نُوليه أولوية باعتباره القسم الثاني لدينا. يتماشى هذا مع مبدأ **الأمان من خلال التصميم** لشركة مايكروسوفت من [مبادرة المستقبل الآمن](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/).

يقدّم بروتوكول سياق النموذج (MCP) قدرات جديدة وقوية لتطبيقات الذكاء الاصطناعي، مع تقديم تحديات أمنية فريدة تتجاوز مخاطر البرمجيات التقليدية. تواجه أنظمة MCP مخاوف أمنية معروفة (كود آمن، أقل امتياز، أمان سلسلة التوريد) بالإضافة إلى تهديدات خاصة بالذكاء الاصطناعي تشمل حقن الطلبات، تسمم الأدوات، اختطاف الجلسات، هجمات المفوض المضلل، ثغرات تمرير الرموز، وتعديل القدرات الديناميكي.

يستعرض هذا الدرس المخاطر الأمنية الأكثر أهمية في تطبيقات MCP — ويغطي المصادقة، التفويض، الأذونات المفرطة، حقن الطلبات غير المباشر، أمان الجلسة، مشاكل المفوض المضلل، إدارة الرموز، وثغرات سلسلة التوريد. ستتعلّم الضوابط العملية وأفضل الممارسات لتخفيف هذه المخاطر مع الاستفادة من حلول مايكروسوفت مثل دروع الطلبات، أمان محتوى أزور، وأمان GitHub المتقدم لتعزيز نشر MCP الخاص بك.

## أهداف التعلم

بنهاية هذا الدرس، ستكون قادرًا على:

- **تحديد التهديدات الخاصة بـ MCP**: التعرف على المخاطر الأمنية الفريدة في أنظمة MCP بما في ذلك حقن الطلبات، تسمم الأدوات، الأذونات المفرطة، اختطاف الجلسات، مشاكل المفوض المضلل، ثغرات تمرير الرموز، ومخاطر سلسلة التوريد
- **تطبيق الضوابط الأمنية**: تنفيذ تدابير التخفيف الفعّالة بما يشمل المصادقة القوية، وصول الأقل امتياز، إدارة الرموز الآمنة، ضوابط أمان الجلسة، والتحقق من سلسلة التوريد
- **الاستفادة من حلول أمان مايكروسوفت**: فهم ونشر دروع الطلبات من مايكروسوفت، أمان محتوى أزور، وأمان GitHub المتقدم لحماية أعمال MCP
- **التحقق من أمان الأدوات**: التعرف على أهمية التحقق من بيانات الأدوات الوصفية، مراقبة التغييرات الديناميكية، والدفاع ضد هجمات حقن الطلبات غير المباشر
- **دمج أفضل الممارسات**: الجمع بين أساسيات الأمان المعتمدة (البرمجة الآمنة، تقوية الخادم، الثقة الصفرية) وضوابط MCP الخاصة للحماية الشاملة

# هندسة وضوابط أمان MCP

تتطلب تطبيقات MCP الحديثة نهج أمني متعدد الطبقات يعالج كلاً من أمان البرمجيات التقليدية والتهديدات الخاصة بالذكاء الاصطناعي. يواصل مواصفة MCP المتطورة بسرعة تحسين ضوابط الأمان، مما يمكّن من تكامل أفضل مع معماريات الأمان المؤسسية وأفضل الممارسات المعتمدة.

توضح أبحاث من [تقرير الدفاع الرقمي لمايكروسوفت](https://aka.ms/mddr) أن **98% من الخروقات المُبلغ عنها يمكن تفاديها بممارسات أمان قوية**. تجمع استراتيجية الحماية الأكثر فعالية بين ممارسات الأمان الأساسية وضوابط MCP الخاصة — حيث تظل التدابير الأمنية الأساسية المثبتة الأكثر تأثيرًا في تقليل المخاطر الأمنية الإجمالية.

## المشهد الأمني الحالي

> **ملاحظة:** تعكس هذه المعلومات معايير أمان MCP اعتبارًا من **5 فبراير 2026**، متوافقة مع **مواصفة MCP 2025-11-25**. يستمر بروتوكول MCP في التطور السريع، وقد تقدم التطبيقات المستقبلية أنماط مصادقة جديدة وضوابط محسنة. يُرجى دائمًا الرجوع إلى [مواصفة MCP الحالية](https://spec.modelcontextprotocol.io/)، [مستودع MCP على GitHub](https://github.com/modelcontextprotocol)، و[وثائق أفضل الممارسات الأمنية](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) لأحدث الإرشادات.

> **نظرة مستقبلية:** تعزز نسخة المرشح للإصدار `2026-07-28` التفويض بشكل أكبر — يجب على العملاء التحقق من معلمة `iss` في ردود التفويض (RFC 9207)، إعلان نوع التطبيق OpenID Connect `application_type` أثناء تسجيل العميل الديناميكي، وربط بيانات الاعتماد المسجلة بسيرفر التفويض المصدر. راجع [ما هو الجديد في MCP: المرشح للإصدار 2026-07-28](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) للحصول على القائمة الكاملة لأساليب التفويض.

### تطور مصادقة MCP

تطورت مواصفة MCP بشكل كبير في نهج المصادقة والتفويض:

- **النهج الأصلي**: تطلبت المواصفات المبكرة من المطورين تنفيذ خوادم مصادقة مخصصة، حيث تعمل خوادم MCP كمزودي تفويض OAuth 2.0 يديرون مصادقة المستخدمين مباشرة
- **المعيار الحالي (2025-11-25)**: تسمح المواصفة المحدثة بتفويض المصادقة إلى موفري الهوية الخارجيين (مثل Microsoft Entra ID)، مما يحسن وضع الأمان ويقلل تعقيد التنفيذ
- **أمان طبقة النقل**: دعم محسّن لآليات النقل الآمن مع أنماط مصادقة مناسبة للاتصالات المحلية (STDIO) وعن بعد (Streamable HTTP)

## أمان المصادقة والتفويض

### التحديات الأمنية الحالية

تواجه تطبيقات MCP الحديثة تحديات متعددة في المصادقة والتفويض:

### المخاطر ومسارات التهديد

- **منطق تفويض خاطئ التكوين**: تنفيذ التفويض المعيب في خوادم MCP قد يكشف بيانات حساسة ويطبق ضوابط وصول بشكل خاطئ
- **اختراق رمز OAuth**: سرقة رمز خادم MCP المحلي تُمكّن المهاجمين من انتحال هوية الخوادم والوصول إلى الخدمات الطرفية
- **ثغرات تمرير الرموز**: التعامل غير السليم مع الرموز يخلق تجاوزات في ضوابط الأمان وفجوات في المساءلة
- **الأذونات المفرطة**: الخوادم ذات الامتيازات الزائدة تنتهك مبدأ أقل الامتياز وتوسع سطح الهجوم

#### تمرير الرموز: نمط معاكس حرج

**تمرير الرموز محظور صراحة** في مواصفة تفويض MCP الحالية بسبب تداعيات أمنية جسيمة:

##### التحايل على ضوابط الأمان
- تنفذ خوادم MCP وواجهات برمجية تابعة ضوابط أمنية حرجة (تحديد المعدل، تحقق الطلبات، مراقبة المرور) تعتمد على التحقق السليم من الرموز
- استخدام العميل المباشر لرموز API يتجاوز هذه الحمايات الأساسية، مما يضعف بنية الأمان

##### تحديات المساءلة والمراجعة  
- لا يستطيع خادم MCP التمييز بين العملاء الذين يستخدمون رموز صادرة من الأعلى، ما يكسر سجلات المراجعة
- تُظهر سجلات خادم الموارد الطرفي مصادر طلبات مضللة بدل الوسطاء الفعليين لخادم MCP
- يصبح التحقيق في الحوادث والتدقيق متعذرًا بشكل كبير

##### مخاطر تسريب البيانات
- تُمكن مطالبات الرموز غير المُحققة الجهات الخبيثة حاملي الرموز المسروقة من استخدام خوادم MCP كوسطاء لتسريب البيانات
- تنتهك انتهاكات حدود الثقة أنماط الوصول غير المصرح بها التي تتجاوز ضوابط الأمان المقصودة

##### مسارات هجوم متعددة الخدمات
- تتيح الرموز المخترقة المقبولة لدى عدة خدمات حركة أفقية عبر الأنظمة الموصولة
- قد تنتهك افتراضات الثقة بين الخدمات عندما لا يُمكن التحقق من مصادر الرموز

### ضوابط الأمان وتدابير التخفيف

**متطلبات الأمان الحرجة:**

> **إلزامي**: يجب على خوادم MCP **عدم قبول** أي رموز لم تُصدر صراحة لخادم MCP نفسه

#### ضوابط المصادقة والتفويض

- **مراجعة تفويض دقيقة**: إجراء تدقيق شامل لمنطق تفويض خوادم MCP لضمان وصول المستخدمين والعملاء المقصودين فقط إلى الموارد الحساسة
  - **دليل التنفيذ**: [إدارة API أزور كبوابة مصادقة لخوادم MCP](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **تكامل الهوية**: [استخدام Microsoft Entra ID لمصادقة خادم MCP](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **إدارة الرموز الآمنة**: تنفيذ [أفضل ممارسات مايكروسوفت للتحقق ودورة حياة الرموز](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens)
  - تحقق من أن مطالبات جمهور الرموز تطابق هوية خادم MCP
  - تنفيذ سياسات تدوير الرموز وانتهائها بشكل صحيح
  - منع هجمات إعادة تشغيل الرموز والاستخدام غير المصرح به

- **تخزين الرموز المحمي**: تأمين تخزين الرموز بالتشفير عند السكون والنقل
  - **أفضل الممارسات**: [إرشادات تخزين وتشفير الرموز الآمنة](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### تنفيذ ضوابط الوصول

- **مبدأ أقل امتياز**: منح خوادم MCP الحد الأدنى من الأذونات المطلوبة للوظائف المقصودة فقط
  - مراجعات وتحديثات دورية للأذونات لمنع توسع الامتيازات
  - **وثائق مايكروسوفت**: [الوصول الآمن بأقل امتياز](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **التحكم بالوصول القائم على الأدوار (RBAC)**: تنفيذ تعيينات أدوار دقيقة
  - تضييق نطاق الأدوار للموارد والإجراءات المحددة
  - تجنب الأذونات العامة أو غير الضرورية التي توسع سطح الهجوم

- **مراقبة مستمرة للأذونات**: تنفيذ مراجعة ورصد دائمين للوصول
  - مراقبة أنماط استخدام الأذونات لاكتشاف الشذوذات
  - تصحيح سريع للأذونات المفرطة أو غير المستخدمة

## تهديدات أمنية خاصة بالذكاء الاصطناعي

### هجمات حقن الطلبات وتلاعب الأدوات

تواجه تطبيقات MCP الحديثة متجهات هجوم متطورة خاصة بالذكاء الاصطناعي لا يمكن لمعظم تدابير الأمان التقليدية التعامل معها بالكامل:

#### **حقن الطلبات غير المباشر (حقن الطلبات العابر للمجالات)**

يمثل **حقن الطلبات غير المباشر** أحد أخطر الثغرات في أنظمة الذكاء الاصطناعي المدعومة بـ MCP. يقوم المهاجمون بإدخال تعليمات خبيثة ضمن محتوى خارجي—كالوثائق، صفحات الويب، الرسائل الإلكترونية، أو مصادر البيانات—التي تعالجها أنظمة الذكاء الاصطناعي لاحقًا كأوامر شرعية.

**سيناريوهات الهجوم:**
- **حقن مستندات**: تعليمات خبيثة مخفية ضمن وثائق معالجة تحفز أفعال ذكاء اصطناعي غير مقصودة
- **استغلال محتوى الويب**: صفحات ويب مخترقة تحتوي على طلبات مدمجة تحرف سلوك الذكاء الاصطناعي عند جمع البيانات منها
- **هجمات عبر البريد الإلكتروني**: طلبات خبيثة في رسائل البريد الإلكتروني تسبب تسريب معلومات أو تنفيذ أفعال غير مصرح بها من المساعدين الذكيين
- **تلوث مصادر البيانات**: قواعد بيانات أو APIs مخترقة تقدم محتوى ملوث لأنظمة الذكاء الاصطناعي

**الأثر في العالم الحقيقي**: يمكن أن تؤدي هذه الهجمات إلى تسريب البيانات، انتهاك الخصوصية، إنتاج محتوى ضار، والتلاعب بتفاعلات المستخدم. للمزيد من التحليل، انظر [حقن الطلبات في MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).

![مخطط هجوم حقن الطلبات](../../../translated_images/ar/prompt-injection.ed9fbfde297ca877.webp)

#### **هجمات تسمم الأدوات**

تستهدف **تسمم الأدوات** البيانات الوصفية التي تحدد أدوات MCP، مستغلة كيفية تفسير نماذج اللغة الكبيرة لوصف الأدوات ومعاييرها لاتخاذ قرارات التنفيذ.

**آليات الهجوم:**
- **تلاعب البيانات الوصفية**: يقوم المهاجمون بحقن تعليمات خبيثة في وصف الأدوات، تعريفات المعايير، أو أمثلة الاستخدام
- **تعليمات غير مرئية**: طلبات مخفية ضمن البيانات الوصفية للأدوات يعالجها نموذج الذكاء الاصطناعي لكنها غير مرئية للمستخدمين البشر
- **تعديل ديناميكي للأدوات ("سحب السجادة")**: أدوات معتمدة من المستخدمين يتم تعديلها لاحقًا لأداء أفعال خبيثة دون علم المستخدم
- **حقن المعايير**: محتوى خبيث مدمج في مخططات معلمات الأدوات يؤثر على سلوك النموذج


**مخاطر الخادم المستضاف**: تقدم خوادم MCP البعيدة مخاطر مرتفعة حيث يمكن تحديث تعريفات الأدوات بعد موافقة المستخدم الأولية، مما يخلق سيناريوهات تصبح فيها الأدوات التي كانت آمنة سابقًا خبيثة. لمزيد من التحليل الشامل، راجع [هجمات تسميم الأدوات (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks).

![مخطط هجوم حقن الأدوات](../../../translated_images/ar/tool-injection.3b0b4a6b24de6bef.webp)

#### **متجهات هجوم إضافية للذكاء الاصطناعي**

- **حقن مطالبات عبر النطاقات (XPIA)**: هجمات متطورة تستغل محتوى من عدة نطاقات لتجاوز ضوابط الأمان
- **تعديل القدرات الديناميكي**: تغييرات في الوقت الحقيقي لقدرات الأدوات التي تهرب من التقييمات الأمنية الأولية
- **تسميم نافذة السياق**: هجمات تتحكم في نوافذ سياق كبيرة لإخفاء تعليمات خبيثة
- **هجمات لبس النموذج**: استغلال حدود النموذج لخلق سلوكيات غير متوقعة أو غير آمنة


### تأثير مخاطر أمان الذكاء الاصطناعي

**العواقب ذات التأثير العالي:**
- **سحب البيانات**: الوصول غير المصرح به وسرقة البيانات الحساسة للمؤسسات أو البيانات الشخصية
- **اختراقات الخصوصية**: الكشف عن المعلومات الشخصية التعريفية (PII) والبيانات التجارية السرية  
- **تلاعب النظام**: تعديلات غير مقصودة على الأنظمة الحرجة وسير العمل
- **سرقة بيانات الاعتماد**: اختراق رموز المصادقة وبيانات اعتماد الخدمات
- **الحركة الجانبية**: استخدام أنظمة الذكاء الاصطناعي المخترقة كنقاط انطلاق لهجمات شبكية أوسع

### حلول أمان الذكاء الاصطناعي من مايكروسوفت

#### **الدروع الذكية للطلبات: حماية متقدمة ضد هجمات الحقن**

توفر دروع الطلبات الذكية من مايكروسوفت دفاعًا شاملاً ضد هجمات حقن الطلبات المباشرة وغير المباشرة من خلال طبقات أمان متعددة:

##### **آليات الحماية الأساسية:**

1. **الكشف المتقدم والتصفية**
   - خوارزميات التعلم الآلي وتقنيات معالجة اللغة الطبيعية تكشف التعليمات الخبيثة في المحتوى الخارجي
   - تحليل في الوقت الحقيقي للوثائق، صفحات الويب، الرسائل الإلكترونية، ومصادر البيانات للكشف عن التهديدات المضمنة
   - فهم سياقي لأنماط الطلبات الشرعية مقابل الخبيثة

2. **تقنيات تسليط الضوء**  
   - تميز بين تعليمات النظام الموثوقة والمدخلات الخارجية المحتمل اختراقها
   - طرق تحويل النصوص التي تعزز ملاءمة النموذج مع عزل المحتوى الخبيث
   - تساعد أنظمة الذكاء الاصطناعي على الحفاظ على تسلسل التعليمات الصحيح وتجاهل الأوامر المحقونة

3. **أنظمة حواجز الترميز والعلامات البيانية**
   - تعريف صريح للحدود بين رسائل النظام الموثوقة ونص المدخلات الخارجية
   - علامات خاصة تبرز الحدود بين مصادر البيانات الموثوقة وغير الموثوقة
   - فصل واضح يمنع خلط التعليمات وتنفيذ الأوامر غير المصرح بها

4. **معلومات التهديد المستمرة**
   - تقوم مايكروسوفت بمراقبة أنماط الهجوم الناشئة وتحديث الدفاعات بشكل مستمر
   - صيد تهديدات استباقي لتقنيات الحقن الجديدة ومتجهات الهجوم
   - تحديثات منتظمة لنماذج الأمان للحفاظ على الفعالية ضد التهديدات المتطورة

5. **تكامل أمان محتوى أزور**
   - جزء من مجموعة أمان محتوى أزور الذكية الشاملة
   - كشف إضافي لمحاولات كسر القيود والمحتوى الضار وانتهاكات سياسات الأمان
   - ضوابط أمان موحدة عبر مكونات تطبيقات الذكاء الاصطناعي

## تهديدات أمان MCP المتقدمة

### ثغرات اختطاف الجلسة

**اختطاف الجلسة** يمثل متجه هجوم حرج في تطبيقات MCP ذات الحالة حيث يحصل أطراف غير مخولين على معرّفات الجلسة الشرعية ويستغلونها لانتحال هوية العملاء وأداء إجراءات غير مصرح بها.

#### **سيناريوهات الهجوم والمخاطر**

- **حقن مطالبات اختطاف الجلسة**: المهاجمون الذين يحملون معرّفات جلسة مسروقة يحقنون أحداثًا خبيثة في خوادم تشارك حالة الجلسة، مما قد يؤدي إلى تنفيذ أفعال ضارة أو الوصول إلى بيانات حساسة
- **الانتحال المباشر**: معرّفات الجلسة المسروقة تتيح استدعاءات مباشرة لخادم MCP تتجاوز المصادقة، معاملة المهاجمين كمستخدمين شرعيين
- **تدفقات استئناف مخترقة**: يمكن للمهاجمين إنهاء الطلبات مبكرًا، مما يجعل العملاء الشرعيين يستأنفون محتوى قد يكون خبيثًا

#### **ضوابط الأمان لإدارة الجلسة**

**المتطلبات الحرجة:**
- **التحقق من التفويض**: يجب على خوادم MCP التي تنفذ التفويض **التحقق من جميع الطلبات الواردة** ويجب ألا تعتمد على الجلسات للمصادقة
- **توليد جلسات آمنة**: استخدام معرفات جلسة مشفرة وآمنة وغير حتمية يتم إنشاؤها باستخدام مولدات أرقام عشوائية آمنة
- **ربط مخصص للمستخدم**: ربط معرفات الجلسة بمعلومات المستخدم باستخدام صيغ مثل `<user_id>:<session_id>` لمنع سوء استخدام الجلسات بين المستخدمين
- **إدارة دورة حياة الجلسة**: تنفيذ انتهاء صلاحية مناسب وتدوير وإبطال لتقليل نوافذ الضعف
- **أمان النقل**: استخدام HTTPS إلزامي لجميع الاتصالات لمنع اعتراض معرف الجلسة

### مشكلة الوكيل المرتبك

تحدث **مشكلة الوكيل المرتبك** عندما تعمل خوادم MCP كوسطاء مصادقة بين العملاء وخدمات الطرف الثالث، مما يخلق فرصًا لتجاوز التفويض من خلال استغلال معرف العميل الثابت.

#### **آليات الهجوم والمخاطر**

- **تجاوز الموافقة عبر الكوكيز**: مصادقة المستخدم السابقة تخلق كوكيز الموافقة التي يستغلها المهاجمون عبر طلبات تفويض ضارة مع عناوين URI لإعادة التوجيه مصممة
- **سرقة رمز التفويض**: كوكيز الموافقة الحالية قد تجعل خوادم التفويض تتخطى شاشات الموافقة وتوجّه الرموز إلى نقاط نهاية يسيطر عليها المهاجم  
- **الوصول غير المصرح به إلى API**: رموز التفويض المسروقة تمكن تبادل الرموز وانتحال هوية المستخدم دون موافقة صريحة

#### **استراتيجيات التخفيف**

**ضوابط إلزامية:**
- **متطلبات موافقة صريحة**: يجب على خوادم MCP الوكيلة التي تستخدم معرفات عميل ثابتة **الحصول على موافقة المستخدم** لكل عميل مسجل ديناميكيًا
- **تنفيذ أمان OAuth 2.1**: اتبع ممارسات الأمان الحالية لـ OAuth بما في ذلك PKCE (إثبات المفتاح لتبادل الرموز) لجميع طلبات التفويض
- **التحقق الصارم من العميل**: تنفيذ تحقق صارم لعناوين URI لإعادة التوجيه ومعرفات العميل لمنع الاستغلال

### ثغرات تمرير الرموز  

يمثل **تمرير الرموز** نمطًا مضادًا صريحًا حيث تقبل خوادم MCP رموز العميل دون تحقق مناسب وتعيد توجيهها إلى واجهات برمجة التطبيقات اللاحقة، مما ينتهك مواصفات تفويض MCP.

#### **الآثار الأمنية**

- **تجاوز الضوابط**: استخدام الرموز المباشر من العميل إلى API يتجاوز ضوابط الحد من المعدل، التحقق، والمراقبة الحيوية
- **فساد سجل التدقيق**: الرموز الصادرة من المصدر العلوي تجعل تحديد العميل مستحيلاً، مما يكسر قدرات التحقيق في الحوادث
- **سحب البيانات عبر الوكيل**: الرموز غير الموثوقة تمكن الجهات الخبيثة من استخدام الخوادم كوكلاء للوصول غير المصرح به للبيانات
- **انتهاكات حدود الثقة**: قد تُنتهك افتراضات الثقة في الخدمات اللاحقة حين لا يمكن التحقق من أصل الرموز
- **توسع الهجوم متعدد الخدمات**: الرموز المخترقة المقبولة عبر خدمات متعددة تمكّن الحركة الجانبية

#### **ضوابط أمان مطلوبة**

**متطلبات غير قابلة للتفاوض:**
- **التحقق من الرموز**: يجب ألا تقبل خوادم MCP الرموز التي لم تصدر صراحة لخادم MCP
- **التحقق من الجمهور**: تحقق دائمًا من مطابقة مدعيات الجمهور في الرمز مع هوية خادم MCP
- **دورة حياة مناسبة للرموز**: تنفيذ رموز وصول قصيرة العمر مع ممارسات تدوير آمنة


## أمان سلسلة التوريد لأنظمة الذكاء الاصطناعي

تطور أمان سلسلة التوريد ليشمل أكثر من مجرد تبعيات البرامج التقليدية ليتضمن النظام البيئي الكامل للذكاء الاصطناعي. يجب على تطبيقات MCP الحديثة التحقق بدقة ومراقبة كافة مكونات الذكاء الاصطناعي، إذ أن كل مكون قد يضيف ثغرات تهدد سلامة النظام.

### مكونات موسعة لسلسلة توريد الذكاء الاصطناعي

**تبعات البرامج التقليدية:**
- مكتبات وأُطر عمل مفتوحة المصدر
- صور الحاويات وأنظمة الأساس  
- أدوات التطوير وخطوط بناء البرمجيات
- مكونات البنية التحتية والخدمات

**عناصر سلسلة توريد خاصة بالذكاء الاصطناعي:**
- **نماذج الأساس**: نماذج مدربة مسبقًا من مزودين متنوعين تتطلب تحقق مصدرها
- **خدمات التضمين**: خدمات تحويل البيانات إلى متجهات وبحث دلالي خارجية
- **موفرو السياق**: مصادر البيانات، قواعد المعرفة، ومستودعات الوثائق  
- **واجهات برمجة التطبيقات الخارجية**: خدمات الذكاء الاصطناعي الخارجية، خطوط أنابيب التعلم الآلي، ونقاط نهاية معالجة البيانات
- **مخرجات النماذج**: الأوزان، التهيئات، والمتغيرات الدقيقة للنماذج
- **مصادر بيانات التدريب**: مجموعات البيانات المستخدمة لتدريب وتعديل النماذج

### استراتيجية أمان شاملة لسلسلة التوريد

#### **التحقق من المكونات والثقة**
- **التحقق من المصدر**: تحقق من الأصل، الترخيص، وسلامة جميع مكونات الذكاء الاصطناعي قبل التكامل
- **تقييم الأمان**: إجراء فحوصات للثغرات ومراجعات أمنية للنماذج، مصادر البيانات، وخدمات الذكاء الاصطناعي
- **تحليل السمعة**: تقييم سجل الممارسات الأمنية لمزودي خدمات الذكاء الاصطناعي
- **التحقق من الامتثال**: التأكد من التزام جميع المكونات بمتطلبات الأمان التنظيمية والمؤسسية

#### **خطوط نشر آمنة**  
- **أمان CI/CD الآلي**: دمج فحوصات الأمان في خطوط نشر التشغيل الآلي
- **سلامة التحف البرمجية**: تنفيذ تحقق تشفيري لجميع التحف المنشورة (الكود، النماذج، التهيئات)
- **نشر تدريجي**: استخدام استراتيجيات نشر متدرجة مع تحقق أمني في كل مرحلة
- **مستودعات التحف الموثوقة**: النشر فقط من سجلات ومستودعات تحقق أمانها

#### **المراقبة المستمرة والاستجابة**
- **فحص التبعيات**: مراقبة مستمرة للثغرات لكافة تبعيات البرمجيات ومكونات الذكاء الاصطناعي
- **مراقبة النماذج**: تقييم مستمر لسلوك النموذج، انحراف الأداء، وشذوذات الأمان
- **تعقب صحة الخدمات**: مراقبة توفر خدمات الذكاء الاصطناعي الخارجية، الحوادث الأمنية، وتغييرات السياسات
- **تكامل معلومات التهديد**: دمج تغذيات تهديد مخصصة لمخاطر أمان الذكاء الاصطناعي والتعلم الآلي

#### **ضوابط الوصول وأقل الامتيازات**
- **أذونات على مستوى المكونات**: تقييد الوصول للنماذج، البيانات، والخدمات بناءً على الحاجة العملية
- **إدارة حسابات الخدمة**: تنفيذ حسابات خدمة مخصصة بأدنى الأذونات المطلوبة
- **تقسيم الشبكة**: عزل مكونات الذكاء الاصطناعي وتقليل الوصول الشبكي بين الخدمات
- **ضوابط بوابة API**: استخدام بوابات API مركزية للتحكم والمراقبة على الوصول إلى خدمات الذكاء الاصطناعي الخارجية

#### **الاستجابة للحوادث والتعافي**
- **إجراءات استجابة سريعة**: عمليات مثبتة لتصحيح أو استبدال مكونات الذكاء الاصطناعي المخترقة
- **تدوير بيانات الاعتماد**: أنظمة آلية لتدوير الأسرار، مفاتيح API، وبيانات اعتماد الخدمات
- **قدرات التراجع**: القدرة على العودة بسرعة إلى نسخ سابقة معروفة جيدة من مكونات الذكاء الاصطناعي
- **التعافي من اختراق سلسلة التوريد**: إجراءات محددة للاستجابة لاختراقات خدمات الذكاء الاصطناعي الموردة من المصدر العلوي

### أدوات التكامل الأمني من مايكروسوفت

توفر **GitHub Advanced Security** حماية شاملة لسلسلة التوريد تشمل:
- **فحص الأسرار**: الكشف الآلي عن بيانات الاعتماد، مفاتيح API، والرموز في المستودعات
- **فحص التبعيات**: تقييم الثغرات للتبعيات المفتوحة المصدر والمكتبات
- **تحليل CodeQL**: تحليل ثابت للكود للكشف عن الثغرات الأمنية ومشكلات البرمجة
- **رؤى سلسلة التوريد**: رؤية شاملة لحالة صحة التبعيات والأمان

**تكامل Azure DevOps و Azure Repos:**
- تكامل فحص الأمان بسلاسة عبر منصات تطوير مايكروسوفت
- فحوصات أمان مؤتمتة في خطوط أنابيب Azure لأحمال عمل الذكاء الاصطناعي
- تطبيق سياسات لنشر مكونات الذكاء الاصطناعي الآمنة

**ممارسات مايكروسوفت الداخلية:**
تنفذ مايكروسوفت ممارسات أمان شاملة لسلسلة التوريد عبر كل المنتجات. اطلع على الطرق المثبتة في [رحلة تأمين سلسلة توريد البرامج في مايكروسوفت](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/).


## أفضل ممارسات أمان الأساس

ترث تطبيقات MCP وتعزز وضع الأمان القائم في مؤسستك. تعزيز ممارسات الأمان الأساسية يحسّن بشكل كبير الأمان الكلي لأنظمة الذكاء الاصطناعي وتطبيقات MCP.

### أسس الأمان الأساسية

#### **ممارسات التطوير الآمن**
- **الامتثال لـ OWASP**: الحماية ضد [أفضل 10 ثغرات تطبيقات الويب OWASP](https://owasp.org/www-project-top-ten/)
- **حمايات خاصة بالذكاء الاصطناعي**: تنفيذ ضوابط لـ [أفضل 10 ثغرات LLMs من OWASP](https://genai.owasp.org/download/43299/?tmstv=1731900559)
- **إدارة الأسرار الآمنة**: استخدام خزائن مخصصة للرموز، مفاتيح API، وبيانات التهيئة الحساسة
- **التشفير الشامل**: تنفيذ اتصالات آمنة عبر كافة مكونات التطبيق وتدفقات البيانات
- **التحقق من المدخلات**: تحقق صارم لكل مدخلات المستخدم، معلمات API، ومصادر البيانات

#### **تقوية البنية التحتية**
- **المصادقة متعددة العوامل**: إلزام MFA لكل حسابات الإدارة والخدمات
- **إدارة التصحيحات**: تصحيح تلقائي وفي الوقت المناسب لأنظمة التشغيل، الأُطر، والتبعيات  
- **تكامل مزود الهوية**: إدارة مركزية للهوية عبر موفري الهوية المؤسسيين (Microsoft Entra ID, Active Directory)
- **تقسيم الشبكات**: عزل منطقي لمكونات MCP لتقليل إمكانيات الحركة الجانبية
- **مبدأ الأقل امتيازًا**: أقل الأذونات المطلوبة لكل مكونات النظام والحسابات

#### **مراقبة الأمان والكشف**
- **تسجيل شامل**: تسجيل مفصل لأنشطة تطبيق الذكاء الاصطناعي بما في ذلك تفاعلات العملاء مع خوادم MCP
- **تكامل SIEM**: نظام مركزي لإدارة المعلومات الأمنية والأحداث للكشف عن الشذوذات
- **تحليلات السلوك**: مراقبة مدعومة بالذكاء الاصطناعي للكشف عن أنماط غير معتادة في سلوك النظام والمستخدم
- **مخابرات التهديد**: دمج تغذيات التهديد الخارجية ومؤشرات الاختراق (IOCs)
- **استجابة للحوادث**: إجراءات معرفّة جيدًا للكشف عن الحوادث الأمنية، الاستجابة، والتعافي

#### **هيكلية الثقة الصفرية**
- **لا تثق أبدًا، تحقق دائمًا**: تحقق مستمر للمستخدمين، الأجهزة، واتصالات الشبكة
- **التقسيم الدقيق**: ضوابط شبكية دقيقة تعزل الأعمال والخدمات الفردية
- **أمان مركز هوية**: سياسات أمان مبنية على الهويات الموثقة بدلًا من موقع الشبكة
- **تقييم المخاطر المستمر**: تقييم ديناميكي للوضع الأمني بناءً على السياق والسلوك الحالي
- **الوصول الشرطي**: ضوابط وصول تتكيف حسب عوامل المخاطر والموقع وثقة الجهاز

### أنماط التكامل المؤسسي

#### **تكامل منظومة أمان مايكروسوفت**
- **Microsoft Defender for Cloud**: إدارة شاملة لوضع أمان السحابة
- **Azure Sentinel**: قدرات SIEM وSOAR سحابية لحماية أحمال عمل الذكاء الاصطناعي
- **Microsoft Entra ID**: إدارة الهوية والوصول المؤسسي مع سياسات الوصول الشرطي
- **Azure Key Vault**: إدارة الأسرار مركزية بدعم من وحدات الأمان الصلبة (HSM)
- **Microsoft Purview**: حوكمة البيانات والامتثال لمصادر بيانات الذكاء الاصطناعي وسير العمل

#### **الامتثال والحوكمة**
- **الامتثال التنظيمي**: تأكد من أن تطبيقات MCP تلبي متطلبات الامتثال الخاصة بالصناعة (GDPR، HIPAA، SOC 2)

- **تصنيف البيانات**: التصنيف المناسب والتعامل مع البيانات الحساسة التي تعالجها أنظمة الذكاء الاصطناعي  
- **سجلات التدقيق**: تسجيل شامل للامتثال التنظيمي والتحقيقات الجنائية  
- **ضوابط الخصوصية**: تنفيذ مبادئ الخصوصية حسب التصميم في هيكلية أنظمة الذكاء الاصطناعي  
- **إدارة التغيير**: عمليات رسمية لمراجعات الأمان لتعديلات نظام الذكاء الاصطناعي  

تخلق هذه الممارسات الأساسية قاعدة أمان قوية تعزز فعالية ضوابط الأمان الخاصة بـ MCP وتوفر حماية شاملة للتطبيقات المدفوعة بالذكاء الاصطناعي.  

## النقاط الرئيسية للأمان  

- **نهج الأمان متعدد الطبقات**: الجمع بين ممارسات الأمان الأساسية (البرمجة الآمنة، مبدأ الأقل امتيازاً، التحقق من سلسلة التوريد، المراقبة المستمرة) مع ضوابط خاصة بالذكاء الاصطناعي لتوفير حماية شاملة  

- **مشهد التهديدات الخاصة بالذكاء الاصطناعي**: تواجه أنظمة MCP مخاطر فريدة تشمل حقن الموجهات، تسميم الأدوات، اختطاف الجلسات، مشاكل الممثل المضلل، ثغرات تمرير الرموز، والصلاحيات المفرطة التي تتطلب تخفيف متخصص  

- **تفوق المصادقة والتفويض**: تنفيذ مصادقة قوية باستخدام موفري الهوية الخارجيين (Microsoft Entra ID)، فرض التحقق الصحيح من الرموز، وعدم قبول الرموز غير المخصصة صراحةً لخادم MCP الخاص بك  

- **منع هجمات الذكاء الاصطناعي**: نشر دروع Microsoft Prompt و Azure Content Safety للدفاع ضد حقن الموجهات غير المباشر وهجمات تسميم الأدوات، مع التحقق من بيانات أدوات التعريف والمراقبة للتغييرات الديناميكية  

- **أمان الجلسة والنقل**: استخدام معرفات جلسة آمنة تشفيرياً وغير حتمية مرتبطة بهويات المستخدم، تنفيذ إدارة دورة حياة الجلسة بشكل صحيح، وعدم استخدام الجلسات لأغراض المصادقة  

- **أفضل ممارسات أمان OAuth**: منع هجمات الممثل المضلل من خلال موافقة صريحة من المستخدم لعملاء مسجلين ديناميكياً، تنفيذ صحيح لـ OAuth 2.1 مع PKCE، والتحقق الصارم من URI إعادة التوجيه  

- **مبادئ أمان الرموز**: تجنب أنماط تمرير الرموز المضادة، التحقق من مطالبات الجمهور للرمز، تنفيذ رموز قصيرة العمر مع تدوير آمن، والحفاظ على حدود ثقة واضحة  

- **أمان سلسلة التوريد الشامل**: معاملة كل مكونات نظام الذكاء الاصطناعي (النماذج، التضمينات، مزودو السياق، واجهات برمجة التطبيقات الخارجية) بنفس الصرامة الأمنية مثل تبعيات البرمجيات التقليدية  

- **تطور مستمر**: مواكبة المواصفات المتطورة بسرعة لـ MCP، المساهمة في معايير مجتمع الأمان، والحفاظ على مواقف أمان متكيفة مع نضج البروتوكول  

- **تكامل أمان Microsoft**: الاستفادة من نظام أمان Microsoft الشامل (دروع Prompt، Azure Content Safety، GitHub Advanced Security، Entra ID) لتعزيز حماية نشر MCP
