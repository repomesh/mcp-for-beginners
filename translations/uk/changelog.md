### Загартування безпеки та усунення вразливостей залежностей

Проведено повний аудит безпеки усіх залежностей та коду прикладів, усунено всі звіти npm advisories та одну знахідку на рівні коду. Після усунення, `npm audit` показує **0 вразливостей** у кожній перевіреній директорії.

#### Вразливості залежностей npm (транзитивні) — виправлено

Проведено аудит усіх 15 закомічених файлів `package-lock.json`. Вразливості були обмежені транзитивними залежностями MCP Inspector dev tool, клієнта OpenAI та MCP SDK; усі наразі усунуті без порушення роботи прикладів:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** та **lab3/code/weather_mcp/inspector**: Оновлено `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), що усунуло advisories у бандлах `ajv`, `brace-expansion`, `diff`, `path-to-regexp` та `ws`. Додано запис npm `overrides`, що примушує патчену версію `shell-quote@1.8.4` для усунення критичного advisory у `concurrently`; повторно згенеровано обидва lock-файли (тепер 0 вразливостей)
- **03-GettingStarted/samples/typescript**: `npm audit fix` оновив транзитивну залежність `qs` (помірна) до патченої версії
- **03-GettingStarted/samples/javascript**: `npm audit fix` оновив транзитивну залежність `hono` (помірна) до патченої версії
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` оновив транзитивну залежність `form-data` (висока) до патченої версії
- **03-GettingStarted/11-simple-auth/solution/typescript**: Згенеровано відсутній `package-lock.json`, щоб проект був відтворюваним та аудиторським (0 вразливостей)

#### Виправлення безпеки на рівні коду (OWASP A03: Ін’єкції)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: Видалено `shell=True` з інструменту `open_in_vscode`. Попереднє `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` дозволяло метасимволам оболонки у шляху до папки інтерпретуватися `cmd.exe` (вектор ін’єкції команд). Тепер запускає безпосередньо розв’язаний `Code.exe` з папкою як аргументом — без оболонки — що функціонально еквівалентно та безпечно

#### Аудит залежностей Python

- Проведено аудит усіх наборів залежностей Python з `pip-audit`. У `05-AdvancedTopics` та `03-GettingStarted/samples/python` виявлено **немає відомих вразливостей** (їхні діапазони `mcp` / `httpx` / `pydantic` / `python-dotenv` резолвляться до поточних патчених версій)
- **09-CaseStudy/docs-mcp/solution/python/requirements.txt**: `pip-audit` виявив транзитивну залежність **`werkzeug` 3.1.1** з трьома advisory DoS через `safe_join` Windows device-name — `CVE-2025-66221`, `CVE-2026-21860` і `CVE-2026-27199` (усі виправлені у 3.1.6). Додано явний пінг безпеки `werkzeug>=3.1.6`, щоб резолвитись на патчену версію; перевірено, що обмеження коректно резолвиться зі стеком `chainlit` / `mcp` / `semantic-kernel`
